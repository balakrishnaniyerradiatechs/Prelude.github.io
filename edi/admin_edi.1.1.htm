EDI Freight Invoice Review Process
An Electronic Data Interchange (EDI) freight invoice can be processed through a series of channels before being approved for payment. This will automate the approval process and eliminate the need to manually enter the freight invoice in Manual Invoice Entry. Your company has the option to run the EDI freight invoice through one or two levels of approval before the manual invoice record is created. If two levels of approval are to be used, the first level freight manager, usually the buyer, will receive the invoice from the EDI conversion for review and approval. The invoice will then be forwarded on to the second level freight manager, typically a branch manager, for review and approval. From there the system will automatically create a manual A/P invoice and a check will be cut to the freight vendor. If a first level freight manager is not assigned, the second level freight manager will receive the invoice directly from the EDI conversion and can release it for payment.
Establishing Identification Codes And Flags
Purchasing Manager Maintenance (/PURCH.MGR.MNT)
Path: Systems >System Parameter > Maintenance > Purchasing Controls > Purchasing Manager Maintenance
•
First level Freight Manager Id: This buyer will be responsible for reviewing freight invoices for the warehouse as they are received through EDI. If this field is blank, the first level freight invoice review process will be by-passed and the EDI invoice will be automatically forwarded to the second level freight manager.
Manager Id Maintenance (/BRANCH.MGR.MNT)
Path: Systems > System Parameter > Maintenance > Order Processing Controls > Manager ID Maintenance
•
2nd Level Freight Mgr Id: This manager that reviews freight invoices for the warehouse after being reviewed by the first level freight manager. If a buyer is not assigned in Purchasing Manager Maintenance, the EDI freight invoice will come directly to this manager Id for review and approval.
NOTE: The manager Id must be set up in Manager Maintenance (System Parameter> Maintenance > Accounts Receivable Controls) before it can be used in this maintenance routine.
User Id Maintenance (/USER.ID.MNT)
Path: Systems > System Parameter > Maintenance > User ID Maintenance
•
Freight Mgr?: This will designate the user as a freight manager and assign the default freight manager Id. This Id is the default for the second level approval process. You must be set up as a freight manager to review freight invoices in the EDI Freight Invoice Second Level Inquiry.
NOTE: The manager id must be set up in Manager Maintenance (System Parameters > Maintenance > Accounts Receivable Controls) before it can be used in this maintenance routine.
EDI G/L Table Maintenance (/EDI.GLTABLE.MNT)
Path: Systems > System Parameter > Maintenance > General Ledger Controls > EDI GL Table Maintenance
There are two General Ledger Table Headings that need to be set for EDI freight invoices. These will identify the accounts used when posting an EDI freight invoice.
•
EDI Inbound Freight: This account is posted to when a manual invoice is created and posted for an EDI freight invoice with a payment method of CC (inbound freight). This account will be the default for any inbound freight invoices created by this process.
•
EDI Outbound Freight: This account is posted to when a manual invoice is created and posted for an EDI freight invoice with a payment method of PP (outbound freight). This account will be the default for any outbound freight invoices created by this process.
EDI Receive File Conversion & Report (/REDIEXPLODE.MAP)
Path: E-Commerce > Electronic Data Interchange > Receive File Conversion > Receive File Conversion
This process must be performed to convert the EDI data that is received. It will determine which freight manager should review the freight invoice. The system will first look at the First Level Freight Manager Id field in Purchasing Manager Maintenance for the invoices warehouse. If this field contains a buyer id, the freight invoice will be placed in that buyers queue for review and approval. If the field does not contain a buyer id, the system will look at the 2nd Level Freight Mgr Id field in Manager Id Maintenance for the invoices warehouse. The freight invoice will be placed in this managers queue for review and approval.
First Level Approval Process (/BQI)
The first level approval process would typically be performed by the buyer. This level is not authorized to alter the freight invoice; only to review, add notes and release it the second level manager.A list of the EDI freight invoices waiting to be approved can be found in the Buyer Queue Inquiry for each buye
Path: Purchasing > Inquiries > Buyer Queue Inquiry
•
Buyer: This should default to the User Id of the buyer logged onto the terminal, but can be changed. This will bring up all open transactions waiting for review for this individual buyer.
•
EDI Freight Invoice Inquiry: Displays the number of freight invoices waiting to be reviewed and released by the buyer. In this example, buyer CDS has 3 EDI freight invoices waiting to be reviewed and released. To view the invoices, put the cursor on this field and press F5- Review.
•
F5-Review : Brings up the EDI Freight Invoice First Level Approval Inquiry screen. In this inquiry, the buyer can review the freight invoice, add notes, and release the freight invoice. This will not allow the freight invoice to be altered.
•
Rel: Determines if the freight invoice will be released to the second level approval manager.
-
Y - To release the freight invoice to the second level manager queue.
-
N - Holds the invoice in this buyers queue.
•
F5 - Review: The detail information for the invoices can be reviewed, but no changes can be made.
•
F8 - 2nd Mgr Notes: Notes to the second level freight manager can be entered for the designated invoice.
•
F2 - Save: This will release the invoices with Rel set to Y. These invoices are removed from this queue.
Second Level Approval Process (/EFSI)
A manager performs the second level approval and will have the authority to alter the invoice amount and release it for payment to the vendor. If a user Id is set up as a freight manager and assigned to a warehouse, Frt will be displayed in the message queue telling the freight manager there are EDI freight invoices to be reviewed and approved.
Version 18 Note: This process now checks to see if the vendors freight invoice already exists in the OAP.HIST file. If so, the system will give a warning message to notify the user that the invoice already exists. The manual invoice will not be created and the freight invoice will be removed from the inquiry screen.
This inquiry is used to review the EDI freight invoices waiting to be approved and released by a manager. This screen is very similar to the first level approval screen except that it is warehouse .
Path: Financial > Accounts Payable >Inquiries > EDI Freight Invoice > Second Level Inquiry
•
App: This determines if the freight invoice will be released for payment to the freight vendor.
-
Y - To release the freight invoice for payment.
-
N - Holds the invoice in this managers queue.
•
F5-Review: The detail information for the designated invoice can be reviewed. The freight amount can be changed if necessary. Any changes to the amount will be reflected in the Total Charge field.
•
F6-G/L Acct: Displays the G/L account definition screen for the designated invoice. Changes to the accounts can be made if necessary.
-
If the payment method in the Freight Invoice record is CC (inbound), the freight account will default to the EDI Inbound Freight account set in the general ledger table.
-
If the payment method for the freight invoice is PP (outbound) then the freight account will default to the EDI Outbound Freight account set in the general ledger table.
-
If the manager approves a freight invoice during the second level approval process and there are no G/L accounts defined in the EDI G/L Table, the system will automatically send the user into this screen to define the accounts.
•
F7-AP Notes: Comments can be viewed or modified for this invoice These comments will be converted into the Remittance Comments on the actual manual invoice when it is created. The first line of the Remittance Comments will print on the check to the freight vendor.
•
F8-Mgr Notes: Any notes there were entered by the buyer from the first level approval process can be viewed and modified.
•
* F-10 Action - Auto Accept All: Allows for all freight invoices to be selected at once.
•
F2-Finish: Will automatically create manual invoices for each freight invoice whose App field is set to ‘Y. All freight invoices for which a manual invoice record was created will be removed from the managers queue. Since the manual invoice is created behind the scenes, the following defaults are used to create the invoice.
•
Batch#: Defaults to the next system generated batch number.
•
Batch Date: Defaults to the current system date.
•
Batch Amount: Defaults to the total amount of all freight invoices released in a session that have the same A/P G/L account number.
•
Vendor Number: Defaults from EDI freight invoice
•
Vendor Invoice Number: Defaults from EDI freight invoice
•
Invoice Date: Defaults from EDI freight invoice.
•
Terms Code: Defaults from the vendor record.
•
Discount Date & Due Date: Calculated the same as Manual Invoice Entry
•
Status: Defaults to Open (O).
•
Invoice Amount: Defaults to total amount of the freight invoice.
Once the checks were printed for payment to the freight vendors, the manual invoice created can be viewed in Vendor Invoice Inquiry.

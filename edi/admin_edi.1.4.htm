EDI Transmissions
This document covers the sending and receiving of EDI data through the Prelude System. The different transmission protocols are not discussed, due to the differing nature of the protocols and VAN (Value Added Network) requirements. The assumption is made that the transmission scripts are already in place, and will be generically referred to as running the EDI session.To use these features, go to E-Commerce > Electronic Data Interchange or type /M:EDI:
•
The File Transfers submenu contains all the menu commands to run an EDI session. The different transmission types that are supported are contained there.
•
The Receive File Conversion submenu is the heart of the EDI system. From this menu, data is converted into the appropriate database format, parsed into the different document types, and moved into (or out of) Prelude.
•
The Transaction Processes submenu is not used in the new EDI Mapping Scheme and will not serve any function.
•
The Reports submenu contains all reports to display EDI control information such as trading partners, host identifiers, etc.
•
The Maintenance submenu contains all commands to update EDI control information such as trading partners, host identifiers, etc.
Create GS/ISA Segments
This is the first process to run when performing an EDI session. This process will bundle all outgoing documents that are ready to send. It will create the appropriate ISA/GS headers for the data, group the documents together by trading partner, and also ready any 997 Functional Acknowledgements. At this point, the data has not been sent. It has simply been bundled and prepared to send together by trading partner, and also ready any 997 Functional Acknowledgements.
Path: E-Commerce Electronic Data Interchange File Transfers Create GS/ISA Segments
At this point, the data has not been sent. It has simply been bundled and prepared to send.
Retransmit Single Document
This process allows a single document number to be retransmitted.
Path: E-Commerce Electronic Data Interchange File Transfers Retransmission Menu Retransmit Single Document
In the above example, an Invoice 100452 is being selected for retransmission. This does not retransmit the document, but the 810 will be selected and bundled when the Create GS/ISA Segments process is run. The documents that may be retransmitted at this point are 850.s, 810.s, and 856.s.
The transmission of an entire batch uses the General Report Interface that is common to all Prelude reports. Simply enter a date and/or time range, then select the batch to send. After selection, proceed directly to the appropriate EDI session to send, there is no need to run the Create GS/ISA Segments process because the data already has this information from the previous send. The purge utility also uses the report interface. After selection, it will purge the selected batches. This should be done occasionally to free disk space.

Data Check & Character Conversion
This process converts the data into an attribute-delimited file, replacing the segment terminators with attribute marks and the element terminators with value marks.
Path: E-Commerce Electronic Data Interchange Receive File Conversion Data Check & Character Conversion
At the same time, the ISA and GS headers are analyzed to see which trading partner the data is from, what ISA/GS versions have been received, and what determines the document type. This is the first process is that is run AFTER an EDI session has been performed. The only input required for this process is a mailbox number. Depending upon the communications setup, this may be a single mailbox number or multiple mailboxes (corresponding to one per VAN). If you are unsure of the appropriate mailbox number, please contact Activant. As this process runs, it will display to the screen the ISA identifiers that data has been received from, as well as a total byte count.
Receive File Conversion
Once the data has been formatted correctly, this process will take this data and create documents in Prelude (i.e. 850.s will become Orders, 810.s will become Vendor Invoices, etc).
Path: E-Commerce Electronic Data Interchange Receive File Conversion Receive File Conversion
This process only prompts for a mailbox number. Once entered, the data will be parsed and an error report will automatically be generated (prompting for print to screen or printer). If this report is blank, no errors were found. If errors did exist, please print this report and fax it to Activant. Documents that contained errors will remain in the EDI system until resolved, while the error-free documents will continue into Prelude.
Once the error report prints, if any documents were received to convert, this conversion process will begin automatically. Occasionally the use may see messages flash along the bottom of the screen. This is perfectly normal output as the conversion process attempts to match incoming data (product numbers, ship-to numbers, etc.) with data in Prelude. Once the process finishes, the user is returned back to the menu. At this point, all data should be converted and EDI transactions now exist in Prelude as normal Prelude documents.
To find any documents that were received through EDI, run the appropriate open document report (i.e. Open Sales Order Detail Report) and select an order source of "E" for EDI.
Summary of the EDI Process
•

•

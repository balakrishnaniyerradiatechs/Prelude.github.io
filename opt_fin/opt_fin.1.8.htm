Setting Up Accounts
Two accounts are used to execute the Vertex logic: VERTEX-EXEC and VERTEX-DATA. The EXEC account contains all shared files used by both the live and test systems. However, a DATA account must exist for both live and test (e.g., XXX-VERTEX-DATA, TEST-VERTEX-DATA). The DATA accounts contain the following 7 files:
•
VTXREG - Register file to accumulate all tax information for invoices generated. Used for reporting and returns modules products.
•
VTXREG.HIST - Archive file for the register file when the purge routine is executed.
•
GEOCONV - Assigning GEOCODES for the CUSTOMER and SHIP-TO files.
•
NUM2000 - Prepares data to be downloaded to the Quantum Returns Module PC product.
•
TAX2000 - Prepares data to be downloaded to the Quantum Returns Module PC product.
•
VTSXTRCT - Prepares data to be downloaded to the Quantum Returns Module PC product.
•
TAXRETN1 - Preparates data to be downloaded to the other returns module PC products.
NOTE: These files must have the q-pointers setup on the live and test accounts pointing to the proper DATA accounts. You must also clear each file when the module is first installed.
Two programs are used to integrate Vertex with the Prelude Application.
•
VERTEX.MNT.CHK - Evaluates the city, state, and zip code to determine the proper geographic code. Used in all interactive screens requiring this interpretation.
•
VERTEX.TAX.CALC - Evaluates the tax authority and calculates taxes. Also performs updates to the register file for tax return reporting. The primary function of this program is to load an array of VTAX that is then passed to a tax calculation program with the results returned to the array. The link area is attached that defines what each element of the array represents.

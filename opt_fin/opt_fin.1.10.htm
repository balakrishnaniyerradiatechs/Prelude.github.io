Establishing Maintenance Flags And Codes
Company Maintenance (/COMPANY.MNT > F6)
Menu Path: Systems > System Parameter > Maintenance > Company Maintenance > F6-Modules
•
Vertex Sales Tax: Determines if the application uses traditional sales tax calculation methods using the Tax Jurisdiction Codes or if the application interfaces to the Vertex/MV™.
-
M -To only activate the maintenance integration so that a customer can be working on defining the GEOCODES properly while still uing the traditional sales tax calculation.
-
Y - Activates both the maintenance and the tax calculation interfaces and asks:
•
Use Taxable Status From Ship-to?: Indicates if the taxable flags at the ship-to or only the customer are ysed. If this flag is set to Y, the ship-to taxable status is used if it is maintained. If the taxable status is empty, the customer taxable status is used.
Warehouse Maintenance (/WM)
Path: Systems > System Parameter > Maintenance > Inventory Controls > Warehouse Maintenance
•
Country: Defines the country where this warehouse is located.
•
City: Defines the city for this warehouse. This field is validated when using GEOCODES.
•
Geographic Code: Defaults when you enter the city, state, and zip code. If the city, state, and zip cross multiple county lines, you must choose a GEOCODE. A GEOCODE is required.
Customer Maintenance (/CM)
Path: Order Processing > Maintenance > Customer Maintenance
•
Country: Defines the country where this customer is located. This should be setup in the default record and overridden when appropriate.
•
City: Defines the city where this customer is located, which is validated against GEOCODES.
•
GeoCode: Defaults when you enter the city, state, and zip code. If the city, state, and zip cross multiple county lines, you must choose the proper GEOCODE. A GEOCODE is required. A GEOCODE of ‘777777777’ defaults for addresses outside of the United States.
The Taxable flag (F5-Add Info -> F6-Sales Tax screen) determines if the customer is taxable at this location.
•
Taxable: Enter Y if tax is to be calculated for this customer.
•
Exempt Number: Enter the tax exemption certificate number for this customer.
•
Expiration Date: Enter the expiration date for the tax exemption certificate.
•
Sales Tax Group: Enter the sales tax group to assign to the customer. This determines if special sales tax rules apply for this customer for selected products or tax groups.
•
Exempt Reason Code: Enter the exempt reason code if applicable. This is passed to Vertex.
Customer Ship-To Maintenance (/CSTM)
Menu Path: Order Processing > Maintenance > Customer Ship-To Maintenance
•
Country: Defines the country where this customer ship-to address is located. This should be setup in the default record and overridden when appropriate.
•
Geographic Code: Defaults when you enter the city, state, and zip code. If the city, state, and zip cross multiple county lines, you must choose the proper GEOCODE. A GEOCODE is required. A GEOCODE of 777777777 defaults for addresses outside of the United States.
•
City: Defines the city for this customer ship to which is verfied for this GEOCODE.
Taxable (F6-Sales Tax) determines if the customer ship-to is taxable at this location.
•
Taxable: Enter Y if tax is to be calculated for this customer.
•
Exempt Number: Enter the tax exemption certificate number for this customer.
•
Expiration Date: Enter the expiration date for the tax exemption certificate.
•
Exempt Reason Code: Enter the exempt reason code if applicable to pass to Vertex.
Prospect Maintenance (/TME)
Menu Path: Marketing > Prospect Maintenance
•
Country: This field defines the country where this prospect address is located.
•
GeoCode: This field is generated automatically when you enter the city, state, and zip code for each prospect created. If the city, state, and zip cross multiple county lines, the system provides a lookup to choose the proper GEOCODE. A GEOCODE MUST be defined for every prospect you setup. A GEOCODE of 777777777 defaults for addresses outside of the US.

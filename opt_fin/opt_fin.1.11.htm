Vertex Menu
Note: Unless otherwise noted, all features are available on the Financial > Vertex menu.
Vertex Test Inquiry (/VTX)
Displays the tax authority and sales tax amounts calculated for a product shipped to a customer from a given location. This routine does not write any records to your database, it is strictly available to see how the system will tax specific customer/ship-from scenarios.
Load Vertex Tax Rates Submenu
This menu contains the utilities needed to load the new Tax Rate tape received each month from Vertex. This tape MUST be loaded before the last day of the month it is received.
Menu Path: Financial > Vertex > Load Vertex Tax Rates
Backup Previous Rates File (/B:VERTEX.VRATES.BKUP)
Clears the Rates backup file and copies the current Rates file to the backup for security.
Load Tax Rates From Tape (/VERTEX.LOAD.RATES)
This routine updates the Rates file from the tape supplied from Vertex. To load the Rates tape.
1
Load the rates tape received from Vertex in the tape drive.
2
Drop to TCL (/TCL).
-
If Unidata, type T-ATT. (Escape back to menu).
-
If AP, perform a SET-DEVICE to the proper tape drive.
3
Execute the load program.
4
Enter password (SYSADM).
5
"Has Tape Been Loaded In Tape Drive" – Answer Y.
6
"Enter Unix Tape Device Path -> /dev/rmt0"
7
Press return if this is the tape device path or override to your tape device path if you use AP.
8
Once the procedure is completed, proceed back to TCL (/TCL)
9
Type "T-DET". This will detach you from the tape drive.
10
Escape back to the menu
NOTE: The program should display incremental counters on the screen and proceed up to a number over 50,000. This takes approximately 15-30 minutes. Run this routine in the evening but can be run during the day if necessary.
Sales Order Rate Validation (/B:VERTEX.TAX.VALIDATE)
This routine performs a test on any current orders awaiting invoicing where the tape load would possibly calculate a different tax amount. Report any problems to Prelude immediately.
Quizip Rebuild (/QZIPBUILD)
Rebuilds the jurisdiction surnames used to assign GEOCODES in the maintenance programs. This routine needs to be run in the evening when employees are not on the system. This could be run once a quarter if necessary. It takes approximately 15 minutes to run. Answer the questions as follows:
•
If creating ZIPPER.WGEO enter ’Y’, or else enter ’N’. Answer Y.
•
Include all Place Names ("ALL") or just legal names? Answer ALL.
Sales Tax Register Report (/VTAX120)
Detail/summary report supplied by ROI Syngineering to provide the tax information used to fill out tax returns. This report requires the password (SYSADM) and then prompts for an invoice date range for the report.
Other Returns Module File Generation (/VTAX250)
This program prepares data for downloading to return modules other than Quantum. A report is supplied by ROI Syngineering which is considered to be very useful when filling out tax returns.
NOTE: These reports should be printed to analyze what is available and to determine which reports should be printed for monthly procedures. If using the returns module capabilities, report printing is optional.
Quantum Returns Submenu
This menu is used to prepare and download data to the Quantum Returns PC product.
Menu Path: Financial > Vertex > Quantum Returns
•
Quantum Returns File Generation (/VTS600) - Generates a summary record for each jurisdiction using the register records.
•
Quantum Returns File Merge (/B:TAX.FILER,B) - Combines all summary records into one continuous record to be downloaded into the quantum software.
•
Quantum Returns File Download (/B:TAX.FILER,D) - Performs the download to your PC (using SBClient) for loading into the Quantum Returns Software.
Vertex Exception Matrix Maintenance (/VEMM)
Defines and maintains the tax exceptions for specific customer/product/ship-to combinations. This could be used for a taxable status exception or a rate override.
Matrix Type: Enter the matrix type to calculate state, city, and county sales tax:
•
Type 1 - Customer Number/Ship-To Number: State, city, or county tax rates can be overridden for a customer/ship-to (e.g., Customer 100 is charged 4% state tax rate when sold to ship-to 1 in the gulf of Mexico.
•
Type 2 - Customer Number/Product Number: The taxable status of a product can be overridden when sold to a customer. State Tax Maximum is per unit sold.
•
Type 3 - Customer Number/Product Line: The taxable status can be overridden for any product defined for the product line when sold. State Tax Maximum is per unit sold.
•
Type 4 - State Nexus Overrides: Include counties or cities to be taxable and exempt all other all other counties and cities for the state defined. Ex. We only charge tax when material is sold in Denver and Colorado Springs in the state of Colorado. All other cities or counties are nontaxable.
•
Type 5 - Other Charges Exemption: Exempt all other charges for a state and/or ship-via. Ex. The state of California does not charge tax for restocking charges. In V17+, the Vertex State Code was increased to 5 digits to allow the matrix to be setup for province codes. The conditional flags determine if the associated exempt flag is ONLY checked if the sales order has no taxable merchandise calculated.
•
Type 6 - Customer Number/State: The state, city, or county tax rates for a customer/state combination can be overridden. Ex. Customer 100 is charged 4% for all ship-to’s in Louisiana.
•
Type 7 - Customer Tax Group/Product Tax Group: The state, city, or county tax rates can be overridden for a customer tax group/product tax group combination. State Tax Maximum is the state tax maximum per unit sold.
•
Type 8 - Product Number/State: State, city, or county tax rates can be overridden for a product number/state. State Tax Maximum is the state tax maximum per unit sold.
Once the matrix type has been selected, the sales tax rates can be established for a customer.
Vertex Exception Matrix Listings (/VEMM.LST)
Provides a listing for each of the Vertex Exception Matrices entered in the previous program.
Rebuild Vertex Exceptions Used (/B:BUILD.VERTEX.EXC)
Rebuilds the vertex exceptions entered in the Vertex Exceptions Matrix Maintenance screens.
Vertex Register Record Purge (/VERTEX.PURGE)
Copies historical records to an archive file. Use for a date range that has been completely filed and no longer needed. Keep 3-6 months of data in the active register file.
Place Of Business Maintenance (/VTX.POB)
Defines the states where you are responsible for filing a return.
•
Vertex Tax Codes: Enter the 2 digit code for the state. F3 lists the codes for all states.
•
Source: Determines the source of the GEOCODE in a sales order transaction.
-
E - To use the entry location.
-
S - To use selling the selling location.
-
P - To use the actual shipping warehouse.
•
Whse:
-
Y - Defines a place of business at the warehouse if you have multiple companies under 1 company.
-
N - Defines a place of business at the company level.
Place Of Business Warehouse Maintenance (/VTX.POB.WHSE)
Defines each state and its Vertex tax code in which you have a warehouse.
Sales Tax G/L Table (/VTX.POB.GL)
Defines the general ledger accounts for each state.
Convert Data Files (/ASSIGN.GEOCODES)
This conversion utility attempts to define a GEOCODE for the customer, ship-to, and warehouse address file based on the city, state, and zip code. This program should only be run once, followed by the Conversion report (following section) to correct the records that were unable to define a GEOCODE. This utility currently runs against the CUSTOMER, SHIP-TO, and WHSE.ADDR files only. It can be run on a validation run only so that the report is generated but the records are not updated with the generated GEOCODE. It is easier to correct the warehouse GEOCODES by hand through Warehouse Maintenance (/WM).
Conversion Report
Displays the customer or ship-to records that could not define a GEOCODE. Note: (If you need to rerun the conversion utility more than once, you must clear the conversion file prior to executing the Convert Data Files again. To perform the clearing, proceed to TCL (/TCL) and type CLEAR-FILE GEOCONV). When that returns, escape to return to the menu. The Warehouse address errors do not show in this report but must be reviewed through the maintenance program.
NOTE: If your Vertex Sales Tax flag in Company Maintenance is set to M, the Customer and Ship-to files can be defined before going live on Vertex. Once converting to Vertex (recommended at the beginning of a month) the open sales orders and quotes must be loaded with the proper GEOCODES.
Vertex Option Parameters (/VERTEX.OPT.MNT)
•
Utilize Florida Surtax Logic: This flag only applies to the state of Florida. In Florida, a surtax is charged up to a certain amount. Once you reach the amount, the surtax no longer applies. This option applies to the Vertex programming that wraps up this surtax logic. Vertex is checking to see that all previous invoicing is added up for an order to determine if the maximum merchandise amount for the surtax no longer applies. The surtax isn't just based upon what is currently shipping out and being charged on an invoice, but is also based upon what has previously shipped out on the order.
•
Send Division: This flag has been added for Versions 18 & higher to determine if the division number is sent as part of the key to the VTXREG file.
-
Y - The Division number is sent to the Vertex engine.
-
N - The Company number is sent to the Vertex engine.

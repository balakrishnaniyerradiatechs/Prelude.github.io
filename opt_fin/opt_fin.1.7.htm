Vertex Module (V17)
Vertex is a software product used by Prelude to calculate, bill and file tax returns in compliance with sales tax laws. Vertex is an integration of the Vertex Sales Tax product for COBOL and RPG that was converted to run in our multi-dimensional operating environment. The porting of the Vertex Sales Tax product was performed by ROI Syngineering in a collaborative effort between ROI and Vertex. ROIâ€™s version uses the same rate files and functional logic that the conventional Sales Tax product performs.
Vertex is licensed by state and provides a Data Module that contains complete state and local rate information updated monthly. The Calculation Module is integrated into Prelude to calculate sales tax and update register files for returns filing. The Returns Module captures the data from billing and generates a PC file to produce state and local tax returns automatically. (Can be purchased separately).
Vertex functions around geographic codes referred to as GEOCODES. These are assigned to customer/ship-tos and warehouses based on the city, state and zip code. Sales tax is calculated based on the source and destination geographic codes for the transaction.
A GEOCODE is a 9-digit number that represents the state, county and city. The first 2 characters define the state, the next 3 characters the county, and the last 4 characters the city. (Example: the code 440850760 breaks down as - 44 Texas, 44085 Collin County, 440850760 Dallas City) All these records are maintained in the VRATES file and are updated monthly via an update tape from Vertex. These records are primarily rates, dates and other information used to calculate tax amounts. They also contain transit taxes and other tax information.
The Vertex Menu includes programs used for the initial setup, along with programs for the monthly routines required to load the rate tapes and print the reports for tax returns. to use Vertex with Prelude, several Prelude Maintenance flags and codes must first be established.

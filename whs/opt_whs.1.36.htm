Work Orders
The work order process creates an order for components to be assembled for a bill of material item. Radio frequency work order picking is used to pick products from pick or bulk bins in a logical picking sequence. The work orders are released and assigned to a picker. The products are moved to a Work in Process bin as they are picked. At completion the Work in Process bin is relieved and the finished good is added to inventory. Outplant Work Orders are used when products are taken out of stock and sent to a vendor for additional processing such as painting, plating, packaging, machining, etc. The finished product is then returned to the distributors warehouse. Refer to Work Order Picking Options for more information on warehouse options.
Maintenance
Bin Location Maintenance
Path: Inventory > Maintenance > Bin Files > Bin Location (BLM)
In Bin Location Maintenance, a Staging Bin and a Work in Process bin are created for each vendor. Another bin can be created if needed for Work in process line stock if components are stored at the vendor site before the work order is entered. For Staging after picking and prior to sending to the vendor, set the following fields:
•
Bin type: S - For staging.
•
Storage type: O - For outplant.
For the Work in Process bin where the product is kept while it is being completed, set the following fields:
•
Bin type: W - For work in process.
•
Storage type: O - For outplant.
For the optional bin to use if components are stored at the vendor site before the work order is entered, set the following fields:
•
Bin type: L - For work in process line stock.
•
Storage type: O - For outplant.
Products are moved to this bin when they are shipped to the vendor.
Product Maintenance
Path: Inventory > Maintenance > Product Maintenance >Product Warehouse Maintenance > F6 Purch Parameters (/PWM > F6)
•
Replenishment Method: The finished product should have a replenishment method of O for outplant or M for manufacture.
Bill of Material Maintenance
Path: BMW/Work Order > Maintenance > Bill of Material Maintenance (/BOMM)
A Bill of Material is normally created with the finished product as the parent. The components and burden item(s) represent the amount the vendor will charge to perform the processing. If these are not standard you can change them in Work Order entry instead of Bill of Material Maintenance. See the Bill of Material Maintenance document on the Bill of Material / Work Order menu for more information.
Burden Maintenance
Path: BMW/Work Order > Maintenance > Burden Maintenance (/BUM)
Burden codes need to be created to reflect a charge to be added to a bill of material or work order. These account for non material costs such as labor, overhead and other expenses directly associated with the assembly of a product.
•
Burden type: the type of charge (outplant, labor)
•
Setup Charge:
-
N - If the charge is per piece.
-
Y - If it is a flat charge for everything sent out.
Entering Work Orders
Path: Warehousing > Work Order Processing >Work Order Entry (/WOE)
This process creates a work order for components to be assembled for a bill of material item. When you release the work order, you must choose a Work In Process bin.
For outplant work order, set Order Type in the header screen to 03 for Outplant work order. When you save the header screen, you must indicate which vendor the BOM will be shipped to. It is assigned to an outplant staging bin.
Picking Work Orders
Work Order Bin Allocation
Path: Warehousing > Work Order Processing >Work Bin Allocation (/WOBA)
When work orders are ready to be picked, the Work Order Bin Allocation process must be run. This determines which bin the components are picked from using the following rules:
1
Work orders are allocated from pick, bulk, and stock type bins.
2
A pick bin percentage is determined. This is done by taking the pick quantity divided by the primary bins maximum quantity. If this pick bin percentage is less than the warehouse percentage, then the primay bin allocation is attempted first.
3
A list of available bins is determined. The main sort is linestock, bulk then pick. If the pick bin percentage is valid, the bin sort is linestock, pick or bulk.
Work Order Picker Assignment / Unassignment
Menu Path: Warehousing > Work Order Processing > Work Order Picker Assignment (/WOPA)
Warehousing > Work Order Processing >Work Order Picker Unassignment (/WOPU)
The work order is assigned to a picker using Work Order Picker Assignment. If the work order needs to be re-assigned it can be unassigned using Work Order Picker Unassignment then re-assigned at that time. One or multiple work orders can be assigned to a picker. The work in process bin must be entered at this time.
Radio Frequency Work Order Picking
Menu Path: Warehousing > Handheld RF Menu > Doc. Picking > Work Order Picking (/RWPE)
Work Order picking allows RF picking by work order number. You are prompted for work order number and number of lines. F3 displays all of your assigned work orders. All line items on the selected work order are displayed showing the product number, description, bin to pick from, lot (if applicable) and quantity to pick. You are guided through the warehouse in the logical picking sequence for each product. As each product is picked it is moved to a cart. After all products have been picked they are moved to a work in process bin which defaults to the bin entered when the work order was released or assigned.
If a partial quantity of a line item is picked, the balance of the pick quantity for that product comes up right after the first pick. The balance of the product is prompted until the entire quantity has been picked or a 0 pick quantity is entered.
Non-RF Work Order Picking
Menu Path: Warehousing > Work Order Processing > Work Order Picking Report (/WO.PICK.RPT)
Menu Path: Warehousing > Work Order Processing >Work Order Picking Confirmation (/WOPC)
As a back up for the radio frequency terminal entry, a report can be printed to pick work orders. The prompts are the same and the report shows the product, quantity, bin to pick from, lot (if applicable) and the work in process bin. Work Order Picking Confirmation records the work order pick if the Work Order Picking Report was used to pick the work order. The products, quantities and bins that were on the report are displayed for confirmation.
Shipping and Receiving Outplant Work Orders
Shipping Outplantt BOMs to a Vendor
Path: Warehousing > Work Order Processing > Outplant Processing > Outplant Shipping Document (/OUTPLANT.SHIP.RPT)
Path: Warehousing > Work Order Processing > Outplant Processing > Outplant Bin Movement (/OUTPLANT.MOVE)
The outplant shipping documents are printed by specifying the staging bin and work order number.You can print three reports:
•
The Outplant Summary Shipment Report prints all components and quantities that are being shipped. This is used to assemble and ship the products.
•
The Outplant Work Order Report shows by work order, components, quantities, assembly instructions and burdens to describe for the vendor what needs to be done.
•
The Outplant Production Group Report sorts all work orders by the product group in the parent product record.
Then use Outplant Bin Movement to identify products sent to the vendor by specifying the staging bin and work order number. This is where the outplant work in process bin is identified, and this is the process that moves the components to that bin.
Receiving Outplant BOMs from Vendors
Path: Warehousing > Work Order Processing > Outplant Processing > Outplant Receiving Document (/OUTPLANT.REC.RPT)
Path: Warehousing > Work Order Processing > Outplant Processing > Outplant Receiving (/OUTPLANT.REC)
When the completed products are returned, a Receiving Documents Report may be printed. This report is selected by entering the Outplant Work in Process bin. The Outplant Receiving process is executed to update inventory with the finished product. The work order number is required. This process completes the work order, puts the finished product in inventory and deducts the components from inventory.
Completeing Work Orders
Menu Path: Warehousing > Work Order Processing > Work Order Completion (/WOC)
Menu Path: Warehousing > Handheld RF Menu > Processes >WO Completions (/RWOC)
Work Order Completion is performed after the product is assembled or received from outplant. This process deducts all components on the work order from the on hand balance in the Work In Process bin. Note that these are the products and quantities on the work order, not necessarily the components that were picked, although they should be the same. This process also puts the completed product into the bin specified at completion. A warning message displays if all lines were not picked upon completion. For outplant work orders, when you complete the work order and receive the finished product, a receiver with the total of the Outplant burden items on the work order is created.

Bin Location Maintenance (/BLM)
Path: Inventory > Maintenance > Bin Files > Bin Location
A bin is created for each physical bin location in the warehouse, including staging, receiving, putaway/pick carts, etc. Many of the fields used to define a bins are used in the warehousing logic. The 3 system bins are:
•
PICK Bin is used to signify that a product was picked and awaiting packing.
•
INV Bin signifies that a product was packed and is waiting invoicing.
•
CYCLE is used in the cycle count subsystem.
•
Warehouse Number: Enter the warehouse number for the bin location.
•
Bin Number: Enter the 1 – 14 alphanumeric bin numeric number.
•
Description: Enter a description of the bin location.
•
Zone: assign each bin to an RF picking by zone if using RF.
•
Cycle Count Zone: Each bin location can be assigned a cycle count zone that can differ from the picking zone. This categorizes bin locations in the cycle counting process in a different manner than for transaction picking. It determines how many times a year products in this zone should be cycle counted.
•
Permanent Code: Determines if a bin should be deleted when the bin is empty.
-
P - Permanent bin.
-
T - Temporary bin.
-
R - Retained temporary bin.
•
Bin Type: Categorizes bin locations. Some of the options are pick, bulk, staging, and dock. The bin types are used in the bin allocation rules to prioritize eligible bins.
•
Pick Reserve: Sets bin level allocation rules. For example, a bin can be set to not available for picking which excludes it from the bins eligible for picking. Options are:
-
P - Pickable
-
R - Reserve
-
N - Not available for picking.
•
Environment Code: Identifies the type of environment where the bin exists (e.g., fire rated, refrigerated, and secured). This is for information only.
•
Storage Type: Describes the physical type of bin. Some options are pallet racks, flow racks and carousel. These types are for information only. Some types have purposes such as truck and outplant.
•
Vendor Number: The vendor that has the product (outplant only).
•
Sort Sequence Code: Controls the logical progression of the picker through the warehouse.
•
Putaway Sort Sequence: Enter the sort sequence code for sorting bins for putaway. If no sort sequence is defined, the system defaults back to the picking sort sequence.
•
Nettable: Controls whether products in this bin are available for sale (e.g., set to for inspection bins).
•
Putaway Status:
-
A - To make this bin active
-
I - To make this bin inactive for putaway status.
•
Auto Replenish: Set to Y to be included in the automatic bin replenishment nightly process. The available is compared to the bin minimum. If the available quantity is below the minimum, a replenishment request is generated up to the bin maximum.
•
Date Established: the date this bin was created (defaults to the system date).
•
Aisle: Enter the aisle number for the bin location.
•
Width, Height, Depth, Volume and Weight: Size information for stock receipts putaway by volume.

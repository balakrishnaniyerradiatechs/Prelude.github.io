Warehousing Options (/WM > F7 > F5)
Path: Systems >System Parameters >Maintenance > Inventory Controls > Warehouse Maintenance > F7- RTS Maintenance > F5 Warehousing
This screen is used to set the warehousing options flag, the sales order picking options, the bin replenishment options, the sales order segment picking options, the stock transfer and the order picking options.
•
Warehousing Flag: This Y/N option controls whether this warehouse is using the Advanced Warehousing Module. This allows a company to use warehousing at some locations and not others.
•
Validate Radio Frequency Entry:
-
Y - Requires that the radio frequency units scan a barcode or re-enter the data value.
-
N - The radio frequency unit can scan the barcode or accept the value with the Enter key.
-
There is also a selective option. This allows you to require scanning of only the product number or the bin location instead of scanning all fields.
•
Pick/Bulk Percent: Determines whether stock transfers and work orders should allocate from a bulk type bin location or the primary bin location. The system uses the pick quantity and determines the percentage required of the primary bins capacity. If the percentage is less, the primary bin is recommended for allocation.
Example: Bin Location Maximum = 100, Quantity Ordered = 30. If the pick/bulk percent is twenty percent, the system will allocate from the bulk bin location. If the quantity ordered is 10, the system allocates from the primary bin location.
Sales Order Picking Options
•
Sales Order Radio Frequency Picking: This option controls if the system should update the radio frequency picking file for sales orders:
-
Y - To pick sales orders with radio frequency units.
-
N - To implement the warehousing philosophy while still processing sales orders via a pick ticket (available for companies who choose to implement radio frequency picking in phases.)
-
S - Indicates that sales order picking is determined by Truck Route Segment.
•
Sales Order Picker Assignment Required: This is accessible if the previous flag is set to Y.
-
Y - Requires that the sales order picker only pick sales order releases that were assigned to the user via Sales Order Picker Assignment.
-
N - The sales order picker can select the next sales order release available.
•
Sales Order Picking Paperless:
-
Y - Suppresses the printing of the pick ticket.
-
N - Prints the pick ticket in addition to updating the radio frequency sales order picking queue.
•
Sales Order Allocation Pickup Ship Via: Allows bin level commitments for sales orders with ship via set to P for pick-up. This is required for Sales Order Radio Frequency Picking.
•
Update Pick Label File: Updates the PICK.LABEL file when orders are processed, which can be used for picking confirmation by line instead of a complete release.
•
Prompt for Route/Ship Via:
-
Y - To prompt for a route or ship via for sales orders.
-
N - To not prompt for a route or ship via for sales orders.
•
Prompt for Dest Bin in RF Pick:
-
Y - To prompt for a destination bin in RF Order Picking for all releases picked during the session.
-
N - To not prompt for a destination bin in RF Order Picking. No destination bin is used.
-
S - To prompt for a destination bin only once in RF Order Picking. This will be used as the destination bin for all releases picked during that session.
•
Prompt for Dest. Bin in PKCE:
-
Y - To prompt for a destination bin in PKCE.
-
N - Defaults the destination bin to pick.
•
Skip from bin: To prompt for a from bin information in the RF order picking.
•
Auto RCE After RF Picking: V18+. Automatically run release confirmation after RF Order Picking.
Bin Replenishment Options
•
Single Bin Replenish: This Y/N option controls whether bin replenishments will attempt to allocate from a single bin location or multiple bin locations for normal replenishment transactions. Emergency Replenishments will allocate from multiple bins if needed.
•
Putaway to Pick Bin: Allows radio frequency bin replenishments to be putaway immediately to the primary bin location, skipping the picking cart. This is used when products are replenished with a full pallet, where the pallet is immediately placed in the primary bin location.
•
Use Replenishment Priorities:
-
N - To sort bins by number of days. This assigns a date sequence based on the current date plus bin onhand days.
-
Y - To sort bins by priority code. Use the online help (F1) for a more information.
•
Allocation Pick Reserve Options: Choose or eliminate replenishment bins:
-
P - For pick bins
-
R - For reserve bins
-
Leave blank for all Pick/Reserve Bins.
•
Replenish Evaluation Method: Determines replenishment requirements by comparing the bin min/max to either the bin onhand or the bin available.
-
O - To compare to the current Bin Onhand
-
A - To compare to the current Bin Available.
•
Replenish Rounding Method: Determines whether to round down replenishment quantities to cases or pallets. If no case or pallet quantity is defined for a product, the default will be one.
-
C - To round down to case quantity
-
P - To round down to pallet quantity.
•
Replenish Bin Sorting Method: Controls how the bins are sorted for evaluation when building the list of bins for sourcing.
-
O - For ascending bin onhand. This helps clean up the supplying bins with smaller bin onhand values, thus releasing bin space for other products.
-
D - For ascending bin transaction date. This performs a FIFO rotation of inventory. Lot control should be used if there is a concern with product age or expiration date.
•
Bin Minimum Percent: The percent used in the Auto Bin Replenishment Min/Max Recalculation. A bin minimum is calculated by taking the average monthly usage in the product warehouse record, multiplied by the Bin Replenishment Minimum %.
•
Bin Maximum Percent: This percent is used in the Auto Bin Replenishment Min/Max Recalculation. A bin maximum is calculated by taking the average monthly usage in the product warehouse record, multiplied by the Bin Replenishment Maximum %.
•
Replenish Bin Sequence and types: Allows the warehouse to prioritize which bin types to use to replenish primary/retail bins (not valid for lot0controlled items):
-
B - Bulk Bin
-
P - Pick Bin
-
L - Line Stock
-
R - Retail Bin
-
K - Bulk Unpackaged
Sales Order Segment Picking Options
Segment picking is used to pick and process orders by truck route, minimizing the time it takes to deliver orders. All documents may require user-forms changes at time and materials. From the same warehousing screen select F6 – Segments.
•
F6 Segments: A picking segment is a list of products grouped together for picking by customer stop. Based on the options described below, these products can be all or part of an order, or a combination of orders based on a single stop I.D.
•
Floored Items Per Segment: Defines how many floored products are assigned to a picking segment. used when picking sales orders by truck route.
•
Floored Cube Per Segment: Defines how much cube a segment contains for a floored segment. For example, if a pallet is ‘x cubes, the segment would be generated with enough products to meet this cube value. Then a new segment is started.
•
Floored Segment Pick Multiple Per Cube: For floored segments, choose Y or N to combine multiple segments in a single picking session to pick up to the cube value.
•
Non-floored Items Per Segment: Defines how many non-floored products are assigned to a picking segment. Used when picking sales orders by truck route.
•
Non-floored Cube Per Segment: Enter the cube limit to limit picking segments by cube. A new segment is started with enough products to meet this cube value.
•
Non-floored Segment Pick Multiple Per Cube: For non-floored segments multiple segments can be combined in a single picking session to pick up to the cube value.
•
Non-floored Segment Require Packing: This option can be set to require that a non-floored segment go through Segment Packing Entry before it can be truck loaded. This would be for repack items.
•
Picking Bin Confirmation: This is used to require validation of the picking bin for segment picking. If N is entered the bin is displayed, but cannot be changed.
•
Default Box Type: Enter the default box type to be used in segment picking. This default is used for products that do not contain a box type code.
•
Segment Destination Bin Confirmation: After the segment was picked in full, the user can be prompted for the destination bin where they are taking the product.
-
Y - Performs a bin move to this destination bin.
-
N - Leaves the product on the To Bin entered during picking.
•
Generate Segments in Ascending/Descending: The picking segments can be sorted by ascending delivery sequence/stop id or descending sequence.
•
Separate Segments Per Release: Combines multiple sales order releases for picking by stop I.D. If this flag is set to N, each release generates a separate picking segment.
•
Separate Segments Per Zone: As each new zone is evaluated, a new segment is generated.
•
Cycle Count Products Not Loaded: Enter Y or N to determine if products that were not updated through route processing should go to the emergency cycle count queue.
•
Route Processing Printer Prompt: Determines if you must choose a printer during route processing.
•
Print Shipping Label in R/F Segment Picking: Enter Y or N to print shipping labels for floored segment products during RF segment picking entry.
•
Print BOL in Route Processing: Enter Y or N to print a delivery bill of lading as a part of the route processing documents.
•
Print Packing List in Route Processing: Enter Y or N to print a packing list as a part of the route processing documents.
•
Print Price Labels in Route Processing: Enter Y or N to print a price labels as a part of the route processing documents.
•
Print Invoices in Route Processing: Enter Y or N to print the invoices as a part of the route processing documents.
•
Print RGAs in Route Processing: Enter Y or N to print any open RGAs for the customer as a part of the route processing documents.
•
Prtr Prompt in Route Processing: Determines if you can choose a printer during route processing.

Sales Order Bin Allocation and Pick Ticket Printing
Path: Warehousing >Sales Order processing > Sales Order Bin Allocation (/SOBA)
This process allocates inventory from the available bin locations and releases the pick tickets. It can be run continuously so that orders can be allocated automatically as soon as the order is set up for picking, or it can be run in a batch mode. The batch process must be run periodically anyway to handle inventory that becomes available for picking due to bin movements, stock receipts, inventory adjustment, etc. This procedure prints a pick ticket or creates a picking queue for radio frequency picking or both, depending on the way the Sales Order Picking Paperless flag is set in the warehouse record.
The order of bin allocation is determined by a set of rules established by your company. These rules define what bin types are available for sales order picking and in what sequence they should be used. Rules are established for both lot controlled and non-lot controlled products. If the product is available in the warehouse but not in a bin type that was identified as available to pick from, then an emergency bin replenishment can be created (see Bin Replenishment), and the order will not be set up for picking. The lines that are available to pick are put on a Bin Queue file ready to be selected by a picker. If pick tickets are being used, they will print at this time.

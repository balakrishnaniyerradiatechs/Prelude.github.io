Transfer Picking Options (F5)
Radio frequency transfer picking is used to pick products from pick and/or bulk bins. The system accumulates the products on to a single shipping transfer until the shipment is processed. From the same warehousing screen select F5- Transfers/work orders.
•
Stock Transfer FIFO Lot Allocation Flag:
-
Y - Uses pure FIFO lot allocation and always allocate the oldest lot first without a check to see if a single lot can completely fill a requirement.
-
N - Allocates the oldest complete lot that can satisfy the requirement, then fills by multiple lots.
•
Stock Transfer Radio Frequency Picking: Controls if the system should update the radio frequency picking file for stock transfers. If the option is N, you can implement the warehousing philosophy while still processing stock transfers via a pick ticket.
•
Stock Transfer Prioritize By Date: Determines if the transfer date is included in the priority for assigning transfers to RF Transfer Picking.
-
Y - The transfer date is used to prioritize transfers to RF Transfer Picking.
-
N - The transfer date is NOT included in the priority for assigning transfers to RF Transfer Picking. This updates the transfers for RF Transfer Picking using the S/T Bin Sorting Method established on the same screen.
•
Stock Transfer Bin Priority and Types: Allows the warehouse to prioritize the bins for transfer alPath:
-
B - Bulk Bin.
-
D - Dock Bin.
-
I - Inspection Bin.
-
K - Bulk Unpackaged.
-
L - Line Stock Bin. Leave S/T Bin Sorting Method blank or it will override this field.
•
Stock Transfer Bin Sorting Method: This is another method of prioritizing bins for stock transfer allocations. Any entry in this field overrides the allocation plan of the previous field (S/T Bin Priority and Type).
-
AA - Ascending Bin Available Quantity
-
AO - Ascending Bin On-Hand Quantity
-
AD - Ascending Bin Transaction Date
-
DA - Descending Bin Available Quantity
-
DO - Descending Bin On-Hand Quantity
-
DD - Descending Bin Transaction Date.
Work Order Picking Options
•
Work Order FIFO Lot Allocation Flag:
-
Y - Uses pure FIFO lot allocation and always allocates the oldest lot first without a check to see if a single lot can completely fill a requirement.
-
N - Tries to allocate the oldest complete lot that can satisfy the requirement, then fills by multiple lots.
•
Work Order Bin Sorting Method:
-
O - Sorts the bins by ascending bin onhand
-
D - Sorts the bins by ascending bin transaction date. This is a secondary sort in the Allocation Priority for non-lot controlled items described below.
•
Work Order Allocation Priority: Allows the warehouse to prioritize which bin types should be used for work order bin allocation.

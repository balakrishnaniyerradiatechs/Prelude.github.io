Picking by Segment
Radio Frequency Truck Order Picking
Menu path: Warehousing > Handheld RF Menu > Doc Picking > Truck Order Picking (/RSPE)
If the ship via assignment maintenance is used, the system determines the next available segment by the user’s ship via code. Each picker is assigned a segment and is guided through the warehouse in the logical picking sequence for each product. As each product is confirmed, each bin is updated on-line. Shipping labels for floored segments are generated at this time since floored products are taken directly to the staging area. The following program prints a paper version of the radio frequency picking process, which can be used in the event a radio frequency unit is inoperable, or as a backup method.
Non-RF Segment Picking
•
Segment Printing (/SEG.PICK.PRINT) - This prints a manual printing report if the RF device is not being used for picking.
•
Segment Picking Entry (/OSPE) - This is the manual picking entry confirmation screen if RF picking is not being used.
Segment Packing
Menu path: Warehousing > Segment Processing > Segment Packing (/SGPM)
Each segment is available for packing after the segment picking has been completed. Segment packing is done for non-floored segments. Floored segments are automatically marked as packed after the picking update is performed. The packer enters the number and type of boxes for each segment. Shipping labels are printed for the segment so that each box has a unique sequential box number.
Loading the Truck
Radio Frequency Truck Confirmation Entry
Menu path: Warehousing >Handheld RF Menu > Processing >Truck Cfm Entry (/TRCE)
The truck loader is shown the next stop ready for loading so that the truck can be loaded in the correct sequence. Each box number is scanned and updated as it is loaded.
Non-RF Truck Loading
Menu path: Warehousing > Segment Processing > Truck Load Maintenance (/TRLM)
This can be used as a manual method of the radio frequency truck loading verification. When a route number is entered, a list of segment numbers is generated for that route for each stop. This also shows the picked and packed flags for each segment. A drilldown is available for each packed segment showing box numbers, box type and a loaded flag. The route loading must start with the first stop on the screen and cannot continue with another stop until the previous stop is completed. Segments within a stop can be loaded as packing is completed.
Route Processing
Menu path: Warehousing > Segment Processing > Route Processing (/RTE.PROCESS.UPDATE)
Route processing occurs when all steps for the route have been completed. When the completed route number is entered, all the sales orders are updated with the quantity shipped and are setup for invoicing.
At this time, printed documents are generated for each route. This can include a bill of lading, an invoice, and also a driver’s manifest - based on the specific document print flag.
Inquiries
Menu path: Warehousing > Segment Processing > Inquiries
•
Sales Order Route Inquiry (/SORI) - This inquiry allows the warehouse manager to view the status of a proposed route before doing the bin level allocation. Displays the total number of orders, line items, and weight.
•
Ship Via Route Inquiry (/SVRI) - Displays the current routes being processed by the warehouse. The information displayed includes ship via, route number and the number of picking segments.
•
Segment Inquiry (/SOPI) - Displays the information for a picking segment, product number, quantity picked, picker id, and packer id.
•
Route Status Inquiry (/RSI) - This inquiry provides the status of each of the segments. The user can enter the specific route for inquiry and the system displays a summary screen showing each stop and if that stop is ready for loading. A drill down is available to show the detailed information for each stop, including the list of segments and the picked and packed flags. An additional drill down is available to show the box number, type and loaded flag for each segment.
•
Freight Status/Boxing Inquiry (/FRSI) - This inquiry shows the status of segments for a ship via and route.
•
Route History Inquiry (/RSI) - Displays the history record for a truck route including the customer number, order number and ship date.
Release Weight Report
Menu path: Warehousing > Segment Processing > Reports > Release Weight Report (/REL.WGT.RPT)
This summary report displays the current weight of all the orders ready for a truck route.

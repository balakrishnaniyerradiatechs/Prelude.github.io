Warehouse Maintenance (/WM)
Path: Systems > System Parameters > Maintenance > Inventory Controls > Warehouse Maintenance
Warehousing flags control updates at the warehouse level. This allows you to use the warehousing module for a main distribution center while still allowing other branches to operate without the parameters in place for radio frequency unit transactions. This screen is used to set the Stock receipt options and the Cycle count options
Stock Receipts Options
From Warehouse Maintenance, choose Action Bar > Addl Info > Bin Locator. The top part of this screen is used to determine where stock receipts are allocated.
•
Default Receiving Dock Bin: This is the bin that stock receipts default to for non-inspection stock products.
•
Default Linkage Receipt Bin: This is the bin that stock receipts default to for linked products.
•
Default Inspection Bin: Default bin for received items that require inspection.
•
Default Return Bin: Default bin for credit memo entry.
•
Default WO Completion Bin: This is the bin work order completion default for BOMs completed.
Cycle Count Options
•
Multiple Bin Locations: This flag must be set to Y to use the cycle count subsystem. It indicates that the warehouse allows a product to reside in multiple bins and that the system will track the quantity on-hand in each bin. If this field is null indicating that this option is set at the system level, the system looks at the Multiple Bin Locations option parameter to determine if cycle counting may be performed.
•
Cycle Count Days: Indicates the number of days a warehouse uses to spread the distribution of scheduled bin counts. This is the estimated number of days in the year that bin counts will be taken. The default for this field is 200 days.
•
Phys Cnt Tolerance%: Defines a discrepancy tolerance for a warehouse when performing a cycle count. When the physical count quantity entered differs from the book on-hand quantity by more than the percentage set in this field, the system requires a confirmation that the entry is correct. Such products are placed into the second level queue requiring the product to be counted again.
•
Update Bin Count 2nd Levl Queue: Determines if the bin count will update the YTD Bin Counts (attribute 25) for every bin on the list in the BIN.LOC file.
-
Y - If the bin count will update YTD Bin Counts in the BIN.LOC file for every bin on the list in the 2nd Level Cycle Count Entry.
-
N - If the bin count will not update YTD Bin Counts in the BIN.LOC file for every bin on the list in the 2nd Level Cycle Count Entry. Only the bins with counts are updated to the YTD Bin Counts field.
•
Update 1st Level Adj as Rejects?: Determines if the product is marked as a reject in the first level of cycle counting.
-
Y or Null - A product is marked as a reject (attribute 27 in BIN.LOC file updated). This occurs in the 1st level queue if the counted on-hand is different from the book on-hand and the product is in tolerances but will need an adjustment to get to the correct inventory on-hand. The Bin Count Statistics Report is updated in the product reject column on the report.
-
N - The product is not marked as a reject (attribute 27 in BIN.LOC file not updated) if the conditions above are met. The Bin Count Statistics Report is not updated in the product reject column on the report.
•
Default P/N in 2nd Level Entry?: Determines if the product number is defaulted in the Product Number field in the 2nd Level Cycle Count Entry.
-
Y - A product number is defaulted in Cycle Count Entry – 2nd Level.
-
N - A product number is not defaulted in Cycle Count Entry – 2nd Level.
•
Update /ECCI Queue?:
-
Y - The Emergency Cycle Count Inquiry is updated from the entries in the system.
-
N - The Emergency Cycle Count Inquiry is not updated from the entries in the system. The Emergency Cycle Count Entry prompts with a message at the warehouse number field that entry is disallowed.
•
Exclude Non-Net Bins 2nd Level: V18+ It determines if the non-nettable bins are included in the list of bins for the product in 2nd Level Cycle Count Entry.
-
Y - The non-nettable bins are excluded from 2nd Level Cycle Count Entry.
-
N - The non-nettable bins are included in 2nd Level Cycle Count Entry.
•
Scan First Level counts: V18+ It determines if products are to be scanned into the system in the Cycle Count - 1st Level Counts:
-
Y - To scan products into the system in the Cycle Count - 1st level entry screen. The quantity is defaulted to the shelf package quantity established in Product Maintenance, but may be overridden.
-
N - Products are NOT scanned into the system in the Cycle Count 1st level entry screen, but are entered once with a total count quantity.

Warehousing After Go-Live - Checklist
Review System Bins
There are five system bins that should be reviewed frequently. If a product has an on-hand quantity in any of these bins that is not committed, then this should be investigated and moved out of the bin as soon as possible via a “bin move”. All five of these bins can be reviewed through Bin Onhand Inquiry (/BOHI) or the Bin Products Onhand Report (/BIN.PRD.OH.RPT).
•
PICK: Generally used by companies that do paper order picking, but use /PLRC or /PKCE for terminal ‘picker’ confirmation. These routines do a bin move to this bin location. Once a sales order is confirmed, the inventory is ‘pulled’ from this bin.
•
INV: This is used as a ‘holding’ bin for “confirmed as shipped” inventory. Once a release is confirmed for shipment and released for invoicing, the system does an automatic bin move to this bin. After a sales order is invoiced the inventory is then ‘pulled’ from this bin.
•
TRNERR: This bin is used if a transfer has been shipped from one warehouse to another but is deleted before the transfer is received into the second location. All of the products on the deleted transfer release go into this bin.
•
CYCLE: If the second level cycle count queue is updated because the counted on-hand quantity is greater than the tolerance allowed; then the current bin on-hand will be adjusted up/down for the count quantity and the system bin will be adjusted up/down the opposite way. This bin should be reviewed after first and second cycle counting has been completed.
•
NULL: If a quantity is found in this bin, contact Customer Service at Prelude as soon as possible to be corrected. To review this bin, go to Bin On-hand Inquiry, and enter a warehouse number then enter past the bin number.
View Reports
You should run several reports regularly to validate bin accuracy, per your warehouse manager (e.g., daily for the first 2-3 weeks after a warehousing go live). Once the manager is confident that products and bins are in balance and moving properly, the frequency can be reduced to weekly or at least monthly.
•
Negative Bin On-hand Report - lists products with a bin on-hand less than zero. Investigate these bins and correct the inventory as soon as possible using bin moves.
Path: Inventory>Reports>Stock Status Reports>Negative Bin On-hand
•
Bin Quantity Discrepancy Report - Lists products with total on-hand that does not match the total of its bin on-hand balances. Report these products to Prelude Customer Service as soon as possible for correction.
Path: Inventory>Reports>Stock Status Reports>Bin Quantity Discrepancy
•
Sales Order Pick Ticket Exception Report - This report can be used to determine why a sales order has products that are not allocating to bins.
Path: Additional Modules>Warehousing>Reports>Sales Order Pick Ticket Exception Report

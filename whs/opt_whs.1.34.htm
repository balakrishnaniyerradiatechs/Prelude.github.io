Multi P/O Stock Receipts
(/RFMSR)
Handheld RF Menu > Processes > Recv & Putaway >Multi P/O Stock Receipts
You can receive products using hand-held RF devices by individual P/O or by a batch of multiple P/Os that are assigned to the same stock receipts batch number. On RF devices, multiple users can access the same P/O silmultaneously.
•
P/O # - The purchase order number to be received.
•
Product Number - The scanned product number and description. If Display Count of Open B/Os in Warehouse Maintenance is Y, the number of open backorders for this line item appears.
•
Qty Recd - The quantity to be received. Enter a negative quantity to correct a receiver if it is not updated, the negative quantity is not greater than the original quantity, and the item appears on one line on the P/O. Also, if the item is a serialized or lot controlled, enter the full quantity.
•
Bin - The to bin for a non-serialized product. For serialized and lot products, the to bin appears when you enter the lot or serial number.
•
Ok to Upd? - Determines if the product number and quantity entered are correct:
-
Y - Updates the line item as received and enter the next part number.
-
N - Changes to the product number or quantity.
•
F5-PBI - The Product Bin Inquiry for this product number and bin location.
•
F6-Add - Choose an option:
-
Reject - Put items on the receiver on a warranty tag for the rejected stock. If you receive partial quantities and reject stock, update the batch to combine items on a warranty tag.
-
Quick Scan - Enter a default quantity and bin for scanned products.
•
F2-Save: Saves the received stock. You must update the batch to move the stock into available inventory.
Receiving Stock without RF
Inventory > Stock Receipts > Stock Receipts Entry (/SRE)
Use to receive stock that is not barcoded. You can receive into the default bin or choose another bin. The default label information for the product appears. You can change the data and print the labels.
Viewing Batch Information
Inventory > Stock Receipts > Batch Stock Receipts > Batch Edit Report (/BSR.EDIT.RPT)
The Batch Edit Report shows RF or batch-received inventory by P/O, batch, and/or warehouse. The report is optional, but it can help you verify the accuracy of items, quantities, and totals of received stock. You can also choose to see only over or under-received lines. You can correct the received quantities in Batch Stock Receipt Entry (/BSRE), run the Batch Edit Report again, and update the batch.

Updating Inventory
Inventory > Stock Receipts > Batch Stock Receipts > Batch Update Procedure (/BSR.UPD)
You must update the batch to update inventory with the received stock. You can update stock for all inventory or you can choose batches to update based on P/O, batch, and/or warehouse number. Run Release Pick Tickets (/REL.PICK.TICK) to fill any open orders with the received material.
Print Product Label By Receiver
Path: Warehousing > Labels > Product Label Print By Receiver
Barcodes can be printed by receiver number, transfer number or manually.
•
Warehouse Number: Enter the warehouse number associated with the receiver number. This field is mandatory.
•
Printer Form Queue: Enter the barcode form queue where the barcodes are to print. The form queue is validated with the barcode label printer established in Warehouse Maintenance.
•
Document Type: Enter the document type for printing the barcode labels for:
-
P - Purchase order receivers.
-
T - Transfer receivers.
-
M - Manual entry. If this option is chosen you are prompted for the product numbers and the number of labels to print for each product. The labels are then printed directly from here.
•
Document Option: Open or received documents can be selected for this process. This option is only accessible if the previous field is set to P or T (V18+).
-
O - Open documents.
-
R - Received documents.
•
Receiver Number: Enter the purchase order receiver number or transfer receiver number depending on the Document Type entered.
If you select P or T for the Document Type and enter a valid Receiver Number, you have the following options for printing barcode labels:
•
Single: Displays the products and quantities received for the receiver number entered. The default label quantity is 0 or Null.
•
Exception: Displays only products and quantities that were received for the receiver number entered. The default label quantity is the Quantity Received/Case Quantity of Product.
•
Complete: Determines the label quantity by Quantity Received/Case Quantity of Product. The system automatically updates the label quantity for the line items received and prints the barcode labels. No additional screen displays.
If you select M for the Document Type, the Manual Product Label Print Entry screen displays. The product number and label quantity can be entered for printing barcode labels.

Putting Stock Away with RF After Posting the Receipt
Warehousing > Handheld RF Menu > Processes > Rec & Putaway > Putaway Ent (/RBPE)
Use this process to move the products to their final bin. Bins are sorted based on the Putaway Sort or Bin Sort Sequence. If you change the bin and Update Primary Bin When Incr Onhand is N and RF Primary Bin Update is Y in User ID Maintenance, you can change the primary bin for the product/warehouse to another type P bin. Enter the warehouse and from bin and choose a putaway method:
•
B - Put the products in a bulk bin where this product exists. Products and their bins appear.
•
P - Primary directs you to the primary bin. All products and their bins appear.
•
M - Manual allows you to enter product, quantity, and destination bin.
•
N - Similar to manual, but displays the primary bin for the scanned product.
•
K - Similar to manual, but displays the bulk bin for the scanned product.
•
V - Volume. If you choose the putaway by volume, all bins should have volume parameters. Bins are analyzed to determine the space available based on proximity to the primary bin and the bins fill-level and partial putaway minimum percents in Zone Maintenance. The putaway quantity may be split among 2-4 bins based on the minimum percent and the bins capacity. For example, if the entire quantity should go in the bin, set the minimum to 100%. To split the quantity in two locations, enter 50%. To split it in four locations, enter 25%.
Linked Products
For linked products, you must indicate the final destination bin, which is used for bin allocation in Warehousing.
Stock Products
•
Only the primary bin if the putaway quantity is less than or equal to the quantity to fill the bin using (putaway qty < (bin max - bin on-hand)). Bin on-hand must be less than the bin minimum.
•
Primary bin and a reserve bin where the product exists on the same aisle if the putaway quantity is more than the quantity needed to fill the primary bin. Bin on-hand must be less than bin minimum. Fill the primary and put the rest in a reserve bin.
•
Only reserve bin(s) on the same aisle as the primary if it can hold all or some of the quantity to be put away up to the zones fill level percent. If it cannot hold all of the putaway quantity, place the remaining quantity in another reserve bin where the product exists on the closest (preferably same) aisle.
•
One empty bin in the same aisle as the primary if the reserve bins cannot hold the entire quantity.
•
Reserve bin(s) in any aisle that contain the product if they can hold part of the quantity to be put away up to the zones fill level percent. Select bins in the aisle closest to the primary bin. Do not use both bulk reserve bin(s) and empty bin(s).
•
Any empty bin on an aisle as close as possible to the primary bin.
Entering Alternate RF Part Numbers
Products in Prelude can be set up with alternate product numbers. The numbers and the corresponding UCC codes can be maintained using radio frequency devices.
Path: Warehousing > Handheld RF Menu > Maintenance > Alternate PN Mnt (/RF.ALT.LOAD.ENT)
This maintenance screen is available for use with the Radio Frequency scanning devices to scan alternate product numbers for a product number.
•
Product Number: Enter the product number to be updated with the alternate product number. The description displays below.
•
Alternates: The 1st existing alternate product number established for the product displays. Use the arrow down key to move through the entire list of alternates.
•
UPC: Determines if the alternate product number is to be established as a UPC code.
-
Y - To update the alternate as a UPC code.
-
N - To not update the alternate as a UPC code.
•
UCC: Determines if the alternate product number is to be established as a UCC code:
-
M - Master pack
-
P - Pallet Pack
-
S - Shelf Pack
-
N - To not update the alternate as a UCC code.
•
UCC Qty: Enter the quantity of the UCC code. This quantity defaults in Serial Ship Container Entry for label printing purposes. In V18+, this also defaults the UCC quantity in the Qty Recd field in the RF Transfer Receipts and Multi Transfer Receipts entry screens when the alternate product number is entered. This can be overridden. If the unit of measure on the stock transfer line is different than what was in Product Maintenance, the message Please verify Qty Recd is prompted to ensure the correct quantity is received.
•
U/M: Enter the unit of measure for the UCC code.
•
Cat ID: Enter the catalog identification code for this UCC code. These codes are setup in Catalog Identification Code Maintenane. You can view the catalog IDs used in sales orders with the Order History Report By Catalog ID Report on the Order Processing menu.
•
Alt Ppg: Enter the alternate product price group for the UCC code. This field defaults to the alternate product price group established for the catalog identification code entered in the Cat ID field.

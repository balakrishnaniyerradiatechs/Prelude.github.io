Replenishing Bins
Normal bin replenishment occurs when a product falls below its bin minimum and a replenishment record is created to bring it back up to its maximum. Emergency replenishments occur when the Sales Order Bin Allocation procedure is run and detects a situation where a quantity on a sales order cannot be picked. A set of rules are established to determine the type of bin to replenish, the type of bin to replenish from, and in what sequence the bins are used. This document explains how to establish the maintenance flags for bin replenishment and the procedures to follow to activate the replenishment process. Radio Frequency procedures for bin replenishment processing are included also.
Bin Replenishment Maintenance
Non-RF Bin Replenishment Maintenance
Path: Warehousing > Bin Replenishments Processing > Bin Replenishment Maintenance (/BRM)
The minimum and maximum for each product / bin combination to be replenished are maintained with this program. When using the different putaway methods, the system calculates the available bins using the minimum and maximum values. Product / bin combinations can be set with frozen controls and a thaw date. The Auto Bin Replenishment Min / Max Recalculation skips items marked as frozen until the thaw date is reached.
•
Warehouse Number: Enter the warehouse number to view the bin replenishment information.
•
Bin Path: Enter the bin location to view the bin replenishment information.
•
Product Number: Enter the product number in the specified warehouse and bin location to view the bin replenishment information.
•
Minimum Quantity: Enter the minimum quantity to have in the bin location before standard bin replenishment is used.
•
Maximum Quantity: Enter the maximum quantity to have in the bin location. When the bin reaches its maximum, the putaway methods distribute the excess items to the appropriate bins.
•
Frozen Control: Determines if the bin is to be included in the Bin Replenishment Process.
-
Y -To not include the bin in the Bin Replenishment Process.
-
N - To include the bin in the Bin Replenishment Process.
-
M - To allow the bin maximum flag to not be included in the bin replenishment process but allow the bin minimum flag to be included. If the bin minimum calculation is greater than the frozen bin minimum, the system sets the bin minimum to the frozen maximum quantity minus 1.
•
Thaw Date: Enter the date the bin was set to frozen control.
•
F2 - Save: Saves the Bin Replenishment information.
RF Bin Replenishment Maintenance
Path: Warehousing > Handheld RF Menu > Maintenance > Bin Rep Mnt (/RFBRM)
This is a radio frequency maintenance process similar to terminal bin replenishment maintenance. The radio frequency unit can scan the bin location, scan the product number, and enter the bin minimum and maximum.
When a bin replenishment record is entered in the RF bin replenishment screen and the Frozen Control flag in Bin Replenishment Maintenance is set to Y or M, the warning message, Frozen Controls are set displays to notify you that the bin replenishment record contains frozen control settings. Min/max values can continue to be entered for the bin replenishment record.
Bin Replenishmen t Processing
Path: Warehousing > Bin Replenishments Processing > Bin Replenishment Processing (/BIN.REPLENISH.UPD)
This nightly process selects all bins with the Auto Replenish Flag set to Y and evaluates each product in those bins. If the current available is below the minimum, then a replenishment request is generated to fill the bin to the maximum. The rules established in the maintenance programs define what type of bin should be replenished from and in what sequence the replenishment from bins are used. If the single bin replenishment in Warehouse Maintenance is set to Y, then inventory is allocated from a single bin. If set to N then inventory is allocated from multiple bins until the entire replenish quantity was met. In V18+, this process includes BUL bins.
Selective Bin Replenishment Process
Path: Warehousing > Bin Replenishments Processing >Selective Bin Replenishment Process (/SEL.BIN.REPL.UPD)
This is similar to the Bin Replenishment process listed above, although the analysis is only done based on a defined range of bin locations.
Bin Replenishment Release Process
Path: Warehousing > Bin Replenishments Processing >Bin Replenishment Release Process (/BIN.REPL.DEL)
This process removes all current open bin replenishment transactions allowing the committed inventory to be available for other transactions. The release is only done for products not currently assigned to a replenishment picker. After a warehouse number is entered, a warning message is displayed.
•
Yes: To release bin commitments.
•
No: To not release bin commitments.
Bin Replenishment Min / Max Update
Path: Warehousing > Bin Replenishments Processing >Bin Replenishment Min/Max update (/BIN.REPL.MIN.MAX.CALC)
This process selects all products with a bin min/max quantity and recalculates these values. The recalculation is done by taking the products average monthly usage, multiplied by the Bin Replenishment Min / Max Percent in warehouse address maintenance.
•
Warehouse Number: Enter the warehouse number to limit the process to a warehouse or leave this field blank to include all warehouses.
•
Use Existing/Create New (E/N): Enter the appropriate action for the process to recalculate the bins.
-
E -To recalculate bin min/max quantities for bins with existing min/max quantities.
-
N -To generate new min/max quantities for bins with no current min/max quantities.
•
Preview or Update:
-
P - To print the preview report and not perform the Bin Replenishment Min/Max Update process.
-
U - To perform the Bin Replenishment Min/Max Update process and skip the preview report.
The bin minimum and maximum fields default to 1 if no monthly usage exists for the product/bin combination. The existing bin min/max figures are replaced with the new calculated values after the process is complete.
Replenishment Picking Report
Path: Warehousing > Bin Replenishments Processing >Replenishment Picking Report (/RELP.PICK.PRINT)
As a backup for the radio frequency terminal entry, it is possible to print a report to use for replenishing. The report shows the product, quantity, From Bin, and To Bin to replenish.
Bin Replenishment Picking
Replenishment Picking Entry
Path: Warehousing > Bin Replenishments Processing >Replenishment Picking Entry (/RPE)
This process records the bin moves if the Replenishment Pick Report is used to do the replenishing. The products, quantities and bins are displayed for confirmation.
•
Warehouse: Enter a valid warehouse for this transaction.
•
User Id: Enter the Id of the user performing the picking process.
•
# of Lines: Enter the number of lines to be picked for this session.
•
From Zone: Enter a zone to start picking from. F3 for options.
•
To Zone: Enter a to zone or leave blank.
•
Emg: Enter Y to select only products for emergency bin replenishments.
•
Product Number: Enter a product number if transactions are to be performed for just one product. Leave blank to bring up all the products in a zone.
•
Move #: Displays the next available Bin Move Number pulled from Counter/Range maintenance.
•
Product Number: Displays the product numbers in the zone.
•
From Bin: The current bin location is displayed.
•
Qty: The current quantity of the bin location is displayed.
•
To Bin: Enter the bin to move the product to.
•
F2 - Save: Saves the bin move information.
Radio Frequency Bin Replenishment Picking
Path: Warehousing > Handheld RF Menu > Doc Picking > Bin Repl. Picking (/RRPE)
Path: Warehousing > Truck Mount RF Menu > Doc Picking > Bin Repl. Picking (/TRRPE)
These programs move selected products from the queue being maintained for radio frequency transactions to their own personal queue. The screen is slightly different for truck mount devices. Products needing replenishment are displayed along with the from bin and quantity. The picker is guided through the warehouse in the logical picking sequence for each product.
•
Warehouse: Enter the appropriate warehouse number.
•
# of Lines: Enter the number of lines to pick for this session. It is best if the picker only selects the number of lines that can be processed in a single picking session.
•
Zone: Enter the picking zone. F3 for options.
•
To Zone: If applicable, enter the zone the products will be moved to.
•
Emergency: Enter Y to select only products for emergency bin replenishments.
•
Product number: Enter the product number to only process transactions for one item, or leave blank to include all in the zone.
If the Putaway to Pick Bin (in Warehouse Maintenance) is set to Y, then the To Bin displays and the product is moved. If the Putaway to Pick Bin is set to N, then the mobile bin (cart) is prompted for. After all products are picked, the method of putaway is prompted for. The options are:
•
Put - Primary bin sequence,
•
Pick - Picked sequence
•
Reverse - Reverse of picked sequence.
Sales Order Bin Allocation
Path: Warehousing >Sales Order Processing >Sales Order Bin Allocation (/SOBA)
This program determines if there is insufficient inventory in a pick bin and if there is available inventory in another bin defined as a replenishment bin. It then generates an emergency bin replenishment request.
The quantity to be replenished is calculated by adding the quantity required for the sales order to the quantity that brings the bin to its maximum inventory level. The sales order is not released for picking until the replenishment is complete.
Bin Replenishment Report
Path: Inventory > Reports > Miscellaneous Reports > Bin Replenishment Report (/BIN.REPLEN.RPT)
This report displays information on bin replenishments based on the primary bin onhand quantity. If a product does not contain any alternate bins with a positive quantity, it is not included in the report. If the product has quantity in the primary bin and at least one alternate (secondary) bin, it is included in the report depending on the selection criteria.

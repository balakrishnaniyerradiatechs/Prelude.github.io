Recosting a P/O
If a vendor invoice does not match the total merchandise received due to a a cost difference, you can recost the P/O. You can also place the invoice on buyer hold so the buyer can recost the P/O (see Placing an Invoice on Buyer Hold).
Recost Limit
Vend Inv Recost% in User Identification Maintenance > F5-Administrative determines the largest percentage that this user can recost a purchase order in Vendor Invoice Reconciliation Entry before the invoice goes on hold. If this field is blank, the user can recost a P/O for any amount.
Update Replacement Cost
Repl Cost Upd Direct Shp in Accounts Payable Parameters Indicates if replacement cost is updated when you change the cost in Vendor Invoice Reconciliation Entry on products that are set up as direct ship only (Availability Type set to D in Product Maintenance > F5-Add Info):
•
Y - Update the cost for direct-ship products.
•
N - Do not update the cost for direct-ship-only products.
To recost a P/O:
1
Depending on when you choose to recost the purchase order, select one of the following:
•
From Vendor Invoice Reconciliation Entry, press F7-Recost.
•
If you get the discrepancy warning message when you try to complete a receiver, choose Recost.
The AP/PO Cost Adjustments screen appears.
2
For each line item, enter either the revised cost per unit of measure for Rev. Cost or the revised discount under Rev. Dsc. When you change a cost or discount, you may be required to enter a cost adjustment reason code. The AP/PO Cost Adjustment Reasons screen appears.
3
Enter an adjustment code and press F2-Save.
Cost Adjustment Reason Codes
S/R Adj. Reason Code in Accounts Payable Parameters determines if a cost adjustment reason code is required when a received cost is changed:
•
Y - Require a cost adjustment reason code when cost is adjusted.
•
N - Do not require a cost adjustment reason code when cost is adjusted.
4
When each line item is correct, press F2-Save.

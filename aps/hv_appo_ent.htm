High Volume AP/PO Matching Entry screen
If you receive invoices from a vendor using EDI, High Volume AP/PO Matching can help to simplify invoice/receiver reconciliation.
Refer to Reconciling High Volume EDI Invoices for more information
Menu Path: Financial > Accounts Payable > AP/PO Subsystem > High Volume AP/PO Matching > High Volume AP/PO Matching Entry
Field and Function Key Descriptions
Field/Function Key
Description
Entry#
Displays the next sequential counter for the transaction. Only one vendor invoice can be used per transaction.
P/O Number
The purchase order number to pay.
Vendor
The vendor number.
Invoice
The vendor’s invoice number associated this payment.
Invoice Date
The date the invoice is issued.
Terms Code
The terms code from the original purchase order.
Discount Date
The discount date from the original purchase order.
Due Date
The date the invoice is due. Based on the Invoice Date and Terms Code.
Merch Amount
The merchandise total for the invoice.
Freight Amount
The freight amount for the vendor invoice if any.
Include?
Determines if the Freight Amount is included in the payment to the vendor:
•
Checked - Includes the freight amount.
•
Not Checked - Does not include the freight amount.
Duty Amount
The duty amount for the vendor invoice if any.
Include?
Determines if the Duty Amount is included in the payment to the vendor:
•
Checked - Includes the duty amount.
•
Not Checked - Does not include the duty amount.
Brokerage Amt
The brokerage amount for the vendor invoice if any.
Include?
Determines if the Brokerage Amt is included in the payment to the vendor:
•
Checked - Includes the brokerage amount.
•
Not Checked - Does not include the brokerage amount.
Misc Amt
Any miscellaneous charges the vendor may be debiting or crediting that are not already included in this entry, (e.g., charges other than merchandise and freight). You can determine if the miscellaneous amount goes to the A/P Miscellaneous Amount G/L Account number or prompts for G/L account numbers.
Discount Amt
The discount amount based on the payment terms.
Invoice Amount
The total amount of the invoice. The total includes the merchandise, freight, duty, brokerage, and miscellaneous charges, if any. The amount does not reflect the discount.
Ship Via Code
The ship via code from the purchase order.
Freight Flag
Indicates if you must pay freight on the P/O:
•
Yes - You must pay freight.
•
No - The vendor pays freight.
F2-Save
Saves the information entered and closes the screen.
F4-Del
Deletes the information.
Action Bar
Notes
Opens the Notes Information screen, which allows you to enter notes for the invoice.
Remit Comment
Opens the Remittance Check Comment screen.

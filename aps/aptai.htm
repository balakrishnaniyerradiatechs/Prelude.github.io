A/P Transaction Audit Inquiry
A/P Transaction Audit Inquiry lists accounts payable transactions (e.g., codes, dates, references, amounts, and balances) for a vendor.
Viewing the A/P Transaction Audit Inquiry
To view A/P transactions:
1
Select Financial > Accounts Payable> Inquiries > A/P Transaction Audit Inquiry to open the A/P Transaction Audit Inquiry screen.
2
Enter a number or keyword for the Vendor or press F3 to search.
3
Choose to Sort by beginning date or ending date.
Field & Function Key Descriptions
Field/Function Key
Description
Vendor
The vendor number.
Primary Contact
The contact name for the transaction.
Phone
The phone number for the contact.
Sort
Determines the order in which the transactions display:
•
Beginning invoice date.
•
Ending invoice date.
The default sort sequence is from Default Parameters Maintenance.
Code
The A/P transaction code:
•
ADV - Post Advance (A/P Manual Invoice Entry).
•
APC - Vendor Invoice Number Change Maintenance (invoice).
•
API - A/P Manual Invoice Entry.
•
CCP - Payment Selection/Print (credit card payment).
•
CHK - Payment Selection/Print & Manual Check Entry.
•
CNV - A/P Transaction Audit Conversion.
•
FWD - A/P Transaction Audit Purge.
•
GCR - Generate Credit Refund.
•
OAR - Recurring Invoice Entry Posting.
•
OER - Create Manual Invoice (A/P Refund).
•
VIC - Vendor Invoice Number Change Maintenance (vendor).
•
VIR - Vendor Invoice/Claim Reconciliation Entry.
•
VOI - Void Check Entry & Void Check Batch Entry.
Date
The date of the A/P transaction.
Reference1
A reference number from one of three areas:
•
The check number if printed.
•
Invoice number if the invoice is reconciled.
•
Invoice order number if the invoice was manually entered.
•
Customer number if an 06 A/P Refund payment method is used to pay off a credit invoice in order processing. This creates a miscellaneous vendor invoice for the customer.
Reference2
The receiver number associated with the vendor’s invoice.
User
The person who completed the transaction.
Amount
The amount of the transaction.
Balance
The unpaid balance on the transaction.
F2-Finish
Closes the A/P Transaction Audit Inquiry screen.
F5-Review
Opens the Vendor Invoice History Inquiry screen.
F6-Report
Opens the A/P Transaction Audit Report screen.
F9-Goto Line
Allows you to enter a reference number for quick access. Valid codes are line number, page number, reference number, B (bottom), or T (top).

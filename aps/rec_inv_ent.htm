Recurring Vendor Invoice Entry screen
You can use recurring vendor invoices for non-inventory expenses that are always paid on the same day of the month and are always the same amount. For example, if you have a rent payment of $1500.00 due at the first of each month, you can set up a recurring invoice for that payment.
Refer to Setting Up Recurring Expense Invoices for more information.
Menu Path: Financial > Accounts Payable > Recurring Vendor Invoices > Recurring Vendor Invoice Entry
Field and Function Key Descriptions
Field/Function Key
Description
Vendor
The vendor number. Use M for a Miscellaneous vendor.
Invoice Number
The invoice number for the recurring invoice. The recurring invoice update process appends the fiscal period to the invoice number.
Transaction Day
The day of the month on which the transaction will occur. If you enter 31 for a month that does not have 31 days, the invoice will post on the last day of that month (e.g., Februaryâ€™s invoice will post on the 28th).
Terms Code
The payment terms for this vendor. Defaults from Vendor Maintenance.
Maximum Payments
The maximum number of invoices that will be paid. limits the number of invoices to generate. If you enter a value in this field, you cannot enter a value in Maximum Period.
Maximum Period
The last period during which invoices will be generated. If you enter a value in Maximum Payments, this field is not accessible.
P/O Number
The purchase order number for the invoice, if one exists.
Invoice Amount
The amount of the invoice.
Merch Amount
The total amount of the invoice that is for merchandise (defaults to the Invoice Amount). Change it if necessary. This amount is used with the Terms Code to calculate the discount.
Discount Amount
The discount amount of the invoice (if any). Discount amounts are calculated using the merchandise amount and accounts payable terms for the vendor.
Ln#
The line count for accounting distribution.
Div
Division number. If your company uses divisions, this represents the highest possible grouping in your company (e.g., Eastern Division, Western Division).
Reg
Region number. A subset of Division if Division is used (e.g. Southwestern Region, Northeastern Region).
Loc
Location number. The warehouse location that represents where the general ledger posting will post.
Dep
Department number. A subset of warehouse. Typically used to separate expenses within a warehouse (e.g., accounting, sales, warehouse).
Ccr
Cost Center number. A subset of Department if Department is used. Typically used to separate expenses within a warehouse or department. (e.g. shipping, receiving, outside sales, inside sales).
Account#
The G/L account number to post the payments to for recurring invoices.
Description
The description of the G/L account.
Dist Amt
The amount distributed to the G/L account number. The total distribution amount must match the invoice total.
Total Distribution
Displays the total amount distributed to the G/L accounts.

Saves the recurring vendor invoice information.

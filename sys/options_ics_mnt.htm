Inventory Parameters screen
The Inventory Parameters screen allows you to set flag and option parameters to control how Prelude operates programs in the Inventory module.
Menu Path: Systems > System Parameters > Maintenance > Option Parameters > Inventory Parameters
Field and Function Key Descriptions
Field/Function Key
Description
Allocate By Age
Indicates if you want to fill backorders by order date:
•
Y - Allocate received material to the oldest sales orders, transfers, and work orders first, if all backorders cannot be filled.
•
N - Do not fill backorders if all orders cannot be filled. A report prints and you must allocate backorders manually. Refer to ”Allocating Inventory Manually” in the Inventory Manual.
Auto Allocate Flag
Determines if inventory is automatically allocated:
•
Y - Inventory automatically allocates when the on-hand quantity is increased (e.g., stock receipts, stock transfer receipts, work orders, and inventory adjustments). You must still run the Allocation Procedure for increases in stock due to credit memos or consignment returns.
•
N - Inventory does not automatically allocate. You must allocate backorders manually.
Refer to ”Allocating Inventory Manually” in the Inventory Manual.
Auto Unallocate Flag
Determines if inventory remains allocated when the available quantity of a product decreases:
•
Y - Inventory automatically un-allocates from committed documents when the available quantity falls below the committed quantity due to an inventory adjustment or a cycle count entry. Line items do not automatically unallocate if they meet the following criteria:
- Sales orders with linked items, a supply code, a released pick ticket, or if the order type is 04, 05, 06, 08, 09, or 10.
- Work orders with linked items, a supply code, or if the work order has an order status other than Entered.
- Transfers with a linked item, a supply code, or a released pick ticket.
•
N - Inventory remains allocated when the on-hand quantity is decreased.
Allocation Method
If Allocate By Age is set to Y, choose one of following methods to allocate received inventory:
•
R - Order date with priorities. Allocates from oldest to newest required date using the following priorities:
- Sales orders.
- Work orders.
- Transfers.
•
D - Order date only. Allocates from the oldest to the newest order date, regardless of document type.
•
O - Order date with priorities. Allocates from oldest to newest order date using the following priorities:
- Sales orders.
- Work orders.
- Transfers.
•
Q - Required date only. Allocates from oldest to newest required date, regardless of document type.
Suppress Allocation Rpt
Indicates if you want the ”Inventory Available for Backorders Report” (Inventory module) to print after the inventory is allocated:
•
Y - Do not print the report.
•
N - Print the report.
Multiple Bin Locations
Determines if inventory is tracked in multiple bins:
•
Y - Items can exist in multiple bins. The system then tracks the quantity on-hand in each bin. This setting automatically sets Validate Bin Locations to Y.
•
N - Inventory is not tracked by bin.
•
M - The primary bin is updated when inventory is received. You have to manually move inventory into secondary bins.
You can override this flag at the warehouse level.
Default Inventory Adjustment Reason Code (unlabeled field)
If Multiple Bin Locations is set to Y or M, the default inventory adjustment reason code to use when inventory is automatically adjusted in the cycle count subsystem.
You can override this flag at the warehouse level.
Validate Bin Locations
Indicates if a bin must be created in Bin Location Maintenance to be entered in Product Warehouse Maintenance:
•
Y - Requires that a bin be created before it can be referenced. This flag is automatically set to Y if Multiple Bin Locations is set to Y.
•
N - Any bin number can be used.
Vendor Part# Xref (Tape)
Indicates if a cross reference file for the vendor part number is maintained when using the price tape updates:
•
Y - Maintain a cross reference file.
•
N - Do not maintain a cross reference file.
Refer to “Updating Prices/Costs Using a Price Tape” in Pricing chapter of the Inventory documentation.
Stock Rec Cost Prompt
Determines if you can edit the cost in the Stock Receipts Entry Line Item Detail screen:
•
Y - You can edit Gross cost.
•
N - You cannot access cost.
•
D - The cost is not displayed.
Safety Allowance %
The safety allowance percent to use as a default when a new product is created in Product Maintenance. This percentage is used to calculate the Order Point.
You can override this percent for a product class in S/A% in the Product Class Settings screen.
You can also override this percent for individual products in Product Purchasing Parameters Maintenance > Safety Allowance %.
If this value is changed, the new percent is stored in all existing products without overrides in Product Purchasing Parameters Maintenance.
Generate UPC Codes
Determines if a new UPC is assigned when a new product is created in Product Maintenance:
•
Y - Generate a UPC each time a new product is created.
•
C - Generate a UPC using the 12-digit checksum logic. The first 11 digits identify the manufacturer and item; the 12th digit is for validation.
•
R - Generate a UPC for products received with a radio frequency device.
•
D - Generate a UPC using the 12-digit checksum logic for products received with a radio frequency device. The first 11 digits identify the manufacturer and item; the 12th digit is for validation.
•
N - Do not generate a UPC code.
When a UPC is created, it is stored with the product as an alternate product number.
If UPC Required (in the Inventory Add. Parameters screen) is set to Y, you must enter a UPC code to receive an item in Stock Receipts Entry.
To automatically create UPCs for all products in inventory that do not already have a UPC, from the Systems menu, select Inventory Utilities > Create UPC Codes For Products.
Refer to “Receiving Products with UPCs in the Inventory” in the Stock Receipts documentation.
UPC Code Prefix (unlabeled field)
A numeric prefix for the UPC. For example, if the prefix is 333 and the next system generated number is 22338, the UPC is 33322338.
Product Keyword Search (unlabeled field)
Determines how to search UPC numbers using product keywords:
•
Y - Search for products first by adding the UPC prefix to the keyword. Then search by alternate part number.
•
N - Search for products using the alternate part numbers before keywords.
Use Check Digit For UPC
If Generate UPC Codes is set to N and you want to use the check sum logic without automatically generating a UPC:
•
Y - The system will not automatically generate a UPC. When you enter N for New in UPC Code in Stock Receipts Entry or in Alternate Product Number Maintenance, the system generates a UPC using the checksum logic.
•
N - Do not use the checksum logic for UPC codes.
Length
The length of the UPC with the check sum:
•
12.
•
13.
•
14.
Product# Prefix
Determines if product prefixes are required when creating a product number in Product Maintenance:
•
Y - Require product number prefixes for new part numbers. You must also enter the Length, which is the number of digits for the product prefix.
•
N - Product prefixes are not required or used in Product Maintenance.
Product Prefixes are created in Product Prefix Maintenance and are used to identify the manufacturer or any other group of products. When you use a prefix to set up part numbers in Product Maintenance, standard information (e.g., price columns, price tape information, product line, vendor number) defaults from the prefix to help expedite the entry of new products. Refer to “Using Product Prefixes” in the Inventory documentation.
Length
The number of digits (up to 3) required for a product prefix, if Product# Prefix is set to Y.
Prod. Lookup Warning
The number of products that can be retrieved by the product keyword search before you are given a warning message. For example, if you enter 50 for Prod Lookup Warning, and you enter a product keyword that can possibly apply to 60 products, a message appears to warn you that the lookup exceeds 50 records. You can then stop the lookup and enter more specific keywords to narrow your search.
Maximum Price Columns
The maximum number of price columns to use in Product Pricing. The Add. Price Columns screen (Product Maintenance > F7-Price/Cost > Action Bar > Add. Prices) is only accessible if this value is higher than 5.
You can use this if you need more price columns in addition to the 5 columns already available, you can use the Add. Price Columns screen to add up to 999 extra price columns. You may want to use this to reduce the chance of employees referencing price columns that are not defined, causing price exceptions. These price columns can be referenced in Customer Maintenance, Product Maintenance, Product Warehouse Maintenance, and Price Matrix Maintenance. The automatic price update programs do not update any more than 5 columns, but if the prices are calculated as a percent or a multiple of another column, they are updated accordingly.
To label any columns in addition to the original 5 columns, select F5-Add Price Columns on the Price Description Defaults screen
Display Fill With Subs
Determines if the substitutes for a fill-with product display on sales orders:
•
Y - Display the substitute items for the fill-with product.
•
N - Do not display substitute items for the fill-with product.
You can set up a fill-with product (Product Maintenance > F6-Xref/Notes > Fill With Product) if you have a part number that can be filled with one or more generic brands of a similar item, but you don’t necessarily want the customer to know the brand they are receiving. The generic product will fill any sales order requirements for the selling product. Refer to “Fill-With Products” in the Inventory documentation for more information.
Omit Fill With Update
Determines if the fill-with product displays on the second line of description for the selling product:
•
Y - Omit the fill-with product on the second line of description.
•
N - Display the fill-with product on the second line of description when the selling product is entered.
Replacement Cost Update
Determines where replacement cost is updated:
•
P - Automatically in Purchase Order Entry.
•
R - Automatically in Stock Receipts Entry.
•
N - No automatic update. It must be maintained in Product or Product/Warehouse Maintenance.
•
O - Optional in Purchase Order Entry.
Prod Vendor Cost Update
Determines where product vendor cost is updated:
•
P - Automatically in Purchase Order Entry.
•
R - Automatically in Stock Receipts Entry.
•
N - No automatic update. It must be maintained in Product Vendor Maintenance.
•
O - Optional in Purchase Order Entry.
Stock Status Default
The default Status Code to assign to new products created in Product Warehouse Maintenance:
•
S - Stock product.
•
N - Nonstock product.
•
Blank - no default.
Allow Overage In SRE
Determines if a linked, over-received item is added to the sales order or put into stock:
•
Y - Allocate the overage to the linked sales order. However, if Multiple P/O Sourcing (Purchasing Parameters screen) is set to Y, the overage will be put into stock.
•
N - Put the overage into stock.
If you over-receive an item by more than the tolerance limits in Warehouse Maintenance, the receiver appears in the Stock Receipts Discrepancy Inquiry.
Product Tagging
Determines how you can look up and enter part numbers in order entry programs and inquiries:
•
N - Product tagging is not activated. You must enter 1 item at a time on the order, then enter the quantity in the detail information screen.
•
Y - Activates the Product Tagging Entry screen for order entry programs to allow you to enter quantities for multiple products to go directly on the order. The Product Tagging Entry screen lists products based on the keyword entered to search for a part number. You can view the available quantity of each item and enter the quantity that you want to add to your order.
•
A - The same as Y, except that the Product Tagging Entry screen also displays for inquiries showing the available and on-order, but you can only select one product at time.
•
I - The Product Tagging Entry screen displays the available and on-order, but you can only select one product at a time.
Auto Recalc Prices
Determines if prices and costs are updated in Product Maintenance and Product Warehouse Maintenance when you save the screen:
•
Y - Prices and costs are recalculated when you press F2-Save.
•
N - You must use F5-Recompute to recalculate the prices and costs.
Prc/Cst Basis Calc Meth
Determines whether prices and costs are calculated using percent markup or gross profit:
•
G - Gross profit. When you enter the gross profit percent that you want to make on a product for a price, the Percent is used with the cost column indicated in Basis to calculate the price.
For example, if Replacement Cost is 10.00.and you want Price 1 to provide 20% gross profit above Replacement Cost, enter 20 under Percent and R under Basis.
When you press F5-Recompute, 10.00 is divided by .80 to come up with a Price 1 of 12.50. Gross profit is 20%.
•
M - Markup. When you enter the markup percent that you want for a product at a price, the Percent is used with the cost column indicated in Basis to calculate the price.
For example, if Replacement Cost is 10.00 and you want Price 1 to be 20% above the value of Replacement, enter 20 under Percent and R under Basis.
When you press F5-Recompute, 10 is multiplied by 1.20 to come up with a Price 1 of 12.00. The gross profit is 16.6%.
Disp Save Prev Pricing
Determines if you want to save old prices when you change a price for a product:
•
Y - Displays a message to allow you to save the previous price.
•
N - Does not display the message or update previous prices.
•
U - Automatically saves previous prices, but does not display a message.
Print Core Receiving Tkt?
Determines if a core receiving ticket prints when a core item is adjusted:
•
Y - Print a core receiving ticket when you adjust inventory for a core item. The core receiving ticket serves as a customer receipt for a returned core.
•
N - Do not print a core receiving ticket when you adjust inventory for a core item. You can manually print the ticket in Core Receipt Ticket Printing.
Refer to “Pending Cores” in the Inventory documentation for more information on setting up and receiving core products.
Copy Whse In Prod Copy?
Determines if only the product header is copied or if the new product is copied into all warehouses that stock the original product:
•
Y - The new product is created in all warehouses that stock the original product.
•
N - The new product is only created in Product Maintenance but not for any warehouses. You can manually add the product to each warehouse in Product Warehouse Maintenance.
Dropship P/O
Serial#
Indicates if serial numbers are required for serialized items on direct ship purchase orders:
•
Y - Prompt for serial numbers.
•
N - Do not prompt for serial numbers.
Refer to “Entering and Tracking Serial Numbers” in the Inventory documentation.
TAI Upd Of Service Items
Indicates if service items appear in the Transaction Audit Inquiry:
•
Y - Service items appear in the Transaction Audit Inquiry.
•
Y - Service items do not appear in the Transaction Audit Inquiry.
#Days Warning In TAI Purge
The number of days that will generate a warning in the Product Transaction Audit Purge. If you enter a number of days less than this amount when purging transaction audit records, you are warned that the number of days is less than this amount but can continue with the purge. If you leave this field blank, no warning appears when purging transaction records.
Capture Weekly Usage
Determines if usage is captured at the company level on a weekly basis:
•
Y - Capture usage by week. Opens the Weekly Usage Options screen to allow you to enter weekly usage parameters.
•
N - Capture usage by month.
If you stock products that have an extremely high sales volume for just a few months of the year, you can use the AWU (average weekly usage) formula to calculate recommended purchase quantities.
Refer to “Tracking Average Weekly Usage” in the Inventory documentation for more information.
F2-Save
Saves the inventory parameter information.
F5-Add. Info
Opens the Inventory Add. Parameters screen.

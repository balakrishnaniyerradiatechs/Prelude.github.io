Accounts Payable Parameters screen
The Accounts Payable Parameters screen allows you to set flag and option parameters to control how Prelude operates for A/P functions.
Menu Path: Systems > System Parameters > Maintenance > Option Parameters > Accounts Payable Parameters
Field and Function Key Descriptions
Field/Function Key
Description
S/R Adj. Reason Code
Determines if a cost adjustment reason code is required when a received cost is changed:
•
Y - Require a cost adjustment reason code when cost is adjusted.
•
N - Do not require a cost adjustment reason code when cost is adjusted.
Fiscal Month Ov. Flag
Determines if the fiscal period can be overridden in accounts payable transactions:
•
Y - Allow overrides to the fiscal period. The fiscal period must be open. An O appears next to the period to indicate that is overridden.
•
N - Do not allow overrides to the fiscal period defaults.
A/P Batch Control
Determines if manual invoices can be entered by batch:
•
Y - You can enter manual invoices by batch. The A/P Batch Information screen appears in Manual Invoice Entry.
•
N - You cannot enter manual invoices by batch.
A/P Receiver Fiscal Flag
Indicates the date to use to determine the fiscal period for received stock:
•
R - Default the fiscal period based on the date stock is received.
•
I - Default the fiscal period based on the date entered when the invoice is reconciled.
A/P Invoice Fiscal Flag
Determines if invoices entered in Vendor Invoice Reconciliation Entry can post to a fiscal period with a status of P:
•
Y - Invoices entered in both Accounts Payable Manual Invoice Entry and Vendor Invoice Reconciliation Entry can post to a fiscal period with a status of P.
•
N - Only invoices entered in Accounts Payable Manual Invoice Entry can post to a fiscal period with the a status of P (open only for accounts payable and journal entries).
A/P Auto D/S Inv Setup
Determines if type 04 (direct ship) sales orders that are linked to direct ship purchase orders are automatically set up for invoicing when the vendor invoice is reconciled with the receiver:
•
Y - Automatically set up direct ship sales orders for invoicing when the vendor invoice is reconciled.
•
N - The direct ship sales order must be manually invoiced.
A/P Auto EDI Recost
Determines if the line item cost on a purchase order is automatically re-costed if the incoming EDI vendor invoice has a lower cost for the line item:
•
Y - Automatically recost the purchase order if the vendor invoice is lower for a line item.
•
N - Do not recost the purchase order when the vendor invoice is lower.
Update Commissions
Determines how to update commission when cost is adjusted in Commission Charges Entry:
•
C - Create a new commission amount for the additional charge.
•
A - Delete the original commission amount and create a new commission amount to include the additional charge. The new invoice date is used.
Warning: If commission records are already purged, new commission records are created for the full commission amount, which overpays commissions.
Refer to ”Adding Charges to an Invoice” in the Commission Subsystem documentation.
Disc. G/L Prefix Default
Indicates which default prefix to use for discounts:
•
A - The accounts payable account prefix.
•
D - The first distribution account prefix.
Misc Detail G/L in APPO
Determines if a G/L account is required when a Misc Amount is entered in Vendor Invoice Reconciliation Entry:
•
Y - Opens the APPO Miscellaneous Amount Entry screen to require a G/L account when a Misc Amount is entered.
•
N - Do not require a G/L account.
Auto Sel Cred Bal Vend
Determines if vendors with a negative balance are included when you use F5-Auto Sel (from the A/P Payment Selection Header screen) when printing checks:
•
Y - Include vendors with a negative balance. After you use the Auto Select option, you must use F6-Manual Sel to add invoices to get the total to zero or greater. You cannot print checks if a vendor has a negative check total in the batch.
•
N - Exclude vendors with a negative balance.
Repl Cost Upd Direct Shp
Indicates if replacement cost is updated when you change the cost in Vendor Invoice Reconciliation Entry on products that are set up as direct ship only (Availability Type set to D in Product Maintenance > F5-Add Info):
•
Y - Update the cost for direct-ship products.
•
N - Do not update the cost for direct-ship-only products.
Invoice Margin
In Vendor Invoice Reconciliation Entry, any discrepancy between a vendor’s invoice and the received amount that is less than this amount will automatically post to the general ledger discrepancy account.
Max. Discrepancy Dollar
The maximum dollar amount that can post to the general ledger discrepancy account when reconciling a vendor invoice. If the difference between the vendor’s invoice and the received amount is more than this amount, you must specify the G/L account to post to.
Max. Discrepancy Percent
The maximum percent amount that can post to the general ledger discrepancy account when reconciling a vendor invoice. If the percent difference between the vendor’s invoice and the received amount is more than this amount, you must specify the G/L account to post to.
Print MICR on AP Checks
Determines if the bank account and routing number print on the checks in MICR (magnetic ink character recognition) format:
•
Y - Print bank information in MICR format (only available for Optio printing).
•
N - Do not print bank information in MICR format.
Create Sub Asset From APIE
Determines if a sub asset is created in Accounts Payable Manual Invoice Entry:
•
Y - Do not update the main asset. Create a sub asset for the add-on capital.
•
N - The main asset for the equipment number is updated with the dollars for add-on capital.
Post Manual Invoice Tax
Indicates if the tax amount for a manual invoice is automatically distributed to the A/P tax account in the general ledger table:
•
Y - Automatically distribute tax to the A/P tax account. The merchandise amount will be the distribution amount.
•
N - Do not post tax to the general ledger. The invoice amount in Accounts Payable Manual Invoice Entry will be the distribution amount, whether or not there is a sales tax.
F2-Save
Saves the accounts payable parameter information.

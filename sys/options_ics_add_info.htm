Inventory Add. Parameters screen
Menu Path: Systems > System Parameters > Maintenance > Option Parameters > Inventory Parameters > F5-Add. Info
Field and Function Key Descriptions
Field/Function Key
Description
Two Step Transfer Process
Determines if inventory at the receiving branch is updated when the transfer is shipped in Transfer Shipments Entry:
•
Y - Use the 2-step transfer process to eliminate the need to receive the transfer. The 2-step transfer reduces the time required to complete a transfer. However, shipping errors may not be caught. You must use Inventory Adjustment Entry to correct any errors.
•
N - Use use the regular 3-step transfer process (enter, ship, and receive).
Transfer Branch Hold
Determines if a transfer goes on hold when the available quantity of an item goes below the safety stock at the shipping warehouse:
•
Y - The transfer goes on Branch Manager Hold when it lowers the quantity of an item below the safety stock if the shipping warehouse is not in the replenishment path.
•
M - The transfer goes on Branch Manager Hold when it lowers the quantity of an item below the safety stock regardless of the replenishment path.
•
N - The transfer does not go on Branch Manager Hold when it lowers the stock of a product below safety.
Auto Transfer Generation
Determines if you can automatically generate a transfer in Sales Order Entry or Work Order Entry:
•
Y - Prompt to generate a transfer.
•
W - Automatically generate a transfer without prompting first.
•
N - Do not automatically generate transfers.
This flag only applies to items with a Supply Code of T (transfer) or R (emergency transfer), or replenishment method of T.
Your user ID must have Enter T/R from S/O in the User Identification Maintenance Additional Information screen to be able to automatically generate transfers.
Rec. Transfers By Buyer
Determines if the recommended stock transfers generated by Nightly Processing are separated by buyer on the Recommended Stock Transfer Report:
•
Y - Recommended stock transfers are separated by buyer. The buyers for transfers are set for each vendor in Vendor Maintenance, but can be overridden in Vendor Line Maintenance.
•
N - Recommended stock transfers are not separated by buyer.
Transfer Freight Prompt
Determines if the Freight Closeout screen should appear upon completion of a transfer shipment:
•
Y - The Freight Closeout screen appears upon completion of a transfer shipment to allow the shipping warehouse to determine the freight amount. If you are using freight matrices, you can override the calculated amount, if necessary.
•
N - The Freight Closeout screen does not appear upon completion of a transfer shipment.
Transfer Receipt Hdr Upd
Determines if you can change and update the Stock Transfer Header screen in Transfer Receipts Entry and Batch Transfer Receipts Entry or if you can only view the header screen:
•
Y - You can edit the Stock Transfer Header.
•
N - You can view, but not edit, the Stock Transfer Header.
Allow Overage In TRE
Determines if an item that is over-received in Transfer Receipts Entry or Batch Transfer Receipts Entry is allocated to a linked sales order. Choose from the following:
•
Y - The excess quantity is allocated to the linked sales order, and the quantity on the sales order is increased to the total received quantity.
•
N - The excess quantity is put into stock. The linked sales order only receives the quantity originally ordered.
Allow Over Ships on Xfers
Determines if an item that is over-received in Stock Receipts Entry or Batch Stock Receipts Entry is allocated to a linked stock transfer. Choose from the following:
•
Y - The excess quantity is allocated to the linked transfer and the quantity on the transfer is increased to the total received quantity.
•
N - The excess quantity is put into stock. The linked transfer only receives the quantity originally ordered.
Vend Claim Orig Invoice?
Determines if you must enter the original vendor invoice number to return an item in Vendor Claim Entry:
•
Y - An invoice number is required.
•
N - An invoice number is not required.
Upd List/Disc In SRE/APPO
Determines if list price is updated with gross price for a product when received in Stock Receipts Entry:
•
Y - Update list price with the gross price of a purchase order when a product is received. Also, if cost is updated in Vendor Invoice Reconciliation Entry, the list price is updated. If the P/O contains a discount, then the product replacement percent is updated with the discount percent if the basis for the replacement cost is list (retail). Do not use this option if Replacement Cost Update (Inventory Parameters screen) is set to Y.
•
N - Do not update list price with gross price when inventory is received.
Reverse Statistics Sequence
Determines if statistics display in reverse sequence (i.e., December to January instead of January to December) for the following screens:
•
Customer Maintenance > F9-Statistics.
•
Vendor Maintenance > F8-Statistics.
•
Vendor Line Maintenance > F6-Statistics.
•
Product Usage Maintenance.
•
Product Warehouse Maintenance > F5-Usage > F6-Summary.
•
Product Warehouse Inquiry > F5-Usage > F6-Summary.
•
Product Parameters Calc Skipped Audit Maintenance.
•
Buyer Queue Inquiry > Purchase Parameters Skipped Items > F5-Review > F5-Perform Maintenance.
•
Customer Contacts > F10-Action > Statistics.
Usage Update During SOE
Determines if usage is captured for product when it is entered on a sales order or when the order is invoiced:
•
Y - Update usage in Sales Order Entry or Credit Memo Entry.
•
N - Update usage when the order is invoiced.
VCRE Allow Nonnet Bin
Determines if you can ship from a non-nettable bin in Vendor Claim Reconciliation Entry:
•
Y - You can ship from a non-nettable bin in Vendor Claim Reconciliation Entry. This option will double commit the inventory until the vendor claim is updated.
•
N - You cannot ship from a non-nettable bin in Vendor Claim Reconciliation Entry.
Transfers To Recv Whse BQI
Determines where the Buyer Queue Inquiry is updated for transfers:
•
Y - Update the Buyer Queue Inquiry for the To warehouse when a stock transfer is generated.
•
N - Update the Buyer Queue Inquiry for the From warehouse when a stock transfer is generated.
Weight Required
Determines if weight is required in Stock Receipts Entry for any product that does not already contain a weight in the product record:
•
Y - You must enter a product’s weight in Stock Receipts Entry. If weight is required and a product does not have a weight, the Stock Receipt Weight Entry screen appears to allow you to enter weight. The weight is then stored for the product (Product Maintenance > Action Bar > Addtl Opts > Package Information) for the next time you receive this item.
•
N - Weight is not required in Stock Receipt Entry.
Refer to “Entering Weight When Receiving Stock” in the Inventory -Stock Receipts documentation.
UPC Required
Determines if a UPC (Universal Product Code) is required when a product is received in Stock Receipts Entry:
•
Y - A UPC is required in Stock Receipts Entry. If you receive a product that does not have a UPC, you must enter the UPC before you can complete the receiver or enter N to let the system assign the next UPC.
•
N - UPCs are not required.
Refer to “Receiving Products with UPCs” in the Inventory - Stock Receipts documentation.
Availability Flag
Determines if the availability in the Product Tagging Entry screen is based on the current warehouse or for all warehouses combined:
•
W - Display availability and on-order quantities for the selected shipping warehouse.
•
C - Display availability and on-order quantities for all warehouses.
Ship Whse Usage Update
Indicates where usage is updated when a product that is non-stock in the selling warehouse is a stock product in the shipping warehouse:
•
Y - Update usage in both the selling and shipping warehouses.
•
N - Update usage in the selling warehouse only.
Phys Inv. Expanded Count
Determines if an expanded count sheet is used to enter physical inventory counts:
•
Y - The Expanded Physical Entry screen displays for each product in the Physical Count Entry screen.
•
N - The Expanded Physical Entry screen is not available.
The expanded count screen allows you to enter multiple quantities or a series of package quantities for a product. The counts are rolled up and a single count is recorded for the product. Refer to “Entering Inventory Counts on an Expanded Screen” in the Physical Inventory chapter of the Inventory documentation.
Phys.Inv. Include Serial#
Determines if you can use serial numbers when you adjust inventory to reflect the physical inventory counts (tag method):
•
Y - Use serial numbers to adjust inventory.
•
N - Do not use serial numbers to adjust inventory.
Refer to ”Taking Physical Inventory Using Tags” in the Physical Inventory chapter of the Inventory documentation
Quality Inspection -AQL
Determines which quality inspection system is used:
•
Y - Activates the Quality Inspection Subsystem module available in the Order Processing module. These procedures allow you to record complaints and customer feedback regarding a defect on a product and to track solutions or corrective actions taken to fix the problem. Refer to the “Quality Inspection Subsystem” in the Order Processing documentation.
•
N - Activates the Quality Control Inspection Level procedure in the Inventory module. These procedures can be used if your warehouse requires an inspection for products received into your warehouse. Refer to “Requiring Quality Control Inspections in Receiving” (Inventory documentation).
Auto Shp Whse Correction
Determines if inventory is automatically adjusted at the shipping warehouse when an item is over-received in Transfer Receipts Entry or Batch Transfer Receipts Entry:
•
Y - Automatically adjust inventory at the shipping warehouse when an item on a transfer is over-received at the receiving warehouse. Lot controlled items are adjusted also.
•
N - Do not automatically adjust inventory. You must manually adjust inventory at the shipping warehouse.
Default Reason Code
If Auto Shp Whse Correction is set to Y, enter a Default Reason Code to assign when inventory is automatically adjusted. Refer to ”Establishing Inventory Adjustment Reason Codes” in the Inventory documentation.
Round Transfer To Pkg Qty
Determines if Nightly Processing rounds to package quantities on recommended stock transfers:
•
Y - Nightly Processing rounds to package quantities when recommending a stock transfer, regardless of turns and maximum quantity calculations.
•
N - Nightly Processing does not round to package quantities when recommending a stock transfer.
•
U - Nightly Processing always rounds up to the nearest whole quantity.
Transfer Consolidation
Determines if batch printing consolidates stock transfers with the same ship-to/ship-from warehouse combination, the same ship via, and the same transfer type:
•
Y - Transfers are consolidated during batch printing. A separate line is created for each line item on the first transfer. Linked sales orders are updated with the new transfer number. This only applies for transfer types 01, 02, and 05.
•
N - Transfers are not consolidated during batch printing.
This flag does not apply to:
•
Warehousing stock transfers.
•
Transfers that use continuous printing.
•
Transfers that have the Unusual Usage Flag set to Y.
•
Transfers that have a future ship date that is different that the other transfers.
A separate line item is created on the transfer if:
•
Ship Complete for the line is set to Y.
•
Emg? is set differently for a line item.
•
Identical line items have different costs.
•
Linked items exists.
Alloc Stock To N/S Items
Indicates if you want to allocate stock to line items on sales orders, transfers, or work orders that have a B supply code and no linkage:
•
Y - Allocate stock to items if the line item has a B supply code and no linkage.
•
N - Do not allocate stock to items with a B supply code and no linkage. You must manually allocate backorders. Refer to ”Allocating Inventory Manually” in the Inventory documentation.
Prorate Transfer Freight
Determines if freight on a partially received transfer is pro-rated by the number of products received:
•
Y - Pro-rate the freight according to the transfer receipt quantity. The freight for each product is allocated according to the percentage set in Product Maintenance (refer to the Freight Information Screen).
•
N - Allocate all freight to the first receipt of the transfer.
F2-Save
Saves the additional inventory parameter information and returns to the Inventory Parameters screen.

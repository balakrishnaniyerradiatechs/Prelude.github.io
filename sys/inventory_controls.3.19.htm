Bin Locator Options screen
The Bin Locator Options screen allows you to set default bin information and flags for cycle counting. Refer to “Cycle Counting” in the Inventory documentation for more information on setting up your system for cycle counting.
Menu Path: Systems > System Parameters > Maintenance > Inventory Controls > Warehouse Maintenance > Action Bar > Addl Opts > Bin Locator Information
Field and Function Key Descriptions
Field/Function Key
Description
Multiple Bin Locations
Determines if a product may exist in more than one bin:
•
Y - A product may exist in multiple bin locations. The system tracks the quantity on-hand in each bin.
•
N - A product can only exist in one bin.
This flag overrides the setting at the system level (Multiple Bin Locations flag in the Inventory Parameters screen).
Default Receiving Dock Bin
The default bin for stock received into your warehouse:
•
Any valid bin number.
•
PRIMARY - Uses the primary bin assigned to each part number.
Default Linkage Receipt Bin
The default bin for products received that are linked to sales orders:
•
Any valid bin number.
•
PRIMARY - Uses the primary bin assigned to each part number.
Default Inspection Bin
The default bin for stock and transfer receipts that require an inspection.
Default Return Bin
The default bin for sales orders that are returned from the customer:
•
Any valid bin number.
•
PRIMARY - Uses the primary bin assigned to each part number.
You can override the bin number in credit memo entry.
If you use the bin locator system, all credit lines default to this location.
If you also use the warehousing module, all products entered in credit memo entry default to this bin.
Default WO Completion Bin
The default bin for work order completions:
•
Any valid bin number.
•
PRIMARY - Uses the primary bin assigned to each part number.
You can override the bin number in work order completions.
If you use the bin locator system, all work order completions default to this location.
If you also use the warehousing module, all products entered in work order completions default to this bin.
Default Vendor Claim Bin
The default bin for vendor claims:
•
Any valid bin number.
•
PRIMARY - Uses the primary bin assigned to each part number.
You can overrode this bin in Vendor Claim Bin Allocation.
Default On The Water Bin
The default receiving bin for on-the-water purchase orders:
•
Any valid bin number.
•
PRIMARY - Uses the primary bin assigned to each part number.
You can override this bin in Stock Receipts Entry.
Default S/T Recv Dock Bin
The default receiving bin for stock transfers:
•
Any valid bin number.
•
PRIMARY - Uses the primary bin assigned to each part number.
RF Physical Tag Method
Determines how the next tag number is generated in RF Physical Tag Entry:
•
O - Increments by one tag number after each count.
•
N - Increments to the next tag number based on the next NEW tag number.
Maintenance Cycle Counting
Cycle Count Days
The estimated number of days this warehouse will count bins. The default is 200 days.
Update Bin Count 2nd Levl Queue
Determines if the bin count updates all of the YTD Bin Counts:
•
Y - The bin count updates YTD Bin Counts for every bin on the list in the 2nd level cycle count entry.
•
N - The bin count does not update YTD Bin Counts for every bin on the list in the 2nd level cycle count entry. Only the bins with counts are updated.
Update 1st Level Adj As Rejects?
Determines if the product is marked as a reject in the first level of cycle counting:
•
Y or Null - A product is marked as a reject in the first level queue if the counted quantity differs from the on-hand, but it is within the tolerances allowed in Cost Tolerance Maintenance (Inventory > Physical Inventory > Cycle Counting Procedure). It does not go to the second level queue. Inventory is adjusted to correct the on-hand inventory. The Bin Count Statistics Report (Inventory > Physical Inventory > Cycle Counting Procedure) displays these products in the Prd Rej column.
•
N - The product is not marked as a reject if the counted quantity is within the cost tolerance limits. Inventory is still adjusted but it does not show up in the Prd Rej column on the Bin Count Statistics Report.
Default P/N In 2nd Level Entry?
Determines if the product number is defaulted in the Product Number field when 2nd level counts are entered:
•
Y - A product number defaults in Part Number in the second level count entry.
•
N - A product number does not default in the second level of count entry. You must manually enter the part number.
Refer to ”Entering Second Level Cycle Counts” in the Cycle Counting chapter in the Inventory documentation
Update /ECCI Queue?
Determines if you use the Emergency Cycle Count Inquiry in the Warehousing module:
•
Y - Activate the Emergency Cycle Count Inquiry.
•
N - Disallows the Emergency Cycle Count Inquiry.
Exclude Non-Net Bins 2nd Level
Determines if the non-nettable secondary bins are included in the list of bins for the product for the second level count:
•
Y - The non-nettable bins are excluded from the list of bins for the product for second level counts.
•
N - The non-nettable bins are included in list of bins for the product for second level counts.
A non-nettable product is part of on-hand inventory but is not available to sell until it is moved into a nettable bin.
Scan First Level Counts
If you use RF devices, indicates if products are scanned into the system for first level counts or if the total count is entered after all products are counted:
•
Y - Products are scanned in the first count. If you find additional quantities of the product in other locations, you can keep scanning the additional quantities to add to the count.
•
N - The total count of the product is entered at one time. The count you enter overrides any count already entered.
Refer to ”Entering 1st Level RF Cycle Counts” in the Cycle Count Chapter in the Inventory documentation.
Cycle Count Auto Update
Indicates if products from a bin are automatically updated during the first level cycle count, if the items were not initially physically counted:
•
Y - Products not counted that have an on-hand balance will go to the second level count queue. If the product has a zero on-hand, it is updated as zero and not sent to the second level count queue.
•
N - If items in the bin are skipped during counting, you are warned and prompted to fill in counts for the remaining items.
2nd Level Bin Sorting
Determines the sort order for the bins in the 2nd level cycle count:
•
S - Sorts the problem bin first in the 2nd level cycle queue bin list.
•
B - Sorts the 2nd level cycle queue by ascending bin number.
F2-Save
Saves the bin locator information.
F5-Default Receiving Bin Range
Opens the Receiving Default Bins screen.
F6-Other Bins
Opens the Other Bin Information screen.

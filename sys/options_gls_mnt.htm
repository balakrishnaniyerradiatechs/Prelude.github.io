General Ledger Parameters screen
The General Ledger Parameters screen allows you to set flag and option parameters to control how Prelude operates for G/L functions.
Menu Path: Systems > System Parameters > Maintenance > Option Parameters > General Ledger Parameters
Field and Function Key Descriptions
Field/Function Key
Description
Multiple Currency
Indicates if your company accepts financial transactions in multiple currencies:
•
Y - Activate the use of multiple currencies.
•
N - All transactions are posted in your native currency.
Refer to “Using Multiple Currency” in the General Ledger documentation for information on setting up multiple currency.
Currency Method
Determines where the currency exchange rate comes from:
•
S - Use the exchange rate from Currency Type Maintenance (General Ledger Controls > Currency Maintenance > Currency Type Maintenance). General ledger accounts and exchange rates are assigned in Currency Type Maintenance for each type of currency other than native currency.
•
D - Use the exchange rate from Daily Currency Exchange Maintenance (General Ledger Controls > Currency Maintenance > Daily Currency Exchange Rate Maintenance). The most recent exchange rate is used. However, if no data exists for the past 9 days, the exchange rate from Currency Type Maintenance applies.
Auto Calc Currency Clng Amt?
Sets the calculation method for posting to the currency clearing account in Accounts Payable Manual Invoice Entry:
•
Y - Use the currency clearing account from Currency Type Maintenance to calculate the currency clearing amount (General Ledger Controls > Currency Maintenance > Currency Type Maintenance).
•
N - Manually calculate the currency clearing amount.
Repl Cost in Vend Currency?
Determines how replacement cost displays on a purchase order:
•
Y - Show replacement cost in the vendor’s currency.
•
N - Convert replacement cost to the native currency on the purchase order.
Display Currency Rate Screen?
Determine if you can change the currency rate in any screens that prompt for a currency:
•
Y - The currency rate screen appears so you can change the currency rate.
•
N - The currency rate screen does not appear.
Re-eval Invoice Currency?
Determines if the currency rate is re-evaluated when an order is invoiced:
•
Y - Re-price sales orders using the currency rate that is in effect at the time of invoicing.
•
N - Do not re-price a sales order when it is invoiced.
Bal G/L Posting By Location?
Determines if posted transactions balance by location during the G/L Balances Posting Routine:
•
Y - The location must balance before posting the transaction. If the General Ledger posting does not balance by location, an audit file is updated. The ”Unposted G/L Error Report” lists transactions that did not balance.
•
N - The balance posting routine verifies that the debits and credits balance but does not check that each location is in balance. The ”Unposted G/L Error Report” lists transaction information for reconciliation.
Refer to ”Balancing by Location” in the General ledger documentation.
Automatic G/L Posting?
 
Determines when transactions post to the general ledger:
•
Y - Opens the Automatic G/L Posting Module Maintenance screen to allow you to enter the G/L modules that are automatically posted to the general ledger as each transaction is processed. You cannot use this option if Multiple Fiscal Periods (Company Maintenance > F5-Add Info) is set to Y.
•
A - Audit mode only. Checks for any potential out-of-balance postings, marks the general ledger posting record, and sends a message to the address in G/L Email Address so that you can correct out-of-balance conditions before posting to the general ledger. A report generates but nothing is posted.
•
N - Transactions are not automatically posted to the General Ledger. You must use the G/L Balances Posting Routine to post transactions from each module to the general ledger.
Refer to “Posting to the General Ledger” in the General Ledger documentation.
G/L Email Address
If you entered Y or A for Automatic G/L Posting? enter the email address for the person to notify of an out-of-balance condition.
Suppress I/C Postings?
Determines if intercompany postings are suppressed for general ledger accounts that are hard coded with a prefix/account in the General Ledger Table Maintenance screen:
•
Y - Do not post to intercompany general ledger accounts that are hard coded with a prefix/account in the General Ledger Table Maintenance screen.
Bal G/L Posting By Location? must be set to N.
•
N - Allow intercompany postings for general ledger accounts hard coded with a prefix/account in the General Ledger Table Maintenance screen.
The following intercompany flags are hard-coded in the general ledger table:
•
Intercompany A/R Clearing.
•
Intercompany A/P Clearing.
•
Transfer Inventory In.
•
Transfer Inventory Out.
•
Intercompany Sales A/R.
•
Intercompany Sales A/P.
F2-Save
Saves the general ledger parameter information.

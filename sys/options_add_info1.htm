Additional Options screen
Menu Path: Systems > System Parameters > Maintenance > Option Parameters > Order Processing Parameters > F5-Add. Info > F5-Add Info
Field and Function Key Descriptions
Field/Function Key
Description
Rate Shop Interface
Indicates the rate shopping method:
•
A - ADS. Prelude’s internal method of rate shopping based on weight and method of delivery. The best rate is calculated in Sales Order Entry using the best estimate based on weights in Product Maintenance (Sales Order Entry > Action Bar > Order Func > Rate Shop).
•
P - PSS (Parcel Shipping System). A third party manifesting system from Software Co-Op, Inc.
Suppress Order Ship Via/Frt Upd In RCE
Indicates if you can update the ship via code and the freight terms when you confirm a shipment in Release Confirmation Entry:
•
Y - If you change the ship via and freight terms on a sales order in Release Confirmation Entry, the changes will not update the sales order.
•
N - If you change the ship via and the freight terms when confirming an order in Release Confirmation Entry, the changes update the sales order and any future releases.
•
S - If you change the ship via and the freight terms when confirming an order in Release Confirmation Entry, only the freight terms are updated.
Suppress COD Terms Code Msg In SOE
Determines if a warning message appears when sales orders are entered for customers with a COD (collect on delivery) terms code:
•
Y - Sales Order Entry will not display a warning message when a sales orders with COD terms is entered.
•
N - Sales Order Entry displays a warning message when a sales orders with a COD terms code is entered.
Oversize Order Value
Defines the amount above which sales orders go on credit hold. For example, enter 50,000 if orders over $50,000 should go on credit hold. The system uses the following hierarchy to determine the oversize order value that puts an order on credit hold:
•
Customer Maintenance > F6-A/R > Oversize Order Value.
•
Warehouse Maintenance > F5-Add info > Oversize Order Value.
•
Order Processing Parameters > F5-Add Info > F5-Add Info > Oversize Order Value.
Maximum Order Discount
The maximum discount for Disc/Allowance on the Sales Order Completion and the Quotation Completion screens. If this field is blank, then any Disc/Allowance can be entered.
You can override this value for a user ID in Max Order Disc in the Administrative Settings screen.
Check For Existing Orders
Determines if a message appears to allow you to add to an open order for a customer before you create a new order for the same customer. This feature only applies to type 01 sales orders that are going to the same ship-to address and for which the customer does not require a purchase order:
•
Y - When you save the Sales Order Header screen, a message appears to allow you to add to non-released orders to the same customer ship-to. If you select Yes, the first type 01 non-released sales order opens to allow you to add products.
•
P - The same as Y except that the system only searches for orders that have a pickable quantity.
•
N - No message displays if open orders exist.
Price Override Reason Prompt
Determines if a price override reason code is required when a price on a stocking item on a sales order or quote is changed:
•
Y - A price override reason code is required when a price is changed so that you can track price overrides.
•
N - A price override reason code is not required when a price is changed.
Price Override Reason Codes are set up in Order Processing > Maintenance > Misc Codes Files > Price Override Reason Codes.
Price Override Code in the User ID Maintenance Additional Information screen determines if a user can override a price on a sales order.
Line By Line Conversion In QOE
Indicates if you can convert a quote to a sales order by line item, from Quote Order Entry:
•
Y - Upon completion of a quote, you can choose Order to convert individual items to a sales order.
•
N - Upon completion of a quote, you can choose Order to convert the entire quote to a sales order.
Display Quote/Meter Info in SVOE
Determines if the Quote/Meter Information screen appears in Service Order Entry to allow you to enter labor, parts, and meter information:
•
Y - The Quote/Meter screen appears in Service Order Entry.
•
N - The Quote/Meter screen does not appear in Service Order Entry.
Display Open Orders Inquiry In SVOE
Determines if open service orders display after you enter the system ID in Service Order Entry:
•
Y - Any existing open service orders appear after you enter the system ID in Service Order Entry.
•
N - Open service orders do not appear in Service Order Entry.
Print Service Documents
Determines if you can print service documents upon completion of a service order:
•
Y - You can print the service document after you complete the order. You must set this flag to Y to use continuous printing.
•
N - You cannot print the service document.
Promotional Pricing
Determines if promotional price matrices are allowed:
•
Y - You can create promotional price matrices.
•
N - Promotional pricing is not allowed.
Price promotions allow you to give customers discounts or free products. You can give a discount based on an order total or give away free products based on a purchased product (e.g., buy one, get one free). You can also include or exclude customers from a promotion. Refer to “Promotional Price Matrices” in the Order Processing documentation.
QFUI Dollar Limit
The amount that determines if a quote appears in the Quote Follow Up Inquiry. Only quotes above this amount are included in the inquiry. For example, enter 15 so that only quotes of $15.01 and higher appear in the Quote Follow Up inquiry.
Pick Ticket Minimum Dollar
The minimum dollar amount required to print a pick ticket. If the total pickable value (pick qty x price for each line item) is greater than this amount, the sales order is released. If the total pickable value for each warehouse (pick qty x price for each line item) is less than this amount, the sales order is not released. This field works with the Pick Ticket Minimum Percent.
Pick Ticket Minimum Percent
The percent of merchandise that must be available before the pick ticket will print. If the percent calculated from the total pickable amount (pick qty x price for each line item) divided by the total on the sales order is greater than this value, the sales order is released. If the percentage calculated from the total pickable amount (pick qty x price for each line item) divided by the grand total on the sales order is less than this value, the order is not released. This field works with the Pick Ticket Minimum Dollar. If both fields have a value and one of the conditions is met, the sales order is released.
If you enter a single pick ticket number in batch pick ticket printing, the system does not check for minimum dollar values or percents. The release prints even if it is below the minimum.
Handling Charge For Pickup Orders
Indicates if handling charges apply to picked up orders:
•
Y - Add a handling charge to orders that are picked up by the customer.
•
N - Do not add a handling charge to orders that are picked up by the customer.
Shipping Whse Split/Reassignment
Determines if the Auto Order Split screen appears in Sales Order Entry if a backordered quantity exists in another warehouse.
•
Y - If the quantity is available in another warehouse, automatically prompt with the Auto Order Split screen in Sales Order Entry to let the user decide if they want to ship part of the quantity from other warehouses.
•
N - Do not show the Auto Order Split screen or allocate any inventory from another warehouse. The product will be backordered.
•
A - Automatically ship part of the line item from a warehouse where it is available (does not apply to kit products).
You can override this setting for a warehouse in Multiple Whse In S/O in the Warehouse Maintenance Additional Information screen.
Refer to “Shipping Split Quantities out of Other Warehouses” in the Order Processing documentation for more information.
Shipping Whse Consolidation
Determines if the Shipping Consolidation screen appears upon completion of a sales order to allow you to consolidate as many lines as possible from a single warehouse.
•
N - Do not show the Shipping Consolidation screen in Sales Order Entry.
•
S - Show the Shipping Consolidation screen in Sales Order Entry. You can then use F7-Consolidate to consolidate as many lines as possible to a single warehouse.
•
A - Automatically consolidate the product and display the summary.
If at least five lines can be filled using the default shipping warehouse, the shipping warehouse for the line items is used instead consolidating the remaining lines from other warehouses.
Geographic Prod Restrict
Determines if Sales Order Entry checks for a Restricted Product Matrix that prevents certain products from shipping to prohibited areas:
•
Y - Sales Order Entry checks for restricted product matrices.
•
N - Sales Order Entry does not check for restricted product matrices.
You can create a restricted product matrix to identify the products or product lines that are not allowed. Within each state, you can limit the restrictions by zip code or warehouse. The system checks the state where the product ships from to determine if the product is allowed. Refer to “Restricting Products for a State or Zip Code” in the Order Processing documentation for more information.
Invoice Days Old
The number of days old an invoice must be to activate a warning in Sales Order Entry/Credit Memo Entry for returned products. When you enter an invoice number that is more than this number of days old, the warning appears but you may complete the return. You could use this feature for products that have an expiration date.
Auto Core Charge?
Determines if a core charge is automatically billed in Sales Order Entry:
•
Y - The core charge is automatically applied to the sales order. The core is billed with the exchange product and so is not tracked.
•
N - The core charge is not automatically applied to the sales order. You can choose to bill the core charge, to not bill the core charge, or to apply a core bank item.
Refer to “Pending Cores” in the Inventory documentation for information on setting up and tracking core products.
Excl ‘*’ Items From MatPrc
Determines if asterisk (*) products are excluded from price matrix calculations:
•
Y - Asterisk items are excluded from price matrix calculations. Pricing from the product or warehouse applies.
•
N - Asterisk items are included in price matrix calculations.
Freight Handling
Determines how freight terms calculate freight in Sales Order Entry and Release Confirmation Entry:
•
Y - Skips Inbound Freight and Outbound Freight in the Sales Order Completion screen and in Release Confirmation Entry if the freight terms code assigned to the sales order has Inbound Freight and Outbound Freight set to N. The freight charge is automatically calculated according to the freight terms.
•
N - Allows you to enter Inbound Freight and Outbound Freight in both the Sales Order Completion screen and in Release Confirmation Entry, to override the system calculated freight no matter how the freight terms are set.
•
A - Works the same as N except that the freight amount in the release header is only carried to the completion screen if the freight terms codes has Inbound Freight and Outbound Freight set to Y. This option allows you to enter freight without determining if the freight should be billed to the customer. The system calculates freight charges according to the freight terms.
Ord Tot Based on Pck Qty?
Determines how the total is calculated for orders with credit card payment terms:
•
Y - Displays the total based on the quantity shipped.
•
N - Displays the total of all products ordered, regardless of backordered items.
Soe Restock Charge Method
Determines if restocking charges are added to returned goods:
•
S (or blank) - Only add restocking charges to material returned to stock.
•
A - Add restocking charges to all types of returned goods.
Restock Charge % in the Company Maintenance Additional Information screen determines the restocking charge percent to use to calculate a restocking charge on credit memos and RGAs.
Restocking Charge (Customer Maintenance > F7 > F5) determines if a restocking charge is calculated as each line item is entered on a credit memo.
Qte Line Conversion in SOE
Determines if you can convert quote line items to the sales order if the product exists on an open quotation:
•
Y - You can convert existing quote line items to the sales order.
Display Open Quote/RGA Message (Order Processing Parameters) must be set to Y and Open Quote Message (Order Processing Parameters > F5) must be set to L or B for this flag to work.
Quote Message All Warehouses (Order Processing Parameters > F5 > F5> F5) determines if all warehouses are included in the list of options available for conversion.
•
N - A message displays to let you know that quotations exists but does not allow you to convert the quote onto the sales order.
Override Contact Name
Determines if Contact, Phone, and Email Address information from a customer ship-to is used on sales orders and quotes:
•
Y - The Contact, Phone, and Email Address information from a customer ship-to record is used on sales orders and quotes. If the ship-to number is changed on a sales order, the information on the sales order changes to the ship-to. If a ship-to does not exist, the information is pulled from the main customer record.
•
N - The Contact, Phone, and Email Address from a customer ship-to is not used on sales orders and quotes. If the ship-to number is changed on a sales order, the information on the sales order will not change.
Print B/O Staging Docs
Determines if backorders print on the first staging document:
•
Y - Show backorders on the first staging document.
•
N - Do not show backorders on the first staging document.
Refer to “Entering Tag & Hold Sales Orders” in the Order Processing documentation for more information.
Post O/B Frt to Shp Whse?
Indicates if Outbound Freight entered in Release Confirmation Entry to bill to the customer posts to the general ledger of the shipping warehouse:
•
Y - Post Outbound Freight to the shipping warehouse.
•
N - The entire amount is income for the selling warehouse.
Ovr Shp Whse Cons Ord?
Determines if the line item shipping warehouse for an 08 consignment order should be the same as the selling warehouse:
•
Y - If the selling and shipping warehouses are different, an 08 consignment order created at invoicing is updated with the selling warehouse for the line items. This allows the G/L and perpetual inventory to be maintained at the selling location. This flag should only be used if the Default Consign Accum Flag in the Default Sort Sequence screen is set to C or A.
•
N - The shipping and selling warehouse do not have to be the same.
Refer to “Using Consignment Orders” in the Order Processing documentation.
P.O. Box Removal
Determines if the PO Box address line is included on the shipping address:
•
Y - Remove the PO Box address line from the shipping address since most carriers do not ship to a PO Box.
•
N - Include the PO Box address line as part of the shipping address.
Prompt For Disc/Allow Reas
Determines if you must enter a discount reason code if you enter a discount amount in Disc/Allowance in the Sales Order Completion screen or the Quotation Completion screen:
•
Y - Require a discount reason code when a discount is entered in the Sales Order Completion screen. Discount reason codes are created in Order Processing > Maintenance > Misc Code Files > Discount Reason Code Maintenance.
•
N - A discount reason code is not required when a discount is entered in the Sales Order Completion screen.
Maximum LDT Commit Days
The maximum number of lead time days that inventory is allowed to be committed on a sales order with a future pick ticket date.
For example, if Maximum LDT Commit Days is set to 5 and you change the Pck-Date on the Line Item Detail screen to a date that is 10 days in the future, the following occurs on the sales order’s line item detail screen:
•
The Qty-Pck changes to zero.
•
The quantity ordered appears in Qty-B/O.
•
The Future Commit (unlabeled field before Qty-Pck) changes to P.
•
5 days before the pick ticket required date, inventory is allocated to the order.
Update Ship Qty / 05 Orders
Determines if the ship quantity on a type 05 (counter invoice) sales order is updated when a backorder is received and allocated to the order:
•
Y - Update the ship quantity when a backorder is received and allocated. If counter invoices print continuously, once the backorder is received and allocated, you must manually go back into the sales order and set it up for invoicing.
•
N - The ship quantity is not updated when the backorder is received and allocated.
Precost Kit From
Rollup
Determines how the cost displays for kit items on sales orders:
•
Y - Automatically calculate the cost of the kit parent based on the cost of the components before entering the Line Item Detail screen and creating the kit. The prices/costs only display when a kit parent is entered as a new line item on a sales order or quote. Once the kit entry screen is saved and the components are changed, the price/costs are totalled accordingly.
If the kit product has S/O Modify set to T (Product Maintenance > F6 > Kit Parent = Y), this flag does not apply. When you enter products using the product tagging screen, no prices/costs are rolled up since components are not yet selected.
•
N - Do not roll up prices/costs from the components when the kit parent item is entered. Prices/costs are rolled up after saving the kit entry screen.
Suspend Orders From Activities
Determines if suspended sales orders or quotes can go on credit hold or administrative hold:
•
Y - Do not place a suspended order or quote on hold, even if it meets the criteria for credit hold or administrative hold. It may still go on hold when the order is no longer suspended.
For direct sales orders, choose Finish to save the direct ship order and not suspend it.
•
N - If a sales order or quote meets the criteria for credit hold or administrative hold, it will go on hold even if it is suspended.
On direct sales orders, you will not get the Finish option but you will be able to send an acknowledgement, even if it is on hold.
F2-Save
Saves the information on the Additional Options screen and returns to the Additional Information screen.
F5-Add More Info.
Opens the More Additional Options screen to allow you to access more order processing option parameters.

File Update Parameters screen
The File Update Parameters screen allows you to set flag and option parameters to control how Prelude updates files and tracks information used for reports and statistics. If any of these reports are not used, turn the flag to N to conserve space and time for processing. You can also use the purge menu to purge saved information from any of these files.
Menu Path: Systems > System Parameters > Maintenance > Option Parameters > File Update Parameters
Field and Function Key Descriptions
Field/Function Key
Description
Lost Sales
Determines if a lost sales reason code is required when a product is deleted from a sales order:
•
Y - A lost sales reason code is required when a product is deleted.
•
N - A lost sales reason code is not required when a product is deleted.
Refer to “Tracking Lost Sales” in the Order Processing documentation.
Pick Error
Determines if pick error information is captured:
•
Y - A pick quantity in Release Confirmation Entry that is different than the quantity indicated on the pick ticket generates a error.
•
N - Pick errors are not tracked.
The Pick Error report in the Order Processing module lists picking errors by warehouse.
Product Catalog
Determines if the product catalog file is maintained:
•
Y - Store product records in both the product file and the product catalog file. This option opens the Catalog Lookup Window Method screen to allow you to determine if you want to separate or combine keyword searches for inventory items and catalog items.
•
N - Only store product records in the product file.
The product file is for normal processing and the product catalog file is for products stored for reference and pricing information. Using the product catalog file allows you to maintain product catalog records without slowing down product file processes. Once a product in the product catalog file is sold, it is moved into the product file. The ”N/S Product Merged Report & Process” in the Order Processing documentation displays the products moved into the product file and allows you to move them back to the catalog file.
If you change this flag setting, you must run the product keyword Xref rebuild to update the change.
Bookings
Determines if bookings information is captured:
•
Y - Keep bookings information whenever an order is entered, changed, or deleted. The Bookings Options screen appears to allow you to determine if booking information is kept for Manufacturer Rep Sales, Quotations, or MITS (3rd party software).
•
N - Do not keep bookings information.
Bookings are the extended amount for which you obtained orders, but are not necessarily invoiced. This information is helpful for companies with long lead times to help view business trends.
Commissions
Determines if commission information is captured when a sales order goes through the B Update process:
•
Y - Update commission information when the sales order is updated in the B Update.
•
N - Do not update commission information during the B Update.
Refer to ”Updating Sales Statistics” in the Commission Subsystem Documentation.
A/R Commissions
Indicates if commission information is updated when the customer pays their invoice:
•
Y - Update commission information when the customer pays their invoice.
•
N - Do not update commission when the customer pays their invoice.
Refer to ”Updating and Paying Commission” in the Commission Subsystem documentation.
Credit Authorization
Determines credit authorization history is captured:
•
Y - Keep credit authorization information (orders released from credit hold). The data appears on the Credit Authorization Report (Order Entry module).
•
N - Do not keep credit authorization information.
Credit Statistics
Determines if credit status history information is captured:
•
Y - Keep credit status information. The “Daily Credit Status Change Report” (Accounts Receivable module) lists all customers who have been put on or taken off auto credit hold or have had their credit code, credit limit, or terms code changed.
•
N - Do not keep credit status information.
Credit Mgr. Release
Determines if credit release information is captured for a credit manager:
•
Y - Keep credit manager release information. The “Credit Manager Release Report ” (Accounts Receivable module) lists information for orders released by a credit manager.
•
N - Do not keep credit manager release information.
Admin Mgr. Release
Determines if administrative order, quote, or invoice hold release information is captured:
•
Y - Keep administrative manager release information. The “Administrative Manager Release Report” (Accounts Receivable documentation) lists all sales orders or quotes released from Administrative Order Hold, Administrative Quote Hold, or Administrative Invoice hold for an Administrative Manager ID.
•
N - Do not keep administrative manager release information.
Customer Job
Determines if customer job information is updated whenever a job record is created and referenced in Sales Order Entry:
•
Y - Maintain customer job information.
•
N - Do not maintain customer job information.
DBS Statistics
Determines if Daily Business Statistics information (e.g., sales, invoiced amounts, inventory value, accounts receivable, and accounts payable information) is captured for the Daily Business Statistics report (Order Processing module):
•
Y - Opens the Daily Business Update Parameters screen to allow you to enter the following information:
A/P Aging Days Interval: The number of days for aging accounts payable (the length of the A/P period).
A/R Aging Days Interval: The number of days for aging accounts receivable (the length of the A/R period).
A/R Write-Off G/L Acct: The general ledger account to record miscellaneous adjustments in Cash Receipts Entry for bad debt write-offs. The account description displays for reference.
Days Back for History: The number of previous days to select for order history to update merchandise and gross profit amounts on the Daily Business Statistics Report.
Bank Account G/L Acct: The main bank general ledger account number. The balance of the account prints on the Daily Business Statistics report below the total amount of Accounts Payable so you can compare liabilities to cash on-hand.
•
N - Daily business statistics are not captured.
Surplus Audit
Determines if audit information is captured each time a buyer purchases inventory when surplus inventory for the same product exists in another warehouse:
•
Y - Capture surplus purchase information. The Surplus Audit report (Purchasing module) displays surplus information to determine if products were purchased when surplus inventory for these products existed in other locations. The report shows the quantity on-hand and the surplus quantities at the warehouse that contained the surplus inventory at the time the products were purchased.
•
N - Do not capture surplus purchase information.
Fill Rate
Determines if fill rate (percent of order line items that are filled) information is captured:
•
Y - Capture fill rate information. The Fill Rate reports (Purchasing module) display the number and percent of stock items that were available when sales orders were first entered (not when invoiced).
•
N - Do not capture fill rate information.
Transfer Fill Rate
Determines if transfer fill rate information is tracked:
•
Y - Capture transfer fill rate information. The Transfer Fill Rate Detail report (Inventory module) displays the percent of transfer line items that are 100% filled before shipping from a warehouse.
•
N - Do not capture transfer fill rate information.
Contact Detail
If you use the Contact Diary Inquiry and the Contact Diary Report (Marketing module), indicate if data is tracked for the contact diary:
•
Y - Track data for the contact diary. The Contact Detail Sources screen opens to allow you to indicate if you want to track information for quotes, sales orders, shipment processing, campaign letters, work order completions, marketing activities, or sample requests.
•
N - Do not track data for the contact diary.
Order Pad
Determines if the order pad information is tracked:
•
Y - Activates the Order Pad Inquiry (Order Processing module) to allow you to search for products purchased by a customer.
•
N - Do not track order pad information.
Order Log
Determines if order log detail information is tracked:
•
Y - Activates the Order Log Inquiry (Order Processing module), which lists all transactions that affected a sales order. Each transaction is preceded by a Code that defines the change or process that affected the quote. The date, time, and user who entered the transaction display also.
•
N - Do not track order log detail information.
Neg. Avail Override
Determines if information is captured when a pick quantity is captured causing the available quantity of the product to become negative:
•
Y - Track negative availability information. The Negative Availability Override report (Order Processing module) lists sales order line items on which the pick quantity was overridden, causing the available quantity to be negative.
•
N - Do not track negative override information.
Work Order Activity
Determines if Work Order Activity is tracked:
•
Y - Track work order activity.
•
N - Do not track work order activity.
Work Order Status
Determines if work order status changes are tracked:
•
Y - Track work order status changes.
•
N - Do not track work order status changes.
Expected Date Change
Determines if expected date changes on purchase orders are tracked:
•
Y - Track changes to the purchase order delivery date.
•
N - Do not track changes to the purchase order delivery date.
Universal Inquiry
Determines if historical cross reference information is maintained for sales orders, purchase orders, work orders, and quotes for the Universal Inquiry (Order Processing module):
•
Y - Maintain cross reference information for sales orders, purchase orders, and work orders for the Universal Inquiry.
•
N - Do not maintain cross reference information for the Universal Inquiry.
Ship-To Keywords
Determines if customer ship-to keywords are used for customer keyword look-ups:
•
Y - Maintain customer ship-to keywords.
•
N - Do not maintain customer ship-to keywords.
Ovr Ship Whse Matrix
Determines if you can create a shipping matrix to override shipping warehouses for specific products:
•
Y - Activates the Override Shipping Whse Matrix (Order Processing module) to allow you to set up shipping matrices so that a product or group of products must always ship out of a designated warehouse.
•
N - Overriding shipping warehouses is not allowed.
Refer to “Overriding Shipping Locations for Products” in the Order Processing documentation.
Putaway Document
Indicates if putaway documents print automatically after each receiver is saved and if the information is tracked:
•
Y - Prints the Putaway Documents Report automatically after completing a stock receipt.
•
A - Prints the Putaway Documents Report after receiving stock and keeps an audit trail of the allocated material.
•
N - Does not print the Putaway Documents Report after receiving stock.
Pending Cores
Determines if you want to track core items to report pending core changes:
•
Y - Track core items to report pending core changes so you can use the pending core procedures. This option allows you to hold off billing the core charge to keep the A/R and gross profit from inflating.
•
N - Do not track pending core changes. You might choose this option if you are only core banking and applying banked cores against new sales in Sales Order Entry.
Refer to “Pending Cores” in the Inventory documentation for more information.
Late Orders
Indicates if you want to track sales orders that shipped after the Required Date:
•
Y - Track late sales orders. The Late Sales Order Report (Order Processing module) lists sales orders that shipped after the Required Date.
•
N - Do not track late sales orders.
Primary Vendor P/O
Determines if information is captured when you purchase products from a non-primary vendor:
•
Y - Track products purchased from a non-primary vendors. The data appears on the “Vendor Exception report” (Purchasing module).
•
N - Do not track primary vendor exception information.
Vendor Cost Audit
Determines if changes to vendor costs (e.g., unit cost, replacement cost, direct ship cost) are tracked:
•
Track changes to vendor cost.
•
Do not track changes to vendor cost.
You can create a customer report to view changes to cost.
Product Maint Audit
Indicates if changes in Product Maintenance are tracked:
•
Y - Track changes to a product or product/warehouse. The Product Maintenance Audit Report by Product and the Product Maintenance Audit Report by User ID (Inventory documentation) list changes in any of the Product Maintenance screens.
•
N - Do not track changes to products.

Determines if changes to a customer record are tracked:
•
Y - Track changes in Customer Maintenance. The data appears on the Customer Maintenance Audit Report by User ID and the Customer Maint Audit Report by Customer (Order Processing module).

N - Do not track changes to a customer.
Contact Maint Audit

•
Y - Track Contact Maintenance information. The data appears on the Contact Maintenance Audit Report by User ID and the Contact Maintenance Audit report by Customer number, including who changed it and what the data was changed to and from.

N - Do not track contact maintenance information.

Determines if changes to a customer ship-to record are tracked:
•
Y - Track changes in Customer Ship-To Maintenance. The data appears on the Ship-to Maintenance Audit Report by User ID and the Ship-to Maintenance Audit Report by Customer (Order Processing module).
•
N - Do not track changes made to a customer ship-to.

N - Do not track changes in Vendor Maintenance.

Y - Track deleted purchase orders. The ”Deleted Purchase Orders Report” (Purchasing module) displays deleted purchase orders and the reasons for deletion.

Quote Job Number Index
Determines if job numbers are maintained and tracked on quotes:
•
Y - The job number on a quote is validated. All quotes that match any portion of the job number display to allow you to choose the job number for the quote.
•
N - Job numbers are not tracked.
Order Job Number Index
Determines if job numbers are maintained and tracked on sales orders:
•
Y - The job number on a sales orders is validated. If so, all sales orders that match any portion of the job number display to allow you to choose the job number for the sales order.
•
N - Job numbers are not tracked.

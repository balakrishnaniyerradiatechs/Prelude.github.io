Rental Parameters screen
The Rental Parameters screen allows you to set flag and option parameters to control how Prelude operates programs in the Rental module. You can override many of these setting for:
•
A product in Product Maintenance > Action Bar > Addtl Opts > Rental Information.
•
A customer in Customer Maintenance > F7 > Order > F5 Add Info > F6-Rental.
•
A rental order in Rental Order Entry > F5-Hdr Scr > Action Bar > Rental.
Menu Path: Systems > System Parameters > Maintenance > Option Parameters > Rental Parameters
Field and Function Key Descriptions
Field/Function Key
Description
Rental Rate Method
The rental rate assignment and billing method:
•
A - Best rental rate always. The rate is automatically evaluated (and pro-rated if necessary) and moved to the best rate based on the actual time material is rented. For example: 5 days moves to one week, 17 days moves to 2 weeks and 3 days.
•
B - Best rental rate - first billing. The rate is automatically evaluated and moved to the best rate based on the actual time material is rented up to the first billing. This billing is either the return of the material or the first periodic billing. Once first billing occurs, when material is returned, the billing is pro-rated by day.
•
C - Best rental rate - use single. Use the rate and rental period as a single rate based on the billing days (F5-Add Info). For example, if billing days are Monday through Friday and the rental item is billed for 7 days, the billing rate is calculated as 1.4 (1 week + 2/5). The total number of billing days is the denominator.
•
D - Daily rental - no exceptions. Always use the rate and rental period as a daily rate. The item is billed at the daily rate for the number of days between the start date and return date (Monday through Sunday plus holidays). For periodic billing, the daily rate is calculated according to the number of days set for the Monthly Billing Cycle in Rental Parameters.
•
N - No best rental rate. Use the rate and rental period as defined on the rental document and round to the nearest whole rental (e.g., 10 days on a daily weekly rental bills as 2 weeks).
Rental Pre Billing?
The default setting for the Rental Information screen (Rental Order Entry > F5-Hdr Scr > Action Bar > Rental > Rental Pre Billing?) but can be overridden for each rental order:
•
Y - Pre-bill the rental order.
•
N - Do not pre-bill the rental order. Bill the customer when it is returned.
Pre-billing allows you to bill rental orders at the beginning of the rental for the expected rental period rather than when it is returned. If the customer returns the rental early, you can issue credit (if Allow Auto Credit is set to Y, the credit is automatically generated). You can combine pre-billing with periodic billing to pre-bill for the next rental period.
Periodic Billing
The default setting for the Rental Information screen (Rental Order Entry > F5-Hdr Scr > Action Bar > Rental > Periodic Billing), but can be overridden for each rental order:
•
Y - Use periodic billing to bill the customer on a monthly basis for items rented over an extended time period.
•
N - Do not use periodic billing.
The periodic billing process checks each line on open rental orders to see if they are ready to bill.
Bill Incidentals
Determines when extra products added to the rental order are billed:
•
Y - Bill immediately when the order is entered.
•
N - Bill when the rental item is returned.
•
P - Bill at the first periodic billing.
Damage Waiver
The default setting in the Rental Information screen (Rental Order Entry > F5-Hdr Scr > Action Bar > Rental > Damage Waiver), but can be overridden for each rental order):
•
Y - Evaluate the rental orders for damage waiver charges.
•
N - Do not check for damage waiver charges.
A damage waiver is an additional charge added to the rental order through a service charge line item (e.g., a charge the customer incurs for repairs required for damaged equipment).
Each product can have a default Damage Waiver Product Number assigned (Product Maintenance > Action Bar > Addtl Opts > Rental Information).
If a customer has a Physical Damage Coverage amount (Customer Maintenance > F7 > Order > F5 Add Info > F6-Rental), then they do not have to pay the Damage Waiver fee.
Environmental Charge
The default setting in the Rental Information screen (Rental Order Entry > F5-Hdr Scr > Action Bar > Rental > Environmental Charge), but can be overridden for each customer (Customer Maintenance > F7 > Order > F5 Add Info > F6-Rental) or for each rental order:
•
Y - Evaluate the order for environmental charges.
•
N - Do not evaluate the order for environmental charges.
Environmental charges are additional charges added to a rental order using a service charge line item (e.g., a charge that is added due to environmental cost associated with the rental product).
Each product can have a default Environmental charge P/N assigned (Product Maintenance > Action Bar > Addtl Opts > Rental Information).
Liability Ins Charge
The default setting in Rental Order Entry > F5-Hdr Scr > Action Bar > Rental > Liability Ins Charge, but can be overridden for each rental order:
•
Y - Evaluate the order for liability insurance charges.
•
N - Do not evaluate the order for liability insurance charges.
A liability insurance charge is an additional charge added to a rental order using a service charge line item (e.g., a charge that is added due to liability insurance costs associated with the rental product).
Each product can have a default Liability Ins Charge P/N assigned (Product Maintenance > Action Bar > Addtl Opts > Rental Information).
If a customer has an amount in Liability Insurance Limit (Customer Maintenance > F7 > Order > F5 Add Info > F6-Rental), then they do not have to pay the Liability Ins Charge.
Rental Items Only in ROE?
Determines if only rental items are allowed in Rental Order Entry:
•
Y - Only allow rental items on rental orders (Product Type N in Product Maintenance).
•
N - Allow both rental and non-rental items on rental orders.
Allow Rental Items in SOE
Determines if rental items are allowed in Sales Order Entry:
•
Y - Allow rental items in Sales Order Entry.
•
N - Do not allow rental items in Sales Order Entry.
Rental items are designated as Product Type N in Product Maintenance.
Rental Grace Period
To provide a grace period to allow for travel time, enter the number of minutes to subtract from the rental return time.
Hours Shifted To Day
The number of hours before the hourly rental rate shifts to a daily rate. Only applies if Rental Rate Method is A or B. You can override this value for a product in Product Maintenance or Product Warehouse Maintenance.
Days Shifted To Week
The number of days before the daily rental rate shifts to a weekly rate. Only applies if Rental Rate Method is A or B. You can override this value for a product in Product Maintenance or Product Warehouse Maintenance.
Weeks Shifted to Month
The number of weeks before the weekly rate shifts to the monthly rate. Only applies if Rental Rate Method is A or B. You can override this value for a product in Product Maintenance or Product Warehouse Maintenance.
Number Of Days For Month
The number of days that defines a month. This value is used to calculate a pro-rated billing for a partial month rental.
Hourly Billing Cycle
If Periodic Billing is set to Y, the number of hours in an hourly rental to determine the next periodic billing cycle (e.g., 9 if 9 is the number of rent hours in a day).
Daily Billing Cycle
If Periodic Billing is set to Y, the number of days in a daily rental to determine the next periodic billing cycle (e.g., 1 for 1 day).
Weekly Billing Cycle
If Periodic Billing is set to Y, the number of days in a weekly rental to determine the next periodic billing cycle (e.g, 1 week = 7 days). This number is also used as the number of days per month for the Auto Credit Calc Method.
Monthly Billing Cycle
If Periodic Billing is set to Y, the number of days in a monthly rental to determine the next periodic billing cycle (e.g, 1 month = 30 days). This number is also used as the number of days per month for the Auto Credit Calc Method.
Rental Type
The default setting for the Rental Information screen (Rental Order Entry > F5-Hdr Scr > Action Bar > Rental > Rental Type) but can be overridden for each rental order:
•
L - Long Term. Update the rental period as MO (month).
•
S - Short Term. The rental period is updated as DAY.
•
P - Rental Purchase.
•
O - On Rent. The rental period is updated according to the Rental Period Default.
Rental Period Default
The default setting for the Rental Information screen (Rental Order Entry > F5-Hdr Scr > Action Bar > Rental > Rental Period Default) but can be overridden for each rental order:
•
HR - Hour.
•
DAY - Day.
•
WK - Week.
•
MO - Month.
•
USR - User defined.
This rental period default is the default rental period for each line item on the rental order:
Rental Header Screen?
Determines if the Rental Information screen automatically appears after you complete the header screen in a rental order:
•
Y - The Rental Information screen automatically appears after you enter the header information on a rental order.
•
N - The Rental Information screen does not automatically appear, but you can still access it from the header screen by selecting Rental from the Action Bar.
Auto Calculate Overtime
Determines if an overtime rate is automatically calculated based on meter hours:
•
Y - Automatically calculate the overtime rate (if you exceed the meter reading).
•
D - Automatically calculate a double overtime rate.
•
N - Do not calculate overtime (if the customer does not charge for overtime).
Sell Rent Prod As Inventory?
Controls how the sale or transfer of a rental product posts to the general ledger:
•
Y - The general ledger updates a rental item as rental inventory and does not reverse accumulated depreciation or recognize a gain/loss.
•
N - The general ledger updates a rental item as a fixed asset.
•
A - The general ledger updates a rental item as a fixed asset. The COGS and sales accounts are reflected.
Print Rental Return Receipt?
Determines if the rental receipt prints:
•
Y - If continuous printing is set up, you are prompted to print a return receipt during rental return to give to the customer.
•
N - No rental return receipt prints.
Company Closing Time
The time your company closes the store. This time is used to calculate the rental rates for hour/day rental calculations.
Company Opening Time
The time your company opens the store. This time is used to calculate the rental rates for hour/day rental calculations.
Prorate Hourly Billing Factor
The value used to calculate the hourly billing rate from a pro-rated daily rate. For example, a value of 6 calculates an hourly rate of 1/6 of the daily rate.
Switch To Post Bill?
If you pre-bill a rental order, the order is changed to a post-bill order after the first billing period:
•
Y - The order is immediately billed for the first rental period, but the remaining rental periods are billed through periodic billing.
•
N - You cannot change a pre-bill rental order to a post-bill rental order. It pre-bills every month.
Auto Recalculate Rates
Determines if a rate change on a rental order affects the rate change for other items on the order:
•
Y - When a rate is changed on a rental order, the same percent change is applied to each new line that is added to the order.
•
N - When a rate is changed on a rental order, no other rates on the order are affected.
Allow Auto Credit
Determines if a credit automatically generates for an early return:
•
Y - When a pre-billed rental order is returned early, a pro-rated price is automatically calculated and the remainder is credited to the customer. You must enter a default credit reason code in the next (unlabeled field) to use for the credit line item.
•
A - If the pre- or post-billed rental order is returned early, the automatic credit is applied to the rental order.
•
N - A credit does not automatically generate for an early return on a rental order.
Auto Credit Calc Method
If Allow Auto Credit is set to Y or A, determine how the credit is calculated:
•
D - Calculate the credit as a daily rate from the monthly rate, regardless of how they were billed.
•
B - Calculate the credit based on the best rate (so that they are credited for the fair amount).
Start Date/Time In RCE?
Determines if a rental start date and time are required in Release Confirmation Entry:
•
Y - Ask for the start date and time of rental in Release Confirmation Entry. You can enter the expected number of rental periods (e.g., hours, days, weeks, months) in Release Confirmation Entry to calculate an accurate expected rental return date. The expected rental return date is calculated by multiplying the expected number of periods times the billing cycles in the billing period for each item on the rental order for all units of measure.
•
N - Do not ask for the start date and time. Use the pick ticket printing date and time.
Cred Chk in Periodic Billing?
Determines if periodic billing is included in the credit limit check:
•
Y - Include periodic billing in the credit limit check. Each time the billing comes through, it will go on credit hold. The customer already has the product, but you may want to know if they are behind on payments.
•
N - Do not include periodic billing in credit limit checks. Even if they are past due, you will not be notified.
Rollup Billing
Determines if all non-serialized lines on a rental order are billed and rolled up into a common billed-through date as a single line when the rental order is invoiced:
•
Y - Roll up the activity into a single line on the invoice. The Rental Rate Method must be set to D (daily) for the rental order and it should only contain non-serialized items. You may want to use this if items are added during the month and you don’t want to have a separate bill for each item. Each item will be pro-rated from the day it was added. Then, the following month, they will all be billed the same.
•
N - Show each activity as an individual line item. Each time an item is added, it will generate a new billing and be tracked separately.
Internal Shop Customer
If you use the Shop Repair Module for returned rental items, you can enter the (generic) customer number to use to generate a shop/repair order at the time of rental return in Rental Order Entry. The Rental Return Status (Product Maintenance > Action Bar > Addtl Opts > Rental Information) for each rental serialized item on the rental order must be set to R (for shop/repair) so that you can generate a shop/repair order when the rental is returned.
Suppress Job Entry
Determines when you can enter the scheduled maintenance information for a returned rental item that requires preventative maintenance.
•
Y - The Job Entry Detail screen appears when you return a rental item so that you can enter scheduled maintenance information (e.g., pay type, repair group). When you complete the rental return, you have the option to create the shop repair order for the preventative maintenance. When you are in the shop repair order, the Job Entry screen does not appear since the default information is already entered.
•
N - When you return a rental item that requires preventative maintenance, the Job Entry screen does not appear until you create the shop repair order for the preventative maintenance. You can enter the scheduled maintenance information at that time.
F2-Save
Saves the rental parameter settings.
F5-Add Info
Opens the Additional Information screen to allow you to enter Billable Rental Days.

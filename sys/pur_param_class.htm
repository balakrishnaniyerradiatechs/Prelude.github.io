Product Class Settings screen
The Product Class settings screen allows you to define classes and set up safety allowance and turns information for products by product class. Refer to “Classifying Products” in the Inventory documentation for more information on product class. Refer to the Buyer Queue “Product Alerts” in the Purchasing documentation for more information on safety allowances.
Menu Path: Systems > System Parameters > Maintenance > Purchasing Controls > Purchasing Parameters > F6-Class Settings
Field and Function Key Descriptions
Field/Function Key
Description
Safety% For Dead Stock
The default safety allowance percentage for all products classified as dead stock. This percentage is applied during End of Month Calendar Close for products classified as dead stock (class 13).
Class
The inventory class number (i.e., 1 through 13, with class 1 being the fastest moving items and class 13 being dead stock). The NEW option at the bottom of the list allows you to set up default product class information (e.g., Days, safety allowance information) for new items added to your system.
Breakdown%
The percentage of the total number of products (classes 1-12) to include in each class.
For example, if you want 7.5% of your products to be included in class 1, enter 7.5. The top 7.5% of products, based on usage value, are assigned to class 1.
Days
The number of days to use to calculate a purchase order quantity. (Forecast/30) x Days = Replenishment quantity.
Used for the CLS Purchase Method products that have a P Replenishment Mtd (refer to the “Product Purchasing Parameters Maintenance screen” in the Inventory documentation).
Turns
The expected number of turns for the products in this class. The turnover is calculated by dividing the cost of stock sold in the last 12 months by the inventory value of the current on-hand quantity. The higher the turns the faster the products are selling, which usually indicates which products are more profitable to sell.
S/A%
The safety allowance percent for this class to calculate Order Point, Minimum, and Safety Allowance Quantity. Also used to allocate stock. You can override this percent for a product in Safety Allowance % in Product Purchasing Parameters Maintenance.
If you want to further define safety allowance quantities to maintain more accurate inventory levels, you can use the Service Factors Screen to assign service level factors for each service level percent.
S/A Low%
The lowest percentage for safety allowance analysis. The Product Alert work queue alerts the buyer when a product’s purchasing parameters should be reviewed if a product is received and the available quantity falls below the safety allowance tolerances defined for the product’s class (S/A Low% x Safety Quantity).
Safety Quantity is set for each product in the Product Purchasing Parameters Maintenance screen.
S/A High%
The highest percent for safety allowance analysis. The Product Alert work queue alerts the buyer when a product’s purchasing parameters should be reviewed if a product is received and the available quantity falls above the safety allowance tolerances defined for the product’s class (S/A High% x Safety Quantity).
Safety Quantity is set for each product in the Product Purchasing Parameters Maintenance screen.
F2-Save
Saves the product class settings information.

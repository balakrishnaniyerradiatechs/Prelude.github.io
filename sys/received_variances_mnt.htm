Over/Under Received Unit Variances screen
The Over/Under Received Unit Variances screen allows you to set up the tolerance amounts that a purchase order can be over or under received or costed in this warehouse without the receiver going on Stock Receipts Discrepancy hold in the Buyer Queue.
Refer to “Stock Receipts Discrepancy “in the Purchasing documentation for more information on receipts discrepancies and releasing receivers from hold.
Receiving Discrepancy Buyer
You must be set up as the Receive Disc Buyer in Warehouse Manager Maint to be able to release orders on Stock Receipts Discrepancy hold for your warehouse.
Menu Path: Systems > System Parameters > Maintenance > Inventory Controls > Warehouse Maintenance > F5-Add Info > F7-Variances
Field and Function Key Descriptions
Field/Function Key
Description
Over Rcvd Unit Variance%
Defines the percent the quantity an item can be over-received before going on hold. To further define the over-received quantity limit, you can enter multiple cost ranges below in From Cost and To Cost to correspond to over-received quantity limits.
The Stock Receipts Discrepancy Inquiry displays a Q reason code for the over-received product. The receiver must be released before Accounts Payable can reconcile it with the vendor invoice.
Under Rcvd Unit Variance%
Defines the quantity an item can be under-received before going on hold. To further define the under-received quantity limit, you can enter multiple cost ranges below in From Cost and To Cost to correspond to over-received quantity limits.
The Stock Receipts Discrepancy Inquiry displays a Q reason code for the over-received product. The receiver does not have to be released to allow Accounts Payable to reconcile it with the vendor invoice.
High $ Cost Variance%
Defines the percentage a received cost can be higher than the purchase order cost.
The Stock Receipts Discrepancy Inquiry displays a C reason code for the held product. The receiver does not have to be released to allow Accounts Payable to reconcile it with the vendor invoice.
Low $ Cost Variance%
Defines the percentage a received cost can be lower than the purchase order cost.
The Stock Receipts Discrepancy Inquiry displays a C reason code for the held product. The receiver does not have to be released to allow Accounts Payable to reconcile it with the vendor invoice.
P/O Auto B/O Cancel Var%
The variance percent that determines if a backorder should be automatically cancelled from a purchase order. If the variance between the open quantity and the received quantity is less than this amount, the backordered quantity is automatically cancelled. No warning message displays. The Purchase Order Log Inquiry displays the ACB code (automatic cancellation of backorder).
This flag does not affect purchase orders items that are linked to sales orders.
From Cost
The beginning cost range for which you want to further define over/under received variances that place a receiver on hold.
To Cost
The ending cost range for which you want to further define over/under received variances that place a receiver on hold.
Over Rec%
The percent by which a product must be over received, within the corresponding cost range, that determines if the receiver goes on hold.
Under Rec%
The percent by which a product must be under received, within the corresponding cost range, that determines if the receiver goes on hold.
F2-Save
Saves the receiving variance information.

Default Sort Sequence screen
The Default Sort Sequence screen allows you to set up a default sort order or value for the Prelude inquiries and entry screens. You can override most of these settings within each program.
Menu Path: Systems > System Parameters > Maintenance > Option Parameters > Default Parameters Maintenance
Field and Function Key Descriptions
Field/Function Key
Description
Lot Inquiry
The default sort sequence for viewing entries in the Lot Inquiry (Inventory > Inquiries > Miscellaneous Inquiries > Lot Inquiry):
•
A - Ascending on-hand balance.
•
E - Expiration date.
•
R - Received date.
Transaction Audit Inquiry
The default sort sequence for viewing entries in the Transaction Audit Inquiry (Inventory > Inquiries > Transaction Audit Inquiries):
•
B - Beginning entries.
•
E - Ending entries.
•
F - Filter options. Allows you to enter additional criteria (e.g. beginning and ending dates, transaction codes) to limit the search.
Available To Promise Days
The default number of days for each time period in the Available to Promise Inquiry (Inventory > Inquiries > Availability Inquiries > Available To Promise Inquiry). The system calculates 5 periods, each with this number of days, starting with the system date. The ending date displays for each period.

Default Consign Accum Flag
Sets the default for Accumulate in the Consignment Types screen (appears for Order Type of 07 or 08 on the Sales Order Header screen):
•
Y - Accumulate orders by customer. All consignment sales orders invoiced for a customer are accumulated on one order. Each time a product is added to the consignment order, it is added as a new line on the sales order.
•
C - Accumulate orders by product. All consignment sales orders invoiced for a product are accumulated on one line on a new consignment order if the selling warehouse and price for the item are the same.
•
A - Works the same as C except that all consignment sales orders invoiced for a product are accumulated on a new consignment. The first price found is used.
•
N - Do not accumulate orders.
•
E - Expense. Consult with your systems administrator for information.

Product Tagging Entry Mode
Determines where the cursor starts in the Product Tagging Entry screen for product entry programs:
•
P - Product. The cursor starts on Product and goes between the Product and the Quantity when you press ENTER.
•
Q - Quantity. The cursor starts on Quantity and goes to the next Quantity when you press ENTER.
When in the Product Tagging screen, you can press F8-Switch Mode to change between the P and the Q options.
Product Tagging Default Sort
The default sort sequence for viewing products in the Product Tagging Entry screen:
•
P - Display products in ascending order by product number.
•
D - Display products in ascending order by product description.

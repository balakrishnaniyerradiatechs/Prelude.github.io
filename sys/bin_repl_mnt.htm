Warehousing Bin Repl Maintenance screen
If you use automatic bin replenishment, you can use the Warehousing Bin Repl Maintenance screen to set rules to determine the type of bin to replenish, the bin to replenish from, and the bin replenishment sequence.
Menu Path: Systems > System Parameters > Maintenance > Inventory Controls > Warehouse Maintenance > F7-RTS Maint > F5-Warehousing > F6-Bin Repl
Field and Function Key Descriptions
Field/Function Key
Description
Single Bin Replenish
Controls whether bins are replenished from a single bin or multiple bins in the normal replenishment process:
•
Y - Bins are replenished from single bins (does not apply to emergency replenishments).
•
N - Bins can be replenished from multiple bins.
Putaway To Pick Bin
Determines if bins are replenished using a 1-step or 2-step put-away process:
•
Y - Use the 1-step process. Bin replenishments are put-away immediately to the primary bin, skipping the picking cart. Use this method when products are replenished with a full pallet.
•
N - Use the 2-step process for put-away; bin replenishments go first to the picking cart then the primary bin.
Utilize Replenish Priorities
Determines how bins are sorted for replenishment:
•
N - Sort bins by number of days supply. A date sequence is assigned using the current date plus bin on-hand days (the number of days supply on -hand based on product usage). The Bin Replenishment report lists the bin with the lowest quantity first. This method could lead to an inefficient sequence used to putaway stock.
•
Y - Sort bins by priority code. The Replenishment Priority Table opens to allow you to set up the priorities. You can enter the beginning and ending number of days supply range and the priority to use to assign replenishments. The Bin Replenishment report is sorted by priority code so that you can move in a logical sequence in the warehouse.
Allocation Pick Reserve Opts
Indicates the method to choose or eliminate bins eligible for replenishment sourcing:
•
P - Pick bins.
•
R - Reserve bins.
•
PR - Both pick and reserve bins.
•
Blank - Include all eligible bins.
Replenish Evaluation Method
Determines replenishment requirements by comparing the bin min/max to either of the following:
•
O - Bin onhand.
•
A - Bin available.
Replenish Rounding Method
Determines whether to round down to case or pallet quantity to replenish the primary bin:
•
C or blank - Round down to case quantities when evaluating a replenishment quantity.
•
P - Round down to pallet quantities when evaluating a replenishment quantity.
If no case or pallet quantity is defined for a product, it defaults to 1.
Replenish Bin Sorting Method
Controls how replenish bins are sorted:
•
O or blank - Sort starting with the smallest bin on-hand quantity. This option can help clean up the supplying bins with smaller bin on-hand quantities, to release bin space for other products.
•
D - Sort starting with the oldest bin transaction date. This option uses a method similar to a FIFO rotation of inventory.
Bin Minimum Percent
The minimum percent to use for the Auto Bin Replenishment Min/Max Recalculation. This value is multiplied by average monthly usage (in the product warehouse record) to calculate the bin minimum.
Bin Maximum Percent
The maximum percent to use for the Auto Bin Replenishment Min/Max Recalculation. This value is multiplied by average monthly usage in the product/warehouse to calculate the bin maximum.
Replenish Bin Seq
Determines the priority sequence for the corresponding Type of replenishment method.
Type
The bin replenishment method for the corresponding Replenish Bin Seq:
•
B - Bulk bin.
•
D - Dock bin.
•
K - Bulk un-packaged.
•
L - Line stock bin.
•
P - Pick bin.
•
R - Retail bin.
This sequence is not valid for lot controlled items.
F2-Save
Saves the bin replenishment information.

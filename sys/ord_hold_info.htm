Authorized Administrative Hold Codes screen
If you have codes set up in Order Processing Parameters to hold sales orders and quotes for pricing or freight restrictions, and if this user is set up as an Administrative Order Manager and/or an Administrative Quote Manager, you can limit the types of order and quote holds that this user is allowed to release.
Refer to the “Administrative Order Inquiry” and the “Administrative Quote Inquiry” in the Order Processing documentation for more information.
Menu Path: Systems > System Parameters > Maintenance > User Identification Maintenance > F5-Administrative > F5-Authorized Codes
Field and Function Key Descriptions
Field/Function Key
Description
Authorized AOH Types
The administrative order hold types that this user is authorized to release:
•
A - Product always on hold.
•
B - Backorder cycle allocations.
•
C - Price overridden below cost.
•
D - Override freight dollars.
•
E - Blanket P/O amount exceeded.
•
F - Override freight terms.
•
G - Price overridden below gross profit %.
•
I - Incompatible ship via.
•
K - Oversized order check.
•
L - P/O generated/linked through S/O.
•
M - Manual admin order hold.
•
N - Price overridden below minimum price.
•
O - Customer always on hold.
•
P - Customer requires a P/O number.
•
Q - Customer invoice amount check.
•
R - Price overridden below default price.
•
S - Quantity over month’s supply test.
•
T - Order with no tax jurisdiction codes.
•
V - Manager High/Low Level Limit GP%.
•
W - Web generated order.
•
X - RX product with no RX.

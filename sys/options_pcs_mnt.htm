Purchasing Parameters screen
The Purchasing Parameters screen allows you to set flag and option parameters to control how Prelude operates programs in the Purchasing module.
Menu Path: Systems > System Parameters > Maintenance > Option Parameters > Purchasing Parameters
Field and Function Key Descriptions
Field/Function Key
Description
Verify Vendor In POE
Determines if the Vendor Verification screen appears when you select a vendor in Purchase Order Entry:
•
Y - The Vendor Verification screen appears to allow you to verify the address of the selected vendor.
•
N - The Vendor Verification screen is skipped.
S/O-P/O, N/S Items Only
Determines if a purchase order can be generated from a sales order for stock items:
•
Y - You can only generate a purchase order from a sales order for non-stock items.
•
N - You can generate purchase orders from sales orders for stock and non-stock items.
Your user ID must have Enter P/O from S/O set to Y in the User Identification Maintenance Additional Information screen to be able to generate a purchase order from a sales order.
Mfg Cert Of Compliance
Indicates if a Manufacturer Certificate of Compliance is required in Purchase Order Entry:
•
Y - Require a Manufacturer Certificate of Compliance.
•
N - Do not require a Manufacturer Certificate of Compliance.
A certificate of compliance is only required for products that have Cert of Comp set to Y in the Lot Maintenance screen (Product Warehouse Maintenance > F9-Lot).
Chem/Phys Test Reports
Indicates if a chemical and physical test report is required in Purchase Order Entry:
•
Y - Prompt for a chemical and physical test report.
•
N - Do not prompt for a chemical and physical test report.
A chemical and physical test report is only required for products that have Chem & Physical set to Y in the Lot Maintenance screen (Product Warehouse Maintenance > F9-Lot).
Incl. Susp S/O For Linkage?
Determines if temporary purchase orders, work orders, and transfers can be created for products linked to suspended sales orders:
•
Y - Nightly Processing generates temporary purchase orders, work orders, and transfers for products linked to suspended sales orders.
•
N - Nightly Processing does not automatically generate temporary purchase orders, work orders, and transfers for products linked to suspended sales orders.
Multiple P/O Sourcing
Determines if multiple sales orders can be linked to a purchase order line item or if one-to-one linkage is required:
•
Y - A single purchase order line can be linked to multiple sales orders. The Purchase Order Display screen does not list the linked sales orders but instead displays the word Multiple for the linked product. In this case, any over-received quantity will not allocate to linked sales orders even if Allow Overage in SRE (Inventory Parameters) is set to Y.
•
N - A purchase order line item can only be linked to one sales order. The quantity allocated to the sales order must be split to a separate purchase order line. The Purchase Order Display screen lists the linked sales order number for each line with a sourced quantity.
S/O-P/O Linkage Work Queue
Activates the Linked Purchase Order Inquiry and the Linked Sales Order Inquiry:
•
Y - The Linked Purchase Order Inquiry (refer to the “Buyer Queue Inquiry” in the Purchasing documentation) is updated when someone changes a sales order linked to a purchase order. Reason codes are set up in the Linkage Queue Reason Code Maintenance screen. The Linkage Queue Parameter Maintenance screen determines the types of sales order changes that cause it to appear in this queue.
Also, the Linked Sales Order Inquiry (refer to the “Employee Queue Inquiry” in the Order Processing documentation) is updated when a purchase order linked to a sales order is updated.
•
N - The Linked Purchase Order and Linked Sales Order inquiries are not available.
Allocate Linked Frt To S/O
Determines if the freight charge is allocated to a linked sales order when a purchase order is received:
•
Y - Freight is allocated proportionally to Inbound Freight for any linked sales orders.
If the calculated freight amount is lower than the Minimum LInked Frt Amt in the Warehouse Maintenance Additional Information screen, the system uses the Minimum LInked Frt Amt from Warehouse Maintenance for Inbound Freight.
•
I - Same as Y, but it also includes the freight cost paid expenses as part of the G/L posting for the receiver.
•
N - Freight for received material is not automatically passed on to linked sales orders.
P/O Cost Displayed List/Disc.
Determines if the list price and the discount are assigned to a purchase order based on the product’s replacement cost Basis and Percent fields. This provides a way to cost a purchase order with a list price and discount from the product or product warehouse record:
•
Y - Retail (List or Price 0) and Percent from Product Maintenance or Product Warehouse Maintenance are assigned to Gross price and Disc% on the P/O Detail Information screen. On the Product Pricing screen, Basis must be set to Price Column 0 and a percent must be defined in Percent. Also, quantity breaks can not be set up.
•
N - The purchase order cost comes from the value set in Product Maintenance, Product Warehouse Maintenance, or Product Vendor Maintenance.
Cancel Sls Order Backorder?
Determines if the remaining backordered quantity on a sales order linked to a purchase order is cancelled if the purchase order backorder quantity is changed to zero:
•
Y - Cancel the backorder on the sales order.
•
N - Do not cancel the backorder on the sales order.
Allow P/O Link For Inv Hold
Determines if a sales order on Administrative Invoice hold can be linked to a purchase order:
•
Y - Allow sales orders on Administrative Invoice Hold to be linked to purchase orders.
•
N - Do not allow sales orders on Administrative Invoice Hold to be linked to purchase orders.
•
D - Allow only direct ship orders that are on Administrative Invoice Hold to be linked to purchase orders.
Suppress Freight Days Prompt
Determines if the Number of Freight Days is required when you change the Delivery Date on a purchase order:
•
Y - The Number of Freight Days is not required when a purchase order delivery date is changed.
•
N - If you change the delivery date on a purchase order at either the header or the line item level, you must enter the estimated Number Of Freight Days it takes for the merchandise to ship to your warehouse.
The delivery date information is used for the Low Priority Expedite and the High Priority Expedite reports generated by Nightly Processing.
Automate P/O Generation From SOE
If a purchase order is generated from a sales order, this flag determines if you can edit the purchase order while in Sales Order Entry, or if you have to go to Purchase Order Entry to edit the P/O:
•
Y - The purchase order is automatically generated from Sales Order Entry without the option to edit it. You can go into Purchase Order Entry to edit the order.
•
N - You can edit the purchase order while still in Sales Order Entry, before generating a P/O.
Your user ID must have Enter P/O from S/O set to Y in the User Identification Maintenance Additional Information screen to be able to generate a purchase order from a sales order.
Automate Transfer Generation
Determines if a transfer is automatically generated for a backorder on a sales order or work order:
•
Y - After you choose to generate the transfer and enter the warehouse number, the transfer is generated and you cannot edit it.
•
N - After you choose to generate the transfer and enter the warehouse number, you are brought into the transfer to edit (if necessary) and complete it.
If a product on a sales order is backordered, you can automatically generate a transfer if:
•
Enter T/R From S/O in your user ID Additional Information screen is set to Y.
•
Auto Transfer Generation in the Inventory Add. Parameters screen is set to Y or W.
•
At least one line item has a supply code (Sc) of T or R. Sc is automatically set to T for non-stock products with a Replenishment Path of T. For stock products, you must manually set Sc to T or R.
Refer to “Linking a Backorder to a Transfer” in the Order Processing documentation for more information.
Prompt In P/O Entry
The number of the field that the cursor moves to in the P/O Detail Information screen after you enter the quantity:
•
1 - Qty-B/O.
•
2 - Del-Date.
•
3 - Promised.
•
4 - Gross.
•
5 - Disc%.
•
6 - Net Cost.
•
7 - Vendor Product#.
•
Blank - Goes to the Accept field to allow you to determine whether on not you want to change any of the remaining fields.
Def Link Delivery Date Meth
Determines the delivery date for the line item when a sales order is linked to a purchase order:
•
P - Use the pick ticket date as the default delivery date.
•
R - Use the required date as the default delivery date.
•
A - Use the number of lead time days as the default delivery date.
This feature does no include direct ship purchase orders with a D supply code. In this case, the purchase order delivery date is always used.
Def Delivery Date Days
The number of default days to add to the purchase order date to display as the delivery date in the Purchase Order Header screen after you save the order.
Del Except Delay Days
The number of days a confirmed delivery date is after the original purchase order delivery date that will cause a purchase order to appear on the Confirmed/Expedite Date Exception report (Purchasing > Reports > Confirmed Date Exception Report).
For example, if Del Except Delay Days is set to 10, and you change the delivery date on a purchase order (header level or line level) to a date that is 11 days past the original delivery date, the P/O will appear on the Confirmed/Expedite Date Exception report.
To enter a confirmed delivery date on a purchase order, enter the date in Delivery Date on either the Purchase Order Header screen or the P/O Detail Information screen and select the Confirmed option.
P/O Expedite Days Late
The number of days that a purchase order must be late to appear on the low level or high level expedite reports (Purchasing > Reports > Nightly Processing Reports).
Defaults to 10 if left blank.
If the number of days is positive (e.g., 10), then the item will appear on the expedite report 10 days after the delivery date on the purchase order.
If the number of days is negative (e.g., -3) then the item will appear on the expedite report 3 days before the delivery date on the purchase order.
Refer to the “High Priority Expedite Report” and the “Low Priority Expedite Report” in the Purchasing documentation.
Vrfq Expiration Days
The number of days to add to the RFQ date on the VRFQ Header Information screen to calculate the Expiration Date for the Vendor Request for Quote.
P/O Pricing Tolerance %
The percent to use to determine if a change to a purchase order cost is outside of the tolerance limits. If you enter a cost that is greater or less than the default price by this percent or more, a warning message appears, but you can still continue with the purchase order.
Maintain Suspended P/Os
Determines if suspended purchase orders are tracked:
•
Y - Any time you suspend a purchase order, you are asked if you want to track the suspended order. If you enter another (non-emergency) order to the same vendor ship-to, you have the option to merge the suspended order with the new order (refer to “Merging with a Suspended Purchase Order” in the Purchasing documentation). The suspended purchase orders appear in the Buyer Queue. Refer to the “Buyer Queue Inquiry” in the Purchasing documentation for more information.
•
N - Suspended purchase orders are not tracked.
Set P/O Date To Output Date
Determines which date to use for the purchase order date:
•
Y - The purchase order date is the date the purchase order is printed, emailed, or faxed.
•
N - The P/O date is not updated when generated. The date originally entered applies.
Blanket P/O At Product Level
Determines if blanket purchase orders can be used by all warehouses (product level) or for an individual warehouse (product warehouse level):
•
Y - Track blanket purchase orders at the product level.
•
N - Track blanket purchase orders at the product warehouse level.
Refer to “Blanket Purchase Orders” in the Purchasing documentation.
Create Separate OTWM Lines?
Determines if separate lines should be created for each purchase order added to the OTW P/O Quantity Entry screen when receiving an on-the-water purchase order:
•
Y - Create a separate line for each purchase order.
•
N - Do not create a separate line for each purchase order.
You can change quantities in an OTW purchase order to reflect the quantity shipped by the vendor. If an item exists on several purchase orders to the same vendor, you may want to choose which purchase order to use or to change quantities on several purchase orders at one time. The OTW P/O Quantity Entry screen provides a worksheet to allow you to view and adjust quantities.
Refer to “Receiving On-the-Water-Merchandise” in the Inventory documentation.
F2-Save
Saves the purchasing parameter information.

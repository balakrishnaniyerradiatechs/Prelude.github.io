System Options for Shop screen
The System Options for Shop screen allows you to set flag and option parameters to control how Prelude operates programs in the Shop Repair module.
Menu Path: Systems > System Parameters > Maintenance > Option Parameters > Shop/Repair Parameters
Field and Function Key Descriptions
Field/Function Key
Description
Identify CN By Model
Determines whether a shop repair order requires the customer or the part number first:
•
Y - Enter the model and serial number first to identify the customer.
•
N - Enter the customer number first, then the model and serial number.
Require Tech On Job
Determines if a technician must be assigned to a job on a shop repair order:
•
Y - You must assign the technician before he/she can clock into a job.
•
N - You do not have to assign a technician. Any technicians that clock into the job are automatically added to the list of assigned technicians.
Require Problem Notes
Determines if problem notes are required for each job. Problem notes give a more complete description of what needs to be repaired:
•
Y - Require problem notes for each job on a shop repair order.
•
N - Do not require problem notes.
Enter Hours Only
Determines if access to Hours is allowed in Technician Billing Entry:
•
Y - Allow access to Hours in Technician Billing Entry to edit start and stop time.
•
N - Do not allow access Hours in Technician Billing Entry.
Update Asset For Capital Repair
Indicates if a sub-asset is created for a shop repair order:
•
Y - Update the main asset for the shop/repair order with the dollars for add-on capital updated in Asset Additions in Asset Maintenance > F7-Books.
•
N - Do not update the main asset. The system creates a sub-asset for the add-on capital.
Continuous Shop Invoice Print
If invoices print continuously, indicate if shop invoices should print continuously also:
•
Y - Print shop invoices continuously.
•
N - Do not continuously print shop invoices.
Repair Group Optional
Indicates if the Repair Group is required in the Shop Repair Job Entry screen:
•
Y - A Repair Group is optional but not required. The system does not check the technician’s repair skill set.
•
N - A Repair Group is required. The repair group must exists in the technician’s skill set in Technician Maintenance.
Auto Shop Charges
Indicates if a separate shop charge is automatically created when you complete a shop repair order:
•
Y - Automatically create a shop charge for each job on the order without prompting. A separate charge generates for each job and is determined by the pay type assigned to the job.
•
N - Do not automatically create shop charges. The Shop Repair Job Entry screens appears when you press F6-Complete from the Shop Repair Recap Information screen, so that you can manually add each shop charge to each job.
Update Equip Expense With ‘List’
If you use shop repair orders to repair rental equipment, indicate if the equipment expense is updated with the list price or the cost of the shop/repair order:
•
Y - Update the equipment expense with the list price from the shop repair order.
•
N - Update the equipment expense with the cost from the shop repair order.
Prorate Labor Cost
Determine how labor cost is calculated:
•
Y - Pro-rate labor cost based on actual time posted to the job.
•
N - Do not pro-rate labor cost. Calculate labor cost based on billable time.
Labor cost is only pro-rated for the first service labor product for each shop/repair job on the order.
Use PLine For Capital Postings
Determine whether to use the G/L account from the product line or the A/R offset account for capital postings:
•
Y - Use the product line.
•
N - Use the A/R offset account.
This flag only applies to shop repair orders with pay types with Capitalize set to Y. If the unit is still on-hand, the posting goes to the inventory account. If the inventory is sold, the posting goes to the cost of goods account.
Whse
The warehouses authorized to work on shop set-up orders. Leave blank to use the warehouse from the sales order.
Description
The warehouse description.
Dep#
The department number to assign the general ledger posting to, if you use departments.
Ccr#
The cost center number to assign the general ledger posting to, if you use cost centers.
Process Inventory As WIP
Indicate if you will use the extra work-in-process step to account for inventory on a shop repair order:
•
Y - Process inventory on a shop repair order as work-in-process (WIP). You must use Release Confirmation Entry to relieve inventory from the on-hand balance and move it into the WIP balance. When the order in invoiced, the quantities move out of WIP. The Shop Inventory Work In Process Report lists products in the Work in Process Account.
•
N - Do not process inventory as work-in-process. When you print the shop repair ticket, inventory balances are not affected until the order is invoiced. Release Confirmation Entry is not used.
The hierarchy for the WIP G/L accounts is as follows:
•
Pay Type / Product Line Maintenance.
•
Pay Type Maintenance.
•
Product Line Maintenance.
•
G/L Table Maintenance.
Convert Estimated Labor
Indicate if estimated labor on a shop repair quote is converted to the shop repair order:
•
Y - Convert estimated labor from a quote.
•
N - Do not convert estimated labor from a quote.
Create Multiple Setups
Indicates if multiple shop repair set up orders can be generated from one sales order:
•
Y - When you enter a line item on a sales order that requires a shop repair set up order, the number of shop repair set up orders generated will equal the quantity of the main item on the sales order.
•
N - Only one shop repair set up order is generated for each line item on a sales order that requires a set up, even if the quantity of the item is more than one.
F2-Save
Saves the Shop/Repair Parameter information.

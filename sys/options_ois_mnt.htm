Order Processing Parameters screen
The Order Processing Parameters screen allows you to set flag and option parameters to control how Prelude operates for order processing functions.
Menu Path: Systems > System Parameters > Maintenance > Option Parameters > Order Processing Parameters
Field and Function Key Descriptions
Field/Function Key
Description
Prompt For Cost
Determines if you are prompted for cost on non-stock products in Sales Order Entry:
•
Y - Prompt for cost.
•
N - Do not prompt for cost.
Curr/Orig Cost for Comm
Determines the cost used for commission cost:
•
C - Use the cost of the product at the time it was invoiced.
•
O - Use the cost of the product at the time the order was entered.
Commission Cost
The cost used to calculate gross profit for salesrep commissions:
•
A - Average cost.
•
S - Standard cost.
•
C - Cost for sales. Opens the Commission W/Linkage Cost Options screen to allow you to determine if commission is calculated using the cost from a linked purchase order, regardless of the cost of sales.
•
R - Replacement cost.
•
U - Original accounting cost; the cost when the item was first received.
This cost can be overridden in Price Matrix Maintenance.
Quote Cost
Determines the cost to use on quotes:
•
A - Average cost.
•
S - Standard cost.
•
C - Cost for sales.
•
R - Replacement cost.
Allow New Prod In S/O
Determines if products can be created within Sales Order Entry or Quote Order Entry:
•
N - You cannot create a new product on a sales order.
•
Y - If you enter a new part number, a dialog box appears to ask if you want to create a new product. If you select Yes, the Skeleton Product Maintenance appears and you can create a new product to add to the inventory file.
•
C - If you enter a new part number, a dialog box appears to ask if you want to create a comment line to reference the product. You can enter a description of the product that appears as a comment. No inventory is affected.
•
B - Both Y & C. If you enter a new part number, a dialog box appears to ask if you want to create a new product or add a comment line. If you choose to create a product, the Skeleton Product Maintenance appears and you can create a new product to add to the product file. If you choose to add a comment, you can enter a description of the product that appears as a comment line.
Multiple Whse In S/O
Determines if more than one shipping warehouse is allowed on the same sales order:
•
Y - You can select the warehouse for each line item.
•
A - Automatically display a list of warehouses and the available quantity of the product in each warehouse any time the quantity available is less than the quantity ordered. You can select a warehouse to ship from. A separate pick ticket prints in each warehouse.
•
N - All lines must use the same shipping warehouse.
You can override this setting for a warehouse in Multiple Whse In S/O in the Warehouse Maintenance Additional Information screen.
Utilize Best Price
Determines if you want to always use the lowest price on customers’ orders:
•
Y - Use the lowest price for the customer, including contracts, price matrices, quantity discounts, or column pricing. You can set Best Price in Customer Maintenance to N to override the system flag if a customer should not receive the best price.
•
N - Follow the pricing hierarchy. You can determine which sequence the system follows for the matrix hierarchy in the Matrix Types and Sequences Maintenance program (Order Processing Controls). The system defaults to a hierarchy of price matrix types (1,2,3,4,5, etc.), unless you change the sequence. You can set Best Price in Customer Maintenance to Y to override the system flag if a customer should receive the best price.
The setting at the customer level overrides this setting.
MSDS/Ship-To Specific
Determines where Material Safety Data Sheet (MSDS) information is maintained:
•
Y - MSDS information is maintained at the customer ship-to level. (Customer Ship-to Maintenance > F5-Add Info > F5-Permit Info > Restricted Use Permit).
•
N - MSDS information is maintained in Customer Maintenance (Order Processing > Maintenance > Customer Maintenance > F7-Order > F5-Add Info > Action Bar > Permits > Restrict Use Permit).
Restricted use permits indicate if the customer can purchase restricted products and which type of restricted products they can buy (restricted products are defined in Restricted Product in Product Maintenance > F5-Add Info > F6-MSDS/Permits).
If the customer does not have a restricted product permit, you can add the permit information in Sales Order Entry which updates Customer Maintenance. Refer to “Assigning MSDS Numbers to Products” in the Inventory documentation for more information.
MSDS/Notice Pick/ Letter
Determines if Material Safety Data Sheet (MSDS) information prints on the pick ticket or is mailed to the customer:
•
Y - The initial sale of an item that requires a MSDS adds a CP type comment line to the order to indicate that this item needs an MSDS.
•
L - The system stores the information to print a report to mail the MSDS to the customer.
•
B - Both Y and L options.
Refer to “Assigning MSDS Numbers to Products” in the Inventory documentation for more information.
Disp Add.Desc In SOE/QOE
Determines if the additional product description displays on quotes and sales orders, both on the screen and on the document:
•
Y - Display the additional description.
•
N - Do not display the additional description.
Additional descriptions are set up in Product Maintenance > F6-Xref/Notes > Add’l Description.
Disp. Shp.Inst. In SOE/QOE
Determines if the instructions screen automatically displays when the header screen opens in Sales Order Entry and Quotation Entry.
•
A - Always display the Instructions screen.
•
N - Never display the Instructions screen.
•
Y - Only display the Instructions screen if instructions exist for the customer in Customer Maintenance > Action Bar > Add’l Options > Instructions.
Override Addl Dep Req
Determines if you can override the Deposit Required on the Sales Order Completion screen for backorders on a sales order:
•
Y - You can override the deposit amount.
•
N - You cannot override the deposit amount.
The Deposit Required is only calculated if the Payment Terms on the sales order header has Deposit Required set to Y in A/R Payment Terms Maintenance.
Pick Ticket Default Days
The number of days to subtract from the required date in the sales order header to calculate the pick ticket date (excluding non-picking days and holidays). Enter T to always default to today.
Pick ticket days are set at multiple levels in the system. The hierarchy to determine which level to use first is:
•
Pick Ticket Def Days in Route Maintenance.
•
Pick Days in Customer Ship-To Maintenance > F5-Add Info.
•
Pick Days in Customer Maintenance > F7-Order > F5-Add Info.
•
Pick Ticket Default Days Warehouse Maintenance > F5-Add Info.
•
Pick Ticket Default Days in Order Processing Parameters.
Refer to “Setting Pick Ticket Rules” in the Order Processing documentation for more information.
Line Item G/P Display
Determines if the gross profit amount and percent appears in the Line Item Detail screen in Sales Order Entry:
•
Y - Display the gross profit amount and percent.
•
N - Do not display the gross profit amount and percent.
C/M Privilege/SOE
Indicates if you can enter negative quantities in Sales Order Entry to issue credit:
•
Y - You can issue credit in Sales Order Entry.
•
N - You must use Credit Memo Entry and/or RGA Entry to issue credit to a customer.
C/M Privilege/SPE
Indicates if you can issue credit in Shipment Processing Entry:
•
Y - You can issue credit in Shipment Processing Entry.
•
N - You cannot issue credit in Shipment Processing Entry.
If you need to enter, confirm, and invoice an order all in one step, you can use Shipment Processing Entry (Order Processing > Optional Entry & Printing > Shipment Processing Entry). The order is set up for invoicing and no pick ticket is generated.
Verify Customer In SOE
Determines if the Customer Verification or the Customer Ship-To Verification screen appears:
•
N - By-pass the customer verification screens.
•
Y - Display the customer’s name and address from Customer Maintenance and require verification.
•
S - Display the customer ship-to name and address from Customer Ship-To Maintenance and require verification.
Allow Direct For Cash/Cod
Determines if you can enter a direct sales order for a cash or COD sales order:
•
Y - Cash or COD direct sales orders are allowed.
•
N - Cash or COD direct sales orders are not allowed.
Prompt In Order Entry
The number of the field that the cursor first goes to in the Sales Order Entry Line Item Detail screen:
•
1 - Qty-Pck (quantity to pick).
•
2 - Qty-Ship (quantity to ship).
•
3 - Qty-B/O (quantity backordered).
•
4 - Sc (supply code).
•
5 - Gross Price.
•
6 - Disc% (discount percent).
•
7 - Net Price.
•
8 - Src (price source).
•
9 - Tx? (taxable flag).
•
Blank - The cursor will go to Accept to let you determine if you want to change any of the remaining fields or accept the information as is.
If you enter a field number, Sales Order Entry calculates and displays price and availability information then goes directly to the field that corresponds to this number.
Prompt in Quote Entry
The number of the field that the cursor first goes to in the Quote Order Entry Line Item Detail screen:
•
1 - Sc (supply code).
•
2 - Gross Price.
•
3 - Disc% (discount percent).
•
4 - Net Price.
•
5 - Src (price source).
•
6 - TX? (taxable flag).
•
7 - Cost.
•
8 - GP% (gross profit percent).
Fut Commit Safety Days
The number of days to add to lead time (Product Vendor Maintenance) to calculate when inventory on a future order should be committed. For example, if Fut Commit Safety Days is set to 10 and you change the Pck-Date on the Line Item Detail screen to a date that is more than 10 days in the future, the following occurs on the sales order’s Line Item Detail screen:
•
The Qty-Pck field changes to zero.
•
The Qty-B/O field is updated with the quantity ordered.
•
An F appears in the Future Commit field (unlabeled field before the Qty-Pck field).
Min S/O-G/P Percent
If the gross profit percent on a sales order is less than this amount, it will appear on the Gross Profit Exception Report (Order Processing > Reports > Daily Order Detail Reports > Gross Profit Exception Report). You can override this percent for a product line in Product Line Maintenance > Gross Profit Test? = Y.
Min S/O-G/P Dollars
If the gross profit amount on a sales order is less than this value, it will appear on the Gross Profit Exception Report (Order Processing > Reports > Daily Order Detail Reports > Gross Profit Exception Report).
Handling Charge
The amount to add as the Handling charge in the Sales Order Completion screen if the merchandise total does not meet the required amount. Once you enter the Handling Charge, the Handling Charge Merchandise Amount screen appears to allow you to enter the minimum merchandise amount that will not incur a handling charge.
Enter 0 if you don’t use handling charges.
The handling charge does not apply if:
•
The ship via is set to P and Handling Charge for Pickup orders is set to N.
•
The sales order has no line items.
•
The sales order total is not checked after the 1st invoice cycle.
•
The merchandise amount is zero or negative.
•
The sales order is a crib replenishment order.
•
Handling is overridden in the Sales Order Completion screen.
•
Handling Charge is set to N for the customer in Customer Maintenance > F7-Order > F5 > Add Info.
Display Open Quote/RGA Message
Determines if a message displays when you first enter a sales order or quote if the customer has open quotations and/or RGAs:
•
Y - A warning message displays if an existing quotation has not expired and is not assigned a Quote Lost Disposition code. If this flag is set to Y, the following flags may be applicable:
Qte Line Conversion in SOE (Order Processing Parameters > F5 > F5) determines if you are warned by line item if a quote exists for a product.
Open Quote Message (Order Processing Parameters > F5-Add Info) determines if the message displays at the header level, line item level, or both.
Include Converted Quotes (Order Processing Parameters > F5-Add Info) determines if converted quotes are included in the warning message.
•
N - No warning message appears if open quotations exist.
You can override this flag for a warehouse in the Warehouse Maintenance Additional Information screen.
Allow Quotes For Prospects
Determines if quotes are allowed for prospects:
•
Y - Quotes are allowed for prospects. Set up a customer as CASH in Customer Maintenance so you can convert a quote for a prospect to a sales order.
•
N - Quotes are not allowed for prospects.
Generate Replacement Equip S/O
Determines if you can create replacement equipment sales orders from an RGA (Return Goods Authorization):
•
Y - When you complete the Line Item Credit Detail screen in an RGA, a message appears to allow you to create a replacement order for the returned product. If you choose to create the replacement order, you can verify the replacement product and quantity, and then enter the replacement order upon completion of the RGA.
•
N - You cannot create a replacement order from an RGA.
Customer# Length
The number of characters allowed for a customer number (e.g., 6, 7, or 8). If you create a customer number that has less than this amount, the system adds preceding zeros. For example, if you enter 6 for Customer# Length, and you create a customer with the number of 345, the customer number appears as 000345.
SOE Expanded Order Review
Determines if the expanded version of the sales order review screen appears in Sales Order Entry > Action Bar > Examine:
•
Y - Displays additional information on the Sales Order Review screen (e.g., pck-date, req-date).
•
N - Displays the standard information on the sales order review screen.
Print CPO on Pick Tickets
Determine if CPO comments print on pick tickets. Choose from the following:
•
Y - CPO comments print on pick tickets.
•
N - CPO comments do not print on pick tickets.
Skip Header in SOE/QOE
Indicates if the Sales Order Header screen is skipped in Sales Order Entry and Quote Order Entry:
•
Y - The header screen will not appear in Sales Order Entry unless a required field is missing data (e.g., customer, purchase order, freight terms, salesrep).
•
Q - Y - The header screen will not appear in Quote Order Entry unless a required field is missing data (e.g., customer, purchase order, freight terms, salesrep).
•
B - The header screen will not appear in either Sales Order Entry or Quote Order Entry.
•
N - The header screen always appears when you first enter a sales order or quote.
Require Cash Customer Info
Determines if customer information is required when you enter a sales order for CASH:
•
Y - You must enter information in all of the customer name and address fields when you enter a sales order for the CASH customer.
•
N - You are not required to enter customer information for a CASH sales order.
Sell Whse for WO within SOE
Determines if the shipping warehouse of the sales order is used as the warehouse for a work order, if the work order is created within the sales order:
•
Y - Use the shipping warehouse of the sales order as the warehouse for the work order.
•
N - Do not use the shipping warehouse from the sales order.
Suppress Default Qty In SOE/QOE
Indicates if the automatic selling package quantity default is suppressed on sales orders:
•
Y - Accept the quantity entered for a product, regardless of the package quantity in Product Maintenance > Selling Pkg Qty.
•
N - When a quantity is entered for a product that is not a multiple of the Selling Pkg Qty set up for the product, automatically default the quantity up to the next multiple of the Selling Pkg Qty. For example, if Selling Pkg Qty is 10 and you enter a quantity of 15, the system will round up to a quantity of 20.
You can override the default quantity but a warning message appears.
Deactivate Auto Whse Generation
Indicates if a product/warehouse record is automatically created when a product is entered on a sales order that did not previously exist in the shipping warehouse (but exists in other warehouses):
•
Y - A product warehouse record is not automatically created when you enter a product on a sales order that does not already exist in the warehouse indicated on the sales order.
•
N - If you enter a product on a sales order that does not exist in that warehouse (but exists in other warehouses), a product warehouse record is automatically created.
F2-Save
Saves the Order Processing Parameters information and closes the screen.
F5-Add Info
Opens the Additional Information screen to allow you to access more order processing parameter options.
F6-Holidays
Opens the Holiday Mnt screen to allow you to enter company holidays.
F7-Add. Frt/Handling
Opens the Additional Frt/Handling screen to allow you to enter freight and handling option parameters.

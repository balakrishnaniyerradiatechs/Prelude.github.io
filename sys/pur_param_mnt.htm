Purchase Parameters Maintenance screen
The Purchasing Parameters Maintenance screen allows you to set company-wide purchasing formula, trend, safety allowance, lead-time, and product class information to use to calculate purchasing parameters.
Menu Path: Systems > System Parameters > Maintenance > Purchasing Controls > Purchasing Parameters
Field and Function Key Descriptions
Field/Function Key
Description
Cost/Ordering
The cost of ordering per purchase order line item (also known as R cost). Used to calculate the Recommended Qty in a product’s purchase parameters. Refer to the “Product Purchasing Parameters Maintenance screen” in the Inventory documentation for more information.
You can override this value for each warehouse in the Cost/Ordering field in the Warehouse Maintenance Additional Information screen.
Cost/Work Order
The scheduling cost per work order line item, used to calculate the Economic Order Quantity for a product. Refer to the “Product Purchasing Parameters Maintenance screen” in the Inventory documentation for more information.
You can override this value for each warehouse in the Cost/Work Order field in the Warehouse Maintenance Additional Information screen.
Cost/Carrying Inv
The cost of carrying inventory, otherwise known as the K cost, used to calculate the Recommended Qty in a product’s purchase parameters. Refer to the “Product Purchasing Parameters Maintenance screen” in the Inventory documentation.
This value is expressed as a percent of inventory.
You can override this value for each warehouse in the Cost/Carrying Inv field in the Warehouse Maintenance Additional Information screen.
ROAI
The percent of cost to use as the return on additional investment (ROAI). This factor is used in the pre-price increase calculation. When you buy in advance of a price increase, you’re assuming that usage and price for the product will not decrease in the future. You would normally need a significant return on your investment to justify this risk. Refer to “Including Future Price Increases” in the Purchasing documentation for more information on the formula for pre-price increase calculations.
Dead Stock Value
The annual cost of goods sold value under which an item is classified as dead stock. Refer to “Classifying Products” in the Inventory documentation for more information.
No/Mths Usage - Max
The maximum number of months of inventory that Nightly Processing recommends for purchasing.
Trend % Maximum
The maximum trend factor that you want to allow the system to calculate. Trend factors are used to account for conditions in your business that could affect forecasting demand (e.g., recessions, business requirements for the same month’s last year usage versus the same month’s this year usage, sales fluctuations due to marketing strategies or promotions). Trend % Maximum in the Warehouse Maintenance Additional Information screen overrides the Trend % Maximum in Purchase Parameters.
Refer to the “Forecasting” documentation for more information on Trends.
Forecast Formula
The formula for non-seasonal products. Used if no formulas exist in Product Line Maintenance, Vendor Line Maintenance, or Product Warehouse Maintenance. Refer to the “Forecasting” documentation for more information on entering formulas.
Forecast Formula - Sea
 
The formula for seasonal products. Used if no formulas exist in Product Line Maintenance, Vendor Line Maintenance, or Product Warehouse Maintenance. Refer to the “Forecasting” documentation for more information on entering formulas.
Product Class Type
Determines how classifications are ranking:
•
V - Rank classification by usage extended value.
•
U - Rank classification by usage quantities.
Product Class Method
Determines how products are classified:
•
W - By warehouse.
•
M - By warehouse/major group.
•
L - By product line.
Refer to “Classifying Products” in the Inventory documentation for more information.
#/Months Classification
The number of months a product must be a stocking product before it is included in the classification calculations. Refer to “Classifying Products” in the Inventory documentation for more information.
Perform Pre Price Update
Determines if Nightly Processing includes pre-price increase information to recommend purchase quantities:
•
Y - Nightly Processing updates the temporary purchase orders on the Recommended Purchase Order Action Report with products that should be purchased due to future price increases. The reason code E (price) indicates that a temporary purchase order was created due to a future price increase. These items also appear on the PrePrice Increase Report.
•
N - Nightly Processing does not create temporary purchase orders for items with future price increases.
Refer to “Including Future Price Increases” in the Purchasing documentation for more information on pre-price increase calculations.
Number Of Lead Times
Determines how many past lead times to use to calculate Average Lead Time in Product Vendor Maintenance. If you do not want unusual lead times to affect the calculation, you can set a maximum value in Lead Time Exception Days.
Cutoff Days For Lead Times
The number of days that determines if a lead-time is too old to use for the average lead time calculation. If blank, 180 is the default.
Lead Time Exception Days
Any lead time that is more than this number of days is excluded in the Average Lead Time calculation in Product Vendor Maintenance. Refer to “Product Vendor Maintenance” in the Inventory documentation.
Out Of Stock Exception
Determines if purchasing parameters are reviewed when a product is out of stock:
•
Y - If a product is out of stock for 2 weeks or more, do not use the usage value to calculate purchase parameters at month end (unless it was overridden in Product Usage Maintenance).
•
N - Use the usage of a product even if it is out of stock.
•
A - Review out of stock products for audit purposes only.
The Product Parameters Calc Skipped Audit Report (Inventory > Processes > Usage Exception Menu) displays products whose purchasing parameters were not recalculated during End of Month Calendar Close. The products on this report appear in the Buyer Queue Inquiry. The buyer can review and/or manually recalculate the purchase parameters if necessary.
Refer to “Product Usage” in the Inventory documentation for more information.
Unusual Usage Exception
Determines if purchasing parameters are reviewed when a product has unusual usage:
•
Y - If a product’s monthly usage is greater than the total of the last 5 months of combined usage, do not use the usage to calculate the product’s purchase parameters at month-end (unless it was overridden in Product Usage Maintenance).
•
N - Use the usage of a product even if it has unusual usage.
•
A - Review unusual usage products for audit purposes only.
The Product Parameters Calc Skipped Audit Report (Inventory > Processes > Usage Exception Menu) displays products whose purchasing parameters were not recalculated during End of Month Calendar Close. The products on this report appear in the Buyer Queue Inquiry. The buyer can review and/or manually recalculate the purchase parameters if necessary.
Refer to “Product Usage” in the Inventory documentation for more information.
Unusual Usage Class
The product class that determines if a product’s unusual usage is copied to the override field during month-end. All products with a class greater than or equal to this value will have unusual usage copied. If the override field already has a value, the unusual usage is not copied.
Unusual Usage Min$
The lowest value that End-of-Month processing ignores for unusual usage calculations. If a product has unusual usage and the value of the unusual usage is less than this amount, the product is not included in unusual usage calculations.
Unusual Usage Max$
The greatest value that End-of-Month processing ignores for unusual usage calculations. If product has unusual usage and the value of the unusual usage is more than this amount, the product is not included in unusual usage calculations.
Spike Exc. - High
The multiple to use to calculate a spike if the actual usage is greater than the forecast. The spike works as an exception trigger to signal products whose actual usage is greater than forecasted usage.
Example: If Spikes Exception -High is 4, the system determines if actual usage is 4 times larger than the forecast for the month. If so, the product appears in the Buyer Queue Inquiry (Purchase Parameters Skipped) with an S exception for the buyer to review. The S exception does not stop the parameters from being recalculated at month-end; it only alerts you to determine if usage should be adjusted to reflect larger than expected sales.
Spike Exc. - Low
The multiple to use to calculate a low spike. This works as an exception trigger to signal products with actual usage less than the forecast.
Example: If you enter .25 for the low spike, the system determines if actual usage is .25 times less than the forecast for the month. If so, the product appears in the Buyer Queue Inquiry (Purchase Parameters Skipped), with an S exception, for the buyer to review. The S exception does not stop the parameters from being recalculated at month-end; it only alerts you to determine if usage should be adjusted to reflect lower than expected sales. Refer to the Forecasting documentation for more information on formulas.
Spike Exc. - Min/Max
The multiple to calculate a spike for min/max items. The spike alerts the buyer when products flagged as min/max items have a usage of more than the maximum quantity settings. If a product has a spike by more than the multiple entered here, it shows up on the buyer’s work queue under Product Alerts. Refer to the Product Alert work queue on the Buyer Queue Inquiry in the Purchasing Module.
Refer to the Forecasting documentation for more information on formulas.
Safety Allowance Method
Determines how safety allowance is calculated:
•
L - Calculate safety stock based on a percentage of usage during leadtime.
•
S - Opens the Safety Factors screen to allow you to enter service level percentages and factors to calculate safety allowance quantities. Refer to the Service Factors Screen for information on how calculate safety quantities.
F2-Save
Saves the Purchase Parameter Information.
F4-Del
Deletes the Purchase Parameter Information.
F5-Forecasting
Opens the Forecasting Parameters screen to allow you to set up rules for forecasting usage.
F6-Class Settings
Opens the Product Class Settings screen to allow you to assign turns and safety allowance percentages to inventory classes.
F7-Product Alerts
Opens the Product Alert Parameters screen to allow you to enter settings that send products to the Product Alert Buyer Queue.

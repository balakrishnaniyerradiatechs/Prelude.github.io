Additional Information screen
The Warehouse Maintenance Additional Information screen allows you to set options for each warehouse that override options set at the company level.
Menu Path: Systems > System Parameters > Maintenance > Inventory Controls > Warehouse Maintenance > F5-Add Info
Field and Function Key Descriptions
Field/Function Key
Description
Cost/Ordering
The cost of ordering per purchase order line item, otherwise known as the R cost, used to calculate the Recommended Qty in a product’s purchase parameters. Refer to the “Product Purchasing Parameters Maintenance screen” in the Inventory documentation for more information.
This value overrides the system level Cost/Ordering in the Purchase Parameters Maintenance screen.
Cost/Work Order
The scheduling cost per work order line item used to calculate the Economic Order Quantity for a product. Refer to the “Product Purchasing Parameters Maintenance screen” in the Inventory documentation.
This value overrides the system level Cost/Work Order in the Purchase Parameters Maintenance screen.
Cost/Carrying Inv
The cost of carrying inventory, otherwise known as the K cost, used to calculate the Recommended Qty in a product’s purchase parameters. Refer to the “Product Purchasing Parameters Maintenance screen” in the Inventory documentation.
This value is expressed as a percent of inventory and overrides the system level Cost/Carrying Inv in the Purchase Parameters Maintenance screen.
Transfer Def. Leadtm
The default lead time for stock transfers. This value is used to calculate purchase quantities at month-end if no value exists for a vendor line (Purchasing > Maintenance > Vendor Line Maintenance > F5-Targets > F5-Add Info > Stock Transfers).
Work Order Def. Leadtm
The default lead time for work orders. This value is used to calculate purchase quantities at month-end if no value exists for a vendor line (Purchasing > Maintenance > Vendor Line Maintenance > F5-Targets > F5-Add Info > Work Orders).
Trend % Maximum
The maximum trend factor that the system is allowed to calculate (default is 100).
This value overrides Trend % Maximum calculated at the system level in the Purchase Parameters Maintenance screen. Trends are used to determine future product demand based on changes in business conditions. Refer to “Forecasting Formulas and Trends” in the Forecasting documentation.
Minimum Linked Frt Amt
The minimum freight amount that is automatically allocated to Inbound Freight on a linked sales order.
This flag works with Allocate Linked Frt To S/O in the Purchase Parameters Maintenance screen. If the freight amount calculated when a purchase order is received is lower than the Minimum LInked Frt Amt, then the Minimum LInked Frt Amt is used for Inbound Freight on the linked sales order.
Oversize Order Value
Defines the amount above which sales orders go on credit hold. For example, enter 50,000 if orders over $50,000 should go on credit hold. The system uses the following hierarchy to determine the oversize order value that puts an order on credit hold:
•
1st - Customer Maintenance > F6-A/R > Oversize Order Value.
•
2nd - Warehouse Maintenance > F5-Add info > Oversize Order Value.
•
3rd - Order Processing Parameters > F5-Add Info > F5-Add Info > Oversize Order Value.
Pick Ticket Default Days
The number of days to subtract from the required date in the sales order header to calculate the pick ticket date (excluding non-picking days and holidays). Enter T to always default to today.
Pick ticket days are set at multiple levels in the system. The hierarchy to determine which level to use first is:
•
1st - Pick Ticket Def Days in Route Maintenance.
•
2nd - Pick Days in Customer Ship-To Maintenance > F5-Add Info.
•
3rd - Pick Days in Customer Maintenance > F7-Order > F5-Add Info.
•
4th - Pick Ticket Default Days Warehouse Maintenance > F5-Add Info.
•
5th - Pick Ticket Default Days in Order Processing Parameters.
Refer to “Setting Pick Ticket Rules” in the Order Processing documentation for more information.
Bin Loc Sort - S/O
Determines the sort sequence for pick tickets if you use bins:
•
B - Sort by bin.
•
C - Sort by customer P/O line number.
•
N - No sort order.
•
P - Sort by product number.
This setting overrides the setting for a company in Bin Loc Sort - S/O in the Company Maintenance Additional Information screen.
Multiple Whse In S/O
Determines if more than one shipping warehouse is allowed on the same sales order:
•
Y - You can select the warehouse for each line item.
•
A - Automatically display a list of warehouses and the available quantity of the product in each warehouse any time the quantity available is less than the quantity ordered. You can select a warehouse to ship from. A separate pick ticket prints in each warehouse.
•
N - All lines must use the same shipping warehouse.
This setting overrides the system level setting in Multiple Whse In S/O in the Order Processing Parameters screen.
Display Open Qte/RGA Msg
Determines if a message displays when you first enter a sales order or quote if the customer has open quotations and/or RGAs:
•
Y - A warning message displays if an existing quotation has not expired and is not assigned a Quote Lost Disposition code. If this flag is set to Y, the following flags may be applicable:
Qte Line Conversion in SOE (Order Processing Parameters > F5 > F5) determines if you are warned by line item if a quote exists for a product.
Open Quote Message (Order Processing Parameters > F5-Add Info) determines if the message displays at the header level, line item level, or both.
Include Converted Quotes (Order Processing Parameters > F5-Add Info) determines if converted quotes are included in the warning message.
•
N - No warning message appears if open quotations exist.
This flag setting overrides the system level setting in Display Open Quote/RGA Message in the Order Processing Parameters screen.
Warehouse Status
Indicates if this warehouse can sell products:
•
Y - This warehouse can sell products.
•
N - This warehouse number cannot be entered on a sales order.
•
V - This warehouse is a virtual warehouse (customer modifications required).
(Unlabeled field)
The default warehouse that receives inventory for the virtual warehouse. Only applicable if Warehouse Status is set to V.
Days Early
The number of days that a product is considered as received early.
If a product is received this number of days before the promise date on the purchase order, the product will appear on the Stock Receipts Exception Report (Inventory > Reports > Stock Receipts Reports > Stock Receipts Exception Report).
Days Late/Orig or Exp Date
The number of days after the original promise or expedite date that a product is considered as received late. If a product is received this number of days after the promise date on the purchase order, the product will appear on the Stock Receipts Exception Report (Inventory > Reports > Stock Receipts Reports > Stock Receipts Exception Report) and affect vendor performance statistics.
Days Late/Latest Prom Date
The number of days after the latest promise date that a product is considered as received late. If a product is received this number of days after the latest promise date on the purchase order, the product will appear on the Stock Receipts Exception Report (Inventory > Reports > Stock Receipts Reports > Stock Receipts Exception Report) and affect vendor performance statistics.
Default Bank Code
The default bank code to use for transactions that do not ask you for a bank code (e.g. a credit memo applied to an existing invoice).
Bank codes are set up in Accounts Receivable Controls in the Bank Account Code Maintenance screen.
Distribution Center
Identifies this warehouse as a distribution warehouse for stock replenishment purposes:
•
Y - This warehouse is a distribution center for other warehouses. Usage on a sales order is tracked through this warehouse rather than the sales order’s shipping warehouse. However, if this flag is set to Y and the customer’s Override D/C Flag is set to N, usage is recorded in the shipping warehouse.
•
N - This warehouse is not a distribution center.
This flag is overridden if a product on a sales order has a Distribution Ctr assigned in Product Purchasing Parameters Maintenance > F7-Add Info. If the product is shipped from any warehouse other than the selling warehouse, usage is updated in the distribution center’s product warehouse record so that a warehouse only maintains enough stock to meet their demands. Refer to ”Recording Product Usage” in the Inventory documentation.
Manifest System Flag
Determines the type of manifest subsystem (Order Processing Module) used for this warehouse:
•
Y - Use Prelude’s UPS module for the manifest subsystem.
•
P - Use PSS (Parcel Shipping System) for the manifest subsystem (no longer supported by Activant Prelude).
•
N - Disables the manifest subsystem if you are using an outside source.
This flag overrides the setting for Manifest Subsystem set in the Company Maintenance Modules screen, for this warehouse only.
Confirmation To ADS Manifest?
Determines if the Manifest System Entry screen appears after the order is confirmed:
•
Y - The Manifest screen appears after you confirm an order in Release Confirmation Entry, Transfer Shipments Entry, or Work Order Issues Entry. Updates the freight information (e.g., amount, weight, number of boxes). Use this option if your packing station is also a freight station.
•
A - Updates the freight information and releases the order for invoicing. Use this option if you only have a single freight station that is separate from the packing station.
•
D - Same as A, but you must use Release Confirmation Entry to update the order and release it for invoicing.
•
N - Update the freight information and release orders during manifest printing at the end of the day.
Only applicable if the Manifest System Flag is set to Y or P.
Carousel Interface
Determines if the warehouse uses the carousel interface:
•
Y - Activates the carousel interface. Only products that have CAR as their primary bin update the non-nettable quantity when received.
•
A - Activates the carousel interface. All products update the non-nettable quantity when received.
•
N - Do not use the Carousel interface.
To use the carousel interface, you must also set the following flags to N:
•
Warehousing Flag in the Warehousing Maintenance screen .
•
Multiple Bin Locations in the Bin Locator Options screen.
Refer to “Receiving Carousel Non-Nettable Products” in the Inventory documentation for more information.
Credit Bureau Override
Determines if the warehouse is using Protobase to authorize credit card transactions or if authorization codes are entered manually:
•
Y - The warehouse is not using Protobase to authorize credit card transactions. You can manually enter an authorization code when completing a sales order. The authorization is performed independently of Prelude.
•
N - The warehouse is using Protobase to authorize credit card transactions.
Prime Contractor Code
The code that prints in the Prime Contractor Code (block 9) on the DD250 form if you print invoices on a DD250 form. This form is a Material Inspection and Receiving Report printed for government customers. Refer to “Printing DD250 Invoices” in the Order Processing documentation.
Generate Release By
Determines if sales order releases are generated by:
•
O - Order. A separate release generates for each sales order.
•
Z - Zone. A separate release generates for each zone on the pick ticket. You must select this option if you set Zone Type to P (Inventory > Maintenance > Bin Files > Zone Maintenance) to include the zone in the pick process. In this case, each bin should have a zone assigned. Refer to “Defining Zones” in the Inventory documentation.
•
L - Line count. A separate release generates for the number of lines entered in the next (unlabeled field). Only includes lines with an available pick quantity. Excludes type 02 (ship complete) orders. Kit parent items are considered as one line.
RF Inv Adj Verification Amt
The largest inventory adjustment extension amount that you can enter for a product when entering counts in the RF inventory adjustment screens. If you enter an adjustment for which the inventory extension is more than this amount, a warning message appears to ask you to confirm that the adjustment is correct.
Bin Loc Sort - W/O
Determines the sort sequence for work order pick tickets if you use bins:
•
B - Sort by bin.
•
N - No sort order.
This flag overrides Bin Loc Sort - S/O in the Company Maintenance Additional Information screen.
F2-Save
Saves the information on the Additional Information screen.
F5-Tax Jur
Opens the Tax Jurisdictions screen to allow you to enter valid tax jurisdictions for the warehouse.
F6-Picking Days
Opens the Picking Rules screen to allow you to designate which days of the week are acceptable to print pick tickets.
F7-Variances
Opens the Over/Under Received Unit Variances screen to allow you to enter acceptable variance amounts for over or under received quantities for the warehouse.
F8-Inc Ship Via
Opens the Incompatible Ship Via Maintenance screen to allow you to enter ship via types that are not compatible for this warehouse.

Purchase Order Requisition Types screen
Purchase order requisition types are set up by Prelude and used when an employee enters a purchase requisition to appear in the buyer queue to request products from a vendor (refer to “Entering a Purchase Requisition” in the Purchasing documentation). You can use the Purchase Order Requisition Types screen to set the default purchase order type that appears in the Purchase Order Header screen and to determine which types are valid for your company.
Menu Path: Systems > System Parameters > Maintenance > Purchasing Controls > Purchase Order Requisition Entry Types
Field and Function Key Descriptions
Field/Function Key
Description
P/O Req Type Default
The purchase order type to default in the Purchase Order Header screen for a purchase order requisition.
Valid P/O Req Types
The purchase types that are valid in Purchase Order Requisition Entry for your company. The available types in Prelude are:
•
01 - Regular. Standard order of merchandise for delivery to your warehouse.
•
02 - Direct Ship. The vendor ships the material directly to your customer. You must specify the sales order number to link to the P/O.
•
03 - Will Call. You pick up the order from the vendor. The merchandise is received automatically when you complete the purchase order, making the P/O a one-step process.
•
04 - Multi Branch. Material purchased for multiple warehouses on one purchase order.
•
05 - Special. Non-stock material required for a special purpose.
•
07 - Capital Assets. Company assets such as office furniture and copiers that are not products in the system.
•
08 - Non Inventory Expense. Similar to the 07 (Capital Assets) type, but for products such as office supplies that are not capital assets.
•
09 - Future. You can enter a future date if the material is not for shipment until a later date. Nightly Processing uses this date to calculate re-order quantities. When the future date arrives, the purchase order automatically changes to a type 01 (Regular).
•
10 - Consigned. Vendor consigned inventory that is tracked and kept separately from regular inventory. Linked, lot-controlled, and serialized items are not allowed on this purchase order type.
•
11 - Recurring Billing. Allows an item to be received multiple times. You can use this to receive rental items multiple times until the rental is complete. Once the rental item is updated as complete, the purchase order is closed.
•
12 - OTW-Stock. On-the-water stock purchase orders used to import from an overseas vendor.
•
13 - OTW-Direct. The same as type 12, except it is for direct shipments to your customer.
F2-Save
Saves the purchase order requisition type information.
F4-Del
Deletes the purchase order requisition type information.

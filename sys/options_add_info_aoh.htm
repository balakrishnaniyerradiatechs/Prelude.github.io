Administrative Order Hold Parameters screen
The Administrative Order Hold Parameters screen allows you to set option parameters for your company to define the restrictions that place an order on Administrative Order Hold. An order must be released from hold in the Administrative Order Hold Inquiry before a pick ticket can print.
Administrative hold options set at the customer or product level override the options set in this screen. Refer to the “Administrative Order Inquiry” in the Order Processing documentation for more information on how to hold and release sales orders.
Menu Path: Systems > System Parameters > Maintenance > Option Parameters > Order Processing Parameters > F5-Add. Info > Administrative Order Hold = Y
Field and Function Key Descriptions
Field/Function Key
Description
AOH Types
The codes that determine what criteria places an order on Administrative Order Hold:
•
B - Backorder Cycle Allocations: If inventory is allocated to the sales order during the Allocation Procedure or when stock is received (if the purchase order is linked after the sales order is invoiced at least once).
•
D - Override Freight Dollars: The Outbound Freight in the Sales Order Completion screen is overridden.
•
E - Blanket P/O Amount Exceeded: A customer’s blanket P/O amount is exceeded.
•
F - Override Freight Terms: The Freight Terms code is changed in the order header.
•
I - Incompatible Ship Via: If the Ship Via Code is incompatible for the customer (Customer Maintenance > F7-Order > F8-Incomp Ship Via) or the product (Product Maintenance > F5-Add Info > Action Bar > Incomp Ship Via).
•
K - Oversized Order Check: The total of the order is more than Oversized Order $ Amount on this screen.
•
L - P/O Generated/Linked Through S/O: The line item is linked to an existing purchase order or if a purchase order is generated through Sales Order Entry.
•
M - Manual: If an order is put on Manual Hold in Sales Order Entry.
•
P - Price Violations: A price is lowered below the default price, cost, or the acceptable gross profit percent. You must also determine if the price violations appear in the Administrative Invoice Inquiry or the Administrative Order Inquiry. If you choose to update the Administrative Order Inquiry, the N option is also available.
•
N - Minimum Price Violations: A line item’s selling price is below the product’s Minimum Price (Product Maintenance > F7-Pricing).
•
S - Ship-Via from Web Requires Authorization: The Ship Via code entered on an order placed on the website must be authorized.
•
T - Order With No Tax Jur Codes: The order is missing Tax Jurisdiction Codes.
•
V - Manager High/Low Level Limit GP%: A line item’s gross profit percent is below the Mgr Low Level Minimum GP% or more than the Mgr High Level GP%. Contract pricing is excluded.
•
W - Web Generated Order: Any order generated from your VIA website.
•
X - RX Product with no RX. A prescription product is entered on a sales order without a prescription number. Refer to “Setting up and Tracking Prescriptions” in the Order Processing Documentation.
•
Z - Credit Card Order – No Authorization: A credit card order is entered without an authorization code. Refer to “Credit Cards” in the Order Processing documentation for more information.
Mgr
Define the manager code from the Warehouse Manager Maintenance screen for each AOH Type, to determine who is authorized to release orders on hold. This code can be overridden in Customer Maintenance.
Max Months Supply For Admin Order Hold
The maximum number of months usage the quantity of an item on a sales order can be before it is placed on hold. This option prevents a single sales order from taking all the available stock for a product causing other orders to backorder. The Administrative Hold Manager can decide if the customer is important enough to inconvenience other customers or if the customer can receive only part of the shipment now and the rest later.
Max Months Supply Cutoff Class
The product class above which a line item is not subject to the Max Months Supply For Admin Order Hold restriction. For example, if the value in this field is 6, then sales orders containing products with an inventory class of 7 through 13 are not checked for the Max Months Supply For Admin Order Hold.
Mgr Low Level Minimum GP%
The minimum gross profit percent allowed for a line item before it is placed on hold. This value can be overridden if a Mgr Low Level G/P% exists for the product line. AOH Type is V, which is only activated if the P (price violation) option is also in effect.
Mgr High Level GP%
The maximum gross profit percent that a line item can be before it is placed on hold. This value can be overridden if a Mgr High Level G/P% exists for the product line. AOH Type is V, which is only activated if the P (price violation) option is also in effect.
Oversized Order $ Amount
If you entered an AOH Type of K, an order will go on hold if the total value is more than this amount.
Admin Hold/unallocate Inv
Indicates if orders on Administrative Order Hold allocate stock:
•
Y - If an order goes on hold, the products are un-allocated and backordered.
•
N - If an order goes on hold, all allocations remain.
F2-Save
Saves the order hold information.

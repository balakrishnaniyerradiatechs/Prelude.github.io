Administrative Invoice Hold Parameters screen
The Administrative Invoice Hold Parameters screen allows you to set option parameters for your company to define the restrictions that place an order on Administrative Invoice Hold. You must use Administrative Invoice Inquiry to release the order before it can be invoiced.
Administrative hold options set at the customer level override the options set in this screen. Refer to the “Administrative Invoice Inquiry” in the Order Processing documentation for more information on how to hold and release invoices.
Menu Path: Systems > System Parameters > Maintenance > Option Parameters > Order Processing Parameters > F5-Add. Info > Administrative Invoice Hold = Y
Field and Function Key Descriptions
Field/Function Key
Description
AIH Types
The codes that determine what criteria places an order on Administrative Invoice Hold:
•
A - Credit Card Order – No Authorization: A credit card order is entered without an authorization code.
•
P - Price Violations: A price is changed to a value that is lower than the default price or lower than the cost. This only applies if Price Override Code for the User ID is set to C or R in the User Identification Maintenance Additional Information screen.
•
S - Salesrep Changed On Order: Salesrep One or Salesrep Two was changed on the Sales Order Header screen.
•
T - Order With No Tax Jur Codes: The order is missing Tax Jurisdiction Codes.
•
U - User Defined. See your Systems Administrator.
Exceed Preauth % Inv Hold
The percent a credit card invoice can be over the pre-authorized amount before it goes on hold (if you pre-authorize credit card orders).
F2-Save
Saves the invoice hold information.

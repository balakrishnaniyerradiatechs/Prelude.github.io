Warehouse Manager Maintenance screen
The Warehouse Manager Maintenance screen allows you to assign a manager code, buyer code, or user ID, to each type of hold queue in each warehouse. Once the manager codes are assigned to a warehouse, the warehouse will appear under Current Warehouses Assigned in the Manager Maintenance screen.
Menu Path: Systems > System Parameters > Maintenance > Warehouse Manager Maintenance
Field and Function Key Descriptions
Field/Function Key
Description
Warehouse Number
The warehouse for which you want to assign manager codes.
Credit Manager
The code for the manager that can release orders from Credit Hold for this warehouse. A user ID must have this code assigned in Credit Manager in their Administrative Settings screen to be able to release orders on credit hold for this warehouse. Refer to “Credit Holds” in the Accounts Receivable documentation for more information.
Admin Order Manager
The code for the manager that can release orders from Admin Order Hold for this warehouse. A user ID must have this code assigned in Admin Order Mgr in their Administrative Settings screen to be able to release orders on Admin Order hold for this warehouse. Refer to the “Administrative Order Inquiry” in the Order Processing documentation for more information.
Admin Invoice Manager
The code for the manager that can release orders from Admin Invoice Hold for this warehouse. A user ID must have this code assigned in Admin Invoice Mgr in their Administrative Settings screen to be able to release orders on Admin Invoice hold for this warehouse. Refer to the “Administrative Invoice Inquiry” in the Order Processing documentation for more information.
Admin Quote Manager
The code for the manager that can release orders from Admin Quote Hold for this warehouse. A user ID must have this code assigned in Admin Quote Mgr in their Administrative Settings screen to be able to release orders on Admin Quote hold for this warehouse. Refer to the “Administrative Quote Inquiry” in the Order Processing documentation for more information.
Branch Manager
The code for the manager that can release orders from Branch Hold for this warehouse. A user ID must have this code assigned in Branch Manager in their Administrative Settings screen to be able to release orders on Branch hold for this warehouse. Refer to the “Branch Release Inquiry” in the Order Processing documentation for more information.
Emg Cycle Counter
The code for the manager that can release orders from the emergency cycle count queue for this warehouse. A user ID must have this code assigned in Emg Cycle Counter in their Administrative Settings screen to be able to access items in the emergency cycle count queue for this warehouse.
Inspection Manager
The code for the manager that can release orders from the Quality Control Inspection Inquiry for this warehouse. A user ID must have this code assigned in Inspection Manager in their Administrative Settings screen to be able to access items in the inspection queue for this warehouse. Depending on how you use the inspection queue, refer to “Releasing Products from Quality Control Inspection” in the Inventory documentation or the “Quality Inspection Subsystem” in the Order Processing documentation.
Imprinting Manager
The code for the manager that can release orders from imprinting hold for this warehouse. A user ID must have this code assigned in Imprinting Manager in their Administrative Settings screen to be able to access items in the Imprint Hold Inquiry for this warehouse, when a vendor’s product number goes on imprinting hold.
You can override this hold queue for a vendor in Vendor Maintenance > F5-Add Info > Imprint Manager.
Purchasing Manager
The code for the manager that can release orders from the Purchase Hold Release work queue for this warehouse. A user ID must have this code assigned in Purchasing Manager in their Administrative Settings screen to be able to access items in the Purchase Hold Release work queue for this warehouse. Refer to “Purchase Hold Release” in the Buyer Queue Inquiry in the Purchasing documentation for more information.
Purchasing Buyer
The buyer code (set up in Purchasing > Maintenance > Buyer Code Maintenance) for the buyer responsible for Product Alerts for this warehouse. If blank, the buyer assigned to each vendor or vendor line is notified of product alerts. Refer to “Product Alerts” in the Purchasing documentation for more information on what causes a product to go on alert in the Buyer Queue Inquiry.
Receive Disc Buyer
The buyer code (set up in Purchasing > Maintenance > Buyer Code Maintenance) for the buyer responsible for releasing orders from Stock Receipts Discrepancy for this warehouse. If blank, the buyer assigned to the purchase order is notified of a receiving discrepancy. Refer to “Stock Receipts Discrepancy” in the Purchasing documentation for more information on what causes a product to go on receiving hold in the Buyer Queue Inquiry.
A/P Manager
The user ID of the person responsible for reviewing invoices on hold in the Accounts Payable Release Inquiry.
Invoices can go on hold in Vendor Invoice Reconciliation Entry if a user exceeds their limits set in A/P Write-Off Limit, Vend Inv Rec Limit, or Vend Inv Recost% set for their user ID in the Administrative Settings screen.
1st Level Frt Buyer
The buyer code (set up in Purchasing > Maintenance > Buyer Code Maintenance) for the buyer responsible for releasing freight invoices from the EDI Freight Invoice work Queue. The EDI Freight Invoice work queue contains freight invoices converted through EDI and is the first level in the EDI freight invoice approval process. Once you release the invoice, it goes to the 2nd Level Frt Manager to set it up for payment.
Refer to “EDI Freight Invoice” in the Purchasing documentation for more information on releasing a freight invoice in the Buyer Queue Inquiry.
2nd Level Frt Manager
The code for the manager that can release orders from EDI Freight Invoice Second Level hold for this warehouse. A user ID must have this same code assigned in Freight Manager in the Administrative Settings screen to be able to release freight invoices from hold for this warehouse. Refer to the “EDI Freight Invoice Second Level Inquiry” in the Accounts Payable documentation for more information.
F2-Save
Saves the warehouse manager information.
F4-Del
Deletes the managers for this warehouse.

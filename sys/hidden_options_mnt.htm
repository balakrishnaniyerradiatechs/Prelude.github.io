Additional Options Menu
Additional Options Maintenance screen
The Additional Options Maintenance screen allows you to set options for Prelude system processes.
Menu Path: Additional Options Menu > Additional Option Parameters
Field and Function Key Descriptions
Field/Function Key
Description
Keep P/O Line After Receipt
Determines if the line items on a purchase order remain after they are received:
•
Y - All line items on a purchase order continue to display in Stock Receipts Entry even after they are received, but you can no longer access the line items.
•
N - Once a line item is received, the next time you go into Stock Receipts Entry for the same purchase order number, the line item no longer appears.
On The Water Conversion
If you use on-the-water receiving to receive merchandise from overseas vendors, this flag determines how on-the-water merchandise is received:
•
P - Use a purchase order to receive on-the-water merchandise. This method creates a new purchase order for the items in transit for the vendor. When the material arrives, you can receive the new purchase order in Stock Receipts Entry. Refer to “OTW Receiving Using a Purchase Order” in the Stock Receipts section of the Inventory documentation for more information.
•
B - Use shipping bins to receive on-the-water merchandise. This method receives the items in transit from the vendor into a shipping bin. When the material arrives, use a bin move to put the items into inventory. Refer to “OTW Receiving Using Shipping Bins” in the Stock Receipts section of the Inventory documentation for more information.
•
Y - Use container number prompting (no longer recommended or supported by Prelude).
No Stage Doc On Ship Complete
Determines if staging documents print for line ship complete orders:
•
Y - Do not print a staging document. Print a pick ticket when all line items are available.
•
N - Print only staging documents as line items are updated with the available quantity and then a pick ticket when all line quantities are available.
Refer to “Shipping Sales Orders Complete” in the Order Processing documentation for information on the options available for shipping orders or line items complete.
Auto Substitute Display?
Determines if a message displays in Sales Order Entry asking if you want to display substitutes when the available quantity is less than the order quantity:
•
Y - A message appears asking if you want to display substitutes when the available quantity is less than the order quantity.
•
N - No message displays when the available quantity is less than the order quantity, but you can press F6 to view available substitutes.
This option works with the S Product Substitute Type (Product Maintenance > F6-xref/Notes > Substitutes).
Refer to “Substitutes” in the Inventory documentation for information on setting up substitute products.
Auto A/R Credit Comments?
Determines if accounts receivable comments (Customer Maintenance > F6 > F6) automatically display when you enter the Accounts Receivable Inquiry:
•
Y - Automatically display comments.
•
N - Do not automatically display comments.
Minimum Sales As Default Qty
Determines whether to use a product’s Minimum Sell Qty, Selling Pkg Qty, or Shelf Pkg Qty (Product Maintenance screen) as the default Quantity in the line item detail screen on sales orders and quotes:
•
Y - Use the Minimum Sell Qty if it exceeds the Selling Pkg Qty.
•
N - The larger of Pkg Qty and Shelf Pkg Qty is used.
Product Number Duplicate Warning
Determines if a warning message displays when a product is entered more than once on a transaction:
•
Y - Display a warning message when a product is entered more than once. Orders that are automatically generated will not receive a warning message.
•
N - Do not display a warning message when a product is entered more than once.
Order Completion Inquiry From RCE
Determines if you can access and update the Sales Order Completion screen in Release Confirmation Entry:
•
A - Allow access to the Sales Order Completion screen for viewing or editing.
•
Y - Allow access to the Sales Order Completion screen for viewing only.
•
N - Do not allow access to the Sales Order Completion screen.
Default Quantity Type For Credits
The default inventory to use in the Line Item Credit Detail screen on a credit memo or RGA when products are returned from customers:
•
R - Return to stock (QtyStk).
•
W - Warranty (QtyWty).
•
D - Damaged (Qty-Dmg).
•
S - Scrapped (Qty-Scr).
You can override this for a credit reason. Refer to “Creating Credit Reason Codes” in the Order Processing documentation for more information.
Deposit Slip Print ABA/Check Number
Determines if deposit slips must include the ABA/check number:
•
Y - Print the ABA/check number.
•
N - Do not print the ABA/check number.
Temporary Customer Maintenance Type
Determines if Temporary Customer Maintenance assigns a valid customer number or a ‘*’ temporary number:
•
Y - Assign a valid customer number to temporary customers.
•
N - New customers are identified with ‘*’. You must convert the customer to a regular customer in order to assign a valid customer number.
You can create a temporary customer in Sales Order Entry by pressing F8 when the cursor is on the Customer field.
DEA Warning Message
Determines if a warning appears in Sales Order Entry if a customer does not have a DEA (Drug Enforcement Agency) permit on file if a product requires a permit:
•
Y - A warning message appears.
•
N - No warning message appears.
Days
The number of days before the DEA permit expires that the warning message appears. Only applicable if DEA Warning Message is set to Y.
One Step Invoicing
Determines if an invoice automatically generates on-line when the pick ticket is confirmed in Release Confirmation Entry:
•
Y - When you complete the sales order, you are taken into the Sales Order Completion screen to allow you to add any freight, taxes, or other charges. A pick ticket/invoice generates when pick tickets print so that it can be used as the packing slip and invoice. When the pick ticket is confirmed in Release Confirmation Entry, the invoice is processed on-line.
•
N - No invoice generates until the order is confirmed in Release Confirmation Entry.You must use Invoice Printing to generate the invoice.
One Step Invoicing Auto Completion
 
Continuous Counter Invoice From RCE
Determines invoices print continuously, immediately after they are confirmed in Release Confirmation Entry:
•
Y - Print invoices continuously after pick tickets are confirmed (prints and processes the order similar to a 05 counter invoice).
•
N - Do not continuously print invoices from Release Confirmation Entry.
Mail Order Mode
Determines if the mail order mode is used for sale orders:
•
Y - Use the mail order mode. When you save a sales order, the sales order completion screen automatically comes up. When you save the sales order completion screen, the cash application screen automatically comes up to allow you to enter payment information (e.g., cash, check, credit card).
•
N - Do not use the mail order mode.
Support Weight By Freight Class
Determines if weight is calculated by freight class:
•
Y - Calculate weight by freight class to appear in place of the Description on release confirmations, pick tickets, and bills of lading.
•
N - Do not use freight class to calculate weight.
Qty Break Included In Gross Price
Determines if gross price includes quantity breaks:
•
Y - Include quantity break discounts when displaying the gross price of a product for a customer.
•
N - Do not include quantity break discounts in gross price.
Update Usage On Drop Ship Orders?
Determines if direct ship orders update product usage:
•
Y - Product usage calculations include direct ship sales order usage.
•
N - Product usage does not include direct ships.
Refer to “Recording Product Usage” in the Inventory documentation.
Include COD Invoices In Service Chg
Determines if COD invoices are included in services charge invoice calculations:
•
Y - COD invoices are included in the base amount.
•
N - COD invoices are not included.
Refer to “Generating Service Charge Invoices” in the Accounts Receivable documentation for more information.
Single Ship-To Default To Null
Determines the default ship-to address to use for customers with only one ship-to:
•
Y - If an account has only one ship-to address, leave the ship-to blank.
•
N - If the account has only one ship-to address, use it as the default for sales orders.
Credit Release Expiration Days
The number of days after a credit manager releases a sales order that the credit check is repeated for the next release.
Days Before Credit Check Repeat
The number of days after a sales order goes on credit hold that the pick ticket printing program should repeat the credit check.
Explode Each S/O To Separate W/O?
Indicates if a separate line item is created in a sales order for each quantity of a serialized BOM:
•
Y - Create a separate line item in the sales order and a separate work order for each quantity. For example, if you enter an order for 3 of a serialized product, the sales order splits the quantity into 3 lines and links each to a separate work order.
•
N - Do not separate the quantity on the sales order.
C/R Upd Cash Activity Always
Determines when the last cash receipt date, amount, and check number are updated for a customer:
•
Y - Always update the cash receipt date, amount, and check number.
•
N - Only update the cash receipt date, amount, and check number when cash is applied to the customer’s account in cash receipts entry.
Qty Avail Ignores Negative Kit Comp
This flag is not applicable at this time.
Convert DOE directly to Order File?
Determines if DOE orders automatically convert to sales orders:
•
Y - Automatically convert DOE orders to the Sales Order Entry File, unless errors exist.
•
N - Always stop at the EDI Order file for manual conversion.
Use Shipping Whse For Intercompany?
Determines if intercompany general ledger postings occur in invoice processing when the default shipping warehouse differs from the line item warehouse:
•
Y - Post to intercompany G/L accounts.
•
N - Do not post to intercompany G/L accounts.
Average Cost Suppress?
Determines if average cost is updated in Skeleton Product Maintenance:
•
Y - Do not update average cost in Skeleton Product Maintenance.
•
N - Update average cost in Skeleton Product Maintenance.
Refer to “Creating Products In Sales Order Entry” in the Order Processing documentation for more information.
Valid Ship-To Entries
Determines if you can duplicate the customer’s main address and/or create a manual ship-to address in Sales Order Entry:
•
B - Allow both duplicate and manual ship-to addresses to be created in Sales Order Entry.
•
D - Only allow duplicate ship-to addresses to be created in Sales Order Entry.
•
N - Do not allow either duplicate or manual ship-to addresses to be created in Sales Order Entry.
•
M - Only allow manual ship-to addresses to be created in Sales Order Entry.
Use Commission Sales Rep For S/A?
Determines if sales analysis is updated for the commission sales rep or the original sales rep assigned to the order:
•
Y - The first salesrep assigned in the commission list of salesreps is used for sales analysis.
•
N - Salesrep One from the sales order is used for sales analysis.
Automatically Complete DOE Order
Determines if DOE (direct order entry) sales orders are automatically completed in the sales order file:
•
y - Automatically complete DOE orders in the Sales Order File. Non-confirmation types will default to PICK. Confirmation types will default to INVOICE.
•
N - Do not complete DOE orders that are set up for pick ticket printing.
Prompt For Overseas Information
Determines if you are prompted for additional overseas vendor information (e.g., Tariff Commodity Code, Country Code, and Tariff rate) when asterisk (*) part numbers are entered on a sales order:
•
Y - Prompt for overseas vendor information for asterisk (*) part numbers.
•
N - Do not prompt for overseas vendor information for asterisk (*) part numbers.
This only applies if the vendor is set up as an overseas vendor (Vendor Maintenance > F5 > F7-Importing).
Use Shipping Whse For Serv Level?
Determines if the service levels for filling orders are evaluated based on the shipping warehouse or the selling warehouse for each line item:
•
Y - Shipping warehouse.
•
N - Selling warehouse.
Pre-Assembled BOM Prompt?
Determines if a message appears to ask if you want to use pre-assembled bill-of-material products in Sales Order Entry:
•
Y - A message appears to allow you to choose if you want to use pre-assembles BOM products.
•
N - Automatically pick pre-assembled BOM quantities.
Mult PN’s Per VPN For Price Tapes
Determines if all part numbers that are assigned to one vendor product number are updated when you run the vendor price tape update:
•
Y - The vendor price tape updates all of the part numbers that are assigned to one vendor product number.
•
N - If more than one part number is assigned to a vendor product number, the vendor price tape will not update any of the part numbers.
F2-Save
Saves the option parameter information.
F5-Addit Info
Opens the ”Additional Information screen” on page 12-7 to allow you to enter additional system options.
F6-ICS
Opens the ”ICS Additional Option Parameters screen” on page 12-9 to allow you to enter additional Inventory module options.

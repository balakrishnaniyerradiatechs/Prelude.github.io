Additional Information screen
Menu Path: Systems > System Parameters > Maintenance > Option Parameters > Order Processing Parameters > F5-Add. Info
Field and Function Key Descriptions
Field/Function Key
Description
Interbranch/Surplus Only
Determines when an order goes on branch hold:
•
Y - Sales orders shipped out of another shipping warehouse will go on Branch Hold if the entire quantity cannot be filled from surplus inventory at the shipping warehouse.
•
N - Sales orders will not go on branch hold when shipping non-surplus inventory out of another warehouse.
Refer to the Branch Release Inquiry in the Order Processing documentation.
Administrative Order Hold
Defines Administrative Order Hold options:
•
N - Do not use Administrative Order Hold.
•
Y - Use Administrative Order Hold. Opens the Administrative Order Hold Parameters screen to allow you to enter order hold criteria.
Refer to the Administrative Order Inquiry in the Order Processing documentation for more information.
Administrative Invoice Hold
Defines Administrative Invoice Hold options:
•
N - Do not use Administrative Invoice Hold.
•
Y - Use Administrative Invoice Hold. Opens the Administrative Invoice Hold Parameters screen to allow you to enter invoice hold criteria.
Refer to the Administrative Invoice Inquiry in the Order Processing documentation for more information.
Administrative Quote Hold
Defines Administrative Quote Hold options:
•
N - Do not use Administrative Quote Hold.
•
Y - Use Administrative Quote Hold. Opens the Administrative Quote Hold Types screen to allow you to enter invoice hold criteria.
Refer to the Administrative Quote Inquiry in the Order Processing documentation for more information.
Partial Shpmnt Invoicing
Determines if an order is invoiced for each release if multiple releases exist:
•
Y - Sales orders are invoiced after each release.
•
N - All releases must be confirmed before the sales order is invoiced.
RFQ Generation
Determines if Vendor RFQs (requests for quotes) can be generated from Quote Order Entry if the quoted products are not available:
•
Y - Vendor requests for quotes can be generated from quotation entry if products are not available.
•
N - Vendor requests for quotes cannot be generated from quotation entry if products are not available.
‘*’ N/S Price/Cost Prompt
Determines if both price and cost are required for nonstock (*) items on sales orders:
•
Y - Request for both list price and cost.
•
C - Request for cost only.
Print B/O On Pick Tickets
Determines if backordered items are included on pick tickets:
•
Y - Include backordered items on the initial pick ticket.
•
N - Do not include backordered items on the pick ticket.
•
A - Always print backordered items on all pick tickets.
You can override this flag for a customer in Customer Maintenance > F7-Order > F5-Add Info > B/O on Pick Tickets.
Written By Commission
Determines if commission is also calculated for the sales order’s Written By (order taker):
•
Y - Calculate commission for the Written By in addition to Salesrep One and/or Salesrep Two.
•
N - Do not calculate commission for the Written By.
Consolidate Consign By Cust/ShipTo
Determines if consignment orders are consolidated by customer number or by ship-to number:
•
C - Consolidate by customer number.
•
S - Consolidate by customer ship-to number.
You can override this setting for a customer in Customer Maintenance > F7-Order > Consign Consolidate Method. Refer to “Using Consignment Orders” in the Order Processing documentation.
Disallow Same Lot# For Different P/N’S
Indicates if the same lot number is allowed for 2 part numbers on sales orders:
•
Y - You cannot use the same lot number for different product numbers on a sales order.
•
N - You can use the same lot number for different product numbers on a sales order.
This overrides the Disallow Same Lot# For Diff P/N’S flag in the ICS Additional Option Parameters screen.
Refer to “Setting Up a Lot Product” in the Stock Receipts section of the Inventory documentation.
Pick Ticket Suppress w/No CC Auth Code
Determines if pick tickets can print for credit card orders without a credit card authorization:
•
Y - Do not print pick tickets for credit card orders without a valid credit card authorization.
•
N - Allow pick tickets to print for credit card orders without a valid credit card authorization code.
Exclude Install Terms From Cred Hold
Indicates if changing payment terms to installment terms causes a sales order to go on credit hold.
•
Y - Orders do not go on automatic credit hold when a sales order’s Payment Terms are changed to installment terms.
•
N - Sales orders go on automatic credit hold when the Payment Terms are changed to installment terms.
Installment payment terms allow payments to be made using an installment plan. A single invoice is created for each payment with a new calculated discount and due date for each invoice. Refer to “Setting up A/R Payments terms in the Accounts Receivable documentation.
Utilize Freight Clearing House Method
No longer applicable.
Sales Tax Matrix
Determines if you can override a tax rate or taxable status for a product or group of products for a customer:
•
Y - You can override the sales tax matrix.
•
N - You cannot override the sales tax matrix.
•
S - Tax is determined by the ship-to state on the order rather than the tax jurisdiction.
Sales Tax Matrix Maintenance allows you to set up sales tax matrices to calculate tax rates based on customers, products, and tax group combinations. When you enter a sales order using the customer/product combination, the tax rate in the sales tax matrix overrides any other sales tax calculations. Sales tax matrices can be set up for the following:
•
1 - Customer/Product.
•
2 - Customer Tax Group/Product.
•
3 - Customer/Product Tax Group.
•
4 - Customer Tax Group/Product Tax Group.
•
5 - Customer Ship-To/Product.
•
6 - Customer Ship-To/Product Tax Group.
Refer to the “Sales Tax Matrix” information in the Order Processing documentation.
Drop Ship Price Matrix
Indicates if separate price matrices are kept for direct shipments:
•
Y - Separate matrices are maintained for direct ships. In this case you must maintain two matrices, one for warehouse sales and one for drop ship (direct) sales, even if they are the same. Price Matrix Maintenance prompts for a direct ship price.
•
N - The same matrix pricing applies to warehouse and direct orders.
Utilizing Split Dial For Icverify
If you use the IC Verify credit card verification system, determine if 2 processing networks are used to process credit cards:
•
Y - IC Verify uses 2 processing networks.
•
N - IC Verify does not use 2 processing networks.
IC Verify is no longer supported by Activant Prelude.
Driver Lic Req/Checks
Indicates if a driver’s license is required when a customer pays for an order by check:
•
Y - A driver’s license is required.
•
N - A driver’s license is not required.
Credit Bureau Auth Code
Defines if you can manually enter the Authorization Code in the Credit Card Information screen on a sales order.
•
Y - Manual authorization codes are allowed.
•
N - Use with the MAPP provider (no longer supported).
•
I - Use the IC Verify online verification software. Opens the Add CC Software Info screen (no longer supported).
•
P - Use the Protobase on-line verification software. You cannot access the Authorization Code in Sales Order Entry because the authorization is done on-line. Opens the Add CC Software Info screen to allow you to enter the credit card software requirements. In situations where a manual authorization code is necessary (e.g., power outage) you can use Sales Order Credit Card Authorization Entry to manually enter an authorization number.
Refer to “Credit Cards” in the Order Processing documentation for more information.
Check Validation Auth Code
Determines if you can manually enter the Authorization Code for a check in the Additional Check Information screen which appears when you select the 02-Check payment type when taking payment for an order:
•
Y - You can manually enter a check authorization code.
•
N - You cannot manually enter a check authorization code. This applies
Cust Security Access
Indicates if the security access feature for customer accounts is activated:
•
A - Activates the security access feature for customers. Only users that have a security access code set up in their User Identification Maintenance Customer Security Access Codes screen can access or maintain customers that have the same access code set in Customer Maintenance > F6-A/R > Action Bar > Security Access.
•
Y - Same as A, except that if a customer does not have a security code, any user can access the account.
•
N - Security access is not activated for Customer Maintenance.
Security access codes are created in the Security Access Code Maintenance screen.
Prod.Security Access
Indicates if the security access feature for products is activated:
•
Y - Activates the security access feature for products. Only users that have a security access code set up in their User Identification Maintenance Product Security Access Codes screen can access or maintain products that have the same access code set in Product Maintenance > F5-Add Info > Prod Security Access.
•
N - Security access is not activated for Product Maintenance.
Security access codes are created in the Security Access Code Maintenance screen.
Vend. Security Access
Indicates if the security access feature for vendors is activated:
•
Y - Activates the security access feature for vendors. Only users that have a security access code set up in their User Identification Maintenance Vendor Security Access Codes screen can access or maintain vendors that have the same access code set in Vendor Maintenance > F5-Add Info > Vend Security Access.
•
N - Security access is not activated for Vendor Maintenance.
Security access codes are created in the Security Access Code Maintenance screen.
Inc Fut Bal/Cred Check
Determines if future orders (orders with a future committed balance) are included in the credit limit check:
•
Y - Future orders are included when calculating a customer’s credit balance for credit hold.
•
N - Future orders are excluded from credit balance checking for credit holds.
Auto Split Whse Shpmnt
Automatic assignment of sales order detail lines to ship from other warehouses:
•
Y - A warning message appears to let the salesperson know that the product can ship from another warehouse to the customer.
•
N - Do not automatically assign line items on a sales order to another shipping warehouse.
•
W - Automatically assign the line item to ship out of another warehouse without warning the salesperson.
Auto B/O Cancellation
Works with Accept Backorders in Customer Maintenance > F7-Order > F5-Add Info to determine if backorders are automatically cancelled:
•
Y - If the customer does not accept backorders, all backorders will be cancelled when the order is confirmed in Release Confirmation Entry. If the customer accepts backorders, the material will backordered for the customer.
•
N - If the customer does not accept backorders, you can override the customer’s backorder flag in Release Confirmation Entry to allow the material to go on backorder anyway. If the customer accepts backorders, the material will be backordered.
Upd ‘07’ Order To Bookings
Determines if type 07 consignment orders update the bookings file:
•
Y - Type 07 consignment orders update the bookings file.
•
N - Type 07 consignment orders do not update the bookings file.
Bookings are the amount for which you obtained orders, but are not necessarily invoiced.
Allocate By Whole Unit
If you use the Unit of Measure Table, you can determine if a fraction of a pickable quantity should be allocated:
•
Y - Do not allocate a quantity until the full quantity for the unit of measure is available. For example, if the unit of measure is CS (case) with 24 each is a case, the quantity is not allocated until a full case of 24 each is available.
•
N - Allocate any quantity available, regardless of the unit of measure. For example, if the customer ordered one case of 24 each and only 12 are available, the system allocates .5 cases.
If a product is purchased, stocked, and sold in different units of measure, you must set up a unit of measure table for the product in Product Maintenance. Refer to “Establishing a Unit of Measure” in the Inventory documentation.
Exclude From Credit Limit
Determines if the credit manager can approve a large sales order and exclude it from the credit limit calculation so that additional sales do not go on credit hold:
•
Y - After a sales order is released in Credit Release Inquiry, a message appears to allow the credit manager to exclude the order from the credit limit check (refer to ”Excluding Sales Orders From the Credit Limit” in the Accounts Receivable documentation).
•
N - The credit manager cannot exclude an order from the credit limit check while in Credit Release Inquiry.
Suppress Credit Messages
Determines if the sales person is notified in Sales Order Entry when a sales order goes on credit hold:
•
Y - Messages regarding the credit status do not appear in Sales Order Entry when the sales order goes on credit hold. The person entering the order will not know if it is on hold.
•
N - When a sales order is entered for a customer on credit hold, a message appears after the customer number is entered to let the salesperson know that the customer is on credit hold. The salesperson can send a note to the credit manager upon completion of the order.
Open Quote Message
If Display Open Quote/RGA Message is set to Y on the Order Processing Parameters screen, indicates where the open quotation warning message appears:
•
H - Header level.
•
L - Line item level.
•
B - Both.
Quote Message All Warehouses (Order Processing Parameters > F5 > F5 > F5) determines if the quote message appears for all warehouses or for the selling warehouse only.
Include Converted Quotes
If Display Open Quote/RGA Message is set to Y on the Order Processing Parameters screen, indicates if converted quotes are included in the warning message:
•
Y - List converted quotes in the warning message.
•
N - Do not list converted quotes.
Upd Sales For Mast/Child
Determines if sales information is updated for both the master and child accounts:
•
Y - Update both master and child account information.
•
N - Only update the child account information.
This flag does not affect how the A/R balance is updated. Master Acct Invoice in Customer Maintenance > F6-A/R determines accounts receivable information.
Sales Order Rationing
Determines if sales order rationing is activated:
•
Y - Sales order rationing is activated.
•
N - Sales order rationing is not activated.
Sales order rationing allows you to limit the quantity of a product or product line that a customer can buy within a designated number of days. For example, if the customer’s ration quantity (set in Customer Maintenance) of an item is 100 units and the ration days for that item is 30, the customer can purchase up to 100 units of that item within the rolling 30-day time frame. Ration comments appear on the sales order when the ration quantity is exceeded.
Refer to “Entering Rationed Products” in the Order Processing documentation for more information.
F2-Save
Saves the parameter information and returns to the Order Processing Parameters screen.
F5-Add. Info
Opens the Additional Options screen to allow you to access additional Order Processing option parameters.

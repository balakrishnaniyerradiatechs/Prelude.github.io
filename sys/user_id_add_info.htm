Additional Information screen
The Additional Information screen in User Identification Maintenance allows you to set additional options and restrictions for a user.
Menu Path: Systems > System Parameters > Maintenance > User Identification Maintenance > F6-Add Info
Field and Function Key Descriptions
Field/Function Key
Description
Display Cost?
Determines if the user can access or view cost and gross profit amounts in order entry and sales inquiry programs for this user:
•
Y - Display commission cost. Determined by how Commission Cost is set in the Order Processing Parameters screen.
•
A - Display accounting cost. Determined by how G/L Cost is set in the Company Maintenance Additional Information screen.
•
N - Do not display cost.
Price Override Code
Determines if this user can override a price on a sales order or quote:
•
A - The user can change the price, even below cost.
•
C - The user can change the price, but not below cost.
•
G - The user can change the price, but if the gross profit falls below the minimum for the product line, the order goes on administrative hold.
•
N - The user cannot access any price fields.
•
R - The user can raise the price above the customer’s default price, but if he/she lowers it below the default price, the order goes on administrative hold.
Refer to the “Administrative Hold Inquiries” in the Order Processing documentation for more information on the criteria that places an order on hold.
Pick Qty Override
Determines if the user can override the pick quantity on a sales order, work order, or transfer:
•
Y - The user can increase or decrease the pick quantity.
•
R - The user can only reduce the pick quantity.
•
N - The user cannot change the pick quantity.
Enter Credit Memos
Determines if the user can enter credit memos:
•
N - The user cannot enter credit memos or RGAs.
•
R - The user can enter RGAs but no credit memos.
•
Y - The user can enter both credit memos and RGAs.
Enter P/O from S/O
Determines if the user can create a purchase order from within Sales Order Entry:
•
Y - The user can create a purchase order in Sales Order Entry.
•
N - The user cannot create a purchase order in Sales Order Entry.
Enter W/O from S/O
Determines if the user can create a work order from within Sales Order Entry:
•
Y - The user can create a work order in Sales Order Entry.
•
N - The user cannot create a work order in Sales Order Entry.
Enter T/R from S/O
Determines if the user can create a stock transfer from within Sales Order Entry:
•
Y - The user can create a stock transfer in Sales Order Entry.
•
N - The user cannot create a stock transfer in Sales Order Entry.
Enter P/O from W/O
Determines if the user can create a purchase order from within Work Order Entry:
•
Y - The user can create a purchase order in Work Order Entry.
•
N - The user cannot create a purchase order in Work Order Entry.
Enter T/R from W/O
Determines if the user can create a stock transfer from within Work Order Entry:
•
Y - The user can create a stock transfer in Work Order Entry.
•
N - The user cannot create a stock transfer in Work Order Entry.
Source P/O from S/O
Determines if the user can link a sales order line item to an existing purchase order:
•
Y - The user can link a sales order line item to an open purchase order.
•
N - The user cannot link a sales order line item to an open purchase order.
Modify GRI Process
Determines if the user can add additional selection criteria to reports and processes that use the general report interface (GRIM):
•
Y - The user can modify GRIM criteria.
•
N - The user cannot modify GRIM criteria.
Schedule GRI Process
Determines if the user can schedule reports and processes using the general report interface (GRIM):
•
Y - The user can schedule reports and processes but cannot access Scheduler Maintenance.
•
A - The user can schedule reports and processes and can access Scheduler Maintenance.
•
N - The user cannot schedule reports and process.
Enforce Auth in GRI
Determines if the user must enter a value in the G/L Prefix fields (e.g., Division, Warehouse) in the GRIM selection screens:
•
Y - The user must enter a value in the prefix fields.
•
N - The user can leave a prefix field blank to select all warehouses, divisions, etc.
Email Reports
Determines if the user can email reports:
•
Y - The user can email reports.
•
N - The user cannot email reports.
Documentation Option
Determines if and how the user can view the on-line documentation accessed from the help key on the Action Bar:
•
F - The user has full access to the on-line documentation including the search key, index, and table of contents for the entire module.
•
I - The user can only view the on-line documentation specific to the screen they are viewing.
•
N - The user cannot view the on-line documentation.
QOE Cst Ov-B/D Supp Cde
Determines if the user can override a cost in Quote Order Entry if the line item has a supply code of B (purchase order) or D (direct purchase order):
•
Y - The user can change the cost in Quote Order Entry, which carries over to the sales order and purchase order. If you use warehousing, Allow Products in RF SRE in the Warehousing Receiving Maintenance screen (Warehouse Maintenance > F7 > F5 > F8) must be set to Y.
•
N - The user cannot change the cost in Quote Order Entry.
Add New Products in SRE
Determines if the user can enter new products that do not exist on the original purchase order when receiving stock:
•
Y - The user can add products when receiving stock.
•
N - The user cannot add products when receiving stock.
Disallow Quote Conversion
Determines if the user can convert a quote into a sales order either from Quote Order Entry or from Sales Order Entry:
•
Y - The user cannot convert a quote to a sales order.
•
N - The user can convert a quote to a sales order.
Refer to “Converting a Quote to a Sales Order” in the Quotation Subsystem section of the Order Processing documentation for more information on methods to convert quotes.
Inspection Bin Movement
Determines if the user can move products from inspection bins into regular inventory bins:
•
Y - The user can move products from inspection bins.
•
N - The user cannot move products from inspection bins.
Refer to “Requiring Quality Control Inspections in Receiving” in the Inventory documentation for information on Quality Control Inspection.
Require Original Invoice#
Determines if the user must enter the original invoice number when entering a credit memo for a returned item:
•
Y - The user must enter the original invoice number.
•
N - The user does not have to enter the original invoice number.
Refer to “Credit Memos” in the Order Processing documentation for more information.
PDA User?
Determines if the user maintains a PDA (personal data assistant) that works with Prelude:
•
Y - The user uses a PDA.
•
N - The user does not use a PDA.
Contact Type
Indicates the type of contact that this user is classified as (if applicable):
•
ALL - All types.
•
CR - Credit department.
•
ENG - Engineering department.
•
FS - Fire/safety department.
•
MGR - Management.
•
MKT - Marketing.
•
NA - Not applicable.
•
RR - Receiving/purchasing.
•
SLS - Sales.
•
SM - Seminars.
•
SRV - Service.
•
TS - Technical support.
•
WTY - Warranty.
Edit Delay
The number of milliseconds SB client will wait before activating Edit and Function keys. Leave blank to use system defaults. This option is used for systems that have timing issues with network protocols or modems.
Enter P/O For Stock
Indicates if the user can add stock items in Purchase Order Entry and Purchase Order Conversion Entry (in addition to items linked to sales orders):
•
Y - The user can add stock items to purchase orders.
•
N - The user cannot add stock items to purchase orders (unless they are linked to a sales order).
Automatic Login From Unix
Unix Id
The user’s Unix ID if using the Unidata/SBclient automatic login. The Bypass SB Login - Menu Update process must be run so that the user can escape and automatically close the SBclient session without the SB login screen.
Sbpassword
The SB password for the Unix Id.
•
NULL - Indicates that no SB password is set up. Only the UNIX login is maintained for the user.
•
If you enter an SB password, it must be established for the SB security user and User Identification Maintenance.
F2-Save
Saves the user ID information.
F5-Cust Sec.
Opens the Customer Security Access Codes screen to allow you enter security access codes that this user can enter to access customers that have the same security access codes.
F6-Prod Sec.
Opens the Product Security Access Codes screen to allow you to enter security access codes that this user can enter to access products that have the same security access code.
F7-Vend Sec.
Opens the Vendor Security Access Codes screen to allow you to enter security access codes that this user can enter to access vendors that have the same security access code.
F8-Bank Sec.
Opens the Bank Code Authorizations screen to allow you to enter the bank codes that this user can access.

Administrative Settings screen
The Administrative Settings screen in User Identification Maintenance allows you to assign a manager code for each work queue that a user has access to. You can also set up limits and restrictions for a user.
Menu Path: Systems > System Parameters > Maintenance > User Identification Maintenance > F5-Administrative
Field and Function Key Descriptions
Field/Function Key
Description
Credit Manager
Determines if the user is a credit manager:
•
Y - The user is a credit manager and can release orders from credit hold in the Credit Release Inquiry. Enter the credit manager code in the next field.
•
N - The user is not a credit manager.
•
A - The same as the Y option but the user can also release an order from credit hold from within Sales Order Entry.
Refer to ”Releasing Orders From Credit Hold” in the Accounts Receivable documentation for more information.
Manager Code (unlabeled field)
The credit manager code set up in the Manager Maintenance screen that determines the work queue and the warehouses this user can release orders from. Once this user is set up as a credit manager, their user ID appears in the Manager Maintenance screen under Current User Assignment Settings.
Authority to Enter Other Manager Codes
(unlabeled field)
Determines if the user can enter other manager codes in the Credit Release Inquiry:
•
Y - The user can enter other manager codes in the Credit Release Inquiry.
•
N - The user can only view and release orders from his/her own credit release queue.

Machine Name
The Computer name from the Information screen on Prelude Notify for the user. This is only necessary if you choose A for Notification Method (next field).
Notification Method
Determines how the user is alerted when one if their work queues requires attention (e.g., Credit Release Inquiry, Branch Release Inquiry, Administrative Order Release Inquiry):
•
A - Prelude Notify. You must download Prelude Notify from the Activant Customer Website and set the Options so that either an animated character or a pop-up window appears anytime a new entry updates the user’s work queue. You must also enter a Machine Name in the previous field.
•
D - Monitor Display. The tool bar icons for each work queue turn red or green.
•
E - Email notification. An email is sent to the user when a work queue requires attention.

Credit Limit Amount
The largest amount for which this user can issue a credit without the order going on administrative invoice hold. If this field is blank, the user can issue a credit for any amount.
C/R Write-Off Limit
The largest amount that this user can write-off as a miscellaneous charge in Cash Receipts Entry. Refer to “Writing off the Unpaid Balance” in the Accounts Receivable documentation. If this field is blank, the user can write-off any amount.
A/P Write-Off Limit
The maximum amount that this user can write-off on a single invoice in Vendor Invoice Reconciliation Entry before the invoice goes on hold in the Accounts Payable Release Inquiry. The invoice can be released by the A/P Manager assigned in the Warehouse Manager Maintenance screen.
If this field is blank, the user can write-off any amount.
Vend Inv Rec Limit
The largest invoice amount that this user can reconcile in Vendor Invoice Reconciliation Entry before the invoice goes on hold in the Accounts Payable Release Inquiry. The invoice can be released by the A/P Manager assigned in the Warehouse Manager Maintenance screen.
If this field is blank, the user can reconcile any amount.
Vend Inv Recost%
The largest percentage that this user can recost a purchase order in Vendor Invoice Reconciliation Entry before the invoice goes on hold in the Accounts Payable Release Inquiry. The invoice can be released by the A/P Manager assigned in the Warehouse Manager Maintenance screen.
If this field is blank, the user can recost a P/O for any amount.
Authorization Limit
The maximum sales order amount for which a user can release an order from credit hold. If this field is blank, the user can release an order for any amount.
Max Order Disc
The maximum amount that this user can enter in Disc/Allowance on the Sales Order Completion and the Quotation Completion screens. If this field is blank, then any Disc/Allowance can be entered.
This field overrides Maximum Order Discount in the Additional Options screen (Order Processing Parameters > F5-Add. Info > F5-Add Info).
Quote Limit
The largest amount for which this user can enter a quote without it going on Administrative Quote Hold. If this field is left blank, the user can enter a quote for any value.
P/O Limit
The largest amount for which this user can enter a purchase order without it going on hold. If this field is left blank, the user can enter a purchase order for any value.
If the warehouse on the purchase order has a Purchasing Manager assigned in the Warehouse Manager Maintenance screen, then the order will go into the Purchase Hold Release work queue for that manager. If no manager is assigned, then the buyer assigned to the purchase order is notified.
IAE Line Limit
The largest amount for which this user can adjust inventory on a single line in Inventory Adjustment Entry for INV (inventory) type adjustments. If this field is blank, the user can adjust inventory for any amount.
View Credit Card #
Determines if the user can see all of a credit card number for a customer account or if they can only see the last 4 digits:
•
Y - The user can see the entire credit card number.
•
N - The user can only see the last 4 digits of the credit card number.
Refer to “Credit Cards” in the Order Processing documentation for more information.
Journal Entry Limit
The largest total amount that this user is authorized to post in a journal entry. Refer to the “Open Journal Entry Inquiry” in the General Ledger documentation for more information.
F2-Save
Saves the Administrative Settings Information.
F5-Authorized Codes
Opens the Authorized Administrative Hold Codes screen to allow you to enter the authorized admin order hold and admin invoice hold codes for this user.

Pay Type Maintenance screen
Each job on a repair requires a pay type. Pay types can specify a vendor or customer, provide warranty information, and indicate how the job should be billed.
Menu Path: Shop Repair > Maintenance > Pay Type Maintenance
Field/Function Key Descriptions
Field/Function Key
Description
Pay Type
A code that represents a type of payment (e.g., BILL - the customer is to be billed, WTY - the product is under warranty and the customer is not to be billed). Up to 6 alphanumeric digits.
Description
A description of the code (e.g., Warranty, Customer Bill).
Customer
An optional field that specifies a customer number to reference a price matrix for pay types.
Price Group
A price group classification for the labor type, if applicable. Price groups are taken from Customer Price Group Maintenance.
Charge Customer?
Indicates if the customer should be billed for jobs with this pay type:
•
Y - The customer is to be charged for jobs with this pay type.
•
N - The customer is not to be charged for jobs with this pay type.
Capitalize?
Determines if the dollars for this pay type are considered no- charge and posted to Add-On Capital for the equipment ID number:
•
Y - If the Charge Customer? flag is set to N, the system creates a general ledger entry when the shop repair order is invoiced. The dollars for this pay type are considered no- charge and posted to Add-On Capital for the equipment ID number.
•
N - The dollars for the shop repair order are not capitalized.
Warranty Flag
Indicates if jobs with this pay type are for warranty work and are submitted to the vendor for reimbursement:
•
Y - Pay type is for warranty work. A vendor claim is automatically created for jobs with this pay type. The vendor in Warranty Vendor# is used for the vendor claim.
•
N - Pay type is not for warranty work.
Submit Electronic
Applies to warranty pay types only. Indicates if the vendor claim for the warranty work will be electronically submitted for payment:
•
Y - Jobs with this pay type are electronically submitted for warranty approval.
•
N - Jobs with this pay type are not electronically submitted for warranty approval.
Require Main SRT?
Applies to warranty pay types only. Indicates if a standard rate table (SRT) is required for jobs with this pay type (refer to Entering Warranty Information for a Job):
•
Y - SRT is required when entering warranty information for a job.
•
N - SRT is not required when entering warranty information for a job.
Require Failed PN?
Applies to warranty pay types only. Indicates if the number of the part that failed is required for jobs with this pay type (refer to Entering Warranty Information for a Job):
•
Y - A failed part number is required for the job.
•
N - A failed part number is not required for the job.
Price Labor At Cost
Determines how Technician Job Billing Entry should price labor line items:
•
Y - Price labor line items at the labor rate set up for the technician in Technician Maintenance (refer to Setting up Technicians).
•
N - Price labor line items at the retail price for the labor type.
(refer to Entering Billable Time for Technicians).
Operating Expense
If this is a rental product, this flag determines if the pay type updates non-operating expenses for the equipment:
•
Y - Posts to Non-Operating Expenses (Rental > Equipment Subsystem > Maintenance > Equipment Maintenance > F8-Fin).
•
N - Do not post to Non-Operating Expenses.
Rework?
Determines if the pay type is used with rework jobs:
•
Y - The pay type if for a rework job (e.g., a shop repair job that is returned for repair). The Rework Flag in Shop Repair Job Entry defaults to Y.
•
N - The pay type is not for a rework job.
Warranty Vendor#
Applies to warranty pay types only. If applicable, the default vendor number to use for warranty submittal.
Shop Charges Code
If applicable, the shop charges code(s) that represent additional charges for this pay type. For more information on shop charges, refer to Creating Shop Charges.
F2-Save
Saves the pay type code information.
F4-Del
Deletes the pay type.
F5-G/L Overrides
Opens the Shop/Repair Payment Type G/L Overrides screen, which allows you to post the pay type to a different general ledger account. If the pay type is no charge, A/R must be offset. This would also need to be offset for a non-inventory expense (type 04 in vendor claims).
F6-Shop Charges
Opens the Ship Charges Screen to allow you to enter the shop charges code(s) that represent additional charges for this pay type. For more information on shop charges, refer to Creating Shop Charges.

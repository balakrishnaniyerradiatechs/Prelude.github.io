Pay Type/Product Line Maintenance screen
By default, all accounts are at the company-level in the GL table. The Pay Type/Product Line Maintenance screen allows you to override those accounts by pay type or by pay type/product line combination.
Menu Path: Shop Repair > Maintenance > Pay Type/Product Line Maintenance
Field/Function Key Descriptions
Field/Function Key
Description
Pay Type
The pay type that requires G/L overrides. Refer to Creating Pay Types for more information.
Product Line
The product line that requires G/L overrides.
G/L Table Headings
The name of the G/L account.
Div
The highest possible grouping in your company (e.g., Eastern Division, Western Division).
Reg
A subset of Division - if Division is used (e.g., Southeast Region, Northwestern Region).
Loc
The warehouse (e.g., Dallas, Tampa).
Dep
A subset of Location/Warehouse. Typically used to separate expenses within a warehouse (e.g., accounting, sales, warehouse).
Ccr
A subset of Department - if Department is used. Typically used to separate expenses within a warehouse or department (e.g., shipping, receiving, outside sales, inside sales).
Account#
The G/L account number.
F2-Save
Save the G/L overrides.
F4-Del
Deletes the G/L overrides.
Action Bar
Print
Prints the G/L overrides.

Open Repair Quote Report
The Open Repair Quote report lists quotes that have not yet been converted into open shop repair orders.
Menu Path: Shop Repair > Shop Quote Subsystem > Open Repair Quote Report
Selection Criteria
Use the following criteria to limit the information to print on the report:
•
Division. The highest possible grouping in your company (e.g., Eastern Division, Western Division).
•
Region. A subset of Division - if Division is used. (e.g., Southeast Region, Northwestern Region).
•
Selling Warehouse. The warehouse quoting the repair.
•
Department. A subset of Warehouse. Typically used to separate expenses within a warehouse (e.g., accounting, sales, warehouse).
•
Cost Center. A subset of Department - if Department is used. Typically used to separate expenses within a warehouse or department (e.g., shipping, receiving, outside sales, inside sales).
•
Customer. The customer receiving the quote.
•
Technician. The person who is assigned to or worked on the job. For more information on technicians, refer to Setting up Technicians.
•
Pay Type. How the repair will be billed (e.g., customer bill, warranty, no charge). Pay types may affect pricing and shop charges. Refer to Creating Pay Types.
•
Repair Group. The type of repair being performed (e.g., electrical, transmission). Refer to Identifying Repair Groups.
•
Quote Date range. A beginning and ending range for the date on which the quote was entered.
•
Model. The part number of the unit that needs to be repaired.
•
Serial Number. The serial number of the unit that needs to be repaired.
Column Descriptions
Column Heading
Description
Quote#
The shop repair quote number.
Job
The job number (e.g., A, B, C) and description. Quotes can have multiple jobs. Refer to Entering a Job.
Product#/
Description
The number and description of the product needed to repair the model.
Customer#/
Description
The number and description of the customer on the order.
Qte-Date
The day the quote was entered.
Model
The model number of the unit being repaired (if any).
Tech
The technician who is assigned to or is working on the repair. Refer to Setting up Technicians.
Serial#
The serial number of the unit being repaired (if any).
Rep Grp
The code for the type of repair (e.g., 01, 02). Refer to Identifying Repair Groups.
Pay Type
How the repair will be billed (e.g., customer bill, warranty, no charge). Refer to Creating Pay Types.
SC
The system-generated supply code. The supply code only appears if the product is back ordered and is being replenished using one of these replenishment methods:
•
B - Purchase.
•
T - Transfer.
•
D - Direct.
•
M - Manufacture.
•
O - Outplant.
•
E - Emergency purchase order.
•
R - Emergency transfer.
Qty Opn
The quantity of the product needed to repair the model.
U/M
The product’s unit of measure.
Prm-Dte
The date by which the repair should be completed.
Status
The status of the job:
•
Completed.
•
Tech clocked in.
•
Open.
•
Quotation.
•
Pending warranty response.
•
Pending warranty submittal.
•
Tech clocked out.
•
Waiting on parts.
•
Not completed/this invoice.
Price
The price for one unit of measure.
Price Ext
The price for the total quantity ordered (Qty-Opn * Price).

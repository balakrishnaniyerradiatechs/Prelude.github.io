Creating the Base and Child Products
A base product is a product that is available for sale in multiple configurations (e.g., size, color, length). You must first set up the configuration codes (refer to Setting Up Configuration Codes) so that you can assign the codes to the base product in Product Maintenance. When you sell the configured product, only the base product is available as a valid product number. When you enter the base product on an order, you must choose from the configured child products available.
To assign a configuration code to a base product:
1
Select Inventory > Maintenance > Product Maintenance > Product Maintenance to open the Product Maintenance Screen.
2
For Product, enter the base product.
3
Press F5-Add Info to open the Additional Information Screen.
4
For Special Operation, enter CB to identify the product as a configurable base product. The Product Configuration Maintenance screen appears.
5
For Sales Analysis Flag, determine where to record sales analysis information. Choose from the following:
•
Parent - Update sales information at the parent (base) level.
•
Child - Update the sales information at the child level.
6
Enter the Configuration Codes established in Configuration Code Maintenance that apply to this product (e.g., size, color, length). Refer to Setting Up Configuration Codes).
7
Enter as many valid Configuration Responses available for the code that apply to the base product (e.g., red, white, blue). You can only enter responses that are set up as Options in Configuration Code Maintenance.
8
Press ENTER to go back to Configuration Codes to enter another code and any valid responses for the product.
Press F5-Response Toggle to move back and forth between Configuration Codes and Configuration Responses to view or make changes.
9
After you enter all the valid codes and responses, press F6-Configuration Worksheet to open the Product Generation Worksheet screen.
All proposed new product combinations based on the configuration codes and responses display. Any existing child products do not appear.
10
Delete any products that are not applicable or override the product description if necessary.
11
Once the list of child products is correct, press F5-Create Children to convert the list into valid child product numbers. The following message appears:
9 children products created.
Continue
12
Press ENTER to complete the product creation.
The new child products numbers have a Special Operation Code of CC (configured child) in the Product Maintenance > F5-Add Info screen. Any changes to the base product, such as price or package information, are automatically applied to the child products if they are the same.
If you want one child product to have a different price than the others, you can override the price in Product Maintenance. When the price for the base product is updated, you must manually update the price for the child product that has a different price.

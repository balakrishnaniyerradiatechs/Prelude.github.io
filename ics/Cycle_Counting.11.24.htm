Cycle Count Second Level Report
After entering the first round of cycle counts (refer to Entering 1st Level RF Cycle Counts), you can request a recount for any products whose first level count created a discrepancy outside the acceptable cost/tolerance limits.
The Cycle Count Second Level Report includes items with discrepancies and any additional items selected for counting (refer to Requesting Additional Cycle Counts).
Menu Path: Inventory > Physical Inventory > Cycle Counting Procedure > Cycle Count Second Level Report
Selection Criteria
Use the following criteria to limit the information to print on the report:
•
Division. The highest possible grouping in your company (e.g., Eastern Division, Western Division).
•
Region. A subset of Division - if Division is used (e.g., Southeast Region, Northwestern Region).
•
Department. A subset of warehouse. Typically used to separate expenses within a warehouse (e.g., accounting, sales, warehouse).
•
Cost Center. A subset of Department - if Department is used. Typically used to separate expenses within a warehouse or department (e.g., shipping, receiving, outside sales, inside sales).
•
Warehouse. The warehouse that requires the recount sheets.
•
Product Page Break. Indicates if you want a page break:
-
Y - Page break by product.
-
N - No page break.
•
Sort D/B/S (w/o Lots). Determines how to sort the report (if there are no lot numbers):
-
D - Problem bins last.
-
B - By bin number.
-
S - Problem bins first.
Column Descriptions
Column Heading
Description
Product Number
Line 1: The product number selected for recounting.
Line 2: The product description.
Avg Cost
The average cost of the product.
Bins
Line 1: The bin location of the product.
Line 2: The secondary bin location for the product if one exists.
Additional locations display if the product exists in additional bins.
Lot Number
The lot number assigned to the product.
P
An asterisk (*) indicates that the item is on the report because of a count discrepancy problem rather than a requested recount.
Date
The date the product went into the second level queue.
Time
The time the product went into the second level queue.
Counter
The user ID of the person who counted the material.
Source
Indicates how the product was added to the second level queue:
•
BCCRE - Bin First Level Cycle Count Request Entry.
•
CCRE - Cycle Count Request Entry.
•
ECCI - Emergency Cycle Count Request Entry.
Disc Qty
The discrepancy quantity for the bin. The difference between the previous on-hand quantity and the 1st level count quantity.
Sys Qty
The current system on-hand quantity from the first level count for the product and bin.
Count Qty
A space to enter the second level count.

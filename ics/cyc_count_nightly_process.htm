Cycle Count Nightly Processing
Cycle Count Nightly Processing updates the bin count queue with the bins to count each day. The bin count queue is sorted by bin creation date to make sure products are counted on a first in/first out basis.
To determine bins to count, Cycle Count Nightly Processing checks:
•
The list of bins already in the pending first level queue for the warehouse that are waiting to be assigned and counted. If a bin already exists, it will not be added again.
•
The status of each frequency in Cycle Count Control Maintenance to determine if it is included in the daily cycle count.
•
The number of bins to count for the day for each frequency (calculated by Cycle Count Monthly Processing).
•
The list of bins for each frequency to determine the number of bins for each frequency.
For example:
If Monthly Processing determines that a zone with a bin count frequency of 6 should have 3 bins counted each day, then Nightly Processing determines the last bin selected and takes the next 3 in the list for the next count cycle.
To run nightly processing for cycle counting:
1
Select Inventory > Physical Inventory > Cycle Counting Procedure > Cycle Count Nightly Processing to open the Cycle Count Process Warehouse screen.
2
Enter the Warehouse number.
3
Choose Execute. You can either press ENTER to start the process or enter a time to start.

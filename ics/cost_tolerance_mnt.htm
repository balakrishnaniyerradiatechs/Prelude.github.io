Cost/Tolerance Maintenance screen
The Cost/Tolerance Maintenance screen defines the percent a count can differ from the on-hand quantity in the first level count before a second level count is required.
Menu Path: Inventory > Physical Inventory > Cycle Counting Procedure > Cost /Tolerance Maintenance
Field and Function Key Descriptions
Field/Function Key
Description
Cycle Count Type
Indicates how variances are calculated:
•
C (or null) - Variances are calculated on the unit cost. For example, if a product with a cost of 59.95 is off by 10% or more, it is placed into the second level count queue.
•
B - Variances are calculated on the total bin value. For example, if a bin contains 50 pieces of an item that costs $1, 5 pieces of an item that costs $10, or 1 piece of an item that costs $50, a variance above 10% places the product in the second level count queue. This method is useful if you stock large quantities of low-priced products.
From Cost
The lowest unit cost or bin value for the tolerance range.
To Cost
The highest unit cost or bin value for the tolerance range.
Tolerance%
The tolerance percentage allowed for the cost range. If the count is off by this percentage or more, then a second count is required.
F2-Save
Saves the cost tolerance information.
F4-Del
Deletes the cost tolerance information.

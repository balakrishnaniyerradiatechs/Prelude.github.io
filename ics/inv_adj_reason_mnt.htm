Inventory Adjustment Reason Maintenance screen
The Inventory Adjustment Reason Code Maintenance screen allows you to create and maintain reason codes to identify why you are Adjusting Inventory Quantities in the Inventory Adjustment Line Detail screen.
Menu Path: Inventory > Maintenance > Misc Code Files > Inventory Adjustment Reason Codes
Field and Function Key Descriptions
Field/Function Key
Description
Inv Adj Reason
The reason code that identifies why inventory is being adjusted.
Description
The description of the reason code.
Update Product Usage
Determines if usage is updated for the product when this code is used to adjust inventory (refer to Recording Product Usage):
•
Y - Product usage is updated for the product. The usage displays in the S/O Usage column in Product Usage Inquiry.
•
N - Product usage is not updated for the product.
Adjustment Account
The general ledger account information to use to offset inventory.
Div
The division number for this account. A division is the highest possible grouping in your company (e.g., Eastern Division, Western Division).
Reg
The region number for this account, if applicable. A region is a subset of Division - if Division is used (e.g., Southeast Region, Northwestern Region).
Loc
The location or warehouse for this account.
Dep
The department number for this account. A department is a subset of warehouse, typically used to separate expenses within a warehouse (e.g., accounting, sales, warehouse).
Ccr
The cost center number for this account. A cost center is a subset of Department - if Department is used. Typically used to separate expenses within a warehouse or department (e.g., shipping, receiving, outside sales, inside sales).
Account
The G/L account number to use as the offset when this inventory adjustment code is used in Inventory Adjustment Entry.
Description
The description of the general ledger account.
F2-Save
Saves the adjustment reason code.
F4-Del
Deletes the adjustment reason code.

Kit Parent Products
A kit parent is a list of components that make up one part number. Depending on the kit components, you can set up a kit parent in Product Maintenance with:
•
A list of components that cannot be changed on an order.
•
A list of components that can be changed on an order.
•
No components so that they can be added when the kit parent is entered on an order.
Once a kit parent is set up, any time you enter the product on a transaction or in an inquiry, KIT displays to let you know that kit components exist for the product. You can choose to explode the kit components onto the order. Refer to “Entering Kit Parent Products” in the Order Processing documentation for more information.
To set up a product as a kit parent:
1
Select Inventory > Maintenance > Product Maintenance > Product Maintenance to open the Product Maintenance Screen.
2
Enter the Product to set up as a kit parent.
3
Press F6-Xref/Notes.
4
Enter Y in Kit Parent to open the Kit Maintenance screen.
5
For Type, choose one of the following:
•
R - Regular. Enter a list of items normally assembled at picking time. Usually inventory is kept at the component level; however, you can assemble the kit ahead of time and keep inventory at the parent level. The parent and the components print on the pick ticket and the parent prints on the invoice.
•
A - Assortment. The same as for Regular except that the parent and components show on the pick ticket and the invoice.
•
D - Display. Enter a group of products sold as one unit. The inventory is kept at the parent level. The parent prints on the pick ticket and the parent and the components print on the invoice.
•
E - Explosion. Enter a group of products in a kit parent. When you enter the kit parent on an order, you can choose to explode the kit components as line items. If you choose not to explode the list of components, you can enter the kit parent item on the sales order as a regular product (same as R type).
Rental Items
If the product is a rental product the following conditions apply:
•
It must be set up as type R (regular) or E (explosion).
•
Both the kit parent and the components must be set up as rental items.
•
Rollup Pricing must be set to N
•
P/O must be set to N.
6
For S/O, determine if you can modify kits on an order. Choose from the following:
•
Modify Components - Allows you to add/change/delete any kit components on a sales order.
•
Automatically Add Components - Creates the kit and components automatically. They cannot be modified on a sales order.
•
Automatically Add and Modify - Creates the kit and components automatically. They can only be modified after initial entry.
•
Tag Components - Opens the Product Tagging screen to prompt for the kit components on a sales order (Kit Type must be set to R to use this option).
7
For Rollup Pricing, determine if the price of the kit parent is calculated as a sum of the prices of the components. Choose from the following:
•
Yes - The sum of the component prices calculate the parent price.
•
No - The price of the kit is the price of the parent product number.
8
For Sas Flag, determine if sales analysis is tracked at the parent level or at the component level. Choose from the following:
•
P or null - Tracks sales analysis at the parent level.
•
C - Track sales analysis at the component level. The components are tracked at their respective selling prices on an invoice. If a discrepancy exists between the kit parent price and the sum of the prices of the components, the discrepancy shows as an Over/Under product in sales analysis reports.
•
D - Track sales analysis at the component level. Each component’s selling price is a percentage of the total cost of the kit. This forces the sum of the kit components to match the kit parent price. The formula to calculate the new component price is (total kit price) x (component ext cost) / total kit cost. Sales Analysis Rebuild looks at this flag for kit products to determine if sales analysis is rebuilt at the kit parent level or at the kit component level.
9
For P/O, indicate if a kit parent should be exploded into its component parts when it is entered on a purchase order. Choose from the following:
•
Yes - Prompt to explode the kit components.
•
N0 - Never explode the kit components.
•
Explode and Recalc Cost - Prompt to explode the kit components and costs and give the option to recalculate the cost so that the total cost of the kit line items equals the cost of the kit parent
10
For Component Number, enter the product numbers that make up this kit.
11
Enter the Quantity of this component required to make one kit.
12
For U/M, enter the unit of measure that overrides this component's default price unit of measure. The unit of measure entered for each component must exist in either the component's default price unit of measure or in the component's unit of measure table.
If a component product has a unit of measure table with a default in the Default S/O U/M field, the unit of measure in the kit parent overrides this default. Refer to Establishing a Unit of Measure for more information on unit of measure.
13
Press F2-Save to save the kit information.

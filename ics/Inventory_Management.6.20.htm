Defining Bins
Detailed description and location information for a bin helps track inventory and replenishment path information for bin on-hand quantities (replenishment is only available with the Warehousing module). Bin locations are assigned to products in the Product Warehouse Maintenance Screen.
Bin Location Listing
To view a list of bins for each warehouse area and zone, select Inventory > Listings > Miscellaneous Listings > Bin Locations.
To define a bin:
1
Select Inventory > Maintenance > Bin Files > Bin Locations to open the Bin Location Maintenance screen.
2
Enter the Warehouse.
3
For Bin, enter up to 14 alphanumeric characters to represent the bin location within the warehouse.
4
Enter a Description to help identify the contents of the bin.
5
If using zones to group bins, enter the Zone where the bin is located. Refer to Defining Zones.
6
For Cycle Count Zone, enter the cycle count zone for this bin. This is required if you cycle count. Refer to Cycle Counting.
7
For Permanent Code, indicate if this is a permanent bin. Choose from the following:
•
P - Permanent bin (default). This is used when a product is normally stored in the same location. Even when the product is out of stock and the bin is empty, the space is reserved for the next stock receipt of the item.
•
T - Temporary Bin. A temporary storage area can be used to hold material on orders that need to be staged before shipment. Deletes the products from the list of products in this bin when the quantity reaches zero. You can determine whether or not to delete the bin when the last product is removed.
•
R - Retained Temporary Bin. Automatically deletes the products from the list of products in this bin when the quantity reaches zero (unless the bin is the primary bin for the product). This option does not delete the bin when the last product is removed. You must delete the product record manually in Bin Location Maintenance.
8
For Bin Type, indicate the type of products in the bin. Only pick, bulk, and retail Bin Types are used with cycle counting. If you use the Warehousing module, the replenishment process uses this information. Choose from the following:
•
B - Bulk. Stores bulk quantities of the product.
•
C - Crib. The Crib Bin Customer/Ship-To Maintenance screen opens to allow you to enter the customer information for this crib/warehouse. Refer to “Using Crib Warehouses” in the Order Processing documentation” for more information.
•
D - Receiving dock. A bin located on the receiving dock.
•
I - Inspection. Temporarily stores products awaiting inspection.
•
K - Bulk un-packaged.
•
L - Work-in-process line stock.
•
O - On-the-water (refer to Receiving Stock for OTW Shipping Bins).
•
P - Pick. Primary picking location.
•
R - Repackaging. Used to repack replenishment material.
•
S - Staging. Temporary storage area to accumulate material before being shipped.
•
T - Retail. A retail sales location.
•
U - Unallocated. Miscellaneous stock usually not available for sale.
•
W - Work in process.
9
For Pick Reserve, indicate the pick status of this bin. Used with the Warehousing module. Choose from the following:
•
N - Not available for picking.
•
R - Pickable.
•
R - Reserved for a customer.
10
Enter the Environment Code, to identify the type of controlled environment for the bin location (for use with the Warehousing module). Choose from the following:
•
E - EPA controlled.
•
F - Fire rated.
•
N - Normal.
•
R - Refrigerated.
•
S - Secured.
11
Enter the Storage Type to indicate how the products in this bin are stored (for use with the Warehousing module). Choose from the following:
•
P - Pallet racks.
•
R - Flow racks.
•
C - Carousel.
•
F - Floor stock.
•
S - Shelf.
•
M - Mobile.
•
W - Work center.
•
T - Truck.
•
O - Outplant.
•
G - Pre-stage.
12
Enter the Vendor Number if you use bin method outplant processing in the Warehousing module.
13
Enter the Sort Sequence to indicate how to sort the bins for picking. Refer to Sorting Bins by Warehouse Location for information on the Bin Location Sort Sequence Procedure.
14
For Putaway Sort Seq, enter the sort sequence code for sorting putaway bins. If this field is blank, the Sort Sequence Code is used.
15
For Nettable Flag, indicate if this bin is nettable. Do not change this flag while users are logged in, if products exist in the bin. You can change this flag at the end of the business day so that Nightly Processing can rebuild non-nettable quantities used in availability calculations. Choose from the following:
•
Y - The bin location is nettable (the material is available to sell).
•
N - The bin location is non-nettable (the material is not available to sell).
16
For Auto Replenish Flag, indicate if this bin is included in the auto replenish procedure (Warehousing module):
•
Y - The bin is included in the auto replenish procedure.
•
N - The bin is not included in the auto replenish procedure.
17
For Putaway Status, indicate if this bin is active and can be used for new stock receipts (information only). Choose from the following:
•
A - The bin is active.
•
I - The bin is inactive.
18
For Date Established, enter the creation date for the bin. Today’s date is the default.
19
Enter the Aisle in the warehouse where the bin is located.
20
Enter the Bin Width, Bin Height, Bin Depth, Bin Volume, and Bin Weight, if this information is required.
21
Press F2-Save to save the bin information.
You can assign bin locations to products in the Product Warehouse Maintenance Screen.
Copying a Bin Location
To copy bin information to create a bin, use the Copy feature on the Action Bar. Enter the New Warehouse number, the New Bin Location and the New Bin Description. The information from the existing bin is copied to the new one.

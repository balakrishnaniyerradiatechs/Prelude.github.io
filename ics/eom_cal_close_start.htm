End of Month Calendar Close
The End of Month Calendar Close updates forecasts, line points, order points, EOQ, and FIFO information. It also determines unusual usage and notifies buyers of any product whose purchase parameters were not recalculated.
You must run the calendar close every month to keep history and performance records up-to-date. You can run month-end more than once if necessary, but be sure to use the same month-end closing date each time you run it for the same month. If at all possible, run the process when very few or no users are on the system so that the product information is accurate.
To run Month End Calendar Close:
1
Select Inventory > Processes > End of Month Calendar Close. The following prompt appears:
Are You Sure You Want to Execute The Calendar Close Procedure?
2
Choose from the following:
•
Yes - Close the calendar month.
•
No - Do not close the calendar month.
3
Once you choose Yes, use the following selection criteria to determine what to include in the end of month calendar close:
•
Enter Month End Closing Date. Enter the last day of the last complete month. The default depends on the current date:
For example:
-
If the system date is between the 1st and the 25th of the month, the default date is the last day of the previous fiscal period.
-
If the system date is the 26th through the end of the month, the default is the last day of the current fiscal period.
•
Exec Prod Classify Rebuild. Determines if you want to rebuild the product classifications automatically before you run the End of Month Calendar Close. Product Classifications are based on the total value of a product that moves through inventory each month.
You can also rebuild product classifications independently of the End of Month procedures. Refer to Classifying Products for information on rebuilding product classes and identifying dead stock.
Choose from the following:
-
Y - Rebuild product classifications.
-
N - Do not rebuild the product classifications.
•
Exec Cus Performance Rebuild. Determines if you want to rebuild the customer performance statistics before you run the End of Month Calendar Close. Sales statistics are used to rank customers by sales and gross profit for the Customer Ranking Report. You can also build customer performance statistics by using Build Customer Performance in the Order Processing module.
Choose from the following:
-
Y - Rebuild the customer performance statistics.
-
N - Do not rebuild the customer performance statistics.
Customer Ranking Options in Default Parameters Maintenance determines how customers are ranked:
•
C - Company.
•
W - Warehouse.
•
T - Territory.
•
A - All.
•
Exec Ven Performance Rebuild. Determines if you want to rebuild the vendor performance statistics before you run the End of Month Calendar Close. A vendor’s performance is ranked according to the number of late or incomplete shipments. You can also rebuild vendor performance statistics independently of End of Month procedures (refer to “Rebuilding Vendor Performance Statistics” in the Purchasing Documentation).
Choose from the following:
-
Y - Rebuild the vendor performance statistics.
-
N - Do not rebuild the vendor performance statistics.
•
Exec VLine Miss Pur Tgt Rpt. Determines if you want to generate the Vendor Line Missed Target Report during the End of Month Calendar Close. This report displays vendor lines that do not have a review cycle target type or amount to calculate review cycle days. Vendor line targets are established in Vendor Line Maintenance.
Choose from the following:
-
Y - Generate the Vendor Line Missed Target Report.
-
N - Do not generate the Vendor Line Missed Target Report.
•
Prnt Prod Prm Calc Skip Aud. Determines if you want to generate the Product Parameters Calc Skipped Audit Report to show all products that did not have purchase parameters recalculated. The buyer queue is also updated with these products to allow you to manually adjust the purchasing parameters.
Choose from the following:
-
Y - Generate the Product Parameters Calc Skipped Audit Report.
-
N - Do not generate the Product Parameters Calc Skipped Audit Report.
Skipped Purchase Parameters
Purchasing Parameters are not recalculated if:
•
Usage overrides were not entered for the months with unusual usage.
•
Monthly usage was greater than the total of the last 5 months of combined usage.
•
A product is out of stock for 2 weeks or more in one month.
Refer to the Product Parameters Calc Skipped Audit Report for more information.
Out of Stock Exception
The Out of Stock Exception flag in Purchasing Controls > Purchasing Parameters determines if purchasing parameters are reviewed when a product is out of stock.
Unusual Usage Exception
The Unusual Usage Exception flag in Purchasing Controls > Purchasing Parameters determines if purchasing parameters are reviewed when a product has unusual usage.
4
Choose Execute. You can either start the report immediately or schedule it for later. When the End of Month Calendar Close starts, the rebuilds and reports generate in the background. The messages at the bottom of the screen indicate which records are being updated.
If you chose to print the Product Parameters Calc Skipped Audit Report, refer to the Product Parameters Calc Skipped Audit Report for detailed information.
If you chose to print the Vendor Line Missed Target Report, refer to the following sample report.

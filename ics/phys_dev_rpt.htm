Physical Inventory Deviation Exception Report
The Physical Inventory Deviation Exception Report lists items with a difference in value between the saved on-hand quantity and the physical count quantity, if it is more than the deviation amount you define.
This report is only valid if you run it before you update inventory (refer to Updating Physical Inventory Counts).
Menu Path: Inventory > Physical Inventory > Inventory Sheet Procedure > Deviation Exception Report
Selection Criteria
Use the following criteria to limit the information to print on the report:
•
Warehouse. The warehouse where the inventory was counted.
•
Physical List Name. The list name to include on the report. Leave blank to include all the lists for a warehouse.
•
Display Primary Bin. Determines if you want the bin number to display. Choose from the following:
-
Y - Display the primary and secondary bin numbers.
-
N - Do not display either bin number.
•
Sort By Primary Bin. Determines the sort sequence. Choose from the following:
-
Y - Sort by primary bin.
-
N - Do not sort by primary bin.
•
Beg. Deviation Value >. If you enter a value, the report only displays items whose deviation quantity multiplied by the unit cost has a total dollar value greater than this amount. You may want to enter a value so that the report only includes discrepancies of a large dollar amount.
For example: If an item is only worth 2 cents, a count discrepancy of 100 is still only 2 dollars, so it is not really worth the re-count.
•
End. Deviation Value <. If you enter a value, then the report only displays items whose deviation quantity multiplied by the unit cost has a total dollar value less than this amount.
Column Descriptions
Column Heading
Description
Product Number
The product numbers in the deviation range between the saved on-hand quantity and the count.
Pri/Sec Bin
The primary and secondary bin numbers, if applicable.
Description
The product description.
Uid
The initials of the person who entered the counts.
Cost
The unit cost of the item.
LinkOH
The on-hand quantity that is linked to a sales order, transfer, or work order.
StockOh
The stock (saved on-hand) quantity.
Book$Value
The dollar amount of the saved on-hand quantity.
PhysCnt
The quantity entered from the physical inventory count.
Dev-Qty
The difference between the saved on-hand quantity and the counted quantity.
Dev$Value
The total dollar amount of the deviation quantity (Dev-Qty x Cost).

Cycle Count First Level Report
The Cycle Count First Level Report prints the list of bins to count in a zone and the number of products to count per bin.
Menu Path: Inventory > Physical Inventory > Cycle Counting Procedure > Cycle Count First Level Report
Selection Criteria
Use the following criteria to limit the information to print on the report:
•
Division. The highest possible grouping in your company (e.g., Eastern Division, Western Division).
•
Region. A subset of Division - if Division is used (e.g., Southeast Region, Northwestern Region).
•
Department. A subset of warehouse. Typically used to separate expenses within a warehouse (e.g., accounting, sales, warehouse).
•
Cost Center. A subset of Department - if Department is used. Typically used to separate expenses within a warehouse or department (e.g., shipping, receiving, outside sales, inside sales).
•
Warehouse. The warehouse to count bins for.
•
Cycle Count Zone. The cycle count zone to include for counting. Cycle count zones are assigned to bins in Bin Location Maintenance. Refer to Setting Up Bins.
Column Descriptions
Column Heading
Description
Zone
The zone for the products to count. Refer to Defining Zones for information on zone assignments.
Description
The description for the zone.
Bin
The bin location for each bin in the zone.
Description
A description of the physical location in the warehouse.
Counter
The counter assigned to the bin if bins are assigned to counters.
Source
Indicates how the product was added to the second level queue:
•
BCCRE - Bin First Level Cycle Count Request Entry.
•
CCRE - Cycle Count Request Entry.
•
ECCI - Emergency Cycle Count Request Entry.
Count Date
The date of the bin’s last cycle count.
# of Counts
The number of times the bin was counted.

Core Order Info screen
The Core Order Info screen allows you to:
•
Apply a banked core to a pending core.
•
Reverse a previously applied core bank.
•
Manually write off a pending core.
Refer to Pending Cores for more information.
Menu Path: Inventory > Processes > Core Bank Issues Entry > F5-Review
Field and Function Key Descriptions
Field/Function Key
Description
Order#
The order containing the pending cores.
Product Number/
Description
The product number and description of each core product on the order.
Core Code
The core code used to group core products. Refer to Creating Core Type Codes.
Pending
Shipped
The number of core products shipped on the original order.
Applied
The number of cores applied against the original order.
Remain
The number of cores outstanding from the customer that still need to be returned.
Bank Avail
The number of cores available in the core bank.
F2-Save
Saves the core order information.
F6-Apply
Allows you to apply a banked core against a pending core or to reverse a previously applied core bank:
•
To apply a banked core against a pending core, enter the quantity to remove from the core bank and apply to the pending core.
•
To reverse a previously applied core bank, enter a negative quantity to reverse an error and reverse the general ledger postings.
F7-Write Off
Allows you to write off a pending core record. The quantity you enter reduces the remaining quantity of the pending core. You are not required to match a core bank record.

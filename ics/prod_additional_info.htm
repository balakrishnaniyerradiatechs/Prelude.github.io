Additional Product Maintenance Information Screen
The Additional Product Maintenance Information screen allows access to additional code and product information.
Menu Path: Inventory > Maintenance > Product Maintenance > Product Maintenance > F5-Add Info > F5-Add Info
Field & Function Key Descriptions
Field/Function Key
Description
Country of Origin
The code that represents the country where this product is produced. Used to calculate duty charges if importing the product from an overseas vendor. Refer to Receiving On-The-Water Merchandise.
Commodity Code
The commodity code used to calculate duty. Refer to Commodity Codes for information on setting up commodity codes. This overrides the Import Code in the Vendor Maintenance Import Information screen when calculating duty.
Harmonizing Unit
The harmonizing unit number required by NAFTA to print on export documents when this item is exported from the US to another country. A purchase order containing this item uses the rate from this unit to calculate duty. Products with a harmonizing unit are subject to a tariff charge. Refer to Tariff Commodity Codes.
Harmonizing Date
The date the harmonizing unit is assigned. Prints on the export documents.
Fav Nation Rate Only
Determines if the product uses the favored nation rate percent for the harmonized tariff:
•
Y - The product uses the favored nation rate percent.
•
N - The product uses the agreement code for the harmonized tariff. If no agreement code exists, the favored nation rate percent applies. Agreement Codes are established in Order Processing Controls > Country Codes to designate special tariff treatments as defined in the Harmonized Tariff Schedule of the United States. Refer to Tariff Commodity Codes.
Family Name
The family code for a product. Product family codes allow you to group products for purchasing purposes. When one product in the group needs replenishment, you can use the Family Replenishment Report to view all the products in the group to determine what else needs to be ordered to meet any minimum ordering requirements. Refer to Product Family Codes for information on setting up family product codes.
Manufacturer Number
The manufacturer number to use for reporting purposes.
Warranty Period Unit
Indicates how warranty time periods are measured, if the product is under warranty to your customer:
•
D - Days.
•
M - Months.
•
Y - Years.
Wty-Per
The warranty period. If there is a value in Warranty Period Unit, this is the number of days, months, or years that the product is under warranty.
If the Service Item Flag is set to W on the Product Maintenance Screen, this information is used to calculate the warranty expiration date.
Wty Labor
The warranty labor period. If the product has a warranty for labor, enter the number of days, months, or years that labor is under warranty (expressed in the same units as Warranty Period Unit).
Wty-Hours
The warranty hours, if the warranty period is based on the number of hours on the equipment. For example, the warranty period could be defined as one year or 1500 hours, whichever comes first. In Shop Repair Order Entry, a warning message alerts you that the unit being repaired is still under warranty.
Recurring Billing
If the product is a non-stock rental item, you must indicate if it can be received multiple times through Vendor Invoice Reconciliation Entry until the customer returns the product:
•
Y - The product can be received multiple times in Vendor Invoice Reconciliation Entry. You might use this in a rent-to-rent situation to generate a recurring billing receiver for a product that is linked to a P/O. The automatic Purchase Order Generation in Sales/Rental Order Entry creates a type 11 (recurring billing) purchase order.
•
N - The product cannot be used on a recurring billing purchase order.
Core Product
Determines if the product is a core product:
•
Y - The product is a core product. All exchange products for this core display. Refer to Core Products for information on setting up exchange products.
•
N - The product is not a core product.
When a customer returns a core product, they receive a discount against a new product. For example, if a battery (the core) is returned, the customer would get a discount on a new battery. Refer to Pending Cores for more information on cores.
Core Type
The core type code set in Core Type Code Maintenance. Refer to Creating Core Type Codes. Used with core banking to group cores. Displays in the Core Bank Inquiry and in Core Bank Issue Entry. Refer to Entering Core Bank Issues.
S/O PTP Dependent
Determines if a pick ticket should print if this product is the only available item on a pick ticket:
•
Y - A pick ticket will not print for an order containing this product unless another pickable item is included on the order. You might use this for a catalog that you don’t want to ship unless products are available to ship.
•
N - A pick ticket will print when this product is available, regardless of the status of any other items on the order.
Resale Exemption
Determines if the product is exempt from sales tax if the customer re-sells it (not applicable for service items):
•
Y - The product is exempt from sales tax.
•
N - The product is taxed.
Allow Description Ov
Determines if you can modify the product description after you enter the product on any transaction:
•
Y - You can modify the product description in an order entry screen.
•
N - You cannot modify the product description in an order entry screen.
F2-Save
Saves the product information in this screen.
F6-Royalty
Opens the Royalty Commission Information Screen to allow you to enter royalty information for the product.

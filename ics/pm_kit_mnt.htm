Kit Maintenance screen
The Kit Maintenance screen allows you to enter a list of components that make up a kit parent item. Refer to Kit Parent Products for information on setting up kits.
Menu Path: Inventory > Maintenance > Product Maintenance > Product Maintenance > F6-Xref/Notes > Kit Parent
Field and Function Key Descriptions
Field/Function Key
Description
Type
Identifies how the kit components are tracked in inventory:
•
R - Regular. A list of items normally assembled at picking time. Usually inventory is kept at the component level; however, you can assemble the kit ahead of time and keep inventory at the parent level. The parent and the components print on the pick ticket and the parent prints on the invoice.
•
A - Assortment. The same as for Regular except that the parent and components show on the pick ticket and the invoice.
•
D - Display. A group of products sold as one unit. The inventory is kept at the parent level. The parent prints on the pick ticket and the parent and the components print on the invoice.
•
E - Explosion. A group of products in a kit parent. When you enter the kit parent on an order, you can choose to explode the kit components. If you choose not to explode the list of components, you can enter the kit parent item on the sales order as a regular product (same as R type).
S/O
Determines if kits can be modified in Sales Order Entry. Choose from the following:
•
Modify Components: Allows you to add/change/delete any kit components on a sales order.
•
Automatically Add Components: Creates the kit and components automatically. They cannot be modified on a sales order.
•
Automatically Add & Modify: Creates the kit and components automatically. They can only be modified after initial entry.
•
Tag Components: Opens the Product Tagging screen to prompt for the kit components on a sales order (Kit Type must be set to R to use this option).
Rollup Pricing
Determines if the kit parent is calculated as a rollup of the prices of the components. Choose from the following:
•
Y - The sum of the component prices calculate the parent price.
•
N - The price of the kit is the price of the parent product number.
Sas Flag
Determines if sales analysis is tracked for the kit parent or the kit component:
•
P or null - Tracks sales analysis at the parent level.
•
C - Tracks sales analysis by component. The components are tracked at their respective selling prices on an invoice. If a discrepancy exists between the kit parent price and the sum of the prices of the components, the discrepancy shows as an Over/Under product in sales analysis reports.
•
D - Tracks sales analysis by component. Each component’s selling price is a percentage of the total cost of the kit. This forces the sum of the kit components to match the kit parent price. The formula to calculate the new component price is (total kit price) x (component ext cost) / total kit cost. Sales Analysis Rebuild looks at this flag for kit products to determine if sales analysis is rebuilt at the kit parent level or at the kit component level.
P/O
Determines if the kit components are exploded when this kit parent is entered on a purchase order:
•
Y - When you enter the product on a purchase order, you have the option to explode the kit parent onto the purchase order.
•
N - You never have the option to explode the kit parent onto the purchase order. Only the kit parent is listed.
•
R - When you enter the kit parent product and cost on the purchase order, you have the option to explode the kit components on the purchase order, including their pro-rated (replacement) cost. You can adjust the components and costs. The kit parent is replaced by a comment line and the components are added to the purchase order as line items.
Component Number
The product numbers that make up the kit.
Description
The product description.
Description 2
The second line of the description for the product, if applicable.
Quantity
The quantity of the component required to make up one kit.
U/M
The unit of measure that overrides this component's default price unit of measure. The unit of measure entered for each component must exist in either the component's default price unit of measure or in the component's unit of measure table.
If a component product has a unit of measure table with a default in the Default S/O U/M field, the unit of measure in the kit parent overrides this default. Sales order kit entry first determines if a component has an override unit of measure set up in product kit maintenance. If an override exists, this unit of measure is used for the component on a sales order. If no override exists, the unit of measure defaults to the component's default price unit of measure.
Refer to Establishing a Unit of Measure for more information on unit of measure.
F2-Save
Saves the kit component information.
F4-Del
Deletes the list of kit components.

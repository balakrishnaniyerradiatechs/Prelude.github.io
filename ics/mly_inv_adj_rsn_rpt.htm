Monthly Inv. Adjustments Report By Reason Code
The Monthly Inventory Adjustments By Reason Code report displays all inventory adjusted for a warehouse for a month or fiscal period. The report is sorted by transaction type, reason code, and date. If you cycle count, you can use this report to monitor cycle count adjustments.
You can select to print the detail line item information or only the total value for each adjustment.
Menu Path: Inventory > Reports > Inventory Adjustment Reports > Monthly Inventory Adjustments by Reason Code
Selection Criteria
Use the following criteria to limit the information to print on the report:
•
Division The highest possible grouping in your company (e.g., Eastern Division, Western Division).
•
Region. A subset of Division - if Division is used (e.g., Southeast Region, Northwestern Region).
•
Warehouse. The warehouse where inventory was adjusted.
•
Department. A subset of warehouse. Typically used to separate expenses within a warehouse (e.g., accounting, sales, warehouse).
•
Cost Center. A subset of Department - if Department is used. Typically used to separate expenses within a warehouse or department (e.g., shipping, receiving, outside sales, inside sales).
•
Transaction Type: The inventory adjustment types to include in the report. Leave blank to list all types of inventory adjustments or choose from the following:
-
INV - Regular inventory.
-
WTY - Warranty inventory.
-
DMG - Damaged inventory.
-
SCR - Scrapped/lost inventory.
•
Adjustment Reason. The reason code, if you want to limit the report to one or more reason codes. Leave blank to include all reason codes. Refer to Establishing Cost Adjustment Reason Codes.
•
Period. The month or fiscal period when the inventory was adjusted.
Column Descriptions
Column Heading
Description
Adj Number
The adjustment number assigned to the transaction. Refer to Adjusting Inventory Quantities.
Whse
The number for the warehouse where inventory was adjusted.
Ent Date
The date the inventory was adjusted.
Type
The type of inventory adjustment:
•
INV - Regular inventory.
•
WTY - Warranty inventory.
•
DMG - Damaged inventory.
•
SCR - Scrapped/lost inventory.
User
The user ID of the person who adjusted the inventory.
Product
Line 1: The product number.
Line 2: The product description.
Line 3: Serial number or lot information, if applicable.
Quantity
The quantity added or deducted from inventory. A positive quantity indicates that inventory was added. A negative quantity indicates that inventory was reduced.
U/M
The unit of measure for the adjusted item.
Cost
The unit cost for the adjusted item.
Ext Cost
The total extended value of the adjusted inventory cost (Quantity x Cost). A positive value indicates that inventory cost was increased. A negative value indicates that inventory cost was reduced.
Adjustment Reason
The adjustment reason code and description for the adjustment. These codes are set up in Inventory Adjustment Reason Code Maintenance (refer to Establishing Inventory Adjustment Reason Codes).
The default reason code from Inventory Parameters is used if the inventory is adjusted during cycle counting. Refer to Cycle Counting.

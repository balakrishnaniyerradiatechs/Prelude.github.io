Setting Maintenance Flags to Use Cores
You must set the following flags to use cores.
File Update Parameters
Pending Cores tracks core items to report pending core changes. This allows you to hold off billing the core charge to keep the A/R and gross profit from inflating. Choose from the following:
•
Y - Track core items to report pending core changes so you can use the pending core procedures.
•
N - Do not track pending core changes. You might choose this option if you are only core banking and applying banked cores against new sales in Sales Order Entry.
Order Processing Parameters
Auto Core Charge? determines if a core charge is automatically billed in Sales Order Entry. Choose from the following:
•
Y - The core charge is automatically applied to the sales order. The core is billed with the exchange product and so is not tracked.
•
N - The core charge is not automatically applied to the sales order. You can choose to bill the core charge, to not bill the core charge, or to apply a core bank item.
Inventory Parameters
Print Core Receiving Tkt? determines if a core receiving ticket prints when a core item is adjusted. The core receiving ticket serves as a customer receipt for a returned core:
•
Y - Print a core receiving ticket when you adjust inventory for a core item. Refer to Receiving Cores in Inventory Adjustment Entry.
•
N - Do not print a core receiving ticket when you adjust inventory for a core item. You can manually print the ticket in Core Receipt Ticket Printing. Refer to Printing Core Receipt Tickets.
BMW/Work Order Parameters
Auto Enter Core Credit Line determines if a core credit line item automatically appears on the work order on the line following the exchange product. Choose from the following:
•
Y - The core credit line is automatically added to the work order.
•
N - The core credit line is not automatically added to the work order.
Default Parameters Maintenance
Core Default Due Days defines the default number of days a customer has to return a core product. This number is used for the default in the Due Date Prompt on a sales order when a core item is not automatically billed.
Product Maintenance > F5-Add Info > F5-Add Info
Core Product determines if a product is a core product. Choose from the following:
•
Y - The item is a core product. If you press ENTER, all exchange products for this core display. Refer to Core Products for information on setting up the exchange products.
•
N - The item is not a core product.
General Ledger Table Maintenance
The following general ledger accounts are required to use pending cores and core banking:
•
Core Receivable.
•
Core Bank Payable.
•
Sales Core.
•
Cost of Sales Core.
•
Core Cost Variance.

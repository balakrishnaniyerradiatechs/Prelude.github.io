PStructures
A product structure or pstructure number is a string of characters that represent multiple kits using an infinite combination of kit component items. For example: To sell doors, you may have just 5 main doors but each door has X number of window kits and X number of door knob kits, etc., that you can use to create the door for a customer.
Although product structures are still a valid option, you may find it easier to use one of the following newer Prelude features to accomplish the same results:
•
Product characteristic codes, which define special ordering requirements for each product you sell (refer to Product Characteristics).
•
Configured products, which create child products for all possible combinations of a product (refer to Configuring Base and Child Products).
To set up a product as a pstructure:
1
Select Inventory > Maintenance > Product Maintenance > Product Maintenance to open the Product Maintenance Screen.
2
Enter the Product to set up as a pstructure.
3
Press F6-Xref/Notes.
4
Enter Y in Pstructure to open the PStructure Definition screen.
5
Enter the Length of the product structure, which is the number of digits a pstructure definition must be when you enter this product on a sales order.
6
Enter the Product Line of the kit parent.
7
Enter the Segment Definition. A segment definition is a kit parent item that joins other kit parent items to comprise the pstructure. When you enter this product and pstructure on a sales order, the system builds a component list of all the combined kit items.
8
Press F2-Save to save the pstructure information.
Example of a PStructure
To enter a pstructure in Sales Order Entry:
1
Enter the product P330S.
2
At the pstructure definition prompt, enter the number 330A197XRAB1525:
Each digit of the pstructure number is referenced by the segment definitions in the product number to determine the kits to add to the order.
Example: In the first segment definition: (1,2,5,6,7) for the product number P330S, each of these numbers indicate a position in the pstructure number to create a kit parent number:
1 references 3
2 references 3
5 references 1
6 references 9
7 references 7

When you enter a kit on a sales order, you only see the components that make up all of the previous kit items. The kit parents are not visible on any printed documents. The pstructure definition (i.e. 330A197XRAB1525) is saved on the transaction.

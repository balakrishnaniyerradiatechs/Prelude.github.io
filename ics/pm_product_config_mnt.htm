Product Configuration Maintenance screen
The Product Configuration Maintenance screen allows you to assign configuration codes to a base product. The configuration codes are set up in Configuration Code Maintenance to identify all the possible options you have when you sell the product (e.g., color, size, length).
Refer to Configuring Base and Child Products for more information.
Menu Path: Inventory > Maintenance > Product Maintenance > Product Maintenance > F5-Add Info > Special Operation = CB
Field and Function Key Descriptions
Field/Function Key
Description
Sales Analysis Flag
Determines where sales analysis information is recorded:
•
Parent - Update sales information at the parent level.
•
Child - Update the sales information at the child level.
Configuration Codes
The codes established in Configuration Code Maintenance that apply to this product (e.g., size, color, length). Refer to Setting Up Configuration Codes.
Configuration Responses
The responses available for the code that apply to this base product (e.g., red, white, blue). You can only enter responses that are set up as Options in Configuration Code Maintenance.
F2-Save
Saves the configuration information.
F5-Response Toggle
Allows you to move back and forth between Configuration Codes and Configuration Responses to view or edit the information.
F6-Configuration Worksheet
Opens the Product Generation Worksheet screen to display all proposed product combinations based on the Configuration Codes and Configuration Responses entered. You can delete any products or override any product descriptions as necessary before you create the child product.

Recording Product Usage
Monthly usage is the primary statistic the system uses to calculate purchasing parameters for a product (refer to the Product Purchasing Parameters Maintenance Screen). To calculate parameters correctly, you must record a product’s usage in the correct warehouse, which may not always necessarily be the warehouse where the product is sold or shipped from.
Customer Maintenance Usage Settings
Shipping Warehouse (Customer Maintenance > F7-Order) identifies the default shipping warehouse for the customer. It also determines which warehouse is updated with usage when a product is sold to the customer. You would want to record usage at the warehouse where the customer normally buys material, even when shipped out of another warehouse (presumably because the normal shipping warehouse was out of stock).
Override D/C (Customer Maintenance > F7-Order > F5 Add Info) determines if usage for this customer follows the distribution center path to record usage:
•
Y - Usage for the customer is not recorded using the distribution center controls in Product Warehouse Maintenance and in Warehouse Maintenance.
•
N - Usage for the customer is updated following the normal distribution center path.
Inventory Parameter Usage Settings
Usage Update During SOE (Inventory Parameters > F5-Add Info) determines if usage is captured for product when it is entered on a sales order or when the order is invoiced:
•
Y - Update usage in Sales Order Entry or Credit Memo Entry.
•
N - Update usage when the order is invoiced.
Ship Whse Usage Update (Inventory Parameters > F5-Add Info) indicates where usage is updated when a product that is non-stock in the selling warehouse is a stock product in the shipping warehouse.
•
Y - Update usage in both the selling and shipping warehouses.
•
N - Update usage in the selling warehouse only.
If the sales order’s selling warehouse is the same as the line item’s shipping warehouse, then usage is recorded in that warehouse.
If the sales order’s selling warehouse is not the same as the line item’s shipping warehouse, then following distribution center path determines where to record product usage:
1
Distribution Ctr in Product Purchasing Parameters Maintenance > F5 Add Info. If this field contains a warehouse number and the customer’s Override D/C Flag is set to N, usage is recorded in the distribution center warehouse rather than the selling or shipping warehouse.
2
Distribution Center in Warehouse Maintenance > F5-Add Info, for the line item’s shipping warehouse. If this flag is set to Y and the customer’s Override D/C Flag is set to N, usage is recorded in the shipping warehouse.
3
Shipping Warehouse in the Customer/Customer Ship-To record. If this field contains a warehouse number, usage from sales orders is recorded in that warehouse rather than the selling or shipping warehouse.
4
The selling Warehouse for the sales order.
Once the system determines where to record usage, it checks the Status Code of the product in that warehouse. If the product is a stock product in the recording warehouse, usage is only updated in that warehouse. If the product is a non-stock product in the recording warehouse, the usage is updated in both the recording warehouse (usually the selling warehouse) and the shipping warehouse.
Distribution Center Path Usage Exceptions
The following order types and product types do not follow the distribution center path to record usage.
•
Transfer Usage: If a product is transferred following its normal replenishment path, then usage is updated in the shipping warehouse.
•
Inventory Adjustment: The Update Product Usage flag in Inventory Adjustment Reason Code Maintenance (refer to Establishing Inventory Adjustment Reason Codes) determines if product usage is updated for the item when you enter the inventory adjustment reason code in Inventory Adjustment Entry (refer to Adjusting Inventory Quantities) to reduce inventory.
•
Work Order Usage: When a bill of material is issued and completed on a work order, the work order usage is updated for the components of the bill of material item.
•
Direct Sales Order Usage: Usage is only recorded for a direct sales order if the Update Usage on Drop Ship Orders flag is set to Y in Additional Option Parameters.
•
Repair Order Usage: Usage is not recorded for products on a type 10 - Repair sales order.
•
Lost Sales Usage: If lost sales are updated and the lost sales reason code has the Update Product Usage flag set to Y, then usage is updated.
•
Credit Memo Usage: If the credit memo credit reason code has Update Product Usage set to Y, then usage is updated.
•
Merged/Superseded Product Usage: If a product is superseded with another product (refer to Supercedure Products), any override usage recorded for sales orders, transfers, and work orders includes the usage for the original product.
If a product is replaced or merged with another product (refer to Replace a Product Universally), usage for sales orders, transfers, and work orders includes the usage of the original product.
•
Related Product Usage: If a product has a Related Product set up in Product Maintenance > F6-Xref/Notes, the Usage Product# determines whether the original product or the related product is updated with usage. When you move product usage, usage for the last 12 months is moved. If you remove this product number, the usage goes back to the original part number. Refer to Related Products.
•
Unusual Usage: Usage is not recorded if a sales order, work order, or transfer has the Unusual Usage Flag set to Y.
•
Consignment Order Usage: The Ovr Shp Whse Cons Ord? flag (Order Processing Parameters > F5-Add Info > F5-Add Info) determines if usage for an 08 consignment sales order updates the shipping or the billing warehouse.

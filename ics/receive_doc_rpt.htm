Purchase Order Receiving Report
If you receive products that arrive without a packing slip, you can print the Purchase Order Receiving Report to list products on a purchase order or multiple purchase orders. A space is provided so you can write in the received quantities.
Menu Path: Inventory > Reports > Stock Receipts Reports > Purchase Order Receiving Report
Generating the Purchase Order Receiving Report
To create a receiving report:
1
For P/O#, enter the purchase order number to receive. You can enter multiple purchase orders on one receiving report.
2
Press F2-Create to generate the report.
Column Descriptions
Column
Description
P/O #
The purchase order number.
Vend#
Line 1: The vendor number.
Line 2: The vendor description.
P/O Date
The date the purchase order was entered.
Lne
The line number for the product on the purchase order.
Product/
Description
Line 1: The product number.
Line 2: The product description.
Line 3: The second line of description for the product, if applicable.
Vendor Part Number
Line 1: The vendor’s part number, if applicable.
Line 2: The linked document number, if any. The prefix indicates the type of document:
•
S - Sales order.
•
T - Transfer.
•
W - Work Order. If EMG appears to the right of the work order number, the work order is marked as an emergency.
Line 3: The customer description if the linked document is a sales order or work order.
Pri/Sec Bin
Line 1: The primary bin number. An asterisk (*) indicates that the on-hand quantity in the bin is zero or below minimum.
Line 2: The secondary bin number.
Line 3: The required date for the item, if applicable.
S
The stock status of the item:
•
S - Stock.
•
N - Nonstock.
•
C - Consignment.
Qty-Opn
The quantity of the line item still open on the purchase order.
U/M
The unit of measure as entered on the purchase order.
Qty-Rec
A blank line where you can enter the received quantity.
SN
An asterisk (*) indicates that the item is serialized.

Entering Second Level Cycle Counts
After you run the Second Level Queue Report (refer to Cycle Count Second Level Report) and count the products in the second level queue, you can enter the recounted quantities.
To enter second level counts:
1
Select Inventory > Physical Inventory > Cycle Counting Procedure > Paper Site Cycle Count Entry (2nd Level) to open the Cycle Count Entry 2nd Level screen.
If you have access to more than one warehouse, the Location Retrieval screen appears first. Enter the appropriate Selling Warehouse, Department Number, and/or Cost Center Number.
2
Enter the Warehouse.
3
For Counter Id, enter the user ID of the counter. This person must be set up as a cycle counter in User Identification Maintenance.
4
Enter the Product from the count sheet. The following displays:
•
Each Bin where the product exists.
•
An asterisk (*) displays under Problem if there was a discrepancy outside the defined tolerance limits.
•
Counted remains blank until the second level count is entered.
•
The Qty, which is quantity entered for the first level count.
If Default P/N in 2nd Level Entry in Warehouse Maintenance is set to Y, the product number defaults in the Product field.
5
To change a quantity in any of the bins listed, highlight the item and press ENTER to open the Cycle Count Entry screen.
6
Enter the second level count in Actual Count.
If the quantity you enter is different than the system quantity, the following prompt displays:
Is The Quantity Entered of ###
The Correct Count Quantity?
7
Choose from the following:
-
Yes - The quantity is correct. The Cycle Count Entry 2nd Level screen returns. An asterisk displays in the Counted Column.
-
No - The quantity is incorrect. You can re-enter the count.
8
Once you correct the second level quantities for each bin, select F2 - Update. The following prompt appears:
An Inventory Adjustment For A Quantity Of ## And A Value Of ## Will Now Take Place! Do You Wish To Continue?
9
Choose from the following:
-
No - Allows you to change the quantities.
-
Yes - Adjusts inventory for the product to reflect the re-counted quantity. The product number is updated with a transaction audit code of ADJ in the Transaction Audit Inquiry.
10
Repeat steps 4-9 to enter counts for each product.
11
Press F6-Exit after all the entries are complete.

Entering Core Bank Issues
You can apply available core bank items to any pending cores for a customer, rather than entering a sales order and invoicing the customer for the item. You can also write-off pending cores for a customer.
To apply core bank items to pending cores:
1
Select Inventory > Processes > Core Bank Issues Entry to open the Core Bank Issues Entry screen.
2
Enter the Customer. If the customer has pending core records but no core bank items, you can only write-off pending cores.
3
For Ret. Type, indicate the return type of pending cores to display. Choose from the following:
•
A - Display all pending cores for the customer.
•
E - Display only pending cores that have a remaining quantity.
The pending core order numbers display along with the quantities Shipped, Applied, and Remaining.
4
Highlight the order to apply the pending core to and press F5-Review. The Core Order Info screen appears to display the pending cores for the order.
5
For each core Product Number, choose one of the following function keys:
•
F6-Apply - Choose from the following:
-
Apply a banked core against a pending core. Enter the quantity to remove from the core bank and apply to the pending core.
-
Reverse a previously applied core bank. Enter a negative quantity to reverse an error and reverse the general ledger postings.
•
F7-Write-Off - Write off the pending core record. Enter the quantity to write off to reduce the remaining quantity on the pending core record. You are not required to match a core bank record.
6
Press F2-Save to save the bank core application.

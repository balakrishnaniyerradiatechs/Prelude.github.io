Product Purchasing Parameters Maintenance Screen
The Product Purchasing Parameters Maintenance screen defines purchasing methods and ordering parameters to determine how Nightly Processing calculates a recommended purchase order, transfer, or work order.
Recommended Quantities
Refer to the “Buyer Queue Inquiry” in the Purchasing Documentation for information on how the recommended quantities calculated by Product Purchase Parameters Maintenance are transmitted to the buyer.
Menu Path: Inventory > Maintenance > Product Maintenance > Product Purchasing Parameters Maintenance
Field and Function Key Descriptions
Field/Function Key
Description
Product
The product number and description.
Warehouse
The warehouse where the product is stocked.
Replenishment Mtd
Defines the replenishment method for the product:
•
P - Purchase. Creates a temporary purchase order. If the product is a stocking product, it is combined with other products in the same vendor line and totals are compared to the vendor line’s targets.
•
D - Direct. Creates a temporary purchase order in the central warehouse.
•
T - Transfer. Creates a temporary transfer.
•
M - Manufactured. Creates a temporary work order for products that are manufactured internally. If you choose this option, the Production Control Parameters screen opens to allow you to enter production control parameters for the work order planner/buyer. Refer to Production Control Groups.
•
O - Outplant. Creates a temporary work order for a product that is out sourced to a manufacturer.
Warehouse (unlabeled field)
Replenishment warehouse. If Replenishment Mtd is T, select a warehouse to transfer from. If Replenishment Mtd is D, select the central warehouse that should purchase this product.
Min / Max Flag
Indicates if the product is replenished using minimum and maximum quantities:
•
Y - Use minimum and maximum quantities to purchase or transfer this product.
•
N - Do not use minimum and maximum quantities to purchase or transfer this product.
Minimum
If Min / Max Flag is Y, enter the minimum quantity to stock. When the product’s stock reaches the minimum, replenishment is recommended.
If Replenishment Mtd is T (transfer), the following formula is used:
Minimum = (Forecast x Lead Time) + Safety Allowance Quantity
Maximum
If Min / Max Flag is Y, enter the maximum quantity to stock. If Replenishment Mtd is T (transfer), the following formula is used:
Maximum = Minimum + (Annual Usage/Turn Target)
Sporadic
A Y indicates if the product has sporadic usage.
•
If you use the Forecasting module, sporadic usage is when a stock product has irregular activity in a set number of months defined in purchasing parameters.
•
If you do not use the Forecasting module, a product is flagged as sporadic when the system changes the min/max settings of a product that has usage of .5 a month or smaller.
Seasonal Flag
Indicates if the item is seasonal:
•
Y - The product’s usage is seasonal.
•
N - The product’s usage is not seasonal.
Frozen Controls
Indicates if purchasing parameters are frozen (refer to Freezing and Thawing Product Purchasing Controls). When you freeze controls for a product, the purchasing parameters calculated by end-of-month processing are ignored. For example, purchasing parameters can be frozen for new products that do not have any usage on which to base purchasing parameters. For these products, you can set minimum and maximum stocking quantities, freeze the controls and set a thaw date after which the product has enough usage for purchasing parameters to create useful data. Choose from the following:
•
Y - The minimum and maximum stocking quantity overrides that you enter are used instead of the purchasing parameters calculated by end-of-month processing until the product is off frozen controls.
•
N - Purchasing parameters calculated by end-of-month processing are used to recommend purchase quantities.
Thaw Date (unlabeled field)
If you entered Y in Frozen Controls, enter a thaw date. After that date, purchasing parameters are calculated based on the product’s usage. If you do not enter a thaw date, the purchasing parameters are frozen indefinitely. Refer to Freezing and Thawing Product Purchasing Controls for information on manually thawing products.
About Overrides:
The data to the right of the following fields are system-calculated purchasing parameters. To override any of these parameters, enter a new value. The system still calculates purchasing parameters for the product but uses the override to determine replenishment requirements.
Purchase Method
Identifies the purchasing method for this product:
•
EOQ - Economic Order Quantity.
•
CLS - Class. Multiply the number of Days (Purchasing Controls > Purchasing Parameters > F6-Class Settings) by (Forecast/30) to calculate a purchase order quantity (for products that have a P Replenishment Mtd).
•
AWU - Average Weekly Usage. Enter the number of weeks to calculate usage. If left blank, the system uses the number of weeks established in Inventory Parameters. Refer to Tracking Average Weekly Usage.
•
DRP - Distribution Requirements Planning. Used with the Forecasting module primarily for products with long lead times.
•
DS - Days Supply. You can enter the number of days supply to keep on-hand.
Weeks to Use (unlabeled field)
Depends on the Purchase Method. If the Purchase Method is:
•
The number of weeks to use to calculate the average weekly usage.
•
The Order value for the current month on the Distribution Requirements Evaluation Inquiry.
•
The number of days supply to keep on- hand.
Leave blank to use the system-level setting in Inventory Option Parameters.
Forecast Formula
If you do not use the Forecasting module, enter the number of months to calculate average monthly usage.
If you use the Forecasting module, press F3 to search for a forecasting formula to use. This formula overrides the setting at the product level.
Forecast
The forecasted usage quantity for the next month.
Order Point
If Purchase Method is EOQ and available stock falls below this quantity, the system recommends replenishment. The formula the system uses to determine Order Point (OP) is:
OP = [(Forecast x Lead Time) x (1 + Safety Allowance%)]/30
Line Point
If Purchase Method is EOQ and available stock falls below this quantity, the product is evaluated as part of a line buy if another product in the line is below its Order Point (OP). The formula to calculate Line Point (LP) is:
LP = OP + [Forecast x (Review Cycle Days/30)]
Review Cycle Days are based on the primary vendor.
Recommended Qty
The quantity that the system recommends that you buy. It can be calculated by using EOQ (Economic Order Quantity), Class, or PDQ (Purchase Discount Quantity). The system rounds the quantity to the nearest standard pack, based on the primary vendor. The maximum recommendation is 12 months’ supply. The minimum recommendation is 2 weeks’ supply or usage during the review cycle, whichever is greater.
The EOQ formula is:
The Class formula is:
Class Quantity = Inventory Class + Forecast
PDQ
If the primary vendor has quantity discounts, PDQ is calculated and used if the Net Cost Per Unit is less than EOQ or Class. To calculate the Net Cost Per Unit, the system uses the following formulas to calculate Investment Cost (IC), Holding Cost, and Total Cost:
Investment Cost = Unit Quantity x Unit Cost at that Lever
Holding Cost = (IC/2) x Cost to Carry x (Unit Quantity/Forecast/12)
Total Cost = IC + Holding Cost
Net Cost Per Unit = Total Cost/Quantity
This number can also be affected by future price increases. Refer to “Including Future Price Increases” in the Nightly Processing section of the Purchasing documentation.
Surplus Threshold
Calculates and displays surplus quantities. Surplus is the difference between quantity available and the system calculated surplus threshold quantity. If this value is overridden, the surplus is the difference between the quantity available and the surplus threshold override quantity.
For EOQ items, this is the Line Point plus the EOQ.
For DRP items, the surplus threshold also includes forecasts during the leadtime of the product. For example, if a product has a 60 day lead time and a 1 month safety quantity of 30, Surplus Threshold is calculated using a quantity of 90.
Safety Allowance %
The safety allowance percentage used to calculate Order Point, Minimum, and Safety Allowance Quantity. Also used to allocate stock. You can enter a percentage here to override the system-level default in Purchase Parameters Maintenance (Systems > Purchasing Controls > Purchasing Parameters > Safety Allowance Method = S > Service %). Refer to the Service Factors screen in the Systems documentation for more on the Service Level Method for calculating safety stock.
Safety Quantity
The minimum quantity that should always be available. The formula to calculate the safety allowance is:
Safety Allowance = [Safety Allowance % x (Forecast x Lead Time)]/30
Inventory Class
The product’s class (0-13) (refer to Classifying Products). Class 0 is for products with less than 12 months of usage (unless you enter an Ann. Usage Override). Class 13 is for products with a usage value less than the Dead Stock Value in Purchasing Parameters Maintenance. Other classes are assigned based on their usage and the class percentages set up in Purchasing Parameters Maintenance. You can automatically recalculate the product class during the End Of Month Procedures (refer to End of Month Calendar Close).
Trend Months
The number of months to calculate Factor. If Trend Months is blank, the Trend% in Product Line Maintenance calculates Trend Quantity. If Trend Months is 0, no Trend Quantity is calculated.
Trend Quantity
The amount that is added to or subtracted from (if negative) the forecast to generate recommended replenishment quantities. The formula used to calculate Trend Quantity is:
Trend Quantity = Forecast x Trend Factor
Factor
The percent change from last year to this year. The formula used to calculate Factor is (CY = Current Year; LY = Last Year; X = Trend Months):
(CY Last X Months Usage - LY Last X Months Usage)/LY Last X Months Usage
Factor can also be entered in Product Line Maintenance as Trend%.
Safety Minimum
The minimum quantity for safety stock.
If Min/Max Flag for this product is set to Y and the Minimum or Maximum is less than the Safety Minimum, it is set to the Safety Minimum.
If Min/Max Flag for this product is set to N and the calculated Order Point, Line Point and Recommended Qty are all less than the Safety Minimum:
•
Min/Max Flag changes to Y.
•
Minimum and Maximum quantities are set to the Safety Minimum.
Set Multiple
The lowest multiple value to keep on the shelf at all times. Nightly Processing uses this number to calculate availability back to this multiple. If the product must be purchased in a set quantity, this is the quantity to maintain availability in this multiple. For example, if you always need to purchase tires in sets of 4, the Set Multiple would be 4.
Last Stockout
The last date the product was out of stock.
Last Replenish
The last date the product was received at this warehouse.
F2-Save
Save changes to the Purchasing Parameters.
F5-Usage
Opens the Product Usage Maintenance Screen, which lists sales order, transfer, and work order usage along with the forecasted usage and forecast formula used. You can override monthly usage.
F6-Recalc Parameters
Recalculates purchasing parameters immediately. If you change the purchasing parameters and do not recalculate them, your changes do not go into effect until end-of-month processing (refer to End of Month Calendar Close).
F7-Add Info
Opens the Purchase Parameters Additional Information Screen to allow you to further define purchase quantities.
F8-Status Inquiry
Opens the Status Inquiry Screen which lists summary information about the product.
Action Bar
Frcst Comparison
Allows you to view the Comparison Forecast Report to compare how each forecast formula would calculate a forecast for the product. Refer to the Comparison Forecast Report in the Forecasting module.
Notes
Opens the Notes Display screen to allow you to enter or edit notes about the purchasing parameters for the product. These notes are shared with the notes screen in Product Warehouse Maintenance and in the Distribution Requirements Planning Inquiry in the Forecasting module.
Safety Alerts
Opens the Product Alert Information Screen to allow you to view any instances where the product is outside of its safety allowance tolerances or whose product usage has exceeded it’s maximum quantity setting. Products Alerts are updated to the work queue of the buyer set up as the purchasing buyer in Warehouse Manager Maintenance.

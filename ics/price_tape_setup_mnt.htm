Price Tape Setup Maintenance screen
The Price Tape Setup Maintenance screen allows you to enter the information required to update prices and costs with a price tape.
Menu Path: Inventory > Price/Cost Update Procedure > Price Tape Update > Price Tape Setup
Field and Function Key Descriptions
Field/Function Key
Description
Name on Diskette or Data File
The name (including the extension) of the file to load. Press F5-Select From tmp to lookup the tmp directory in UNIX to find the correct file name.
Vendor
The vendor to update.
Warehouse
The default warehouse to use for any new products created. Warehouse numbers are not necessary if you are only updating existing locations.
Key to Build System Product
The information required to create your product number from the tape’s data:
•
If your part number is the same as the vendor’s, enter the number position on the tape that contains the vendor’s part number.
•
If your part number consists of the first three characters of the vendor’s name, connected by a dash and linked to the vendor part number, enter “TOR-”:3. The 3 characters inside the quotes represent the vendor. The number 3 represents the third field of the tape to use as a component of your part number.
•
If you want to only include a portion of the field, use square brackets, e.g., 3[x,y]. The number 3 represents the field number to extract from. The x is the starting position of the field to extract. The y determines how many characters to extract.
Fixed Length or Delimited
The type of file to use with the price tape update:
•
F - The price tape is a fixed length file; each piece of data has a set number of characters.
•
D - The price tape is a delimited file; a character separates the data into fields, (e.g., -, tab).
Price Effective Date
The date the prices takes effect if a price is loaded into a future price. Otherwise, this field can be used as a memo field.
Load and Display Non Update Data?
Determines the items to display on the report:
•
Y - Display non-updated products on the Pre-Update Report.
•
N - Display only updated items on the Pre-Update Report.
Create Products Not On System?
Indicates if the price tape update should create products that are on the price tape but not in the system:
•
C - If products exist on the price tape that do not exist in your system, create the new product records in the catalog file (products sold by the vendor but that are not active products in your system).
•
P - If products exist on the price tape that do not exist in your system, create the new product records in the inventory file (includes stock and non-stock products).
•
N - Do not create any new products.
Skip P/N if Vendor doesn’t match?
Indicates if the product number should be skipped if the product/ warehouse/vendor does not match the vendor in the Price Tape Setup screen:
•
Y - Do not update the product if the primary vendor does not match.
•
N - Update the product even if the primary vendor does not match.
Tape Data
Fld#
The next available field number if entering multiple fields.
Position
The starting position on the diskette for this record if using fixed length. The field number and character if using a delimited field.
Length
The length of the field.
Format
The number of decimals in the raw data file. For example:
•
Enter 3 if 12000 should display as 12.000.
•
Enter .3 if the decimal is already set in the correct place, (e.g., 12.000).
•
Enter D if the field is a date.
File Name
The system filename to update. For example, enter PRODUCT to update the product file.
File Data
Dictionary
The dictionary location where the information from the tape will be stored.
Format
The number of decimals in the raw data file. For example:
•
Enter 3 if 12000 should display as 12.000.
•
Enter .3 if the decimal is already set in the correct place, (e.g., 12.000).
•
Enter D if the field is a date.
Override
Indicates if you want to override the existing data with data from the price tape:
•
Y or null - Overwrite the data.
•
N - Do not overwrite the data.
Conv
Y indicates that the data must be converted in order for the products to be updated.
F2-Save
Saves the price tape setup information.
F4-Del
Deletes the price tape setup record.
F5-Select From tmp
Allows you to lookup the tmp directory in UNIX to find the correct file name.
F7-Copy to New Record
Allows you to copy the price tape information for another vendor.

Setting Up Product Characteristic Codes
Product characteristics define the information required to sell a product. If you sell a product that has characteristic codes assigned, you must enter the information necessary to fill the order.
To create product characteristic codes:
1
Select Inventory > Maintenance > Product Control Files > Product Characteristic Codes to open the Product Characteristics Code Maintenance screen.
2
For Characteristics Code, enter the code to assign for the characteristic information. For example, if you create a code called BLADE and assign it to a product, you are required enter the required blade information each time you sell the product.
3
Enter the Description of the product characteristic code (e.g., size, length, color).
4
Enter the Style Template that determines how product characteristics appear on a sales order when you sell a product with this code. Choose from the following:
•
1 - The Product Characteristic Information screen appears when you enter the sales order and requires you to enter an additional description of the product.
•
2 - The system pulls in the product characteristic information from the last time this customer/contact/product combination was sold.
•
3 - A list of previous sales orders for the customer/product combination displays to allow you to select product characteristics from a previous order.
5
If you chose 3 for Style Template, you can enter a Service Product# to add to the sales order when a new product characteristic is created.
6
For Description, enter the types of information to request when an item with this characteristic code is sold. For example, for blade information you may request a size, length, and/or handle type.
7
For Mandatory, indicate if the type of information requested for each Description is mandatory when an item with this product characteristic is sold. Choose from the following:
•
Y - Requires the corresponding Description information to sell the product. For example, if you enter size for Description (in step 6) and Y for Mandatory, then you must enter a blade size number any time you sell a product with this characteristics code.
•
N - Does not require the Description information to sell the product.
8
For History, indicate if the product characteristic information is available on future sales orders:
•
Y - The product characteristic information is available for future orders.
•
N - The product characteristic information is not available for future orders. You must re-enter the product characteristic information each time the customer buys the product.
The Product Characteristics Additional Information screen appears to allow you to enter validation information for each characteristic. If you enter information in this screen, the Add Info field on the Product Characteristics Code Maintenance screen displays a Y.
9
For Default Process, enter the default information to display when the salesperson is required to enter characteristics information.
10
For Validation Process, enter the type of information required by the salesperson to validate the characteristics entered:
•
List of valid values (e.g., Y or N).
•
Key on another file.
•
Expression.
•
Range of values.
•
Pattern match.
•
User subroutine.
•
Length.
•
Translate codes.
•
No spaces.
•
Unique values only.
•
Mandatory field.
•
Integer (no decimal point).
•
Existing data file.
•
Validation list expression.
•
Field input restrictions.
•
Call process.
11
For Intuitive Process, identify the selection process that provides valid responses when the salesperson presses F3.
12
For On Line Help, enter the instructions to appear to assist the salesperson.
13
For Notes Before Field, enter the information notes to appear before the salesperson sees the product characteristics description.
14
For Notes After Field, enter the notes or process name to appear after the salesperson sees the product characteristics description. These notes provide additional information when the item is entered on a sales order.
15
Press F2-Save.
Refer to Assigning Product Characteristic Codes to Products for information on using the characteristic code for a product.

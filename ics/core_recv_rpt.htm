Cores Received Report
The Cores Received Report displays information on cores returned from customers, including cores applied to pending cores or placed in the core bank. Refer to Receiving Cores in Inventory Adjustment Entry for more information.
Menu Path: Inventory > Reports > Inventory Adjustment Reports > Cores Received Report
Selection Criteria
Use the following criteria to limit the information to print on the report:
•
Customer Range. A beginning and/or ending customer to limit the cores to view. Leave both fields blank to view cores received from all customers.
•
Received Date Range. A beginning and/or ending received date to limit the cores to view. Leave both blank to view cores received for all dates.
•
Core Product: The core product number to view. Leave blank to view all core products for the above criteria.
Column Descriptions
Column Heading
Description
Product Number
The core product returned from the customer.
Description
The product description.
Dte Rcvd
The date the core product was received/returned from the customer.
Adj/Ord Number
The inventory adjustment transaction number.
Retn Type
The exchange type that explains why the core was returned:
•
E - Exchange. The customer will exchange the core item for another product.
•
R - Repair and return. The customer wants you to repair and return the core product.
•
U - Unused. The core product is not used.
•
W - Warranty. The core exchange product is under warranty.
•
O - Other.
Source
Indicates where the received core was applied:
•
Core Bank - The customer returned the core and it was applied to the customer’s core bank (open payable)
•
Pending - The customer returned the core and it was applied to a pending core invoice for the customer (open receivable).
Quantity
Received
The quantity received from the customer.
Applied
The quantity applied to a pending core.
Remain
The quantity not applied to a pending core.

New Product Record Audit Report
The New Product Record Audit Report displays products created on sales orders. You can use this report to verify and/or complete product maintenance on these items (refer to the Maintaining a Product by Warehouse).
After the report runs, a dialog box appears to allow you to purge the product information so that it does not appear the next time you run the report.
Creating New Products
Allow New Prod In S/O in Order Processing Parameters must be set to Y to allow products to be created when entering a sales order.
Menu Path: Inventory > Reports > Miscellaneous Reports > New Product Record Audit Report
Selection Criteria
Use the following criteria to limit the information to print on the report:
•
Warehouse. The warehouse containing the new products.
•
Buyer. The buyer assigned to the product’s primary vendor.
Column Descriptions
Column Heading
Description
Product
Line 1: The product number created on a sales order.
Line 2: The product description.
Line 3: The second line of description for the product, if applicable.
Line 4: The vendor part number, if one exists.
Whse
The warehouse where the new product was created.
Pline#
The product line assigned to the product.
Vendor
The primary vendor for the new product.
Vline#
The vendor line assigned to the product.
PPGrp
Product price group. Refer to Establishing Product Price Groups for information on price groups.
PrcTbl
The discount break table set up in the Price Column Quantity Breaks Screen for the product.
CstTbl
The discount break table set up in the Cost Quantity Break Table Screen for the product.
Retail
The retail price set up for the new product.
Price 1
The column 1 price entered for the new product.
Price 2
The column 2 price entered for the new product.
Price 3
The column 3 price entered for the new product.
Rep-Cost
The replacement cost.

Cycle Count Monthly Processing
Monthly processing for cycle counts updates daily counts and bin count frequencies. You must run Monthly Processing when you first set up cycle counting and any time you change the maintenance settings (e.g., bins added/deleted, count frequency changed, etc.). Otherwise, you do not have to run it every month.
Cycle Count Monthly Processing determines:
•
The bins to add or delete since the process last ran.
•
The number of bins to count per day for each frequency.
(Bin Count Frequency x number of bins per zone) / number of count days for the warehouse.
For example:
-
A zone has a bin count frequency of 4, which means that each bin should be counted 4 times per year. Bin Count Frequency is established in Zone Maintenance.
-
The total number of bins in that zone is 150.
-
The warehouse counts bins 200 days per year. Cycle Count Days is established in Warehouse Maintenance.
-
The system calculates 3 as the number of daily counts for the zone. (4 x 150) / 200 = 3 bins a day.
•
The bins to count. The system selects the appropriate bins and sorts them by warehouse, bin count frequency, cycle count zone, and sort sequence and then sets them in a queue for Cycle Count Nightly Processing.
To update cycle count records:
1
Select Inventory > Physical Inventory > Cycle Counting Procedure > Cycle Count Monthly Processing to open the Cycle Count Process Warehouse screen.
2
Enter the Warehouse number to update.
3
Choose Execute. You can either press ENTER to start the process or enter a time to start.

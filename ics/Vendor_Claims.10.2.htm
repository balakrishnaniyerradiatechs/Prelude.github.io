Setting Flags for Vendor Claims & Warranty Inventory
The following maintenance flags determine G/L postings, entry requirements, return rules, and transmission methods for vendor claims and warranty replacements.
General Ledger Table Maintenance
Set the following G/L accounts for posting warranty inventory:
•
Inventory Warranty Return:
-
Debited when a product is put into warranty inventory on a credit memo or rejected in stock receipts.
-
Credited when the credit memo is posted in Vendor Claim Reconciliation Entry.
•
Inventory Warranty Scrapped: The same as Inventory Warranty Return, except products are scrapped rather than returned to the vendor.
•
Inventory Warranty Return Accrual:
-
Credited when warranty inventory is returned to the vendor but the vendor credit memo is not yet received.
-
Debited when the credit memo is posted in Vendor Claim Reconciliation Entry.
•
Inventory Warranty Scrapped Accrual: Warranty returns that are scrapped rather than returned to the vendor are posted as credits to this account. The claim is entered but the vendor credit memo is not yet received.
Inventory Parameters > F5-Add Info
Vend Claim Orig Invoice? determines if you must enter the original vendor invoice number to return an item in Vendor Claim Entry:
•
Y - An invoice number is required.
•
N - An invoice number is not required.
Vendor Maintenance > F5-Add Info > F6-Warranty > F5-Rules
The Vendor Return Rules screen allows you to set up rules for open warranty tags to automatically generate vendor claims. The claims appear in the buyer queue for review (refer to the Buyer Queue Inquiry in the Purchasing documentation). Enter the following information:
•
Date: The date the vendor return rules take effect.
•
Rules: An explanation of the vendor’s return rules. You can print and view these on the Vendor Return Rules Exception Report (Purchasing > Reports > Vendor Return Rules Report). You can also view the return rules in the Warranty Tag Information screen when you enter a warranty tag for an item (refer to Moving Products into Warranty Inventory).
•
Auto Claim Creation Rules: These values work in conjunction with the Vendor Claim Creation From Tags - Rules Based process to automatically create vendor claims. Refer to Generating Vendor Claims from Open Warranty Tags.
-
Minimum Value: Enter the minimum value required for open warranty tags to generate a vendor claim.
-
Minimum Quantity: Enter the minimum quantity required for open warranty tags to generate a vendor claim.
-
Minimum Weight: Enter the minimum weight required for open warranty tags to generate a vendor claim.
-
Maximum Days: Enter the maximum days required for open warranty tags to generate a vendor claim. Open warranty tags that are older than this number of days will generate a vendor claim.
Vendor Maintenance > F5 Add Info
V/C RMA# Required? determines if you must enter a return material authorization (RMA) number to enter a vendor claim for this vendor:
•
Y - An RMA is required.
•
N - An RMA is not required.
Vendor Claims Doc Options determines the default method for transmitting vendor claims to the vendor:
•
B - Print and fax.
•
E - E-mail.
•
M - Print and e-mail.
•
N - Print.
•
Y - Fax.
If using email, the following hierarchy within Vendor Maintenance determines which email address to use:
•
1ST: F5-Add Info > F6-Warranty: Contact & Email Address.
•
2ND: F7 - Contacts: Purchasing Name and E-Mail Address.
•
3RD: F7 - Contacts > F5-Add. Contacts: Sales Rep Name and Sales Rep Email Address.
•
4TH: F7 - Contacts: Expedite Name & E-Mail Address.
•
5TH: E-mail Address on the main screen in Vendor Maintenance.

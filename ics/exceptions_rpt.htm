Usage Exception Report
The Usage Exception Report displays unusual usage and stockout information generated from the Usage Exception Evaluation Procedure. Refer to Evaluating Unusual Usage.
Menu Path: Inventory > Processes > Usage Exception Menu > Usage Exception Report
Selection Criteria
Use the following criteria to limit the information to print on the report:
•
Warehouse. The warehouse(s) for which you want to view usage exceptions.
•
Product Line. The product line to limit the report to just one product line. Leave blank to include all product lines.
•
Product Range. The range of product numbers to limit the report to a group of products or leave blank to include all products.
•
Class. The product classification code(s) to limit the report, or leave blank to include all classes.
•
Exception Code. The types of usage exceptions to include on the report:
-
U - Products with unusual usage.
-
O - Products that are out of stock.
-
S - Products with a spike in usage (Forecasting module).
Column Descriptions
Column
Description
Product
The product with a usage exception.
Description
The product description.
Usage
The calculated monthly usage for the product.
Forecast
The calculated monthly forecast.
Exceptions
The type of exception:
•
U - Unusual usage. Monthly usage is greater than the total of the last 5 months of combined usage.
•
O - Out of stock. A product is out of stock for 2 weeks or more in one month.
•
S - Spike. Monthly usage is above or below the minimum or maximum levels set for the product by the percentage defined in Spike Exception - Min/Max set in Purchasing Parameters Maintenance (Systems menu). Refer to “Entering Formulas/Parameters by Company” in the Forecasting documentation.

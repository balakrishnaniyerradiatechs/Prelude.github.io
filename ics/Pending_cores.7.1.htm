Pending Cores
If you use and exchange core products for customers, you can set up core product records and track the pending cores for each customer.
A core product is the reusable part of a product that is returned by the customer either for credit or for a discount against a new product. You can set up an exchange product for the core, to identify the product that the returned core is associated with. For information on setting up a core and exchange products, refer to Core Products.
A pending core is created when you sell an exchange product to a customer but do not charge them for the core (reusable) part of the item. The core charge is in a pending status until it is billed to the customer or a core bank item is applied to it. A pending core is an open receivable.
A core bank item is a core product that the customer owns, but returns to you for their future use. The customer may apply the value of the core bank item when they buy a new product or have the old one repaired. A core bank item is an open payable.
When the customer returns the core you can apply it to the pending core by:
-
Adjusting inventory for the returned core by applying the returned core to the open sales order or returning it to a core bank. Refer to Receiving Cores in Inventory Adjustment Entry.
-
Using Core Bank Issues Entry to credit the customer for the returned core. Refer to Entering Core Bank Issues.
-
Using the Automatic Core Application Process to automatically link the pending core to the bank record. Refer to Applying Available Core Bank Items Automatically.
Example:
An alternator has a core value because part of it is reusable. It is put on an engine when it is new; then when it needs to be replaced it can be rebuilt and used again. When the customer buys the rebuilt alternator they are expected to return the old one (the core). If they don’t return it, the customer’s price of the rebuilt alternator includes a core charge.
You can handle the core charge in one of the following ways:
-
Bill the customer for the core charge.
-
Bill the customer, then issue a credit for the core charge when they return the core.
-
Do not bill the customer but track it as a pending core for the customer.
-
If the customer returns the core before it needs to be rebuilt, put it into the core bank for the customer.
The customer can store as many core products as they want in their core bank for future rebuilt alternators.

Related Reports
Description
Cores Received Report
Cores received from a customer whether the core was applied to a pending core record or placed in the core bank.
“Pending Cores Due Report” (Order Processing module)
Pending cores created when an exchange product was invoiced and the Bill Core Charge is set to N.
“Core Returns Due Aging Report” (Order Processing module)
Pending cores not yet received. You can see what is left to return and when it is expected to come in.
“Core (Issues) Sold on Exchange Report” (Order Processing module).
Displays core items sold on exchange (e.g., pending cores created because an exchange product was invoiced).

Assigning MSDS Numbers to Products
Once MSDS information is set up in Material Safety Data Sheet Maintenance (refer to Entering MSDS Numbers), you can assign the MSDS number to a product.
Order Processing Parameters
MSDS/Ship-To Specific determines if MSDS information is maintained in Customer Maintenance or Customer Ship-to Maintenance:
•
Y - Maintain MSDS information at the customer ship-to level (Customer Ship-To Maintenance).
•
N - Maintain MSDS information at the customer bill-to level (Customer Maintenance).
If you change the ship-to on a sales order or quote that has a restricted product, the system checks to see if the restriction applies to the new ship-to address.
 
MSDS/Notice Pick/Letter determines if MSDS information prints on the pick ticket or is mailed to the customer:
•
Y - The initial sale of an item that requires a MSDS adds a CP type comment line to the order to indicate that this item needs an MSDS.
•
L - The system stores the information to print a report to mail the MSDS to the customer.
•
B - Both Y and L options.
The customer is flagged as having received an MSDS for each item. The system will not add a comment or send a letter for subsequent sales of the same item to the customer unless the MSDS is revised.
To assign an MSDS number to a product:
1
Select Inventory > Maintenances > Product Maintenance > Product Maintenance to open the Product Maintenance screen.
2
Enter the Product.
3
Press F5-Add Info > F6-MSDS/Permits to open the MSDS/Restricted Permits Information Screen.
4
For MSDS Flag, enter Y to indicate that an MSDS is required for the product.
5
For MSDS Number, enter the MSDS number created in Material Safety Data Sheet Maintenance. Refer to Entering MSDS Numbers.
6
For Restricted Product, indicate if restrictions apply to the product. Choose from the following:
•
Y - Normal restrictions. The product requires that the customer have a restricted use permit to purchase this product. You must enter Usage Codes to define the restrictions. The customer must have the same usage codes set up in Customer Maintenance (Permits Information) to buy this product.
•
M - Matrix restrictions. Each state where this product is sold must be defined in Usage Codes along with the level of restriction.
•
N - No restrictions exist for this product.
7
For Usage Codes, enter the code that describes the type of restriction for the product. These codes are established in Restricted Usage Code Maintenance (Order Processing > Maintenance > Customer Control Files > Restricted Usage Codes). The customer must have the same usage code in their Customer Permit Information screen (Customer Maintenance > F7-Order > F5-Add Info > Action Bar > Permits) to buy the product.
8
For Level, enter the level of restriction for the usage code. This only applies if Restricted Product is set to M. Choose from the following:
•
0 - Do not sell.
•
1 - Restricted.
•
2 - General.
•
3 - No permit required.
9
Press F2-Save to save the MSDS information.
Sales Orders & Quotes
When you enter a line item on an order that contains an MSDS number, a comment line is automatically added referencing the revision number, last revision date, and bin number.
If you change the ship-to on a sales order or quote that has a restricted product, the system checks to see if the restriction applies to the new ship-to address.
Pick Ticket Printing
If you use Imaging, when the pick ticket prints, the system prints the MSDS image for each line item that contains an MSDS number.

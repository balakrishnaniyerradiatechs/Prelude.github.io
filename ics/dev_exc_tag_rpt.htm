Deviation Exception Report by Tag
The Deviation Exception Report by Tag lists items, sorted by tag number, with a difference between the saved on-hand quantity and the physical count quantity. It only includes products with a count entered. You can use this report to determine which tags/bins to recount.
This report is only valid if you run it before you update inventory (refer to Updating Physical Inventory From Tag Counts), and after you consolidate counts (refer to Consolidating Product Counts).
Menu Path: Inventory > Physical Inventory > Inventory Tag Procedure > Deviation Exception Report by Tag
Selection Criteria
Use the following criteria to limit the information to print on the report:
•
Warehouse. The warehouse where the inventory was counted.
•
Rental Product. Indicates if you want to include rental products on the report. Choose from the following:
-
Y - Include rental products.
-
N - Exclude rental products.
Column Descriptions
Column Heading
Description
Product Number
The product number with the count deviation.
Description 1
The product description.
Description 2
The second line of the product description, if applicable.
Physical Tag
The tag number assigned to the product.
Bin Number
The bin location of the product.
Count
The physical count entered for the product.
Bin Onhand
The saved on-hand quantity.
Avg-Cost
The cost of the product.
Dev $ Value
The total dollar amount of the deviation quantity (Bin Onhand - Count) x Avg-Cost).

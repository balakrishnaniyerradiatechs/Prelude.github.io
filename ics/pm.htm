Product Maintenance Screen
Product Maintenance allows you to create or edit basic product information that is common to all warehouses and vendors. From this screen you can access and edit usage, pricing, and purchasing information unique to each warehouse and vendor.
Menu Path: Inventory > Maintenance > Product Maintenance > Product Maintenance
Field & Function Key Descriptions
Field/Function Key
Description
Product
The product number to create or edit.
Est Date
The date the product was created.
Description
The product description.
Description Two
The second line of the product description, if applicable.
Product Line
The product line for this product (mandatory field). Product lines group similar products together for selling purposes (e.g., sales analysis, discounts, product campaigns, pricing). For more information, refer to Maintaining Product Lines.
Product Type
Identifies the product type used to group products for reports:
•
F - Finished good.
•
R - Raw material.
•
B - Both a finished good and a raw material.
•
E - Expense item.
•
N - Rental. If this is the first time you are setting up this product, opens the Rental Module Screen.
•
U - Used item.
•
P - Part. Same as raw material but is used for customer reporting purposes.
Minimum Sell Qty
The minimum quantity of this product that you should sell. If you enter an quantity on a sales order or quote for less than this amount, a warning appears. You can either default to the minimum or leave the quantity as entered.
Minimum Sales as Default Qty in Additional Options Maintenance (systems menu) determines whether to use a product’s Minimum Sell Qty, Selling Pkg Qty, or Shelf Pkg Qty (from this screen) as the default Quantity in the line item detail screen on sales orders and quotes:
•
Y - Use the Minimum Sell Qty if it exceeds the Selling Pkg Qty.
•
N - The system checks Pkg Qty and Shelf Pkg Qty and uses whichever one is larger.
Selling Pkg Qty
The selling package quantity. If you enter an quantity on a sales order or quote that is not a multiple of this amount, a warning appears and allows you to round up to the next selling package quantity. For example, if the Selling Pkg Qty is 10 and you enter 16, you can choose to either leave the 16 or to round up to 20.
If the Suppress Default Qty in SOE/QOE flag in Order Processing Parameters to Y, the warning message does not appear and you can enter any quantity.
Minimum Sales as Default Qty in Additional Options Maintenance (systems menu) determines whether to use a product’s Minimum Sell Qty, Selling Pkg Qty, or Shelf Pkg Qty (from this screen) as the default Quantity in the line item detail screen on sales orders and quotes:
•
Y - Use the Minimum Sell Qty if it exceeds the Selling Pkg Qty.
•
N - The system checks Pkg Qty and Shelf Pkg Qty and uses whichever one is larger.
Shelf Pkg Qty
The default shelf package quantity for stocking purposes. No message appears if you change the quantity on a purchase order.
Minimum Sales as Default Qty in Additional Options Maintenance (systems menu) determines whether to use a product’s Minimum Sell Qty, Selling Pkg Qty, or Shelf Pkg Qty (from this screen) as the default Quantity in the line item detail screen on sales orders and quotes:
•
Y - Use the Minimum Sell Qty if it exceeds the Selling Pkg Qty.
•
N - The system checks Pkg Qty and Shelf Pkg Qty and uses whichever one is larger.
Transfer Pkg Qty
The default transfer package quantity Nightly Processing uses to recommend a stock transfer. If you enter a quantity of less than this amount on a transfer, the system automatically rounds up to the next package quantity. A warning message appears if you override the quantity but does not prevent you from overriding it.
Stock/Buy Ratio
Works with the Buy and Stock units of measure to determine the number of units to increase/decrease inventory for each unit received. Nightly Processing recommends a purchase quantity divided by this stock/buy ratio.
For example, if the Buy unit of measure is CS, the Stock unit of measure is EA, and the Stock/Buy Ratio is 12, for every unit you receive, your inventory increases by 12. If you receive 5 cases, your inventory increases by 60 (5 x 12). You must enter pricing in terms of the Price unit of measure and costs in terms of the Stock unit of measure.
You cannot use this function with the U/M Table.
Special Operation
The special operations code for the product:
•
B - Belting (Lot Number Flag must be set to Y and the Belting module must be activated). Opens the Special Operations Keywords screen to allow you to enter a vendor number to search for the product in the belt search program.
•
BF - Belting finished good (Lot Number Flag must be set to Y and the Belting module must be activated).
•
BC - Base child product. The product is a child product of a base master product (Special Operations code = BM). The Special Operations Master screen appears and you can enter or view the base master product. The stocking unit of measure should be the same for both the master and the child products. Both the base child and the base master products display in product lookup screens. Nightly Processing does not recommend a purchase order for base child products.
•
BM - Base master product. Any base child products (Special Operations code = BC) assigned to this product display. You may want to use this if you have a product that comes in different package quantities. Both the base child and the base master products display in the tagging screen on a sales order so that you can choose the proper package. Any time a field within the base master product is altered, you have the option to update all base child products also (if the information is the same). The stocking unit of measure must be the same for both the master and the child products.
•
CB - Configurable Base. Opens the Product Configuration Maintenance screen to set the product as a base product (unless it exists on open transactions). Refer to Configuring Base and Child Products.
•
CC - Configured Child. Indicates if the product is a child of a configurable base product. Refer to Configuring Base and Child Products.
•
G - Gasket Square Inch. Opens the Gasket Related Product Maintenance Screen to allow you to enter bulk parts associated with the remnant product. This information is required for the Gasket/Hose Subsystem.
•
GC - Gasket Kickoff Code. Opens the HConfigurator Maintenance Screen to allow you to enter configurator components for a gasket kickoff code. This option is only applicable for the Gasket Hose Subsystem.
•
H - Hose (Lot Number Flag must be set to Y and the Belting module must be activated). Opens the Special Operations Keywords screen to allow you to enter hose/vendor information to assist in product/belting searches.
Special Operations (continued)
 
•
HC - Hose Configurator. Opens the HConfiguration Maintenance Screen to allow you to enter attribute options for a configurator class. This option is only applicable for the Gasket Hose Subsystem.
•
HK - Hose Kickoff Code. Opens the HConfigurator Maintenance Screen to allow you to enter configurator component information for a hose kickoff code. This option is only applicable for the Gasket Hose Subsystem.
•
L - Length. Requires length when you enter an X for length in generic products. Opens the Special Operations Length Info screen to allow you to enter a length pattern and an increment to use for length rounding.
•
LW - Remnant product (Lot Number Flag is set to Y). You can enter a Base Product Number, which is the original product before it is cut. Refer to Minimum Remnant Quantity in the Lot Information Screen.
•
PL - Piece/length. You can specify the number and length of the drums or reels. For example, if you enter 5 pieces at a length of 10 ft., the system automatically calculates 50 ft. Refer to Minimum Remnant Quantity in the Lot Information Screen.
•
RL - Rotation and Length. Requires rotation and length when you enter an X for length in generic products.
•
RX - Prescription. Opens the RX Group Maintenance screen to allow you to enter the prescription group for this product. Refer to “Tracking Physician Prescriptions” in the Order Processing Documentation for more information.
Obsolete Flag
Determines if the item is obsolete:
•
Y - The product is obsolete. You can sell the available quantity but cannot generate a backorder. You cannot enter the product on a purchase order.
•
N - The product is not obsolete.
Service Item Flag
Indicates if the product is a service item for expense:
•
Y - The product is a service item (e.g., labor, expense, sublet). No inventory quantity is updated. You must use a product line set up just for service items so that the general ledger accounts are overridden and inventory is not tracked (refer to the Product Line Maintenance Screen).
•
N - The product is not a service item and is tracked in inventory.
Service Item Type (unlabeled field)
If you entered Y in Service Item Flag, you must enter a Service Item Type to post to the correct general ledger account:
•
C - Core. Work Order Completions tracks the value to add it to the core value of the item.
•
E - Expense. Non-inventory expense (e.g., packing supplies, equipment fees).
•
J - Job. Fee for a job performed.
•
L - Labor. Charge for labor performed.
•
M - Material. Accessory parts not tracked in inventory (e.g. catalogs, flyers).
•
S - Sublet. Work performed outside of your company.
•
W - Extended Warranty. The warranty expiration date is calculated using the Warranty Period Unit information for the product in the Additional Product Maintenance Information Screen. When a warranty product is entered on a sales order, the Warranty Product screen appears.
U/M Table
Indicates if a unit of measure table is required for this product. If the product is purchased, stocked, and sold in different units of measure, you can require a unit of measure table:
•
Asterisk (*) - A unit of measure table already exists for the product.
•
Y - A unit of measure table is required for the product. The U/M Table Maintenance screen appears to allow you to enter a unit of measure table. The unit of measure table uses Buy, Stock, and Price to affect how the system buys, stocks, and sells the item. It can also track inventory using multiple units of measure and inventory factors.
Refer to Establishing a Unit of Measure.
•
N - A unit of measure table is not required.
You cannot use Stock/Buy Ratio with a unit of measure table.
Buy
The unit of measure to use when you purchase the product from a vendor. However, a Buy unit of measure defined at the product/warehouse/vendor level overrides this unit of measure. Refer to Maintaining a Product by Warehouse/Vendor.
If you buy and stock at a different units of measure, you can either use Buy/Stock Ratio or the U/M Table. Refer to Establishing a Unit of Measure.
Stock
The unit of measure to use for stock in your warehouse. Enter all costs for the product and product-warehouse level in the terms of the Stock unit of measure.
If you buy and stock at a different units of measure, you can either use Buy/Stock Ratio or the U/M Table. Refer to Establishing a Unit of Measure.
Price
The unit of measure to use to sell to your customers. Enter all prices at the product and the product-warehouse level in terms of the Price unit of measure.
Refer to Establishing a Unit of Measure.
F2-Save
Saves the product information.
F4-Del
Deletes the product.
F5-Add Info
Opens the Additional Information Screen to allow you to enter flag settings, codes, and other information to store in the master product.
F6-Xref/Notes
Opens the Product Xref/Notes Screen to allow you to enter cross-reference and related product information for a product.
F7-Pricing
Opens the Product Pricing Screen to allow you to enter costs and discounts to calculate prices.
F8-Warehouse
Opens the Product Warehouse Maintenance Screen to allow you to enter product information specific to a warehouse.
Action Bar
Move
Moves the product to the catalog file if it is currently in the product file or to the product file if it is currently in the catalog file. Catalog products are moved to the inventory file during physical inventory so that they will appear on the count sheets (refer to Merging Catalog Products into Inventory Count Lists).
The N/S Product Merged Report & Process displays the products moved into the product file and gives you the option to move them back to the catalog file.
Copy
Only available before you enter a product number. The Product Maintenance Copy screen opens to allow you to copy all information from one product number to a new one. Refer to Copying a Product to Create a New Product for information on copying products.

Attachments - Allows you to attach pictures or files to the product.
•
Imaging - If you use imaging, you can view the following document images related to the product.

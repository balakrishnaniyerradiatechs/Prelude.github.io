Transaction Audit Inquiry
The Transaction Audit Inquiry displays the transactions that affected the on-hand quantity or average cost of a product in a warehouse. You can also view the detail information for each transaction.
Viewing Transactions
To view the transactions for a product:
1
Select Inventory > Inquiries > Transaction Audit Inquiries > Transaction Audit Inquiry to open the Transaction Audit Inquiry screen.
2
Enter the Product.
3
Enter the Warehouse number.
4
For Sort Sequence, determine how to display the transaction information:
•
B - Display entries in ascending order, from the oldest to the newest. If you enter a beginning date, only transactions on or after the date display.
•
E - Display entries in descending order, from the newest to the oldest.
•
F - Filter the transactions to display. You can enter beginning and ending dates and the transaction codes to display (Refer to the list of Transaction Audit Inquiry Codes). In this case, the true on-hand, average cost, and linked cost may not represent a sequential audit of the product’s activities.
The transaction codes display along with reference numbers, values, and on-hand quantities.

U/M
The stocking unit of measure from Product Maintenance.
Linked Value
The value of on-hand inventory linked to sales orders, work orders, or transfers.
Stock Value
The value of on-hand stock (unlinked) inventory.
Code
The code representing the transaction that affected the product. Press F3 to see a list of codes. Refer to Transaction Audit Inquiry Codes.
Date
The date the transaction took place.
Reference 1
A reference number for the associated transaction (e.g., sales order number, purchase order number, invoice number, receipt number). Refer to Reference 1 on the list of Transaction Audit Inquiry Codes.
Reference 2
A reference number for the associated transaction (e.g., sales order number, purchase order number, invoice number, receipt number). Refer to Reference 2 on the list of Transaction Audit Inquiry Codes for an explanation of each reference number.
CN/VN/Whse
The customer number, vendor number, or warehouse number associated with the transaction.
Units
The number of units going into or out of inventory. A negative value means the amount was deducted from inventory.
Value/Unit
The cost per unit for the transaction.
Cost
The new average cost of the item after the transaction.
Onhand
The total on-hand balance of linked and unlinked inventory.
F2-Finish
Closes the inquiry screen.
F5-Review
Allows you to view the detail and historical information of the highlighted transaction. The type of transaction code selected determines the type of display screen that appears.
F6-Report
Allows you to send the inquiry information to the screen, the printer, an email, or a fax. The resulting report displays the bins, lots, and serial numbers of the transactions.
F7-Expanded
Displays the Expanded Transaction Audit Inquiry screen which includes the customer, warehouse, or vendor description.
F9-Goto Line
Allows you to enter a line number, page number, or reference number to go directly to.
sF5-Prev Group
Returns to the previous page if more than one page of transactions are listed. The number of records that display depend on Records to Block in Company Maintenance > Action Bar > GUI Options.
sF6-Next Group
Forwards to the next page if more than one page of transactions are listed. The number of records that display depend on Records to Block in Company Maintenance > Action Bar > GUI Options.
Action Bar
By Bin
Opens the Transaction Audit by Bin Inquiry to display transactions for the product by bin.
By Lot
Opens the Transaction Audit By Lot Inquiry to display transactions for the product by lot number.

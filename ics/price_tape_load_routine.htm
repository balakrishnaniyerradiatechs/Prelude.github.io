Price Tape Load Routine screen
Once you set up the required price tape information (refer to Setting Up a Price Tape), you can use the Price Tape Load Routine screen to load the price tape.
Menu path: Inventory > Processes > Price/Cost Update Procedure >Price Tape Update > Price Tape Load
Field and Function Key Descriptions
Field/Function Key
Description
Name On Diskette or Data File
The full name of the price tape.
Vendor
The vendor associated with the price tape.
File Type
Indicates the price tape source:
•
C - CD. You first must go to UNIX as root and type
mount /cdrom. When finished with the load, type unmount /cdrom.
•
D - DOS diskette or the /tmp UNIX directory.
•
P - Data already stored in the Pick/UniData file.
Warehouses
The default warehouse to use for any new products created in the price tape load. Warehouse numbers are not necessary if you are only updating existing locations.
Key to Build System Product Number
Information required to create your product number from the tape’s data:
•
If your part number is the same as the vendor’s, this field displays the number position on the tape that contains the vendor’s part number.
•
If your part number consists of the first three characters of the vendor’s name, connected by a dash and linked to the vendor part number, this field displays as “TOR-”:3. The 3 characters inside the quotes represent the vendor. The number 3 represents the third field of the tape to use as a component of your part number.
Fixed Length or Delimited
The type of file for the price tape update:
•
F - A fixed-length file that has the data as a set number of characters for each field.
•
D - A delimited file that has a value separating data into fields (e.g., tab, comma).
Price Effective Date
The date the prices takes effect, if a price is loaded into a future price field.
Load and Display Non Update Data?
Determines the items to display on the report:
•
Y - Display non-updated products on the Pre-Update Report.
•
N - Display only updated items on the Pre-Update Report.
Create Products Not On System?
Indicates if the price tape update should create products that are on the price tape but not in the system:
•
C - If products exist on the price tape that do not exist in your system, create the new product records in the catalog file (products sold by the vendor but are not active products in your system).
•
P - If products exist on the price tape that do not exist in your system, create the new product records in the inventory file (includes stock and non-stock products).
•
N - Do not create any new products.
Tape Data
Fld#
The next available field number if entering multiple fields.
Pos
The starting position on the diskette for this record. If using a delimited field, then enter the field number or return to advance automatically.
Length
If using fixed length, the length of the field. If using delimited, the single character that separates the fields from each other (e.g., : or “ or C9 for tab delimited).
Format
The number of decimals in the raw data file.
File Name
The file to update (e.g., PRODUCT, PRD-VEND).
File Data
Dictionary
The dictionary name where the field from the tape is stored.
Format
The number of decimals in the raw data file.
F2-Load
Loads the price tape information. The product numbers are built and stored in the proper format in the PRICE.WORK file.
The Pre-Update report allows you to verify the prices before you update the products (refer to Running the Price Tape Pre-Update Report).

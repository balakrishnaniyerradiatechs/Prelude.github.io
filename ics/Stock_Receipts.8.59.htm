Stock Receipts Discrepancy Inquiry
The Stock Receipts Discrepancy Inquiry displays receivers on hold for the buyer to review. Depending on the reason why the receiver went on hold, it may or may not need to be released from hold before Accounts Payable can reconcile it with the vendor invoice.
Receiving Discrepancy Manager
Your Buyer Code must be set up as the Receive Disc Buyer in Warehouse Manager Maintenance to be able to release orders on Stock Receipts Discrepancy hold for your warehouse.
Reasons Why Stock Receivers Go On Hold
Stock receivers may go on hold for the following reasons:
•
Manual buyer hold: To manually place a receiver on hold while in Stock Receipts Entry, select the Buyer Hold option on the Action Bar. You can enter notes to the buyer to explain why you are putting the receiver on hold.
The Stock Receipts Discrepancy Inquiry displays an M reason code for the held product. The receiver must be released before Accounts Payable can reconcile it with the vendor invoice.
•
Rejection: You can reject an item in Stock Receipts Entry (refer to Rejecting Stock) if the vendor sent an incorrect or damaged product.
The Stock Receipts Discrepancy Inquiry displays the rejection reason code for the held product. The receiver must be released before Accounts Payable can reconcile it with the vendor invoice.
•
Over-received quantity: You can set over-received tolerance percentages (Warehouse Maintenance > F5-Add Info > F7-Variances > Over Rcvd Unit Variance%) to define the percentage the quantity an item can be over-received before going on hold. To further define the over-received quantity limit, you can enter multiple cost ranges to correspond to over-received quantity limits.
The Stock Receipts Discrepancy Inquiry displays a Q reason code for the over-received product. The receiver must be released before Accounts Payable can reconcile it with the vendor invoice.
•
Under-received quantity: You can set under-received tolerance percentages in (Warehouse Maintenance > F5-Add Info > F7-Variances > Under Rcvd Unit Var Pct) to define the quantity an item can be under-received before going on hold.
To further define the under-received quantity limit, you can enter multiple cost ranges to correspond to over-received quantity limits.
The Stock Receipts Discrepancy Inquiry displays a Q reason code for the over-received product. The receiver does not have to be released to allow Accounts Payable to reconcile it with the vendor invoice.
•
Over cost tolerance: You can set a high cost variance percent (Warehouse Maintenance > F5-Add Info > F7-Variances > High $ Cost Var Pct) to define the percentage a received cost can be higher than the purchase order cost.
The Stock Receipts Discrepancy Inquiry displays a C reason code for the held product. The receiver does not have to be released to allow Accounts Payable to reconcile it with the vendor invoice.
•
Under cost tolerance: You can set a low cost variance percent (Warehouse Maintenance > F5-Add Info > F7-Variances > Low $ Cost Var Pct) to define the percentage a received cost can be lower than the purchase order cost.
The Stock Receipts Discrepancy Inquiry displays a C reason code for the held product. The receiver does not have to be released to allow Accounts Payable to reconcile it with the vendor invoice.
For information on receivers placed on hold for quality control inspection, refer to Requiring Quality Control Inspections in Receiving.
Releasing a Stock Receiver from Hold
To release stock receivers on hold:
1
Select Inventory > Inquiries > Stock Receipt Inquiries > Stock Receipts Discrepancy Inquiry to access the Stock Receipts Discrepancy Inquiry screen.
2
Enter the Buyer.
3
Press ENTER to display the receivers on hold for the buyer.
4
Highlight the product number to review and/or release and select one of the following:
•
F5-Review to open the Purchase Order History Display screen to view detail information for received items.
•
F6-Rejection Info to open the Line Item Quality Control screen to review the rejection reason and any notes entered.
5
Release the order by entering a Y in Release.
6
After you review and release the receivers, press F2- Finish to close the screen.
Field & Function Key Descriptions
Field / Function
Description
Buyer
The buyer responsible for reviewing and releasing the products on hold. The buyer number assigned to the primary vendor for each product determines which buyer queue contains each product on hold.
Product
The product number on hold.
Description
The description of the product.
P/O Number
The receiver that is on hold.
Qty-Opn
The quantity remaining on the purchase order that was not received.
Qty-Rec
The quantity received on this receiver.
Reason
Displays the reason code for why the receiver was put on hold:
•
Q - The quantity received was over or under the allowed tolerance percent set in Warehouse Maintenance.
•
C - The cost of the item is over or under the allowed cost tolerance percent set in Warehouse Maintenance.
•
M - The receiver was manually placed on hold in Stock Receipts Entry (Action Bar > Buyer Hold).
•
The rejection reason code entered in Stock Receipts Entry (detail Information Screen > F5-Reject Claim).
Release
Allows you to release the receiver:
•
Y - Release the receiver.
•
N - Do not release the receiver.
F2-Finish
Closes the screen.
F5-Review
Opens the Purchase Order History Display screen to allow you to review the purchase order.
F6-Rejection Into
Opens the Line Item Quality Control screen to display the rejection reason, quantity, warranty information, and any notes.
Action Bar
Alternate Buyer
Opens the Alternate Buyer screen to allow you to send the receiver to another buyer’s work queue.

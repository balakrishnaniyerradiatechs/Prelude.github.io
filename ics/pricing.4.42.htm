Setting Up a Price Tape
Price Tape Setup allows you to enter the information necessary to update prices and costs with a price tape, CD, diskette, or UniData file.
Always update on your test account first.
To set up a price tape:
1
Select Inventory > Processes > Price/Cost Update Procedure > Price Tape Update > Price Tape Setup to access the Price Tape Setup Maintenance screen.
2
For Name on Diskette or Data File, enter the name (including the extension) of the file to load. (e.g., PRC 2005.csv). Press F5-Select From tmp to look for the file in the tmp directory in UNIX.
3
Enter the Vendor for the update.
4
Enter the Warehouses to contain the new products created. This step is not necessary if you are only updating existing locations.
5
For Key to Build System Product, enter the information required to create your product number from the tape’s data:
•
If your part number is the same as the vendor’s, enter the number position on the tape that contains the vendor’s part number.
•
If your part number consists of the first three characters of the vendor’s name, connected by a dash and linked to the vendor part number, enter “TOR-”:3.
-
The 4 characters inside the quotes represent the vendor.
-
The number 3 represents the third field of the tape to use as a component of your part number.
•
If you want to only include a portion of the field, use square brackets (e.g., 3[x,y]).
-
The number 3 represents the field number to extract from.
-
The x is the starting position of the field to extract.
-
The y determines how many characters to extract.
6
For Fixed Length or Delimited, select the type of file:
•
F - The price tape is a fixed length file; each piece of data has a set number of characters.
•
D - The price tape is a delimited file; a character separates the data into fields, (e.g., -, tab).
7
For Price Effective Date, enter the date the prices takes effect if a price is loaded into a future price/cost field.
8
For Load and Display Non Update Data?, choose from the following:
•
Y - Display products on the Pre-Update report even if they do not exist in the system and will not be updated.
•
N - Do not display items products on the Pre-Update report that have no product match in the system.
9
For Create Products Not On System?, indicate if you want to add products that are not in the system:
•
C - Create new products in the catalog file, but not in inventory.
•
P - Create new products in inventory.
•
N - Do not create any new products.
10
If Create Products Not on System is set to P or C, you can enter a product line for the products created during the price tape update.
11
For Skip P/N if Vendor Doesn't Match?, indicate if you want to update the product if its primary vendor does not match the vendor on the price tape:
•
Y - Do not update the product if the primary vendor does not match.
•
N - Update the product even if the primary vendor does not match.
12
For Position and Length under Tape Data, depending on which option you chose in step 6, choose from the following:
•
If you chose fixed length, enter the starting position on the diskette for this record and then the length of the field.
•
If you chose delimited, enter the field number and then the single character that separates the fields from each other (e.g. , : or “, or C9 for tab delimited).
13
Enter the Format of the field. For example:
•
Enter 3 if 12000 should display as 12.000.
•
Enter .3 if the decimal is already set in the correct place, (e.g., 12.000).
•
Enter D if the field is a date.
14
For Filename, enter the system filename to update. For example, enter PRODUCT to update the product file.
15
Under File Data, enter the Dictionary name where the field from the tape is stored.
16
Enter the Format of the File Data field. For example:
•
Enter 3 if 12000 should display as 12.000.
•
Enter .3 if the decimal is already set in the correct place, (e.g., 12.000).
•
Enter D if the field is a date.
17
For Override, indicate if you want to override the existing data with data from the price tape:
•
Y or null - Overwrite the data.
•
N - Do not overwrite the data.
18
For Conv., enter Y if this field should be converted before updating.
Do NOT save the price tape until you validate the Conv field. Make sure that if you choose to overwrite, it properly writes to the TAPE.FMT file. This file is used during the Price Tape Update to tell the system where to load the data from the tape. Contact Activant for assistance.
19
Press F7-Copy to New Record to copy these settings for another price tape.
20
Press F2-Save to save the price tape setup information.
Once the setup is complete, you can load the price tape. Refer to Loading a Price Tape.

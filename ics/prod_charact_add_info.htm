Product Characteristics Additional Information screen
The Additional Information screen in Product Characteristics Code Maintenance allows you to enter validation information required for each characteristic description.
Menu Path: Inventory > Maintenance > Product Control Files > Product Characteristics Codes > F5-Add Info
Field and Function Key Descriptions
Field/Function Key
Description
Description
The description from the Product Characteristics Code Maintenance screen that identifies the information required when an item with this characteristics code is sold.
Default Process
The default information to display when the salesperson enters characteristics’ information.
Validation Process
The type of information required by the salesperson to validate the characteristics entered:
•
List of valid values (e.g., Y or N).
•
Key on another file.
•
Expression.
•
Range of values.
•
Pattern match.
•
User subroutine.
•
Length.
•
Translate codes.
•
No spaces.
•
Unique values only.
•
Mandatory field.
•
Integer (no decimal point).
•
Existing data file.
•
Validation list expression.
•
Field input restrictions.
•
Call process.
See your systems administrator for more information.
Intuitive Process
The selection process that validates responses.
On Line Help
The instructions to assist the salesperson.
Notes Before Field
The notes to appear before the salesperson sees the product characteristics description. These notes provide additional information to help sell the product.
Notes After Field
The notes (or process name) to appear after the salesperson sees the product characteristics description. These notes provide additional information when the item is sold.
F2-Save
Saves the validation information.
F5-Invoke Text Editor
Works as a carriage return if you use the character screen.

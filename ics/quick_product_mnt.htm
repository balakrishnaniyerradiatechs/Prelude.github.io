Using Quick Product Maintenance
If you want to set up a product for a warehouse or several warehouses without having to enter all of the detail information right away, you can use Quick Product Maintenance to set up the basic product information necessary to enter a product on an order. You can then go back later and complete the detail and pricing information in Product Maintenance and Product Warehouse Maintenance.
Menu Path: Inventory > Maintenance > Product Maintenance > Quick Product Maintenance
Field & Function Key Descriptions
Field/Function Key
Description
Product
The product number to create. Existing products cannot be edited in Quick Product Maintenance.
Description
Description of the product.
Description Two
The second line of description for the product, if applicable.
Product Line
The product line for this product (mandatory field). Product lines group similar products together for selling purposes (e.g., sales analysis, discounts, product campaigns, pricing). Refer to Maintaining Product Lines.
Product Type
The product type:
•
F - Finished good.
•
R - Raw material.
•
B - Both a finished good and a raw material.
•
E - Expense item.
•
N - Rental item. Opens the Rental Module Screen.
•
U - Used item.
Unit of Measure
Buy
The default buying unit of measure for purchase orders.
Stock
The default stocking unit of measure used to calculate the on-hand inventory.
Price
The default selling price unit of measure.
List
The published list price.
Price 1 - Price 3
Default customer prices. Each customer is assigned a price code to use for this product. You can set prices by product or by product/warehouse.
Replacement Cost
The replacement cost of the item for all warehouses and vendors. You can override this cost at the product/warehouse level (Warehouse Pricing Screen), the company vendor level (refer to Setting Up Company Vendor Cost), or the product/vendor/warehouse level (Product Vendor Maintenance Screen). You can also change replacement cost when you purchase or receive the product, depending on how Replenishment Cost Update is set in Inventory Option Parameters.
Product Prc Group
The product price group used to group products for special pricing. Product price group codes are established in Product Price Group Maintenance. Refer to Establishing Product Price Groups.
Primary Vendor
The primary vendor for the product. Nightly Processing uses this vendor to recommend a purchase order.
Vendor Product
The vendor’s product number.
Minimum Sell Qty
The minimum selling quantity for this product. If you enter an order for less than this amount, a warning message displays but you are not required to increase the amount.
Selling Pkg Qty
The selling package quantity. If you enter an quantity that is not a multiple of this amount, a warning appears and allows you to round up to the next selling package quantity. For example, if the Selling Pkg Qty is 10 and you enter 16; you can choose to either leave the 16 or to round up to 20.
Shelf Pkg Qty
The default shelf package quantity for this product for stocking purposes. No message appears if you change the quantity.
Weight
The default weight of the product in pounds.
Whse
The warehouses that stock this product. Enter ALL to stock the product in all warehouses.
Mn/Mx
Determines if minimum and maximum stock quantities are used to generate purchase quantities:
•
Y - The warehouse uses min/max levels for replenishment.
•
N - The warehouse does not use min/max levels for replenishment.
Min
If Mn/Mx is set to Y, the minimum amount of stock to keep at the warehouse for this item.
Max
If Mn/Mx is set to Y, the maximum amount of stock to keep at the warehouse for this item.
Frz Cntrl
Indicates if purchasing parameters are frozen (refer to Freezing and Thawing Product Purchasing Controls). When you freeze controls for a product, the purchasing parameters calculated by end-of-month processing are ignored. For example, purchasing parameters can be frozen for new products that do not have any usage on which to base purchasing parameters. For these products, you can set minimum and maximum stocking quantities, freeze the controls and set a thaw date after which the product has enough usage for purchasing parameters to create useful data. Choose from the following:
•
Y - The minimum and maximum stocking quantity overrides that you enter are used instead of the purchasing parameters calculated by end-of-month processing until the product is off frozen controls.
•
N - Purchasing parameters calculated by end-of-month processing are used to recommend purchase quantities.
Thaw Date (unlabeled field)
If you entered Y in Frozen Controls, enter a thaw date. After that date, purchasing parameters are calculated based on the product’s usage. If you do not enter a thaw date, the purchasing parameters are frozen indefinitely. Refer to Freezing and Thawing Product Purchasing Controls for information on manually thawing products.
Stat
The stocking status code:
•
S - Stock. The product is a regular stock item for this warehouse and replenished according to purchase parameters.
•
N - Nonstock. The product is not a stock item for this warehouse. You can only purchase this product if it is linked to a sales order.
•
C - Consignment. The product is on consignment from the vendor.
Meth
Defines the replenishment method for the product:
•
P - Purchase. Creates a temporary purchase order. If the product is a stocking product, it is combined with other products in the same vendor line and totals are compared to the vendor line’s targets.
•
D - Direct. Creates a temporary purchase order in the central warehouse.
•
T - Transfer. Creates a temporary transfer.
•
M - Manufactured. Creates a temporary work order for products that are manufactured internally. If you choose this option, the Production Control Parameters screen opens to allow you to enter production control parameters for the work order planner/buyer. Refer to Production Control Groups.
•
O - Outplant. Creates a temporary work order for a product that is out-sourced to a manufacturer.
Whse
The replenishment warehouse:
•
If Meth is T, the warehouse to transfer from.
•
If Meth is D, the central warehouse that should purchase this product.
Vline
The vendor line number for the product. Vendor lines group products to meet buying requirements.
StdPk
The standard purchasing package quantity for the product. You can modify this quantity for each vendor in the Product Vendor Maintenance Screen.
Bin
The bin location for the product for each warehouse.
F2-Save
Saves the product information.
F5-Add Info
Allows you to enter additional information regarding the product. Refer to the Additional Information Screen.
F6-Xref/Notes
Allows you to enter cross-reference, substitute, and other product information. Refer to the Product Xref/Notes Screen.
F7-U/M Table
Opens the U/M Table Maintenance screen. Refer to Establishing a Unit of Measure.
Action Bar
Price Qty Breaks
Opens the Quantity Break Table Screen to allow you to enter prices for an item based on quantity breaks.
Cost Qty Break
Opens the Cost Quantity Break Table Screen to allow you to enter cost based on quantity breaks.
Language Codes
Opens the Product Language Code Mnt Screen to allow you to enter the product description in another language.

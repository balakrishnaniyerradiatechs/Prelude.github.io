Price Tape Update Routine screen
The Price Tape Update Routine screen allows you to update the prices and costs information generated in the Price Tape Load (refer to Loading a Price Tape).
Menu Path: Inventory > Processes > Price/Cost Update Procedure > Price Tape Update > Price Tape Update
Field and Function Key Descriptions
Field/Function Key
Description
Name On Diskette
The full name of the price tape.
Vendor
The vendor associated with the price tape.
Warehouses
The default warehouse to use for any new products created in the process. Warehouse numbers are not necessary if you are only updating existing locations.
Key to Build System Product
Information required to create your product number from the tape’s data:
•
If your part number is the same as the vendor’s, this field displays the number position on the tape that contains the vendor’s part number.
•
If your part number consists of the first three characters of the vendor’s name, connected by a dash and linked to the vendor part number, this field displays as “TOR-”:3. The 3 characters inside the quotes represent the vendor. The number 3 represents the third field of the tape to use as a component of your part number.
Fixed Length or Delimited
The type of file to use with the price tape update:
•
F - The price tape is a fixed length file; each piece of data has a set number of characters.
•
D - The price tape is a delimited file; a character separates the data into fields, (e.g., -, tab).
Price Effective Date
The date the prices takes effect if a price is loaded into a future price field.
Load and Display Non Update Data?
Determines the items to display on the report:
•
Y - Print non-updated items on the Pre-Update Report.
•
N - Print only updated items on the Pre-Update Report.
Create Products Not On System?
Indicates if the price tape update should create records for products not currently on the system:
•
C - Create records in the catalog file.
•
P - Create records in the product file.
•
N - Skip all missing product records.
Tape Data
Fld#
The next available field number if entering multiple fields.
Pos
The starting position on the diskette for this record. If you are using a delimited field, then enter the field number.
Length
If using fixed length the length of the field. If using delimited, the single character that separates the fields from each other (e.g., : or “ or C9 for tab delimited).
Format
The number of decimals in the raw data file.
File Name
The filename to update.
File Data
Dictionary
The dictionary file to update.
Format
The number of decimals in the raw data file.
F2-Save
Starts the price tape update. A dialog box appears and gives you the option to save the previous prices. You must also choose whether to update all warehouses are just the selected warehouses.

Bin Location Maintenance screen
The Bin Location Maintenance screen contains information for a bin to help track and identify inventory for replenishment and cycle counting. Bin locations are assigned to products in the Product Warehouse Maintenance Screen.
Refer to Using Bins for more information.
Menu Path: Inventory > Maintenance > Bin Files > Bin Locations
Field and Function Key Descriptions
Field/Function Key
Description
Warehouse
The warehouse where the bin exists.
Bin
The alphanumeric number that identifies the bin.
Description
A descriptive name of the bin to help identify the contents of the bin.
Zone
The zone where the bin exists, if using zones to group bins. Refer to Defining Zones.
Cycle Count Zone
The zone that controls the bin’s cycle count so that you can print cycle count sheets by zone, if using Cycle Counting. If this field is blank, the Zone already established for the bin is used.
Permanent Code
Indicates if this is a permanent bin:
•
P - Permanent bin (default). This is used when a product is normally stored in the same location. Even when the product is out of stock and the bin is empty, the space is reserved for the next stock receipt of the item.
•
T - Temporary Bin. A temporary storage area can be used to hold material on orders that need to be staged before shipment. Deletes the products from the list of products in this bin when the quantity reaches zero. You can determine whether or not to delete the bin when the last product is removed.
•
R - Retained Temporary Bin. Automatically deletes the products from the list of products in this bin when the quantity reaches zero (unless the bin is the primary bin for the product). This option does not delete the bin when the last product is removed. You must delete the product record manually in Bin Location Maintenance.
Bin Type
The type of products in the bin. Only pick, bulk, and retail Bin Types are used with cycle counting. If you use the Warehousing module, the replenishment process uses this information:
•
B - Bulk. Stores bulk quantities of the product.
•
C - Crib. Allows you to enter the customer information for a crib/warehouse.
•
D - Receiving dock. A bin located on the receiving dock.
•
I - Inspection. Temporarily stores products awaiting inspection.
•
K - Bulk un-packaged.
•
L - Work-in-process line stock.
•
O - On-the-water bin for imported products.
•
P - Pick. Primary picking location.
•
R - Repackaging. Used to repack replenishment material.
•
S - Staging. Temporary storage area to accumulate material before being shipped.
•
T - Retail. A retail sales location.
•
U - Un-allocated. Miscellaneous stock usually not available for sale.
•
W - Work in process.
Pick Reserve
The pick status of this bin (used with the Warehousing module):
•
N - Not available for picking.
•
R - Pickable.
•
R - Reserved for a customer.
Environment Code
The type of controlled environment for the bin location (used with the warehousing module):
•
E - EPA controlled.
•
F - Fire rated.
•
N - Normal.
•
R - Refrigerated.
•
S - Secured.
Storage Type
Indicates how the products in this bin are stored (used with the Warehousing module):
•
P - Pallet racks.
•
R - Flow racks.
•
C - Carousel.
•
F - Floor stock.
•
S - Shelf.
•
M - Mobile.
•
W - Work center.
•
T - Truck.
•
O - Outplant.
•
G - Pre-stage.
Vendor Number
The vendor for bin method outplant processing in the Warehousing module.
Sort Sequence
Indicates how to sort the bins for picking. Refer to Sorting Bins by Warehouse Location for information on the Bin Location Sort Sequence Procedure.
Putaway Sort Seq
The sort sequence code for sorting putaway bins. If this field is blank, the Sort Sequence Code is used.
Nettable Flag
Indicates if this bin is nettable:
•
Y - The bin location is nettable (the material is available to sell).
•
N - The bin location is non-nettable (the material is not available to sell).
This flag cannot be changed if users are on the system and products exist in the bin. It must be changed at the end of the business day so that Nightly Processing can rebuild non-nettable quantities used for availability calculations.
Auto Replenish Flag
Indicates if this bin is included in the auto replenish procedure (Warehousing module):
•
Y - The bin is included in the auto replenish procedure.
•
N - The bin is not included in the auto replenish procedure.
Putaway Status
Indicates if this bin is active (information only). Choose from the following:
•
A - The bin is active.
•
I - The bin is inactive.
Date Established
The date the bin was created.
Aisle
The aisle containing the bin.
Bin Width
The width measurement of the bin.
Bin Height
The height measurement of the bin.
Bin Depth
The depth measurement of the bin.
Bin Volume
The cubic volume of the bin.
Bin Weight
The weight of the bin.
F2-Save
Saves the bin information.
F4-Del
Deletes the bin location.
F5-Cycle Count History
Opens the Cycle Count History screen to display cycle count and product information for the bin.
Action Bar
Copy
Allows you to copy existing bin information to create a new bin location.

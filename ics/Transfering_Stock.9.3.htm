Defining Transfer Types
Each transfer starts with a default transfer type, usually 01. You can change the default transfer type when you begin the transfer, but you cannot change it after you save the header screen.
Stock Transfer Type
You can set up a Transfer Type Default and Valid Transfer Types in Inventory Controls > Stock Transfer Entry Types.
Valid transfer types are:
•
Regular Order - Type 01: Moves inventory items from one warehouse to another.
•
Non-Inventory - Type 02: Moves non-inventory items (e.g., office supplies, packing materials) from one warehouse to another. The products are not set up in the system and are not tracked as inventory.
•
Warranty Inventory - Type 03: Moves warranty tag inventory from one warehouse to another. The items must exist in the system as warranty inventory. Refer to Transferring Warranty Inventory.
•
Crib Replenishment - Type 04: If the receiving warehouse is a crib warehouse, the transfer type defaults to 04. Replenishment is handled as a standard transfer for product/bin locations. Nightly processing generates a recommended transfer to restock the crib. Transfer Entry requires you to enter a Destination Bin. Refer to the Product Whse Crib Information Screen for information on setting up a crib warehouse.
•
Ship Line Item Complete - Type 05: The transfer will not ship a line item if the line is not completely available to ship.

Product Maintenance Audit Report by Product
The Product Maintenance Audit Report by Product displays product changes, sorted by product number. You can choose to list changes made in any of the Product Maintenance screens.
After the report runs, a dialog box appears to allow you to purge the product information so that it does not appear the next time you run the report.
Recording Product Changes
Product Maint Audit? in File Update Parameters must be set to Y to record changes in Product Maintenance for this report,
Menu Path: Inventory > Reports > Miscellaneous Reports > Product Maintenance Audit Report > Product Maintenance Audit Report by Product
Selection Criteria
Use the following criteria to limit the information to print on the report:
•
Product. The product for which data was changed. Leave blank to view changes to all products.
•
Entry Date Range. The beginning and ending dates when the data was changed.
•
File Type. The type of data that was changed. Choose from the following or leave blank to view all changes:
-
P - View changes in Product Maintenance.
-
W - View changes in Product Warehouse Maintenance.
-
V - View changes in Product Vendor Maintenance.
•
Sort by Date. The chronological sort sequence for the report:
-
Y - Sort the changes by date, then by warehouse.
-
N - Sort the changes by warehouse (default).
Column Descriptions
Column Heading
Description
Product
The product number changed.
Whse
The warehouse that contains the product that was changed, if the change was made in Product Warehouse Maintenance.
Vend#
The vendor number, if the product/warehouse/vendor data was changed.
Type
The type of product record changed:
•
P - The product record was changed.
•
W - The product/warehouse record was changed.
•
V - The product/warehouse/vendor record was changed.
Ent-Date
The date the change was entered.
Ent-Time
The time the change was entered.
User
The user ID number of the person who changed the data.
Description
The description (name) for the user ID.
Description
The description of the field that was changed.
Before Value
The value of the field before the change.
After Value
The value of the field after the change.

Core Bank Issues Entry screen
The Core Bank Issues Entry screen identifies a customer’s available core bank items to apply to pending cores or to write off pending cores for a customer.
Refer to Entering Core Bank Issues for more information.
Menu Path: Inventory > Processes > Core Bank Issues Entry
Field and Function Key Descriptions
Field/Function Key
Description
Customer
The customer with pending cores. If the customer has pending core records but no core bank items, you can only write off pending cores.
Ret.Type
The type of pending core records to display:
•
A - All pending cores for the customer.
•
E - Only pending cores with a remaining quantity.
Pending
Order#
The order number the exchange product was originally sold on. Also the pending core number.
Shipped
The number of core products shipped on the original order.
Applied
The number of cores applied against the original order.
Remain
The number of cores outstanding from the customer that still need to be returned.
F2-Save
Clears the screen.
F5-Review
Opens the Core Order Info screen to allow you to view and apply pending cores.

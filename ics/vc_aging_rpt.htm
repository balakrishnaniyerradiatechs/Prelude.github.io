Vendor Claims Aging Report
The Vendor Claims Aging report displays claims updated in Vendor Claim Update Processing (refer to Updating Vendor Claims) but not yet reconciled with a vendor credit or replaced with another product (refer to Entering Warranty Replacements). You can use this report to follow up with the vendor on old claims that still need a credit or a replacement.
Menu Path: Inventory > Reports > Vendor Claim Reports > Vendor Claim Aging Report
Selection Criteria
Use the following criteria to limit the information to print on the report:
•
Division. The highest possible grouping in your company (e.g., Eastern Division, Western Division).
•
Region. A subset of Division - if Division is used (e.g., Southeast Region, Northwestern Region).
•
Warehouse. The warehouse where the claims were entered.
•
Department. A subset of warehouse. Typically used to separate expenses within a warehouse (e.g., accounting, sales, warehouse).
•
Cost Center. A subset of Department - if Department is used. Typically used to separate expenses within a warehouse or department (e.g., shipping, receiving, outside sales, inside sales).
•
Vendor. The vendor to include on the report or leave blank to include all vendors.
•
Summary or Detail. The type of aging information to include in the report. Choose from the following:
-
S - The summary of each claim for each vendor.
-
D - The detail product information of each claim for each vendor.
•
Sort By (W)arehouse / (V)endor. The report sort sequence. Choose from the following:
-
W - Sort by warehouse.
-
V - Sort by vendor.
Column Descriptions
Column Heading
Description
Date/Clm
Line 1: The date the vendor claim was entered.
Line 2: The vendor claim release number (e.g., the claim number with -1, -2).
Vend#
The vendor number.
Tag#/
Doc#
Line 1: The warranty tag number if the claim was generated for a warranty tag.
Line 2: The document number associated with the warranty tag. The prefix indicates how the tag was created:
•
S - The tag was created when a credit was issued to the customer for the returned product (refer to Creating Warranty Tags in Credit Memo Entry).
•
A - The tag was created when the product was adjusted into warranty inventory (refer to Creating Warranty Tags in Inventory Adjustment Entry).
•
No prefix - The tag was created when the product was rejected in receiving (refer to Creating Warranty Tags in Stock Receipts Entry).
Product# / Description
Line 1: The product number.
Line 2: The description of the product.
Line 3: The second line of description of the product, if applicable.
Quantity
The quantity of the product on the vendor claim that was scrapped or returned to the vendor.
Current
The total value of vendor claims updated in Vendor Claims Update Processing less than 30 days ago.
31-60
The total value of vendor claims updated in Vendor Claims Update Processing 31 to 60 days ago.
61-90
The total value of vendor claims updated in Vendor Claims Update Processing 61 to 90 days ago.
91-120
The total value of vendor claims updated in Vendor Claims Update Processing 91 to 120 days ago.
121&Over
The total value of vendor claims updated in Vendor Claims Update Processing over 120 days ago.
Tot Value
The total value of vendor claims updated in Vendor Claims Update Processing that are waiting for a credit or replacement.

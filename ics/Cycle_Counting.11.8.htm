Cost Tolerance Maintenance
Cost tolerances determine if a product requires a recount after the first level cycle count. You can define tolerance levels by cost range, bin value, or unit cost. These tolerances override the Phys Cnt Tolerance % field in Warehouse Maintenance.
To set cost tolerance limits for cycle counts:
1
Select Inventory > Physical Inventory > Cycle Counting Procedure > Cost/Tolerance Maintenance to open the Cost/Tolerance Maintenance screen.
2
For Cycle Count Type, indicate how variances are calculated:
•
C (or null) - Variances are calculated on the unit cost. For example, if a product with a cost of 59.95 is off by 10% or more, it is placed into the second level count queue.
•
B - Variances are calculated on the total bin value. For example, if a bin contains 50 pieces of an item that costs $1, 5 pieces of an item that costs $10, or 1 piece of an item that costs $50, a variance above 10% places the product in the second level count queue. This method is useful if you stock large quantities of low-priced products.
3
For From Cost, enter the lowest unit cost or bin value for the tolerance range.
4
For To Cost, enter the highest unit cost or bin value for the tolerance range.
5
For Tolerance%, enter the tolerance percentage allowed for the cost range. If the count is off by this percentage or more, then a second count is required. If you ever want products to go into the second level queue, enter 100.
6
Press F2-Save to save the cost tolerance information.

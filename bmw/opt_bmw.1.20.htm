Automatically Generating Work Orders for Scheduled Maintenance
Nightly Processing automatically generates a temporary work order for equipment records that are scheduled for maintenance. The system tracks the last maintenance date and determines if it is time for the next scheduled maintenance based on a date, the number of days past the last maintenance, or the number of hours/wear past the last maintenance hours/wear. Temporary work orders are only generated for products whose Equipment Flag in Product Maintenance is set to Y and that are set up as a bill-of-material.
Note: This feature does not work with Daily Processing.
Scheduled Maintenance Code Maintenance (/SMCM)
Path: Rental > Equipment Subsystem > Maintenance > Scheduled Maintenance Code Maintenance
This program determines when to schedule maintenance for a piece of equipment. It defines each maintenance routine performed on equipment and the days and hours between each scheduled maintenance entry.
•
Maintenance Code: Enter the code for a scheduled maintenance routine. For example: OC- oil change, TC- tire change, TU- tune up, etc.
•
Description: Enter the description for the maintenance code being created.
•
Days: Enter the number of days between each maintenance period. For example: in the above screen, an oil change is performed every 90 days.
•
Hours: Enter the amount of use between maintenance periods. This could be hours of use, mileage, or wear measurements such as on a drill bit (.001). Hours are defined in the Measurements Required field in Product Maintenance. For example (using the above screen): if the products Measurements Required field is set to H (hours), an oil change will be performed every 3000 hours. If the products Measurements Required field is set to O (odometer), an oil change will be performed every 3000 miles.
Equipment Maintenance (/EM > Action Bar > Parts List)
Path: Rental > Equipment Subsystem > Maintenance > Equipment Maintenance > *Action > Parts List
Each scheduled maintenance entry for a piece of equipment is defined and tracked in this program. The Parts List Entry is used to set up the parts needed for each maintenance routine performed on a piece of equipment.
•
Product#/Description: Enter the valid product numbers needed for each scheduled maintenance routine performed on the piece of equipment.
•
Maint Code: Defines which scheduled maintenance routine the corresponding product is used for. For example: in the above screen, all parts used in an oil change are identified with the code of OC1. These codes are set up in Scheduled Maintenance Code Maintenance.
•
Quantity: Enter the quantity of each product to use for this scheduled maintenance.
•
F5 – Maint Required: This option in the Parts List screen is used to view or change the dates for a piece of equipment’s next scheduled for maintenance. An example of the screen is shown below.
•
Maint Code: Identifies each maintenance routine scheduled for the piece of equipment. These codes represent the products entered on the previous screen and cannot be added, changed, or deleted in this screen.
•
Last Date: Defines the last date the corresponding maintenance routine was performed on the piece of equipment. This field is updated when a work order for the scheduled maintenance routine is completed in Work Order Completions.
•
Next Date: Defines the next date the corresponding maintenance routine should be performed on the piece of equipment. This field is updated when a work order for the scheduled maintenance routine is completed in Work Order Completions. The system calculates the next scheduled maintenance date based on the Days defined in Scheduled Maintenance Code Maintenance for the corresponding maintenance code. For example: if the last maintenance date is 11/05/03 and the number of days between maintenance periods is 30, the next scheduled maintenance date will be 12/05/03.
If setting up a piece of equipment for the first time, use this field to enter the date each scheduled maintenance entry should first be performed. If both fields are left blank the system schedules maintenance the next time Nightly Processing is run.
•
Last Hours: This field defines the measurement (hours, mileage, wear) of the piece of equipment the last time the corresponding maintenance routine was performed. This field is updated when you enter the current measurement in Work Order Completions.
If setting up a piece of equipment for the first time, enter the date each scheduled maintenance entry should first be performed in the Next Date field. If both fields are left blank the system schedules maintenance the next time Nightly Processing is run.
Nightly Processing (/NP)
Path: Purchasing > Shortcut: /NP
Temporary work orders for equipment scheduled for maintenance are created in Nightly Processing. The system only creates a temporary work order for one code at a time. In other words, if a piece of equipment needs two codes to be scheduled for maintenance, Nightly Processing only picks up one to create a temporary work order for. The second code is picked to create a temporary work order after the work order for the first code was completed and processed.
The system runs through a series of checks to determine if a temporary work order should be created for scheduled maintenance on a piece of equipment. It first checks the Next Date field in Equipment Maintenance for the piece of equipment. If that date is the current system date or earlier, then a temporary work order is built for the scheduled maintenance.
If the Next Date has not occurred, the system compares the current system date with the Last Date field in Equipment Maintenance. If the difference between them is greater than or equal to the value stored in the Days field in Scheduled Maintenance Code Maintenance for the maintenance code then a temporary work order is built for the scheduled maintenance. The logic being that the Next Date field may be manually changed but the scheduled maintenance routine should be performed on the piece of equipment every so many days.
If neither of the above conditions are met, the system compares the Current Meter Hours field in Equipment Maintenance with the Last Hours field in the Parts Lists Maintenance Dates screen. If the difference between them is greater than or equal to the value stored in the Hours field in Scheduled Maintenance Code Maintenance for the maintenance code, then a temporary work order is built for the scheduled maintenance.
If one of the above conditions is met, Nightly Processing generates a temporary expense (type 05) work order, which prints on the Recommended Maintenance Work Order Report produced by Nightly Processing.
Work Order Entry (/WOE)
Path: BMW/Work Order > Work Order Entry
Temporary work orders created by Nightly Processing for scheduled maintenance are converted in Work Order Entry using the F5-Convert Work Order function available at the Bill Number prompt.
•
Temporary Work Order Number: Enter the temporary work order number created by Nightly Processing to be converted to an expense work order.
•
Product Number: The system automatically pulls in the product number and description for the selected temporary work order.
•
Correct Work Order Number?: Prompts you to verify that the temporary work order number entered is the correct one. Set to Y to convert the temporary work order to a type 05 expense work order.
Once the conversion process is initiated, the system pulls in the components from the equipment parts list for the scheduled maintenance being performed. For example: if the work order is for an oil change, then all the products in the parts list that have an OC1 maintenance code are pulled into the work order
Since the system creates a type ‘05 expense work order, you are prompted for the Expense Reason Code and the Maintenance Code. The Expense Reason Code field allows you to define an override expense account to post the work order to. The Maintenance Code field displays the scheduled maintenance code the work order was created for.
Expense work orders post to a defined expense account instead of to inventory. The expense account is defined by the expense reason code, if one is given, or by the Expense Work Order header in G/L Table Maintenance. The Expense Work Order account defined in the G/L table is used if an expense reason code is not specified or if the expense reason code does not contain an override account number.
Note: An expense work order for scheduled maintenance can be manually created as well. Create a type 05 work order for the desired piece of equipment. Enter the maintenance code for the scheduled maintenance being performed and the serial number for the piece of equipment it is being performed on. The system automatically pulls in the products from the parts list for the maintenance code entered and updates the maintenance dates when the work order is completed.

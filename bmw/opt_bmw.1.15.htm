Automatically Generating Transfers From Work Order Entry
Prelude provides the ability to automatically generate transfers from in Work Order Entry for line items with a supply code of T for Transfer or R for Emergency Transfer. This can be defined by user to determine who can generate transfers.
Inventory Parameters (/OPTIONS.ICS.MNT)
Path: Systems > System Parameter > Maintenance > Option Parameters > Inventory Parameters > F5-Add Info
•
Auto Transfer Generation: Defines how automatic transfer generation from in Work Order Entry works in your system. Note: This flag works with both Sales Order Entry and Work Order Entry.
-
N - To not allow automatic generation of transfers from in Sales Order Entry or Work Order Entry.
-
Y - To display a dialog box in Sales Order Entry and in Work Order Entry to ask if a transfer should automatically be generated. This allows you to determine if a transfer should be generated at that time or not.
-
W - To automatically prompt for transfer information. The only difference between this option and the Y option is that this option forces you to generate the transfer.
User Identification Maintenance (/USER.ID.MNT)
Path: Systems > System Parameter > Maintenance > User Identification Maintenance > F5-Add Info
•
Enter T/R from W/O: Determines if this user can generate stock transfers from in Work Order Entry.
-
Y - The user is allowed to generate stock transfers from in Work Order Entry.
-
N - The user is not allowed to generate stock transfers from in Work Order Entry.
Work Order Entry (/WOE)
Path: BMW/Work Order > Work Order Entry
If the Auto Transfer Generation flag is set to Y or W in Inventory Parameters and if your Enter T/R from W/O flag is set to Y in User Identification Maintenance, then you may generate a transfer from in Work Order Entry for line items with a T (transfer) or R (emergency transfer) supply code. This process allows you to transfer in products needed to fill the work order from another warehouse.
•
Sc: Supply code. Enter a T or R for each item that needs a transfer to fill the work order
Note: An automated product load function does not generate a transfer request. For example, when a bill-of-material is set up with components, those components are immediately pulled into a work order for that bill-of-material. Even though one of the line items may have a T supply code, the transfer generation process is not activated unless you walk through the line item and enter the transfer information. Each line item with a T or R supply code must be walked through to activate the transfer generation request. Another example is when a bill-of-material is exploded on the work order.
When a line item detail screen with a T or R supply code is exited, the system checks that your company allows auto transfer generation and that your user id allows you to generate such transfers. If both of these criteria are met, you are asked if you wish to generate a transfer for the product (below). Note: this prompt is not displayed if the Auto Transfer Generation flag is set to W. Instead, you are immediately taken into the Auto Transfer Generation screen.
•
No: To not enter transfer information for the current line item at this time. Nightly Processing picks these items up and recommends a transfer.
•
Yes: To enter the transfer information for the current line item. The Auto Transfer Generation screen is allows you to define the warehouse to transfer the product from.
•
Shipping Warehouse: Enter the warehouse to transfer the product from. This warns you if the transfer line item will be backordered at the ‘From location.
When you exit the work order, the system checks for line items with a T or R supply code that contain transfer information. If any exist, you are prompted to proceed with transfer update (below). This prompt is not displayed if the Auto Transfer Generation flag is set to W. Instead, you are immediately taken into the Generate Stock Transfers screen.
•
No: To not generate transfers at this time. Nightly Processing will pick these items up and recommend a transfer.
•
Yes: To generate the transfers for the work order. The Generate Stock Transfers screen displays the accumulated transfer information set up for the current work order for all From Whse To Whse supply code combinations.
•
From Whse: Displays the warehouse the transfer material will be sent from.
•
To Whse: Displays the warehouse the transfer material will be sent to.
•
Type: Displays the supply code or the type of transfer. A separate transfer is generated for an R supply code than a T supply code even if they are both being transferred from the same warehouse. This is due to the R (emergency or ship immediate) transfer.
•
Ln#: Displays the number of line items that will be linked to the transfer.
•
T/R#: Displays the transfer number after the transfer is created.
•
Printed: Indicates whether or not the transfer is set to print.
•
F2-Generate All: Generates a transfer for all From Whse/To Whse/ Type combinations. For example, in the screen above, this function key generates two transfers: one for warehouse 20 for two line items and one for warehouse 30 for one line item.
•
F5-Edit: Allows you to make changes to the current transfer. This function key cannot be selected if the transfer has not been generated.
•
F6-Generate: Generates a transfer for the current From Whse/To Whse/Type combination. For example, in the screen above, if the cursor were on the second line then a transfer is only generated for warehouse 30.
•
F8-Review : Allows you to view the current newly created transfer in inquiry mode. No changes may be made.
•
F9-Continue: Exits the Generate Stock Transfers screen after all transfers were generated. This function key can also be used to exit the screen without generating the transfers. In which case, Nightly Processing will pick these items up and recommend a transfer.
When transfers are generated by using either F2-Generate All or F6-Generate, the system checks to see if there is an existing transfer on the system for the From/To combination and if its an emergency transfer. Then it checks for the same Ship-Via. If it finds an existing transfer that meets this criteria that has not been printed yet it puts the line items on that transfer. If it doesn’t, it puts the line items on a new transfer.
After the transfer is created and linked to the work order, the following dialog box displays allowing you to either set the transfer up for printing or to suspend it.
This prompt is displayed for each transfer created. For example, using the previous Generate Stock Transfers screen, this prompt would display twice: once for From 20 To 10 and once for From 30 To 10.
Once the stock transfer was shipped and received, the inventory from the stock transfer updates the work order making it available for pick ticketing.

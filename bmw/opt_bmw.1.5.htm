Perpetual Work Order
A perpetual work order is similar to a type 05 Expense work order except that it does not close until you close it, it cannot be linked to a sales order, and the bill number cannot be a rental product. This type work order can be used to accumulate expenses and then post those expenses when the work order is run through Work Order Completions. Any items on the work order that were issued are posted and removed from the work order but the work order itself remains open until you tell it to close. This allows you to continue to accumulate expenses on the same work order.
This type work order can be used to track expenses for odd jobs, repairs in shop, normal wear and tear maintenance, building repairs. For example, if a company has hundreds of chain hoists that need to be replaced every so often, a perpetual work order is used to expense the products used for maintenance. Monthly, the perpetual work order goes through Work Order Completions to post the expense items issued, but the work order remains open for the next months expenses. Since this type work order is similar to an expense work order, work order expense reason codes must be set up. These codes not only define the reason for the expense but also allow you to define separate G/L expense accounts for parts, labor, and sublet (work done outside your company).
General Ledger Table (/GLTABLE.MNT)
Path: General Ledger > Maintenance > General Ledger Table
If perpetual work orders are to be used, an Expense Work Order General Ledger Table Heading must be set up. Work Order Completion posts to this account instead of to the Inventory account, but can be overridden in the work order expense reason code.
•
Expense Work Order: Debits are posted to this account when a perpetual type work order is processed in Work Order Completions. It is used for in-house expenses.
Work Order Expense Reason Code Maintenance (/WERM)
Path: BMW/Work Order > Maintenance > Work Order Expense > Reason Code Maintenance
Expense reason codes can be created and maintained for expense and perpetual work orders. A reason code may be entered for a perpetual work order to define why it is being expensed as well as the accounts to use when posting the work order in Work Order Completions. You may define a separate account for parts, labor, and sublet (outplant) for each reason code.
•
W/O Expense Reason #: Enter the two digit expense reason code to create or edit.
•
Description: Enter the description for the reason code.
•
Add-on Capital: Determines if the Add-on Capital bucket for the equipment id number is updated for maintenance and labor expenses for a type 05 expense work order.
•
G/L Table Headings:
-
Parts Expense: This account is used to post any line item on the work order that is a non-service product, has a service type of M for Material, or is not defined as a labor or sublet expense.
-
Labor Expense: This account is used to post any line item on the work order that has a service or burden type of L for Labor.
-
Sublet Expense: This account is used to post any line item on the work order that has a service type of S for Sublet or a burden type of O for Outplant.
If any of these headers do not have an account defined, the system posts to the Expense Work Order account defined in G/L Table Maintenance. For example (using the accounts defined in the above reason code), if a line item on a work order falls into the parts expense category, the posting for that line item is to the Expense Work Order account defined in the G/L table. This is due to the Parts Expense account being blank in the reason code. If a line item on the work order falls into the labor expense category, the posting for that line item will be to the Labor Expense account defined in the reason code.
Work Order Entry (/WOE)
Path: BMW/Work Order > Work Order Entry
In the work order header screen, set the Order Type field to 06 Perpetual Work Order. A perpetual work order is only allowed if the work order is not linked and if the bill number is not a rental product. The following box prompts for additional expense information.
•
Expense Reason Code: Enter the code to define why the work order is expensed. This assigns the expense accounts to post to in Work Order Completions. If left blank, Work Order Completions posts to the Expense Work Order account defined in the general ledger table.
•
Maintenance Code: Leave this field blank for Type 06 perpetual work orders.
The system automatically sets the Quantity Ordered field in the header screen to 1 and it cannot be changed. If a work order is changed from type 05 Expense to type 06 Perpetual, then the Quantity Ordered is set to 1 and the line item quantities are adjusted as well.
This type work order is different from other work order types in that when the header screen is exited, you are not prompted for a serial number for a serialized bill number since we are not concerned with tracking expenses against a piece of equipment. You will also notice that if the bill number has components set up in Bill of Material Maintenance, those components are not pulled into the work order so that you start out with an empty work order.
Enter the components to be expensed on the work order. Print and issue the work order as usual. Since this work order is perpetual, multiple entries and issues may be performed over the life of the work order. Burdens and other functions of the work order system are the same.
Work Order Issues Entry (/WOIE)
Path: BMW/Work Order > Work Order Processing > Work Order Issues Entry
This process works the same way as for a regular work order. On-hand inventory is relieved for the components. A credit is posted to the Inventory account and a debit is posted to the Inventory Work In Process account for the extended cost of all products issued. Work Order Completions (/WOC)
Path: BMW/Work Order > Work Order Completions
Though a perpetual work order works the same way as a regular work order on the front end, Work Order Completions processes a perpetual type work order differently than a regular work order.
•
A regular work order performs the following when completed:
-
Increases on-hand inventory for the bill of material product.
-
Credits the Inventory Work In Process account.
-
Debits the Inventory account for the extended cost of the work order.
•
An expense type work order when completed perform the following debits/credits:
-
Does not increase on-hand inventory for the bill of material product.
-
Credits the Inventory Work In Process account for the extended cost of the work order.
-
Debits one or more Expense account.

In Work Order Completions, the system posts all items that were issued on a perpetual work order. The system then removes the items from the work order. If a line item was partially issued, then the line item is adjusted to the quantity remaining. If a line item was fully issued, then the line item is removed from the work order. If the work order is completely issued (meaning that there are no lines left open on the work order) then you are prompted to close the order (below).
•
Yes: The work order is closed.
•
No: The work order remains open with no component lines. (default).
If the work order contains items that have not been issued, it remains open and you are not prompted to close it. This allows you to issue the remaining expenses or to add additional expenses.

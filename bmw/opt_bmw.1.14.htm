Bill of Material (BOM)
A bill of material is the combination of component items from stock, that are put together to create a new item to be used in the Prelude Work Order system. Once a bill of material was created, it can be modified and the components can be replaced if necessary.
This document will cover the following topics regarding a bill of material:
•
Fixed /Variable Bill of Material
•
Bill of Material Maintenance
•
Bill of Material Component Replacement
•
Bill of Material Log Inquiry
In Bill of Material maintenance, each bill of material will be specified as fixed or variable. This determines how components, quantities and costs will be updated in the system.
Fixed Bill of Material
In a fixed bill of material the components and quantities do not change. When a fixed bill of material is entered on a work order, the work order is printed and completed, eliminating the issue process. The completion process will make all the necessary updates, including relieving component inventory and increasing finished goods inventory. There will be no work in process for these types of bill of material items. In addition, the finished good of a fixed bill of material will always update the standard cost for the bill of material item.
If a component or quantity needs to be changed, it can be changed in work order entry. In this case, work order completion will make the appropriate updates based on the work order components and quantities, but will consistently update the finished good at the bill of material cost, no matter what changes are made.
Fixed bill of material items are only to be used by companies who fully understand how they operate all the way through the work order process. If using the fixed bill of material items, please discuss the process with your installer, and/or make sure you completely understand how everything is updated.
Variable Bill of Material
A variable bill of material is the preferred way to use the Prelude work order system. A work order created for a variable bill of material does not assume that the quantities and/or components will be consistent. Items and quantities can be added, deleted and/or changed. Backorders can exist and will be handled accordingly. Releases will be printed for components available to pick and will show what is actually being used to create the finished good. The components stay in work in process until the work order is completed, at which time the components are taken out of work in process and the finished good on-hand inventory is increased.
By using variable types, the user is able to track exact inventory through work in process. Actual components and quantities are issued and the completion process only updates what was issued. The cost of these finished goods will be updated as actual or standard as specified in Bill of Material Option Parameters. Actual will update the cost based on actual usage and standard will update based on standard cost of the bill of material.
Creating and Maintaining a Bill of Material
Bill of Material Maintenance (/BOMM)
Path: BMW/Work Order -> Maintenance -> Bill of Material Maintenance
To create a bill of material (BOM), the product number must exist as a valid product in your product file. All the information entered in bill of material Maintenance will default into each work order created for the bill of material. All information can be changed in Work Order Entry except the fixed/variable flag, which can only be changed in Bill of Material maintenance.
Once a bill of material was created, changes can be made using this same program. Components that exist on multiple Bills of Materials can be changed all at once using the BOM Component Replacement Program. Any changes or updates made can be viewed in the Bill of Material Log Inquiry program
•
Bill Number: Enter the product number to be used to create a bill of material. This product number must already exist as a valid product in your product file. To copy a BOM number, there are several options to choose from using the function keys.
•
F5-Dup BOM: This will duplicate an existing bill of material to create a new one.
•
F6-Dup W/O History: Will create a bill of material from a closed work order record.
•
F7-Dup Open W/O: Creates a bill of material from an open work order record.
Once the bill of material number is created, the Bill of Material Header screen appears.
•
Assembly Days: Enter the number of days it takes to make this bill of material item.
•
Bill Type: Enter the billing type:
-
R - For regular (default)
-
F - If a formula is to be used. Use the F5 key to create a formula for billing.
•
Creation Date: This will default to the current date.
•
Revision Number: This field can be used for a revision number for when changes are made to the BOM.
•
Revision Date: This field can be used for a revision date to track revisions of the BOM.
•
Drawing: Enter a drawing number if applicable.
•
Rollup Pricing:
-
Y - If the prices of the components should be rolled up to determine the price of the parent.
-
N - If the prices should not be rolled up.
•
Sales Analysis:
-
P - If sales analysis should be updated at the parent level
-
C - If sales analysis should be updated at the component level
-
D - To distribute the actual selling price of the parent to each component, based a percentage of total cost.
•
Fixed/Variable BOM: See explanation at the beginning of this document on Fixed/Variable Bills of Material.
-
F - Fixed Bill of Material. This type will not go through the work order issue process and will update as a standard bill of material. There will be a cost associated with a fixed bill of material based on the components. The work order completion process will perform all the quantity updates, including relieving component inventory and increasing finished goods inventory, in one step. Work Order Completion will update the finished good at the BOM cost, instead of actual cost.
-
V - Variable Bill of Material. This type will go through the work order issue process and it will be updated with actual usage and cost or at standard cost determined by the bill of material option parameter flag.
•
Warehousing AlPath: This field works with the W/O Allocation % field in Product Maintenance. (Additional Options - Warehousing screen)
-
O - Will over allocate the components using the percentage set up for each component in Product Maintenance. It will take the pick quantity, calculate the over percentage, and round up to the nearest selling package quantity for each component.
-
N or null - Will not use the over percentage calculation. This should be used if the bin available is less than the over percentage calculated.
•
Assembly Instructions: Enter any instructions that default to work orders created for this bill of material. These print on the work order pick ticket if specified in user-forms.
•
F2 - Save: Saves the information input into the header screen and allows for the entry of the component items for the bill of material.
•
Component / Description: Enter each component part number that will make up the parent item. The description will default from the product file. (If no components are entered on this screen, they can be entered when the work order is created. This is only recommended if the components are not consistent.)
•
Asm: If the component is a kit parent or another BOM, a ‘* will appear in this column. Enter through the line item to explode the components onto the bill of material.
•
U/M: Unit of measure will default from the product file.
•
Qty - Asm: Enter the quantity it takes of each item to assemble the bill of material.
•
F2 - Save: Saves the bill of material component listing.
Bill of Material Component Replacement (/BCRE)
Path: BMW/Work Order -> Processes -> Bill of Material Component Replacement
A product that exists as a component on one or more bills of material can be replaced with another product and will be reflected on all bills of materials with this component.
•
Bill of Material Product Line: If applicable, enter the bill of material Product Line.
•
Component Number To Replace: Enter the old component number that is being replaced.
•
Replacement Component Number: Enter the new number that will replace the existing number on all associated Bills of Materials.
•
F2 - Process: Completes the transaction.
Bill of Material Log Inquiry (/BMLI)
Path: BMW/Work Order -> Inquiries -> Bill of Material Log Inquiry
This inquiry screen displays any updates that have occurred to a specified bill of material in Bill of Material Maintenance. to determine which fields will be tracked with the CHG update code, the fields must already be established in System parameters for BOM Change Log Setup and BOM Line Change Log Setup. See the Log Inquiry Setup document on the System Options Menu.
Version 18 Note: When a bill of material is updated from the BOM Component Replacement Entry process, the system updates this inquiry with the ADD code for the component added and the DEL code for the component removed. In previous versions, the system only updated these codes if changes were manually made in Bill of Material Maintenance.
•
Code: These codes explain what updates were made to the bill of material:
-
ADD - Line item added
-
ASG - Assignment or creation of BOM
-
CHG - Header or line item changed
-
DEL - BOM deleted
-
DUP - Duplicated through Bill of Material Maintenance
-
ENT - Entered an existing BOM
-
LDE - Line item deleted
•
Reference Number: If a product was changed or added, this field references the product. If a bill of material or work order was duplicated, the BOM number is referenced.
•
F5 - Review: This is used to review a change or update. If a bill of material was deleted, this function is not accessible. Put the cursor on any of the line items to bring up the following detail screen:
•
F5 - Hdr Screen: The detail from the bill of material header can be displayed.
•
F7 - Notes: Any notes regarding changes can be viewed using this function key.

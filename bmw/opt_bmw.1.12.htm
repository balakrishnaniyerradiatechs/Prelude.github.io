Mass Entry of Labor for Work Orders
A technician can enter and allocate a base number of hours across multiple work orders without having to enter a Miscellaneous Entry for each work order. The system updates each work order listed with the burden code, number of hours worked, rate (cost) and price if the work order is linked.
Rates may be defined for the Type of work done for linked and non-linked work orders giving you the ability to cost linked work orders at a different amount. For example, overtime may have a different rate than regular time, or labor for a linked work order may have a different rate than regular work orders. You also can to add the burden code to the work order as one line item or as individual line items for each entry.
Burden Maintenance (/BUM)
Menu Path BMW/Work Order > Maintenance > Burden Maintenance
Burden codes and labor rates account for labor for work orders. Some of the fields on this screen pertain to work order timesheet entries and do not affect service orders.
•
Burden Code: Enter the burden code to be created or maintained. This code is used in Work Order Timesheet Entry to define which rate to allocate to a work order.
•
Description: Enter the description of the burden code to be displayed for reference.
•
Burden Type: Enter the burden type of the burden code.
-
L - For labor.
-
A - For accounts payable.
-
O - For outplant processing.
•
Rate: Enter the amount to be charged if this burden code is used in timesheet entry to update a work order.
•
Unit of Measure: Enter the units in which the burden code is measured (e.g., hours).
•
Setup Charge Flag: Determines if there is a one-time setup fee for this burden code.
•
Include/Parent Cost: Determines if the burden value should be included in the finished product cost. This flag setting pertains to outplant processing.
•
Accumulate Burden Entries: Determines if the burden entries from a Work Order Timesheet Entry are added to the work order as one line for all burden types or individually for each burden type (e.g., a work order is entered more than once in a Work Order Timesheet Entry for different Types).
-
N - A separate line item is created for each entry.
-
Y - All entries for a burden code are accumulated on a single line item.
•
F5-Burden Rates: Burden rates can be established for each burden code. This screen allows you to set up Type codes along with a rate and a price. The rate is the cost of the labor. The price is the hourly charge to the customer and is only used for linked work orders.
•
Type: This code in Work Order Timesheet Entry to define which rate to use in a W/O:
-
RT - Regular Time.
-
OT - Overtime.
-
FT - Field Time.
-
FOT - Field Overtime.
-
PT - Premium Time.
-
RTL - Regular Time Linked.
-
OTL - Overtime Linked.
•
Rate: Defines the rate for the corresponding type. This is the cost on the work order for the burden item.
•
Price: Defines the price for the corresponding type for a work order linked to a sales order (i.e. customer repairs). This price tracks to the work order for the burden item if the work order is linked to a sales order.
•
Product Number: Enter the product number to be used for posting to a service order when this burden code type is used in Service Timecard Entry.
NOTE: If you allocate time by mechanic or technician, you can set up each person as a burden code to track hours and billing rates. If the mechanics or technicians bill at the same rate, you can use only one labor burden code for all labor costs. Or you can set up burden codes per department or group of personnel. For example, welders may have a different rate than the technicians who wire panels.
Work Order Timesheet Entry (/MTSE)
Path: BMW/Work Order > Processes > Work Order Timesheet Entry
This program provides for mass entry of labor allowing a base number of hours to be allocated across multiple work orders. The rate and price for each work order is determined by the type code entered. This option cannot be used for burden codes without types and rates as explained in the section on Burden Maintenance.
•
Burden Number: Defines the burden code (mechanic/technician) to be added to the work orders. Each code (mechanic/technician) must be set up in Burden Code Maintenance with defined types and rates.
•
Tech/User: Enter the technician for these timesheet hours.
•
Date: Defines the date the labor was done. This sets the posting period for the miscellaneous entry created behind the scenes.
•
Total Hours: Defines the number of hours the mechanic worked on work orders on the specified day. This amount allocates to the individual work orders.
•
WO# : Defines each work order the mechanic worked on that day. These are the work orders that will be updated with the burden code, hours, rate, and price (if the work order is linked). F5-Comments can be added before entering the work order number. These comments are added as Line Item notes to the work order.
•
Hours: Defines the number of hours the mechanic worked on the corresponding work order on the specified date. The sum of this column must balance to the value in the Total Hours field before you can complete and exit the transaction.
•
Burden: Enter the burden code in Burden Maintenance to be used for updating labor hours for the document number entered.
•
Type: Defines the type of work done and to locate the rate (and price if linked) that will be charged to the work order. The rate (and price if linked) is located in Burden Maintenance for the type entered.
•
Rate: Displays the rate found in Burden Maintenance for the burden/type combination. This rate defines the cost that carries over to the work order for the burden item.
•
Extension: Displays the rate amount extended by the value in the Hours field.
•
Price: Displays the price found in Burden Maintenance for the burden/type combination. This price is only displayed if the work order is linked. This is the price that carries over to the work order for the burden item.
•
Extension: Displays the price extended by the value in the Hours field. This extension is only displayed if the work order is linked.
When this screen is processed, the system performs a payroll (P) type Miscellaneous Entry behind the scenes. Each work order listed is updated with the burden code, the AsmQty and ActQty fields, the number of hours, the cost with the rate for the corresponding work order/burden, and the price. If the work order is linked, the price for the corresponding work order/burden is updated also.
If Accumulate Burden Entries is set to Y, all burden types are accumulated and placed on the work order as one line item. If this flag is set to N, each burden type on the work order as individual line items.
Work Order Log Inquiry (/WLI)
Path: BMW/Work Order > Inquiries > Work Order Log Inquiry
The MIS code is updated when the Work Order Timesheet Entry is processed.
The Reference # is the burden code and miscellaneous entry transaction number. F5-Review displays the Miscellaneous Entry Inquiry screen.
Work Order Inquiry & Work Order History Inquiry (/WOI & /WOHI)
Path: BMW/Work Order > Inquiries > Work Order Inquiry
Path: BMW/Work Order > Inquiries > Work Order History Inquiry
•
F5 - Review -> F10-Examine: This screen is updated with the entry id, user id, and number of hours for each burden added to the work order.

Expense Type Work Order
An expense type work order is mainly used for in-house maintenance on rental or fleet equipment where the entire transaction is expensed. When this type work order is completed, the system relieves on-hand inventory for the components used but the on-hand for the unit itself does not increase. Though an expense work order can be used for other things, this document mainly covers routine maintenance on a piece of equipment.
There are several maintenance steps required before using expense work orders. When using an expense work order for equipment maintenance, the equipment part number should be set up as a bill-of-material. This allows you to track expenses for each piece of equipment. Work order expense reason codes must also be set up. These codes not only define the reason for the expense but also allow you to define separate G/L expense accounts for parts, labor, and sublet work performed outside your company.
General Ledger
A General Ledger Table Heading must be set up if expense work orders are used. Work Order Completions post to this account instead of to the Inventory account. This account is overridden if an account is defined in the work order expense reason code.
•
Expense Work Order Debits are posted to this account when an Expense type work order is processed in Work Order Completions. It is used for in-house expenses.
Work Order Expense Reason Code Maintenance (/WERM)
Path: BMW/Work Order > Maintenance > Work Order Expense Reason Code Maintenance
This maintenance program creates and maintains expense reason codes used for expense type work orders. A reason code may be entered for an expense work order to define why it was expensed and the accounts to post to in Work Order Completions. You may define a separate account for parts, labor, and sublet (outplant) for each reason code.
•
W/O Expense Reason #: Enter the two digit expense reason code to create or edit.
•
Description: Enter the description for the reason code.
•
Add-on Capital: Determines if the Add-on Capital bucket for the equipment id number is updated for maintenance and labor expenses for a type 05 expense work order.
-
Y - The Add-On Capital bucket on the Equipment Financial Info screen is updated with the dollars for Maintenance Expense - Parts (work order component costs) and Maintenance Expense - Labor (labor cost - miscellaneous entries).
-
N - The Add-On Capital bucket on the Equipment Financial Info screen is NOT updated. Maintenance Expense - Parts (work order component costs) and Maintenance Expense - Labor (labor cost - miscellaneous entries) are updated.
•
G/L Table Headings:
-
Parts Expense: This account is posts non-service line items on the work order that with a service type of M for Material or is not a labor or sublet expense.
-
Labor Expense: This account is used to post any line item on the work order that has a service or burden type of L for Labor.
-
Sublet Expense: This account is used to post any line item on the work order that has a service type of S for Sublet or a burden type of O for Outplant.
If any of these headers do not have an account, the the Expense Work Order account is posted to. For example (using the accounts defined in the above reason code), if a line item on a work order is in the parts expense category, that item posts to the Expense Work Order account in the G/L table. This is due to the Parts Expense account being blank in the reason code. If a line item on the work order falls into the labor expense category, the posting for that line item will be to the Labor Expense account defined in the reason code.
Equipment Maintenance (/EM > Action Bar > Parts List)
Path: Rental > Equipment Subsystem > Maintenance > Equipment Maintenance > * Action > Parts List
The Parts List Entry option sets up the parts needed for each maintenance routine performed on a piece of equipment. This is only one method used to enter components on a type 05 expense work order. The other methods are described in the Work Order Entry section.
•
Product#/Description: Enter the valid product numbers needed for each maintenance routine performed on the piece of equipment. For example, products were entered for an oil change and a tire change.
•
Maint Code: Defines which scheduled maintenance routine the corresponding product is used for. For example: in the above screen, all parts used in an oil change are identified with the code of OC1. This code is set up and maintained in Scheduled Maintenance Code Maintenance (/SMCM).
•
Quantity: Defines the quantity of the product to use when the piece of equipment is repaired.
•
F5-Maint Required: Used to view or change the dates used to determine when a piece of equipment is next scheduled for maintenance. This option should only be used if you are letting the system keep up with your scheduled maintenance. For more information on scheduled maintenance see document Automatic Work Order Generation for Equipment Scheduled Maintenance.

Work Order Entry (/WOE)
Path: BMW/Work Order > Work Order Entry
The Work Order Entry program is used to enter an expense work order for routine maintenance. There are three methods available to load products needed for routine maintenance. The first method is the most efficient, but the other methods can be used.
Pull Components By Using A Maintenance Code
•
Bill Number: For expense orders, this should be the serial number or equipment Id number of the item being serviced. The F3 lookup can be used to search by equipment id number or serial number. This means that the product number must also be set up as a bill of material. This allows the expenses to be tracked against a piece of equipment but only if the product tracks by serial number. For example, product number 9801 is set up for a Bobcat front-end loader. A bill number 9801 should also be set up in Bill Of Material Maintenance.
In the work order header screen, set Order Type to 05 Expense. This Expense Information screen appears.
•
Expense Reason Code: Enter the code to define why the work order is expensed. This assigns the expense accounts to post to in Work Order Completions. If left blank, Work Order Completions posts to the Expense Work Order account in the G/L table.
•
Maintenance Code: Used to automatically load components from the equipment parts list. Multiple maintenance codes may be entered. The system automatically pulls in the products from the equipment parts list for the maintenance code(s) entered. You may leave this field blank and define the components at the work order line item level.
When the work order header screen is exited and if the bill number tracks by serial number, you are prompted for the serial number of the piece of equipment being serviced. This is how the system knows which equipment record to update with the maintenance/expense information.
When the serial number is entered, the rest of the information - if available - appears. In the work order line item screen, components are displayed if a maintenance code was entered in the work order header and if the piece of equipment has components in its parts list for the defined maintenance code. Print and issue the work order as usual. Burdens and other functions of the work order system are the same. Though the previously described way to automatically pull in the products needed for a routine maintenance is the most efficient, there are two other ways to achieve the same result:
Pull Components From The Equipment Parts List
When using this method the system displays all the components defined in the equipment parts list allowing you to select the ones you want to add to the work order.
•
Available: the quantity available in the warehouse for the product.
•
Asm: Displays ‘* if the corresponding product is set up as a bill of material.
•
Asm Quantity: the assembly quantity from the parts list for the quantity.
•
Add To Order: Indicates which products you wish to add to the work order. Change to Y to add it to the order.
•
Quantity: Enter the quantity of the product add to the order. A quantity should be entered even if it is the Asm Quantity. If a quantity is not entered, the products are pulled into the work order without assembly quantities.
•
Exp: Indicates if a bill of material product (indicated by ‘* in Asm) should be exploded on the work order.
-
Y - All the components in the bill of material will be placed on the work order.
-
N - Only the bill of material is placed on the work order. For example, using the bill of material 9801-OC, if this flag were set to N then 9801-OC is placed on the work order. If this flag were set to Y, then the components making up 9801-OC are placed on the work order.
•
F5-Accept All: Pull the components in the equipment parts list into the work order line item screen.
•
F2-Save: All components whose Add To Order flag is set to Y are pulled into the work order line item screen.
Create a Product Number and Bom
The components on the bill of material would be the products used to perform the routine. For example, 9801-OC is created in Product Maintenance and in Bill Of Material Maintenance for an oil change on a 9801. The bill of material is created with the components of the oil change (oil, oil filter, and labor). The bill of material is then added to the equipment parts list or simply entered on a work order.
On the work order line item, enter the bill of material that identifies the service being performed (i.e. 9801-OC). The system asks if the components of the bill of material should be exploded onto the work order. If you respond Yes, all the components in the bill of material will be placed on the work order. If you respond No, then only the bill of material will be placed on the work order. For example, using the bill of material 9801-OC, if you were to respond No then 9801-OC would be placed on the work order. If you were to respond Yes, then the components making up 9801-OC are placed on the work order.

Work Order Completions (/WOC)
Path: BMW/Work Order > Work Order Completions
Work Order Completions for expense work orders are different than a regular work order.
•
A regular work order when completed: Increases on-hand inventory for the bill of material product; credits the Inventory Work In Process account and debits the Inventory account for the extended cost of work order.
•
An expense type work order when completed: Does not increase on-hand inventory for the bill of material product; credits the Inventory Work In Process account for the extended cost of the work order and debits one or more Expense accounts.
In this example, the oil and filter posted to the Expense Work Order account defined in the G/L table and the labor posted to the account defined in the reason code. Work Order Completions also updates the equipment record if the bill of material is the actual equipment product and if it is serialized. These can be viewed in Equipment Maintenance or Equipment Inquiry under the F8-Financial function key.
The F5-Review Maint Exp function key drills down into an Equipment Maint Expense Inquiry screen where all work orders that were expensed against the current piece of equipment are displayed. From this inquiry screen, you may drill down into Work Order History Inquiry to view the actual work order.
Note: Only Add-On Capital, Maintenance Expense-Parts, Maintenance Expense-Labor, and Sublet Maint Expense-Labor are updated from work orders. The Add-On Capital field is ONLY updated from work orders if the work order expense reason code has the Add-On Capital flag set to Y in Work Order Expense Reason Code Maintenance. All of these are updated from Manual Invoice Entry when asset tracking is turned on in the account and the asset tracking charge type is set to one of these types.
V18+: You can create a fixed asset record for an '05' type expense work order. It is a sub asset of the original fixed asset. Also, the Master Asset Entry and Sub Asset Disp screens in Asset Maintenance show the Sub Asset record for the Master Asset or the Master Asset for the Sub Asset record.

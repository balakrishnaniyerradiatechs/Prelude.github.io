Work Order Rebates
Work order rebates can be calculated and accumulated by Prelude. These are used when a proof of sale is required from your vendor to receive an agreed upon dollar amount back for items sold to a customer. Price Matrix Maintenance defines the components for a bill of material to calculate the rebate amount to the customer. A rebate report is available for printing rebate information for a customer or invoice date range.
Note: Only components on work orders that are linked to sales orders are included in calculating rebate amounts when the work order is completed. The rebate amount of each component item is rolled up to the bill of material item on the sales order. If a rebate was established for the bill of material item in Price Matrix Maintenance, the rebate amount is overridden with the amount rolled up from the component items.
Product Maintenance (/PM > F7-Pricing)
Path: Inventory > Maintenance > Product Maintenance > F7-Pricing
Enter a valid selling price for the component item to calculate a rebate on the price.
Price Matrix Maintenance (/PMM)
Path: Order Processing > Maintenance > Pricing Structures > Price Matrix Maintenance
Price Matrix Maintenance can be used to setup rebate information for all the available price matrices. There are ways to enter rebates using either a net cost or a rebate amount. The way the rebate is calculated depends on the value entered in the Mtd field.
•
Rebate Mtd:
-
C - Calculates the rebate amount (average cost – net rebate cost). Enter the Net Rebate Cost in Rebate.
-
R - Calculates the rebate amount (replacement cost – net rebate cost). Enter the Net Rebate Cost in Rebate.
-
D - For a chain discount. This prompts you to enter a chain discount percentage.
-
T - For quantity table discount. Enter the table number in the Table field.
-
* - For a percentage of a price/cost field. Enter a column in the Basis field to calculate the percent.
-
$ - For a rebate cost. This can be used even with non-product matrices but a warning appears. This allows you to use a net rebate price even when no product is identified.
•
Table: This field is only accessible if the previous field is set to T for quantity table discount. Enter the table to be used with this price matrix for rebates.
•
Rebate: Enter the rebate cost for this matrix if using C, R or $ in the Mtd field.
•
Basis: If the Rebate Mtd field is set to *, enter the appropriate code to indicate which column price or cost to use as a basis. Options are:
-
0 - Suggested Retail.
-
1 - Price 1.
-
2 - Price 2.
-
3 - Price 3.
-
4 - Price 4
-
5 - Price 5
-
A - Average Cost
-
B - Product Qty Break. Uses the products quantity break table before applying the discount defined in this table. This should only be used for percentage discounts.
-
C - Customer Default Column. Uses the customer's default price level before applying the discount defined in this table.
-
O - Cost for Sales.
-
R - Replacement Cost.
-
S - Standard Cost.
•
Pct: Enter the discount percentage for this price matrix. This percentage is applied to the value in the Basis field to calculate the rebate amount for each line item. The rebate amount is a percentage over a price column if a price was specified in the basis field or a percentage over a cost if a cost was specified in the basis field.
Sales Order Entry (/SOE)
Path: Order Processing > Sales Order Entry
When a sales order is entered that has a price matrix established for the components and the sales order is linked to a work order, a rebate amount is calculated for the sales order when the work order is completed.
The F10-Examine screen for the line item displays the rebate amount. The rebate amount is a total of all rebates for the individual components.
•
Example: If using the price matrix established with R , The replacement cost for the component of the bill of material item was $7.50. The rebate cost was $5.00. The rebate amount for the component is $2.50 (7.50 – 5.00). This amount is rolled up to the bill of material item on the sales order.
Note: In this example one component is being used for the bill of material item. If multiple components were established for the bill of material item, the rebate amount for each component item would be rolled up to the bill of material item on the sales order.
Process Invoices (‘B Update) (/BUPDATE.PROCESS)
Path: Order Processing > Processes Process Invoices (‘B Update)
This process updates the rebate amount of each sales order to the REBATE file. The rebate cost is subtracted from the commission cost for the salesman.
Rebate Report (/REBATE.RPT)
Path: Order Processing > Reports > Miscellaneous Reports Rebate Report
Displays the rebates for the division number, selling warehouse, vendor number, customer number, and invoice date range entered. The Process Invoices (‘B Update) procedure must be completed for the rebates to be included on the report. This report pulls the rebate information from the REBATE file. For bill of material items, there is a note referencing that the rebate amount is a result of a roll up from a work order.

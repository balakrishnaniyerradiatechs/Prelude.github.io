Work Orders and Bills of Material for Special Services
When an inventory product needs to be sent to an outside processor to have some special service performed, this modified product may become either an inventory item or a component of another Bill of Material.
The following example may fit the requirements for most companies that need to issue raw materials from inventory and ship to another source for a finishing process. The steps are as follows:
1
Set up regular product numbers for the Finished Product and the Finishing Service to be performed.
2
Add product notes to the Finishing Service product to carry to the purchase order and fully explain procedures to be done.
3
Establish a bill-of-material for the finished product. Include the raw material(s) and the processing service(s) as separate lines on the bill. Add instructions to appropriately direct the shipment of the raw material with the purchase order for the finishing process.
4
Create a work order for the finished product and release the raw material on a work order pick ticket. Link a purchase order for the finishing service.
5
Print both the work order ticket and purchase order pick ticket.
6
When the raw material is pulled to ship to the finisher, issue the raw material. This moves it from regular inventory to Work In Process. Ship with the purchase order to the finisher.
7
Upon receipt of the finished product, receive the Finishing Service, print the work order pick ticket, issue the Finishing Service, and finally complete the work order to move the finished product to inventory.
BMW/Work Order Parameters (/OPTIONS.BMW.MNT)
Path: Systems > System Parameter > Maintenance > Option Parameters > BMW/Work Order Parameters
The following flags must be set as noted below to use this process.
•
Auto Build-To-Order WO: Determines if the work order is automatically created from in Sales Order Entry. Set to N for this process so that you will be prompted to create a work order.
•
Auto Release W/O: Determines if the work order is automatically released when processing in Work Order Entry if no backorders exist for the components. Set to N for this process so that you are prompted to release the work order.
•
Released By User Id: Determines if the system should prompt for a user id when the work order is released for production. This flag should be set to Y for this process to prompt for a user id.
Example:
•
The Finished Product: Left Handed Pink Widget.
•
Bill Of Materials: 2 Yellow Gizmos imprinted with the word PINK.
•
Outside Source: Prime Time Printers.
•
Non-stock item to be ordered to build the finished product: 1 Whatchamacallit
•
Product Numbers established:
-
LH-PINK-WIDGET: Status Code S (or N) - Repl Path M
-
GIZMO-YEL: Status Code S - Repl Path P.
-
PRINT-PINK: Status Code N - Repl Path P.
-
WHATCHA: Status Code N - Repl Path P.
The following steps are used when adding special services to a work order.
8
A sales order is entered for BIGCUST for 100 LH-PINK-WIDGET since the current stock level is zero and because it is a non-stock item.
9
A work order is generated for the production of the 100 widgets. This creates a work order pick ticket to pull the 200 Yellow Gizmos, along with a purchase order to send them off for imprinting
10
A purchase order is also issued to procure the required 100 WHATCHAs.
11
The 200 Gizmos are received, and the work order pick ticket is produced to issue the imprinting (PRINT-PINK). The 200 newly imprinted Gizmos are completed through the Work Order Completion process.
12
When the 100 WHATCHAs are received, the final work order pick ticket is printed, the Left Handed Pink Widgets are assembled, and completed.
13
The sales order pick ticket is printed, picked and confirmed.
14
The product is shipped and invoiced to BIGCUST and the customer pays for the merchandise.

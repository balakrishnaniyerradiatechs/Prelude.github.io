Work Order Entry Vendor Claim Expense Type Line Items
When credit is expected from a vendor for products that were repaired in-house, these products can be expensed on a work order. This is accomplished by defining the lines on the work order that are to be expensed (claimed for credit) and then creating a vendor claim with a miscellaneous charge line item for the expected credit amount. The vendor claim may be automatically generated after the work order is completed through Work Order Completions.
Note: This feature works best when the returned unit is repaired and returned to stock instead of linked to a sales order for the customer. If a work order contains expense line items and it is linked to a sales order, the price for the expense line items does not roll up into the price of the bill-of-material on the sales order. However, the cost does roll up.
Work Order Expense Reason Code Maintenance (/WERM)
Path: BMW/Work Order > Maintenance > Work Order Expense Reason Code Maintenance
This creates and maintains work order expense reason codes. Expensing a work order line item requires that a reason code be entered for the transaction. This reason code defines why the item is expensed as well as the account used when posting the expense line item in Work Order Completions. The defined accounts are used for the miscellaneous charge line item when the vendor claim is automatically generated. You may define a separate account for parts, labor, and sublet (outplant) for each reason code.
•
W/O Expense Reason #: Enter the two digit expense reason code to be created or maintained.
•
Description: Enter the description for the reason code.
•
Add-on Capital: Determines if the Add-on Capital bucket for the equipment id number is updated for maintenance and labor expenses for a type 05 expense work order.
-
Y - The Add-On Capital bucket on the Equipment Financial Info screen is updated with the dollars for Maintenance Expense - Parts (work order component costs) and Maintenance Expense - Labor (labor cost - miscellaneous entries).
-
N - The Add-On Capital bucket on the Equipment Financial Info screen is NOT updated. The dollars for Maintenance Expense - Parts (work order component costs) and Maintenance Expense - Labor (labor cost - miscellaneous entries) are updated in their respective buckets.
•
G/L Table Headings:
-
Parts Expense: This account is used to post any line item on the work order that is a non-service product, has a service type of M for Material, or is not defined as a labor or sublet expense.
-
Labor Expense: This account is used to post any line item on the work order that has a service or burden type of L for Labor.
-
Sublet Expense: This account is used to post any line item on the work order that has a service type of S for Sublet or a burden type of O for Outplant.
If any of these headers do not have an account defined, the system posts to the Expense Work Order account defined in G/L Table Maintenance. For example, if a line item on a work order falls into the parts expense category and a parts expense account is not established, the posting for that line item will be to the Expense Work Order account defined in the G/L table. This is due to the Parts Expense account being blank in the reason code. If a line item on the work order falls into the labor expense category and a labor expense account is established, the posting for that line item will be to the Labor Expense account defined in the reason code.
Note: Vendor Claim Entry requires that the account number used for a miscellaneous charge line item be set up in Vendor Claim Misc Charges G/L Numbers Maintenance (System Parameter -> Maintenance-> Inventory Controls menu). Failure to do so will result in the inability to maintain the vendor claim miscellaneous charge line items.
Work Order Entry (/WOE)
Path: BMW/Work Order > Work Order Entry
Enter the work order as usual. Open the line item detail screen for each product that needs a vendor credit.
•
Type: Set to V (vendor claim expense). This tells the system to track this item as a vendor claim expense allowing you to create a vendor claim once the work order was completed and to post the item accordingly. This brings up the following prompt:
•
Expense Reason Code: Defines why the item is expensed as well as the account used when posting the expense line item in Work Order Completions. The account defined for the reason code in Work Order Expense Reason Code Maintenance is also used for the vendor claim miscellaneous charge line item when the vendor claim is automatically generated.
•
Vendor Number: Enter the vendor to claim the credit from. This field defaults to the products primary vendor but may be overridden.
Print and Issue the work order as usual. The posting from Work Order Issue credits the Inventory account and debits the Inventory Work In Process account for the extended cost of all products issued. The expense accounts are not affected at this time.
Work Order Completions (/WOC)
Path: BMW/Work Order > Work Order Processing > Work Order Completions
Complete the work order as usual. Cost is accumulated for each line item on the work order with the Type field set to V. Expense line items are excluded from re-averaging of cost.
Work Order Completion posts a debit to the Inventory account for the extended cost of all line items with Type not set to V, a credit to the Inventory Work In Process account for the full extended cost of the work order (reversing the posting from Work Order Issues), and a credit to the expense account defined by the reason code for the extended cost of line items with Type set to V. If the work order contains more than one expense reason code and different accounts are referenced, the system accumulates cost by reason code and posts accordingly. Once the work order is complete you can to continue with the vendor claim update.
•
No: Brings back the work order completion screen. You must then manually create the vendor claim with a miscellaneous charge line item and reference the correct account. Be aware that at this time there are no reports to inform you of expensed items that are not on a vendor claim.
•
Yes: Displays the Generate Vendor Claim for Expense Items screen allowing you to generate the vendor claim(s) for the expensed items. This screen displays each vendor a claim needs to be created for, the warehouse it will be created in, and the number of lines that will be placed on the claim. The vendor claim number is displayed after the claim is created.
•
F2 - Generate All: Creates vendor claims for all vendors listed.
•
F6 - Generate: Only creates a vendor claim for the vendor the cursor is currently positioned on.
A regular (type 01) vendor claim is created for the vendor(s) indicated. A miscellaneous charge line item is created for each reason code. The amount is the total of all lines with that reason code/vendor combination.

Note: The account number in the vendor claim should not be changed. The vendor claim must contain the same expense account number as was posted to in Work Order Completions. If the account number is changed, Vendor Claim Update Processing posts a credit to an account different from the one debited in Work Order Completions thus creating an out-of-balance for both accounts involved.

Work Order Log Inquiry (/WLI)
Path: BMW/Work Order > Inquiries > Work Order Log Inquiry
The Work Order Log Inquiry allows you to review transactions that have occurred when creating or editing work orders. In addition, you have the capability of viewing other information such as Header and Line notes as well as detailed information such as Cost and Linkage in the Examine screen.
•
F5 - Review -> * Action -> Notes -> Header Notes: In V18+, the Header Notes screen was updated to display the Vendor Claim number once the work order was completed in Work Order Completions and the vendor claim is created for the vendor claim expense type line items.
•
5 - Review -> * Action -> Examine: In V18+, the Examine screen was updated to now display the Expense Reason Code once the work order was completed and the vendor claim is created for the expense type items entered on the work order.

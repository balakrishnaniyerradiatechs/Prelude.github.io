Work Order Overview
Work orders use a structured process to build bill of material (BOM) items or to create new items (finished goods) from components. This document covers the following basic stages used for work orders.
•
Work Order Entry: A work order is created for a BOM which pulls in the associated components and quantities.
•
Work Order Printing: The work order is printed and the components are pulled in the warehouse.
•
Work Order Issue Entry: The work order is issued in the system which relieves component inventory. (Fixed type bill of material items will not go through the issue process). The warehouse performs the assembly of the components or the work required to obtain the finished product.
•
Work Order Completion: The work order is completed in the system which increases inventory for the finished good (BOM), making it available for stocking, selling, or transferring.
The Work Order Log Inquiry program and associated codes are also explained in this document.
Work Order Entry (/WOE)
Path: BMW/Work Order > Work Order Entry
This program is used to enter or edit a work order to build a bill of material.
•
W/O Number: Enter over NEW or enter N to select a new order number.
•
Bill Number: Enter the BOM number to be assembled on this work order. This can include part of the description, equipment I.D. number, serial number or other search criteria determined by your company. If the BOM does not exist, you may be able to create it on the fly in work order entry if allowed.
•
W/O Status: Displays the work order status: ENTERED or RELEASED.
After you enter a BOM number, the default information from BOM maintenance appears on the header screen.
•
Order Type: Enter the work order type. Select 01 for regular work order.
•
Warehouse Number: Enter the warehouse number for this work order.
•
Quantity Ordered: Enter the quantity to be created from the components for the bill of material. This number is multiplied by the component assembly quantity to determine an open quantity.
•
Entry Date: Defaults to the date the work order is entered.
•
Required Date: This is the date the work order is required to be completed. This is calculated from the assembly days entered in BOM maintenance. It takes the entry date plus assembly days to calculate a required date. This can be changed here if needed.
•
Pick Ticket Date: The date the pick ticket should print based on BOM option parameters.
•
Written By: Defaults to the user id of the person entering the work order.
•
Buyer Number: If the BOMs replenishment path for this warehouse is M, the planner number appears.
•
Work Order Type: This is only to be used with repair type work orders. All others will be N for non-billable.
•
Assembly Days: Defaults from Bill of Material Maintenance, but can be changed if needed. This represents how many days it should take to assemble this work order.
•
Job Number: Enter a job number if needed.
•
Rollup Pricing: Defaults from BOM maintenance, but can be changed per work order. Determines whether the components prices should roll up to calculate the parent price.
•
Emergency Flag: Used with the warehousing module to prioritize picking.
•
Fixed/Var BOM: Displays whether the bill of material is Fixed or Variable.
•
Assembly Instructions: Pulls from BOM maintenance but can be added, changed or deleted.
•
Revision#: Displays the revision number setup for the BOM.
•
Rev. Date: Displays the revision date setup for the BOM item.
•
Drawing: Displays the drawing number setup for the BOM item.
•
F2-Save: Processes the header screen.
If backorders exist, a message displays to let you know that backorders exist and appropriate supply codes are defaulted. Supply codes are used to determine the replenishment path. The default is pulled from the product warehouse file. For example, if the component item is non-stock, the supply code defaults to a B for buyout, which means the item must be linked. If the item has a transfer replenishment path, the supply code defaults to a T. The supply code can be changed if needed based on your company parameters. The components and quantities are pulled into the work order from BOM maintenance. You can add/delete components, or change quantities on this screen.
•
Product# / Description: The product number and description from the product.
•
Asm: If the component is a kit parent or another BOM, a * appears in this column. If you enter through the line item, you can to explode the components into the bill of material.
•
Asm-Qty: Displays the assembly quantity of the component to assemble one BOM.
•
Qty-Opn: Displays the open quantity for this work order. This quantity is calculated as the assembly quantity multiplied by the order quantity from the header screen.
•
F2-Save: Processes the work order. If the Auto Release W/O flag is set to Y in BMW/Work Order Parameters, the following message does not appear.
•
No: To not release the work order for production. The Work Order Release Inquiry can be used to release and schedule work order production. The work order cannot be printed until it is released.
•
Yes: To release this work order for production and setup for printing. If continuous printing is established for work order tickets, the ticket prints immediately. If not, use the Work Order Printing Process to print tickets.
•
Return: To return back to the work order screen to make additional updates.
Note: To prompt for the partial release of work orders, see the Partial Release of Work Orders document.
To make changes to an existing bill of material:
•
Delete a component: Press the back slash key, (\), on the line item to delete and press Enter.
•
Add a component: Insert a line or add the component to the bottom of the components. Press F9, then B to get to the bottom quickly. /Keys will show your insert key, if unknown. When adding nonstock (*) items, if you want both the price and cost to be prompted for, see Nonstock Price/Cost in Work Order Entry.
•
Quantity change: Go to the item to be changed, press Enter to get to quantity prompt and change the assembly quantity. This is the quantity it takes to make one of the finished item (BOM). The system performs the multiplication to determine the open quantity.
•
Other: If you need to change anything in the detail box, enter N at the Accept prompt and you can change the pick quantity (if allowed), the back order quantity, supply code, price or the billable type (repair only).
Work Order Printing (/WOP)
Path: BMW/Work Order > Work Order Processing > Work Order Printing
If continuous printing is not set up for work orders, use this program print work order pick tickets by batch.
•
Warehouse Number: Enter the warehouse number.
•
Work Order Number: Enter the number to print or leave blank to print all work orders for this warehouse.
•
Printer Form Queue: Enter the printer number.
•
Print Work Order: Prints the work orders.
•
Mask Print Test: To print a mask or test of the work order to make sure the paper is aligned.
•
Abort Printing: To abort the printing process.
After printing is complete, a box displays with the last release number that printed.
If all work order tickets printed correctly, select Continue. Otherwise, you can choose to reprint one work order or all work orders in the batch if there was a printing problem such as a paper jam.
•
F5-W/O Pick Ticket Reprint: Allows the work order pick ticket to be reprinted.
Work Order Issues Entry (/WOIE)
Path: BMW/Work Order > Work Order Processing > Work Order Issues Entry
The work order issue process decreases the on-hand of the components and increases Work in Process.
•
W/O Release Number: Enter the work order release number. For example; 100263-3 represents the third release for work order number 100263. Additional releases include backorders or components added to a work order. If all components print and are picked on the first release, and no additional components are used on this work order, there will only be one release.
•
Bill Number: Displays the bill of material item entered on the work order release.
•
Process Date: Enter the current date, T for today. This updates the work order with an issued date.
This brings up the components screen to verify that the components that were printed are being issued. Changes can be made as necessary on this screen.
•
Total Cost: In V18+, a running total of the components issued for the work order. This does not include any labor costs updated from Miscellaneous Entry or Work Order Timesheet Entry, but is strictly used for the components on the work order.
Important Note: This field is not updated with a dollar amount for the first release (-1). The totals begin calculations on the second release (-2) for the first release (-1). The Total Issue Cost on the third release (-3) reflects the total cost for the first and second releases, and so on.
•
F2 - Save: When finished with the work order issue process.
At this time, the warehouse should be assembling the finished item. If there are left over components that were issued but not used, a negative issue must be created to move the components from Work in Process back to regular inventory. Also, if there are components that were used and not issued, an additional issue must be created to relieve the extra components from regular inventory. Both of these examples can be handled with a manual issue (explained below). The following message displays to post and update the work order.
•
Yes: Posts and updates the work order as issued and sets it up for completion.
•
No: To not post and not update the work order as issued. The work order can be viewed and updated later.
•
Manual Issue: In Work Order Issue Entry, enter the work order number (without a release number) in the release number field. You will be asked if this is a manual entry. Answer Yes and on the component screen enter the component to issue or un-issue. Enter a positive quantity for an additional issue entry, enter a negative quantity for a negative issue.
F2 when finished with the manual issue. After all components are correctly issued and the finished good is completed, you are ready to complete the work order in the system. Note: In V17+, you can re-open a work order linked to an invoiced sales order in Work Order Issues Entry. The G/L posting is updated to the cost of goods sold (COGS) account instead of indirect/linked inventory.
Work Order Completion (/WOC)
Path: BMW/Work Order > Work Order Completions
Work Order Completion completes the work order by removing the components from the Work in Process inventory and increasing the on-hand inventory for the finished good (BOM).
•
W/O Number: Enter the work order number to be completed.
•
Bill Number: The bill number from the work order defaults.
•
Completion Date: Defaults to the current date. This is the date the system updates the finished good as complete. Once the date is entered, the fiscal period is displayed to the right of the date.
•
Completed By: Enter your user id or initials.
•
Quantity Completed: Enter the quantity completed of the finished item.
•
F2-Update: Performs the work order completion.
•
F5–Review W/O: To review the work order.
•
F6 - Completion History: Displays all completed quantities and completion dates for the work order.
You can skip this step so that work orders are completed and closed while in the Work Order Issues Program. See the document, Allow Complete in Work Order Issues for more information.
Work Order Log Inquiry (/WLI)
Path: BMW/Work Order > Inquiries > Work Order Log Inquiry
This inquiry program displays any actions or changes performed on a work order in Work Order Entry.
•
Code: This is the action that was performed on this work order. The definition of the codes are as follows:
-
ADD - Line item added.
-
ALB - Allocation of backorder by age.
-
ASG - Work order number assigned.
-
ASS - Picker assigned.
-
BIN - Bin level allocation.
-
CHG - Header/line item changed.
-
CLS - Work order close.
-
CNV - Temporary work order conversion.
-
CPL - Work order completed.
-
DEL - Work order deleted.
-
ENT - Re-entered work order.
-
ESC - Escape from work order.
-
FUT - Future line item changed to pickable status.
-
ISS - Work order Issue.
-
LDE - Line item deleted.
-
MIS - Work order reopened in Miscellaneous Issues Entry.
-
OPN - Work reopened in miscellaneous entries.
-
OUT - Outplant received.
-
PCK - Pick ticket printed.
-
QCV - Work order quote conversion.
-
QTE - Work order number assigned.
-
RDE - Release denied.
-
REC - Linked item received.
-
REL - Released for production.
-
UNA - Pick Assigned.
•
Reference Number: Displays the product, release or picker number associated with the code.
•
F5 Review: Displays the detailed information for the highlighted action.

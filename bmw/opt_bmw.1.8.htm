Outplant Work Orders
The Outplant Work Order system is used when products are taken out of stock and sent to a vendor to have work performed such as painting, plating, packaging, machining, etc. The finished product is then returned to your warehouse to be sold. Outplant Work Orders can only be used if your company is using Warehousing.
Product Warehouse Maintenance (/PWM > F6)
Menu: Inventory > Maintenance > Product Maintenance > Product Warehouse Maintenance > F6 Purchasing Parameters
The Replenishment Method for the finished product must be set up in the Purchasing Parameters screen.
•
Replenishment Mtd: This must be set to O for Outplant.
•
Planner: Enter the buyer id of the person responsible for the production of this item.
•
Production Multiple: Enter the amount per production to be manufactured.
•
Production Minimum Months: Enter the minimum number of months usage to be manufactured at a time.
•
Production Group: If applicable, enter a production group to assign this item to.
Bill of Material Maintenance (/BOMM)
Menu: BMW/Work Order > Maintenance > Bill of Material Maintenance
A Bill of Material is normally created with the finished product as the parent. The products that are being sent out are the components. The burden items represent the amount the vendor charges to perform a process. If these are not standard they can be entered / changed in Work Order Entry instead of Bill of Material Maintenance.
Burden Maintenance (/BUM)
Menu: BMW/Work Order > Maintenance > Burden Maintenance
Burden items are created to reflect the outplant charges.
•
Burden Code: Enter the Outplant burden code to be created or maintained.
•
Description: Enter the description of the burden code to be displayed for reference.
•
Burden Type: Set to O for outplant.
•
Rate: Enter the amount to charge if this burden code is used in timesheet entry to update a work order.
•
Unit of Measure: Enter the units in which the burden code is measured (e.g., hours).
•
Setup charge Flag:
-
N - If it is a charge per piece
-
Y - If it is a flat charge for everything sent
•
Include/Parent Cost: Determines if the burden value should be included in the finished product cost for outplant processing.
•
Accumulate Burden Entries: Determines if the burden entries from a Work Order Timesheet Entry are updated to the work order as one line for all burden types or individually for each burden type. This flag covers the scenario where a work order is entered more than once in a Work Order Timesheet Entry for different Type codes.
-
N - A separate line item is created for each entry.
-
Y - All entries for a burden code are accumulated on a single line item.
•
F5-Burden Rates: Burden rates can be set for each burden code. This screen allows you to set up Type codes and a rate and a price. The rate is the cost of the labor. The price is the hourly charge to the customer and is only used for linked work orders.
Bin Location Maintenance (/BLM)
Path: Inventory > Maintenance > Bin Files > Bin Locations
Two bins must be created for each vendor using outplant processing.
For the first bin location, Bin Type is set to S Staging and Storage Type is set to O Outplant. This is where the product is staged after picking and before sending to the vendor. For the second bin location, Bin Type is set to W Work in Process and Storage Type is set to O Outplant. The product remains here while it is at the vendor site.
Work Order Entry (/WOE)
Path: Warehousing > Work Order Processing > Work Order Entry
In Work Order Entr or Work Order Completions, Order Type: 03 – Outplant enter a vendor: You can enter an outplant staging bin in Work Order Entry or Assignment.
The work order is allocated, assigned, and picked as normal to outplant staging bin. When the components are ready to be shipped – it could be by individual work order or several, the outplant shipping documents are printed by specifying the staging bin and work order number. It is possible to print three reports from this process. Each is prompted for separately so only those needed can be printed.
Shipping Documents (OUTPLANT.SHIP.RPT)
Menu: Warehousing > Work Order Processing > Outplant Processing > Shipping Documents
Several reports can be printed that pertain to outplant shipping. The 1st report is the Outplant Summary Shipment Report. This prints all components and quantities that are being shipped. It is then used to assemble and ship the products.
The second report is the Outplant Work Order Report. It includes all components, quantities, assembly instructions, and burden codes for a work order. This is sent to the vendor to describe what needs to be done. This report appears after the Outplant Summary Shipment Report.
The third report is the Outplant Production Group Report. It sorts all work orders by the production group in the parent product record. This report is prompted after the Outplant Work Order Report is printed.
Outplant Bin Movement (/OUTPLANT.MOVE)
Path: Warehousing > Work Order Processing > Outplant Processing > Outplant Bin Movement
This process identifies products sent to the vendor by specifying the staging bin and work order number. The components are then moved into the designated Work in Process bin.
Receiving Documents (/OUTPLANT.REC.RPT)
Path: Warehousing > Work Order Processing > Outplant Processing > Receiving Documents
When the completed product(s) are returned, a receiving document report may be printed to display information on work orders available for outplant receiving.
Outplant Receiving (/OUTPLANT.REC)
Path: Warehousing > Work Order Processing > Outplant Processing >Outplant Receiving
This process updates the finished product into inventory.
The work order is completed, the finished product goes into inventory while the individual components are deducted from inventory.
Vendor Invoice Reconciliation Entry (/VIRE)
Path: Accounts Payable > AP/PO Subsystem > Vendor Invoice Reconciliation > Invoice Reconciliation Entry
When a work order was completed and received, the finished product creates a receiver for the total of the burden items on the work order. Vendor Invoice Reconciliation Entry is then used to pay the vendor for services.

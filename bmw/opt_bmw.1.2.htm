Work Center Activity Maintenance
A work center can be created as a burden code in Burden Code Maintenance and then allocated to a work order. When the work order is entered in the work center, the date and time starts for the order, then stops when the work order is re-entered. You are prompted at both times with a message allowing you to select if you wish to 'start' or 'stop' this process. If you select 'No' to either, you cannot continue with the work order. This allows the technician hours from each work center to be allocated to the work order as a Miscellaneous Entry.
Start and stop times cannot be modified. The system updates each work order listed with the burden code (work center), the number of hours worked, the rate (cost) and the price if the work order is linked. Rates may be defined for the type of work done and/or for linked and non-linked work orders giving you the ability to cost linked work orders at a different amount. For example, overtime may have a different rate than regular time, or labor for a linked work order may have a different rate than regular work orders. You can to add the burden code to the work order as one line item or as individual line items for each entry.
Burden Maintenance (/BUM)
Path: BMW/Work Order > Maintenance > Burden Maintenance
Burden codes and labor rates can be established in Burden Maintenance to further customize the work centers.
•
Burden Code: Enter the burden code (work center) to be created or maintained. This field is user defined and allows unlimited types to be entered. This code is used in Work Center Activity Maintenance to define which rate to allocate to a work order.
•
Description: Enter the description of the burden code to be displayed for reference.
•
Burden Type: Enter the burden type of the burden code.
-
L - For labor.
-
A - For accounts payable.
-
O - For outplant processing.
•
Rate: Enter the amount to be charged if this burden code is used in timesheet entry.
•
Unit of Measure: Enter the units in which the burden code is measured (e.g., hours).
•
Setup Charge Flag: Determines if a one-time setup fee is charged for this code.
•
Include/Parent Cost: Determines if the burden value should be included in the finished product cost. This flag setting pertains to outplant processing.
•
Accumulate Burden Entries: Determines if the burden entries from a Work Order Timesheet Entry are updated to the work order as one line for all burden types or individually for each burden type. This flag covers the scenario where a work order is entered more than once in a Work Order Timesheet Entry for different Type codes. This flag determines if these two entries are added to the work order as one line item or as separate line items.
-
N - A separate line item is created for each entry.
-
Y - All entries for a burden code are accumulated on a single line item.
•
F5-Burden Rates: Burden rates can be established for each burden code. This screen allows you to set up Type codes along with a rate and a price. The rate is the cost of the labor. The price is the hourly charge to the customer and is only used for linked work orders.
•
Type: This code is used in Work Order Timesheet Entry to define which rate to allocate to a work order. The following are examples of type codes as displayed in the above screen:
-
RT - Regular Time.
-
OT - Overtime.
-
FT - Field Time.
-
FOT - Field Overtime.
-
PT - Premium Time.
-
RTL - Regular Time Linked.
-
OTL - Overtime Linked.
•
Rate: Defines the rate for the corresponding type. This is the cost that tracks to the work order for the burden.
•
Price: Defines the price for the corresponding type for a work order linked to a sales order (i.e. customer repairs). This price tracks to the work order for the burden item if the work order is linked to a sales order.
•
Product Number: Enter the product number to be used for posting to a service order when this burden code type is used in Service Timecard Entry.
NOTE: If allocating time by mechanic or technician, you may want to set up each person as a burden code to track hours and billing rates. If the mechanics or technicians bill at the same rate, only one labor burden code can be established to cover all labor costs. Or you can set up burden codes per department or group of personnel. For example, welders may have a different rate than the technicians responsible for wiring panels.
Work Center Activity Maintenance (/WCM)
Path: BMW/Work Order > Work Order Processing > Work Center Activity Maintenance
•
Work Order: Enter the work order number to be updated with work center activity. Once the work order is entered, the system checks if the work order is currently in process and if already in the work center. If the work order is NOT in process for a work center yet, the system prompts with the following message:
•
No: To not start the date and time for the work center for this work order.
•
Yes: To start the date and time for the work center for this work order.
If the work order is currently in process for a work center, the system prompts with the following message:
•
No: To not continue with entry of the work order for updating a work center.
•
Yes: To continue with entry of the work order for updating a work center. This brings up the following screen.
•
Work Center: Defines the work center burden code to add to the work order. Each burden code must be setup in Burden Code Maintenance with defined types and rates.
•
User Id: Enter the technician who performed the work center hours.
•
Start: Defines the start date and time the work order was updated into the work center.
•
Stop: Defines the date and time the work order was stopped and closed out of the work center. This date sets the posting period for the miscellaneous entry created behind the scenes.
•
E.T. (Hr. Min): Defines the estimated number of hours and minutes the work order was in the work center. The start and stop fields are used to calculate the estimated time. This time is multiplied by the work center (burden) rate and updated to the work order.
•
Work Order: Displays work orders processed in Work Center Activity Maintenance for the user in the center.
•
Item: Displays the bill of material item that is associated with the work order.
•
Description: Displays the bill of material item description associated with the item number on the work order.
When this screen is saved and a work order is stopped from the work center, the system performs a payroll (P) type Miscellaneous Entry behind the scenes. The system also updates the work order listed with the burden code, the AsmQty and ActQty fields with the number of hours, the cost with the rate for the corresponding work order/burden, and the price, if the work order is linked, with the price for the corresponding work order/burden.
Note: Multiple work centers are NOT allowed to be in process for a single work order. The work center in process must be stopped before the next work center for the work order can be started.
Work in Process Inquiry (/WPI)
Path: BMW/Work Order > Inquiries > Work in Process Inquiry
This inquiry displays work orders for a warehouse that are currently in process for a work center.
•
Warehouse Number: Enter the warehouse number to display work orders in process.
•
Sort: The sort order of the data in the inquiry screen. Defaults based on Default Parameters Maintenance.
•
Filters: You can filter by customer, product line, work order status to limit the records that appear.
•
W/O#: Displays the work order numbers that are in process for the warehouse.
•
Item/Description: Displays BOM number and description that are associated with each work order.
•
W/C: Displays the work center (burden code) entered in Work Center Activity Maintenance.
•
Start: Displays the start date and time updated at the time of entry in Work Center Activity Maintenance.
•
Stop: Displays the stop date and time updated at the time of entry in Work Center Activity Maintenance. If the work order is in process in the work center, the reference In-Process displays.
•
User: Displays the user id entered in Work Center Activity Maintenance.
Work Order Log Inquiry (/WLI)
Path: BMW/Work Order > Inquiries > Work Order Log Inquiry
MIS indicates that the work order stopped in Work Center Activity Maintenance for a Miscellaneous Entry.
The Reference # is the work center (burden code) and miscellaneous entry number. Press F5-Review to see the Miscellaneous Entry Inquiry screen.
Work Order Inquiry & Work Order History Inquiry (/WOI & /WOHI)
Path: BMW/Work Order > Inquiries > Work Order Inquiry
Path: BMW/Work Order > Inquiries Work Order History Inquiry
The F10-Examine screen is updated with the entry id, user id, and number of hours for each burden added to the work order.

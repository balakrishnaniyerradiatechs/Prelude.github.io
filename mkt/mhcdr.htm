Contact Diary Report
The Contact Diary Report lists closed marketing activities and tracked transactions (refer to Selecting Files to Update for more information on tracking transactions). You can view all activities and transactions for a customer, prospect, or contact.
Menu Path: Marketing > Contact Diary System > Contact Diary Report
Selection Criteria
Use the following criteria to limit the information on the report:
•
Prospect. The customer or prospect on the activity or transaction.
•
Mail To. The number of the mail-to address, if you use mail-to addresses.
•
Sort Option. The order for the activities and transactions on the report:
-
Contact.
-
Document Number.
-
Entry Date.
-
Product Number.
-
Serial Number.
-
User Id.
•
Contact Type. The reason for the contact (e.g., Sales, Credit).
•
Contact Method. How the customer or prospect was contacted:
-
EM - Email.
-
FAX - Fax.
-
ML - Mail
-
MT - Meeting.
-
TH - Telephone.
•
Contact#. The code for the contact on activities and transactions.
•
Diary Code. Identifies and groups activities. Refer to Identifying Diary Codes.
•
Entry Date. The date on which the activity or transaction was entered.
•
User Id. The user ID of the sales rep on the activity or transaction.
•
Product. The product number associated with the activity or transaction.
•
Serial. The serial number associated with the activity or transaction.
•
Document Type. The type of transaction:
-
S - Sales order.
-
W - Work order.
-
Q - Quote.
•
Document Number. The number associated with the Document Type.
Column Descriptions
Column Heading
Description
Prospect
The customer or prospect number on the activity or transaction.
Mail-To
The number of the mail-to address on the activity, if you are using mail-to addresses.
Description
The description of the prospect or customer.
Phone
The customer’s phone number.
Ent Date
The action date for the activity or transaction. For closed activities, the action date is the date on which the Activity occurred.
Ent Time
The action for the activity or transaction. For closed activities, the action date is the date on which the Activity occurred.
User Id
The user ID of the person who entered the activity or transaction.
Type
The reason for the contact (e.g., Sales, Credit).
Contact Method
How the customer or prospect was contacted:
•
EM - Email.
•
FAX - Fax.
•
ML - Mail.
•
MT - Meeting.
•
TH - Telephone.
Contact Name
The name of the customer or prospect contact on the activity or transaction.
Diary Code
The diary code that identifies the type of activity. Diary codes are used to group similar activities for reporting. Refer to Identifying Diary Codes.
Document
The document type and number of the activity or transaction. Document types are:
•
S - Sales order.
•
Q - Quote.
•
W - Work order.
•
M - Marketing activity.
•
D - Contact diary entry.
•
R - Sample request.
•
T - To do (scheduled activity).
Product
The product(s) associated with the activity or transaction.
Serial#
The serial number(s) for the products associated with the activity or transaction.

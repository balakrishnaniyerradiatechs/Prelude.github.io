Contact Maintenance Audit Report by User ID
The Contact Maintenance Audit Report by User ID lists each time data was changed for a contact, including who changed the data and what the data was changed from and to. To use this report, Contact Maint Audit in File Update Parameters must be Y. If your company does track this data, you can purge it after you run the report or keep it for future reference.
Menu Path: Marketing > Reports > Contact Maintenance Audit Reports > Contact Maintenance Audit Report By UserId
Selection Criteria
Use the following criteria to limit the information to print on the report:
•
User Identification. The user ID of the person to review.
•
Entry Date range. The date(s) on which the contact was changed.
Column Descriptions
Column Heading
Description
Customer/Contact/
Description
Line one - The number of the customer or prospect and the contact number.
Line two - The name of the contact.
Type
Indicates if the contact is for a customer or prospect:
•
C - The contact is for a customer.
•
P - The contact is for a prospect.
Ent-Date
The date on which the data was changed.
Ent-Time
The time at which the data was changed.
User
The ID of the user who changed the data.
Description
The name of the person who changed the data.
Data Changed
Description
The field description of the data that was changed.
Before Value
The data that was in the field before the user changed the data.
After Value
The data that was in the field after the user changed the data.

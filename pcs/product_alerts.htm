Product Alerts
The Product Alert work queue alerts the buyer when a product’s purchasing parameters should be reviewed and/or adjusted for the following reasons:
•
Min/Max Spike: When usage for a min/max product, with frozen purchasing parameters, exceeds the maximum usage quantity setting. This alert helps the buyer to adjust min/max settings for new products until a sufficient amount of history is available to accurately calculate product usage. The buyer must be set up as the purchasing buyer in Warehouse Manager Maintenance.
You can freeze a product’s purchasing parameters by entering Y in Frozen Controls in the Product Purchasing Parameters Maintenance screen. When you freeze controls for a product, the purchasing parameters calculated by end-of-month processing are ignored.
•
Safety Allowance: When the product is received and the available quantity falls above or below the safety allowance tolerances defined for the product’s class. Safety allowance tolerances are calculated by multiplying:
-
S/A High% x Safety Quantity
-
S/A Low% x Safety Quantity
Safety Quantity is set for each product in the Product Purchasing Parameters Maintenance screen.
S/A High% and S/A Low% are set by company for each product class in the Product Class Settings screen (Purchasing Controls > Purchasing Parameters > F6-Class Settings).
•
Early Warning: When a product’s usage exceeds a pre-defined percentage of its Forecast quantity by the day of the month defined in Product Alert Parameters. For example, if by the 7th of the month, actual usage for the product exceeds 75% of the Forecast quantity, it will appear in the buyer queue.
The Product Alert Parameters screen (Purchasing Controls > Purchasing Parameters > F7-Product Alerts) allows you to set a day of the month to evaluate a product’s usage and the percentage of the product’s Forecast that causes an Early Warning alert. If a product’s usage by that day exceeds the defined percent of its Forecast, an Early Warning product alert appears in the Buyer Queue.
A product’s Forecast is set up in Product Purchasing Parameters Maintenance.
•
Lost Sales: When a sale was lost. The lost sale reason code displays to identify why the sale was lost.
Update Product Alerts in Lost Sales Reason Code Maintenance (Order Processing Maintenance > Misc Code Files > Lost Sales Reason Codes) must be set to Y so that a lost sale with this reason code updates the Buyer Queue.
Reviewing and Adjusting Purchasing Parameters
Purchasing Buyer
You can assign a Purchasing Buyer in Warehouse Manager Maintenance (systems menu) to the buyer responsible for Product Alerts for a warehouse. If blank, the buyer assigned to each vendor or vendor line is notified of product alerts.
To review and adjust purchasing parameters for products on alert:
1
Select one of the following methods to access the Buyer Product Alert Inquiry screen:
•
Purchasing > Inquiries > Buyer Queue Inquiry. Place the cursor in front of Product Alerts and press F5-Review.
•
Purchasing > Inquiries > Buyer Queue Inquiries > Product Alerts. Enter the Buyer.
The items on product alert appear.
2
Highlight the item to review and select F5-Review to open the Product Purchasing Parameters Maintenance screen. Refer to the “Product Purchase Parameters Maintenance screen” in the Inventory documentation for detailed information on each field.
3
If necessary, you can adjust the Minimum, Maximum, Safety Allowance %, or Forecast values to more accurately match the actual usage of the item.
4
Press F2-Save to save any changes. You then have the option to remove the item from the work queue.
5
After reviewing and adjusting all the items on alert, press F2-Save to close the screen.
Field & Function Key Descriptions
Field / Function
Description
Product Number
The product number on alert whose Product Parameters should be reviewed.
Description
The description of the product number.
Whse
The number for the warehouse where the item’s Product Parameters should be reviewed.
Reason
The reason why the product is on alert:
•
Min/Max Spike - When usage for a min/max product, with frozen purchasing Parameters, exceeds the maximum usage quantity setting. This alert helps the buyer to adjust min/max settings for new products until a sufficient amount of history is available to accurately calculate product usage. The buyer must be set up as the purchasing buyer in Warehouse Manager Maintenance.
•
Safety Allowance - When the product is received and the available quantity falls above or below the Safety Allowance tolerances defined for the product. Safety tolerances are calculated by multiplying the S/A High% or the S/A Low% in the Product Class Settings screen (Purchasing Controls > Purchasing Parameters > F6-Class Settings) by the Safety Quantity set up for the product (Product Purchasing Parameters Maintenance).
This setting is not applicable for min/max products or linked products. Products with a safety allowance of zero are not included.
•
Early Warning - When a product’s usage exceeds a pre-determined percentage of it’s Forecast quantity by a pre-defined day of the month. The product’s Forecast is set up for a product in Product Purchasing Parameters Maintenance.
This alert is activated in Purchasing Controls > Purchasing Parameters > F7-Product Alerts. You can set the day of the month for the system to evaluate the usage and the percentage of the Forecast that the product’s usage must exceed to activate the alert.
•
Lost Sales - The lost sale reason code displays to identify why the sale was lost. Lost sales reason codes are created in Order Processing Maintenance > Misc Code Files > Lost Sales Reason Codes.
Alert Date
The date the item went on alert.
F2-Save
Closes the Buyer Product Alert Inquiry screen.
F5-Review
Opens the Product Purchasing Parameters Maintenance screen for the product to allow you to adjust the Minimum, Maximum, or Safety Allowance quantity for the item.
F6-II
Opens the Inventory Inquiry screen to allow you to view the inventory status of the item.
F7-TAI
Opens the Transaction Audit Inquiry screen for the item to allow you to view transactions involving the item.

P/O Vendor Exception Report
The Vendor Exception report displays products that were purchased from a vendor other than the product’s primary vendor. After the report runs, a dialog box appears to allow you to purge the product information so that it does not appear the next time you run the report.
This information is only tracked if the Primary Vendor P/O flag in File Update Parameters is set to Y.
Menu Path: Purchasing > Reports > P/O Reports > P/O Vendor Exception
Selection Criteria
Use the following criteria to limit the information to print on the report:
•
Warehouse. The warehouse for which the material was purchased.
•
Vendor. Vendor number to select for the report. The vendor that the products were bought from, not the primary vendor.
•
Buyer. Typically the buyer assigned to the vendor. The buyer number can be overridden in Purchase Order Entry.
Column Descriptions
Column Heading
Description
P/O #
Purchase order number.
P/O Date
Date the purchase order was entered.
Vendor
The number of the vendor where the product was purchased from.
Description
The description of the vendor where the product was purchased from.
Byr
Buyer assigned to this purchase order.
Whse
The warehouse assigned to receive the material.
Product
Line 1: Products on the purchase order.
Line 2: Description of the product.
Quantity
Total quantity of the product purchased on this purchase order.
Primary
Primary vendor number for this product (Product Warehouse Maintenance).
Description
Description of the product’s primary vendor.

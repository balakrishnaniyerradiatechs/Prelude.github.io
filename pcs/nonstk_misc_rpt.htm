Manual Purchasing Requirements Report
The Manual Purchasing Requirements report lists products that need to be purchased but were not placed on a recommended purchase order due to a discrepancy (e.g., no primary vendor, obsolete item).
Menu Path: Purchasing > Reports > Nightly Processing Reports > Manual Purchasing Requirements Report
Selection Criteria
Use the following criteria to limit the information to print on the report:
•
Central Warehouse. The warehouse that requires the items.
Column Descriptions
Column
Description
Product
Line 1: Product to order.
Line 2: The description of the product.
Whse
The warehouse that requires the product.
Document
The document number to which the product is linked. The prefix indicates the document type:
•
S - Sales order.
•
W - Work order.
•
T - Transfer.
Ln#
The line number of the item on the linked document.
Sc
The line item’s supply code:
•
B - Purchase order.
•
E - Emergency purchase order.
•
D - Direct ship purchase order.
•
R - Emergency stock transfer.
•
T - Stock transfer.
•
W - Work order.
Ref #
The number of the customer or warehouse that requires the item.
Description
The description of the customer or warehouse that requires the item.
Qty-B/O
The quantity backordered.
U/M
The unit of measure of the item as entered on the sales order.
Cost
The cost per unit of measure.
Ext. Cost
The extended dollar value of the backordered item.
Message
The reason the product was not recommended for purchase:
•
T/R Request Error. The transfer entry request has an error.
•
Manual Linkage. Product manually linked to a purchase order.
•
Buyout Override.
•
No Primary Vendor assigned (Product Vendor Maintenance).
•
Obsolete Product (Product Maintenance > Obsolete Flag =Y).
•
Work order. The requirement is for a work order.

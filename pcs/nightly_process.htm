Nightly Purchasing Process screen
The Nightly Purchasing Process screen allows you to schedule Nightly Processing. The defaults for this screen are set up in System Parameters > Purchasing Controls > Nightly Purchasing Setup.
Refer to Nightly Processing for more information.
Menu Path: Purchasing > Processes > Nightly Processing - Scheduled
Field and Function Key Descriptions
Field/Function Key
Description
Weekend Execution
Determines if you are running Nightly Processing on the last day of the business week, normally a Friday:
•
Y - It is the last day of the business week.
•
N - It is not the last day of the week.
Daytime Execution
Determines if Nightly Processing is run during the day. Nightly Processing rebuilds the current statistics file to determine product availability. All open documents are evaluated and the on-order and committed information is rebuilt:
•
Y - Nightly Processing is being run during the day. In this case, data is not cleared and rebuilt.
•
N - If Nightly Processing is not being run during the day. All files are updated.
Run Daily Business Update
Determines if you want to run the Daily Business Update, which updates all sales, bookings, inventory value, accounts receivable, and accounts payable information by warehouse.
•
Y - Nightly Processing updates Daily Business Statistics.
•
N - Nightly Processing does not run the Daily Business Statistics Build. You must run the Daily Business Statistics Build separately.
Warehouse Number
The warehouse numbers to include in this Nightly Processing run. The reports to run for each warehouse cannot be changed. If you want to change the report to run, refer to Running Nightly Processing.

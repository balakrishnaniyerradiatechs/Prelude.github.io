Purchase Order Requisition Header screen
The Purchase Requisition Header screen appears after you select a vendor in Purchase Requisition Entry. You can review or edit information regarding the vendor. Refer to Entering a Purchase Requisition for more information.
Menu Path: Purchasing > Processes > Purchase Requisition Entry > F5-Hdr Scr
Field and Function Key Descriptions
Field/Function Key
Description
P/O Type
The type of purchase requisition:
•
01 - Regular.
•
02 - Direct ship.
•
03 - Will call.
•
05 - Special.
Ship-From
You can choose from multiple ship-from addresses for each Vendor:
•
If the system does not find a ship-from address for the vendor, then this field is set to D to use the default address established in Vendor Maintenance.
•
If a ship-from address is established for the vendor, the system uses the ship-from address from Vendor Ship-From Maintenance.
•
If multiple ship-from addresses are available for the vendor, you are prompted to choose a ship-from address.
Refer to Vendor Ship-From Maintenance for more information.
P/O Date
The date the purchase requisition is generated.
Delivery
The date the purchase requisition is scheduled for delivery.
Promised
The date the merchandise is scheduled for delivery at the warehouse.
Contact
The purchasing contact name from Vendor Maintenance.
Terms Code
The accounts payable terms code from the vendor record to determine the payment terms for this purchase requisition. The codes are set up in A/P Terms Maintenance.
Ship Via
The ship-via code from the vendor record to represent the method for shipping the merchandise.
Buyer
The buyer code to represent the person responsible for this vendor’s products. The default is from the buyer assigned to this vendor in Vendor Maintenance, but you can override it. Refer to Setting Up a Buyer.
Confirmed P/O
Determines if the purchase requisition is confirmed with the vendor:
•
Y - If the purchase requisition is confirmed to the vendor. In this case, the message “Confirming P/O” prints on the purchase order.
•
N - The purchase order is not confirmed with the vendor.
Unusual Delivery
Determines if the delivery time for this purchase requisition is out of the ordinary:
•
Y -The delivery time on this purchase requisition is not the normal length of time for this vendor. This tells the system not to use the delivery date to calculate lead time.
•
N - The delivery time for this order is normal and is used to calculate lead time and rank vendor performance.
Emergency P/O
A Y in this field indicates that the purchase requisition was created in the Emergency/Direct buyer work queue and action should be taken immediately. In this case the delivery date is not used to calculate lead time.
Freight Flag
Indicates to account payable if you are responsible for freight:
•
Y - You are responsible for freight charges billed when this purchase requisition is converted to a purchase order.
•
N - You are not responsible for any freight charges billed when this purchase requisition is converted to a purchase order. This is the default on recommended purchase orders created by Nightly Processing that meet the Frt Paid Amount in the Vendor Line Target Parameters Screen.
Instructions
Instructions to print on the purchase requisitions.
F2-Save
Saves the header information and opens the Purchase Requisition Entry screen.
F5-Add Info
Opens the Additional Information screen to enter additional information regarding this purchase requisition.
F6-Ship-From
Opens the Ship-From Address screen to view or edit the vendor’s ship-from address for this purchase requisition.
F7-Ship-To
Opens the Ship-To Address screen to view or edit the address where the vendor will ship the order.

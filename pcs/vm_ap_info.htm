A/P Information Screen
The A/P Information screen allows you to enter the vendor’s accounts payable invoicing information.
Menu Path: Purchasing > Maintenance > Vendor Maintenance > F6-A/P
Field and Function Key Descriptions
Field/Function Key
Description
Payable Type
Classifies the vendor based on your settings in Vendor Payable Type Maintenance (e.g., Inventory, Expense). Payable Types are used to select invoices to pay. Refer to the Accounts Payable Module for more information on payable types.
Payable Method
Identifies how the vendor is paid and sets the default for Vendor Invoice Reconciliation Entry, Vendor Claim Reconciliation Entry, and Manual Invoice Entry:
•
R - Regular.
•
B - Automatic bank deduction.
•
E - Electronic funds transfer. Refer to the Bank Information Screen.
•
C - Credit card.
Status (T/H/I)
Indicates how the vendor is used in the system:
•
T - Trade. The vendor is used for inquires and transaction entries. Payments are processed as usual.
•
H - Hold. The vendor is used for inquires and transactions, but orders go on hold.
•
I - Inactive. The vendor is inactive and can only be referenced for inquires and reports.
Pay On Receipt?
Indicates if the vendor’s invoices should be reconciled when stock is received or when the invoice is received:
•
N - The vendor invoices are reconciled when the invoice is received through Vendor Invoice Reconciliation Entry.
•
A - When stock is received, the invoice number, invoice date, and freight amount are required. You must complete the process in Vendor Invoice Reconciliation Entry or Pay On Receipt Vendor APPO Mass Update.
•
Y - When stock is received, an invoice number, invoice date, and freight amount are required. The invoice is automatically reconciled and ready for payment.
For more information, refer to the Accounts Payable Module.
Include GST/HST?
Indicates if the vendor uses GST/HST (Canadian taxes):
•
Y - GST/HST should be included for this vendor.
•
N - GST/HST should not be included for this vendor.
Update Commissions
Indicates if commissions should be updated if the actual cost on the invoice is different from the cost on the purchase order for linked items:
•
Y - Update commissions if the cost is different.
•
N - Do not update commissions if the cost is different.
Single Chk Per Inv?
Indicates if you want to print a single check for each invoice for this vendor in A/P check processing:
•
Y - A check is printed for each invoice. You may want to use this so that all invoices on a project are not held up if just one invoice has not arrived. This could also be used for the miscellaneous vendor.
•
N - All invoices for a vendor are combined in one check.
Unearned Disc Days
The number of days past the discount day (based on your Terms Code) that you will take a discount. This overrides Take Lost Discounts? when paying vendors. For example, if the vendor’s Unearned Disc Days is 10, the discount is taken for 10 days past the discount day. Therefore, if the discount date on the invoice is October 1, the discount is taken until October 11, even if you enter N for Take Lost Discounts?. If you print checks after October 11, the discount is not taken even if you enter Y for Take Lost Discounts?.
VAT Registration #
The value added tax number if applicable.
Pay To Vendor
References the vendor number to pay when purchasing from this vendor. Both vendors should have the same setting for Pay On Receipt?.
CC Payment Vendor
If Payable Method is C (Credit Card), this field is the credit card vendor (e.g., Visa, MC). When you reconcile the vendor’s invoice, the invoice shows as paid and the invoice goes to the credit card vendor to be paid monthly.
Off Discount Code
A code that represents the discount negotiated with the vendor (e.g. advertising rebates), that is not reflected when the merchandise is received, but appears when you reconcile the invoice. Refer to Creating Off-Discount Codes.
On Discount Codes
A code that represents the additional discount negotiated beyond your normal merchandise amount. The discount is reflected on your merchandise invoices. Refer to Creating On-Discount Codes.
F2-Save
Saves the Accounts Payable information.
F5-G/L
Opens the Vendor G/L Information Screen to allow you to enter default general ledger accounts to use for manual invoices (e.g., payments for rent or electricity).
F6-1099 Info
Opens the Current Year 1099 Information Screen to allow you to view and edit the vendor’s 1099 information.

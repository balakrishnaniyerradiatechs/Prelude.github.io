Service - Overview
This document provides an overview of the Service module available in Prelude.
System Maintenance
Company Maintenance (/COMPANY.MNT)
To use the Service module, the service flag must be established in Company Maintenance.
Path: System Parameter > Maintenance > Company Maintenance > F6-Modules
Order Processing Parameters
Path: System Parameter > Maintenance > Option Parameters > Order Processing Parameters > F5-Add Info > F5-Add Info
•
Display Quote/Meter Info in SVOE: Y - show the Quote/Meter Information screen in Service Order Entry, N - do not show the Quote/Meter Information screen in Service Order Entry.
•
Display Open Orders Inquiry in SVOE: Y - show the open orders inquiry for the system id number if orders exist, N - do show the open orders inquiry for the system id number.
•
Print Service Documents: Y - show the print box in Service Order Entry to print the service order (required for continuous printing), N - do not show the print box in Service Order Entry.
Warehouse Maintenance (/WM)
Path: System Parameter > Maintenance > Inventory Controls > Warehouse Maintenance > F6-Cont Setup
•
Service Document: Y - continuously print service documents, N - print service documents by batch.
Counter/Range Maintenance
Path: System Parameter > Maintenance > Counter/Range Setup > Service Module
•
Agreement Number Range. Create and maintain service agreement counter range numbers by warehouse.
•
System Number Range. Create and maintain the system counter range numbers by warehouse.
Service Orders
Path: Service
Service Order Entry (/SVOE)
This entry screen is used to enter service orders for services performed. Enter a systems id to determine which system is being serviced. In V16+, the secondary engineer was added to the Service Order Information screen. This field defaults from the customer, customer ship-to number, or system id. Previously, the secondary engineer was the sales rep #1. By keeping the sales rep and engineer fields separate, the reps can be paid commissions on service orders.
Service Quote Entry (/QVOE)
This entry is identical to Quotation Order Entry for entering standing quotes (type 04). Once the service quote is created, the Quotation Log Inquiry is updated for service quote entered. In V17+, when service quotes are converted to service orders using the Standing Quote Conversion process, the order date for the service order is updated with the conversion date. Previously, the order date for service orders was updated with the quote date entered in Service Quote Entry.
Service Agreement Entry (/SAE)
Use this option to create and maintain service agreements, which includes what service is provided to the customer, terms of the service, and the systems and components that are covered.
In V16+, if Customer Price Index (CPI) is Y, the service agreement contract has annual price increases in the Consumer Price Index (CPI). If CPI is N, the service agreement contract does not have annual price increases. Note: CPI is only for query/reporting.
In V17+, you are asked if you want to bill the floor plan customer to use a third-party customer. Choose now to bill to the original customer on the service agreement.
Service Assignment (/SRVA)
This option will assign the service order to a service technician for repair.
Service Document Printing (/SVP)
Prints service orders by batch. Set Print Service Documents in Order Processing Parameters to setup printing. The Order Log Inquiry is updated with the SRV code once a service document prints.
•
Warehouse: the warehouse number to limit the service document printing to or leave blank for all.
•
Sales Order Number: enter the sales order number to limit the service document printing to a sales order number or leave this field blank to include all sales order numbers.
•
Printer Form Queue: enter the printer form queue number to print the service documents.
•
Service Document Date: enter the date to print on the service document form.
•
Service Document Message: the message to print on the footer of the service document form.
Timecard Process
Path: Service > Timecard Process
This process allows you to enter and allocate hours for a mechanic or technician to multiple service or work orders. The system updates each service order listed with the burden code, the number of hours worked, the rate (cost) and the price. Rates may be defined for the type of service.
Service Timecard Entry (/STCE)
This option allocates a base number of labor hours across multiple service orders. The rate for each service order is determined by the type code.
•
Batch Number: the batch number of the service timecard entry to be edited or updated. Enter ‘NEW’ to assign the next available batch number for new service timecard entries.
•
Entry Date: the date the labor was performed. The date defaults to the system date.
•
Employee: enter the employee user id associated with the timecard entry.
•
Warehouse Number: the warehouse number associated with the timecard entry.
•
Total Hours: the number of hours the employee performed service on the service orders on the specified day. This field can be compared to the employee’s timecard.
•
Dtype: the document type update with the labor hours: S - service order, W - work order, or N - no billing (allows you to enter hours that are not related to a service order or work order.
•
Document#: the service or work order number to be updated with labor hours. Skipped if Dtype is N.
•
Labor: the burden code in Burden Maintenance to update labor hours for the document.
•
Type: the type of service labor hours for the order (e.g., OT for overtime hours).
•
Bill-Hours: the number of hours to be billed to the service order.
•
Pay-Hours: the number of hours the employee actually worked on the service. These hours may be different than Bill-Hours if non-billable time occurred.
•
UB-Code: the unbilled code from Valid Response Maintenance (UBC code) if Bill-Hours and Pay-Hours do not match.
Timecard Audit Report (/SRV.TC.AUDIT.RPT)
This report displays unprocessed timecards.
•
Batch Number: enter the batch number to limit the report to a batch number or leave blank for all.
•
Warehouse Number: warehouse number to limit the report to or leave blank for all warehouses.
•
Entry Date: date of the timecard to limit the report to a date or leave blank to include all entry dates.
Timecard Batch Process
This process updates all unposted timecard entries. Billable hours for service orders are added using the product number in Burden Maintenance for the burden type code on the timecard entry. Billable hours for work orders are added is updated with the burden code on the timecard entry.
•
Batch Number: the batch number to limit the process to or leave blank for all.
•
Entry Date: entry date of the timecard to limit the process to a date or leave blank to for all dates.
•
Warehouse Number: warehouse to limit the process to or leave blank to include all warehouses.
Order Log Inquiry (/OLI)
TME appears when the Timecard Batch Process runs for each burden type code on the service order. Reference # is the product number for the type. If the product number is on the service order hours are added to the service order, Reference # shows the prefix “Added To” before the product number. Line item notes for each burden type are added to the service order with the user id, labor, and type.
Path: Order Processing > Inquiries > Sales Order Inquiries > Sales Order Log Inquiry
Service Maintenance
Path: Service > Maintenance
•
Systems Maintenance (/SYSM) - A system record is a room or department with equipment. The header information applies to the whole system. The detail information lists the components. Note: In V16+, the tertiary engineer field was added.
•
Equipment Types Maintenance (/SETM)
•
Warehouse Departments Maintenance (/SWDM) - Add and maintain service warehouse departments. Rates and costs associated with working in each department can be set.
•
Customer Maintenance (/CM) - Add and maintain customer numbers associated with the service module. InV16+, the sales rep 1, sales rep 2, and tertiary engineer fields were added to the Service Information screen (Action Add’l Options menu).
•
Customer Service Comments Maintenance (/CMSC) - Add and maintain service comments to be displayed when the customer number is entered in Service Order Entry.
•
Customer Ship-To Maintenance (/CSTM) - Add and maintain the customer ship-to records. In V16+, the sales rep 1, sales rep 2, and tertiary engineer fields were added to the Service Information screen (Action Add’l Options menu).
•
Burden Maintenance (/BUM) - Enter time sheet hours worked on sales order or work orders. You can create a burden code and labor rates for that burden code in Burden Maintenance. Some of the fields on this screen pertain to work orders timesheet entries and do not affect service orders.
-
Code: the burden code to create or maintain.
-
Description: the description of the burden code to be displayed for reference.
-
Type: the type of the burden code: L - for labor, A - accounts payable, O - outplant processing.
-
Amt: the amount to charge if this burden code is used in Work Order Timesheet Entry to update a work order.
-
U/M: the units in which the burden code is measured (e.g., labor would be measured in hours).
-
F5-Burden Rates: type codes along with a rate and a price. The rate is the cost of the labor. The price is the hourly charge to the customer and is only used for linked work orders.
-
Type: type code to use with the burden code in Service Timecard Entry to define which rate to allocate to the service order (e.g., RT– Regular Time, OT– Overtime, FT– Field Time, FOT– Field Overtime, and PT– Premium Time, RTL– Regular Time Linked, OTL– Overtime Linked).
-
Rate: this field defines the rate for the corresponding type. This is the cost that will track to the service order for the burden item.
-
Price: this field defines the price for the corresponding type for a work order linked to a sales order (e.g., customer repairs). This price tracks to the work order for the burden item if the work order is linked to a sales order.
-
Product Number: the product number (typically a service item) to be used for updating a service order when this burden code type is used in Service Timecard Entry.
If allocating time by mechanic or technician, you may want to set up each person as a burden code to track hours and billing rates. If the mechanics or technicians bill at the same rate, you can set up one labor burden code for all labor costs. Or you can set up burden codes per department or group.
Inquiries
Path: Service > Inquiry
•
Service Order Inquiry (/SVOI) - Service orders for a customer number.
•
Service Quote Inquiry (/SVQI) - Service quotes entered in Service Quote Entry.
•
Service Agreement Inquiry (/SAI) - Service agreement data for a service agreement number.
•
Customer Inquiry (/CI) - Customer information for a customer.
•
Customer Ship-To Inquiry (/CSTI) - Customer ship-to information.
•
Systems Inquiry (/SYSI) - Information entered in System Maintenance.
Service Listings
Path: Service > Listings
•
Service Agreement Listing (/SRV.AGREE.LST) - service agreements.
•
Person Titles Listing (/SRV.TITLE.LST) - service person titles depending on the criteria entered.
•
Warehouse Departments Listing (/SRV.DEPT.LST) - warehouse departments.
•
Equipment Types Listing (/SRV.TYPE.LST) - equipment types depending on the criteria entered.
•
Systems Listing - information on all or selected system records based on the criteria.
Reports
Path: Service > Reports
•
Service Order Summary Report (/SRV.ORD.SUM.RPT) - data on service orders in summary format. Criteria include order date, customer number, and status.
•
Daily Service Invoice Register (/DLY.SRV.REG.RPT) - invoiced service orders for a date.
•
Monthly Service Invoice Register (/MLY.SRV.REG.RPT) - invoiced service orders for a fiscal period.
•
Preventative Maintenance Due Report (/PM.DUE.RPT) - preventative maintenance due for the systems in Systems Maintenance.

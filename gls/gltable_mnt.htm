General Ledger Table Maintenance
The general ledger table lists all the general ledger account numbers available for posting transactions to the general ledger. Initially, Prelude provides you with a Default Chart of Accounts, but you can add or edit account numbers as necessary.
To access the general ledger table:
1
Select Financial > General Ledger > Maintenance > General Ledger Table to open the General Ledger Table Maintenance screen.
You can also access the general ledger table from General Ledger Controls on the Systems menu.
Depending on how you set up your general ledger hierarchy (refer to Setting up the General Ledger Hierarchy) the associated Division, Region, Location, Department, and Cost Center display for each general ledger account.
2
Verify or edit the prefix information for each G/L Table Heading, to determine how transactions post:
•
Div - Division. The highest possible grouping in your company (e.g., Eastern Division, Western Division). Enter X for a wildcard if the division should be taken from the transaction being posted.
•
Reg. Region. A subset of Division - if Division is used (e.g., Southeast Region, Northwest Region). Enter X for a wildcard if the region should be taken from the transaction being posted.
•
Loc. Location - The warehouse or location that receives credit for the sale. Enter X for a wildcard if the location should be taken from the transaction being posted.
•
Dep. Department. A subset of warehouse. Typically used to separate expenses within a warehouse (e.g., accounting, sales, warehouse). Enter X for a wildcard if the department should be taken from the transaction being posted.
•
Ccr. Cost Center. A subset of Department - if Department is used. Typically used to separate expenses within a warehouse or department (e.g., shipping, receiving, outside sales, inside sales). Enter X for a wildcard if the cost center should be taken from the transaction being posted.
Do not use wild cards for departments or cost centers on balance sheet accounts (i.e., a standard that lists the most liquid accounts first).
The Account that displays is the general ledger account number to use when posting to this account. Refer to the Default Chart of Accounts. You can edit the account if necessary.
3
Press F2-Save to save any changes made to the general ledger table.
Unused Accounts
If an item on the table does not apply to your company’s activities, create a new account number for it but mark it Do Not Use somewhere in the description of the account (e.g., 9901 DNU – Rebate Receivable, 9902 DNU – Core Receivable). Always use a new account number for each item that you don’t expect to have general ledger activity. Then if Prelude does post to the account, the transaction is easier to research.
Intercompany Accounts
Suppress I/C Postings? (Option Parameters > General Ledger Parameters) Determines if intercompany postings are suppressed for general ledger accounts that are hard coded with a prefix/account in the General Ledger Table Maintenance screen:
•
Y - Do not post to intercompany general ledger accounts that are hard coded with a prefix/account in the General Ledger Table Maintenance screen.
Bal G/L Posting By Location? must be set to N.
•
N - Allow intercompany postings for general ledger accounts hard coded with a prefix/account in the General Ledger Table Maintenance screen.
The following intercompany flags are hard-coded in the general ledger table:
•
Intercompany A/R Clearing.
•
Intercompany A/P Clearing.
•
Transfer Inventory In.
•
Transfer Inventory Out.
•
Intercompany Sales A/R.
•
Intercompany Sales A/P.

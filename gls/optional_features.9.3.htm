Maintenance Options for Multiple Fiscal Periods
You must use the following maintenance programs to set up multiple fiscal periods.
Company Maintenance
To use multiple fiscal periods in your company, set Multiple Fiscal Periods (Company Maintenance > F5-Add Info) to Y.
Location Maintenance
Each location points to its own a fiscal account. Separate Fiscal (Systems Menu > General Ledger Controls > Location Maintenance) defines the fiscal account a location belongs to and points to the fiscal period to post to. For example, Dallas is pointed to the MAR account, Houston to the JUN account, and Tulsa and San Antonio are pointed to the OCT account. Tulsa and San Antonio will share a fiscal account since they have the same fiscal year.
If Multiple Fiscal Periods in Company Maintenance is set to Y but a Separate Fiscal account is not designated for a location, then that location posts to the parent account.
Multi-Fiscal Period Maintenance
Multi-Fiscal Period Maintenance (Systems Menu > General Ledger Controls) allows you to define fiscal periods for each fiscal account. You can use this as a cross-reference for each fiscal period/company to indicate which fiscal period to post to in each fiscal account. For example, a transaction posted on the parent account in period 07-05 may post to 07-12 in the fiscal account.
Multiple Fiscal Account Copy Procedure
If you want to copy all chart of accounts from the parent account to individual fiscal accounts, you can use the Multiple Fiscal Account Procedure (see your Systems Administrator). Once the fiscal accounts and locations are set up, this process copies existing records to each fiscal account. The Fiscal Account Name is the name of the fiscal account you are setting up. All appropriate division/region/location type records are copied.
You must complete this procedure before using Chart of Account Duplication to assure that each fiscal account and the parent account have identical divisions, locations, and departments.
Chart of Accounts Description Maintenance
When you change an account description in Chart of Account Description Maintenance (refer to Changing a G/L Account Description), each fiscal account is automatically updated when an item is added or changed in the parent account.
For example, if you change the description for account 4100 from DIRECT SALES to SALES-DIRECT, account 4100 in each fiscal account is updated with the new description. When you create a new account description, it is copied to each fiscal account. Chart of Accounts descriptions should be maintained in the parent account instead of in the fiscal accounts.

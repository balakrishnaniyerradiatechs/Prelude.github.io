Entry Programs that use Project IDs
The following entry programs require a valid project ID if the G/L account number entered has Project ID Tracking set to Y (refer to Maintenance Requirements for Project Tracking).
Journal Entry
When you enter a G/L account number with Project ID Tracking set to Y, you must enter the project ID after you enter the debit or credit amount. The Project G/L Inquiry displays the project ID and the debit/credit amount for the G/L account number once the journal entry is posted.
If you enter a Reversal Date for the journal entry, you are not required to enter a project ID since the G/L account be will reversed out in the following month.
Recurring Journal Entry
When you enter a G/L account number with Project ID Tracking set to Y, you must enter the project ID after you enter the debit or credit amount. The Project G/L Inquiry displays the project ID and debit/credit amount for the G/L account number once the recurring journal entries are posted.
If you enter a Reversal Date for the journal entry, you are not required to enter a project ID since the G/L account be will reversed out in the following month.
If you use the Copy feature for the recurring journal entry, the project ID is copied to the new recurring journal entry.
Cash Receipts Entry
When you write off a miscellaneous amount in Cash Receipt Entry (M/C set to M), and you enter a G/L account number that has Project ID Tracking set to Y, you must enter the project ID after you enter the debit amount. The Project G/L Inquiry displays the project ID and debit amount for the G/L account number once the cash receipt is posted.
A/P Manual Invoice Entry
When you enter an G/L account number in the Accounts Payable Manual Invoice Entry screen that has Project ID Tracking set to Y, you must enter the project ID after you enter the Dist Amt. The Project G/L Inquiry displays the project ID and debit/credit amount for the G/L account once the invoice is posted.
Vendor Invoice Reconciliation Entry
If you write off a miscellaneous amount in Vendor Invoice Reconciliation Entry, the G/L Accounts for Discrepancy screen appears when you enter an amount in Misc Amt. If you enter a G/L account that has Project ID Tracking set to Y, you must enter a project ID after you enter the Dist Amt. The Project G/L Inquiry displays the project ID and debit/credit amount for the G/L account once the invoice is posted.
Option Parameters
The G/L Accounts for Discrepancy screen only appears if Misc Detail G/L in APPO in Accounts Payable Parameters is set to Y.

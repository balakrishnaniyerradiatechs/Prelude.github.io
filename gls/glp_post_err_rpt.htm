Unposted G/L Error Report
The Unposted G/L Error report lists transactions that failed the G/L Balance Posting Routine (refer to Posting to the General Ledger). Transactions may fail the G/L posting due to a location being out of balance, a multi-value mismatch, a missing account number, or debits and credits being out of balance. You can use this information to research errors.
After you run the report, you can choose to purge the information so that it is not included the next time you run the report.
Warning: Contact Activant if you have posting errors. Do not purge the data until after the problem is corrected.
Menu Path: Financial > General Ledger > Reports > Unposted G/L Error Report
Column Descriptions
Column Heading
Description
Period
The fiscal period of the failed posting. The dash (-) is replaced by a T (e.g, 07T04) so that it does not affect the general ledger until the problem is solved. Contact Activant Prelude Support.
MD
The general ledger module of the failed posting. Refer to the General Ledger Module Code Definitions for an explanation of each module code.
Counter
The General Ledger Posting transaction counter number.
Document Id
The document number for the associated transaction.
Account
The general ledger prefix and account number.
Description
The description of the general ledger account.
Debit
The amount debited to the general ledger account.
Credit
The amount credited to the general ledger account.
+/-
Total out-of-balance amount for the transaction.
Posting Errors
The reason for the out-of-balance error (e.g., Debits, credits out of balance).

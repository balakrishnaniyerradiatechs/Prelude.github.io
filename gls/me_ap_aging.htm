Saving Vendor A/P Aging Balances for Month-End
The Month-End Vendor A/P Aging saves all accounts payable balances for a fiscal period to update the accounts payable reports and inquiries with the aging status of invoices due to vendors. You should only run this process once a month, overnight, on the month-end cutoff date.
To run the Month-End Vendor A/P Aging:
1
Select Financial > General Ledger > Processes > End of Month End Closing > Month-End Vendor A/P Aging. The following message appears:
This procedure will capture all vendor accounts payable aging buckets as of the point in time this process is run. We will save the future, current, 1 to 30, 31 to 60, 61 to 90, 90 & over. This process should only be run at the month end cutoff and should be executed overnight. This process should only be run once a fiscal period. This will allow you to inquire on the aging status for a vendor for any fiscal period captured. Continue With Procedure?
2
Choose from the following:
•
No - Cancels the procedure. Returns to the menu.
•
Yes - Opens the Month End A/P Aging Save Routine GRIM screen.
3
Enter the fiscal Period that you are closing.
4
Press ENTER. The following message appears:
Do You Wish To..............
5
Choose from the following:
•
Execute: Begins the month-end save.
•
Schedule: Allows you to schedule the process to run later or to occur monthly.
•
Save Params: Allows you to save the input parameters to use again later.
•
Return: Returns to the selection criteria.
•
Exit: Cancels the process.

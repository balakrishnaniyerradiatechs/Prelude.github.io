Cost Adjustments (CA)
Cost adjustments only adjust average cost. When cost is adjusted, the following postings occur:
•
Inventory is debited for a positive adjustment, or increase, in cost.
•
Inventory is credited for a negative adjustment, or decrease, in cost.
•
The offsetting entry is posted to the account assigned to the cost adjustment reason code. If none exists, it defaults to the Cost Adjustment account in the G/L table (refer to the Default Chart of Accounts
The transaction amount is calculated by the difference between the current (old) average cost and the new average cost multiplied by the current on-hand quantity. To adjust cost, try to correct any error by reversing the transaction that caused the error, instead of adjusting cost. Typically you can correct many cost errors in Vendor Invoice Reconciliation Entry by using negative receivers.
Average Cost Equation (for unlinked transactions only)
[(Avg. cost X previous stock on-hand) + (quantity x transaction cost)] / new stock on-hand.

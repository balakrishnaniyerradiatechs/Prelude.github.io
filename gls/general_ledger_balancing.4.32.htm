Balancing by Location
System parameters determine if journal entries and general ledger postings should balance by location. The Unposted G/L Error Report lists transactions that did not balance due to a location being out of balance.
General Ledger Parameters
Bal G/L Posting By Location? determines if posted transactions balance by location during the G/L Balances Posting Routine (refer to Posting to the General Ledger):
•
Y - The location must balance before posting the transaction. If the General Ledger posting does not balance by location, an audit file is updated. The Unposted G/L Error Report lists transactions that did not balance.
•
N - The balance posting routine verifies that the debits and credits balance but does not check that each location is in balance. The Unposted G/L Error Report lists transaction information for reconciliation.
Journal Entry
When you adjust the general ledger using a journal entry (refer to Journal Entries), if G/L Posting By Location in General Ledger Parameters is set to Y, the system verifies that the entry balances by location. If the transaction does not balance, a warning message appears and you must enter a password. You must either balance the entry by location or enter a valid password before you can save the journal entry.
G/L Balance Posting Routine
When you post transactions from each module to the general ledger (refer to Posting to the General Ledger), if Bal G/L Posting By Location in General Ledger Parameters is set to Y, the system verifies that the transactions, credits, and debits balance by location. If posting errors are found, the following occurs:
•
Each transaction is marked by replacing the dash in the fiscal period with a T. For example, if the period for the transaction is 07-08 then it changes to 07T08 to keep the transaction from going though the posting routine again until it is fixed.
•
The audit file is updated.
•
The message “G/L Posting Errors Occurred. Please Call Prelude!” appears. Enter a support case with Activant Prelude.
•
The Unposted G/L Error Report lists any transactions not posted due to balancing discrepancies.

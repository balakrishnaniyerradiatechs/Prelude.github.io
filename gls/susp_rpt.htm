Suspense Account Posting Report
The Suspense Account Posting report lists suspended account entries that occur when the general ledger update references an invalid account number. The system posts to the suspense account when it cannot find a valid account number in the Chart of Accounts. For example, if the inventory account is set up as a wildcard (e.g, XX 1200) and the new selling location of 50 1200 does not exists, then it will post to the suspense account.
Menu Path: Financial > General Ledger > Reports > Suspense Account Posting Report
Selection Criteria
Use the following criteria to limit the information to print on the report:
•
Posting Period. The fiscal period of the invalid account postings.
Column Descriptions
Column Heading
Description
G/L Number
The invalid general ledger account prefix that the system tried to post to.
Description
The description of the general ledger account, if applicable.
History Key
The document number (e.g., order number).
Module
The general ledger module and description. Refer to the General Ledger Module Code Definitions for further explanation of each module code.
Debit Amount
The amount debited to the suspense account.
Credit Amount
The amount credited to the suspense account.

Saving Customer A/R Aging Balances for Month-End
The Month-End Customer A/R Aging Save saves all customer A/R balances for a fiscal period to update the A/R reports and inquiries with the correct aging status of customer accounts (e.g., AvgPay, AvgOvr, DaysSO in the Accounts Receivable Inquiry). You can schedule this process but should only run it once a month, overnight, and on the month-end cutoff date.
 
Accounts Receivable Parameters
Save M/E A/R Stmt Detail
•
Y - Saves detail A/R Information at month end for month-end aging inquiries and reports so that you can re-run statements from previous months.
•
N - Month-end statement information is not saved. This option is recommended to keep your files from getting too big.
To run the Month-End Customer A/R Aging Save:
1
Select Financial > General Ledger > Processes > End of Month End Closing > Month-End Customer A/R Aging Save. The following message appears:
This procedure will capture all customer accounts receivable aging buckets as of the point in time this process is run. We will save the future, current, 1 to 30, 31 to 60, 61 to 90, 90 & over and a disputed invoices flag. This process should only be run at the month end cutoff and should be executed overnight. This process should only be run once a fiscal period. This will allow you to inquire on the aging status of a customer for any fiscal period captured. Continue With Procedure?
2
Choose from the following:
•
No - Cancels the procedure. Returns to the menu.
•
Yes - Opens the Month End A/R Aging Save GRIM screen.
3
Enter the fiscal Period that you are closing.
4
Press ENTER. The following message appears:
Do You Wish To..............
5
Choose from the following:
•
Execute: Begins the month-end save.
•
Schedule: Allows you to schedule the process to run later or to occur monthly.
•
Save Params: Allows you to save the input parameters to use again later.
•
Return: Returns to the selection criteria.
•
Exit: Cancels the process.

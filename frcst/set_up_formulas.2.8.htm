Generating Forecasting Data
Forecasting data is regenerated by:
•
Running Nightly Processing.
•
Running Inventory End-Of- Month procedures.
•
Generating Historical Forecasts.
Nightly Processing
Menu Path: Purchasing > Processes > Nightly Processing
Nightly Processing evaluates the replenishment status of all products in each warehouse and recommends the purchasing requirements for the next business day. This process, normally run at the end of the day, reviews all open purchase orders, sales orders, work orders and inventory transfers to calculate the net available inventory for each product. If the net available is below the order point, line point, or minimum (whichever purchase method is being used), a recommended purchase order, direct purchase order, or inventory transfer is created depending on the product’s replenishment method.
For more information, refer to “Nightly Processing” in the Purchasing documentation.
 
Inventory End of Month Calendar Close
Menu Path: Inventory > Processes > End of Month Calendar Close
The Inventory End of Month Calendar Close analyzes the forecasting data and hierarchy levels to calculate a forecast quantity for each product (refer to Setting Up Forecasting Formulas and Trends). The forecast, formula, and exception fields displayed on the Product Usage Inquiry screen are updated. Inventory control files including forecast, trend percent maximum, line point, order point, and EOQ are also updated.
 
Historical Forecasts
Menu Path: Systems > System Parameter > Processes > ADS Utilities > Inventory Utilities > Generate Historical Forecasts
You can generate historical forecasts to capture each month’s forecast for each product and store it to compare against actual usage. The forecast column in the previous year and current year usage screens are updated for each product warehouse record. The Product Usage Inquiry screen in Distribution Requirements Evaluation Inquiry shows the traditional usage information along with the forecast to compare forecasting performance. You can calculate the forecast for previous months to compare actual usage to what would have been forecasted usage. Forecast updates are calculated based on warehouse, product line, and the number of previous month’s usage.
 
The Generate Historical Forecasts process could take a long time to run based on the number of months chosen and the database size. Execute the procedure one warehouse at a time (possibly by product line) and calculate using 12-24 months.

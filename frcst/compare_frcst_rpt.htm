Comparison Forecast Report
The Comparison Forecast report allows you to compare the accuracy of each formula to actual usage by:
•
Re-forecasting actual usage for each product using each formula.
•
Calculating the deviation rate for the last 12 months. The number of months is a variable depending on company settings.
•
Identifying the formula that provides the lowest deviation rate based on the actual usage vs. forecast.
•
Providing a method to update the product with the suggested formula.
You can view the current formula and forecast for each product and compare it to the recommended formula and forecast to determine the best performance.
For a forecast formula to be included in the report, the Forecast Analysis flag in Forecast Formula Maintenance must be set to Y. Refer to Creating Forecasting Formulas.
The following products are excluded from formula calculations:
•
* (Asterisk) products.
•
Products with frozen purchasing controls (Frozen Controls set to Y in Purchasing Parameters Maintenance).
•
Non-stock products (Status Code set to N in Product Warehouse Maintenance).
•
Products flagged as having sporadic usage (Sporadic set to Y in Purchasing Parameters Maintenance).
Menu Path: Purchasing > Forecasting Subsystem > Comparison Forecast Report
Selection Criteria
Use the following criteria to limit the information to print on the report:
•
Enter Month End Closing Date. The date of the month end to include in the report. For example, if today is May 10, use April 30 as the month end date.
•
Warehouse. The warehouse where the material is stocked.
•
Purchase Method. The purchase method to review (e.g., EOQ, DRP, CLS, AWU). Established in the product warehouse record.
•
Product Line. The product lines to include in the report.
The first time you use this report, start by using just one product line to determine how the formulas work.
•
Product Range. The beginning and ending product numbers to limit the report to a set of product numbers.
•
Class. The product classifications of the items to include.
•
Primary Vendor. The primary vendors to include. Primary vendors are assigned in Product Vendor Maintenance.
•
Frozen Controls. Determines if you want to include products whose purchasing controls are frozen:
-
Y - Only include products whose purchasing controls are frozen. (Frozen Controls set to Y in Product Purchasing Parameters Maintenance).
-
N - Only include products whose purchasing controls are not frozen (Frozen Controls set to N in Product Purchasing Parameters Maintenance).
-
Blank - Include all products whether or not their purchasing controls are frozen.
•
Output Presentation. Indicates whether you want a summary or a detail report:
-
D - Print a detail report to show each formula and how it performed based on a deviation percent.
-
S - Print a summary report to show the current and suggested formula.
-
U - Print a summary report to show current and suggested formulas and usage for the past 12 months.
•
Display Formula Changes Only?. Indicates if you want to print all formulas or only those that recommend a change:
-
Y - Display products for which the comparison analysis suggests a different formula than what is assigned to the product.
-
M -Display all results regardless of changes.
•
Update Best Formula?. Indicates whether or not you want to update the formulas:
-
N - Print the comparison forecast report without updating the product.
-
Y - Update each product with the best calculated formula and prints the report for audit purposes.
-
U - Update each product with the best calculated formula but does not print the report.
•
# Of Months For Evaluation. The number of months to evaluate the deviation error rate when comparing actual to forecasted usage for the formulas evaluated. For example, enter 12 to compare the formulas for the last 12 months of history versus the forecasts to determine the best performing formula over the 12-month period.
•
Average Error Rate. The error rate cutoff that the current forecast formula for the product must not exceed. Allows you to focus on products where the current forecast formula is not working as well as you’d like.
•
Best Forecast Difference. Limits the list to only include products whose recommended forecast deviated by more or less than the current forecast by this value. This option allows you to double check products with a large swing in usage and identify any change in usage patterns:
-
A positive value includes products whose best forecast has increased over the current forecast by this value or higher.
-
A negative value includes products whose best forecast has decreased under the current forecast by this value or lower.
Column Descriptions
Column Heading
Description
Product/Description
Line 1: The product number.
Line 2: The product description.
Whse
The warehouse where the material is stocked.
Est-Date
The date the product was created for this warehouse.
CFormula
The current formula assigned to the product.
Cfrcst
The current forecast for the product.
Bformula
The best formula recommended for the product for the best performance.
Bfrcst
The forecast quantity if using the best formula recommended.
Diff
The difference between the current formula or the recommended best formula.
Cost
The cost of the product.
Investment
The dollars that would be invested if you used the recommended best formula.
Formula
A list of formulas in your system.
Frcst
The forecast quantity that would be recommended if using each of the corresponding formulas.
Dev %
The percent deviation between actual and forecasted usage.
Low
The lowest deviation percent or best formula.

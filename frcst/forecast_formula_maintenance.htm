Creating Forecasting Formulas
You can create forecasting formulas in Forecast Formula Maintenance to calculate forecasts for each product, product line, vendor line, or warehouse. You can assign a weight to each month if the usage for some months should be considered more than usage in other months. The system provides several commonly used formulas; however, you can create additional formulas for situations unique to your business.
To print a listing of existing forecasting formulas and descriptions, select Purchasing > Forecasting > Forecast Formula Listing.
Forecast Formula Maintenance
To create or modify a forecast formula:
1
Select Purchasing > Forecasting Subsystem > Forecast Formula Maintenance to access the Forecast Formula Maintenance screen.
2
Enter the Formula Id to create or maintain or press F3 to search for existing formulas.
3
Enter a Description of the formula.
4
To calculate trend, you can either enter a trend percentage for each month of the year, or enter the number of trend months. Choose from the following:
•
Enter the number of Trend Months to use to calculate trend factor and trend quantity. Leave this field blank to calculate trends at other levels or if you are not using trends.
You can use trend forecasting to determine future product demand based changes in business conditions. The calculated trend quantity is added to or subtracted from the forecast thus affecting order point, line point, and purchasing quantity. Trend Months at this level can be overridden by Trend Months at the Product Warehouse level.
If you calculate the Trend Factor using Trend Months, the following rules apply:
•
If last year’s usage is zero, the Trend Factor is zero.
•
If last year’s usage is less than zero and the current year’s usage is greater than zero, the Trend Factor is 100.00.
•
If the calculated Trend Factor is greater than 100, the Trend Factor is 100.00.
•
If the calculated Trend Factor is less than -100, the Trend Factor is set to -100.00.
•
If the trend is not always positive or always negative for the number of months indicated, the Trend Factor does not apply.
•
Press F5-Trend Months to open the Trend % by Month screen.
Enter the trend percentage for each month. These values override any value entered in Trend Months and becomes the initial default for trend percentage for this formula.
You can set a Trend % Maximum in Warehouse Maintenance > F5-Add Info it you do not want the system to calculate a trend factor above a certain percent.
5
For Seasonal Flag, select one of the following:
•
Yes - To apply a seasonal pattern analysis by weighting months from last year (e.g., months 10-12).
•
No - To apply a non-seasonal pattern analysis by weighting months that just occurred (e.g., months 1-6).
6
For Forecast Analysis, select Yes to include the formula in the Comparison Forecast Report for analysis. This report reviews all formulas to recommend the formula with the lowest error rate.
7
Enter the Weight to apply to each month’s actual usage to calculate the upcoming forecast. Each Month listed indicates which previous months are used in the forecasting calculation. For example, 2nd denotes two months back.
8
Press F2-Save to save the forecast formula.
Exclude Warehouses From Formulas
If you want to prevent a warehouse from using a formula, from the Action Bar, select Exclusions and enter any warehouses that should not use this formula. You many want to exclude warehouses for seasonal formulas if different geographic locations do not have the same seasonal purchasing requirements.
Copy Formulas
To copy the information from one formula to create a new one, before you enter a formula, select Copy from the Action Bar and enter the Original Formula ID and the New Formula ID to create. Modify the new formula as necessary.

Forecast Calculation for 6 Month Average Formula
To calculate the forecast for October, multiply the usage for each of the last 6 months by the weight factor for each month. Add the 6 totals and divide the sum by the total of the weight factors.
If each of the 6 previous months has a weight factor of 1:
-
Actual usage x weight for the last 6 months: (43x1) + (37x1) + (22x1) + (40x1) + (80x1) + (60x1) = 282
-
Total weight: 1+1+1+1+1+1 = 6
-
Forecast for October: 282/6 = 47
Forecast Calculation for Seasonal 3/1-0 Month Trend Formula
To calculate the forecast for October, multiply the usage of each of the months 10, 11, and 12 months back by the weight factor for each month. Add the 3 totals and divide by the total weight factor.
If the weight factor for December =1, November = 2, and October = 3:
-
Actual usage x weight for the 3 designated months: (90x1) + (97x2) + (99x3) = 581
-
Total weight: 1+ 2 + 3 = 6
-
Forecast for October: 581/6 = 97
 
Field and Function Key Descriptions
Field/Function Key
Description
Formula Id
The code that identifies the forecast formula.
Description
The description of the formula.
Trend Months
The number of months to calculate the trend factor and trend quantity. Trend Forecasting is used to forecast product demand based on past sales. The calculated trend quantity is added to or subtracted from the forecast thus affecting order point, line point, and purchasing quantity. Trend Months at this level can be overridden by Trend Months at the Product Warehouse level. When calculating the Trend Factor using Trend Months, the following rules apply:
•
If last year’s usage is zero, the Trend Factor is zero.
•
If last year’s usage is less than zero and the current year’s usage is greater than zero, the Trend Factor is 100.00.
•
If the calculated Trend Factor is greater than 100, the Trend Factor is 100.00.
•
If the calculated Trend Factor is less than -100, the Trend Factor is -100.00.
•
If the trend is not always positive or always negative for the number months indicated, the Trend Factor does not apply.
Seasonal Flag
Determines if the formula is for seasonal or non-seasonal products:
•
Yes - To apply a seasonal pattern analysis by weighting months from last year (e.g., months 10-12).
•
No - To apply a non-seasonal pattern analysis by weighting months that just occurred (e.g., months 1-6).
Forecast Analysis
Determines if you want to include the formula in the Comparison Forecast Report for analysis. This report reviews all formulas to recommend the formula with the lowest error rate.
Month
Indicates which previous months are used in the forecasting calculation. For example, 2nd denotes two months back.
Weight
The weight to apply to each month’s actual usage to calculate the upcoming forecast.
F2-Save
Saves the forecast formula.
F4-Del
Deletes the forecast formula.
F5-Trend Months
Opens the Trend % By Month screen. You can enter trend percents from January through December. These values override the Trend Months in Forecast Formula Maintenance and work as an initial default for trend percentage when the formula is used.
Action Bar
Copy
Allows you to copy an existing forecasting formula to create a new formula with similar features. You can then modify the new formula. This feature is only accessible from the Formula Id Field.
Exclusions
Opens the Warehouse Exclusion screen to allow you to enter any warehouse numbers to exclude from this formula. You many want to exclude warehouses for seasonal formulas if different geographic locations do not have the same seasonal purchasing requirements.

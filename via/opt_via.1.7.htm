Access
The Access Rights screen shows authorizations for accessing options on the web page.
Access – Check any of the following to indicate that the user can…
•
Place Order: Place an order.
•
Ship To Sel. Change the ship to address at checkout.
•
Manual Ship To: Enter a manual ship to address at checkout.
•
Convert Quote: Convert a quote to a sales order on the Quote Detail Inquiry page.
•
CC Entry: Enter credit card information. Required is based on the Cc Required Flag.
•
Templates: Save the sales order cart contents for future use.
•
Mult Favorites: Group products in multiple favorites lists.
•
Required Date: Enter a required product date on sales order.
•
Include In Statistics: Be included in the Activity Statistics information.
•
Request Password Change: Be continuously asked to change their password after logging on to VIA.
•
Add to List: Add to the Favorites List on the web page.
•
Ship Whse: Change the shipping warehouse.
•
Hold Orders: Indicates if all sales orders for this user go on administrative order hold.
•
Ship Via: Change the ship via on the Checkout page.
•
Multi Carts: Put items in multiple carts for future use (e.g., a cart for each month).
•
Print Views: View a printable version of shopping lists or of any page with lists.
•
Comments: Enter comments on the website.
•
Dup Items: Enter product on order multiple times.
Sales Rep/Internal Access Functions - Check any of the following to indicate that a salesrep can…
•
Create New Customer: Create a new customer in Customer Maintenance.
•
Change Price in Quick OE: Change a product price in Quick Order Entry.
•
Hold Release: Release orders from Credit, Admin Order, or Admin Invoice Holds.
Calculation & Control - Check any of the following to indicate that a user can…
•
Cust. Price: See their best price or price 0.
•
Calc Shipping: Sees flat rate shipping on the checkout Totals page.
•
CC. Required: Enter credit card information if a credit card is required.
•
Account Mgr: Change a customer’s password from the Admin Password page.
•
Min/Max: Indicates if minimum/maximum settings are enforced.
•
Available Policy: Name of program that calculates availability. Overrides Web Policies Maintenance.
•
Notifications: Indicates if an acknowledgement is emailed or faxed to the web user. Determines if the user can choose Ack options on the Checkout page.
•
# Months: The number of months' historical data that the order pad will display.
•
Submit Type: Indicates if the user can place: O=Order Only, Q=Quote Only, B=Both.
•
Allow Splits: Move non-stock products to a quote or split the items off and delete them.
•
Allow Msrp Entry: Change the MSRP during line item entry on the sales order.
Scope Restrictions/Misc.
•
Scope List: Name of the list of products the user can view. The list is from List Maintenance.
•
Industry: A grouping of customers.
•
F5-View Options: Refer to View Options.
•
F7-Order Types: Chose the order types the user can enter: 01 Regular, 03 Tag & Hold, 06 Confirmation, 12 Invoice When Complete, 14 Ship Line Item Complete, 16 Order Projection, 18 Crib Replenishment, 02 Ship Complete, 04 Direct Ship, 07 Regular Consignment, 13 Ship Backorder Complete, 15 Show Order, 17 Manufacturer Rep
•
F8-Qualifiers: If a product has qualifiers in Product Maintenance, enter the qualifiers here. You can then restrict viewing based on the qualifier. If the field is blank you can view all products.
•
Action Bar-Cust List: Refer to Action Bar - Customer List .
View Options
This screen lets you choose what a user can see. The flags in Company Web Policies override these flags.
•
MSRP: Manufacturer’s suggested retail price.
•
Available: Product quantity availability if the system-level setting agrees.
•
Price: Product price.
•
All Activities: All sales orders or only those you created.
•
Flyover: Help by passing the cursor over a field.
•
Substitutes: Product substitutes.
•
Accessories: Product accessories.
•
Stock Status: Stock status (S or NS).
•
Cost: Product cost.
•
Price Source: Product price source.
•
Items Per Page: The number of products and hearders to display per page (1–200).
Menu Options
List of menu items that appear for the user
•
Menu Security For: The menu that should appear.
•
Lang Res: The text that should appear.
•
Description: Menu item description (not editable).

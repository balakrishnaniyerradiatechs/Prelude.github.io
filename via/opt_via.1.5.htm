Hooks
General User Hooks
•
Global: At the beginning of all VIA program (e.g., global variables used in other slots).
•
Keyword Search: To amend or add data to product keywords.
•
Keyword Search After: After searching but before processing to sort or alter data from the search.
•
Shopping List After: To sort or alter the data from the shopping list build routine.
•
After Product Read: To sort or alter the data from the product read build routine.
•
Product Description: To add data to the product description lines separated with Char (124).
•
Product Web Description: To add data to the product web description lines.
•
Initial Display Data: To amend or add data when Product Display Information is built. Use values 38-40 of the variable INFO for the data. VIA does not put these fields on the UI. You must place these fields on the UI.
•
Final Display Data: To re-sort or amend data that is built but not sent to UI. When you remove or re-order data, update all corresponding multi-values.
•
Set Customer Info: Before the customer info is sent back to VIA in GET_CUSTOMER_INFO.
•
Set Ship-via Info: After altering the ship via data sent to VIA in GET_CUSTOMER_INFO.
•
User Hook Stock Status: To amend the display of the Stock Status field.
•
Product Lookup Before: During searching or retrieving shopping list data at the beginning of the Product Lookup build program.
•
Warehouse Load: To add logic to filter the display of warehouses in VIA.
•
Filter Products after Build Info: After the final selection of products to filter the data.
Inquiry User Hooks
These subroutines affect the Order Inquiry screens (Open Order, Order History, Invoice, and Quote).
•
Inquiries Initial Slot: At the beginning of the inquiries to initialize data.
•
Inquiries Header Data: During the processing of every document in the order inquiry screens.
•
Inquiries Header After: After all documents are processed in the order inquiry screens.
•
Inquiries Detail Data: During the processing of each line item in the order inquiry screens.
•
Inquiries Detail After: After each line item is processed in the order inquiry screens.
•
All Orders Inquiry Select: In all order inquiries replacing the current Select Criteria subroutine.
•
Order Pad Initial Slot: Before any processing at the beginning of the Order Pad logic.
•
Orderpad After: At the end of the Order Pad selection to filter orders.
Maintenance Hooks
•
Customer Maintenance Read: After the customer record is read to display data when the page loads.
•
Customer Maintenance Before: Before processing.
•
Customer Maintenance After: After processing but before the data is saved to the CUSTOMER file.
•
Update Cart: To amend or add data as the Update Routine processes each line item.
•
Update Cart Product Loop: To amend or add data as the Update Routine processes each line item, before updating the line.
•
Update Cart After: At the end of the Update Cart Routine.
•
Checkout Process Before DOE: Before the order is created to add/amend data.
•
Checkout Process After DOE: After the order is created to update any related order.
•
DOE Order Create Product Loop: During the product number loop while creating CONVERT.DATA records for DOE.CONVERT.
•
DOE Order Create Product After: During the product number loop before saving CONVERT.DATA.LINE.
•
Checkout Before Screen Loads: To change shopping cart data in GET_CHECKOUT_DEFAULTS.
•
User Hook Convert Customer: From Convert Customer in Web User Maintenance.
•
Before Processing Order: Before processing the order.
•
Calc Sales Tax At Checkout: To non-standard sales tax calculation (contact VIA support).
•
Registration Before Write: In the Registration page before the record is saved.
•
Check Cart: Before going to the checkout page.

Web Password Parameters
Allows you to determine password requirements.
•
Alpha/Numeric/Combination: Indicate if passwords are (A)lpha, (N)umeric, or a (C)ombination or both.
•
Min Alpha Reg: The minimum number of alphabetic characters required (0 – 99).
•
Min Numeric Reg: The minimum number of numeric characters required (0 – 99).
•
Min Length: The minimum length for passwords.
Check any of the following to indicate that:
•
Allow Same As User: User IDs can be passwords.
•
Allow Spaces: Passwords can contain spaces.
•
Case Sensitive: Passwords are case-sensitive.
•
Case Sensitive User Id: User IDs are case-sensitive.
•
Allow Forgot Password Option: Users can be emailed a forgotten password (must use Optio or VSIFAX).
Web Properties
This screen enables you to configure parameters for the web site.
•
Website Base Url: The URL of the web site.
•
Domain: The domain name that pertains to the website in this field
•
Hostname: The name of the host server in this field.
•
Baseaddress: The base address from which the VIA process will begin.
•
Imageaddress: The location where the images for the web site are stored.
•
Thumbnail Address: The location where the thumbnail images for the website are stored.
•
Advertise Image: The location where the advertising images are stored.
•
Advertising Timer: The number of milliseconds the advertising banner appears before displaying the next banner (1000 – 999999999).
•
Guest User Verbiage: The text that appears when guest users click Add To Cart.
•
Logo Gif: The name of the logo image (*.gif) for the web site.
•
Logo Destination: The path to the logo image (*.gif) for the web site.
•
Logo Overtext: The text that appears the cursor hovers over the logo (1-30 characters).
•
Splash Gif: The name to the default splash image (*.gif) to appear when a user logs in.
•
DataBase Listener: If using Supply Chain Prelude generated outbound transactions, the listener that services transactions sent to remote system.

Printer Administration
This document will explain how to use the "Printer Administration" and "Hold Printer Administration" programs for Unidata customers on Unix. to use this tool, the Unix user must have the `printq' group listed in the Group Sets field of their Unix user. When executed, the process will list all System Builder printers that are set up with valid Unix queues along with the printer description, Unix queue, and status.
Printer Administration (/PRINTER.ADMIN)
This screen was developed to make control of Unix printers more user friendly. With this program, jobs can be deleted, held, released and moved.
•
F2-Finish: Exits screen
•
F5-Refresh: Will refresh the status of each printer.
•
F6-Enable: Executes the Unix `enable' command on the specified printer.
•
F7-Disable: Executes the Unix `disable' command on the specified printer.
•
F8-Jobs: Displays a queue of current jobs including the file path, job owner and job status as shown below. From here jobs can be deleted, held, released and moved.
The following functions can be accessed within the Jobs screen:
•
F2-Finish: Exits screen
•
F4-Delete Job: Executes the Unix `qcan' command to delete the specified job.
•
F5-Hold Job: Executes the Unix command to change the job status to HOLD.
•
F6-Release Job: Executes the Unix command to release the job and change the status to QUEUED.
•
F7-Move: Executes the Unix command to move the job to another Unix Queue
•
F9-Refresh: Refreshes the screen to update the current list of jobs
Note: To narrow list of printers, append a number to /PRINTER.ADMIN. For example, to only see System Builder printers that start with 5, type "/PRINTER.ADMIN,5" or place it on a customer menu.
Hold Printer Administration (/HOLD.PRINTER.ADMIN)
The Hold Printer Administration screen was created to aid in the management of the _HOLD_ file. This screen will show the hold job number, record size, created date, user who created the job and the heading of the report. The heading column may not always show a heading depending on the how the report was set up.
•
Owner: Enter the name of the owner of the job to limit the list to only jobs for a user, or leave blank to include all users on file. This will bring up the following screen:
Hold jobs can be deleted by using the slash (\) key.
The following functions can be accessed from this screen:
•
F2-Save: Exits the screen
•
F5-Review: Each job can be reviewed.
•
F6-Search for String: Will search for and display occurrences of a string of text.
•
F7-Last 20 Lines: Provides quick access to the `Grand Total' information.
•
F9-Print Job: Prompts the user for a desired Unix queue and then executes the UniData `SPOOL' command.

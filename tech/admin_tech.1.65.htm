Logging Off of a Customer’s Computer
Whether debugging a problem, helping a customer perform a process, or loading modified or new software, it will often be necessary to dial into a customer’s computer over the modem line. Logging off when finished seems rather elementary, but can save the customer from fraud, vandalism, or accidental damage when done properly. Logging out should always end the session completely, so that the next person dialing in must enter a login ID and password. A clean exit of the system ensures that the application closed and the customer’s database is secure.
•
Exiting the Application Software - The first step is to exit the application software. Always go back to a menu by hitting the Escape Key (to exit) or the F2 Key (to save). Once at a menu, hit “0” (zero) to exit System Builder and go to TCL (ECL on UniData).
•
Exiting UniData on Unix - At ECL, type “EXIT” <return>. This will exit UniData and go to the Unix shell. To exit Unix, type “exit” <return> or CTRL “D”. Hang up the modem.

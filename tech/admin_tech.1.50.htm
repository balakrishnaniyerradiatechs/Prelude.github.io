PrleudeFTP Processing
This FTP Processing document will provide an overview on the set up process necessary for establishing the required information needed to use the FTP process for both Outbound and Inbound file transfers.
FTP Processing (/FTPP)
The FTP Processing screen enables the user to enter the necessary IP Address, File Names, and Path information that are required to process Outbound and Inbound transfers.
Path: Systems‡ Computer Utility‡ File Transfer‡ FTP Processing
•
Name: Enter the FTP name that has been created for this FTP transaction.
•
Description: Enter the description of this FTP process.
•
IP Address: Enter the IP address to use when retrieving inbound data or transferring outbound data.
•
User: Enter the Login user to be used for this FTP session.
•
Password: Enter the password.
Outbound Transfers:
•
File Type: Enter the internal file type that is to be used when sending data. Currently the valid options are 'D' Database or 'O' Operating system.
•
Database File Name: Enter the database file name that has been created for this FTP session.
•
Operating System Path: Enter the Operating system path that is to be used when sending data. It is required that the path name always begin with '\' (i.e \/tmp/ftp.data.directory).
•
External Path: Enter the external system-operating path to be used to send data to. It is required that the path name always begin with '\' (i.e \/tmp/ftp.data.directory)
•
Archive Path: Enter the operating-system path in which items transferred out will be archived with the record key as well as a date and time stamp.
•
Delete Data: Enter 'Y'es if the outbound data on this machine is to be deleted once this process has been performed successfully. However, only 'S' and 'D' file types can be deleted.
Inbound Transfers:
•
File Type: Enter the internal file type that is to be used when receiving in data. Currently the valid options are 'D' Database or 'O' Operating system.
•
Database File Name: Enter the database file name that has been created for this FTP session.
•
Operating System Path: Enter the Operating system path that is to be used to receive in data. It is required that the path name always begin with '\' (i.e \/tmp/ftp.data.directory). This is a Unix directory that is located on the local machine.
•
External Path: Enter the External system-operating path that is to be used to receive in data. It is required that the path name always begin with '\' (i.e \/tmp/ftp.data.directory)
•
Archive Path: Enter the operating-system path in which items transferred in will be archived with the record key as well as a date and time stamp.
•
Delete Data: Enter 'Y' to if the inbound data on the host side is to be deleted (if permission is granted) once the process has been completed.
F5-FTP Log
The FTP Log shows detailed information about transfer processes for a date.
F6-FTP Session
•
Session Name: Enter the name of the FTP process to be transferred. This session can be executed promptly or scheduled to be transferred at a future date and time.
F7-Item Lists
The Item Lists option allows a custom list of records to be established for the FTP Process. In addition to adding new FTP records items to the list, the user also has the ability of renaming existing FTP records with a different name. The functionality of using the '*' (asterisk) as a wild card before or after the FTP process being searched for, has been added to simplify the look up procedure when searching for a record. Important Note: Renaming a list item is ignored if used with the wildcard feature.
•
Item List: Enter the name of the FTP record that is to be renamed.
•
Rename List: Enter the name that the existing FTP record is to be changed to.
F8-Additional Info
•
Email Address: Enter the e-mail address of the individual to be e-mailed session results.
•
Email Option: The Email Option screen allows the user to determine which types of FTP process results are to be e-mailed to the address previously established. Enter 'F' if only Failed sessions are to be e-mailed. Enter 'S' if only Successful sessions are to be e-mailed. Enter 'A' if All sessions are to be e-mailed, or 'N' if you do not wish to send any e-mails pertaining to session results.
•
Sleep Time: Enter the number of seconds required to wake the external server to process a FTP session.

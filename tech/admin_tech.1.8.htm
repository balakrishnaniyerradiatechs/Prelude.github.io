PSI Copy - Copying Live to Test
This procedure copies your data from your live account to your test account for training and testing for Unix/Unidata users. Before going live on Prelude, copy your data and flags to train users with the most up-to-date information. Then copy your data for testing. Please read this entire document before copying data:
WARNING: Nothing should be running on source or target account of any kind, including background jobs, nightly processing, or any update to the system. No users should be logged into system other than the user issuing the copy command. Not enforcing this may corrupt or lose data in target account.
Verify that no processing is occurring, and stop and start Unidata immediately before PSICOPY is run.
Do not copy an account from Unix or NT level without verifying your intentions with Activant Inc. Copying from Unix or NT level corrupts data, greatly damages both accounts, and severely interrupts business.
To copy the data files:
1
From the LIVE account, type /TCL <enter>
2
At TCL, type: PSICOPY /ud/XXX-DATA /ud/TEST-DATA <enter> (XXX is your account name.)
After you press ENTER, you will see the list of files being copied to the test account. When the copy is finished, you will receive a message to check the copied files.
3
Log out of the live account and back into the TEST account to check the data.
Use the following steps in the test account after the copy prior to Golive.
1
Go into /WM and add TEST to the end of the warehouse description/name. This will print on any documents that print warehouse address (e.g., P/Os, Invoices, and Packing Lists) so you know they are from the test account and not the live account.
2
Check /WM‡F6 to verify that the documents are set to print to the correct printers for testing.
3
Go into COMPANY.MNT and add TEST to the end of the company description/name. This will print on any documents (e.g., reports) so you know which ones are from the test account.
4
In the TEST account, go to /TCL and CLEAR-FILE PLATFORM. This will take ALL the passwords off the TEST account. If you do not want to remove all the passwords on TEST, do not run this command.
5
Prior to GOLIVE, if you are using the continuous printer, make sure it is running after the copy. /CD shows if a continuous printer is active. You will probably need to start the continuous printer with each copy. /CONT.PTR.PROCESS starts and stops the continuous printer.
Note: If you are installing VIA, check with your consultant or the VIA department before you perform a psicopy.
Note: If you have 2 Versions of VIA (Live & Test are Not the Same Version of VIA. DO NOT RUN PSICOPY

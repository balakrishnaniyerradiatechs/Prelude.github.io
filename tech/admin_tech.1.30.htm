Host to PC Download
Note: When using any of the SBClient file transfer features, you must be out of Prelude menus and at TCL.
This function allows you to share files between your PICK or UNIDATA system and a PC using a variety of formats that extract and sort your records before they are downloaded. The Host to PC transfer is located under Transfer menu in SBClient, once this is started you will see the following screen:
•
Host File: The host file is the file on your PICK/UNIDATA system that you wish to download. If you know the filename you can type in the name or you can click the square on the right hand side of the screen for a list of files to choose from. Note: You can use the browse box to help you “build” each field in the download.
•
Dictionary: This prompt for dictionary will generally always be the same name as the host file prompt.
•
PC File: The name of the PC file that you wish to download data into (e.g., C:\DIRECTORY\FILENAME).
•
Select Records: This prompt can be answered a variety of ways:
-
Leave the field blank or place an “*” for all records in the file.
-
Enter id’s such as “001000100” “001000200”.
-
Build a select statement. The format of this statement will be similar to a TCL or ACCESS select statement that must begin with the word “WITH” and have each criteria separated by “AND WITH”, for example, WITH STATE = “TX” AND WITH CITY = “DALLAS”.
-
Enter a list name that was created in TCL in the format GET-LIST LISTNAME.
•
Sort By: Enter the fields that you wish to sort the download by in the format BY FIELD1 BY FIELD2.
•
Display Fields: These are the actual data fields out of the host file that will be downloaded to the PC. If you do not specify any display fields the entire file may be downloaded. If you are using the Fixed or Sylk formats you are required to enter display fields.
Format
•
Fixed: The file is downloaded in fixed lengths, which are determined by the length in the dictionary items.
•
CSV: (Comma separated value) Each field is separated by a comma and alphanumeric data is enclosed in quotes.
•
Tab/Space delimited: Same as comma delimited except the commas are changed to either tabs or spaces. Alphanumeric fields will not be enclosed in quotes (“ “).
•
ASCII: Will download the data in the same format as PICK/UNIDATA, with each attribute on a different line.
•

•
PC File: The name of the PC file that you wish to download data into (e.g., C:\DIRECTORY\FILENAME).
•
Host File: The name of the file on the host computer that data will be uploaded into. This file must already exist on the host computer, SBClient will not create this file for you.
•
Dict: The name of the file dictionary that field definitions will be used from. Usually the same as the file name.
•
Id: Determines how the id’s on the uploaded records will be created. There are three options for this:
-
Leave the id field blank to force SBClient to use the first field in the list as the definition for the id.
-
Specify a literal list of id’s such as “000100” “000200”
-
Use a literal string with a mask. For example using R.3 or R.### will create sequential id’s using R and a three digit number after it. R.## or R.2 will use R and a two digit number after it.
•
Fields: Data definitions from the file dictionary that tell SBClient where to store the data on the host computer.
Format
Fixed length transfer: The attribute number of the field definition will determine where in the host file that piece of data will be stored. The length of the field will determine which piece of data is extracted from the PC file. For example, the CUST.NUM dict has a length of 6 and points to position 1. The first 6 characters of PC data are loaded into attribute 1 of the host file. CUST.DESC points to field position 2 and has a length of 15. The next 15 characters of the file are loaded into attribute 2 of the host file. The record that looks:
000100CUSTOMER NAME CITY STATE
000200CUSTOMER NAME CITY STATE
Then the field list should look like this:
CUST.NUM CUST.DESC CITY STATE
 Field length:
CUST.NUM 6
CUST.DESC 15
CITY 10
STATE 10
•
CSV transfer: For this type of transfer you must list the fields in the order that they are on the PC. The field position/subposition will determine where that data is loaded on the host file.
•
Transfer mode: Allows you to choose what happens if the destination host file already exists. The only 2 options are overwrite or preserve. If overwrite is chosen the entire record will be overwritten by the upload, there is no way to update only a attribute in a file.

Version 18 and Higher
You can use Btree indexes to process entries, lookups, and reports more quickly in Prelude. The standard software comes with a set of defined indexes that you cannot delete. You can add fields to files and enter new files. The CONTROL record has a list of existing indexes.
Creating Indexes (/REGEN.INDEX)
Path: System Parameter > Processes > Create Index Procedure
•
Filename - The files for which indexes exist. You can add files if necessary..
•
Rebuild - Indicates if that file’s indexes should be rebuilt: Y - rebuild indexes for the file; N - do not rebuild indexes for the file (default).
•
F2-Process - Creates the Btree indexes for the files with Rebuild set to Y.
•
F5-Rebuild All - Sets Rebuild to Y for all files. Press F2 to rebuild all indexes.
•
F6-Review Indexes - Shows standard indexed fields for a highlighted file.
•
F7-User Defined Indexes - Shows user-defined indexed fields for a highlighted file.
Example: Adding Indexes to ORDER.HISTORY
Choose System Parameter > Processes > Create Index Procedure. The Create Indexes screen appears.
1
Click on ORDER.HISTORY under Filename and press F6 to view existing indexes. The Review Index Items screen appears.
2
Since the two fields you want to index, INV.DATE and INV.NUM, are not listed, press F2 to go back to the Create Indexes screen.
3
Press F7 to view the User Defined Indexes screen.
4
Type INV.DATE, press ENTER, type INV.NUM, and press F2.
5
To view all index fields, press F6 to open the Review Index Items screen. Your new definitions appear with [User] added to the end of the field name.

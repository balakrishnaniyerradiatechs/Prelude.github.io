Optio Fax Confirmations
When faxing documents using Optio, the system normally sends the sender of the fax an email confirming the fax transmission. The email tells the user information such as whether the fax was actually received or not, the type of document sent, the fax number, etc. A valid email address must be established for the sender in User ID Maintenance. The number of emails sent to a user can be quite large when running a process such as batch invoice or statement faxing. To avoid this, custom programming is available to send all fax confirmation emails to a report file instead of individual users’ email addresses. The report and purge options described in the document are standard in the software. To use the report file for fax confirmations, enter a modification request.
Optio Fax Confirmation Report (/FAX.CONF.RPT)
This report will provide information on the fax confirmations sent to the file. The report sorts by document type or status. This report may take awhile to run as the system processes all fax confirmations sent to the report file since the last time the report was printed. Invoice numbers, purchase order numbers, etc., do appear if they were generated by batch. If the documents were generated individually, the document numbers are updated.
Path: Order Processing > Reports > Miscellaneous Reports > Optio Fax Confirmation Report
In V16+, a prompt is displayed to inform the user that incoming email confirmations exist on the system.
Note: A backup file must be used for this prompt to be displayed.
If 'Yes' is selected, the incoming fax confirmation emails are processed. The percentage completed of processing the incoming emails is displayed on the bottom of the screen. Once the percentage completed = 100%, the report is available for printing.
If 'No' is selected, the incoming fax confirmation emails are not processed at this time. The report is available for printing. The next time the user selects the report, this prompt will be displayed again to allow the incoming fax confirmation emails to be processed and included in the report.
•
Date: enter the date to limit the report to a date or leave this field blank to include all dates.
•
Document Type: enter the document type to limit the report to a document type or leave this field blank to include all document types. Select ‘CA’ for credit application, ‘CJ’ for customer job invoice, ‘CL’ for collection letter, ‘DM’ for debit memo, ‘IN’ for invoice, ‘PO’ for purchase order, ‘QT’ for quote, ‘RG’ for RGA, ‘AC’ for acknowledgement, ‘ST’ for statement, ‘VC’ for vendor claim, or ‘VR’ for vendor request for quote.
•
Buyer: enter the buyer number to limit the report to a buyer number or leave this field blank to include all buyer numbers.
•
Vendor: enter the vendor number to limit the report to a vendor number or leave this field blank to include all vendor numbers.
•
Sender: enter the sender number to limit the report to a sender or leave this field blank to include all senders.
•
Status (A)ll or (F)ailure: enter ‘A’ to include all fax confirmation statuses or ‘F’ to include only failed statuses. Failure statuses would include the following: Busy Signal on Line, Connection Failed, and T.30 protocol error.
•
Sort By (D)ocType, (S)tatus: enter ‘D’ to sort by document type or ‘S’ to sort by status. If this field is left blank, the report will sort by date.
Optio Fax Confirmations Purge
This purge routine will remove all Optio fax confirmation records from the CONF.FLAT.FILE up to the starting date entered. This process should be run weekly or monthly depending on the volume of faxes normally sent out since this file can become very large. The Purge Log Report has been updated with Purge Type 36 (Fax Confirmations) to limit the report to only include purged information for Optio fax confirmations.
Path: System Parameter > Processes > Purge Utility Menu > Fax Purge Optio Fax Confirmations

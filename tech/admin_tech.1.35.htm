Programming a MC9060 RF Gun
Before you can start you will need information from the customer as to how they want the guns setup:
1
Make sure you have followed the instructions on how to setup your PC to program an RF MC9060.
2
Confirm the Server host IP (the Prelude server). This is often different to the IP address that Activant uses to connect to the server so what you need is the IP address that the users will be using to connect to the Prelude server.
3
Whether they will be using Static or Dynamic IP addressing to supply the guns with IP addresses.
4
Valid IP's for the guns if the guns will be using Static IP addresses (note this step is not required if they use Dynamic)
5
The Subnet mask (not always obvious!)
6
The IP address of the Default Gateway
7
The IP address of the DNS server
8
The ESSID. This is the Wireless ID. If the customer already has AP’s and Guns ask them to confirm this. If none is specified then the default ESSID is 101.
9
Will Encryption be used? The MC9060 provides WEP and Kerberos encryption. Note that the AP must be setup to match this.
The answers to these questions determine how much work you need to do! This is because of the problem mentioned below where the gun loses settings when cold-booted or it loses power for an extended period.
The rule is that If the customer:
•
does not require Static IP’s setup (steps 2 and 3)
•
requires the default ESSID (101, step 7) and
•
does not require Encryption
then the setup is significantly simpler and requires much less time to do. This is because you do not need to power-off IP addresses, which saves a lot of time.
If ANY of the above ARE REQUIRED - Static IP, non-default ESSID or Encryption – then you MUST do step “4. Power-off IP address issue” below. Otherwise all the settings you put in will be lost the next time the gun is booted or loses power. So it is very important to determine what the customer wants in advance before you start to program the guns.
Confirm the version of Wavelink TelnetCE on the gun and upgrade if required.
-
Place a battery in the gun, then place the gun in a programming cradle. Power up the gun (if not powered up automatically) by pressing the red button. You will be presented a “+” symbol. Using the stylus, run through the calibration steps (clicking on the cross and saving).
-
Now double-click on the TelnetCE icon to open it up, then on About… to see the version. If it is older than 4.25-21 you need to do this step. If not, move onto the next step. To load the newer version of Wavelink, open the shortcut to telnetconfig you created on the desktop to get the window below.
-
If you need to send the Wavelink application to the gun (because the one on the gun is older than 4.2521) then click the button marked Application & Config to install the files (note that it uses ActiveSync to load the files onto the Gun. Once the transfer is complete, go to the gun and run the Telnet session. Look at the version (About) to check that it is version 4.25-21. Prior versions have a bug which causes CAPS-lock to work incorrectly. This problem appears to be fixed in version 4.25-21.
NOTE: If you clicked on “Application & Config” you will not need to do step 3 below.
Configuring Wavelink on the RF Gun
-
On the gun itself, within TelnetCE go to Options/Configure/Host Profiles, and when prompted enter the password (which is “system”, by default). If you have followed step 1 you should already see a Profile called PRELUDE. Open this then rename the Profile with an alias which corresponds to the name of the company to whom the guns will be delivered (for example, if the gun is due to go to NTP, then make the alias name NTP. For Address enter the IP address of the host which corresponds to that customer (for example, if the customer is NTP, their IP address currently is 208.58.3.130). If you do not see any profiles in this screen check you have followed step 1 correctly. Or if the profile has been deleted, you can click on Add then add a new Profile. Within the profile, set emulation to VT220. Then click on Config then the Display tab, uncheck Vertical Scrollbar and Horiz Scrollbar. Click on Hide menu, press top left button (F14), press X. Click in the Hide keyboard, press the top right button, press X. Click OK, Save then Done.

KeyMacro 4200 =
KeyMacro 4300 = "&"
KeyMacro 4400 = "*"

Next you need to fix a problem with the gun forgetting the IP address when the battery runs completely flat.
Power-off IP address issue
There is a problem with the MC9060s which means that it will lose the static IP address or its encryption setup (if used) if the battery runs so low that the gun powers down or if the gun is cold booted. This procedure is for making an IP address allocation be retained even when the battery runs out. In order to make a Mobile Companion profile Persistent (stay through a cold boot) on a Gemini (MC90xx Series Terminal) you must create two customized registry files with the desired settings and place them in the ROM area of the terminal. One registry file configures the TCP/IP settings for the terminal and the other contains the rest of the settings.
Symptoms:
Once the desired Mobile Companion Profile has been modified, saved and selected, the terminal is cold booted or is allowed to rest out of charger long enough to lose all battery life: all settings are lost.
Cause:
This information is stored only in the RAM or Volatile memory of the terminal, which gets rewritten upon a cold boot or battery loss.
Solution:
The registry files will be created using the existing registry of the terminal. Configure the terminal with a Mobile Companion profile that you wish to make persistent. Be sure to make the right choices for this profile including, Profile Name, ESSID, Encryption Settings and IP settings. IMPORTANT: This example will provide guidance for using a Mobile Companion profile that uses DHCP as it’s IP scheme and either 40 bit WEP, 128-bit WEP or Open System (no encryption) as its encryption method.
Once the terminal is configured with the desired Mobile Companion profile, you’ll want to make an RF/Network connection using that profile to be sure it is activated.
1
After configuring the profile and connecting successfully with that profile, connect the terminal to the PC (with Embedded Virtual C installed) via an ActiveSync session. Once the terminal is configured with the desired Mobile Companion profile, you’ll want to make an RF/Network connection using that profile to be sure it is activated.
2
While the terminal is connected, run the Embedded Visual C++ application. From the Tools menu, choose Remote Registry Editor and select the PDT9000 device from the pop-up window.
3
Wait as the connection is being established.
4
Expand the directory tree for the PDT9000 device and select the following key:
HKEY_LOCAL_MACHINE/COMM/Netwlan1/Parms.
5
With Parms selected, choose “Export Registry File” from the Registry menu at the top of the page.
6
Save the file as a .reg file and name it so that you’ll remember it is the Parms key.
7
Now, select the following key: HKEY_LOCAL_MACHINE/SOFTWARE/Symbol Technologies, Inc/Profiles.
8
With Profiles selected, choose “Export Registry File” from the Registry menu at the top of the page.
9
Save the file as a .reg file and name it so that you’ll remember it is the Profiles key.
10
WARNING: Once these files are on your PC, do not double click them. Always right click to see a menu and choose to edit with notepad. Double clicking will merge these files with your PC’s registry and might cause bad things to happen.
11
Wait as the connection is being established.
12
On the PC, and edit the exported registry files as follows:
-

-
Edit the registry file by adding your own IP values for these four fields: Default Gateway, Subnet Mask, IP Address and DhcpWINS. At the end of this procedure you should have two registry files, which you have copied across to the Application folder on the gun, both names beginning PSI to make them easily identified.
Courier Font
This step is to provide the gun with the necessary font files required to display the horizontal and vertical bar characters which are used around the RF menu screens. If you do not do this step these bars will not be displayed correctly. This step is fast. Copy the two files “cour.ttf” and “CourierFont.cpy” from the PC to the gun in the Application folder.
Testing
You should be able to cold-boot the gun and when you go to the Mobile Companion to check the IP address matches the details you entered. Note: if the gun is using DHCP it will pickup an address at Prelude. If static IP it should correctly display all those details. Similarly you should see any changes you made to the ESSID and Encryption. If not, you will have to redo the above steps.
Double-click on the Wavelink terminal emulator and hit enter. The gun should attempt to connect to the host server. Because that server is on the customer site, this will timeout, which is to be expected because the customer’s ADS server is not accessible from the Prelude network.
If you wish to further confirm all is well, you can create a second Host profile for the development server at Prelude and try connecting to that. But this will only work if the gun has a valid Prelude IP address, so usually it is more practical just to test that you can run the terminal application, rather than changing all the settings to make it work at Prelude then changing them all back to the customer settings. But this can be useful for diagnostic purposes.
This completes the programming of a 9000 series gun.

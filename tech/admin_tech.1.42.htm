VSI-Fax FAQs
Q. What version of VSI-Fax am I running? What is my serial number for VSI-Fax?
-
A. At the Unix level, log in as root and ENTER vfxadmin license -I
Q. How do I know if VSI-FAX is running?
-
A. At the Unix level, log in as Root and enter the following command vfxstat-r or vfxstat | more (for more details). A message displays if VSI-Fax goes down. 
The Server on the F80 has been operating since 05/11 at 20:00.                        
                                                                   
Device       Comments                  State                           
-----------  --------------------      -------------                   
C/S          <none>                    		Idle                            
fax01        <none>                    	Idle                            
lb           LoopBack                  	Idle                            
lcr          LCR hold queue            	Not running [Holding queue only]
sm           SendMail                  		Idle                            
 
This example represents how the fax device appears to determine if the device is idle or in the process of sending. If State is anything other than Idle or Sending, please notify Activant.
Q. How do I stop, start or restart the server?
-
A. At the Unix level, log in as Root and ENTER one of the following commands:
To stop - vfxsched start
To start - vfxsched start
To restart - vfxsched restart
Once the command is completed, it will return with the message "Done".
Q. Where does VSI-FAX reside on my Unix box?
-
A. VSI-FAX resides in the /usr/vsifax3 directory.
Q. How much free disk space do I need for VSI-FAX to run?
-
A. VSI-FAX needs at least 60 mb free in the /usr directory to run. Prelude suggests that at least 100 meg be free. If statements are to going to be run, it should be taken into account the extra large jobs they produce. Therefore, Prelude suggests having at least 200 meg free if running statements.
Q. What do I do if my disk space in /usr falls under 60MB?
-
A. A purge script is available within the /usr/vsifax3 directory. To run this purge, ENTER /vfpurge at the command line. The script will display the process in which to shut down the system. Running this process will also prompt the user to enter the number of days that the information is to be kept. Prelude recommends keeping the information a maximum of 7 days. The script is generated will prompt the user to restart once the purge process is complete. In addition, the script can also be added to the 'Cron' tab for the script to be run at night.
Q. Where are the VSI-FAX logs kept?
-
A. VSI-FAX logs are kept in two separate areas, /usr/vsifax3/spool/logs and /usr/vsifax3/spool/dbs.
Q. How do I send a test fax at Unix?
-
A. At the Unix level, log in as Root and ENTER the following command:
vfx-n <fax number> /etc/motd
 Once this command has been completed, the job number that VSI-FAX assigns is received.
Q. How do I send a test e-mail at Unix?
-
A. At the Unix level, log in as Root and ENTER the following command:
vfx-n <email address> /etc/motd
Once this command has been completed, the job number that VSI-FAX assigns is received.
Q. How do I cancel a VSI-FAX job?
-
A. You can cancel the job from Via/VF in Prelude or at the Unix level. From the Unix level, log in as Root and ENTER the following command:
vfxcancel <job number>
Q. Where is VSI-FAX turned on in Prelude?
-
A. The VSI-FAX option is located in the /COMPANY.MNT> F6-Modules> Fax Integration screen.
Q. Where do I setup email confirmations for users?
-
A. All users should have an e-mail address in User Id Maintenance (/USER.ID.MNT). All confirmations for documents (except P/Os) go to the sender's e-mail address. P/O confirmations are sent to the buyer's e-mail address. Check Buyer Maintenance (/BM) to view which users are assigned to which buyers. In addition, you can use fax control maintenance to setup blanket control of confirmations for each document type. If an e-mail address exists in VSI-FAX options, confirmations are automatically sent to that address. If the system cannot find a valid e-mail address in any of these locations, the confirmation is sent to the e-mail address in the fax integration screen in Company Maintenance.
Q. Where are the VSI-FAX confirmation logs?
-
A. The confirmation logs are stored in /usr/vsifax3/spool/logs under vmail.
Q. What do I do if VSI-FAX is running but not faxing?
-
A. Stop and start the VSI-FAX scheduler.
Q. What if I stopped and started it and it still isn’t faxing.
-
A. Check the disk space and phone line.
Q. What if I stopped and started it and the phone line is working.
-
A. Reset the VSI-FAX modem to confirm that it is or is not working correctly.
Q. What if I want to change the information on the cover page?
-
A. The information on the coverage is generated from the information in Company Maintenance (/COMPANY.MNT). Therefore, any changes made in Company Maintenance will be reflected on the cover page.
Q. What if we change our exchange server address?
-
A. You can edit the IP address and server host name in the vsisrv.ini file in the /usr/vsifax3/lib directory. After this is completed, restart VSI-FAX.
Q. Where do I get VSI-FAX documentation? What if I need to download the VSI-FAX software again?
-
A. VSI-FAX is supplied by Esker’s. You can get documentation, contact Tech Support, download patches and utilities as well as knowledge base articles.

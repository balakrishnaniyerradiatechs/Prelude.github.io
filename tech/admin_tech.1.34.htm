SBClient FAQs
Exporting Data from Host to PC
1
Go to TCL.
2
Choose Transfer > Host to PC.
3
Enter data on the screen that appears. Host file and dictionary file are the same. PC file is where to save the data. Select records specify which records to transfer. For Format, use fixed length (for Excel, use CSV format or a Smart Query). click Start Transfer. If the transfer succeeds, 800 appears, followed by complete. If an error occurs use the LIST command with the file, dictionaries, and selection criteria in the transfer to see what the output is. If that fails, the data specified may not exist.
Exporting Data with Smart Query
1
At TCL, type SBCVERSION to verify that the first 3 numbers of the client version are the same as the host version (e.g., 5.3.7 in the figure below).
2
Choose Utilities > Smart Query.
3
Enter data on the screen that appears. Dict file and Data file are the same. Select specifies which records to transfer. For Output, choose how to transfer data (e.g., Excel). Run the query to the screen at least once before using Excel. To use Excel, make sure that it is open, then click Run Query.

Cleaning TUXFER Files
1
At TCL, type TU.CLEAN. If a message appears about a TUXFER file choose No.
2
Type SELECT VOC = “[TUXFER]” and press ENTER.
3
If any files are found, type DELETE VOC and press ENTER.
4
From UNIX, type cd /ud/<account> and press ENTER.
5
Type ls *TUXFER* and press ENTER.
6
If any files are found, type rm *TUXFER* and press ENTER.
964 ERROR
1
Manually enter the Access statement in the Host-to-Pc/Smart Query at TCL.
Example: To download the City, State and Zip from the customer file, yet you are getting a 964 error, type LIST CUSTOMER CITY STATE ZIP at the TCL level. This will determine if the errors are generated at TCL. If so, the error will need to be corrected on the Host file for the download to be successful.
2
Enter the syntax and spelling in the Host-to-Pc/Smart Query windows correctly. If this is misspelled or is not the exact syntax that was used in the Host-to-PC/Smart Query fields at TCL, an error will occur. If the problem persists, it is possible that a syntax issue has in fact occurred. Therefore, it is recommended that the user click on the box with the three dots to the right of the field to allow SBClient to create the fields and syntax.
3
It is also possible that the TUXFER.DATS files that handle the file transfers have become corrupt. It is recommended that the user checks or perform the following process.
4
At a TCL prompt, type TU.CLEAN and hit <enter>. This process will delete the TUXFER.DATA files. Each file is associated with a port. Each time port # 14 processes a download with Host-to-Pc/Smart Query, it uses the TUXFER.DATA.14 file.
If you still receive the error, contanct your system administrator remove all TUXFER.DATA pointers and files from both the VOC/MD entry as well as the Unix account directory.

SBClient Setup in GUI
If your GUI session seems to be slow, check these SBClient settings:
1
If you are in a GUI screen, right-click the icon in the top left of your window and click Switch to Terminal Display.
2
On the main SBClient screen, choose Setup > GUI Parameters.
3
Choose one of the following:
-
Check Pad TCP Packets and set it to 1500 unless you are using XP and connecting to Prelude from a remote site (non-VPN).
-
If you are using XP on a remote connection (non-VPN), uncheck Pad TCP Packets.
4
Click OK.
5
On the main SBClient screen, choose Setup > Communication and click Advanced.
6
Uncheck Enable Echo and Buffered Output.
7
Click OK twice.
8
Choose File > Save Configuration.
9
To get back to your GUI screen, right-click the icon in the top left of the window and click Switch to GUI.
201/202 Host Item Not Found
If you receive this error message, one or more of the records may be corrupted. Contact your system administrator to establish if any files being accessed contain null or corrupted records.
Y2K Check
'Possible Y2K problem! To disable this message, add a record called Y2KCHECK into the TUDEFN. Place a 0 in attribute 1.' In the more current versions of SBClient, you can search for Y2K compatibility for any date fields. This process should be disabled. The system administrator should add a record called Y2KCHECK in the TUDEFN file. Within the TUDEFN file, place a 0 (zero) in attribute 1.
Cannot Initialize ROC
If you are receiving this error message, this indicates that the version of SBClient on your PC does not match the SB+/System Builder on your host. to verify the version of SB+ currently running, type the command SBCVERSION at TCL. If the first two digits of the SBClient version do not match the first two digits of the SB+/System Builder version, Smart Query will not work. However, the Host-to Pc transfer will continue to work. Have your system administrator contact Prelude about the possibility of getting the utility versions to match.
Time Out Starting Server
If you are receiving this error message, it is possible that there is a bottleneck of data that is being dragged across the connection. Example: Typically, serial connections are slower than network connections therefore, we suggest that you wait and retry the process again later. If you are still having issues with this process, try logging off and back on.
General
If you receive an error while in either Host-to-Pc or Smart Query, click CANCEL to exit the download window. Then re-enter the Host-to-PC/Smart Query. Occasionally, the initial error causes the connection to hang. Therefore, exiting out and re-entering will reset the connection.

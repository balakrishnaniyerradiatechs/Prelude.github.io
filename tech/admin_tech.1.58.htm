Purging Data
Purge Utility Menu
Path: Systems > System Parameter > Processes > Purge Utility Menu
You should purge most periodically to delete old records and keep Prelude running efficiently. Each of the following purge options prompts for a cutoff date for deleting old records among other selection criteria

Quotations

Accounts Payable
OAP History Purge
Open invoiced accounts payable history records with an account balance of zero.
Receive History Purge
Historical receiving records.
Open Receive Purge
Open receivers 90 days or older for a specified warehouse.
Vendor Master Purge
Vendor master records with no A/P balance or open purchase orders. Works the same as manually deleting the records in Vendor Maintenance (F4- delete). The VEND and VEND.STAT files are the only files affected by this process.
Accounts Receivable
Accounts Receivable History
A/R History records for invoices for a specified date.
Cash Receipts History
Cash Receipts History information for a specified posting date.
Credit Collections Notes History
Credit Collection Notes for a specified entry date.
Transaction Audit
Removes all A/R Transaction Audit Inquiry records prior to the specified date. Once purged, customer records will be updated with a FWD indicating the accumulated balance of all A/R transactions were purged.
Customer Purge
Removes customer records with zero A/R or Order Balance for a saved-list name or criteria. Same results as deleting the customer in Customer Maintenance. Only the CUSTOMER and CUST.STAT files are updated with this purge.
BMW/Work Order
Work Order History
Completed work orders for a specified warehouse
Work Order Log
Historical work order log records and all related files.
Bill of Materials Log
Historical bill of material log records.
Warehousing
Transaction Audit File
Creates a balance forward for purged quantity.
Pick Label
Picking labels.
Pick Label History
Picking label history records and all related files.
Emergency Cycle Count Queue
Emergency cycle count queue records for the specified Manger id.
Warehouse Activity
Warehouse records.
Pick Route History
Pick route history, segment history and box number history.
Staging Order Batch
Moves completed BPTP batches into history file.
Staging Order Batch History
BPTP history records.
Cores
Pending Cores
Pending core credit records that were completely applied.
Core Bank
Core bank records that were completely applied.
General Ledger
General Ledger History
General ledger history records for the selected fiscal year.
Fax Purge
Fax Information
Fax information records.
Optio Fax Information
Optio fax confirmation records from the CONF.FLAT.FILE. This should be run weekly or monthly depending on the volume of faxes.
Miscellaneous
Employee Activity (time card) purge
Employee activity file records.
Pkg Content Purge
Package History Content records.
Saved Lists Mass Purge
Deletes records in the SAVEDLISTS file based on the number of days old or a list of report generated items.
Vendor Master Purge Error Report
Includes information on the vendor master records not deleted with the vendor Master Purge. Run this report after the Vendor Master Purge. The report is overwritten each time you purge the data.
Customer Master Purge Report
Includes information on the customer records that were not deleted with the Customer Purge. If Keep Customer Main Audit flag to Y in File Parameters, the Customer Maintenance Audit Reports list purged customer records.
Purge Log Report
Displays all purges that were run based the start date and time, end date, warehouse, and user id.

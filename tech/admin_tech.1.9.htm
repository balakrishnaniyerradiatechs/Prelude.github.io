Prestores
Prestore commands are an easy way to manipulate data within many different records of a file without editing each record separately (e.g., update the same attribute in many records of a file with the same data). The most common way to handle this mass replacement is to select the records you want to change and save these records in a save-list. Then EDIT the save-list to begin the prestore process. Following are examples of how to perform a prestore in a Unidata environment as well in a Pick environment.
Warning!! When running prestores, if the attribute you are editing does not exist, Prelude udpates the last attribute in the record. For example if your prestore should change attribute 210 and the last attribute in the record is attribute 205, the prestore updates attribute 205.
A prestore in unidata has the following syntax: =prestore.name `command.string`=prestore.name
The prestore is executed from the editor. For large amounts of data, run a select statement to get a save-list of the records for the prestore. Once you have the save-list, you can begin manipulating the records. On the first record you can define your prestore and then execute it. Below is an example of a prestore run on a save-list of records in the Fix file. This prestore will go to attribute 1 and replace the text CTVTEST with the text CTV. The prestore then saves the record and moves to the next record until all records in the save-list are updated. 
:SELECT FIX WITH F1 = "CTVTEST" 
2 records selected to list 0. 
>ED FIX

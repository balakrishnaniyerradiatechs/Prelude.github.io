Identifying Bad Record Keys
The error message .SELECT LIST ERROR. received sometimes when running a report can be an indication that there exists one or more records which have a bad key. The key would typically contain a multivalue mark or an extended character that causes the OE to treat the key as two or more separate keys. Using these subparts of the original key, the system is not able to read a record, resulting in these error message.
Steps to Finding a Bad Key
To find bad keys, create the following three dictionary items in the file with the suspected bad keys:
•
A dictionary item referencing the raw key value. Name it K.FileName 
•
A dictionary item referencing the key value converted for extended and nonprintable characters. Name it CONV.K.FileName.
•
A dictionary item comparing the above two dictionary items, returning either a .Y. (Bad Key) or .N. (Good Key). Name it BAD.KEY

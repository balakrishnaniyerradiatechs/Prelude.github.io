Create Unidata TEST and TEST-DATA Accounts from Tape
There are four major steps to adding a TEST account:
1
Check for amount of free disk space
2
Restore the data files
3
Change the VOC pointers to point to the correct data account
4
Reset permissions in the new account

4
Note the number in the 'Free' column where /ud is mounted.
5
Determine the size of your LIVE account:
# cd /ud
# du -sk LIVE (replace the word 'LIVE' with the name of your LIVE account)
The statement will return data similar to: 684452 LIVE
6
Then check to see how big your LIVE-DATA account is:
# cd /ud
# du -sk LIVE-DATA (replace the word 'LIVE' with the name of your LIVE data account)
The statement will return data similar to the following: 1027721 LIVE-DATA
7
Total the size of the LIVE and LIVE-DATA accounts: 684452 + 1027721 = 1712173
8
Subtract the total from the amount of 'Free' space to determine if you have enough space to add the accounts in /ud: 2687168 - 1027721 = 1154995
Restore the Data Files
If the answer is a positive number, you can create the new accounts. If the answer is a negative number, contact Tech Support to determine if more space or file systems can be added.
Creating the New Accounts
If there is enough space in /ud:
1
Insert your existing backup tape into the tape drive
2
# cd /ud
3
# mkdir RESTORE (the contents of RESTORE will move to TEST at the end of the process)
4
# mkdir RESTORE-DATA (the data in RESTORE-DATA will move to TEST-DATA at the end of the process)
**NOTE** The process below will be completed twice if you are creating both TEST and TEST-DATA. The first time through will be to create TEST, and the second time through will be to create TEST-DATA. The example below is based on creating TEST. When restoring TEST-DATA, replace 'RESTORE' with 'RESTORE-DATA'.
# smit sysback
5
Choose Backup & Recovery Options > Restore Data from a Backup Tape
6
Enter /dev/rmt0 Xmm Tape Drive
7
Enter Backup Sequence Number [1] (May take a few minutes.)
8
Select - Directory (all files within)
9
Search word or ENTER for full list [/ud/LIVE] (replace the word 'LIVE' with your live account name)
10
Move cursor to top item, /ud/LIVE, and press F7.

When complete, it should say successful in upper right corner.
11
Press F3 until you get back to a # sign.
12
Repeat the process to restore your LIVE-DATA account to RESTORE-DATA.
13
Once the accounts are restored to RESTORE and RESTORE-DATA, move the data to TEST and TEST-DATA:
14
# cd /ud/RESTORE
15
# mv LIVE /ud/TEST (replace the word 'LIVE' with your live account name)
16
# cd /ud/RESTORE-DATA
17
# mv LIVE-DATA /ud/TEST-DATA (replace the word 'LIVE' with your live account name)
18
# cd /ud
19
# rmdir RESTORE
20
# rmdir RESTORE-DATA
*** NOTE: The data account must be named XXXX-DATA for PSICOPY to function correctly ***
After Contacting Tech Support
Tech Support will restore the accounts so that you can move them into the proper unix directories:
1
# cd /ud/TEST/LIVE (replace the word 'LIVE' with your live account name)
2
# mv * /ud/TEST (may have to be done in multiple steps, i.e. A*,B*)
3
# cd /ud/TEST
4
# rmdir LIVE (replace the word 'LIVE' with your live account name)
5
# cd /ud/TEST-DATA/LIVE-DATA (replace the word 'LIVE' with your live account name)
6
# mv * /ud/TEST-DATA (may have to be done in multiple steps, i.e. A*, B*)
7
# cd /ud/TEST-DATA
8
# rmdir LIVE-DATA (replace the word 'LIVE' with your live account name)
Point VOC Pointers to TEST-DATA
To point to TEST-DATA, log into TEST and:
1
At TCL, type SELECT VOC WITH F1 = "DIR" "F" <enter>
2
ESEARCH VOC <enter>
3
String: '-DATA' <enter>
4
At the TCL prompt '>' enter AE VOC <enter>
5
Enter =P1 R99*LIVE*TEST*`FI` =P1 <enter> (replace the word 'LIVE' with your live account name)
6
View the list of VOC pointers on the screen. They should point to TEST-DATA.
Set Permissions And Ownership
*** Make sure you are logged into unix as root ***
# cd /ud
# chmod -R 777 TEST TEST-DATA
# chown -R root TEST TEST-DATA
# chgrp -R ud TEST TEST-DATA
Ensure Users Can Access the TEST Account
To grant users access to TEST:
1
From the SBMENU, go to Admin > Security> Group Security Setup
2
In the 'Group Id Code' field, type in 'ROOT'. Add 'TEST' to either the list of 'Accounts with No Restrictions' or 'Accounts with Restrictions', based on what you want your users to be able to access. Always start at the top of the tree when adding accounts. If you add an account to a group, the same account must be accessible from that group's parent. Proceed down the tree adding 'TEST' to the appropriate list of accounts for each group that you want to have access to the TEST account.

Creating New Accounts
Accounts are maintained on the system for each customer. As new customers are added, an account is created based on the customer’s name (typically 3-4 letters or the comapny name). New accounts will be duplicated from the most recent version of the software. All customers have an account on the Advanced Pick (F50) machine. Unidata customers,also have an account on the Unidata machine (G30). In the following example, a new customer account will be created for Franks Supply, Inc. The naming convention for new accounts uses the first letter of the company’s name, in this case FSI.
To create a new account on the Unidata platform, perform a copy function in Unix. Copy the main account directory, currently /ud/V14.0, to the new directory name. Once the new directory has been created, the Unix permissions need to be set. If the permissions were correct on the directory you copied from, the permissions should be correct on the new directory as well.
Log in as root *
Change directory to /ud cd /ud
Copy an existing directory to the new directory name cp –R /ud/V14.0 /ud/FSI
Change directory back to /ud cd /ud
Setup the correct permission for the new account. While in the /ud directory, issue the following commands.
chmod -R 777 /ud/FSI
chgrp -R ud /ud/FSI
chown -R root /ud/FSI

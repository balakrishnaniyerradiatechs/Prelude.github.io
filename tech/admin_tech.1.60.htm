Purging Customer Data
The Customer Purge process removes all customer records without A/P balance or open orders for the saved-list name or criteria entered. This purge option process is the same as manually deleting the customer master record within Customer Maintenance using the F4-Delete option.
Note: CUSTOMER and CUST.STAT files are the ONLY files that are updated during this purge process. All other files that pertain to the customer (Order history, A/R History, etc.) are NOT updated.
Customer Master Purge
This process will purge all customer master records from a saved-list or criteria entered that have no A/R balance or Order Balance (open sales orders). If the customer number exists in the BOOKS.CUST file (existing books) and the Books/Samples flag (Company Maintenance >F6-Modules screen) is set to 'Y', the customer number will NOT be deleted.
Path: System Parameter >Processes >Purge Utility Menu >Accounts Receivable >Customer Purge
When this process is selected, a warning message is displayed for reference to the user:
Once the user selects to continue with the purge process, the Customer Purge Utility appears with the follwoing section criteria:
•
Save-List Name: enter a save-list name that contains an external selection performed of the list of customers to be purged. The user would create this save-list file manually.
•
Description: enter the partial description of the customers to be selected for purging. This field does not apply if the Save-List Name field has a value.
•
State: enter the state to limit the purge routine to a state or leave this field blank to include all states. This field does not apply if the Save-List Name field has a value.
•
Zip Code: enter the zip code to limit the purge routine to a zip code or leave this field blank to include all zip codes. This field does not apply if the Save-List Name field has a value.
•
Start Date <=: enter the date the customer was established to limit the purge routine to only include customers established on or before a date. The Start Date field in Customer Maintenance > F5-Add Info screen is updated when the customer master record is created. This date is checked when a date is entered here. This field does not apply if the Save-List Name field has a value.
•
Current Year Purchases <=: enter the current year purchases amount to limit the purge routine to only include customers with current year purchases less than or equal to the amount entered. The current year purchases information for each customer can be viewed in Customer Maintenance > F8-Cy & Py $ screen. This dollar amount is checked when a value is entered here. This field does not apply if the Save-List Name field has a value.
•
Previous Year Purchases <=: enter the previous year purchases amount to limit the purge routine to only include customers with previous purchases less than or equal to the amount entered. The previous year purchases information for each customer can be viewed in Customer Maintenance > F8-Cy & Py $ screen. This dollar amount is checked when a value is entered here. This field does not apply if the Save-List Name field has a value.
If the purge process did not delete certain customers, the following message appears after the process is completed: "Errors Occurred During Deletion, Please Review Customer Master Purge Report!"
Customer Master Purge Report
This report includes information on the customer master records that were NOT deleted when the Customer Purge process was performed.
Path: System Parameter >Processes >Purge Utility Menu >Accounts Receivable >Customer Purge Report
This report is overwritten each time the Customer Purge process is performed. For example, if the purge process did not delete certain customers, the report would be updated with the customer numbers and reason why each customer could not be deleted (open balance). If the purge process deleted all customers successfully, the report would display blank as no customer numbers would need to be reviewed. It is recommended that this report be printed after the purge process is executed each time. If the customer was not deleted due to existing books, the reference “Books Exist for the Customer” is updated on the report.
Note: If the Keep Customer Maint Audit flag is set to 'Y' in File Update Parameters, the Customer Maintenance Audit Reports (Purchasing > Reports > Customer Audit Report menu) are updated with the customer numbers that have been deleted during the Customer Purge process.

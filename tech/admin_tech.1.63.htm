Protobase FAQ
Q: “Batch Functions Locked Out. Fix Error in Batch to release!”
The settlement will not be completed if an error occurs due to the timing out of a modem, the modem being down or if the Protobase software is corrupted. If this happens, a “lock” will be placed on Prelude to prevent that batch from being altered any further until the issue has been resolved.
During the settlement/batch close process, two reports will always be generated: The ‘Pre-Settlement’ report and the ‘Settlement’ report. The Pre-Settlement report is printed prior to actually closing the settlement to notify the user that the settlement is about to be closed. Once the settlement dials out, Prelude will prompt the user to print the settlement report.
If an error occurs during the settlement, a lock will occur, therefore displaying the error message in which indicates that a lock has been placed on Prelude. Immediately after selecting <ENTER>, the user will be prompted to print the second report. However, this does not indicate that the batch has closed. What will be displayed at the bottom of this second report is the error message from the Protobase software, identifying what the problem was that occurred. If the problem reported was in reference to the modem being timed out (which is a common issue within the Amex Processing Network), simply remove the lock in Prelude and attempt to close the batch later.
to delete this lock, go to TCL and type the following: 
   DELETE CONTROL PB.BATCH.LOCKOUT
Contact Prelude for assistance if any error messages other than the one listed is displayed.
For every failed attempt at settling, a new lock will be created on Prelude. Therefore, requiring the user to remove the lock once again. This lock will only take place at the time of settlement.
Once the lock is in place and is not removed, this error will continue to be displayed if a settlement or a post authorization is attempted. Pre-authorizations will continue as usual.
Q: Is Protobase Running?
Prelude only sells a Unix version of Protobase. To confirm it is running, type the following command in Unix:
ps –ef | grep pb
The following will be displayed if Protobase is running in ‘Demo’ mode:
*Note the “d” on the ./pbdirect process line
root	18220	108158	0	Mar 24	0:00	pbcom /dev/tty66 PBD 0 0
root	64152	1	0	Apr 23	0:00	./pbdirect . -qd
The following will be displayed if Protobase is running in ‘Live’ mode:
*Note there is not a “d” listed on the ./pbdirect process line
root	18220	108158	0	Mar 24	0:00	pbcom /dev/tty66 PBD 0 0
root	64152	1	0	Apr 23	0:00	./pbdirect . -q
A “d” listed on the ./pbdirect process line indicates that Protobase is running in DEMO mode. Prelude is designed so that at the time of each pre-authorization, post-authorization or settlement, Prelude will verify that Protobase is running. If Protobase is not currently running, the Protobase software will automatically be started by Prelude. Protobase will be started in either the Demo or Live mode based on a Prelude flag in Order Processing Parameters > F5 > Credit Bureau Auth Code > Live/test Account.

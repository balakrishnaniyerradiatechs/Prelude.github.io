File System Administration
As System Administrator, one of your most important tasks is to ensure that your files are properly sized and error-free. Therefore, you should check the file system:
•
Quarterly after your files are stable. If your files are not stable, check the file system as frequently as required.
•
If the system abnormally halts or has major reliability events, resize all accounts.
•
After you starting using a new module or upgrade your version of Prelude.
To check the file system:
1
Set up each account - define the account in SYSMAN. Refer to Defining Accounts.
2
Create file sets - identify which files you want to analyze and resize in each account. Refer to Creating and Editing the File Set.
3
Analyze the file sets - check files for corruption, typically due to system abnormalities, and generate the information needed to fix a damaged file. Refer to Analyzing the File Set.
4
View errors - list which files have errors. Refer to Guide Statistics Report.
5
Refine the file set - choose which files you want to resize. Refer to Refining the File Set.
6
Repair damaged files - fix the files with the fixfile utility. Refer to Repairing Data Corruption.
7
Log off users - if users are on the system, the data will be corrupted. Refer to Preventing Users from Logging On.
8
Resize files - resize the files you selected. Refer to Resizing the File Set.
9
Allow users to logon. Refer to Allowing Users to Logon.
10
Restore files - if the data cannot be repaired, you can restore data from your backups. Please call Activant Technical Support to restore data.
Best Practice
Analyze the files on a Thursday night so that you can call Activant Technical Support on Friday if needed. Fix the files on Friday and resize them over the weekend.
About SYSMAN
Prelude developed SYSMAN (a SYStem MANagement utility) to resize files and analyze data integrity. It is typically loaded in /ud/SYSMAN. To access the SYSMAN account, type /LOGTO SYSMAN. The SYSMAN menu appears.
To access the SYSMAN account, you must belong to an SB group with access to SYSMAN.
Defining Accounts
You must identify the accounts on your machine to resize files. To identify accounts:
1
Choose Maintenance > Accounts. The Account Maintenance screen appears.
2
For Account Name, enter the subdirectory that contains the UniData files or press F5-Locate Accounts to search for accounts. You can edit the generated list of accounts to remove any unwanted accounts.
3
For Path, enter the full path that contains the account subdirectory. Because System Builder does not allow a leading slash to be entered in a data field, the initial slash in the path name is implied.
Do not manually set up SYSMAN as an account in Account Maintenance. If you log into SYSMAN and attempt to resize it, you will lose data.
Creating and Editing the File Set
You must create and edit the file set when you initially configure SYSMAN and when you add files to the accounts (e.g., Prelude upgrades). To create or edit a file set:
1
Choose Resize Procedures > Build/Maintain File Sets. The Build File Set screen appears.
2
In File Set, enter the name of the set of files.
3
Enter a Description of the file set.
4
In File Name, press F3. The Locate Files for Analysis screen appears.
5
For Account, press F3 to see the accounts that are defined in Account Maintenance (refer to Defining Accounts).
You may leave Minimum Size and Maximum Size blank to find all files in the account.
6
For Minimum Size, enter the smallest file size (in bytes) that you want to include in the set. Useful for restricting analysis to only large files.
7
For Maximum Size, enter the largest file size (in bytes) that you want to include in the set. Useful for restricting analysis to only small files.
8
For Dict/data/both, choose one of the following:
-
A - select only data files.
-
D - select only dictionary files.
-
B - select data and dictionary files.
9
For Browse Files, choose one of the following:
-
Y - allows you to edit the list of selected files.
-
N - automatically uses all files that match the selection criteria.
Analyzing the File Set
To analyze files, SYSMAN uses a Unix-level UniData utility called guide. The utility analyzes hashed files, generates statistics, and suggests files for optimizing to ensure data integrity. The files in the following table are created when you analyze files. You can rename these files.
File
Description
GUIDE_ADVICE.LIS
Management advice about files that are poorly sized or corrupted.
GUIDE_ERRORS.LIS
Structural errors in the file set.
GUIDE_STATS.LIS
Detailed statistical information about the file set.
GUIDE_FIXUP.DAT
Damaged groups for use with fixfile, dumpgroup, or fixgroup.
Because you lose data in files over 2Gb, this report warns you if a file approaches 2Gb.
If these files already exist, a 6-digit timestamp is appended to the filename before a new file is created. The most current files do not have this time stamp. The system does not overwrite previously generated files. Therefore, you must periodically purge these files.
To establish a time to analyze the files:
1
Choose Resize Procedures > Run GUIDE on Selected Files. The Set Program Run Time screen appears.
2
Select a File Set to analyze.
3
For Starting Time, enter the time you want to start analyzing the files.
Carefully choose the time you want to analyze the files because if the files are changing while being analyzed, you may see warning messages for errors that do not exist.
4
If you want to keep users off the system while analyzing the files, refer to Preventing Users from Logging On.
If the process finds any errors or a file approaching 2Gb, a warning message instructs you to contact Activant immediately. You must:
•
Correct any corrupt files as soon as possible to contain the problem and minimize data loss. As with any file corruption, the last resort is to restore the corrupted file from the last good file save.
•
Change any file approaching 2Gb to a dynamic file.
Viewing Errors
The Guide Error Report lists the file name, account, number of errors, current file dimensions, suggested file dimensions, and the guide error log for files with errors. Refer to Analyzing the File Set for more information on finding errors.
Refining the File Set
You must refine the file set to allow the files to adjust for growth factors (defined in Maintenance > Default Factor) and to exclude files before you resize files. You must refine the file set even if you do not change anything. To refine files:
1
Choose Resize Procedures > Refine File Set. The Refine File Set screen appears.
2
Enter the name of the File Set you want to refine. The files in the set appear with an error count (should be zero) and the current dimensions of the file.
3
Edit the Suggested Module and/or BlkSz as needed.
4
Edit the GF% (growth factor) as needed.
5
To exclude a file from resizing, change Status to Y to mark it as done/skipped.
The function buttons are described in the following table.
Button
Description
F2-Save
Saves the file list, making these files available for resizing.
F5-Errors
Displays the output of GUIDE_ERRORS.LIS for the current file.
F6-Stats
Displays the output of GUIDE_STATS.LIS for the current file.
F7-Advice
Displays the output of GUIDE_ADVICE.LIS for the current file.
F8- Reset Status
Resets Status to null or N, which signifies that the file should be resized according to parameters in this screen.
F9-Resort
Resorts the list by account, file name, suggested modulo, or suggested size.
Repairing Data Corruption
Being able to fix corrupted data is not a substitute for good, reliable, regular backups!
If a file is corrupted, either Activant technical support or a qualified system administration should repair the damage. As the files are repaired, any data that cannot be fixed is deleted. Therefore, although you may repair a damaged file, data may still be lost. If data is lost, you may need to restore data.
To repair a file, you must be familiar with the layout of the account and file structures of the database and know how to:
•
Access the UNIX shell.
•
Move through a UNIX file system with the cd command.
•
List files with the ls command.
•
Remove UNIX files with the rm command.
To repair a damaged file:
1
Use the Guide Error Report to determine which files are damaged. Refer to Guide Statistics Report.
2
Go to the UNIX shell.
3
At the shell prompt, change into the directory with the damaged file.
4
Type guide file.name where file.name is the name of the damaged file.
5
Type ls GUIDE* and locate the GUIDE_FIXUP.DAT file.
6
Choose one of the following:
-
If GUIDE_FIXUP.DAT exists, the damaged file has errors and must be repaired. Continue to step 7.
-
If GUIDE_FIXUP.DAT does not exist, the file does not have any errors. Go back to step 1 for the next file.
7
Type fixfile -dtempfile -f to fix the damaged file. You may replace tempfile with any file name. If you need to edit this file, call Activant Technical Support.
8
Type rm tempfile to remove the tempfile.
9
Type rm GUIDE* to remove the guide output files.
10
Type rm -r file.name_dir to remove the temporary directory with the restored records where file.name is the name of the damaged file
11
Repeat steps 4-6 to ensure the that the file is repaired.
12
Repeat this process for each file that needs to be repaired.
Preventing Users from Logging On
If you log off before enabling logons, no one can logon. Please call Activant Technical Support for help.
When you resize files, all users must be logged off, including phantom processes (e.g., continuous port, report scheduler). If users are on the system, data may be lost. When you analyze files, you may prevent users from logging on so that the Guide Error Report does not show false errors. To ensure that users are logged off:
1
Bring UniData down and back up.
2
Login to SYSMAN twice in case the resize process aborts and logs you out when Inhibit Logons is activated. Then you can use the second login to Enable Logons (refer to Allowing Users to Logon).
3
Once all users are off the system, choose Resize Procedures > Inhibit Logons. The Inhibit Logons screen appears.
4
Make sure that SB and your user ID are listed in Non Inhibit Users. All other users will not be able to logon to the system. However, users already on the system are not logged off.
5
In Logon Inhibited Text Message For Users, enter a message that users will see when they try to logon.
Resizing the File Set
Users should not be on the system while files are being resized. If a record in a file is created, modified, or deleted while the file is being resized, the file will be corrupted. Therefore, you must schedule file resizing for when all users are logged off. Because SYSMAN separates the functions of analysis and resizing, you can prepare data for resizing and schedule the resizing for a time when the system is inactive, which is typically during the middle of the night or over a weekend.
Once you refine the file set and all users are logged off , including continuous printers and the scheduler (refer to Preventing Users from Logging On), you can resize files. To resize files:
1
Choose Resize Procedures > Resize Files. The File Resizing screen appears.
2
Choose a File Set to resize.
3
For Start Date/Time, enter the date and time at which you want to begin resizing files. Files are not resized before this time.
4
For End Date/Time, enter the date and time at which you want resizing to end. Files are not resized after this time. If you enter only a date, resizing ends at 12:01am that day or after the file being resized is completed.
Start Date/Time and End Date/Time allow you to reserve time during nightly processing for file sizing. However, they are optional.
5
Press F2. The following popup screen appears.
6
Choose one of the following:
-
Increased - Resize files that need to be larger than the current size. Use this option to ensure that files that have outgrown their current size are resized.
-
Decreased - Resize files that need to be smaller than the current size. Use this option to resize files after purging data.
-
All Files - Resizes all files on which guide was run. Use this option at go-live and after any significant data changes.
-
Cancel - Goes back to the main menu.
Allowing Users to Logon
After the files are analyzed or resized, you must allow other users to logon. Choose Resize Procedures > Enable Logons.

Account Save/Restore
A file save procedure will place on a backup medium a copy of all data with the exception of the database/operating system. The data can then be restored if necessary at a later time. This is a means of ensuring that what is contained in the database is accurate. In Unidata, the cpio command copies file archives into and out of a cpio archive. There are 3 mutually exclusive ways to select the action to be performed: copy in, copy out, and pass.
Copy In Mode
The command “cpio -i” (copy in) extracts files from the standard input which has resulted from a previously executed “cpio -o” command. Extracted files are conditionally created and copied into the current directory tree based on the options selected. The permissions of the files will be the same as those of the files from which they were copied. The owner and group will be the same unless the current user is the super-user. If so, the owner and group will be the same as those of the previous “cpio -o” command. If the “cpio -i” command tries to create a files that already exists, and the existing file is the same age or newer, cpio will issue a warning message and not replace the files. The -u option can be used to overwrite, unconditionally, the existing file.
Copy Out Mode
The command “cpio -o” (copy out) reads the standard input to obtain a list of path names and copies those files into the standard output together with path name and status information. Output is padded to a 512-byte boundary by default, or to a user specified block size (with the -B or -C options) or to some device-dependent block size when necessary.
Pass Mode
The command “cpio -p” (pass) reads the standard input to obtain a list of path names of files that are conditionally created and copied into the destination directory tree based on the options selected. When using the -i or -o options, if cpio reaches the end of a medium (such as the end of a diskette), and the -O and -I options are not used, cpio prints the following message:
To continue, type device/file name when ready.
To continue, you must replace the diskette and type the character special device name (for example: /dev/rdiskette) and press RETURN. Simply pressing RETURN causes the cpio process to exit.
Options
The following options can be appended in any sequence to the -o, -i, or -p options.
•
-a - Reset access times of input files after they have been copied.
•
-A - Append files to an archive. The -A option requires the -O option. Valid only with archives that are files, or that are on floppy diskettes or hard disk partitions.
•
-b - Reverse the order of the bytes within each word. (use only with the -i option)
•
-B - Block input/output 5120 bytes to the record. The default buffer size is 512 when this and the -C options are not used. This option does not apply to the -p option. It is only meaningful with data directed to or from a character special device.
•
-c - Read or write header information in ASCII character form for portability.
•
-C n - Block input/output n bytes to the record where n is replaced by a positive integer.
•
-d - Create directories as needed.
•
-E file - Specify an input file (file) that contains a list of filenames to be extracted from the archive (one filename per line)
•
-f - Copy in all files except those in patterns
•
-H - Read or write header information in header format. Always use this option or the -c option when the origin and destinations machines are different types.
•
-I file - Read the contents of file as an input archive. If file is a character special device and the current medium has been completely read, replace the medium and press RETURN to continue on to the next medium. This option is only used with the -i option.
•
-k - Attempt to skip corrupted file headers and I/O errors that may be encountered. If you want to copy files from a medium that is corrupted or out of sequence, this option lets you read only those files with good headers. Use only with the -i option.
•
-l - Whenever possible, link files rather than copying then. (use only with the -p option)
•
-L - Follow symbolic links. The default is not to follow symbolic links.
•
-m - Retain previous file modification time. This is ineffective on directories that are being copied.
•
-M - Define a message to use when switching media. When you use the -o or -I options and specify a special character device, you can use this option to define the message that is printed when you reach the end of the medium.
•
-O file - Direct the output of cpio to file.
•
-P - Preserve ACLs. If the option is used for output, ACLs if existing are written along with other attributes to the standard output. ACLs are created as special files with a special file type. This option should not be used with the -c option.
•
-r - Interactively rename files. If the user types a carriage return alone, the file is skipped. If the user types a “.” the original pathname will be retained. (Not available with cpio -p)
•
-R id - Reassign ownership and group information for each file to user.id. (ID must be a valid login ID form /etc/passwd). This option is only valid for the super-user.
•
-s - Swap bytes within each half word
•
-S - Swap halfwords within each word.
•
-t - Print a table of contents of the input. No files are created.
•
-u - Copy unconditionally (normally, an older file will not replace a newer file with the same name).
•
-v - Verbose. Print a list of the file names. When used with the -t option, the table of contents looks like the output of an “ls -l” command.
•
-V - Special verbose. Print a dot for each file read or written. Useful to assure that cpio is working without printing out all file names.
•
-6 - Process a UNIX system sixth edition archive format file. Use only with the -i option.
•
Format of each type with acceptable options:
-
cpio -i [bBcdkmPrsStuvV6 ] [-C bufsize ] [-E file]
-
cpio -o [aABcLPvV ] [-C bufsize ]
-
cpio -p [adlLmPuvV
Example:
On the G30, the backup/restore is done using the following commands:
To tape: find . -print|cpio -ocduvB > /dev/rmt0
From Tape: cpio -icduvB < /dev/rmt0

Debugging BASIC Programs
There are command differences associated with the Pick debugger and the Unidata debugger. This document will cover some of the basic differences in Pick and Unidata debugger commands. For more advanced debugging topics, please refer to the before mentioned help pages.
Debug Statement Format
The correct statement format to execute the debugger in both Pick and Unidata programs is as follows:
IF USER.ID = “XXX” THEN DEBUG
When the DEBUG command is encountered, execution of the program is temporarily halted, and the debugger is invoked. Another way to enter a debugger is to press the “break” key while a BASIC program is executing. Involuntary ways of entering a debugger occur if a runtime error is encountered, or if an “abort” statement is executed in the program.
Debugger Commands
The general layout of this section will be a Pick debugger command and its equivalent Unidata command, followed by a description.
E{n m}
Specifies a certain number of lines, n, to execute before returning to the debugger. In the Unidata debugger, m specifies the line at which to begin execution, defaulting to the current line.
V
Toggles on or off the display of the current source line when stepping through the program. When the C function is on, the line is displayed AFTER it has been executed. With the V function in Unidata, the line is displayed BEFORE it has been executed.
This command steps through the program by the number of lines specified by the E command. For example, if E1 is specified previously, <cntl>+J will execute 1 line of code (and display it if the C function has been specified). <cntl>+J with E2 and C specified will execute and display 2 lines of code, and so on.
With Unidata, the E command not only specifies the number of lines to execute, it actually executes the lines (and displays them if the V command has been invoked).
G{line number}
Exits the debugger and restarts program execution with the line specified and continues until the end of the program. The default beginning is the current line. You cannot specify the number of lines to execute.
\{variable}
Displays the value of any variable within the runtime code being executed. In both cases the operator is given the opportunity to change the requested value. If a <return> is entered at the "=" prompt, the symbol remains unchanged. Otherwise, any value entered is placed into the variable.
$
Displays the name of the program being debugged, and the current line number.
H
Displays a brief alphabetic listing of the available debugger commands.
END
Terminates the run-time portion of a BASIC program, exits the debugger, and returns control to TCL.

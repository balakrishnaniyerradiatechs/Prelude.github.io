The vi Editor
‘vi’ (which stands for ‘visual’) is a powerful and complex editor available in the Unix system. It is an interactive text editor designed to be used with a CRT terminal. It provides a window of about 20 lines long in the file you are editing. You can move the window up and down through the file. You can move to any part of any line on the screen. The additions and changes you make to the file are reflected in what you see on the screen. This editor has approximately 100 commands. The ‘vi’ editor can, however, be unfriendly to new users. Therefore, Pick and Unidata provide a way to directly access the Unix file through the file pointer. After the file pointer is created in the MD or VOC, the file can be edited using more familiar editors such as ‘ED’, ‘SE’, and etc.
File Pointers
File pointers for Pick and Unidata are in the following formats:
Pick
001  Q
002 
003  Unix full path to the directory that contains the file to be edited
An example of the Pick file pointer to the Unix file ‘/tmp/unixdir/unixfile’ would be
001  Q
002
003  /tmp/unixdir/
Unidata
001  DIR
002  Unix full path to the directory that contains file to be edited
003  D_VOC
An example of the Unidata file pointer to the Unix file ‘/tmp/unixdir/unixfile’ would be
001  DIR
002  /tmp/unixdir/
003  D_VOC
Environment Issues
The shell environment variable ‘TERM’ must be correctly set for vi to work properly. Use the shell command “echo $TERM” to display the current ‘TERM’ assignment. To change it, use one of these commands:
TERM=xxx;export TERM
or
export TERM=xxx
Where xxx is the actual terminal type you are using, i.e. vt100, wy60. vi is case sensitive. The command ‘a’ is different than the command ‘A’. Make sure that your <CAPS LOCK> is off when working in vi.
Starting vi
Before you can use any of the following commands, you must start vi using one of the following commands:
vi file_name
or
vi /Unix_path/file_name
vi recognizes both relative and absolute UNIX path names.
Modes
‘vi’ operates in three modes:
•
Command - the mode you’re in when you start vi. You can move the cursor, copy, delete, or paste text, and invoke the other two modes.
•
Text input - interprets all keystrokes, except <ESC>, as data. You can enter the text you want to store in the file. To enter this mode from the command mode, type ‘a’ or ‘i’. Press <ESC> to return to the “command” mode. These commands are discussed in detail later.
•
External - type a ‘:’ (colon) to access this mode and save your work, exit vi, or access other advanced features. When in doubt, press <ESC> a few times until you hear the beep.
Basic Commands
These commands can make your stay in vi much smoother. There are scores of other commands with their own uses, but for the occasional vi user, these commands will get you through most editing situations. It won’t necessarily be quick, but you can do all the necessary housekeeping you run into in text files.
Key
Action
h
Move one character to the left
j
Move down one line, in place
k
Move up one line, in place
l
Move one character to the right
You may be able to use the “arrow keys” on some terminals.
To enter text you need to enter text input mode by:
Key
Action
a
Enter "append" mode (e.g., add text at the end of a line)
i
Enter “insert” mode (e.g., entering new text before existing text)
<ESC>
Return to "Command Mode"
The ‘x’ command allows you to delete one character at a time. This command is only active in command mode.
Key
Action
x
Delete the current character
When you finish entering and editing text, you can save the file and exit vi with two “external” mode commands:
Key
Action
:w
Write the current file to disk (e.g., :w saves the file using the path and name from the command line you used to start vi. You can rename the file using ‘:w new_name’)
:q
Exit vi. An error appears if you have not saved the file.
:q!
Exit vi without saving
:wq
Save the file and exit vi.
Always preced “External” mode commands with a ‘:’ (colon). When you type the colon the cursor goes to the last line on the screen because many external commands also take arguments. Also, you can combine many simple “external” commands (eg., :wq).

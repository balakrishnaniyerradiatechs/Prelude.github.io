Continuous Printers
Continuous printers are print servers that allow documents to be printed automatically upon completion of the transaction. Each document is put into a queue when a transaction is complete. The continuous print process continuously checks the queue and then prints any available items. If not using continuous printing, the batch mode is used to manually print all of the designated documents at one time.
When using continuous invoice printing, the invoice is updated during the continuous printing process. In the Pick environment an actual port has to be reserved for the continuous printer. This port must be identified and have no devices attached to it. In the Unidata environment a phantom process is created for the continuous printer so no actual port is used. Any number can be chosen as the designated continuous port.
Continuous Printer Port Maintenance (/PTR.TABLE.MNT)
Path: Systems > System Parameter > Maintenance > Printer Set-up > Continuous Printer Port Maintenance
This process is used to set up continuous printer ports for use in printing forms and labels continuously.
•
Continuous Port: Enter the continuous printer number that was designated as the continuous port.
•
Description: Enter the description or the location of this printer.
•
Primary Function: Enter the primary function for this continuous port. Choose from the following:
-
I - Invoice printing.
-
J - Work order quote ticket printing.
-
L - Shipping label printing.
-
O - Purchase order printing.
-
Q - Quotation printing.
-
S - Product label printing.
-
T - Transfer ticket printing.
-
W - Work order pick ticket printing.
•
Form Queue Number: Enter the printer form number for the printer assigned to the primary function. This is where the mask prints.
•
Invoice Message: This optional field can be used to enter a message line to print on invoices when the continuous printer is used.
•
Quotation Message: This optional field can be used to enter a message line to print on quotations when the continuous printer is used.
•
Action Bar> User Fields: These can be used for custom programming for the continuous printer.
Warehouse Maintenance (/WM > F6)
Path: Systems > System Parameter > Maintenance > Inventory Control > Warehouse Maintenance> F6 - Printing
Documents such as invoices, quotations, purchase orders, etc., must print continuously for each warehouse.
Enter Y for each document type that is to print continuously. Enter the port number used to control continuous printing for this document and the form queue of the actual printer for this document.
•
F5-Cont Labels Setup: Allows for labels such as stock receipts labels, shipping labels, etc., to be setup with the continuous printer.
•
F6 - NP Printers: This function key was added for Versions 18 and higher to assign printers for Nightly Processing. (In previous versions, this was under Warehouse Maintenance -> F8-Printers).
Continuous Port Start / Stop (CONT.PTR.PROCESS)
Path: Systems > System Parameter > Maintenance > Printer Set-up > Continuous Port Start/Stop
Each time a document flag is changed for continuous printing in Warehouse Maintenance, the continuous printer must be stopped and started again in Continuous Port Start/Stop. Also, the continuous printer must be started after the system is taken down, rebooted or if there were any programming changes made to the printing program. to start/stop the continuous printer, enter the continuous port number.
After entering the port number, the Continuous Printer Procedure option is displayed.
•
Start Printer: Starts the continuous printer. The mask prints on the form queue from Continuous Printer Port Maintenance.
•
Halt Printer: Stops the continuous printer.
•
Exit: To not perform any stop/start functions for the continuous printer procedure.
Unidata Continuous Printers (/CD)
This program can be used to view the status of a continuous printer on a Unidata platform. It shows what phantoms are running and the process id of the phantom. If necessary, this process id can be used to kill the continuous printer.
When logging on to the continuous printer, it sets up the system Q pointers if not found. The program to set this up on Unidata is in PROGS CONT.LOGON.UDT.
When logging off a continuous printer on the Unidata platform, it writes STOP in attribute 1 of the QUEUE.ASSN file. This is removed when logging the printer back on to the continuous logon program. If it is necessary to debug through the continuous printer then the STOP in attribute 1 must be manually removed.
to debug through a continuous printer problem in Unidata, at /TCL type in CONT.ACCTNAME.PORT NUM. An example of this on the ATOZ Test account would be to type in CONT.TEST.057. Test is the account name and 057 if the continuous port number used on the test account. The program can then be debugged to find any problems. To see which document is currently printing: go to/TCL and type in LIST QUEUE.ASSN F1.
PICK Continuous Printers
to set up a continuous printer on the test account, there must be a port available without a device attached to it.
When logging on to the continuous printer, it sets up the system Q pointer if there is not one found. The Q pointer on the Pick system is set up in the process CONT.LOGON.
If the continuous printer stops working on the Pick system, use the WHERE command at /TCL to determine what the port is doing.
•
- If nothing displays then there is no process logged on to that port.
•
- If there is a D in the Stat column, then the continuous printer is in a debug state.
•
- If there is a rdl.getchar and brinput in the Stat Column then the continuous printer is waiting for some one to press return.
To debug through the continuous printer program
12
Go to /TCL.
13
Type in CONT.DRIVER.
14
At the ? prompt type in TEST.
15
Enter the company number.
16
Enter the printer queue number.
At this time the continuous print program can be debugged.
To see which document is currently printing: go to/TCL and type in LIST QUEUE.ASSN A1.

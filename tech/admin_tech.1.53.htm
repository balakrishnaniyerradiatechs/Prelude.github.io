Send Page Feature
Operator can send a message to a pager of a single user or of multiple users. Users being paged must have a Pager Id set up in User Identification Maintenance and their pager number set up in Unix. The message server must be running at the Unix level for paging to operate.
Message Server
The client is required to purchase paging software of their choice. Activant uses HipLink as the message server but another Unix paging software may be used. User should consult the Unix paging software they are using on how to set up the users and the pager numbers.
HipLink (for System Administrator only)
HipLink is the message server Activant currently uses. It resides at the Unix level so the user must shell out to Unix (!sh) to access it. At the Unix prompt type “hiplink”. HipLink is used to start and stop the message server and to maintain users. All users who can receive messages via a pager should be entered and maintained. The user’s pager ID, pager number, and service are required for this operation. This can be set up in the Config System > Users option. Using the example from Customer Maintenance, Pat Smith would be “Psmith|9319005|PageNet”. To exit type and save type “:Q”. To exit without saving type “:Q!”.
User Identification Maintenance
The Pager Id field (F6-Personal screen) must be established in User Identification Maintenance.
Path: System Parameter > Maintenance > User Identification Maintenance > F6-Personal
•
Pager Id: this field defines the user’s pager ID as it is used at the Unix level. This maintains a cross-reference between Pick where the user resides and Unix where the pager number is stored. For example: if user Pat J. Smith (PJS) has a message server ID of “Psmith” then “Psmith” would be the Pager Id for user PJS.
Send Page (/PAGE)
Type /PAGE to send a message to a pager number or multiple pager numbers.
•
User ID: this field defines the user(s) that will receive the message. If manually entering multiple users, the user IDs should be comma delimited. For example: if a message needs to be sent to users AJT, LJT, and KAB then “AJT,LJT,KAB” would be entered.
•
Message: this field is where the operator will type the message to be sent to the pager(s).
•
Sender ID: this field defines who sent the message. This is a free-form field that defaults to the operator’s user ID but may be overridden. This is sent with the message so the user receiving the message will know who sent the pager message.

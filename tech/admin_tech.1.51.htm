Convert Decimal Places
WARNING: Only execute this process when no users are on the system as it could corrupt data.
This process will convert decimal places in dictionary items and processes based on the Price/Cost/Qty values set up in System Parameters, and then will regenerate source code for the processes that previously had source code generated. This process will generally be executed on the client’s Live account and the TEST account will be updated using the COPY.DICT process. Please note this process should only be executed if the client’s price/cost decimals are different from the standard Prelude price/cost decimals.
Price/Cost/Qty Maintenance
This option is used to establish/maintain the number of decimal places that will be used for prices, costs, and quantities. Normally these decimals would only be maintained before a client goes live, changing the decimals after data has been loaded will only change dictionaries and processes and will have no effect on data.
This maintenance routine is located in System Parameters > Maintenance > Inventory Controls.
Convert Decimal Places Process
The Convert Decimal Places Process is basically the same for Unidata or Pick except for the prompts at the beginning. This process is located in System Parameters > Processes.
If you select “Yes” to continue, the next prompt displayed will be different depending on the system platform.
The prompt is: Enter the FULL PATH of the account for “-DATA” files /ud/DEV-DATA
You must enter the correct FULL path and case (upper and lower) in Unidata or the Convert Decimal Process will not run correctly. The correct path format should be /ud/ACCOUNT.NAME-DATA. ACCOUNT.NAME is the account where the process will run.
After you enter the correct path or account name, the process selects items to convert and begins converting dictionaries and processes. When the data is converted and the dictionaries and processes are updated, you must regenerate System Builder expressions. The Convert Decimal Places process automatically executes the REGEN.EXP process. Unless you want to baby sit the REGEN.EXP process error messages that appear, you can start the ‘Regenexp’ Script in SBCLIENT. To execute the script, choose Script > Run Script and highlight regenexp and click OK
When the REGEN.EXP process has finished, then you can stop the script by clicking Script > Stop Script.

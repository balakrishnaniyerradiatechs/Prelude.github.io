Host Library
Before using SBClient you must have the SBClient host utilities installed on your host computer. Any client on version 11.1 or greater of Prelude already has the host utilities; however, you must verifry the version of the host utilities matches your version of SBClient. To do this you will have to create and run a small BASIC program.
At TCL type:
ED TUBP VERSION.TEST
New item
Top
.
I
001+
CALL TU.GET.VERSION(VERSION,TEST)
002+
PRINT “SBClient: “: VERSION<1>
003+
PRINT “SBHost: “: VERSION<2>
004+
Press Return
Top
.
FI
BASIC TUBP VERSION.TEST
RUN TUBP VERSION.TEST
When you run this program the first line tells you what your PC version of SBClient is, the second line tells you what version the host programs are. If these 2 versions are different, you must install the host programs.
Note: Please consult Prelude before installing host utilities to the live account.
To install the host program, go to the Transfer – Host Program Installation menu in SBClient. Accept the default Source Directory of Host Software and choose the appropriate database (Advanced Pick or Unidata). Note that the host program installation process must be run from TCL.
If you have multiple versions of SBClient in use then you may not want to install the host utilities on your live account. Doing so will setup the host programs for that version and disable the host programs for users on other versions. Instead create a new account on the system called SBHOST.X.X where X.X is the version number, then log over to that account and install the new host utilities there.
Note: Most problems that we research about Host To PC, PC To Host and Smart Query data transfers are caused by incompatible versions of host programs and SBClient. If you are having problems with any of these three data transfer functions make sure that the host utility programs and the SBClient versions match.

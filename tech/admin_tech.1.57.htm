Scheduler Control Maintenance (/SCHED.CONTOL.MNT)
This option allows a user whose Schedule GRI Process flag is set to ‘A’ to maintain control records that govern the scheduling of reports.
Path: Jobstream > Scheduler > Scheduler Control Maintenance
•
Sleep Time: this field defines how long the phantom will sleep before it wakes up to look for jobs to run. In the above example, the scheduler will sleep five minutes between each run. If a relatively small time frame is entered, the system may get bogged down running phantom reports. Please monitor your system carefully till the timing is correctly set to meet your needs. If a time report like the Emergency Purchase Order Report needs to be ran quicker, you may want to set this at a smaller time frame. Running the scheduler every hour may not be soon enough to catch these items.
•
History Records: this field defines the number of old schedule records to keep in history. At this time these records are stored in history but there is not an inquiry to view them.
•
Max Phantoms: this field defines the number of jobs the scheduler will run at a time. When the scheduler “wakes up”, it determines how many jobs are still running. It will then check for jobs that need to be run and will queue however many it takes to meet the maximum number of jobs. For example, in the screen displayed above, Maximum Phantoms is set to 16 jobs. When the scheduler wakes up, it determines that there are 12 jobs still running and that there are 5 jobs that need to run. So the scheduler will start the next 4 jobs. Had there only been 3 jobs that needed to run, the scheduler would have started all 3 jobs.
•
Max Work Phantoms: this field works with the Work Begins and Work Ends fields to define how many jobs the scheduler will run at a time during the “Work” hours. When the scheduler “wakes up” during the “work” hours, it determines how many jobs are still running. It will then check for jobs that need to be run and will queue however many it takes to meet the maximum number of jobs during work hours. For example, in the screen displayed above, the phantom will only run 6 jobs at a time between the hours of 8:30 a.m. and 6:45 p.m. When the scheduler wakes up, it determines that there are 2 jobs still running and that there are 8 jobs that need to run. So the scheduler will start the next 4 jobs. Had there only been 3 jobs that needed to run, the scheduler would have started all 3 jobs. At all other times, the scheduler will run up to 16 jobs at a time.
•
Work Begins: enter the time of day when work normally begins at the company.
•
Work Ends: enter the time of day when work normally ends at the company.

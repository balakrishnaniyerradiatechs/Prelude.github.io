Lists
A list is an item containing selected item ID’s which can be retrieved and used to manipulate or report data. These lists are stored using a SAVE-LIST command. The lists can then be retrieved and used by means of other list commands. These commands include COPY-LIST, DELETE-LIST, EDIT-LIST, GET-LIST, and SORT-LIST.
Lists may be saved in a system table called SAVEDLISTS. Unidata displays ‘>’ rather than ‘:’ when the active list has data. If a list name is not specified for any list command the default list “0” is used.
A saved list that exceeds approximately 34,810 characters is saved in multiple parts. Each part has an extension to the specified list name beginning with 000 and incrementing sequentially. Although it is not necessary to specify each part of the saved-list record ID when the GET.LIST command is executed, you must specify each part if the copy command or the AE command is used.
SAVE.LIST
The SAVE.LIST command in Unidata serves the same function as in Pick. The format is:
SAVE.LIST{{listname}
Listname is the name of the list to be retrieved. If no listname is specified, the default of “0” is used.
Example:
SELECT CUSTOMER
40000 records selected to list 0
>SAVE.LIST CUST
40000 keys saved to 7 records.
LIST SAVEDLISTS = “[CUST]”
SAVEDLISTS
CUST000
CUST001
CUST002
CUST003
CUST004
CUST005
CUST006
7 records listed
Note: The “sample” command may be used in Unidata in the same way as “sampling” in Pick except that the “sample” command automatically selects a sampling quantity of 10.
GET.LIST
The GET.LIST command retrieves a select list previously saved with the SAVE.LIST command. Any process that can be executed after a select statement can also be executed after a GET.LIST statement. Records stored in a list can be retrieved repeatedly using a GET.LIST command. Once you retrieve a list with the GET.LIST command it is assigned the default list number “0” unless you assign a different list number. The format for this command is:
GET.LIST{{acct.name}list.name}
If the list does not reside in the SAVEDLISTS file in the current account, the acct.name parameter can specify its location using a full pathname. If GET.LIST is used with no parameters, default list 0 is retrieved.
COPY.LIST
The COPY.LIST command copies a saved list to another list on the same account. It can be used to store an entire list as a record in a Unidata file or send a saved list to the printer or terminal. The format is:
COPY.LIST old.listname new.listname {-options}
•
Old.listname is the name of the existing list and new.listname is the name of the list to be copied to. If new.listname is not specified, Unidata prompts for a TO list. There are 2 options for the TO list:
•
Destination list – the name of the list to which you are copying.
•
[DICT] (filename) {record.id} – the filename and record to which you are copying. If this option is used, the source list becomes a record in the specified file. If no record.id is specified, Unidata uses the listname as the record.id.
•
Options:
-
‘D’ Deletes after copying. This cannot be used with the ‘p’ or ‘t’ option.
-
‘N’ Suppresses auto paging. The display doesn’t stop until list end is reached. Only valid with the ‘t’ option.
-
‘O’ Overwrites if the list already exists.
-
‘P’ Sends output to the printer.
-
‘T’ Sends output to the terminal.
DELETE.LIST
The DELETE.LIST command deletes a list from the savedlists table. If the list does not exist or you do not have permission to delete the list, an error message will display. The list will be deleted without prompting for confirmation that you want the list deleted.
Examples:
DELETE.LIST CUST
‘CUST’ deleted.
If the list does not exist:
DELETE.LIST CUST
DELETE.LIST CUST failed.
EDIT.LIST
The EDIT LIST command will allow you to edit a saved list in the savedlists file. The list listname is opened in the editor. The data in the list can then be edited. If the size of the list is large, and multiple list records are present, the EDIT.LIST command will display the first part of the savedlist record, then prompt you to display subsequent parts.
Example:
EDIT.LIST CUST
CUST
Top of ‘CUST000’ in ‘SAVEDLISTS’, 2012 lines, 34,806 characters
00100001
00100002
00100003
Quit “CUST000” in file “SAVEDLISTS” unchanged.
Do you want to continue with CUST001? (y/n/q)
?
SORT.LIST
The SORT.LIST command sorts an existing saved list in record.ID order. The format is:
SORT.LIST listname {-options}
•
Listname is the name of an existing list.
•
Options:
-
‘R’ Sorts the elements in descending order.
-
‘N’ Sorts the elements according to their numeric values.
If the ‘R’ option is not used, Unidata performs an ascending sort. If ‘N’ is specified, NULL values are treated as zeros, otherwise they are treated as the smallest values in the list.
Example:
SORT.LIST CUST (R
CLEARSELECT
The CLEARSELECT command clears an active select list. This command is used at the active select list prompt, “>”. The format is:
CLEARSELECT
Example:
SELECT CUSTOMER
234  records selected to list 0.
CLEARSELECT
The CLEARSELECT can be used from a GET-LIST command as well.

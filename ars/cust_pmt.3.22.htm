Fast Cash Receipts Exception Report
After you update the batch (refer to Updating a Batch), any transactions that did not update will appear on the Fast Cash Receipts Exception Report. You can use High Volume Cash Receipts Maintenance (refer to Manually Matching and Editing Invoices) to adjust any un-posted transactions before running the update again. The Fast Cash Receipts Exception report must be empty before you can purge a batch (refer to Purging a Batch).
To view any update exceptions:
1
Select Financial > Accounts Receivable > High Volume Cash Receipts > Exception Report to open the High Volume Cash Exception Report screen.
2
For Beginning Batch and Ending Batch, enter the number of the first and last batch you want to view. Leave blank to view all exceptions.
3
For Sort Sequence, choose a sort method:
•
C - Customer.
•
B - Bank.
4
Press ENTER and choose an output.
 
Column Descriptions
Column Heading
Description
Reference Number
The reference number from High Volume Cash Receipts Detail Entry.
Amount
The amount to be paid toward the invoice.
Invoice#
The invoice that was posted to.
A/R Balance
The customer’s A/R balance.
Pay Error
If Check Error is 99, Pay Error indicates the source of the error:
•
03 - Underpaid.
•
04 - Overpaid.
•
05 - Amount was written off to the discrepancy account.
You must go to High Volume Cash Receipts Maintenance (refer to Manually Matching and Editing Invoices) to adjust any transactions.
Batch#
The number of the batch.
Seq#
The transaction number for a customer’s invoices.
Bank Code
The code for the bank.
Check#
The number of the customer’s check.
Check Amount
The amount of the customer’s payment.
Chk Date
The date the payment was issued.
Cust#
The customer number.
Description
The customer name.
Method Entry Type
MANU indicates that the batch was entered manually. See your System Administrator if you use other codes.
Check Error
If the check had errors, 99 appears.

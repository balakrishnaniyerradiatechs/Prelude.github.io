Administrative Manager Release Report
The Administrative Manager Release report lists all sales orders or quotes released from Administrative Order Hold, Administrative Quote Hold, or Administrative Invoice hold for an Administrative Manager ID. Refer to the Administrative Hold Inquiries in the Order Processing documentation for more information on administrative holds.
Admin Mgr Release in File Update Parameters must be set to Y to keep Administrative manager release information.
Menu Path: Financial > Accounts Receivable > Reports > Admin Manager Release
Selection Criteria
Use the following criteria to limit the information to print on the report:
•
Division. The highest possible grouping in your company (e.g., Eastern Division, Western Division).
•
Region. A subset of Division - if Division is used (e.g., Southeast Region, Northwest Region).
•
Department. A subset of warehouse. Typically used to separate expenses within a warehouse (e.g., accounting, sales, warehouse).
•
Cost Center. A subset of Department - if Department is used. Typically used to separate expenses within a warehouse or department (e.g., shipping, receiving, outside sales, inside sales).
•
Warehouse. The selling warehouse that receives credit for the sale, regardless of which warehouse ships the merchandise.
•
Admin Manager. The Administrative Manager ID for the released orders (e.g., AI1, AQ1, AO1).
•
Admin Type. The type of administrative hold:
-
I - Administrative Invoice Hold.
-
O - Administrative Order Hold.
-
Q - Administrative Quote Hold.
•
Released By. The user ID of the person that released the orders.
•
Released Date range. The beginning and ending dates that the orders were released from hold.
Column Descriptions
Column Heading
Description
Whse
The warehouse that sold the material.
Customer
The customer number.
Description
The customer description.
Type
The type of administrative hold:
•
I - Administrative Invoice Hold.
•
O - Administrative Order Hold.
•
Q - Administrative Quote Hold.
Document
The sales order, invoice, or quote number.
W/B
The user ID of the person that entered the order.
Amount
The value of the order/invoice/quote that was released from hold.
Hdr
The reason code to indicate why the order went on hold (refer to the “Administrative Hold Inquiries” in the Order Processing documentation). Codes in this column refer to the holds affecting the order header:
Administrative Invoice Hold reason codes:
•
A - Credit card order – no authorization.
•
C - Credit memo amount exceeded.
•
D - Sales order’s negative deposit exceeds user’s limit.
•
I - Customer always on invoice hold.
•
M - Manually placed on administrative hold.
•
P - Customer requires a P/O number.
•
Q - Customer invoice amount check.
•
S - Salesrep was changed.
•
T - No tax jurisdiction code.
•
X - Exceeds the pre-authorized percent.
•
0-9 - User defined codes.
Administrative Order Hold reason codes:
•
A - Automatic hold for conversion orders.
•
B - Backorder cycle allocations.
•
C - Conversion errors from DOE.
•
D - Override freight dollars.
•
E - Blanket P/O amount exceeded.
•
F - Override freight terms.
•
I - Incompatible ship via.
•
K - Oversized order amount exceeded.
•
L - An item was linked to an existing P/O, or a P/O was created through Sales Order Entry.
•
M - Manually placed on administrative hold.
•
O - Customer always on order hold.
•
P - Customer requires P/O number.
•
Q - Customer invoice amount check.
•
S - Ship via from web requires authorization.
•
T - No tax jurisdiction codes.
•
W - Web generated order.
•
Z - Credit card order with no authorization.
•
0-9 - User defined codes.
Administrative Quote Hold reason codes:
•
D - Override freight dollars.
•
F - Override freight terms.
•
I - Incompatible ship via.
•
J - Exceeded the quote limit for the Written By user ID.
•
L - All the customer’s quotes go on Administrative Quote Hold (Admin Hold Override set to Q in the customer’s A/R Information screen).
•
M - Manual hold.
•
O - Customer always on quote hold.
•
P - Customer requires a P/O number.
•
Q - Customer invoice amount check.
Line
The reason code to indicate why the order went on Administrative Hold (refer to the Administrative Order Hold Inquiries in the Order Processing documentation). Codes in this column refer to the holds affecting the line items:
Administrative Invoice Hold reason codes:
•
C - Price overridden below cost.
•
G - Price overridden below GP%.
•
R - Price overridden below customer default price.
Administrative Order Hold reason codes:
•
A - Product always on hold.
•
C - Price overridden below cost.
•
G - Price overridden below GP%.
•
I - Incompatible ship via.
•
N - Price overridden below minimum price.
•
R - Price overridden below customer default price.
•
S - Quantity over month’s supply test.
•
V - A line item’s gross profit percent is below the Mgr Low Level Minimum GP% or more than the Mgr High Level GP% (designated in Order Processing parameters).
Administrative Quote Hold reason codes:
•
A - Product always on hold.
•
C - Price overridden below cost.
•
G - Price overridden below GP%.
•
I - Incompatible ship via.
•
N - Price overridden below minimum price.
•
R - Price overridden below customer default price.
•
V - Manager low level/high level limit GP%.
Rel-Date
The date the order was released from hold.
Time
The time the order was released from hold.
User
The user ID of the person that released the order from hold.

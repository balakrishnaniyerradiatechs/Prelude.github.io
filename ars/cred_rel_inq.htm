Releasing Orders From Credit Hold
When a sales order goes on credit hold, it must be released by a credit manager in Credit Release Inquiry before a pick ticket will print. Your user ID must be set up as a credit manager to release orders from credit hold. Refer to Setting up Credit Managers for more information.
To view or release orders from credit hold:
1
Select Financial > Accounts Receivable > Inquiries > Credit Release Inquiry to open the Credit Release Inquiry screen. The Credit Manager Id for the branch defaults if you are set up as a credit manager.
2
Choose from the following Sort options:
•
Order Number.
•
Order Date.
•
Required Date.
•
Order Type.
•
Sales Order Dollars.
3
For Filters, choose from the following:
•
No - List all orders on credit hold.
•
Yes - Allows you to limit the list of orders by:
-
Customer.
-
Required date range.
-
Customer P/O number.
-
Job number.
-
Order total.
-
Order Type.
4
Press ENTER. Sales orders on credit hold appear.
5
To review an order or customer information before releasing it, choose from the following:
•
F5-Review - Opens the Sales Order Display screen to allow you to review detail information on the sales order.
•
F6-A/R Inq - Opens the Accounts Receivable Inquiry to display Accounts Receivable balance information for the customer.
6
To release orders from credit hold, choose from the following:
•
F7-Release All - To release all orders on hold. The following message appears:
Choose from the following:
You have requested to release all the orders remaining in this queue at one time. Are you sure you want to continue?
-
No - Returns to the Credit Release Inquiry screen.
-
Yes - Releases all the orders in the queue.
•
F8-Release - To release an individual order. The following message appears:
Do You wish to release order from credit hold?
7
Choose from the following:
•
Yes - Releases the order from credit hold and allows you to
-
Send notification to the salesperson’s work queue. Orders released from hold appear under Order Processing > Inquiries > Employee Queue Inquiry > Order Message Inquiry.
-
Print a pick ticket.
-
Exclude the sales order from the customer’s credit limit check, if applicable (refer to Excluding Sales Orders From the Credit Limit).
•
Yes W/ Notes - Same as Yes but allows you to attach notes to the sales order.
•
No - The order is not released and remains in the queue.
•
No W/Notes - The same as No but allows you to enter notes regarding the sales order and to send a message to the employee queue to explain why you are not releasing the order.
8
After you release the orders credit hold, press F2-Save to close the inquiry screen.

•

•

Once moved, use F7-Toggle to see orders in the hold queue. If necessary, you can then release the order from the hold queue.
F2-Finish
Closes the inquiry screen.

Opens the Sales Order Display screen to display detail information for the sales order.

Opens the Accounts Receivable Inquiry to display Accounts Receivable balance information for the customer.

Releases all orders on the inquiry screen.

Allows you to release the order from hold, send a message to the employee’s queue, and print the pick ticket.

Allows you to enter a line number, page number, or reference number to go directly to.
Action Bar
Refresh
Refreshes the information on the screen if you made any changes.
Toggle
Allows you to go back and forth between the Credit Release Inquiry screen and the hold queue. Orders can be released from either queue.
Alt Mgr
Allows you to send the highlighted order to another manager’s work queue.
Collect Notes
Opens the Credit Collections Notes Inquiry screen to allow you to view credit collection notes regarding this order (refer to Entering/Viewing Credit Collection Notes).
Comm/Notes
Opens the Credit Release Notes/Comments screen to allow you to enter one of the following:
•
N - To attach notes to this sales order. The notes appear in the sales order header and are user/time/date stamped.
•
C - To attach a comment line to the sales order. Comments appear as a CB comment line and are added to the end of the order.
The default for this field is set in Default Parameters Maintenance > Credit Release Msg Type.
 
Credit Hold Reason Codes
Reason Code
Description
ACR
Additional company rules. See your systems administrator for more information.
A/R
The Auto Customer Credit Hold Procedure put the customer on credit hold because of overdue A/R invoices. Refer to Automatically Placing Customers on Credit Hold.
CHG
Payment terms were changed in the Sales Order Entry header screen. Only applicable if Credit Hold A/R Terms Change is set to Y in Accounts Receivable Parameters.
CST
The order is larger than the Oversized Order Value set in Customer Maintenance.
EX
Excessive months since last sale. Only applicable if Cred Hold/#Mo Last Sale is set to Y in Accounts Receivable Parameters.
INS
Payment Terms on the sales order were changed to installments. Only applicable if Exclude Install Terms From Cred Hold is set to Y in Order Processing Parameters.
MAN
The order was manually placed on hold (refer to Manually Placing an Order on Credit Hold).
OVR
The customer is over their Credit Limit (set in Customer Maintenance). When you enter a sales order for this customer, if the total value of the customer’s A/R balance and open sales orders (excluding long-term future orders) exceeds the credit limit, the order goes on credit hold.
PD
Undisputed past due. You can mark an invoice as disputed in Manual Invoice Entry (refer to Manually Entering an Invoice) or in Accounts Receivable Maintenance (refer to Editing an A/R Invoice).
REL
Exceeded credit hold release amount.
SYS
The order is larger than the Oversize Order Value set at the system level in ”Order Processing Parameters.
TMP
Temporary customer.
WHL
The order is larger than the Oversized Order Value set in Warehouse Maintenance.

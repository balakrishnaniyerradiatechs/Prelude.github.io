Sequencing Follow-Up Collection Activities
For each Credit Collection type, you can create a sequence of follow-up collection activities for overdue customers. Credit Collections Nightly Processing generates the follow-up activities depending on how many days the oldest customer invoice is past due and how many days elapsed since the last activity.
To create a credit collection follow-up table:
1
Select Financial > Accounts Receivable > Collections Subsystem > Maintenance > Credit Collections Follow-Up Maintenance to open the Credit Collections Follow-Up Maintenance screen.
2
For Credit Collections Type, enter the credit collections type code to create a schedule for. These codes are created in Credit Collections Type Maintenance and are used to group similar customers for collections. Refer to Defining Credit Collections Types.
3
Enter a Security Access Code so that the list of follow-up activities only applies to customers with the same access code. This field is only accessible if you have Bypass Security Access in Coll set to N in Accounts Receivable Parameters.
4
Use the activity table at the bottom of the screen to define the interval between activities and the action to take at each interval:
•
For Days on the first row, enter the number of days past the customer's oldest invoice due date that collection procedures should begin. The remaining activities are based upon the number of days past the last activity. These number of days are overridden if values are entered in Maximum Amount Interval Days and Maximum Amount First Action Days.
•
For Activity, enter the code, set up in Credit Collections Activity Maintenance (refer to Creating Credit Collections Activity Codes), that defines the activity that takes place when the corresponding number of Days passes and the customer is not yet current.
Enter as many rows of Days and Activities as necessary to define this collection schedule.
5
To schedule a final activity, if the customer is still not current after all the activities are exhausted, enter the following:
•
Final Action Interval Days. The number of days after the last Activity (defined on the activity table) that you will take final action against the customer.
•
Final Action Activity Code. The code for the final action that you will take. This activity continues to occur every number of Final Action Interval Days until the customer becomes current or is manually removed from collections.
6
If you want to set an override to the number of days in the activity table (i.e., for large past due balances) to determine when collection activities begin, enter the following:
•
Maximum Delinquent Amount. When the customer’s past due balance goes above this amount, the first collection activity will take place in the number of days defined in Maximum Amount First Action Days.
•
Maximum Amount Interval Days. The override number of days between activities, that goes into effect if a customer's open balance is greater than the value in the Maximum Delinquent Amount. This number overrides all other number of days.
•
Maximum Amount First Action Days. The override number of days in the first line of the activity table that goes into effect if the customer's open balance is greater than the Maximum Delinquent Amount.
7
Press F2-Save to save the list of follow-up activities for this credit collections type.
Credit Collections Nightly Processing uses this information to determine when each follow-up activity should be scheduled for each customer. Refer to Generating Credit Collections Activities.

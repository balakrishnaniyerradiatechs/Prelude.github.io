Creating Charge Back Reason Codes
If you want to charge an amount back to the customer (e.g., bounced check), you must enter a charge back reason code to explain why you are charging the customer (refer to Issuing Charge Back Invoices).
Accounts Receivable Option Parameters for Charge Back Invoices
Consol. Chargeback Invoices determines if charge back invoices are consolidated onto one invoice:
•
Y - Charge back invoices are consolidated into one invoice per reason code for a customer in Cash Receipts Entry.
•
N - Each charge back invoice is on a separate invoice for a customer in Cash Receipts Entry.
Charge Back Inv By Cust P/O? determines if a customer P/O number is required for each reason code in Cash Receipts Entry:
•
Y - A customer P/O is required for each charge back invoice. In this case, you cannot consolidate charge back invoices (Consol. Chargeback Invoices must be set to N).
•
N - Customer P/O numbers are not required for each charge back invoice.
To create a charge back reason code:
1
Select Financial > Accounts Receivable > Maintenance > Charge Back Reason Maintenance to open the Charge Back Reason Code Maintenance screen.
2
For Charge Back Reason, enter a 1 or 2 digit alphanumeric code to use when charging a customer back in Cash Receipts Entry.
3
Enter the Description that explains the charge.
4
If applicable, enter any Comments to further explain the charge.
5
Press F2-Save to save the charge back reason code.
Refer to Printing Charge Back Invoices for information on transmitting the invoice to the customer.

Credit Collections Follow-Up Maintenance screen
For each Credit Collection type, you can use the Credit Collections Follow-Up Maintenance screen to create a sequence of follow-up collection activities for overdue customers. Credit Collections Nightly Processing generates the follow-up activities depending on how many days the oldest customer invoice is past due and how many days elapsed since the last activity without the customer becoming current. Refer to Sequencing Follow-Up Collection Activities for more information.
Menu Path: Financial > Accounts Receivable > Collections Subsystem > Maintenance > Credit Collections Follow-Up Maintenance
Field and Function Key Descriptions
Field/Function Key
Description
Credit Collections Type
The credit collections type code for which you want to create follow-up activities. These codes are created in Credit Collections Type Maintenance (refer to Defining Credit Collections Types) and are used to group similar customers for collections.
Security Access Code
Only accessible if you have Bypass Security Access in Coll set to N (refer to Accounts Receivable Parameters).
Final Action Interval Days
The number of days after the last Activity (defined on the activity table) that you will take final action against the customer. The final action is defined by the Final Action Activity Code. The final action is repeated this number of days until the customer becomes current or you manually remove them from collections.
Final Action Activity Code
The code for the final action that you will take. This activity continues to occur every number of Final Action Interval Days until the customer becomes current or is manually removed from collections.
Maximum Delinquent Amount
When the customerâ€™s past due balance goes above this amount, the first collection activity will take place in the number of days defined Maximum Amount Interval Days and Maximum Amount First Action Days.
Maximum Amount Interval Days
The override number of days that goes into effect if a customer's open balance is greater than the value in the Maximum Delinquent Amount. This number overrides all other number of days.
Maximum Amount First Action Days
The override number of days in the first line of the activity table that goes into effect if the customer's open balance is greater than the Maximum Delinquent Amount.
Days
The number of Days past the customer's oldest invoice due date that collection procedures should begin. The remaining activities are based upon the number of days past the last activity. You can schedule an unlimited number of activities.
Activity
For Activity, enter the code, set up in Credit Collections Activity Maintenance (refer to Creating Credit Collections Activity Codes), that defines the activity that takes place when the corresponding number of Days passes and the customer is not yet current.
Description
The description of the activity code.
F2-Save
Saves the follow-up information.
F4-Del
Deletes the follow-up information for this credit activity.

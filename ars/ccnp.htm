Running Credit Collections Nightly Processing
To determine which customers to include in credit collections activities and which activities should be scheduled, Credit Collections Nightly Processing evaluates the following:
•
Customer and system options to determine when a customer is included in credit collections activities. Refer to Defining Customers/Invoices to Include in Credit Collections.
•
The date of the last collection activity (if any). Credit Collection activities are defined in Creating Credit Collections Activity Codes.
To run Credit Collections Nightly Processing:
1
Select Financial > Accounts Receivable > Collections Subsystem > Maintenance > Processes > Credit Collections Nightly Processing to open the Credit Collections Nightly Processing screen.
2
For Credit Collections Type, enter a credit collection type or leave blank for all credit collection types (refer to Defining Credit Collections Types).
3
For Cust Security Access, choose from the following:
•
If Bypass Security Access in Coll in Accounts Receivable Parameters is set to N, enter the security access code. Only customers with the same security access code are selected. If the security access code prompt is left blank, then only customers with no security access code assigned are selected.
•
If Bypass Security Access in Coll in Accounts Receivable Parameters is set to Y, you can skip this field.
Once Nightly Processing is complete, if another activity is required (refer to Sequencing Follow-Up Collection Activities), the activity is placed in one of the following queues depending on the type of activity scheduled:
•
Credit Collections Schedule Entry (refer to Entering Scheduled Credit Collection Activities).
•
Credit Collections Letter Generation. (refer to Generating Scheduled Collection Letters).

Closing the A/R Credit Card Batch
The Batch Close tells Protobase to complete the transfer of money from your customer’s account into your bank account. This process closes all post authorization transactions, including the credit card transactions created in Sales Order Entry (refer to the “Credit Card” documentation in the Order Processing module).
To close an A/R credit card batch:
1
Select Financial > Accounts Receivable > Processes > A/R Credit Card > Credit Card A/R Batch Close to open the Credit Card Batch Close Process screen.
2
Enter the Selling Warehouse number for the batch close.
3
Enter a Credit Card Type if you use multiple merchant numbers for a credit card type, otherwise accept the default of VISA.
4
For End Date, enter the post-authorization date. If the batch was post-authorized yesterday, enter yesterday’s date.
5
For Close all Branches, choose from the following:
•
Y - Close all merchant IDs if you have more than one merchant ID per branch.
•
N - Only close the branch entered in Selling Warehouse.
6
Once you choose Execute, the following message appears:
Do you wish to Print Preview or Close the Batch?
7
Choose from the following:
•
Print - Prints the Protobase Administrator Terminal Report to allow you to verify the credit card transactions for the current batch. The batch does not close until you run this process again and select Close.
•
Close - Prints the Protobase Administrator Terminal Report. After the report prints you can then close the batch. A final settlement report prints to list what the bank says you transmitted and it approves.
8
Match the batch total on this report with the batch total on the ARCC Balancing Report. Any discrepancies indicate a failure between Prelude and Protobase (i.e., if the post-authorization was sent to Protobase multiple times or if Protobase did not receive a post-authorization request). If the reports do not match, then there is a Protobase problem and you must adjust the client’s card immediately so that it does not carry over to the next day. Call Activant if you need help balancing.
Invoice Detail Inquiry
Once the credit card batch is closed, the invoices are updated as paid in the Invoice Detail Inquiry.
Protobase Terminal Report - Column Descriptions
Column Heading
Description
Ref #
The credit card batch number.
Account
The customer’s credit card number.
Exp
The expiration date of the credit card.
Type
The Protobase terminology for transaction type (e.g., prior authorization).
Amount
The amount charged to the credit card, including tax and other charges.
Date
The date the credit card was used to pay for the transaction.
Bus.
The date the transaction was post authorized.
Auth#
The credit card authorization code from protobase
Cashier
The order taker (written by).
Order Num.
The sales order number.
AVS
The Address Verification Service approval number if you use AVS.
Zip Code
The zip code of the billing address for the credit card.
Billing Address
The billing address for the credit card.
Credit Totals for Terminal
Card Type
The type of credit card.
# Auth
The number of transactions authorized for this terminal.
Auth $
The total amount authorized for this terminal since the last close.
# Sale
The number of sales made at this terminal since the last close.
Sales $
The total sales generated at this terminal, including authorized and non-authorized.
# Ret
The total amount of returns for this terminal.
Return $
The total value of returned material.
The remaining fields may apply to some Protobase customers but are not used by Prelude.

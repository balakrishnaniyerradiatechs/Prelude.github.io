Issuing Charge Back Invoices
To charge a customer for fees associated with the invoice (e.g., bounced check), you can enter the charge back in Cash Receipts Entry.
Accounts Receivable Option Parameters for Charge Back Invoices
Consol. Chargeback Invoices determines if charge back invoices are consolidated onto one invoice:
•
Y - Charge back invoices are consolidated into one invoice per reason code for a customer in Cash Receipts Entry.
•
N - Each charge back invoice is on a separate invoice for a customer in Cash Receipts Entry.
Charge Back Inv By Cust P/O? determines if a customer P/O number is required for each reason code in Cash Receipts Entry:
•
Y - A customer P/O is required for each charge back invoice. In this case, you cannot consolidate charge back invoices (Consol. Chargeback Invoices must be set to N).
•
N - Customer P/O numbers are not required for each charge back invoice.
To charge back a customer:
1
In the Cash Receipts Invoice Detail Entry screen, choose C for M/C. Depending on how your Account Receivable Option Parameters are set to handle charge backs, one of the following occurs:
•
If Chargeback Inv by Cust P/O? is set to Y, you can continue with the cash application. When you press F2-Save to finish the batch, the Cash Receipts Chargeback By Customer P/O Number screen appears.
•
If Chargeback Inv by Cust P/O? is set to N, the Charge Back Entry screen appears.
2
For Reason, enter the charge back reason code to explain why you are charging the customer. Refer to Creating Charge Back Reason Codes for more information.
3
Enter the Customer P/O if customer P/O numbers are required (only applicable if Option Parameter Chargeback Inv by Cust P/O? is set to Y).
4
For Debit Amount, enter the amount you are charging back to the customer.
5
Enter additional reason codes and debit amounts until the total of the Debit Amount equals the total Amount Left To Distribute.
6
Press F2-Save to save the charge back information and to return to the Cash Receipts Invoice Detail Entry screen. The Balance for the invoice line should now be zero if there are no other discrepancies.
When you finish applying cash for this check number, the Charge Back Invoice Entry screen appears.
7
Choose one of the following to enter the Invoice Number to bill the customer:
•
Enter NEW for the next sequential invoice number.
•
Enter any number to assign to the invoice. If you enter the original number, you must add a character to it so that it is not exactly the same.
8
Enter the Invoice Date for the charge back invoice.
9
If necessary, edit the Comments regarding the charge back. The comments are added when the reason code is created (refer to Creating Charge Back Reason Codes).
10
Press F2-Save to complete the charge back invoice. Depending on how Accounts Receivable Option Parameters are set to consolidate charge backs, one of the following occurs:
•
If Consol Chargeback Invoices is set to Y, all charge backs for each reason code are consolidated into one invoice per customer.
•
If Consol Chargeback Invoices is set to N, a separate invoice is created for each charge back, even if they have the same reason code.
11
Repeat steps 7-10 for each additional charge back invoice to create.
Refer to Printing Charge Back Invoices for information on transmitting the charge back invoice to the customer.

Charge Back Invoice Entry screen
The Charge Back Invoice Entry screen appears after you finish applying cash to a batch in which you created a charge back to a customer. Refer to Issuing Charge Back Invoices.
Menu Path: Financial > Accounts Receivable > Cash Receipts Entry > F5-Enter Detail > enter C for M/C > complete cash receipt
Field and Function Key Descriptions
Field/Function Key
Description
Charge Back Reason Code
The charge back reason code to explain why you are charging the customer. Refer to Creating Charge Back Reason Codes for more information.
Customer P/O Number
The customer purchase order number, if applicable.
Charge Back Amount
The amount charged back to the customer.
Warehouse Number
The warehouse responsible for the sale.
Invoice Number
The invoice number to use to bill the customer for the invoice amount (e.g., you can enter the original number but you must add a character to it so that it is not exactly the same).
Invoice Date
The date of the charge back invoice.
Comments
Any comments regarding the charge back invoice.
F2-Save
Completes the charge back invoice. Depending on how Accounts Receivable Option Parameters are set to consolidate charge backs, one of the following occurs:
•
If Consol Chargeback Invoices is set to Y, all charge backs for each reason code are consolidated into one invoice per customer.
•
If Consol Chargeback Invoices is set to N, a separate invoice is created for each charge back, even if they have the same reason code.

Product Maintenance
To setup a product for rationalization, go to Inventory > Maintenance > Product Maintenance > Product Maintenance > Action Bar > Addtl Opts > TPCx Information. The TPCx Information screen appears.
Field/Function Key Descriptions
 
Field/Function Key
Description
Vendor
The rationalized TPCx vendor for the item. The rationalized vendor is based on the list of vendors in the Purchasing Controls screen in Product Maintenance > Action Bar > Addtl Opts > Purchasing Controls. The vendor you rationalize first is used.
Vendor Product
The vendor product number for the item.
UPC Number
The UPC (Universal Product Code) for the item.
Alternate 1
A secondary number that TPCx used to match this part with a manufacturer’s part on the hub (e.g., UCC or other product alternates).
Stockable
Indicates if this item is stockabe for TPCx:
•
Y - the item is stockable.
•
N - the item is not stockable.
Rationalized
Indicates if this product is rationalized (updated by TPCx):
•
Null - never rationalized.
•
Y - currently rationalized.
•
N - formerly rationalized.
Dead Item
Indicates if this product is dead stock (updated by TPCx):
•
Null - never dead stock.
•
Y - currently dead stock.
•
N - formerly dead stock.
If you:
•
Delete a product with TPCx information, Prelude notifies TPCx.
•
Use Universal Product Replacement, Prelude sends a request to TPCx to delete the old product and rationalize the new product .
•
Use Universal Product Merge, Prelude requests rationalization from TPCx only if the item being merged into is already rationalized.

Renting from a Vendor and Renting to a Customer
You can rent an item from a vendor so you can rent it to a customer by receiving and billing the rental item multiple times until the rental is complete. Typically, non-stock product is ordered and linked to a type 11 Recurring Billing purchase order. When you receive the vendor’s invoice and reconcile it with the receiver in Vendor Invoice Reconciliation Entry, the vendor is setup for payment and the customer is invoiced for the same amount if the product is non-rental. If it is a rental product then the customer is invoiced through periodic billing. The rental/sales order and purchase order stay open until the rental item is returned and the final bill is received from the vendor.
Purchase Order Entry Types
Path: System Parameter > Maintenance > Purchasing Controls Purchase Order Entry Types
Purchase order type 11 (Recurring Billing) must be available in Purchase Order Entry Type maintenance before it can be used in Purchase Order Entry. If this purchase order type is not listed simply add it to the list.
Product Maintenance (/PM)
For each product (typically nonstock), you can indicate if recurring billing can be used:.
-
Y - use to recurring billing in Vendor Invoice Reconciliation Entry until the rental is complete.
-
N - do not use recurring billing.
Path: Inventory > Maintenance > Product Maintenance F5-Add Info > F5-Add Info
Recurring Billing Steps
Recurring billing may be done with a rental or non-rental product. Using a rental product allows you to use the periodic billing process if the order is set up for periodic billing. Using a non-rental product allows you to by-pass periodic billing and bill the customer immediately after receiving the bill from the vendor.
1
Enter the non-stock product on an order.
-
For a Rental Product: In Rental Order Entry (or Sales Order Entry) enter an part with a Product Type of N (rental) and with Recurring Billing set to Y.
-
For a Non-Rental Type Product: In Sales Order Entry enter an non-stock part without a Product Type of N and with Recurring Billing of Y.
In both cases, the supply code is D when the line is linked to a type 11 recurring P/O.
2
Link the rental product to a type 11 Recurring Billing P/O either through the automatically in rental/sales order entry or manually in purchase order entry.
3
Start the clock on the rental using Vendor Invoice Reconciliation Entry. On the first receiver, enter a rental start date and time for rental products. You can start the clock before receiving the item by updating the start and time in VIRE, but not the A/P information.
4
Receive the P/O as many times as needed. The original order quantity is never decreased; the purchase order remains the way it was when first entered. When you save these receivers and are asked if the rental is complete, choose No so that the pick quantity on the is updated but the linkage is not removed and the P/O stays open.
5
When you finish renting the item, close the P/O in VIRE. When you save the final receiver in VIRE, choose Yes when you are asked if the rental is complete. The rental/sales order is not set up for invoicing. You must manually set up the last invoice to the customer for billing.
6
Bill the customer. For rental products, the order is not set up for invoicing. Instead periodic billing picks it up and invoices it. For non-rental products, the linked sales order is set up for billing if A/P Auto Direct Invoice Setup is Y. If the A/P Auto Direct Invoice Setup system option is set to ‘N’, then you must manually set the order up for invoicing. The invoice is generated the next batch.

Rental Start Date & Time
Rental start date and times are used to calculate the available days an item is on rent. Dates and times can be entered in Rental Order Entry or Release Confirmation Entry.
Rental Parameters
Path: System Parameter > Maintenance > Option Parameters > Rental Parameters
In Rental Parameters, Start Date/Time in RCE? determines if rental start date and time are required in Release Confirmation Entry. Choose one of the following
•
Y - ask for the start date and time of the rental in Release Confirmation Entry.
•
N - do not as for the start date and time; default the date and time to the pick ticket printing date and time.
Rental Order Entry (/ROE)
Enter a default rental start time and date in the Rental Header in Rental Order Entry.
Path: Rental > Rental Order Entry
•
Rental Start Date: the rental start date to default in the Rental Information screen in Release Confirmation Entry.
•
Rental Start Time: the rental start time to default in the Rental Information screen in Release Confirmation Entry.
The date and time in the Rental Header screen default in Release Confirmation Entry but can be changed. If these fields are blank, the default is the system date and time in Release Confirmation Entry. If a date and time are entered in the Rental Header and Start Date/Time in RCE? is N, the pick ticket printing date and time are used.
Release Confirmation Entry
If Start Date/Time in RCE? is Y in Rental Parameters, you can enter the expected number of rental periods (e.g., hours, days, weeks, or months) in Release Confirmation Entry to calculate an accurate expected rental return date. The expected rental return date is calculated by multiplying the expected number of periods times the billing cycles in the period for each item on the rental order for all units of measure.

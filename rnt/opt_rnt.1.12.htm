Exchanging Rental Items
The rental exchange process is used to switch out a unit in the middle of the rental. For example, the unit breaks down and therefore must be replaced with another. This process allows you to exchange a serialized item on a rental order or to exchange the original product for an entirely different product. The system tracks the time on rent for the original unit so the system can post rental income and days on rent for the unit. The new unit being placed on rent takes the place of the original unit on the same rental order without having to close the rental order and open a new one. This process works with both serialized and non-serialized products.
Setting Up Rental Exchanges
Rental Exchange Counter Range Setup
This maintenance routine is used to establish and maintain the number range and current number to be used for rental exchanges. These must be set up before a rental exchange can be performed.
Path: System Parameter > Maintenance > Counter/Range Setup Rental Exchanges
Warehouse Maintenance (/WM)
Continuous printing for rental exchanges can be established in Warehouse Maintenance.
Path: System Parameter > Maintenance > Inventory Controls > Warehouse Maintenance > F6
•
Cont. Rental Exchange: determines how rental exchange documents are printed.
-
Y - use continuous printing for rental exchange documents.
-
N - use Rental Exchange Printing by batch.
Rental Exchange Reason Code Maintenance (/RXCM)
All rental exchanges require a reason code to be entered to define why the exchange took place. Enter as many reason codes as are needed and a description for each.
Path: Rental > Maintenance Rental Exchange Reason Code Maintenance
Entering a Rental Exchange (/REE)
Path: Rental > Processes > Rental Exchange Entry
This entry program is used to perform the actual exchange of the items on the rental order. The top portion of this screen will look similar to the one displayed below.
•
Exchange Number: enter ‘N’ or ‘New’ to assign the next available system generated rental exchange number or enter number of a suspended rental exchange.
•
Rental Order Number: the original rental order number for the item being exchanged. The Customer field is automatically filled in when the rental order number is entered.
After the rental order number is entered, you are taken into the exchange order header screen (below). The information entered in this screen is strictly for the exchange and will not affect the original rental order.
•
Entry Date: defaults to the system date and time, not the date of the exchange.
•
Ship-To Number: this field is used to print the ship-to information on the rental exchange document if the new rental unit must be delivered or the original rental unit picked up. For the rental exchange only; does not change the rental order.
After the rental exchange order header has been filled in, you are taken into the line item detail screen (below). This is where the original rental unit and the exchange rental unit are entered. Both products must be either serialized or non-serialized. You cannot replace a serialized unit with a non-serialized unit or vice versa.
•
Original Product Number: enter the rental product that is being exchanged from the original rental order.
•
New Product Number: enter the rental product number that is replacing the original rental unit or enter ‘R’ to repeat the original product number.
After the original product and the new product numbers are entered, you are taken into the line item detail screen (below) and prompted for detailed information about each. Non-serialized products will only prompt for the exchange quantity, reprice flag, and reason code. All the other fields are for serialized products.
•
Xchg-Qty: if the items are serialized, this field defaults to ‘1’ and may not be changed. If the items are non-serialized, enter the quantity being exchanged. If the product is serialized, you can only exchange one at a time. If the original line on the rental order has a quantity greater than one, the line item on the rental order will have to be split before the exchange can be made.
•
Original Serial Number: enter the serial number for the unit that is being exchanged. If the item is serialized, a serial number must be entered. This field is not prompted if the item is non-serialized.
•
Original Meter Out: defaults to the Meter Out reading from the rental order. Required if Measurements Required in Product Maintenance is H (hour meter) or O (odometer).
•
Original Meter In: the current meter reading for the unit being exchanged. Required if Measurements Required in Product Maintenance is H (hour meter) or O (odometer).
•
If wear measurements or fuel charges are required for the unit being exchanged, the system prompts for these on an additional screen (below) and adds additional lines to the original rental order for these charges.
•
New Serial Number: enter the serial number for the unit that is replacing the returned unit. A serial number does not have to be assigned at this point but will be required before completing the rental order. This field is not prompted if the item is non-serialized.
•
New Meter Out: the meter reading for the replacing unit if the product’s Measurements Required in Product Maintenance is set to H (hour) or O (odometer).
•
New Wear Out: the wear measurement for the replacing unit if the product’s Measurements Required in Product Maintenance is set to W (wear).
•
Reprice: this field defines if the rates for the old unit will be carried over to the new unit. This flag is used to reprice the rental order for the new product number. This field will default to ‘Y’ if the new product number is different than the original product number. If the products are the same, the default is ‘N’. This flag is mainly used when the new product number is different than the original product number or if the new product number has a different rate structure than the original product number. If this flag is set to ‘Y’, the rental rates for the new product are calculated for the rental order. If this flag is set to ‘N’, the new product will retain the pricing from the original product.
•
Reason: enter the code that describes the reason for the exchange. This is a mandatory field. Reason codes are set up in Rental Exchange Reason Code Maintenance (/RXCM).
After all the exchange line items have been entered, use the F2-Save function key to exit the entry program. At this point, you can print, complete, or suspend the exchange or to return to the line item detail screen.
Do You Wish To...
•
Print - set the rental exchange up for printing. The rental exchange document can be used for delivery and pickup of the exchange items.
•
Suspend - saves the rental exchange until you return to the order. Suspended rental exchanges appear in the Rental Exchange Inquiry and Open Rental Exchange Report.
•
Complete - close the exchange transaction. Enter the date and time of the exchange. This date and time stop the rental on the original product and to start the rental on the new product. The line item must be manually adjusted for down time.
At this point, the products are switched on the rental order along with serial numbers and meter readings if applicable. The system keeps track of the charge for the original unit’s rental period and adds it to the charge for the new unit’s rental period. When a serialized product is exchanged, the status is set to ‘C’. The status will have to be manually changed through Rental Status Maintenance.
Printing a Rental Exchange Document
You can print a rental exchange document for a selected rental exchange, rental exchanges for a warehouse, or all rental exchanges. This document can be used to deliver and/or pick up the rental unit from the customer’s site. The rental exchange document the rental exchange number, rental order number, original product, and new product and serial numbers and meter readings if available.
Path: Rental > Document Printing > Rental Exchange Printing
•
Rental Exchange Number: enter the rental exchange number to be printed.
•
Warehouse Number: the warehouse to limit the printing or leave blank to for all exchanges.
•
Printer Form Queue: enter the printer form queue where the document should print.
Path: Rental > Document Printing > Rental Exchange RePrinting
•
reprint a rental exchange document to the form queue entered.
•
Rental Xchg History #: the rental exchange history number to reprint.
•
Warehouse Number: the warehouse to limit the printing or leave blank to for all rental exchanges.
•
Printer Form Queue: enter the printer form queue where the document should print.
Inquiries & Reports
Order Log Inquiry (/OLI)
Path: Order Processing > Inquiries Sales Order Inquiries Sales Order Log Inquiry
When a rental order is exchanged, the Order Log Inquiry is updated with the XCH code. The Reference Number is updated with the rental exchange number. Drill down into the Rental Exchange Entry is available. Use the examine feature on the line item to view serial numbers and other information.
Rental Exchange Inquiry (/RXI)
Lists customer rental exchanges that were entered but not completed. You can drill down into a rental exchange to view the document.
Path: Rental > Inquiries Rental Exchange Inquiry
Rental Exchange History Inquiry (/RXHI)
Lists customer rental exchanges. You can drill down into a rental exchange to view the document. The screen is similar to Rental Exchange Inquiry except the exchange date and time appear instead of the entry date and time.
Path: Rental > Inquiries Rental Exchange History Inquiry
Open Rental Exchange Report
Path: Rental > Reports Open Rental Exchange Report
Lists open rental exchanges for warehouse(s), customer number(s), and/or entry date range. This report displays the original and new product, and detail information such as serial numbers, meter readings, and charges.
Rental Summary Inquiry
The Rental Summary Inquiry lists rental orders for a customer and job. The quantity is the total units of the product entered for a job.
Path: Rental > Inquiries > Rental Summary Inquiry (/PRJI)
•
Product Number: the product number.
•
Warehouse: the warehouse number used on the rental orders for the product number.
•
Customer Number: the customer number used on the rental orders.
•
Job Number: the job number used on the rental orders.
•
Quantity On Order: the quantity of the total number of rental orders for the product number, warehouse, customer number, and job number entered is displayed.
•
F5-Detail opens the Sales Order Rental information.
•
F5-Review Order opens the Sales Order Inquiry for the rental order.
•
F7-Log opens the Order Log Inquiry for the rental order.

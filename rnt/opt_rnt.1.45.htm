Periodic Billing
This process determines if any open rental orders set up for periodic billing need to be invoiced for the current billing cycle and generates billing entries for those orders. This procedure checks each line item on the rental order to evaluate if it is available for billing. Periodic Billing may be run for a order, customer, and/or warehouse.
Path: Rental > Processes Periodic Billing
•
Billing Date: enter the date for the system to check when selecting open rental orders to verify if a rental has completed a billing period.
•
Maximum Days In Cycle: this field defaults to the number of days that translates to a month cycle for billing purposes. Basically, this is the minimum number of days past the on-rent or bill-through date before you can bill the order again. This field defaults to 2 weeks (14 days) plus the value in the Days Shift To Week option parameter. (Three weeks is normally considered a month in the rental industry. Therefore this field defaults to the shortest number of days considered a month which is 2 weeks plus whatever it takes for another week.) If the Days Shift To Week option parameter is set to ‘4’ then 18 days is the minimum number of days the item is on rent before you can bill it for another month. Remember that this is 18 days past the on-rent date (if first pass) or the bill-through date (if already billed once through periodic billing).
•
Order Number: enter the order number to be invoiced during periodic billing or leave the field blank to generate invoices for all orders that meet the qualifications for periodic billing.
•
Warehouse Number: enter the warehouse number to only invoice orders for a warehouse or leave the field blank to generate invoices for all warehouses that meet the qualifications for periodic billing.
•
Customer Number: enter the customer number to only invoice orders for a customer or leave the field blank to generate invoices for all customers that meet the qualifications for periodic billing.
When this screen is exited, the system will select all open rental orders set up for periodic billing. It then compares the Billing Date with the On-Rent date (if first pass through periodic billing) or the Bill-Through date (if already billed once through periodic billing) to determine if the Max Days In Cycle has been reached or exceeded. If so, the order is set up for invoicing. For example, the Billing Date is set to ‘07/14/02’ and the Max Days In Cycle is set to ‘18’. A rental order whose on-rent date is 06/30/02 will not be selected during this run because it is not 18 days past the on-rent date. A rental order whose bill-through date is 06/24/02 will be selected during this run because it is 18 days past the bill-through date. The bill-through date on this rental order would be updated to 08/11/02 (based on a 28 day month). This allows you to get your money sooner.
If a rental order exceeds the customer's credit limit when periodic billing is performed and the flag, Cred Chk in Periodic Billing?, is set to ‘Y’, the rental order will go on credit hold. Once the rental order is released from credit hold, the rental order is setup for invoicing.

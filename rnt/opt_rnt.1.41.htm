Rental Return – Rental Order Entry (/ROE)
Path: Rental > Rental Order Entry
When a pre-billed rental order is returned, you are prompted for the return information as usual. If the Allow Auto Credit option parameter is set to ‘Y’, the system will check the actual return date against the expected return date or rental bill thru date if one is entered. If the item is returned before the rental period expires, a message is prompted to allow the user to issue a credit for the days not used.
The Order Has Early Returns. Do You Wish To Create Credits?
If ‘No’ is selected, a credit memo line is not created on the rental order. If ‘Yes’ is selected, a credit memo line is created on the rental order for the number of days not used. This credit memo line can be referenced to the original invoice number. When a monthly pre-billed rental order is returned early, the daily rate is calculated by taking the monthly rate divided by the monthly billing cycle days. When a weekly pre-billed rental order is returned early, the daily rate is calculated by taking the weekly rate divided by the weekly billing cycle days. The rate is then multiplied by the number of days returned early for credit against the rental order.
Note: The credit is always issued in Days (i.e., the rental period (U/M) for the credit item is DAY). No shifting occurs.

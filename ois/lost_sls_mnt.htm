Lost Sales Reason Maintenance screen
If your company tracks lost product sales for sales analysis, you can use the Lost Sales Reason Maintenance screen to create lost sales reason codes that identify why a product was deleted on a sales order. Each time you delete an item, you can determine whether or not to update product usage for the lost sale (refer to Tracking Lost Sales).
Menu Path: Order Processing > Maintenance > Misc Code Files > Lost Sales Reason Codes
Field and Function Key Descriptions
Field/Function Key
Description
Lost Sales Reason#
The 1 or 2 digit alphanumeric code that represents an order cancellation reason code.
Description
The description of the code that explains why the order was lost (e.g., price too high, out of stock, late delivery).
Update Product Usage
Determines if product usage is updated on a sales order or quote when the product is deleted:
•
Y - Update product usage for the lost sale.
•
N - Do not update product usage for the lost sale.
F2-Save
Saves the lost sales reason code.
F4-Del
Deletes the lost sales reason code.

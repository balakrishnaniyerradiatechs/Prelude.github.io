Using Administrative Order Hold for Credit Cards
If you want to require that credit card sales orders have an authorization code before the pick ticket prints, you can set up an Administrative Order Hold Option for credit cards.
Pre-Authorizing Sales Orders in Release Confirmation Entry
If you pre-authorize credit card sales orders in Release Confirmation Entry (refer to Confirming Releases), do not use the Administrative Order Hold option for credit cards. The pick ticket prints without an authorization code. When you confirm the shipment, the Sales Order Completion screen automatically appears so you can pre-authorize the sale. In this case, you must activate Administrative Invoice Hold (refer to Using Administrative Invoice Hold for Credit Cards) for credit card orders so that the order cannot be invoiced without an authorization code.
To activate Administrative Order Hold for credit card sales orders:
1
In Order Processing Parameters, select F5-Add Info to open the Additional Information screen.
2
For Administrative Order Hold, enter Y to open the Administrative Order Hold Parameters screen.
3
For AOH Types, enter Z (Credit Card Order - No Authorization). If a credit card sales order does not have an authorization code, it will go on Administrative Order Hold and the pick ticket will not print. The order must be released from hold before the ticket prints
Refer to the Administrative Order Inquiry for information on all the order hold options and releasing orders from hold.
4
Press F2-Save twice to save the Administrative Order Hold Parameters.

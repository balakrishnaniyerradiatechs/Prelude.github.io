Additional Order Information Screen
The Additional Order Information screen allows you to enter printing, pricing, invoicing, and other transaction requirements for a customer.
Menu Path: Order Processing > Maintenance > Customer Maintenance > F7-Order > F5-Add Info
Field and Function Key Descriptions
Field/Function Key
Description
Invoice Print Cycle
Determines how often invoices print/fax/email for this customer:
•
D - Daily.
•
W - Weekly.
•
M - Monthly.
•
S - Semi-monthly.
Refer to Printing Invoices.
Allow Qty Disc
Determines if quantity discounts are allowed for this customer:
•
Y - The customer receives quantity discounts.
•
N - The customer does not receive quantity discounts.
•
H - The customer receives the highest quantity discount regardless of the quantity purchased on each sales order.
Refer to the Pricing Hierarchy.
Price Pick Tick
Determines if prices appear on picking tickets and packing lists:
•
Y - Print prices.
•
N - Do not print prices.
Refer to Setting Pick Ticket Rules.
Retail Pricing
 
Determines if retail prices are calculated for this customer:
•
Y - Calculate retail prices.
•
N - Do not calculate retail prices.
Refer to Retail Price Matrices.
Accept Substitutes
Determines if the customer allows automatic substitution:
•
Y - If a product is backordered, an available substitute is placed on the order (depending on how Product Substitution Type is set in Product Maintenance > F6-Xref/Notes > Substitutes). Refer to “Substitute Products” in the Inventory Documentation.
•
N - If a product is backordered, SUB displays on the Line Item Detail screen to inform you that the product has substitutes but the product is not automatically substituted. You can press F6-Subs to view available substitute products. You are warned that the customer does not accept substitutes but still allowed to accept the substitute.
Chain Discount Flag
Indicates if chain discounts exist for the customer:
•
Y - Opens the Chain Discounts screen to allow you to enter the chain discount percent and description. Chain discounts allow you to add multiple discounts for a final price. For example, if you have chain discounts of 10% and 20%, the system multiplies the sales price by .9 and multiplies that value by .8.
•
N - Chain discounts do not apply for this customer.
Add. Discount%
An additional discount percent that applies to each product.
Sales Tax Group
The sales tax group assigned to the customer. The tax groups are created in Valid Responses Maintenance and assigned to a Customer Tax Group in the Sales Tax Matrix Maintenance Screen. This information applies if some products purchased by the customer are taxed differently than their regular tax jurisdiction.
Unusual Usage
Determines if product usage is recorded from this customer’s sales orders to determine purchase quantities:
•
Y - Exclude the product usage recorded from sales to this customer.
•
N - Include usage from this customer.
Item# On Pk/Inv
No longer applicable.
Prohibit Sales
Indicates if you can enter sales orders or quotes for this customer:
•
Y - Sales orders and quotes are allowed.
•
N - Sales orders and quotes are not allowed.
Blind Shipment?
The default for the Blind Shipment flag in Sales Order Entry (F5-Header > F5-Add Info) for this customer. A blind shipment indicates that the sales order will ship to this customer without your logo:
•
Y - The sales order is sent as a blind shipment. The Blind Shipment Logo screen appears to allow you to enter the path to the logo image. If an image has read/write permissions, enter HOME/images/<image_name> (<image_name> is the file name). A blind shipment packing list prints the customer information in the logo area. The bill-to and ship-to addresses are used as the sales order’s ship-to address.
•
A - The same as Y, except that when you enter a sales order, you are asked if you want to send it as a blind shipment.
•
N - Sales orders for this customer are not blind shipments. Your logo is included on the pick ticket and packing list.
Packing List?
Determines if a packing list should print for this customer:
•
Y - Packing lists print when the order is confirmed in Release Confirmation Entry.
•
N - Packing lists do not print for sales orders for this customer.
Refer to Printing Packing Lists for more information.
Accept Backorders
Sets the default for Accept Backorders on sales orders for this customer. You can override this flag for each order in Sales Order Entry (F5-Hdr Scr > F5-Add Info):
•
Y - The customer normally accepts backorders.
•
N - The customer does not typically accept backorders.
•
R - Allows you to override this flag in Release Confirmation Entry. This option only works if the Auto Backorder Cancellation flag in Order Processing Parameters is set to N. You are notified in Release Confirmation Entry that the customer normally does not accept backorders but you can override it if necessary.
Cancel Amount
If the customer does not normally accept backorders (Accept Backorders is set to N) and the backorder is less than this amount, the backorder is cancelled. Orders over this amount are not cancelled.
Source Number
The source code that identifies how you acquired this customer. These codes are established in the Source Code Maintenance Screen.
Mail Invoices
Determines if confirming copies of counter sales invoices are sent to this customer:
•
Y - Confirming invoices for counter sales orders are sent to the customer.
•
N - Confirming invoices for counter sales orders are not sent to the customer.
Override D/C Flag
Determines if the Distribution Center path is followed to record usage when sales orders for this customer are invoiced:
•
Y - The Distribution Center controls in Warehouse Maintenance and Purchasing Parameters Maintenance that record usage are ignored.
•
N - The normal Distribution Center path is followed to determine which warehouse is updated with usage.
Refer to “Product Usage” in the Inventory documentation.
Mfg Name On Invoice
Determines if the manufacturer’s name should appear on the customer’s invoices for lot-controlled products:
•
Y - The manufacturer’s name is pulled from the lot control information and appears on the customer’s invoices.
•
N - The manufacturer’s name does not appear on customer invoices.
Test Rpts Required
Indicates if manufacturer test reports are required for this customer:
•
Y - Manufacturer test reports are required.
•
N - Manufacturer test reports are not required.
This is for information only and is used for custom reporting.
Mfr C Of C Required
Determines if a Manufacturer Certificate of Compliance is required:
•
Y - A manufacturer certificate of compliance is required.
•
N - A manufacturer certificate of compliance is not required.
This is for information only and is used for custom reports.
Critical Inspection
Determines if a critical inspection is required:
•
Y - A critical inspection is required.
•
N - A critical inspection is not required.
This is for information only and can be used for custom reports.
Approved Vendors
If lot control is in effect, then a product may have more than one manufacturer. This flag determines if you must choose approved manufacturers when you enter a sales order for this customer:
•
Y - You can specify a manufacturer for products sold to this customer.
•
N - You cannot specify a manufacturer for products sold to this customer.
Refer to the “Lot Information screen” in the Inventory documentation for more information on lot products.
Print Alt On Invoice
Determines if alternate part numbers should appear on the customer’s invoices:
•
Y - Alternate part numbers appear on invoices.
•
N - Alternate part numbers do not appear on invoices.
Summary Invoices
Determines if summary invoices print on cycle for this customer:
•
Y - Summary invoices print on cycle.
•
N - Summary invoices do not print.
•
R - Summary invoices print off cycle. You can print the summary invoices at any time, rather than on a schedule.
Refer to Printing Summary Invoices for more information.
Consolidate Invoices
Indicates when consolidated invoices print, if applicable:
•
D - Daily.
•
W - Weekly.
•
M - Monthly.
•
Blank - Consolidated invoicing is not used for this customer.
This value must match the value in Consolidated Invoice Printing to print invoices in the batch. Refer to Printing Consolidated Invoices.
Consolidate Type
If Consolidate Invoices has a value, you can determine the types of invoices to print:
•
B - Both sales and service invoices.
•
N - Neither sales or service invoices.
•
S - Sales invoices only.
•
V - Service invoices only.
Allocation Priority
Determines where the customer ranks when the Allocation Procedure allocates received stock to backorders. You can rank a customer from A to Z, with A being the highest priority and Z being the lowest. Customers with no priority code are selected last. You can override this setting in Sales Order Entry (F5-Hdr Scr > F5-Add Info > Allocation Priority).
Delivery Chg/PckUp
Indicates if delivery and/or pick-up charges are billed to the customer:
•
D - Delivery charges.
•
P - Pick-up charges.
•
B - Both delivery and pick-up charges.
•
N - Neither delivery or pick-up charges.
This field is used primarily with the rental module when equipment is delivered or picked up at a site.
You can override this value in Sales Order Entry > F5-Hdr Scr > Delivery/Pick Up.
Pick Days
The number of days to subtract from the required date to calculate the pick ticket date. This number overrides the Pick Ticket Default Days in Warehouse Maintenance and Order Processing Parameters. Enter T to default to today.
Refer to Setting Pick Ticket Rules.
Inv Document Option
Determines how invoices are transmitted to the customer:
•
B - Both fax and print.
•
E - Email only.
•
M - Email and print.
•
N - Print only.
•
Y - Fax only.
Refer to Printing Invoices.
Qte Document Option
Determines how quotes are transmitted to the customer:
•
B - Both fax and print.
•
E - Email only.
•
M - Email and print.
•
N - Print only.
•
Y - Fax only.
Refer to Printing Quotes.
Ack Document Option
Determines how acknowledgments are transmitted to the customer:
•
E - Email only.
•
M - Email and print.
•
N - No acknowledgment.
•
F - Fax only.
•
C - Print and fax.
•
T - Text email.
Refer to Sending Acknowledgements for information on setting up and sending sales order acknowledgements.
RGA Document Option
Determines how RGAs are transmitted to the customer:
•
B - Both fax and print.
•
E - Email only.
•
M - Email and print.
•
N - Print only.
•
Y - Fax only.
Refer to Printing RGAs.
Handling Charge
Indicates if handling charges are billed to the customer if the invoice does not meet a minimum amount:
•
Y - Handling charges are billed.
•
N - Handling charges are not billed.
The amount of the Handling Charge is set up in Order Processing Parameters.
Sum & Reprice Flag
Indicates if this customer is eligible for Sum & Reprice quantity break discounts:
•
Y - Sum & Reprice applies to this customer when enough products within the same product group are purchased in the amount that meets a quantity break discount.
•
N - Sum & Reprice discounts do not apply to this customer.
Sum & Reprice is a process that combines products in a defined group and applies a discount if the total number of items purchased in the group meets a quantity break level. Separately these products may not meet the quantity break price, but combined, they meet a discount break level.
Products can be grouped in one of two ways:
•
By product (Product Maintenance > F5-Add Info > Sum & Reprice Flag = Y). You can also assign a product group (Grp) and/or a product table (Tbl).
•
By a price matrix.
Refer to Pricing Hierarchy and Sum & Reprice and Quantity Break Discounts.
Invoice To Ship-To
Determines where invoices are mailed, faxed, or emailed:
•
Y - Invoices are sent to the ship-to address on each sales order.
•
N - Invoices are sent to the bill-to address, but the ship-to address for each order is referenced on the invoice.
Non-Restricted U/M
Determines if you can use a unit of measure in Sales Order Entry that is not set as a valid unit of measure in Product maintenance (Valid in SOE flag in a product’s unit of measure table) when you enter a sales order for this customer:
•
Y - You can change the unit of measure to any valid unit of measure in a product’s unit of measure table, when selling products to this customer.
•
N - You can only change the unit of measure if the Valid in SOE flag is set to Y for a product’s unit of measure.
Refer to “Establishing a Unit of Measure” in the Inventory documentation
Best Price
Determines if the best price available is used for this customer’s orders:
•
Y - All price column, matrix prices, and special prices are evaluated to calculate the lowest price possible for this customer.
•
N - The pricing hierarchy is used to determine a price.
This setting at the customer level overrides the setting at the system level. Refer to the Pricing Hierarchy.
Print B/O Notice
If you print backorder notices, this flag determines if backorder notices print when a product for this customer goes on backorder:
•
Y - Backorder notices print.
•
N - Backorder notices do not print.
Refer to Printing Backorder Notices.
Restocking Charge
Determines if a restocking charge is added to each line item on a credit memo or RGA:
•
Y - A restocking charge is calculated for each line item when credit is issued.
•
N - Restocking charges are not calculated for each line when credit is issued.
This flag works with the Soe Restock Charge Method flag in Order Processing Parameters > F5-Add Info > F5 > Add Info.
Email Ship Notice
Determines if the customer receives shipping confirmations when sales order releases are confirmed in Release Confirmation Entry:
•
P - Print shipping confirmations.
•
F - Fax shipping confirmations.
•
E - Email shipping confirmations.
•
Y - Text email shipping confirmations.
•
N - Do not send shipping confirmations.
For more information on setting up your system for shipping confirmations, refer to Sending Shipping Confirmations.
Contact Required
Determines if a contact name is required on sales orders or quotes for this customer:
•
Y - If you leave the Purchasing Contact (Sales Order Header screen) blank, a message warns you that a contact is required.
•
N - A contact name is not required on sales orders or quotes.
B/O On Pick Tickets
Determines if backorders are included on pick tickets:
•
Y - Include backordered items on pick tickets.
•
N - Do not include backordered items on pick tickets.
•
A - Always include backordered items on all pick ticket releases.
•
Blank - Use the flag setting in Order Processing Parameters > F5-Add Info > Print B/O On Pick Tickets.
F2-Save
Saves the customer ordering requirements information.
F5-G/L Prefix Override
Opens the Customer G/L Prefix Override screen to allow you to override general ledger account numbers set up in the general ledger table.
F6-Rental
Opens the Rental Module Screen to allow you to enter rental information for the customer if you are using the Rental Module.
F7-Whsing
Opens the Warehousing Information Screen to enter staging and picking information for the customer if you are using the Warehousing Module.
Action Bar > Permits Information
Opens the Customer Permit Information Screen to allow you to enter permit information if the customer requires DEA, Hazmat, or other permit information.

Auto Reverse Order screen
The Auto Reverse Order screen allows you to reverse a customer invoice so that it is voided out of the system. This option works the same as a credit memo (refer to Credit Memos & RGAs), but may be more useful than a typical credit memo if you want to reverse an entire sales order, rather than just give credit for a single returned item. Refer to Voiding an Invoice.
Menu Path: Order Processing > Sales Order Entry > Action Bar > Auto Reverse
Field and Function Key Descriptions
Field/Function Key
Description
Invoice #
The invoice number for the order that you want to reverse.
Customer
The customer that the invoice is charged to.
Written By
The order taker that entered the sales order.
Invoice Date
The date the order was invoiced.
Invoice Amount
The total value of the invoice.
Credit Reason Code
The credit reason code to explain why the invoice is being reversed. Refer to Creating Credit Reason Codes. The restocking charge percent for the credit reason code is calculated for each line on the order.
Default Return Type
Indicates where the inventory on the sales order will be returned to:
•
D - Damaged inventory.
•
R - Return to stock.
•
S - Scrapped inventory.
•
W - Warranty return inventory.
Default Warranty Tag#
The warranty tag number (if you entered W for Default Return Type). Enter NEW to assign a new warranty tag number.
Shipping Whse Assign
Determines the shipping warehouse:
•
O - Original (default). The shipping warehouse for every line item is the same as the shipping warehouse on the original order.
•
V - Override. Allows you to change the shipping warehouse for the order. Enter the new warehouse number in the following field.
Shipping Warehouse (unlabeled)
The new shipping warehouse number if you entered V for Shipping Whse Assign.
Order Type
Defaults to the 06 confirmation order type, but you can change if necessary. Refer to Sales Order Types.
F2-Save
Reverses the charges on the invoice. Refer to Voiding an Invoice.

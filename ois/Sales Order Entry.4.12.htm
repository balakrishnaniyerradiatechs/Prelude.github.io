Entering Non-Stock Products
The method for entering non-stock products depends on your company policies. Check with your manager or systems administrator for information or how you should handle non-stock products so that inventory and the general ledger are updated correctly. Prelude recommends that you set up an asterisk (*) part number for each vendor for each warehouse to use for non-stock products purchased from each vendor. For example, create the product *KLE for the Klein vendor.
The following examples are commonly used methods for entering non-stock products.
To enter non-stock products on the Sales Order Entry screen:
1
For Product, choose one of the following methods to add products to the order:
•
If the product is non-stock in the shipping warehouse, enter the product number. The Line Item Detail screen appears. Go to step 6.
•
If the product is not a stock product in any warehouse and this is a one time buy, you can enter the product using the asterisk (*) along with the vendor prefix. An asterisk part number should be set up for each vendor for each warehouse. The Product Description screen appears.
2
For Description 1, enter the part number or a description.
3
If necessary, you can enter an extended description in Description 2.
4
Press ENTER. The Cost Prompt screen appears.
5
Enter the Product Cost for the non-stock item.
Price/Cost Prompt
Order Processing Parameters > ‘*’ N/S Price/Cost Prompt Indicates if both list price and cost are required for nonstock (*) items:
•
Y - Prompt for both list price and cost.
•
C - Prompt for cost only.
The Line Item Detail screen appears.
6
Enter the Quantity of the product that the customer is ordering.
7
Verify the U/M (selling unit of measure) and the Whse (shipping warehouse).
8
Enter N for Accept and fill in the following fields as necessary:
•
Qty-Pck and Qty-Shp. The quantity picked and the quantity shipped default to zero for non-stock products. Generally the product is unavailable and must be ordered.
•
Qty-B/O. The quantity backordered generally defaults to the quantity ordered for non-stock products. Refer to the following for information on backorders:
-
To link a backorder for a stock product to an existing purchase order, refer to Sourcing a Backorder to an Existing Purchase Order.
-
To create a purchase order for a backordered product, refer to Linking a Purchase Order to a Backorder.
•
Sc. The supply code indicates where the backorder will be supplied from and varies depending on the product’s replenishment path and product settings. Commonly used options are:
-
B - Purchase order.
-
E - Emergency purchase order.
-
D - Direct ship purchase order.
-
R - Emergency stock transfer.
-
T - Stock transfer.
-
W - Work order.
•
Gross Price defaults to the Suggested Retail/List Price for an asterisk (*) product or to the List Price from Product Maintenance for existing products. Change as necessary.
•
Src. The source code for the price. Refer to Price Source Codes for definitions of the source codes.
•
Tx?. Indicate if the product is taxable:
-
Y - The customer should be taxed for this product.
-
N - The product is tax exempt.
•
Edit the Cost, Gp%, and Profit as necessary. If you change any one of these fields, the other two fields update accordingly.
•
Req-Date, Pck-Date, Promised. Enter the required date, pick ticket date, and the promised date for the product, if known.
9
Press F2-Save to close the Line Item Detail screen. If the non-stock product is an asterisk (*) item, the N/S Additional Information screen appears to allow you to enter more information for the product.
10
For Vendor Product, manually enter the vendor’s product number or use one of the following to print information on the purchase order:
•
D1 - Loads Description 1 that you entered in the Product Description screen (step 1).
•
D2- Loads Description 2 that you entered in the Product Description screen (step 1).
11
If known, enter the Lead Time for the product to help determine when the product should be purchased.
12
Enter the Buying U/M if it is different than the selling U/M.
13
Enter the Weight and Cube volume, if known.
14
Depending on system settings, the supply code for the product, and the product’s availability in other warehouses, a yes/no dialog box may appear to ask if you want to generate a purchase order or a transfer for the product. If so, choose from the following:
•
No - The product goes on backorder and appears in the buyer’s work queue to request a purchase.
•
Yes - Depending on whether a purchase order or transfer is recommended, refer to one the following:
-
Linking a Purchase Order to a Backorder.
-
Linking a Transfer to a Backorder.
15
Continue adding products to the order or press F2-Save to complete the order. Refer to Finishing the Sales Order.

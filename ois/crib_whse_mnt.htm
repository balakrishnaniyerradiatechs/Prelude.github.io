Crib to Warehouse Cross Reference Maintenance screen
When you import sales orders for crib products using the Direct Order Entry module, you must designate the crib or bin information in Crib to Warehouse Cross Reference Maintenance. Refer to Cross Referencing Customer Crib Orders.
Menu Path: Order Processing > Maintenance > Customer Control Files > Customer Crib Maintenance > Crib To Warehouse Maintenance
Field and Function Key Descriptions
Field/Function Key
Description
Crib Number
The number assigned to the crib.
Warehouse
The warehouse associated with the crib.
Bin
The bin number if managing cribs by bin.
Default Department
The default department code to use for sales orders.
Shrinkage Department
The shrinkage department code if shrinkage is set up for a customer ship-to account.
P/O Number
The main purchase order number to use for the crib.
Shrink Acct Code 1
The shrinkage accounting code to use if shrinkage is used for a customer and ship-to record. This accounting code is not general ledger related but can be used for crib reports to find lost inventory.
Crib Type
The company associated with the crib, for example:
•
GM - General Motors.
•
D - Delphi.
•
F - Ford.
F2-Save
Saves the crib cross reference information.
F4-Del
Deletes the crib cross reference information.

Customer Maint Audit Report by Customer
The Customer Maintenance Audit Report by Customer displays any data changes in Customer Maintenance. You can run the report for 1 or more customers.
After you run the report, you can choose to purge the product information so that it is not included the next time you run the report.
Track Changes
Customer Maint Audit in File Update Parameters must be set to Y to track changes in Customer Maintenance.
Menu Path: Order Processing > Reports > Miscellaneous Reports > Customer/Ship-To Reports > Customer Audit Reports > Customer Maintenance Audit Report By Customer
Selection Criteria
Use the following criteria to limit the information to print on the report:
•
Customer. The number for the customer whose changes you want to view.
•
Entry Date Range. The beginning and ending dates that the customer data was edited.
Column Descriptions
Column Heading
Description
Customer Number/Description
The customer name and description.
Type
Indicates if the data was changed for a customer or prospect:
•
C - Customer.
•
P - Prospect.
Ent-Date
The date the customer record was edited.
Ent-Time
The time of day that the customer record was edited.
Userid
The user ID of the person that changed the record.
Description (top line)
The description or name of the user.
Description (second line)
The field name that changed.
Before Value
The data in the field before the change.
After Value
The data in the field after the change.

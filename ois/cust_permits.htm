Customer Permit Information Screen
If you sell products that require a permits from your customer, you can enter and update the customer’s permit and restricted product information in the Customer Permit Information screen. If the customer does not have a restricted product permit, you can add the permit information while in Sales Order Entry. The customer’s record is updated with the information.
Ship-To Addresses
The MSDS/Ship-To Specific flag in Order Processing Parameters determines if this information is stored for a customer at the ship-to level or the bill to level.
If you change the ship-to on a sales order or quote that has a restricted product, the system checks to see if the restriction applies to the new ship-to address.
Refer to Restricting Products for a State or Zip Code for information on setting up restricted product matrixes.
Refer to “Assigning MSDS Numbers to Products” in the Inventory documentation for more information on restricted products.
Menu Path: Order Processing > Maintenance > Customer Maintenance > F7-Order > F5-Add Info > Action Bar > Permits
Field and Function Key Descriptions
Field/Function Key
Description
DEA Permit Number
The customer’s DEA (Drug Enforcement Agency) permit number, if applicable.
Expire Date
The date the DEA permit expires.
On File?
Indicates if the DEA permit is on file in your office:
•
Y - The permit is on file.
•
N - The permit is not on file.
State Permit Number
The customer’s state permit number, if applicable.
Expire Date
The expiration date of the state permit.
On File?
Indicates if the state permit is on file in your office:
•
Y - The permit is on file.
•
N - The permit is not on file.
Restrict Use Permit
Indicates if the customer can purchase restricted products, and which type of restricted products they can buy (restricted products are defined in the Restricted Product field in Product Maintenance > F5-Add Info > F6-MSDS/Permits):
•
Y - The customer can purchase normal restricted products only (Restricted Product set to Y in Product Maintenance). The usage codes that you enter in Usg Codes must match the usage codes set up for the product. Usage codes are set up in the Restricted Usage Code Maintenance Screen.
•
M - The customer can purchase matrix and normal restricted products (Restricted Product set to M or Y in Product Maintenance). They must have a state permit for matrix-restricted products. The customer’s level of restriction must be defined for each state usage code. When you enter a restricted product on a sales order for this customer, this flag is checked in the customer record to see if it matches the codes for the product. If you change the ship-to on a sales order or quote that has a restricted product, the system checks to see if the restriction applies to the new ship-to address. Refer to Restricting Products for a State or Zip Code for information on restricted product matrices.
•
N - The customer may not purchase restricted products.
On File?
Indicates if the restricted use permit is on file in your office:
•
Y - The permit is on file.
•
N - The permit is not on file.
Usg Codes
The usage codes from the Restricted Usage Code Maintenance Screen that define the type of restricted products the customer is certified to buy:
•
If Restrict Use Permit =Y, enter the restricted usage codes. A restricted product must have the same usage code for the customer to be able to buy it.
•
If Restrict Use Permit = M, enter each state where this customer has a license to purchase matrix-restricted products. A sales order’s ship-to state must be defined before the customer can buy a matrix-restricted product.
Permit Holder
The name of the person who is authorized to buy this type of restricted product for the customer. If this name does not exist in this customer’s employee list, then you have the option to add it.
If Restrict Use Permit is set to M or Y or if the Req Permit Holder flag in Restricted Usage Code Mnt is set to Y, then you must enter a permit holder before continuing.
Permit Address Line 1
The first line of address for the permit holder.
Address Line 2
The second line of address for the permit holder.
City
The city of the permit holder.
State
The state of the permit holder.
Zip
The zip code number.
Permit Number
The certificate number that allows the customer to buy this type of restricted product.
Exp-Date
The expiration date of the certificate. The customer may not purchase this type of restricted product after this date.
Level
If Restrict Use Permit is set to M, enter the level of restriction for this customer:
•
0 - No license.
•
1 - General.
•
2 - Restricted.
•
3 - Dealer.
F2-Save
Saves the customer’s permit information.

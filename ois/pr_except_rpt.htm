Price Override Audit Report
The Price Override Audit Report lists all sales orders on which a price was manually entered or changed.
Price Override Reason Codes
If the Price Override Reason Code Prompt (Order Processing Parameters > F5-Add Info > F5-Add Info) is set to Y, you must enter a price override reason code when you override a price on a sales order or quote. This code displays under the first column on the Price Override Audit Report.
Price Override Reason codes are established in Price Override Reason Code Maintenance (Refer to Creating Reason Codes for Price Overrides).
The Price Override Code in User ID Maintenance for your user ID determines if you can override a price. The following options are available:
•
A - You can change the price, even below cost.
•
C - You can change the price, but not below cost.
•
G - You can change the price, but if the gross profit falls below the minimum defined for the product line, the order goes on Administrative Order Hold.
•
N - You cannot access any price fields.
•
R - You can raise the price, but if you lower it at all, the order goes on Administrative Order Hold.
Menu Path: Order Processing > Reports > Daily Order Detail Reports > Price Override Audit
Selection Criteria
Use the following criteria to limit the information to print on the report:
•
Division. The highest possible grouping in your company (e.g., Eastern Division, Western Division).
•
Region. A subset of Division - if Division is used (e.g., Southeast Region, Northwest Region).
•
Department. A subset of warehouse. Typically used to separate expenses within a warehouse (e.g., accounting, sales, warehouse).
•
Cost Center. A subset of Department - if Department is used. Typically used to separate expenses within a warehouse or department (e.g., shipping, receiving, outside sales, inside sales).
•
Warehouse. The selling warehouse. The selling warehouse is the warehouse that receives credit for the sale, regardless of which warehouse ships the merchandise or where the order was entered.
•
Override Date <. The last save date for which you want to include overrides. All price overrides before this date will appear on the report.
•
Customer. The customer numbers to include.
•
Product Type. The product types to include. Choose one or more of the following:
-
B - Finished good & raw material.
-
E - Expense item.
-
F - Finished good.
-
N - Rental.
-
P - Part.
-
R - Raw material.
-
U - Used.
•
Product Line. The product line numbers to include.
•
Product. The products to include.
Column Descriptions
Column Heading
Description
Customer/Rsn Code
Line 1: The customer number.
Line 2: The customer description.
Line 3: The price override reason code from Price Override Reason Code Maintenance, that explains why the sales person changed the price. Only applicable if Price Override Reason Prompt is set to Y in Order Processing Parameters. Refer to Creating Reason Codes for Price Overrides.
Slm/SO#
Line 1: The user ID of the sales person assigned to the order.
Line 2: The sales order number that contains the price override.
#
The number of times the order went through the invoice cycle.
P
Indicates the pick ticket print status:
•
S - The order was suspended and no pick ticket printed.
•
P - The product is available and a pick ticket will print during the next print cycle or when the date to print is reached.
•
Blank - The pick ticket is not ready to print.
I
Indicates the order’s invoice status:
•
Y - The order is flagged for invoicing.
•
H - The order is on hold and cannot be invoiced until released. Refer to the Administrative Invoice Inquiry.
•
Blank - The order is not yet flagged for invoicing.
Product
Line 1: The product description.
Line 2: The second line of description for the product, if applicable.
Ov Date
The date the price was overridden.
C
Indicates if the product is part of a kit:
•
K - Kit parent.
•
C - Kit component.
•
Blank - The product is not part of a kit.
S
Line 1: The line item’s supply code (if one appears on the sales order):
•
B - Purchase order.
•
E - Emergency purchase order.
•
D - Direct ship purchase order.
•
R - Emergency stock transfer.
•
T - Stock transfer.
•
W - Work order.
Line 2: The linked document number, if the product is linked to a purchase order, transfer, or work order.
Pline
The product line number.
#
The quantity of the product originally entered on the sales order.
Typ
Line 1: Unit of measure.
Line 2: The product type:
•
F - Finished good.
•
R - Raw material.
•
B - Both finished goods and raw material.
•
E - Expense item.
•
N - Rental product.
•
U - Used.
•
P - Part.
Orig/Ext.
Line 1: The original price of the item on the sales order before it was overridden.
Line 2: The extended original value of the total quantity of the item (quantity x Price).
Line 3: Currency code, if you use multiple currencies.
Price/Ext.
Line 1: The new price of the item after the override.
Line 2: The extended price of the total quantity of the item after the override (quantity x Price).
Cost/Ext.
Line 1: The cost of the item.
Line 2: The extended cost of the total quantity of the product (quantity x Cost).
GP%/ID
Line 1: The gross profit percent earned using the new price for the product.
Line 2: The user ID of the person who edited the cost.

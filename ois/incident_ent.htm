Quality Incident Entry screen
A Quality Incident is the collection of all feedback entries (refer to Quality Feedback Entry and corrective action entries (refer to Quality Corrective Action Entry) made for a complaint. You can use the Quality Incident Entry screen to tie together all parts of the Quality Inspection Subsystem to maintain quality control on products from manufacturers and to improve service to customers (refer to Collecting Quality Control Incident Information).
When you create a Quality incident, not all of the information may be available to fill in all of the fields. It is normally held open for awhile to allow you to add accumulative feedback and corrective action information as it is received.
Menu Path: Order Processing > Additional Subsystems > Quality Inspection Subsystem > Quality Incident Inquiry
Field and Function Key Descriptions
Field/Function Key
Description
Incident#
The incident number.
Entry Date
The date the incident was entered.
User Id
The user ID of the person that entered the incident.
Status
The status of the incident:
•
O - Open.
•
C - Closed. You cannot close the incident until all corrective actions are completed.
•
I - In process.
•
U - Unassigned.
•
W - Waiting for response.
Description
An explanation of the problem. This may include a combination of complaints all regarding one product or employee.
Incident Type
The type of incident such as customer return or complaint.
Sources
Type
The source type of the incident:
•
F - Feedback entry. For Document, enter the feedback number created in Quality Feedback Entry.
•
I - Inspection result entry. For Document, enter the inspection result number (refer to Inspection Results Entry).
•
R - Return sales order. For Document, enter the credit memo number.
Document
The feedback number, inspection result number, or the credit memo number.
Description
The description of the entry.
Agent
The agent assigned to the incident.
Inspection Type
The type of inspection:
•
Feedback.
•
Inspection.
Corrective Actions
Action#
The corrective action number associated with the incident.
Status
The status of the corrective action:
•
Open.
•
Closed. You cannot close the incident until all corrective actions are completed.
•
In process.
•
Unassigned.
•
Waiting for response.
Agent
The quality inspection agent assigned to the corrective action.
Title
The summary title of the corrective action.
F2-Save
Saves the quality incident information.
F4-Del
Deletes the quality incident information.
F5-Review
Allows you to review the highlighted action or entry.
Action Bar
Notes
Opens the Notes information screen to allow you to enter information recording the action.
Output
Allows you to print, email, or fax the feedback information.

Ship-to Maintenance Audit Report by UserId
The Ship-To Maintenance Audit Report by User Id displays, for each user ID, changes in Customer Ship-To Maintenance.
After you run the report, you can choose to purge the product information so that it is not included the next time you run the report.
Track Changes
Ship-To Maint Audit in File Update Parameters must be set to Y to track changes in Customer Ship-To Maintenance.
Menu Path: Order Processing > Reports > Miscellaneous Reports > Customer/Ship-To Reports > Ship-To Audit Reports > Ship-To Maintenance Audit Report by UserId
Selection Criteria
Use the following criteria to limit the information to print on the report:
•
User Identification. The user ID of the person that edited customer ship-to data.
•
Entry Date Range. The beginning and ending dates that the customer ship-to data was edited.
Column Descriptions
Column Heading
Description
Customer Number
The customer name and description.
Shipto
The customer ship-to number that represents a shipping address.
Type
An S indicates that this is a customer ship-to record.
Ent-Date
The date the customer’s record was edited.
Ent-Time
The time of day that the customer’s record was edited.
User
The user ID of the person that made the change.
Description (top line)
The description or name of the user.
Description (second line)
The field name that changed.
Before Value
The data in the field before the change.
After Value
The data in the field after the change.

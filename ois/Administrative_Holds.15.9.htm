System Level Administrative Order Hold Options
Option parameters define the restrictions that place an order on Administrative Order Hold, and who can release them, at the system level for your company. Administrative hold options set at the customer or product level override the options set at the system level.
To define Administrative Order Hold parameters:
1
Select Order Processing Parameters > F5-Add Info to open the Additional Information screen.
2
For Administrative Order Hold, enter Y to open the Administrative Order Hold Parameters screen.
3
Under AOH Types, enter one or more of the following codes to determine what criteria places an order on Administrative Order Hold:
•
B - Backorder Cycle Allocations: If inventory is allocated to the sales order during the Allocation Procedure or when stock is received (if the purchase order is linked after the sales order is invoiced at least once).
•
D - Override Freight Dollars: The Outbound Freight in the Sales Order Completion screen is overridden.
•
E - Blanket P/O Amount Exceeded: A customer’s blanket P/O amount is exceeded. Refer to Tracking Blanket P/O Information for a Customer.
•
F - Override Freight Terms: The Freight Terms code is changed in the Sales Order Header > Order Additional Information Screen.
•
I - Incompatible Ship Via: If the Ship Via Code is incompatible for the customer (refer to Customer Level Administrative Order Hold Options) or the product (refer to Product Level Administrative Order Hold Options).
•
K - Oversized Order Check: The total of the order is more than the Oversized Order $ Amount field on this screen.
•
L - P/O Generated/Linked Through S/O: The line item is linked to an existing purchase order or if a purchase order is generated through Sales Order Entry.
•
M - Manual: If an order is place on hold Manual Hold in Sales Order Entry (refer to Manually Placing Orders on Administrative Order Hold).
•
P - Price Violations: A price is changed to a value that is lower than the default price, lower than the cost, or below the acceptable gross profit percent. You must also determine whether you want the price violations to appear in the The Administrative Invoice Inquiry or the Administrative Order Inquiry. If you choose to update the Administrative Order Inquiry, the N option is also available:
•
N - Minimum Price Violations: A line item’s selling price is below the product’s Minimum Price (Product Maintenance > F7-Pricing or Product Warehouse Maintenance >F7-Price/Cost). This option is only available if the P option is in effect.
•
S - Ship-Via from Web Requires Authorization: The Ship Via entered on an order placed on the website must be authorized.
•
T - Order With No Tax Jur Codes: The sales order does not contain Tax Jurisdiction Codes in the sales order header Tax Jurisdiction Information Screen.
•
V - Manager High/Low Level Limit GP%: A line item’s gross profit percent is below the Mgr Low Level Minimum GP% (Step 7) or more than the Mgr High Level GP% (step 8). Contract pricing is excluded.
•
W - Web Generated Order: Any order generated from your VIA website.
•
X - RX Product with no RX. A prescription product is entered on a sales order without a prescription number. Refer to Setting up and Tracking Prescriptions.
•
Z - Credit Card Order – No Authorization: A credit card order is entered without an authorization code. Refer to Credit Cards.
4
For Mgr, you can define a manager code for each AOH Type, to determine who is authorized to release orders for this type of hold. This code can be overridden in Customer Maintenance (refer to Customer Level Administrative Order Hold Options).
5
For Max Months Supply For Admin Order Hold, enter the maximum number of months usage the quantity of an item on a sales order can be before it is placed on hold. This option prevents a single sales order from taking all the available stock for a product causing other orders to backorder. The Administrative Hold Manager can decide if the customer is important enough to inconvenience other customers or if the customer can receive only part of the shipment now and the rest later.
6
For Max Months Supply Cutoff Class, define the product class above which a line item is not subject to the Max Months Supply For Admin Order Hold restriction. For example, if the value in this field is 6, then sales orders containing products with an inventory class of 7 through 13 are not checked for the Max Months Supply For Admin Order Hold.
7
If you entered an AOH Type of V, for Mgr Low Level Minimum GP%, define the minimum gross profit percentage allowed for a line item before it is placed on hold. This value can be overridden if a Mgr Low Level G/P% exists for the product line (refer to Product Line Level Administrative Order Hold Options).
8
If you entered an AOH Type of V, for Mgr High Level GP%, define the maximum gross profit percentage that a line item can be before it is placed on hold. This value can be overridden if a Mgr High Level G/P% exists for the product line (refer to Product Line Level Administrative Order Hold Options).
9
If you entered an AOH Type of K, enter the Oversized Order $ Amount. An order will go on hold if the total value is more than this amount.
10
For Admin Hold/unallocate Inv, indicate if you want to unallocate stock from an order on Administrative Order Hold. Choose from the following:
•
Y - If an order goes on hold, the products are un-allocated and backordered.
•
N - If an order goes on hold, all allocations remain.
11
Press F2-Save to return to the Additional Information screen.
12
Press F5-Add Info to open the Additional Options screen.
13
For Suspend Orders From Activities, determine if suspended sales orders can go on credit hold or administrative hold. Choose from the following:
•
Y - Do not place a suspended order or quote on hold, even if it meets the criteria for credit hold or administrative hold.
•
N - If a sales order or quote meets the criteria for credit hold or administrative hold, it will go on hold even if it is suspended.
14
Press F2-Save 3 times to save the Order Processing Parameter Information.

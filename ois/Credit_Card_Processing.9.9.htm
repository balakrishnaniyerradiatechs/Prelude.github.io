Integrating Protobase with Prelude
If you use Protobase with Prelude, you must enter software information to determine where the credit card information is stored. Once you activate Protobase, you cannot access the Authorization Code in Sales Order Entry because the sales are authorized on-line.
To integrate the Protobase software with Prelude:
1
In Order Processing Parameters, select F5-Add Info to open the Additional Information screen.
2
For Credit Bureau Auth Code, select P to use the Protobase on-line verification software. The Add CC Software Info screen opens to allow you to enter software requirements.
3
For Live/Test Account, indicate which account to use:
•
L - Live account.
•
T - Test account (to practice - live data is not affected).
4
Enter the Directory Path that Protobase uses to save data.
5
For Multiple Line Swipe, choose from the following:
•
Y - Use the 2-line screen to capture the full credit card swipe.
•
N - Use the single line screen (recommended).
6
For Archive Data, choose from the following:
•
Y - Save raw data transactions in the UNIX archive folder.
•
N - Do not save data.
7
For Add Markup To Pre-Auth Amt, choose from the following:
•
Y - Add a percent markup to the sales order to account for tax, freight, and handling. You can set up tables in Credit Card Markup Table Maintenance to calculate markups (refer to Adding a Markup to the Pre-Authorization Amount).
•
N - Do not add a percent markup. Pre-authorization is for the amount on the order.
8
For Credit Card Industry Type, choose from the following:
•
R - Retail. The card is swiped.
•
D - Direct marketing, mail order. The card information is given over the phone.
•
Blank - If the card is swiped, it is assumed to be retail. If the card is not swiped, it is assumed to be direct marketing.
9
Press F2-Save to save the Credit Bureau information and to return to the Additional Information screen.

Ship Via Maintenance Screen
The Ship Via Maintenance screen allows you to set up codes to represent shipping methods or carriers. You can assign these codes to sales orders and purchase orders to designate the method of shipment. You can also set up a default ship via code in Customer Maintenance and Customer Ship-To Maintenance as the method most commonly used by the ship-to customer.
Menu Path: Sales Order Entry > Maintenance > Customer Control Files > Ship Via Codes
Field and Function Key Descriptions
Field/Function Key
Description
Ship Via
The one or two-digit alphanumeric code to represent a method of shipping.
Description
A description of the shipping method or carrier name.
Freight Type
Determines how freight is calculated for this ship via code, if you use the Amount and Freight$ in the Freight Charges table on this screen:
•
$ - Freight is calculated based on the dollar amount.
•
# - Freight is calculated based on the weight (in Product Vendor Maintenance).
Class
The code that identifies the shipment class:
•
1 - Next day.
•
1A - Next day AM.
•
1S - Next day AM (Saver).
•
2- Second day.
•
2A - Second day AM.
•
3 - Third day.
•
A - Rate shop all classes.
•
C - Common carrier.
•
D - Delivered.
•
G - Ground transportation.
•
I - International.
•
I1 - International next day.
•
I2 - International second day.
•
N - Next day/Saturday.
•
NA - Next day Saturday AM.
•
W - Will call.
The Daily Business Statistics Report uses this information to accumulate sales information by shipping method.
Charge/
Backorders?
Determines if freight is calculated and charged for backorders:
•
Y - Charge freight on backorders.
•
N - Do not charge freight on backorders.
Incremental Charge?
Indicates if freight dollars are accumulated by incremental value:
•
Y - Charge freight by increment of amount ordered. For example, charge $10 for every $2000 of merchandise.
•
N - Do not charge freight by increment. Charge using the Amount and Freight$ in the Freight Charges table.
Freight Surcharge?
Determines if freight surcharges apply to this ship via:
•
Y - Freight surcharges apply.
•
N - Freight surcharges do not apply.
Common Carrier
Indicates if this ship via type is a common carrier:
•
Y - This ship via represents a company (e.g., Fedex, UPS).
•
N - This ship via is not a company (e.g., our truck, will call).
Pending Freight
Indicates if the system captures information to process pending freight invoices though a freight clearing house (no longer supported by Prelude):
•
Y - Process freight invoices through a freight clearing house.
•
N - Do not process freight invoices through a freight clearing house.
Carousel Emg Pick
Indicates if this ship via code automatically sends a message to the carousel interface that the order is an emergency:
•
Y - This ship via code sends a message to the pickers that the order is top priority.
•
N - This ship via code does not send an emergency code to the carousel interface.
Refer to “Receiving Carousel Non-Nettable Products” in the Inventory documentation for more information.
Truck Route
Determines if this ship via code is used with truck routing or van bins:
•
Y - Orders with this ship via code are considered for truck routing.
•
M - Orders with this ship via code are flagged as mobile (delivery by salesrep) and are used with van bins in Sales Order Entry. Van bins are set up in Bin Location Maintenance (Inventory Module) as Unallocated and are used to identify a salesrep.
When the pick ticket prints, it only displays the customer’s name and the van bin assigned to the order. When the release is confirmed, inventory is moved to the assigned van bin.
•
N - Use neither truck routes nor van bins.
Max Pack Weight
The maximum acceptable weight of each package for this ship via code. If you enter an order with this ship via code and the package weight exceeds this Max Pack Weight, the order goes on Administrative Order Hold.
Max Dimension
The maximum acceptable length, depth, or width for this ship via code. If you enter a product that has a dimension that is larger than the Max Dimension (from Product Vendor Maintenance), the order goes on Administrative Order Hold.
Max Total Weight
The maximum acceptable total order weight for this ship via code. If the total weight of an order exceeds this Max Pack Weight, the order goes on Administrative Order Hold.
Shipper Number
The shipper number from the manifest system or the 3rd party shipper, if applicable.
Service Type
The service type, if you use the manifest system:
•
0 - Same day.
•
1 - Next day standard.
•
1A - Next day AM.
•
2- Second day.
•
2A - Second day AM.
•
3 - Third day select.
•
4 - Commercial ground.
•
4R - Residential ground.
•
ID - International deferred.
•
IE - International express.
•
IS - International standard.
Ship Via Family
The ship-via family code for this carrier. Ship-via family codes are established in the Ship Via Family Code Maintenance Screen to group multiple ship-vias. For example, UPS red, UPS blue, UPS red could be grouped as a ship via family.
Scac Code
The Standard Carrier Alpha Code that identifies the carrier, if applicable. You can set up SCAC codes in the Standard Carrier Alpha Code Maintenance screen (Order Processing > Maintenance > Misc Code Files > Standard Carrier Alpha Code Maintenance).
Freight Charges
Frt Table Limit
The maximum order value that uses the freight table. An order over this amount will not include estimated freight.
Amount
If the total value of the order is this amount or lower, the order qualifies for the corresponding freight charge.
Freight$
The freight amount to charge for the corresponding order amount.
F2-Save
Saves the ship via information.
F4-Del
Deletes the ship via code.
F5-Warehousing
Opens the Warehousing Information Screen. If you use the Warehousing Module, you can enter warehouse picking information.
F6-Add Info
Opens the Ship Via Freight Info Screen. You can enter freight information that is compatible with the ship-via code.
F7-Surch Tbl
Opens the Freight Surcharge Table Screen. You can enter surcharges for freight amounts.
F8-Add Chrg
Opens the Ship Via Additional Charge Screen. You can enter an additional charge to add to orders depending on the merchandise value.
Action Bar
Addtl Optns
Provides access to the following additional options:
•
PSS Manifest Interface Screen. You can enter information for the Parcel Shipping Manifest Interface.
•
Via Overrides Screen. You can enter information regarding your website.
•
Credit Card Options. You can enter a credit card markup table number set up in Credit Card Markup Table Maintenance (Order Processing Controls). Refer to Setting up Your System for Credit Cards.
•
Picking Days. Allows you to enter the picking days for the ship via code for a warehouse. This overrides the picking days in Warehouse Maintenance but can be overridden in Route Code Maintenance. Refer to Printing Pick Tickets for detailed information on pick ticket rules and the hierarchy used to determine when to print pick tickets.

Open Sales Order Summary Report
The Open Sales Order Summary Report lists open customer sales orders that are not yet fully invoiced.
Menu Path: Order Processing > Reports > Daily Order Summary Reports > Open Sales Order Summary
Selection Criteria
Use the following criteria to limit the information to print on the report:
•
Division . The highest possible grouping in your company (e.g., Eastern Division, Western Division).
•
Region. A subset of Division - if Division is used (e.g., Southeast Region, Northwest Region).
•
Department. A subset of warehouse. Typically used to separate expenses within a warehouse (e.g., accounting, sales, warehouse).
•
Cost Center. A subset of Department - if Department is used. Typically used to separate expenses within a warehouse or department (e.g., shipping, receiving, outside sales, inside sales).
•
Selling Warehouse. The warehouse that receives credit for the sale.
•
Order Date Range. The beginning and ending order dates.
•
Customer. The customers for which you want to view open sales orders.
•
Order Type. The sales order types to include on the report. Choose one or more of the following:
-
01 - Regular.
-
02 - Ship complete.
-
03 - Tag & hold.
-
04 - Direct ship.
-
05 - Counter invoice.
-
06 - Confirmation.
-
07 - Regular consignment.
-
08 - Confirmation consignment.
-
09 - Return goods.
-
10 - Repair orders.
-
12 - Invoice when complete.
-
13 - Ship backorder complete.
-
14 - Ship line item complete.
-
15 - Show order.
-
16 - Order projection.
-
17 - Manufacturer’s representative.
-
18 - Crib replenishment.
•
Sort By Cust, W/B, Slsm.The sort order for the report:
-
C - Customer number.
-
S - Sales representative number.
-
W - Written-by user ID.
•
Page Break On SalesRep/WB. Determines if the report should start a new page for each sales representative or written-by user ID:
-
Y - Start a new page for each sales rep number or written-by user ID, depending on which sort option you chose.
-
N - Do not start a new page for each sales representative or written-by user ID.
•
Written By. The user ID person who entered the orders.
Column Descriptions
Column Heading
Description
Customer
The customer number.
Customer Name
The customer description.
Order#
The sales order number.
I
Indicates the order’s invoice status:
•
Y - The order is flagged for invoicing.
•
H - The order is on hold and cannot be invoiced until released. Refer to the Administrative Invoice Inquiry.
•
Blank - The order is not yet flagged for invoicing.
Customer P/O#
The customer’s purchase order number.
Ord Date
The date the order was entered.
Req Date
The date the customer requires the order.
#
The number of times the order went through the invoice cycle.
Order-Total
The total selling price of the order.
Order-Cost
The total cost of the order.
Gross Prof
The gross profit amount earned for the order.
GP%
The gross profit percent earned for the order.

Creating Reason Codes for Price Overrides
If you want to track price overrides on sales orders or quotes, you can create override reason codes in Price Override Reason Code Maintenance. System settings allow you to determine who can change prices and if price override reason codes are required.
Option Parameters
Price override Reason Prompt (Order Processing Parameters > F5-Add Info > F5-Add Info) determines if a price override reason code is required when a price on a stocking item on a sales order or quote is changed:
•
Y - A price override reason code is required when a price is changed.
•
N - A price override reason code is not required when a price is changed.
User Identification Maintenance
Price Override Code (User Identification Maintenance > F6-Add Info) determines if a user can override a price without placing the order on Administrative Order Hold:
•
A - The user can change the price, even below cost.
•
C - The user can change the price, but not below cost.
•
G - The user can change the price, but if the gross profit falls below the minimum for the product line, the order goes on Administrative Order Hold.
•
N - The user cannot access any price fields.
•
R - The user can raise the price above the customer’s default price, but if he/she lowers it below the default price, the order goes on Administrative Order Hold.
Refer to the Administrative Order Inquiry for information on releasing orders from hold.
Related Report
Description
Price Override Audit Report
All sales orders on which a price was manually entered or changed.
 
To create price override reason codes:
1
Select Order Processing > Maintenance > Misc Code Files > Price Override Reason Code Maintenance to open the Price Override Reason Code Mnt screen.
2
For Price Override Reason Code, enter a two digit code to use on sales orders and quotes to identify a price override reason.
3
Enter the Description of the override code.
4
Press F2-Save to save the reason code information.

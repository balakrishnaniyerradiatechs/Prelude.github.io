Catalog Order History Report
If you use catalogs to sell your products, you can use the Catalog Order History report to view the products sold by vendor and catalog number. Once you create a catalog number (refer to Creating a Catalog ID Code) to identify a catalog, you can assign it to part numbers in Product Maintenance > F6-Xref-Notes > Alternates > Catalog Id.
Menu Path: Order Processing > Reports > Monthly Detail Reports > Catalog Order History Report
Selection Criteria
Use the following criteria to limit the information to print on the report:
•
Invoice Date Range. The beginning and ending invoice dates to include.
•
Primary Vendor. The primary vendor assigned to a product in Product Vendor Maintenance.
•
Catalog Id Code. The catalog assigned to a product in Product Maintenance > F6-Xref/Notes > Alternates. Catalog Id numbers are created in the Catalog ID Code Maintenance screen.
Column Descriptions
Column Heading
Description
Vend#
The vendor number.
Product Number/ Description
Line 1: The product number.
Line 2: The product description.
Vendor Product Number
The vendor’s product number, if applicable.
Alternate Product #
The alternate product number if applicable.
Cat ID
The catalog number created in Catalog ID Code Maintenance, if applicable. Refer to Creating a Catalog ID Code for more information.
Total Units
The total quantity invoiced.
Total Net Price
The total selling price for the entire quantity of the product (Total Units x Price).
Total Cost
The total cost for the entire quantity of the product (Total Units x Cost).
Stk UOM
The stocking unit of measure set up for the product in Product Maintenance.
Qty in Stk UOM
The quantity per unit of measure as defined in Product Maintenance for the Stock unit of measure. Defaults to 1 if a unit of measure table does not exist.

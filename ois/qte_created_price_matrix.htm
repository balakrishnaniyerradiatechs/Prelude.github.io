Quote/Order Created Price Matrix screen
If you entered a quote or sales order for a customer and you want to save the pricing information as a matrix, you can use the Quote/Order Created Price Matrix to create a price matrix for future use. You may only use this feature for price matrix types 1, 2, 6, 7, and 19. The system copies the quote information and allows you to modify it as necessary.
Menu Path: Order Processing > Maintenance > Pricing Structures > Price Matrix Maintenance > Quote/Order Created Price Matrix
Field and Function Key Descriptions
Field/Function Key
Description
Quote
The quote number to use for the price matrix information.
Order
The order number to use for the price matrix information.
Matrix Type
The type of matrix to create:
•
1 - Customer/Product.
•
2 - Customer/Customer Product Number.
•
6 - Customer Price Group/Product.
•
7 - Customer Price Group/Customer Product Number.
•
19 - Customer Ship-to/Product.
Matrix Option
Determines how the matrix is updated:
•
O - Overwrites the target matrix with the source matrix. The original target matrix is deleted. You are warned if the target matrix exists and allowed to exit without overwriting.
•
I - Inserts the source matrix at the beginning of the target matrix and retains the existing target price matrix data. You can either insert the entire source matrix or a single line of the source matrix. You may want to use this option to update promotional or temporary pricing.
•
A - Appends the source matrix to the end of the target matrix and retains the existing target price matrix data. You can either append the entire source matrix or a single line of the source matrix into the target matrix. You may want to use this option to update future prices that are effective after the existing matrix expires.
•
D - Deletes all the price matrices for the customers defined only if the same combination of matrices exists for the customer or customer price groups. For example, to delete a price matrix for a product, the customers must have the same product on their matrices as the source matrix. Only records with the same product are deleted. The price matrices that have range numbers as part of the matrix must have the same range number for the customers or customer price groups to allow you to delete them.
Price Method
Enter $ to update as a dollar amount. The net price from the quote is used for the matrix.
Start Date
The date the price matrix becomes effective.
Expiration Date
The date the price matrix is not longer valid.
Warehouse
The warehouse number if the matrix is for one warehouse only. Otherwise, leave blank to include all warehouses.
Job/Customer P/O
The customer job number or purchase order number if the matrix only applies to a specific job or purchase order.
Contact
Indicates if you want to create the price matrix as a contract:
•
Y - Create the matrix as a contract.
•
N - Do not set the matrix up as a contract.
Contract Number
The contract number to assign to the contract, if you entered Y for Contract.
Override
Indicates if the matrix price can be overridden on a sales order:
•
Y - The price can be overridden on a sales order.
•
N - The price cannot be overridden on a sales order.
•
H - The price can only be changed to a higher price on a sales order.
F2-Create Matrix
Creates the price matrix.

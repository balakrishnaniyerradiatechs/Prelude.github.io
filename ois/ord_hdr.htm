Sales Order Header screen
The Sales Order Header screen allows you to enter the header information for a sales order. Most of the fields on this screen default from the customer’s account but you can change the information as necessary. Refer to Entering Sales Order Header Information.
Menu Path: Order Processing > Sales Order Entry > F5-Hdr Scr
Field and Function Key Descriptions
Field/Function Key
Description
Order Type
The sales order type that defines the shipping and backorder conditions. Available types may include:
•
01 - Regular.
•
02 - Ship Complete.
•
03 - Tag and Hold.
•
04 - Direct Ship.
•
05 - Over the Counter.
•
06 - Confirmation.
•
07 - Consignment.
•
08 - Confirmation Consignment.
•
09 - RGA.
•
10 - Repair.
•
12 - Invoice When Complete.
•
13 - Ship Backorder Complete.
•
14 - Ship Line Item Complete.
•
15 - Show.
•
16 - Order Projection.
•
17 - Manufacturer Rep.
•
18 - Crib Replenishment.
Refer to Sales Order Types for a complete explanation of each order type.
Purch Contact
The person at the customer’s location who is purchasing the material. If you use Contact Management, you can track sales information by purchasing contact.
Ship-To
The customer’s ship-to address. If this is not correct, choose from the following:
•
Enter an existing ship-to addresses for this customer.
•
D - Duplicate the customer’s main address for the ship-to address. This is the default if there are no ship-to addresses set up for the customer.
•
M - The Ship-To Address Screen appears to allow you to manually enter a new, one-time address for this order.
•
P - Customer pick-up.
•
C - Use the purchasing contact address for the ship-to address.
•
CP - End user pick-up.
Contact
The name of the person placing the order. The default comes from the Customer Contacts Screen (Customer Maintenance > F8-Contact) if one is available. Otherwise it defaults from the Contact name on the Customer Maintenance screen.
Customer P/O#
The customer’s purchase order number. This field may be mandatory and may require a pre-determined format depending on how Require P/O Number is set in the customer’s Order Information Screen.
Cust Release#
The customer release number. This field may be mandatory depending on how Require Release Number is set in the customer’s Order Information Screen.
Job Number
The number assigned to the job if you use Job Accounting or Progress Billing to track job numbers. A job number may be mandatory depending on how Require Job Number is set in the customer’s Order Information Screen.
Ship Via
Indicates how the customer will receive the material. Ship via codes are set up in the Ship Via Maintenance Screen. Incompatible ship vias for a customer are set up in the Customer Maintenance Incompatible Ship Via Maintenance Screen. or the Customer Ship-to Maintenance Incompatible Ship Via Maintenance Screen.
Freight Terms
The freight terms code that determines if inbound or outbound charges apply to the sales order when it is confirmed in Release Confirmation Entry. Freight codes are established in the Freight Terms Maintenance Screen and assigned to the customer the Freight Information Screen.
Payment Terms
The accounts receivable payment terms code for sales orders that indicates when payment is due, payment conditions, and any applicable discounts. These codes are set up in A/R Terms Maintenance and assigned to customers as a default in the Customer Maintenance Screen.
Email Address
The email address for the associated Contact.
Fax
The fax number for the customer. The default comes from the contact for the customer if one exists in the Customer Contacts Screen. Otherwise the fax number on the Customer Maintenance Screen is used.
Phone
The phone number for the customer.
Ext
The phone extension number.
Sell Warehouse
Defaults to the warehouse established for this customer in the Customer Maintenance Order Information Screen. You can change the selling warehouse if you want a different warehouse to receive credit for the sale.
Ship Warehouse
Defaults to the warehouse established for this customer in the Customer Maintenance Order Information Screen. You can change the shipping warehouse if you want the material to ship out of a different warehouse.
Written By
Defaults to the person logged on to the terminal.
Salesrep One
The primary sales representative assigned to this customer. Sales representatives are defined in the Sales Representative Maintenance Screen.
Salesrep Two
The secondary sales representative assigned to this customer (e.g., an inside salesrep). Sales representatives are defined in the Sales Representative Maintenance Screen.
Entry Date
The date this order is first entered.
Order Date
Defaults to today’s date. You can change it if the customer ordered the material on a different date.
Required
The date the customer requested to receive the material.
Pick Ticket
The date the pick ticket will print. This date is calculated according to how you set maintenance options for pick tickets (refer to Setting Pick Ticket Rules). If you use truck routes, this field automatically updates.
Order Source
The order source that indicates how you obtained this order. Commonly used examples are:
•
C - Customer call in.
•
D - Direct order entry.
•
E - Electronic data interchange.
•
K - Special order-telemarketing.
•
L - Sales representative.
•
M - Mail in.
•
O - Default order source if one is not set in Customer Maintenance or Customer Ship-to Maintenance.
•
R - Remote order entry.
•
S - Sales force automation.
•
T - Telephone sales.
•
W - Will call.
•
X - Special order.
Refer to Defining Order Source Codes for information on creating order source codes.
Route Number
If you use delivery trucks and follow delivery routes, you can enter a route number set up in the Route Maintenance Screen that contains valid delivery date and time information for each route.
The default comes from the customer’s Customer Route Maintenance Screen or the Ship To Route Maintenance Screen, depending on how the order is entered.
Delivery Seq
If you entered a Route Number, this number indicates where in the delivery sequence the customer will receive the order.
Currency
The code that indicates the type of currency that this customer uses, if you use multiple currency. The default is set in the customer’s A/R Information Screen.
Curr Rate
The currency exchange rate.
F2-Save
Opens the Sales Order Entry screen. Refer to Entering Sales Order Line Items.
F5-Add Info
Opens the Order Additional Information Screen to allow you to enter or view sales representative, freight, shipping, and payment information for the order.
F6-Ship-To
Opens the Ship-To Address Screen to allow you to verify or change the shipping address.
F7-Tax Jur
Opens the Tax Jurisdiction Information Screen to allow you to view or edit tax jurisdictions and tax exempt information.
F8-Instructions
Opens the Instructions screen to allow you to enter any special instructions for the sales order, if applicable. Press F3 to view a list of existing standard instruction messages (refer to Creating Standard Instructions). You can also set up standard instructions for each customer to default for all of their sales orders (refer to the Customer Maintenance Instructions Entry Screen).
Disp. Shp.Inst. In SOE/QOE in Order Processing Parameters determines if the instructions screen automatically displays when the header screen comes up in Sales Order Entry and Quotation Entry.
•
A - Always display the Instructions screen.
•
N - Never display the Instructions screen.
•
Y - Only display the Instructions screen if instructions exist.
F9-Credit Info
Opens the Credit Information Screen to display the credit limit and A/R balance for the customer.
Action Bar - Add Info
Route Info
Opens the Order Route Information screen to allow you to enter delivery times and requirements for delivery routes.
Repairs
On Type 10 repair orders, this opens the Repair Order Type Screen to allow you to select the repair order type to determine payment.
Consignment
On consignment orders, this key opens the Consignment Types screen to allow you to define consignment information for the order.
Fut Commitments
Opens the Future Commitment Override screen to allow you to commit inventory for future orders.
Rental
Opens the Rental Information screen to allow you to enter billing information for rental orders.
Commission Split
Opens the Commission Splits screen to allow you to enter two or more sales representatives who will split the commission for this sales order. Refer to Splitting Commission Between Salesreps for more information.
Commission Override
Opens the Commission Header Override screen to allow you to override the method for calculating commission for the sales representative assigned to the order. Refer to Overriding Commission For a Salesrep for more information.

Additional Credit Card Information screen
The Additional Credit Card Information screen appears when you enter a Pay Type of 03 (bank card) in the Cash Application screen when completing a credit card sales order (refer to Pre-Authorizing a Sales Order). Depending on system settings and company procedures, you can use one of the following methods to obtain credit card information:
•
If the order has credit card Payment Terms and default credit information (refer to Setting up Your System for Credit Cards), the Credit Card information defaults in the appropriate fields.
•
If no credit card information exists, enter the Credit Card Type (e.g., Visa, M/C), the Credit Card Number, and the Expiration Date.
•
If you have a credit card swipe machine, press F6-Swipe Card and swipe the card to fill in the information.
Menu Path: Order Processing > Sales Order Entry > Complete > Cash Application screen > Pay Type = 03-Bank Card
Field and Function Key Descriptions
Field/Function Key
Description
Credit Card Type
The type of credit card (e.g., Visa, Master Card, American Express). These are set up in Credit Bureau Maintenance (refer to Assigning Credit Bureau Codes).
Credit Card Number
The customer’s credit card number.
Expiration Date
The expiration date shown on the credit card.
CVV Number
The 3-digit code on the back of the card that is used by some companies to prevent theft. Not stored in the system.
Authorization Code
The approval code from the credit card authorization company. This code appears after you press F5-Authorize to obtain authorization from the credit card verification company (refer to Pre-Authorizing a Sales Order). You can also manually enter the authorization code (refer to Manually Adding an Authorization Code).
Address Line One
The first line of address for the card user if you use the AVS (address verification service). Once an AVS response is received, this information is discarded. If you do not wish to use address verification, leave blank.
Zip Code
The zip code of the card user if you use AVS.
Card Id#
The purchase card ID number, if applicable. This is an additional ID for American Express or Visa cards to define the user of a corporate card. For example, a company with multiple stores such as Home Depot may have one account but each location could have an additional ID number.
Description
A code that is passed on to credit card companies for Protobase. This information is used by American Express purchase cards and allows you to enter 4 lines of text stating what the purchase was for and prints on the statement. These fields are only used with Level II reporting.
F2-Save
Saves the credit card information and returns to the Cash Application screen.
F5-Authorize
Sends the authorization request to the credit card authorization company. If the authorization is successful, the approval code information appears. This code guarantees the credit card for 3 to 30 days (depending on the customer’s bank) and at the same time deducts the amount of the sales order from the cardholder’s available-to-buy. At this point the credit card company is liable. If the transaction is not post-authorized before the approval code expires, then the pre-authorization code becomes void and the sales order amount is released back to the card holder’s available-to-buy balance.
F6-Swipe Card
Allows you to swipe the credit card to fill in the credit card information (if you have a credit card swipe machine).

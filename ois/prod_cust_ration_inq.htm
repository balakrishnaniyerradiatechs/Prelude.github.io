Customer Rationing Inquiry
The Customer Rationing Inquiry displays rationed products and quantities for a customer. Sales order rationing allows you to limit the quantity of a product or product line that a customer can buy within a designated number of days. For example, if the customer’s ration quantity of an item is 100 units and the ration days for that item is 30, the customer can purchase up to 100 units of that item within 30 days. Ration comments appear on the sales order when the ration quantity is exceeded. Refer to Entering Rationed Products.
Ration types are set up in Rationing Maintenance (refer to “Customer Ration Types” in the Inventory documentation) and assigned to customers in the Customer Maintenance Order Information Screen.
Sales Order Rationing in Option Parameters > Order Processing Parameters must be set to Y to use Rationing Maintenance.
Viewing Customer Rationing Information
To view rationing information for a customer/product:
1
Select Order Processing > Inquiries >Customer Info Inquiries > Customer Rationing Inquiry to open the Customer Rationing Inquiry screen.
2
Enter the Customer #.
3
Enter the Product Number that is rationed.
4
Enter the Vendor# that the rationed product is purchased from.
Rationing information for the customer/product/vendor appears.
Field & Function Key Descriptions
Field/Function Key
Description
Customer Ration Type
The ration type code assigned to the customer in the Customer Maintenance Order Information Screen. The ration type codes define the quantity a customer can buy and are established in Rationing Maintenance in the Inventory module. All available type codes display under Cust Ration Type.
Start Date
The date that rationing begins for this product/vendor.
End Date
The date that rationing ends for this product/vendor.
Ration Days
The number of days the rationing cycle is in effect. The customer can only purchase the ration quantity during this cycle. Once the number of days ends, the next ration cycle starts and the ration quantity is available again. The number of ration days keeps cycling until the End Date is reached.
Cust Ration Type
The type codes (from Ration Type Code Maintenance) to correspond to each Ration Qty. These type codes are used in Ration Type in Customer Maintenance.
RationQty
The ration quantity associated with each Customer Ration Type. For example, type A could be setup with a ration quantity of 5000, type B with a quantity of 7500.
Ration Dates
The beginning and ending dates for the ration cycle. The length of each ration cycle is determined in Ration Days.
Used Qty
The quantity of the product that the customer has used so far in this ration cycle.
F2-Save
Closes the inquiry screen.

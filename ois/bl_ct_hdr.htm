Header Information screen
The Header Information screen in Blanket Contract Maintenance allows you to enter the terms and date information for a blanket contract.
Menu Path: Order Processing > Maintenance > Pricing Structures > Blanket Contract Maintenance > Blanket Contract Maintenance > F5-Hdr Scr
Field and Function Key Descriptions
Field/Function Key
Description
Description
The description of the contract to display on reports and screens when the contract number is entered.
Start Date
The date the prices will take effect. The default is today’s date.
Expiration Date
The date the contract will expire.
Warning Days
The number of days before the contract expiration date that you want a warning to display for sales orders with this contract. Leave blank if you do not want a warning message to appear.
Contract Amount
The negotiated amount for the contract. When the customer’s purchases against the blanket contract reach this value, a warning message displays in Sales Order Entry. Leave blank if you do not want a warning message to appear.
Terms Code
The default A/R terms code to use for the customer when using this contract. These terms override the terms in Customer Maintenance.
Ship Via Code
The default ship via code to use for the customer’s orders when this contract is used. This code overrides the ship via code for the customer in Customer Maintenance.
Freight Terms
The default freight terms to use for the customer’s orders when this contract is used. These terms override the terms in the ship via code for the customer in Customer Maintenance.
Supplying Warehouse
The Supplying Warehouse to designate which warehouse will use the Beginning On Hand (BOH) / Normal Order Quantity (NOQ) values for purchasing. This only applies for custom purchasing. Leave blank for standard software.
Auth. Add. Products
Determine if products can be added to a sales order if they do not exist on the blanket contract. Choose from the following:
•
Y - Allow products on a sales order that do not exist on the blanket contract.
•
N - Do not allow products on a sales order that do not exist on the blanket contract.
Auth. Price Ov.
Determines if the sales price for the products can be overridden on a sales order.
•
Y - Allow the sales price for the product to be overridden on the sales order.
•
N - Do not allow the sales price for the product to be overridden on the sales order.
Auth. BOH/NOQ Mnt.
Determines if the Beginning On Hand / Normal Order Quantity values are maintained in a custom maintenance program:
•
Y - Maintain Beginning On Hand / Normal order Quantity values for a custom purchasing program.
•
N - Use standard software.
Accept Backorder
Determines if the customer accepts backorders when purchasing the products on the blanket contract. Choose from the following:
•
Y - The customer accepts backorders.
•
N - The customer does not accept backorders.
F2-Save
Returns to the Blanket Contract Maintenance screen.
F5-Add. Cust
Opens the Blanket Contract Customers screen to allow you to add sub-accounts if the customer is set up as a master account. Refer to
Creating a Blanket Contract for information on sub- accounts.

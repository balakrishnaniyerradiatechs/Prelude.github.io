Shipping Sales Orders Complete
A ship complete order is used to make sure that merchandise does not ship to the customer until the entire quantity of a product or the entire quantity of an order is available. You can use one of the following methods for ship complete orders:
•
The 02 Ship Complete order type will not ship any products on an order until the entire order is available.
•
The 14 Ship Line Item Complete order type will not ship an individual line item until the full quantity of just that line is available. It is similar to an 01 order type, except that Ship Line Item Comp in the Line Item Detail Additional Information screen for each line item is automatically set to Y.
•
You can also use an 01 Regular order type and then manually change Ship Line Item Comp in the Line Item Detail Additional Information screen to Y for any line item.
System flags determine when pick tickets or staging documents print. A staging document contains the customer name and “Ship Complete” instead of the ship-to address or shipping instructions so that warehouse personnel cannot attempt to ship the incomplete order. As the backordered material is received, it is allocated to the order and additional staging tickets print (releases are not created). When the last item is received and the order or line is ready to ship complete, the final pick ticket prints with a ship-to address.
Additional Option Parameters
No Stage Doc on Ship Complete: Determines if staging documents print for line ship complete orders:
•
Y - Print a pick ticket when all line items are available.
•
N - Print only staging documents as line items are updated with available quantity and then a pick ticket when all line quantities are available.
Order Processing Parameters
Print B/O on Pick Tickets: Determines if backordered line items are included on pick tickets:
•
Y - Include backordered line items on the initial pick ticket only.
•
N - Do not include backordered line items on pick tickets.
•
A - Include backordered line items on all pick tickets.
Print B/O Staging Doc: Determines if backordered line items are included on staging documents:
•
Y - Include backordered line items on the initial staging document only.
•
N - Do not include backordered line items on staging documents.
Ship Complete By Warehouse: Determines if you can enter a different warehouse for each line item on a type 02 (ship complete) sales order:
•
Y - Allow entry of multiple warehouses for line items on type 02 sales orders.
•
N - Do not allow entry of multiple warehouses for type 02 sales orders. A message appears to let you know that you cannot change the warehouse.
A ship complete sales order behaves differently depending on availability of inventory and your system staging flags:
•
If all of the inventory is available when the sales order is flagged for pick ticket printing, a ship complete pick ticket prints for a regular order.
•
If some of the inventory is unavailable and No Stage Doc on Ship Compete in Option Parameters is set to N, the unavailable inventory is backordered and the available inventory is staged. If your company stages partial orders in hold areas (refer to Using Warehouse Hold Areas for Tag & Hold Orders) until they are complete and ready to ship, a staging ticket prints for only the available quantities.
•
If some of the inventory is unavailable and No Stage Doc on Ship Compete in Option Parameters is set to Y, the unavailable inventory is backordered and the available inventory is allocated. If your company does not stage ship complete orders (e.g., because of bulky material, insufficient space for staging, small orders), staging tickets do not print. When the order is entered, available inventory is committed to the order and unavailable inventory is backordered. As the backordered material is received, it is allocated to the order. After all material is received and available to pick, a complete pick ticket prints and a release is created.
The 02 Ship Complete Order Type takes precedence over the shipping warehouse matrix (refer to Overriding Shipping Locations for Products).

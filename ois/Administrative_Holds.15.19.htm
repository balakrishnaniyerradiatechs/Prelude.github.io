User Level Administrative Invoice Hold Options
You can set restrictions by user ID to place orders on Administrative Invoice Hold.
To define Administrative Invoice Hold options for a user:
1
Select User ID Maintenance > F6-Add Info to open the Additional Information screen.
2
For Price Override Code, determine if this user can override a price on a sales order:
•
A - The user can change the price, even below cost.
•
C - The user can change the price, but not below cost.
•
G - The user can change the price, but if the gross profit falls below the minimum for the product line, the order goes on Administrative Invoice Hold.
•
N - The user cannot access any price fields.
•
R - The user can raise the price above the customer’s default price, but if he/she lowers it below the default price, the order goes on Administrative Invoice Hold.
3
Press F2-Save to return to the User Identification Maintenance screen.
4
Press F5-Administrative to open the Administrative Setting screen.
5
For Credit Limit Amount, choose from the following:
•
The largest credit memo or RGA amount that the user can enter without the order going on Administrative Invoice Hold.
•
Blank - No credit memos or RGAs will go on invoice hold.
•
0 (zero) - All credit memos or RGAs will go on invoice hold.
6
Press F2-Save twice to save the invoice hold information.

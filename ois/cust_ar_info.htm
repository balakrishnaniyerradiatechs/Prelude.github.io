A/R Information Screen
The A/R Information screen allows access to accounts receivable information (e.g., credit limit, payment terms and conditions, collection information).
Menu Path: Order Processing > Maintenance > Customer Maintenance > F6-A/R
Field and Function Key Descriptions
Field/Function Key
Description
Auto Credit Review
Determines if this customer is placed on credit hold by the Automatic Credit Review Process which is run periodically:
•
Y - The customer is placed on or off credit hold by the Auto Credit Hold Procedure (refer to “Automatically Placing an order on Credit Hold” in the Accounts Receivable documentation). If an invoice is more than the number of days past due indicated in Auto Credit Days, and not flagged as disputed, then this flag is automatically changed to H (automatic credit hold). You can temporarily take the customer off hold by changing the H back to a Y until the Auto Credit Hold Procedure runs again. Other settings such as credit limits may affect this flag also.
•
H - The customer is on automatic credit hold.
•
N - The customer is not included in the Auto Credit Hold Procedure.
Auto Credit Days
The number of days past due the customer’s payments must be before they are subject to the Auto Credit Hold Procedure. Auto Credit Review must be set to Y. This number overrides the Days Past Due set in Accounts Receivable Parameters on the Systems menu.
Credit Limit
The customer’s credit limit. When you enter a sales order for this customer, if the total value of the customer’s A/R balance and open sales orders (excluding long-term future orders) exceeds the credit limit, the order goes on credit hold.
Credit Code
The credit control code that determines if and when orders can be entered for the customer:
•
A - You must enter an Authorization Code in Sales Order Entry after you enter the customer number and before you enter the header information. The authorization codes are set up in Accounts Receivable Controls.
•
E or blank - Sales orders are subject to the customer’s Credit Limit. If the customer is over their credit limit, the pick ticket will not print until the order is released from credit hold.
•
H - Sales orders can be entered but are always held until the credit manager releases them. You must enter the credit reason abbreviation code (established in Valid Response Maintenance).
•
N - No sales orders or quotes can be entered for this customer.
Credit Reason Code
(unlabeled field)
If Credit Code is set to H, enter the credit reason code (from Valid Response Maintenance) that explains why this customer’s orders always go on hold. If no codes exists, the code MAN (manual) is used.
Credit Manager ID
The credit manager ID override for this customer. This ID overrides the credit manager ID for the warehouse.
Admin Hold Override
Determines if the customer’s sales orders and quotes should always or never go on Administrative Order/Invoice/Quote Hold, regardless of other settings:
•
B - Always place sales orders for this customer on both Administrative Invoice and Administrative Order Hold.
•
I - Always place sales orders for this customer on Administrative Invoice Hold.
•
N - Never place an order for this customer on hold. You can still manually hold orders while in Sales Order Entry. Refer to Manually Placing Orders on Administrative Order Hold.
•
O - Always place sales orders and quotes for this customer on Administrative Order hold.
•
Q - Always place quotes for this customer on hold.
Refer to Administrative Hold Inquiries for more information.
Service Charge %
The percent to use to calculate service charges on overdue invoices, if applicable. This percent is multiplied by the total invoice amount to determine the service charge.
Service Charge Days
The number of days past due a customer’s payment on an invoice must be before a Service Charge % is added. This number overrides the Current Days Limit in Auto Service Charge Entry.
Master Account Flag
Determines if this customer is a master account:
•
Y - This customer is a master account and one or more sub-accounts can be linked to this account. This flag cannot be changed to N unless the sub-accounts are unlinked. You might use this if you set up a blanket contract for a customer (refer to Blanket Contracts) and you want to use the special pricing for sub-accounts assigned to this customer.
•
N - Not a master account.
The Master Account Inquiry lists all existing sub-accounts for a master account.
Master Account
The master account number if this customer is a sub-account. Both the sub-account and the master account can be updated for sales, returns, and gross profit. If you use blanket contracts and you want this customer sub-account to use the blanket contract pricing, you must enter the master account number here. Refer to Creating a Blanket Contract for more information on adding customers to contracts.
Master Acct Invoice
If this customer is a sub-account and a Master Account is assigned, this flag determines which account is updated with accounts receivable information and where invoices are sent:
•
N - The sub-account is updated with A/R information and receives the invoices.
•
Y - The master account is updated with A/R information and receives the invoices.
•
A - The master account is updated with A/R information, but the sub-account receives the invoices.
You can override this flag in Sales Order Entry > F5-Hdr > F5-Add Info.
Upd Sales For Mast/Child in Order Processing Parameters > F5-Add Info determines if sales history information is updated for both the master and child accounts or just the child account.
D&B Rating
The Dun and Bradstreet rating, which is a combination of the customer’s net worth and composite credit appraisal. The composite credit appraisal helps determine the credit worth of the company.
D&B Date
The date the D&B rating was established.
D&B Number
The D-U-N-S (Data Universal Numbering System) number assigned to this customer by D&B. This 9-digit identification sequence is recognized internationally in EDI and global e-commerce transactions.
D&B PayDex Score
The score (out of 100 points) that rates this customer’s payment history.
Currency Indicator
The country code that indicates the type of currency that this customer uses, if you use multiple currency.
Override Exch Rate
The negotiated currency exchange rate for this customer, which overrides the Daily Exchange Rate or the Currency Rate Table that normally applies to sales orders. This field is only accessible if the Currency Indicator is not your native currency.
Admin Order Mgr ID
The Administrative Order Manager for this customer, if necessary. This ID overrides the Administrative Order Manager in Warehouse Maintenance. Refer to the Administrative Order Inquiry for information on releasing orders from hold.
Admin Invoice Mgr ID
The Administrative Invoice Manager ID for this customer, if necessary. This ID overrides the Invoice Manager ID for the warehouse. Refer to the Administrative Invoice Inquiry for information on releasing orders from hold.
Admin Qte Mgr Id
The Administrative Quote Manager for this customer, if necessary. This ID overrides the Administrative Quote Manager in Warehouse Maintenance. Refer to the Administrative Quote Inquiry for information on releasing orders from hold.
Oversize Order Value
The order value that is too large for this customer. Any orders over this amount will go on credit hold.
The system uses the following hierarchy to determine the oversize order value that puts an order on credit hold:
•
1st level: Customer Maintenance > F6-A/R > Oversize Order Value.
•
2nd level: Warehouse Maintenance > F5-Add info > Oversize Order Value.
•
3rd level: Order Processing Parameters > F5-Add Info > F5-Add Info > Oversize Order Value.
F2-Save
Saves the accounts receivable information.
F5-Add Info
Opens the Additional A/R Information Screen to access statement requirements and other customer information.
F6-Comments
Allows you to enter notes or comments regarding this customer:
•
Y - The Credit Comments screen appears to allow you to enter credit comments for this customer.
•
N - The Standard Notes Display screen appears to allow you to view or edit notes regarding the customer’s A/R information.
Edit Credit Comments in Accounts Receivable Parameters indicates how notes are entered and viewed in Accounts Receivable Inquiry, Customer Invoice Inquiry, and Customer Maintenance.
•
Y - Enter free form credit comments.
•
N - Use the standard Notes Entry screen for credit comments.
F7-Credit Card
Opens the Credit Card Information Screen to access credit card information.
F8-Floor Plan
Opens the Floor Planning Maintenance Screen to access financial floor plan information.
Action Bar
Security Access
Opens the Security Access Maintenance screen to allow you to enter access codes so that only users that have the same security access code can view this customer.
Only accessible if Cust Security Access in Order Processing Parameters is set to Y or A.
You can create security codes in Security Access Code Maintenance (Systems menu > Miscellaneous Controls) and then assign the codes to a user in User Identification Maintenance > F6-Add Info > F5-Cust Sec.
Cred Ref
Opens the Financial References Screen to access credit application review and reference information.
Cred Ref Inq
Displays credit reference information for the customer.
Cash Rec Notes
Allows you to enter cash receipts comments for this customer.

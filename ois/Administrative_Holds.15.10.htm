User Level Administrative Order Hold Options
You can set restrictions by user ID to place an order on Administrative Order Hold for price overrides.
To define Administrative Order Hold options for a user:
1
Select User ID Maintenance > F6-Add Info to open the Additional Information screen.
2
For Price Override Code, determine if this user can override a price on a sales order:
•
A - The user can change the price, even below cost.
•
C - The user can change the price, but not below cost.
•
G - The user can change the price, but if the gross profit falls below the minimum for the product line, the order goes on Administrative Order Hold.
•
N - The user cannot access any price fields.
•
R - The user can raise the price above the customer’s default price, but if he/she lowers it below the default price, the order goes on Administrative Order Hold.
3
Press F2-Save 2 times to save the price override information.
The Price Override Audit Report displays all overridden prices.

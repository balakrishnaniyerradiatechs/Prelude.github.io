Inspection Results Entry
Once a product is inspected, use Inspection Results Entry to record the results and to identify the document where the product was first found to be defective. You can also recommend a sample size to inspect based on the lot size (refer to Setting up Inspection Levels for Sample Sizes).
Default Pass/Fail Bins
If you use bins, select Warehouse Maintenance > Action Bar > Addl Opts > Bin Locator Information > F6-Other Bins to set up default bins for products in Inspection Results Entry:
•
Default Q/C Pass Bin - Default bin location for products that pass inspection.
•
Default Q/C Fail Bin - Default bin location for products that fail inspection.
These bins are overridden by default pass/fail bins set up for an agent in Quality Agent Maintenance (refer to Creating Quality Agent Codes).
To enter inspection results:
1
Select Order Processing > Additional Subsystems > Quality Inspection Subsystem > Inspection Results Entry to open the Inspection Results Entry screen.
2
For Inspection#, choose from the following:
•
Press ENTER over NEW to create a new inspection number.
•
Enter an existing inspection number.
•
Press F3 to search for previously created inspection entries.
3
Status is not accessible and refers to the current status of the inspection result. Options include:
•
O - Open.
•
P - Pass.
•
F - Fail.
4
For Inspect Type, identify how the inspection request was generated:
•
C - Customer return.
•
P - PO receipts.
•
V - Vendor Performed.
•
W - W/O Manufacturing.
Once you enter the Inspect Type, the default Procedure# appears under Inspection Plan (refer to Assigning Default Procedures to Inspection Types).
5
Enter the Agent, that reported the defect or is assigned to the inspection (refer to Creating Quality Agent Codes).
6
For Document#, if applicable, enter the document number that corresponds with the inspection type (e.g., purchase order number, credit memo number, work order number):
•
Ln# refers to the line on the purchase order, credit memo number, or work order number.
•
Source identifies the vendor if you entered a purchase order number for Document#.
7
Enter the Product# to inspect.
8
For Lot Size, enter the total quantity received.
9
Verify the following under Inspection Plan:
•
Procedure# - Defaults to the procedure established as the default in Quality Inspection Procedure Default Maintenance (refer to Assigning Default Procedures to Inspection Types).
•
Sample Size defaults to the recommended sample size to inspect, based on the Lot Size. The lot size information from Sample Size code Letter Maintenance (refer to Setting up Inspection Levels for Sample Sizes) determines the sample size (refer to Using Single Sampling Tables).
•
Std. Level is the recommended General Inspection Level based on the lot size. This information is from Sample Size code Letter Maintenance (refer to Setting up Inspection Levels for Sample Sizes). If you change the level, the Sample Size adjusts according to the value set up in sampling tables (refer to Using Single Sampling Tables).
10
For Number/Cartons, if applicable, enter the number of cartons in the shipment.
11
For Type, enter the code for the severity of the problem found during inspection. Choose from the following:
•
C - Critical.
•
J - Major.
•
N - Minor.
12
For Code, enter the defect code that represents the problem found.
13
For Qty, enter the defective quantity found.
14
Enter the Description of the problem.
Press F5-Summary to view a summary of inspection statistics by severity type. (i.e., pass, fail).
Press F6-Instructions to view any inspection instructions set up for the product (refer to Assigning Inspection Procedures for Products).
15
Press F2-Save to save the inspection results information.
16
If the product failed inspection, the following message appears:
Warning, This lot of product has failed acceptable levels!
Choose from the following:
•
Acknowledge Message.
•
Message Acknowledged.

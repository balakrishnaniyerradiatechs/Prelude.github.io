Maintaining Customer Department File Information
When you set up a crib warehouse to contain products in a customer’s warehouse, you can group the products by department and accounting codes. You can also enter the customer’s purchase order numbers to reference each accounting code if an accounting code is required for each line item on a sales order (refer to the Customer Maintenance Acct Code Pattern Match Screen).
If you have a custom program set up for electronic data transmission, you can use the department file information to create the sales order.
To group products by accounting codes:
1
Select Order Processing > Maintenance > Customer Control Files > Customer Crib Maintenance > Customer Department Maintenance to open the Customer Department File Maintenance screen.
2
Enter the crib Customer Number.
3
Enter the Customer Ship-To number, if applicable.
4
Enter the Department Number to assign to the crib customer. The department is used to separate expenses within a warehouse.
5
Enter the Description of the department, for just this customer.
6
For Shrinkage, indicate if inventory shrinkage is updated for the Customer Ship-to number. Inventory shrinkage refers to inventory that decreases in quantity without being sold (e.g., theft, accidents). Choose from the following:
•
Y - Update shrinkage.
•
N - Do not update shrinkage.
7
For Acct Code 1, enter the crib accounting code to display in the Crib Information screen when a crib product is entered on a sales order.
8
For Code 2, enter the second crib accounting code to display the Crib Information screen when a crib product is entered on a sales order.
9
For Customer P/O#, enter the customer’s purchase order number to use that must match the accounting code for each crib line item on a sales order.
10
For PO Ln#, enter the line number from the original purchase order.
11
Press F2-Save to save the customer department information.

Customer Maintenance Audit Report by User Id
The Customer Maintenance Audit Report by User Id displays changes made in Customer Maintenance. You can run the report for 1 or more user IDs.
After you run the report, you can choose to purge the product information so that it is not included the next time you run the report.
Track Changes
Customer Maint Audit in File Update Parameters must be set to Y to track changes in Customer Maintenance.
Menu Path: Order Processing > Reports > Miscellaneous Reports > Customer/Ship-To Reports > Customer Audit Reports > Customer Maintenance Audit Report By User Id
Selection Criteria
Use the following criteria to limit the information to print on the report:
•
User Identification. The user ID of the person that edited customer records.
•
Entry Date Range. The beginning and ending dates that the customer records were edited.
Column Descriptions
Column Heading
Description
Customer Number
The customer name and description.
Type
 
Indicates if the data was changed for a prospect or customer:
•
C - Customer.
•
P - Prospect.
Ent-Date
The date the customer record was edited.
Ent-Time
The time of day that the customer record was edited.
User
The user ID of the person that changed the record.
Description (top line)
The description or name of the user.
Description (second line)
The field name that was changed.
Before Value
The data in the field before the change.
After Value
The data in the field after the change.

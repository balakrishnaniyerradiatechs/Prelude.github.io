Releasing Orders from Administrative Order Hold
The Administrative Order Inquiry contains sales orders put on hold for pricing or other ordering restrictions defined by your company (refer to Defining Options for Administrative Order Hold for information on how to place orders on hold). The order must be released from hold before a pick ticket can print. Your user ID must be set up as an Administrative Order Manager to release orders from the Administrative Order Inquiry.
Tracking Released Sales Orders
To track orders released from Administrative Order Hold, enter Y for Admin Mgr. Release in File Update Parameters.
To review or release orders on Administrative Order Hold:
1
Select Order Processing > Inquiries > Administrative Order Inquiry to open the Administrative Order Inquiry screen. The Administrative Order Manager ID will default if you are an Administrative Order Manager.
2
Press ENTER. Sales orders on Administrative Order Hold appear.
3
Move the cursor to the order that you wish to review or release and choose from the following:
•
F5-Review - Opens the Sales Order Display screen to allow you to review the sales order.
•
F6-SOE - Opens the Sales Order Entry screen to allow you to modify the sales order. You can also release the order and send a message to the queue of the employee who entered the order.
•
F7-Toggle - Allows you to move back and forth between the original list of orders on the Administrative Order Inquiry screen and the orders in the hold queue. To send an order to the hold queue, enter one of the following under Reviewed:
-
C - Customer review.
-
I - Internal review.
-
S - Salesrep review.
•
F8-Release - Allows you to release the order from hold and send a message to the queue of the employee that entered the order.
4
Repeat step 3 for each order that you want to review, edit, or release.
5
Press F2-Save to close the inquiry screen.

Order Value
The total purchase price for the products on the sales order.
User
The user ID of the person that entered the sales order (Written By).
Hdr
The reason code to indicate why the order went on Administrative Order Hold (refer to Defining Options for Administrative Order Hold). Multiple codes may display for each order. Codes in this column refer to the holds affecting the order header:
•
A - Automatic hold for conversion orders.
•
B - Backorder cycle allocations. If inventory is allocated to the sales order during the Allocation Procedure or when stock is received (if the purchase order is linked after the sales order is invoiced at least once).
•
C - Conversion errors from DOE.
•
D - The Outbound Freight in the Sales Order Completion screen was overridden.
•
E - A customer’s blanket P/O amount was exceeded. Refer to Tracking Blanket P/O Information for a Customer
•
F - The Freight Terms code was changed in the Sales Order Header Order Additional Information Screen.
•
I - The Ship Via Code is incompatible for the customer (refer to Customer Level Administrative Order Hold Options) or the product (refer to Product Level Administrative Order Hold Options).
•
K - The total of the order is more than the Oversized Order $ Amount field on Administrative Order Hold Parameters screen. Refer to System Level Administrative Order Hold Options.
•
L - An item was linked to an existing P/O, or a P/O was created through Sales Order Entry.
•
M - Manually placed on administrative hold. Refer to Manually Placing Orders on Administrative Order Hold.
•
O - The customer is always on order hold. Refer to Customer Level Administrative Order Hold Options.
•
P - The customer requires a P/O number. Refer to Customer Level Administrative Order Hold Options.
•
Q - The invoice amount is more than the Admin Invoice Amt in the Customer Blanket P/O Numbers Screen, if Require P/O Number (Customer Maintenance > F7-Order) is set to I.
•
S - Ship via from web requires authorization.
•
T - The sales order does not contain Tax Jurisdiction Codes in the sales order header > Tax Jurisdiction Information Screen.
•
W - Web generated order.
•
Z - A credit card order is entered without an authorization code. Refer to Credit Cards
•
0-9 - User defined codes.
Line
The reason code to indicate why the order went on Administrative Order Hold (refer to Defining Options for Administrative Order Hold). Multiple codes may be displayed for each order. Codes displayed in this column refer to the holds affecting line items on the order:
•
A - The product is always on hold. Set in Product Maintenance > F5-Add Info > Always Hold Order = Y.
•
C - The price was overridden below cost.
•
G - The price was overridden below GP%.
•
I - The Ship Via Code is incompatible for the customer (refer to Customer Level Administrative Order Hold Options) or the product (refer to Product Level Administrative Order Hold Options).
•
N - The price was overridden below the product’s Minimum Price (Product Maintenance > F7-Pricing or Product Warehouse Maintenance >F7-Price/Cost).
•
R - The price was overridden below the customer’s default price.
•
S - The quantity is over the month’s supply test defined in Max Months Supply For Admin Order Hold in the Administrative Order Hold Parameters screen (refer to System Level Administrative Order Hold Options).
•
V - A line item’s gross profit percent is below the Mgr Low Level Minimum GP% or more than the Mgr High Level GP% (designated in Order Processing parameters and can be overridden in Product Line Maintenance).
•
X - A prescription product was entered without a prescription. Refer to Setting up and Tracking Prescriptions.
Message
Any notes sent to the Administrative Hold Manager by the Written By sales person when the order went on hold.
Reviewed
Indicates if the order was reviewed within this inquiry:
•
Y - This order was reviewed.
•
N - This order was not reviewed.
If you want to move the order to a hold queue for review, enter one of the following:
•
C - Customer review.
•
I - Internal review.
•
S - Salesrep review.
Once moved, you can use F7-Toggle to see orders in the hold queue. If necessary, you can then release the order from the hold queue.

F8-Release

F9-Goto Line
Allows you to enter a line number, page number, or reference number to go directly to.

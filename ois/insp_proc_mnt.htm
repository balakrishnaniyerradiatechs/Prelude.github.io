Quality Inspection Procedure Maintenance screen
You can use the Quality Inspection Procedure Maintenance screen to set up acceptable quality levels (AQL) for each type of inspection procedure (e.g., by vendor, customer return, work order, or P.O. receipt).
Menu Path: Order Processing > Additional Subsystems > Quality Inspection Subsystem > Maintenance > Quality Inspection Procedure Maintenance
Field and Function Key Descriptions
Field/Function Key
Description
Procedure#
The inspection procedure number.
Description
The explanation of the inspection procedure.
Inspection Type
The type of inspection this procedure number represents:
•
C - Customer return.
•
P - PO receipts.
•
V - Vendor performed.
•
W - Work order manufacturing.
Version
The inspection procedure version that is currently active.
Date (unlabeled field)
The date this version went into production.
Previous Version
The previous version number used for this procedure. Updated when you enter a new version.
Date (unlabeled field)
The date the previous version went into effect.
Acceptable Quality Levels (AQL)
Table
The type of sample table to use for accept and reject levels for this procedure number:
•
S1 – Single Sampling (normal). Refer to the Normal Inspection Table.
•
S2 – Single Sampling (tightened). Refer to the Tightened Inspection Table.
•
S3 – Single Sampling (reduced). Refer to the Reduced Inspection Table.
Standard Level
The ANSI General Inspection Level to use from Sample Code Letter Maintenance (refer to Setting up Inspection Levels for Sample Sizes).
Critical
The percent that must pass inspection if the severity code is critical.
Major
The percent that must pass inspection if the severity code is major.
Minor
The percent that must pass inspection if the severity code is minor.
F2-Save
Saves the quality inspection procedure information.
F4-Del
Deletes the quality inspection procedure information.

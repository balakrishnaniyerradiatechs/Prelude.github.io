Lost Sales Entry screen
If you want to record a lost sale for a product without having to enter a sales order (e.g. if a customer asked for a product that you did not have), you can use Lost Sales Entry to record a lost sale for a customer/product/warehouse.
Menu Path: Order Processing > Optional Entry and Printing > Lost Sales Entry
Field and Function Key Descriptions
Field/Function Key
Description
Customer
The customer to whom you lost the sale.
Product / Description
The product number for which you lost the sale.
Sell Warehouse
The warehouse that would have received credit for the sale.
Salesrep #1
The salesperson assigned to the order that would have received credit for the sale.
Quantity
The quantity of the product for which you lost a sale.
U/M
The product’s selling unit of measure.
Net Price
The customer’s price for the product.
Extension
The total cost of the extended quantity lost (Quantity x Net Price).
Reason Code
The Reason Code that explains why the sale was lost and determines if product usage is updated. Lost sales reason codes are set up in Lost Sales Reason Code Maintenance. Refer to Creating Lost Sales Reason Codes.
F2-Save
Updates the lost sale.

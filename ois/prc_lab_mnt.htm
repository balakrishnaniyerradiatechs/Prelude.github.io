Price Label Format Maintenance screen
The Price Label Format Maintenance screen allows you to design the layout of a price label.
Menu Path: Order Processing > Maintenance > Pricing Structures > Price Label Format Maintenance
Field and Function Key Descriptions
Field/Function Key
Description
Format Code
The 1 - 6 digit alphanumeric code to represent the label design.
Printer Type
Determines how the labels print:
•
Z - SPL language code (zebra printer).
•
E - EPL Language code.
•
O - Other.
Stickers Across
The number of labels to print on a row (the number of labels across the form).
Sticker Spaces
The number of spaces between each label in the row.
Sticker Width
The maximum number of characters to print across on a single label.
Sticker Depth
The number of lines to print on a single label. The system takes into account one line between each label.
Type
Defines where the printed data comes from. You can enter multiple lines to print on the label, with a different Type for each line:
•
C - Customer file.
•
S - Customer ship-to file.
•
CPN - Customer part number file.
•
P - Product header file.
•
W - Product warehouse file.
•
PV - Product vendor file.
•
V - Vendor file.
•
O - Order file.
•
OL - Order line file.
•
D - Current date.
•
L - Literal.
Attr
The location in the raw data file (e.g., from /FLI) where the printed data is extracted from.
Mult Value
The position where the data is extracted from if the Attr is multi-valued. If the Attr is not multi-valued, enter zero.
Line
The line number on the label where the data will print. The fields to print must be in ascending order (e.g., 1, 2, 3, 4, 5).
Print Pos
The starting position for the data on the label (normally 0 or 1). To indent a field by 2 positions, enter 3.
Length
Defines the number of characters to print on a line. This cannot be longer than the Sticker Width.
Start Pos
Defines which digit of the part number is the beginning digit to print on the label. Normally you would set this to 1 to print starting with the first character. But, if you use product prefixes and do not want to print a 3 digit prefix, enter 4 to start with the 4th character of the part number.
InDec
If the data field is a price, this field defines the number of decimal places to store in its internal format.
For example, a price of 55.385 is stored internally as 55385. If you enter 3 in In Dec, then the price is stored as 55.385. If you enter 2 in In Dec, the price is stored as 553.85.
OutDec
Indicates the number of decimal prices to use to print the price.
For example, if the internal price is 55385 (internal decimal of 3) and you set Out Dec to 3, then 55.385 will print. If you set Out Dec to 2, then 55.39 will print.
Just
Indicates how to justify the printed data on the label:
•
R - Right justified.
•
L - Left justified.
•
D - Date. The information will print as mm/dd/yy.
Supp Lead Zero
Indicates if leading zeros are suppressed when you print data:
•
Y - Suppress leading zeros. For example, customer 000001 prints as 1.
•
N - Do not suppress leading zeros.
Cde Num Fld
Indicates if you want to encrypt the price label data by assigning a character to each number from 0 - 9. Choose from the following:
•
Y - You want to encrypt the data to print on a label. Select F6 to open the Coded Number field screen to enter a character to print for each number.
•
N - You do not want to encrypt the printed data.
•
B - Allows you to enter barcode type and dimension information.
F2-Save
Saves the price label information.
F4-Del
Deletes the price label information.
F5-Shelf Sticker
Opens the Shelf Label Format Maintenance screen to allow you to enter the same information as in the Price Label Format screen to print shelf labels for your warehouse.
F6-Coded Numeric Field Definition
If you chose Y for Code Num Fld, you can use this to open the Coded Number Field screen to allow you to enter a character to print for each number. For example, if you enter D for 3, F for 5, and K for 9, then 55.39 prints as FFDK.

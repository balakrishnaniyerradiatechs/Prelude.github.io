Creating a Price Matrix From a Quote or Sales Order
If you entered a quote or sales order for a customer and you want to save the pricing information as a matrix, you can use the Quote/Order Created Price Matrix to create a price matrix for future use. You can only use this feature for price matrix types 1, 2, 6, 7, and 19. The system copies the quote/order information and allows you to modify it as necessary. If multiple customer price groups exists for the customer, you must enter the price group to update. You can insert, append, override, or delete price matrices.
To create a price matrix from a quote:
1
Select Order Processing > Maintenance > Pricing Structures > Price Matrix Maintenance > Quote/Order Created Price Matrix to open the Quote/Order Created Price Matrix screen.
2
Enter either the Quote or the Order number that you want to use for price matrix information.
3
Enter the Matrix Type that you want to create. Choose from the following:
•
1 - Customer/Product.
•
2 - Customer/Customer Product Number.
•
6 - Customer Price Group/Product.
•
7 - Customer Price Group/Customer Product Number.
•
19 - Customer Ship-to/Product.
4
For Matrix Option, choose from the following to determine how the matrix is updated:
•
O - Overwrites the target matrix with the source matrix. The original target matrix is deleted. You are warned if the target matrix exists and allowed to exit without overwriting.
•
I - Inserts the source matrix at the beginning of the target matrix and retains the existing target price matrix data. You can either insert the entire source matrix or a single line of the source matrix. You may want to use this option to update promotional or temporary pricing.
•
A - Appends the source matrix to the end of the target matrix and retains the existing target price matrix data. You can either append the entire source matrix or a single line of the source matrix into the target matrix. You may want to use this option to update future prices that are effective after the existing matrix expires.
•
D - Deletes all the price matrices for the customers defined only if the same combination of matrices exists for the customer or customer price groups. For example, to delete a price matrix for a product, the customers must have the same product on their matrices as the source matrix. Only records with the same product are deleted. The price matrices that have range numbers as part of the matrix must have the same range number for the customers or customer price groups to allow you to delete them.
5
For Price Method, enter $ to update as a dollar amount. The net price from the quote is used for the matrix.
6
For Start Date, enter the date the price matrix becomes effective.
7
Enter the Expiration Date for the matrix.
8
Enter the Warehouse number if the matrix is for one warehouse only. Otherwise, leave blank to include all warehouses.
9
For Job/Customer P/O, enter a customer job number or purchase order number if you want the matrix to only apply to a job or purchase order.
10
For Contract, indicate if you want to identify the price matrix as a contract:
•
Y - Flag the matrix as a contract.
•
N - Do not flag the matrix as a contract.
11
If you entered Y for Contract, enter a Contract Number to assign to the contract.
12
For Override, indicate if the matrix price can be changed:
•
Y - The price can be overridden on a sales order.
•
N - The price cannot be overridden on a sales order.
•
H - Prices can be raised but not lowered below gross price.
13
Press F2-Create Matrix to create the price matrix.

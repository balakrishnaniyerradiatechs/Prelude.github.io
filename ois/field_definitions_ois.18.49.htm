Credit Memo Entry screen
The Credit Memo Entry screen is used to issue credit to a customer for returned or damaged material. Refer to Entering a Credit Memo for more information.
Menu Path: Order Processing > Optional Entry and Printing > Credit Memo Entry
Field and Function Key Descriptions
Field/Function Key
Description
Order Number
The credit memo number.
Customer
The customer number.
Product
The product for which the customer is to receive credit.
Description
The description of the product.
N
An asterisk (*) indicates that notes exist for the line item.
R
If the product is from Vendor Request for Quote Entry, a Q indicates that a RFQ (request for quote) was sent to the vendor. Refer to Creating a Vendor RFQ from a Quote.
AO
If the line item is placed on Administrative Order hold, the code for the violation appears:
•
A - Product always on hold.
•
C - Price overridden below cost.
•
G - Price overridden below GP%.
•
I - Incompatible ship via.
•
N - Price overridden below minimum price.
•
R - Price overridden below customer default price.
•
S - Quantity over month’s supply test.
•
V - A line item’s gross profit percent is below the Mgr Low Level Minimum GP% or more than the Mgr High Level GP% (designated in Order Processing parameters).
Refer to the
AI
If the line item is placed on Administrative Invoice hold, the code for the violation appears.
•
A - Credit Card Order – No Authorization: A credit card order is entered without an authorization code.
•
P - Price Violations: A price is changed to a value that is lower than the default price or lower than the cost. This only applies if Price Override Code for the User ID is set to C or R (refer to User Level Administrative Invoice Hold Options).
•
S - Salesrep Changed On Order: Salesrep One or Salesrep Two was changed on the Sales Order Header screen.
•
T - Order With No Tax Jur Codes: The order does not contain Tax Jurisdiction Codes.
•
U - User Defined.
Refer to the Administrative Invoice Inquiry.
Linkage
The document number if the product is linked to another order. The prefix indicates the document type:
•
P - Purchase order.
•
T - Transfer.
•
W - Work order.
Quantity
The quantity for which the customer is receiving credit.
U/M
The unit of measure for the product.
Whse
The warehouse to which the product is returned.
Price
The unit price of the product.
Quantity
The total quantity of products for which the customer is receiving credit.
Weight
The total weight of all products on the sales order.
Profit
The total amount deducted from gross profit earned.
Gp%
The total gross profit percent of the credit memo.
Amount
The total amount of credit issued to the customer for the returned products.
F2-Save
Allows you to:
•
Complete the credit memo.
•
Print the pick ticket.
•
Suspend the credit memo.
•
Return to the Credit Memo Entry screen to view or edit.
F4-Del
Deletes the credit.
F5-Hdr Scr
Opens the Sales Order Header screen to allow you to view or edit the credit memo header information.
F6-Totals
Opens the Order Totals screen to allow you to view or change the total quantity, weight, profit, and dollar amount of the order.
F8-Order Pad
Opens the Order Pad Inquiry to allow you to search for products, by description, previously purchased by a customer.
F9-Goto line
Allows you to enter a line number, reference number, or page number to go directly to.

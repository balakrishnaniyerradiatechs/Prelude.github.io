Expected Date Change Notification Report
The Expected Date Change Notification report lists sales orders linked to purchase orders, transfers, or work orders on which the delivery date was changed.
After you run the report, you can choose to purge the product information so that it is not included the next time you run the report.
Menu Path: Order Processing > Reports > Miscellaneous Reports > Expected Date Change Notification Report
Selection Criteria
Use the following criteria to limit the information to print on the report:
•
Salesrep. The salesperson assigned to the order.
•
Document Type. The type of document the sales order is linked to. Choose from the following or leave blank to view all documents:
-
P - Purchase Order.
-
W - Work Order.
-
T - Transfer.
Column Descriptions
Column Heading
Description
Cust#/Description
The customer number and description.
Contact/Phone Number
The customer contact name and phone number, if applicable.
S/O#
The sales order number linked to the purchase order, work order, or transfer.
Product Number/ Description
Line 1: The product number.
Line 2: The product description.
Date Before
The expected delivery date before the source document was changed.
Date After
The revised expected delivery date on the source document.
Source Document
 
The linked document number that contains the backordered item. The prefix indicates the document type:
•
P - Purchase order.
•
W - Work order.
•
T - Transfer.
Usr
The user ID of the person that changed the order.
Date
The date the order was changed.
Time
The time the change was made.

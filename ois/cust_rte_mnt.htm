Customer Route Maintenance Screen
If you use delivery trucks and routes, you can use the Customer Route Maintenance screen to establish route and time information for a customer, including special delivery requirements. If a customer has more than one route, Sales Order Entry chooses the best route based on the order date.
Menu Path: Order Processing > Maintenance > Customer Maintenance > F7-Order > Action Bar > Add’l Options > Route
Field and Function Key Descriptions
Field/Function Key
Description
Route#
The default route number for this customer that indicates the route normally used to deliver merchandise to this customer. Route numbers are set up in the Route Maintenance Screen and contain delivery day and time information for each route.
Description
The description of the route.
Del Seq
If you use standard delivery routes, the number that indicates where in the delivery sequence the customer will receive their order.
Add’l Collection Amount
If you use route processing, you can enter an amount to collect from the customer to appear on the driver’s manifest.
Beginning Delivery Time
The earliest time that the customer accepts deliveries.
 
Restricted Start Time
The beginning of the time frame during which the customer does not accept deliveries, if applicable.
Restricted End Time
The ending of the time frame during which the customer does not accept deliveries, if applicable.
Ending Delivery Time
The latest time in the day that the customer accepts deliveries.
Delivery Requirements
The delivery requirement code if the customer has special delivery requirements.
Dynamic Routing Longitude
The longitude to use with interfacing dynamic routing software (custom modification used with 3rd party software).
Dynamic Routing Latitude
The latitude to use with interfacing dynamic routing software (custom modification used with 3rd party software).
Dynamic Routing Zone
The zone to use with interfacing dynamic routing software (custom modification used with 3rd party software).
F2-Save
Saves the customer route information.

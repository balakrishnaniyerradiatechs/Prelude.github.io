Quality Incident Report
The Quality Incident report lists information gathered from the Quality Incident Entry.
Menu Path: Order Processing > Additional Subsystems > Reports > Quality Incident Report.
Selection Criteria
Use the following criteria to limit the information to print on the report:
•
Incident Number. The incident number to view. Leave blank to view all incident numbers that meet the remaining criteria.
•
Title. The corrective action entry title to limit the report to a specific title. Leave blank to include all title descriptions .
•
Entry Date range. The beginning and ending incident entry dates to include.
•
User Id. The user ID number for which you want to view incident entries. Leave blank to include incident entries for all user IDs.
Column Descriptions
Column Heading
Description
Incident
The incident number created in Quality Incident Entry.
Description
The description of the incident.
Ent-date
The date the incident was entered.
Status
The status of the incident. Options include:
•
O - Open.
•
C - Closed.
•
I - In process.
•
U - Unassigned.
•
W - Waiting for response.
Incident Type
The type of incident (e.g., product, employee).
Type
The source type of the incident:
•
F - Feedback entry.
•
I - Inspection results entry.
•
R - Return sales order.
Document
The document number depending on the Type:
•
The feedback number if Type is F.
•
The credit memo number if Type is R.
•
The Inspection number of Type is I.
Description
The description of the problem.
Agent
The quality agent assigned to the incident.
Inspection Type
The source type of the incident
•
Feedback. The incident was created in Quality Feedback Entry.
•
Inspection. The incident was created in Inspection Results Entry.
•
Return. The incident was created on a credit memo.

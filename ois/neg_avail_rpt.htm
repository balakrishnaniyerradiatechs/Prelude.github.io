Negative Availability Override Report
The Negative Availability Override report lists sales order line items on which the pick quantity was overridden, causing the available quantity to be negative.
After you run the report, you can choose to purge the product information so that it is not included the next time you run the report.
Neg. Avail Override in File Update Parameters must be set to Y to track negative override information.
Menu Path: Order Processing > Reports > Miscellaneous Reports > Negative Availability Override Report
Selection Criteria
Use the following criteria to limit the information to print on the report:
•
Warehouse. The shipping warehouse.
•
User Id. The user ID of the person that entered the order.
Column Descriptions
Column Heading
Description
User Id
The user ID and name of the person who entered the sales order (written-by user ID).
Product Number
The product with a negative available balance.
Whse
The shipping warehouse.
Source Key
The order that contains the discrepancy. The prefix indicates the order type:
•
S - Sales order.
•
T - Transfer.
•
W - Work order.
Qty-Ord
The quantity ordered by the customer.
Bef-Pck
The quantity defaulted from the Pick Qty before it was overridden.
Aft-Pck
The pick ticket quantity after the override.
Bef-Avl
The quantity the system showed as available before the quantity was overridden.
Aft-Avl
The quantity the system showed as available after the quantity on the sales order was overridden.
Prt
The port number from which the pick ticket printed.
Ent-Date
The date the order was entered.
Ent-Time
The time the order was entered.

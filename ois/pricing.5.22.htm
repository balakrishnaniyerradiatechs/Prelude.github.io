Creating a Retail Price Matrix
You can use retail price matrices to give discounts and markups to a customer for a period of time by using one of the following:
•
A net price.
•
A percentage markup on a gross price.
•
A gross profit percent.
•
A markup above the net price.
Retail Price Matrix Listing
Retail Price Matrix Listing (Order Processing > Maintenance > Pricing Structures > Retail Price Matrix Maintenance > Retail Price Matrix Listing) lists retail prices matrices by type.
To create a retail price matrix:
1
Select Order Processing > Maintenance > Pricing Structures > Retail Price Matrix Maintenance > Retail Price Matrix Maintenance to open the Retail Price Matrix Maintenance screen.
2
For Matrix Type, enter a valid matrix type set up in Retail Price Matrix & Sequence Maintenance.
3
For Warehouse, enter the selling location if you want this matrix to apply to only one warehouse. Leave blank if the matrix is for all warehouses.
4
The next screen that appears depends on the Matrix Type. For example, if you select Matrix Type 1 (Customer/Product), the Customer/Product Retail Matrix screen appears.
5
Enter the appropriate criteria for the matrix type. In this case, you must enter the Customer and the Product.
The Retail Price Matrix Input screen appears.
For each line, enter pricing information for a date range. Since you can enter multiple lines for a matrix, enter each line in chronological order.
6
For Beg-Date, enter the date that the special pricing is to begin. The start date for the first line defaults to the system date. All remaining lines default to the day after the previous expiration date. For example, if the previous line has an expiration date of 06/30/06, then the next line’s beginning date defaults to 07/01/06. If the previous line does not have an expiration date then the next line is not available.
7
For Exp-Date, enter the date that the special pricing is to end. Leave blank if there is no expiration date.
8
For Mtd, choose the method to calculate the retail prices:
•
$ - Enter a set Price for the product. Not valid for product range matrices.
•
* - Enter a markup or a discount to calculate the price. Choose from the following:
-
Enter the %Markup to earn over the gross price.
-
Enter a Basis and a Percent. The basis indicates which column price or cost to use from Product Maintenance. The percent is multiplied by the basis to calculate the price. The discount entered will be a percentage over a price column if a price column was entered in basis or a percentage over a cost if a cost was entered in basis.
•
G - Enter the % Markup for the gross profit percentage that you want to earn.
•
N - Enter a % Markup above net price.
9
If applicable, enter a Key Code to print retail price stickers.
10
For PPFlg, indicate if this price is for a promotion:
•
Y - Mark the price as a promotion (information only).
•
N - Do not mark the price as a promotion.
Retail Price Recalc Utility
If you change a price in Retail Price Matrix Maintenance, you can use the Retail Price Recalculation Utility to update all open orders that contain the customer/product combination.
Order Processing > Maintenance > Pricing Structures > Retail Price Matrix Maintenance > Retail Price Recalc Utility

Manually Deleting a Sales Order
When you delete a sales order, depending on system settings, you may be required to enter a reason code to indicate why you are cancelling the order (refer to Creating Order Cancellation Reason Codes). You may also be required to enter a lost sales reason code (refer to Creating Lost Sales Reason Codes).
If you delete an order with open releases, you are warned that pick tickets already exist. If you do delete the order, you will not be able to bill the customer for material already received.
To delete a sales order:
1
From the Sales Order Entry screen, select F4-Del. The following message appears:
Are You Sure You Wish To Delete
2
Choose from the following:
•
No - Returns to the Sales Order Entry screen.
•
Yes - Opens the Reason for Deletion screen to allow you to enter an Order Cancellation Reason Code. Refer to Creating Order Cancellation Reason Codes for information on creating order cancellation reason codes.
File Update Parameters
To track deleted sales orders, set Deleted Orders in File Update Parameters to Y. The salesperson that deletes the order must enter an order cancellation reason code.
Once you enter the order cancellation reason, if your company tracks lost sales to update product usage, the following message displays:
Do You Want To Update Lost Sales?
3
Choose from the following:
•
No - Exits the sales order.
•
Yes - Opens the Lost Sale Reason Code screen to allow you to enter a Lost Sale Reason Code. Refer to Creating Lost Sales Reason Codes.
File Update Parameters
To track lost product sales, you must set Lost Sales in File Update Parameters to Y. The salesperson that deletes the order must enter a lost sales reason code.

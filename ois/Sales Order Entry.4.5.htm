Entering Sales Order Header Information
Once you select a customer/customer ship-to account (refer to Selecting a Customer), the Sales Order Header screen appears. Most of the fields on this screen default from the customer’s account or ship-to account, but you can change the information as necessary.
Option Parameters
Skip Header in SOE/QOE in Option Parameters > Order Processing Parameters indicates if the Sales Order Header screen is skipped in Sales Order Entry and Quote Order Entry:
•
Y - The header screen will not appear in Sales Order Entry unless a required field is missing data (e.g., customer, purchase order, freight terms, salesrep).
•
Q - Y - The header screen will not appear in Quote Order Entry unless a required field is missing data (e.g., customer, purchase order, freight terms, salesrep).
•
B - The header screen will not appear in either Sales Order Entry or Quote Order Entry.
•
N - The header screen always appears when you first enter a sales order or quote.
To enter sales order header information:
1
Enter the Order Type for the sales order (refer to Sales Order Types for an explanation of each order type).
2
For Purchasing Contact, if applicable, enter the person at the customer’s location who is purchasing the material. If you use Contact Management (refer to the Marketing Documentation), you can track sales information by purchasing contact.
3
For Ship-To, accept the customer’s ship-to address or choose from the following:
•
F3 - Lists existing ship-to addresses for this customer to allow you to select the correct address.
•
D - Duplicates the customer’s sold-to address for the ship-to address (default if no ship-to addresses exist for the customer).
•
M - Opens the Ship-To Address Screen to allow you to manually enter a new, one-time address for this order.
•
P - Indicates that the customer will pick-up the order.
•
C - Uses the purchasing contact’s address for the ship-to address.
•
CP - Indicates that the end-user will pick-up the order.
Option Parameters
Valid Ship-to Entries (Additional Option Parameters) determines if duplicate and manual ship-to address are allowed in the Sales Order Header screen:
•
B - Allow both duplicate and manual ship-to entries.
•
D - Allow only duplicate ship-to entries.
•
M - Allow only manual entries.
•
N - Do not allow either duplicate and manual ship-to entries.
 
Ship-To Keyword Search (Option Parameters > Order Processing Parameters > F5 > F5 > F5) determines which screen appears when you press F3 in the Ship-To field:
•
Y - Display the Ship-To Keyword Selection screen. You can enter a keyword to lookup ship-to numbers.
•
N or blank - Display the GRIM Screen to lookup up customer ship-to addresses.
4
For Contact, enter the name of the person placing the order. The default comes from the Customer Contacts Screen if one is available. Otherwise, it defaults from the Contact name on the Customer Maintenance screen or the Customer Ship-To Maintenance screen.
5
Enter the Customer P/O#. The customer’s purchase order number may be mandatory and may require a pre-determined format depending on how Require P/O Number is set in the customer’s Order Information Screen.
6
If applicable, enter a Cust Release#. A customer release number may be mandatory depending on how Require Release Number is set in the customer’s Order Information Screen.
7
If you use Job Accounting, Progress Billing, or if you track job numbers, you can enter a Job Number. A job number may be mandatory depending on how Require Job Number is set in the customer’s Order Information Screen.
Job Number Index
Order Job Number Index (File Update Parameters) determines if job numbers are maintained and tracked:
•
Y - The job number on a sales orders is validated. If so, all sales orders that match any portion of the job number display to allow you to choose the job number for the sales order.
•
N - Job numbers are not tracked.
8
For Ship Via Code, indicate how the customer will receive the material. Press F3 to view only compatible ship vias based on the customer, customer ship-to, and warehouse. Ship via codes are set up in the Ship Via Maintenance Screen and assigned to customers in the Customer Maintenance Incompatible Ship Via Maintenance Screen or the Customer Ship to Maintenance Incompatible Ship Via Maintenance Screen.
9
Verify or change the Freight Terms code that determines if inbound or outbound charges apply to the sales order when it is confirmed in Release Confirmation Entry. Freight codes are established in the Freight Terms Maintenance Screen and assigned to the customer the Freight Information Screen.
10
Verify or change the accounts receivable Payment Terms code for sales orders that indicates when payment is due, payment conditions, and any applicable discounts. These codes are set up in A/R Terms Maintenance and assigned to customers as a default in the Customer Maintenance Screen.
11
Verify or change the Email Address, Fax, and Phone if necessary.
12
For Sell Warehouse, accept the default warehouse for this customer (from the Customer Maintenance Order Information Screen) or change the selling warehouse if you want a different warehouse to receive credit for the sale.
Selling Warehouse
Selling Allowed in Warehouse Maintenance > F5-Add Info must be set to Y to allow you to enter the warehouse as a selling warehouse.
13
For Ship Warehouse, accept the default warehouse for this customer (from the Customer Maintenance Order Information Screen) or change the shipping warehouse if you want the material to ship out of a different warehouse.
14
For Written By, accept the default of the person logged on or change if necessary.
15
Salesrep One and Salesrep Two are assigned to the customer in the Customer Maintenance Screen. Change if necessary.
16
For Order Date, accept the default of today’s date or change the date if the customer ordered the material on a different date.
17
For Required , enter the date the customer requested to receive the material.
18
For Pick Ticket, accept the default date or change as necessary. This date is calculated according to how you set maintenance options for pick tickets (refer to Setting Pick Ticket Rules). If you use truck routes, this field automatically updates.
19
For Order Source, indicate how you obtained this order. Commonly used examples are:
•
C - Customer call in.
•
D - Direct order entry.
•
E - Electronic data interchange.
•
K - Special order-telemarketing.
•
L - Sales representative.
•
M - Mail in.
•
O - Default order source if one is not set in Customer Maintenance or Customer Ship-to Maintenance.
•
R - Remote order entry.
•
S - Sales force automation.
•
T - Telephone sales.
•
W - Will call.
•
X - Special order.
Refer to Defining Order Source Codes for information on creating order source codes.
20
If you use delivery trucks and follow delivery routes, you can enter a Route Number set up in the Route Maintenance Screen that contains valid delivery date and time information for each route. The default comes from the customer’s Customer Route Maintenance Screen or the Ship To Route Maintenance Screen, depending on how the order is entered.
21
If you entered a Route Number, enter a Delivery Seq to indicate where in the delivery sequence the customer will receive the order.
22
If you use multiple currency, enter the Currency code that indicates the type of currency that this customer uses. The default is set in the customer’s A/R Information Screen.
23
To further verify requirements for the sales order, use any of the following keys on the Sales Order Header screen:
•
F5-Add Info to open the Order Additional Information Screen.
•
F6-Ship-To to open the Ship-To Address Screen.
•
F7-Tax-Jur to open the Tax Jurisdiction Information Screen.
•
F8-Instructions to enter any Instructions for this sales order, if applicable (refer to Creating Standard Instructions). You can also set up standard instructions for each customer to default for all of their sales orders (refer to the Customer Maintenance Instructions Entry Screen).
Disp. Shp.Inst. In SOE/QOE in Order Processing Parameters determines if the instructions screen automatically displays when the header screen comes up in Sales Order Entry and Quotation Entry. You can set the flag to always display the Instructions screen, never display the Instructions screen, or only display the Instructions screen if instructions exist in the customer record.
•
F9-Credit Info to open the Credit Information Screen.
•
Action Bar > Add Info to access the following:
-
Route Info. Opens the Order Route Information screen to allow you to enter delivery times and requirements for delivery routes.
-
Repairs. On Type 10 repair orders, this opens the Repair Order Type Screen to allow you to select the repair order type to determine payment.
-
Consignment. On consignment orders, this opens the Consignment Types screen to allow you to define consignment information for the order.
-
Fut Committments.Opens the Future Commitment Override screen to allow you to commit inventory for future orders.
-
Rental. Opens the Rental Information screen to allow you to enter billing information for rental orders.
-
Commission Split.Opens the Commission Splits screen to allow you to enter two or more sales representatives who will split the commission for this sales order. Refer to Splitting Commission Between Salesreps for more information.
-
Commission Override.Opens the Commission Header Override screen to allow you to override the method for calculating commission for the sales representative assigned to the order. Refer to Overriding Commission For a Salesrep for more information.
24
Once all the header information is correct, press F2-Save to open the Sales Order Entry screen. Refer to Entering Sales Order Line Items.
Check For Existing Orders
Check for Existing Orders (Option Parameters > Order Processing Parameters > F5 > F5) determines if a message appears to allow you to add to open orders for a customer before you create a new order for the same customer:
•
Y - When you save the Sales Order Header screen, a message appears to allow you to add to non-released orders to the customer ship-to. If you select Yes, the first type 01 non-released sales order opens to allow you to add products.
•
P - The same as Y except that the system only searches for orders that have a pickable quantity.
•
N - No message displays if open orders exist.
This feature only applies to type 01 sales orders that are going to the same ship-to address and for which the customer does not require a purchase order.

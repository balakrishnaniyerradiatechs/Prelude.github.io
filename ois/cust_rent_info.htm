Rental Module Screen
If you use the Rental Module, you can use the Rental Module screen to set up insurance information and rental rate methods for a customer.
Menu Path: Order Processing > Maintenance > Customer Maintenance > F7-Order > F5-Add Info > F6-Rental
Field and Function Key Descriptions
Field/Function Key
Description
Liability Insurance Limit
The customer’s liability insurance limit, if the customer rents a product that requires insurance (set up in Product Maintenance > Action Bar > Addtl Options > Rental Information).
Liability Expiration Date
The date the customer’s liability insurance expires.
Physical Damage Coverage
The amount of physical property damage coverage that the customer has, if applicable. You can use this information to determine if you should charge a damage waiver when renting to this customer.
You can assign a Damage Waiver Product Number to a product in Product Maintenance > Action Bar > Addtl Options > Rental Information.
Physical Damage Expiration Date
The date the physical property damage insurance expires.
Rental Rate Method
The rental rate and billing method to use with this customer:
•
A - Rates are automatically evaluated and moved to the better rate based on the actual time material is rented. The best rate is calculated and automatically pro-rated.
•
B - Rates are automatically evaluated and moved to the better rate based on the actual time material is rented up to the first billing. This billing would be either the return of the material or the first periodic billing. Once the material is returned and first billed, it is pro-rated by day.
•
C - Uses the rate and rental period as a single rate based on the billing days established. If billing days are Monday-Friday and the rental item is billed for 7 days, the billing rate is calculated as 1.4 (1 week +2/5). The total number of billing days is used as the denominator (e.g., Monday - Friday has 5 billing days).
•
D - Uses the rate and rental period as a daily rate always. The rental item is billed for the daily rate for the number of days between the start date and the rental return date. The system calculates all days (Monday - Sunday & holidays). For periodic billing, the daily rate is calculated according to the number of days established for the Monthly Billing Cycle in Rental Parameters.
•
N - Uses the rate and rental period defined on the rental document and rounds to the nearest whole rental (e.g., 10 days is charged as 2 weeks).
Rental Contract Customer
The customer number whose pricing you want to use, if applicable. If you have a rental pricing contract set up for another customer, you can use the same pricing contract for this customer.
Salesrep#1
The number for the default primary sales representative to assign to rental orders for this customer.
Salesrep#2
The number for the default secondary sales representative (e.g., inside sales rep) to assign to rental orders for this customer, if applicable.
Environmental Charge
Determines if the customer’s rental orders should be evaluated for environmental charges:
•
Y - Evaluate the customer’s orders to determine if environmental charges apply.
•
N - Do not check for environmental charges.
The Environmental Charge P/N for a product is set up in Product Maintenance > Action Bar > Addtl Options > Rental Information.
You can override this flag for each order in the Rental Order Entry header screen. If this flag is left blank, the flag setting in Systems > Maintenance > Option Parameters > Rental Parameters is used.
F2-Save
Saves the customer’s rental information.

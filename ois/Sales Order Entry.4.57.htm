Using Consignment Sales Orders
A consignment order (07 order type) is used when material is removed from on-hand inventory and shipped (or reserved for a customer) but not invoiced until later. When the pick ticket release is confirmed, a zero dollar invoice is created and the inventory is moved from on-hand to consigned inventory. An 07 becomes an 08 (Confirmation Consignment) order type when the order is invoiced. The line items accumulate on the 08 order according to the Accumulate flag in the Consignment Types screen (appears when you enter an Order Type of 07 or 08 on the Sales Order Header screen). If the material is returned to stock, the inventory can be added back into the on-hand balance without ever affecting accounts receivable.
Order Processing Parameters
Consolidate Consign By Cust/Ship To. Determines if consignment orders are consolidated by customer number or by ship-to number. You can override this setting for a customer:
•
C - Consolidate by customer number.
•
S - Consolidate by customer ship-to number.
Upd ‘07’ Order To Bookings. Determines if type 07 consignment orders update the bookings file:
•
Y - Type 07 consignment orders update the bookings file.
•
N - Type 07 consignment orders do not update the bookings file.
Ovr Shp Whse Cons Ord. Determines if the line item shipping warehouse for the 08 consignment order should be the same as the selling warehouse:
•
Y - If the selling and shipping warehouses are different, an 08 consignment order created at invoicing is updated with the selling warehouse for the line items. This allows the G/L and perpetual inventory to be maintained at the selling location. Only use this flag if the Default Consign Accum Flag (explained below) is set to C or A.
•
N - The shipping and selling warehouse do not have to be the same.
Customer Maintenance
Consign Consolidate Method:
•
C - Consolidate at the customer level.
•
S - Consolidate at the customer ship-to level.
•
Blank - Use the system level setting in Order Processing Parameters.
Default Parameters Maintenance
Default Consign Accum Flag. Sets the default for Accumulate in the Consignment Types screen (appears for Order Type of 07 or 08 on the Sales Order Header screen):
•
Y - Accumulate orders by customer. All consignment sales orders invoiced for a customer are accumulated on one order. Each time a product is added to the consignment order, it is added as a new line on the sales order.
•
C - Accumulate orders by product. All consignment sales orders invoiced for a product are accumulated on one line on a new consignment order if the selling warehouse and price for the item are the same.
•
A - Works the same as C except that all consignment sales orders invoiced for a product are accumulated on a new consignment. The first price found is used.
•
N - Do not accumulate orders.
•
E - Expense. Consult with your systems administrator for information.

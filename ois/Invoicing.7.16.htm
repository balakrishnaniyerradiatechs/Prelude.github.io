Printing Invoices Using the DD250 Form
Any sales orders set up to print an invoice on a government DD250 form are not included in the standard invoice printing. You must use DD250 Invoice Printing to print the invoices.
To print invoices on DD250 forms:
1
Select Order Processing > Invoice Preview and Printing > DD250 Invoice Printing to open the DD250 Invoice Printing screen.
2
Enter the Sales Order Number for which you want to print a DD250 invoice or leave blank to print invoices for all confirmed sales orders that are set up to print DD250 invoices.
3
Enter the Warehouse number or leave blank to include all warehouses.
4
Enter the Invoice Date to print on the invoices and to use to calculate due dates and discounts.
5
Enter the Printer Form Queue to determine where the invoices print.
6
For Invoice Cycle, enter the invoice print cycle for which you want to include invoices. Each customer is assigned an Invoice Print Cycle in the Customer Maintenance Additional Order Information Screen. Only orders for customers whose Invoice Print Cycle matches this selection are included with these invoices. Choose from the following:
•
D - Daily.
•
W - Weekly.
•
M - Monthly.
•
S - Semi-monthly.
7
For Print Source, indicate which types of invoices to include. Choose from the following:
•
I - Invoices only.
•
C - Credit memos only.
•
B - Both invoices and credit memos.
•
M - Mail duplicates of counter invoices to customers whose Mail Invoices flag is set to Y in the Customer Maintenance Additional Order Information Screen.
8
Press F2-Save to print the invoices. The following message appears:
Load Invoice Forms In Printer & Align Visually
9
Choose one of the following:
•
Print Invoices - Prints the invoices.
•
Mask Print Test - Tests the alignment of the invoice forms in the printer. After the forms are correctly aligned, select Print Invoices.
•
Abort Printing - Cancels the invoice print process.
10
After you select Print Invoices, the following message appears:
Invoice Printing Complete....
11
Choose one of the following:
•
Process - The invoices printed successfully.
•
Reprint All - None of the invoices printed successfully. This option is also useful if only a couple of invoices printed successfully from a large print job. The system re-prints the invoices and this message appears again.
•
Single Reprint - To re-print invoices individually. This option is useful if a couple of invoices jammed in the printer. Enter the sales order number for the invoice you want to re-print.
•
Abort - Cancel all invoicing.
12
Once you select Process, the following message appears:Choose one of the following:
Are You Sure You Want to Process?
•
Reprompt - Keep this dialog box open.
•
Yes - Complete invoice processing, update the Order Log Inquiry and move the order to the Sales Order History Inquiry.
•
No - Return to step 11.
To re-print an invoice, select F5-Invoice Re-print from the DD250 Invoice Printing screen. You can either enter the invoice number or the original sales order number.
 
DD Form 250 Field Descriptions
Field/Function Key
Description
Block 1
PROCUREMENT INSTRUMENT IDENTIFICATION (CONTRACT) NO.
The customer purchase order number from the sales order.
 
ORDER NO.
The sales order release number.
Block 2
SHIPMENT NO.
The shipment number from the DD250 Information screen.
Block 3
DATE SHIPPED
The date the material shipped. For example, September 10, 2006 displays as 2006SEP10.
Block 4
B/L
The bill of lading number from the DD250 Information screen.
TCN
The transaction control number from the DD250 Information screen.
Block 5
DISCOUNT TERMS
Payment Terms from the sales order header Order Additional Information Screen.
Block 6
INVOICE NO./DATE
The invoice number and date.
Block 7
PAGE OF
The page number (hard coded).
Block 8
ACCEPTANCE POINT
The acceptance point from the DD250 Information screen that indicates where the material will be inspected:
•
S - At the source.
•
D - At the destination.
•
N - Neither.
Block 9
PRIME CONTRACTOR CODE
The Prime Contractor Code from Warehouse Maintenance.
Block 10
ADMINISTERED BY CODE
The administered-by code from the DD250 Information screen. Refer to Creating Administered-By Codes.
Block 11
SHIPPED FROM
The address from block 9 if Ship From Code is left blank on the DD250 Information screen.
SHIPPED FROM CODE
Indicates where the Shipped From Code came from:
•
S - From block 9.
•
M - Manually entered.
SHIPPED FROM FOB
The FOB Point from the DD250 Information screen.
Block 12
PAYMENT WILL BE MADE
The Payment By Code from the DD250 Information screen (refer to Creating Payment-By Codes).
Block 13
SHIPPED TO
The customer ship-to address from the sales order.
Block 14
MARKED FOR CODE
The Marked for Code from the Government Information screen that appears when you enter line items on the sales order.
Block 15
ITEM NO
The Government Item Number from the Government Information screen that appears when you enter line items on the sales order.
Block 16
STOCK/PART NO/DESCRIPTION
Includes the following:
•
The alternate product number entered in the Product field.
•
The product description.
•
Mil number from the Government Information screen.
•
Product number.
•
The cage code from the Government Information screen.
•
FMS (Foreign Military Service) contract notes from the Government Information screen.
Block 17
QUANTITY SHIP/REC’D
The quantity shipped on the sales order.
Block 18
UNIT
The unit of measure.
Block 19
UNIT PRICE
The unit price from the sales order.
Block 20
AMOUNT
Quantity shipped x unit price.
Block 21
CONTRACT QUALITY ASSURANCE
The quality assurance representative from the DD250 screen.
Block 22
RECEIVER’S USE
No data is entered in this field. The customer fills it in.
Block 23
CONTRACTOR USE
Contractor notes entered in the DD250 screen.

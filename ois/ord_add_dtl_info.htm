Line Item Detail Additional Information screen
The Line Item Detail Additional Information screen allows you to enter additional customer and vendor information for a product on a sales order.
Menu Path: Order Processing > Sales Order Entry > Line Item Detail screen > F5-Add Info.
Field and Function Key Descriptions
Field/Function Key
Description
Expected Date
The date the product is expected to ship to the customer.
Cust P/O Line#
The line number from the customer’s purchase order, which defaults from the line number on the sales order.
Customer Product
The customer’s part number for the product. Refer to Setting Up Customer Product Numbers.
Ship Via
The ship via method for this line item, if it is to be shipped by a different method than the other items on the order.
Accept B/O
Determines if the customer will accept backorders for this product:
•
Y - The customer will accept a backorder for this product.
•
N - The customer will not accept a backorder for this product.
This setting overrides the Accept Backorders setting in the sales order header Order Additional Information Screen for this line only.
Controlling Product
Determines if the product is a control product:
•
Y - The product is a control product which means that any additional products on the sales order will not ship until this product is available to ship:
•
N - The product is not a control product.
Only one control product is allowed per sales order (refer to Using a Control Product).
Ship Line Item Comp
Determines if the line item is held until the full quantity is available to ship:
•
Y - Do not ship this line item until the full quantity is available. Only available for sales order types 01 and 14.
•
N - Ship any available quantity and then ship the rest as it becomes available.
Refer to Shipping Sales Orders Complete.
Customer Acct Code
The customer’s accounting code for the product, if required by the customer. Require Acct Code in the customer’s Order Information Screen (Customer Maintenance > F7-Order) determines if the customer requires an accounting code for each line item on a sales order or quote. If an account code pattern exists for the customer in the Acct Code Pattern Match Screen, you must also match the accounting code pattern.
Vendor Contract#
The vendor’s contract number. If you use purchase order EDI transmissions and this line is linked to a purchase order, the price defaults from the price matrix but can be overridden up until the line is linked to a purchase order.
Vendor Quote#
The vendor’s quote number. If you use purchase order EDI transmissions and this line is linked to a purchase order, the price defaults from the price matrix but can be overridden up until the line is linked to a purchase order.
Vendor Salesman#
The vendor’s sales person number if you use EDI transmissions and this line is linked to a purchase order.
Campaign Priority Code
The campaign priority code that the customer is responding from, if you use the Marketing Module and have campaigns set up.
Substituted Product
The substitute product number if a product was substituted for this line item. You can also use this field to record usage for a product that you would normally have entered if it was available. For example, if you sell a product that was not what the customer really wanted, you would want to record usage for the product that they would have bought if it was available.
Auto Cancellation Date
The date by which a backorder will be cancelled if the product is not received. If the line is picked or linked when this date is reached, it will not be deleted.
Unusual Usage Flag
Indicates if the usage for this product is unusual and therefore should not update the product’s usage history:
•
Y - The sale of this product is unusual and usage should not be updated.
•
N - The sale of this product is normal and usage should be updated.
F2-Save
Saves the line item information.

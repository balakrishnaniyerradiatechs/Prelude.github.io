Customer Product Number Maintenance screen
If a customer wants to use their own part numbers when they place an order, you can use Customer Product Number Maintenance to enter customer part numbers to reference your own product numbers. If you enter the customer part number on a sales order, your own part number automatically appears on the order. If several customers use the same part numbers to reference products (e.g. divisions of a large company), you can create the part numbers for the parent customer and then share the information with other customers.
You can also use Customer Product Number Maintenance to set up customer part numbers for a customer price group so that the products may be used for all customers assigned to the same Price Group in the Customer Maintenance Order Information Screen.
Refer to Setting Up Customer Product Numbers.
Menu Path: Order Processing > Maintenance > Order Entry Control Files > Customer Product Numbers
Field and Function Key Descriptions
Field/Function Key
Description
Customer/Cust Prc Grp
The customer or customer price group:
•
The customer number if you want to enter a product number based on a customer. To share this customer part number with other customers, enter this parent customer number in the CPN Customer Number field in the Order Information Screen (Customer Maintenance > F7-Order).
•
The customer price group if you want to set up a product number by price group. Price groups allow you to assign the same pricing, prices matrices, or discounts to a group of customers. You can create price groups in the Customer Price Group Maintenance Screen and then assign one or more price groups to each customer in the Customer Maintenance Order Information Screen.
Cust Product Number
The customer’s part number that corresponds to your system part number. When you enter this number on a sales order, your corresponding Product number appears.
Product
Your product number that corresponds to the customer’s part number.
Description One
The first line of description for the product, if it is different than what exists in Product Maintenance.
Description Two
The second line of description for the product, if it is different than what exists in Product Maintenance.
Code Number One
Any special code that may apply to this customer’s product number. This information can be used on reports, pick tickets, or invoices.
Code Number Two
Any additional codes, if applicable.
Accounting Code
The accounting code if the customer requires an accounting code (Customer Maintenance > F7-Order > Require Acct Code = Y). This code is the default for Customer Acct Code on the Line Item Detail Additional Information screen when the product is entered on a sales order. Refer to the Acct Code Pattern Match Screen for more information on accounting codes.
Resale Exemption
Indicates if the customer product number is considered tax exempt:
•
Y - The customer part number is tax exempt.
•
N - The customer part number is taxable.
Package Qty
The quantity in a package, if it is different than what exists in Product Maintenance. This information appears on reports.
F2-Save
Saves the customer product number information.
F4-Del
Deletes the customer product number.
F5-Inventory Inquiry
Opens the Inventory Inquiry screen to display product availability information for the product.

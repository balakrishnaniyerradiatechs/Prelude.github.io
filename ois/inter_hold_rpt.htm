Interbranch S/O Hold Report
The Interbranch S/O Hold report lists all orders on branch sales order hold. An order will go on branch hold if the entire quantity of a product cannot be filled from surplus inventory at the shipping warehouse. The surplus quantity is defined in the product in Product Purchasing Parameters Maintenance. The order must be released by a branch manager before the pick ticket will print (refer to the Branch Release Inquiry for more information on branch hold).
The Interbranch/Surplus Only flag in Order Processing Parameters must be set to Y to use interbranch sales order hold.
Menu Path: Order Processing > Reports > Daily Order Detail Reports > Interbranch S/O Hold
Selection Criteria
Use the following criteria to limit the information to print on the report:
•
Division. The highest possible grouping in your company (e.g., Eastern Division, Western Division).
•
Region. A subset of Division - if Division is used (e.g., Southeast Region, Northwestern Region).
•
Department. A subset of warehouse. Typically used to separate expenses within a warehouse (e.g., accounting, sales, warehouse).
•
Cost Center. A subset of Department - if Department is used. Typically used to separate expenses within a warehouse or department (e.g., shipping, receiving, outside sales, inside sales).
•
Warehouse. The shipping warehouse.
•
Sort By 1) Shp 2) Sell Whs. Determines if you want to sort the report by shipping warehouse or selling warehouse:
-
1 - Sort by selling warehouse.
-
2 - Sort by shipping warehouse.
Column Descriptions
Column Heading
Description
Order#
The order number on interbranch hold.
Sell Cust#
The customer to receive the material.
Port
The port number where the sales order was entered.
Rep1
The Salesrep #1 assigned to the sales order.
Product Number
The product number.
Description
The product description.
Whse
The warehouse shipping the material.
U/M
The unit of measure of the product as entered on the sales order.
Quantity
The quantity of the product entered on the sales order.
Status
(unlabeled field)
The interbranch hold status description:
•
Not processed. No action taken yet.
•
Rejected. Will not be released from hold.
•
Accepted. Released from hold.
Currency Indicator
(unlabeled field)
The type of currency, if you use multiple currency.

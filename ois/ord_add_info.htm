Order Additional Information Screen
The Order Additional Information screen displays default information for the customer from the Customer Maintenance Screen or the Customer Ship-To Maintenance Screen. You can change any of the information as necessary. This screen may automatically appear when you save the Sales Order Header screen if any required information is missing for this customer.
Menu Path: Order Processing > Sales Order Entry > F5-Hdr Scr > F5-Add Info
Field and Function Key Descriptions
Field/Function Key
Description
Fob Description
The freight on board description.
Purchasing Agent
If the customer has authorized purchasing agents identified in their Customer Authorized Purchasing Agents Screen (Customer Maintenance > F8-Contact > F6-Purch Agents), you must enter an authorized purchasing agent who can enter an order for this customer.
Quote#
If the order was converted from a quote (refer to Converting a Quote to a Sales Order), the quote number displays.
Freight Shipping Type
The freight shipping type of arrangement you have with this customer for orders shipped by UPS, FedEx, and RPS, if you use the Parcel Shipping System (PSS):
•
CBS - Consignee billing.
•
COL - Freight collect.
•
TPB - Third party billing.
The default is set in the customer’s Freight Information Screen.
UPS Account Number (unlabeled)
The consignee’s UPS account number, if applicable.
Acknowledge Flag
Determines if and how an acknowledgment is sent to the customer for this order:
•
E - Email only.
•
M - Email and print.
•
N - No acknowledgment.
•
F - Fax only.
•
C - Print and fax.
•
T - Text email.
•
Y - Print only.
The default is from Ack Document Option in the customer’s Additional Order Information Screen. Refer to Sending Acknowledgements.
Fax Number
(unlabeled field)
The fax number, if you are faxing the acknowledgement to the customer.
Territory
The territory number used to group customers for sales and marketing reports. These codes are set up in the Territory Maintenance Screen and assigned to a customer in the Additional Order Information Screen or the Customer Ship-To Maintenance Screen.
Floor Plan
Indicates if this customer uses Floor Plan Financing (i.e., another customer is responsible for payment):
•
Y - This customer uses another customer as a financial institution (refer to the Using Floor Plan Financing for information on setting up a floor plan customer).
•
N - This customer does not use floor plan financing.
Floor Plan Customer Number (unlabeled field)
If you entered Y for Floor Plan, enter a floor plan customer who is responsible for payment, whose credit limit is checked, and whose A/R balance is updated for this sales order. The default for the customer is set up in the customer’s Floor Planning Maintenance Screen.
Specifier#
The customer number associated with the book for this sales order. This field is only accessible if you use the book/sample subsystem.
Auto Cancellation Date
The date by which this order will be cancelled by Nightly Processing if a backorder still exists. If the order still has backordered lines when this date is reached, each line will be cancelled if no linkage or pick quantity exists. You can override this setting at the line item level. Refer to Automatically Cancelling a Backorder.
Add. Discount%
An additional discount percent that applies to each product sold to this customer. The default comes from the customer’s Additional Order Information Screen.
Chain Discount
Indicates if chain discounts exist for the customer:
•
Y - Opens the Chain Discounts screen to allow you to enter the chain discount percent and description. Chain discounts allow you to give multiple discounts to a customer. For example, if you have chain discounts of 10% and 20%, the system multiplies the sales price by .9 and then by .8 to determine the final price.
•
N - Chain discounts do not apply for this customer.
The default comes from the customer’s Additional Order Information Screen.
Master Account Inv
If this customer is a sub-account and a Master Account is assigned, this flag determines which account is updated with accounts receivable information and which address prints on invoices:
•
N - The sub-account is updated with A/R information and receives the invoices.
•
Y - The master account is updated with A/R information and receives the invoices.
•
A - The master account is updated with A/R information, but the invoices are sent to the sub-account.
The default master account information comes from the customer’s A/R Information Screen.
Print DD250 Flag
Determines if a DD250 invoice generates for this order instead of a regular invoice:
•
Y - The customer requires a DD250 form.
•
N - The customer does not require a DD250 form.
The DD form 250 is a material inspection and receiving report that prints for government customers. The default is set in the customer’s Government Information Screen. Refer to Entering a Sales Order that Requires a DD250 Invoice.
Price Pick Tick
Determines if prices should appear on picking tickets and packing lists for this sales order:
•
Y - Print prices.
•
N - Do not print prices.
The default is from the customer’s Additional Order Information Screen.
Allocation Priority
You can enter a priority code if you want to override the default priority code set in the customer’s Additional Order Information Screen. Allocation priority determines where the customer ranks when the Allocation Procedure allocates received stock to backorders. You can rank a customer from A to Z, with A being the highest priority and Z being the lowest. Customers with no priority code are selected last.
Blind Shipment
Determines if the sales order is sent as a blind shipment:
•
Y - The sales order is sent as a blind shipment. The packing list prints the customer information in the logo area. The bill-to and ship-to addresses print the sales order’s ship-to address.
•
A - The same as Y, except that when you enter a sales order for this customer, a message appears to ask whether or not to send the order as a blind shipment.
•
N - Sales orders for this customer are not blind shipments. The company logo is included on the pick ticket and packing list.
A blind shipment means that the sales order will ship to this customer, without your company logo. The default is from the customer’s Additional Order Information Screen or the Ship-To Additional Information Screen.
Supp Inv Print/Fax
Determines if invoices generated by this order should be faxed and/or printed:
•
Y - Suppress the printing or faxing of invoices for this order.
•
N - Print/fax invoices for this order.
Accept Backorders
Determines the default for every line on the sales order, to indicate if the customer accepts backorders:
•
Y - The customer normally accepts backorders.
•
N - The customer does not typically accept backorders but you are prompted to determine whether or not this customer accepts backorders when entering the line items.
•
R - Allows you to override this flag in Release Confirmation Entry. This option only works if the Auto Backorder Cancellation flag in Order Processing Parameters is set to N. You are notified in Release Confirmation Entry that the customer normally does not accept backorders but you can override it if necessary.
The default is from Accept Backorders in the customer’s Additional Order Information Screen.
Chg Frt/Hnd Surchg
Indicates if additional freight and handling surcharges (calculated in Order Processing Parameters > F7-Add Frt/Handling) are added to this order:
•
Y - Freight surcharges apply.
•
N - Freight surcharges do not apply.
Two flags in Customer Maintenance (F7-Order > F7-Frt Info) determine if and when surcharges are added to a customer’s order:
•
Charge Frt/Hndlg Surcharge determines if surcharges are added.
•
Addl Frt/Handling Chg determines whether surcharges are added to Outbound Freight or Handling on the Sales Order Completion screen.
Refer to the customer’s Freight Information Screen for more information on calculating surcharges.
Unusual Usage Flag
Indicates if usage for this order is considered unusual and if usage should be updated for the products sold on this sales order:
•
Y - Unusual usage. Do not update usage for the products sold.
•
N - Not unusual usage. Update usage for the products on this sales order.
Refer to “Product Usage” in the Inventory documentation for more information.
Consolidated Inv?
If the customer is not set up to normally use consolidated invoicing (refer to Consolidate Invoices in the Additional Order Information Screen), you can determine if this order should be consolidated for invoicing:
•
Y - Consolidates invoices for this order only.
•
N - Uses the customer’s default setting.
Delivery/Pick Up
Indicates if delivery and/or pick-up charges are billed to the customer:
•
D - Delivery charges.
•
P - Pick-up charges.
•
B - Both delivery and pick-up charges.
•
N - Neither delivery or pick-up charges.
This field is used primarily with the Rental module when equipment is delivered or picked up at a site. The default comes from the customer’s Additional Order Information Screen.
Email Ship Notice
Determines if shipping confirmations are sent when releases for this order are confirmed in Release Confirmation Entry:
•
P - Print shipping confirmations.
•
F - Fax shipping confirmations.
•
E - Email shipping confirmations.
•
Y - Text email shipping confirmations.
•
N - Do not send shipping confirmations.
The default comes from the customer’s Additional Order Information Screen. Refer to Sending Shipping Confirmations for more information.
Language Indicator
The code that indicates which language appears on pick tickets, packing lists, quotes, and invoices for this customer. The default comes from the customer’s Order Information Screen or the Ship-To Additional Information Screen.
Replacement Equip
Indicates if this sales order is to replace material returned by the customer:
•
Y - The sales order is for replacement equipment.
•
N - The sales order is not for replacement equipment.
F2-Save
Closes the Order Additional Information screen.

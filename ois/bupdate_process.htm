Updating Sales Statistics (‘B’ Update)
Updating inventory and sales statistics after products are sold is a two-part process. Invoicing (also known as the ‘A’ update), updates inventory and posts to G/L accounts. After invoicing, you must run the ‘B’ Update to update files, commissions, rebates, contracts, sales statistics, and other sales analysis information. Each time you run the ‘B’ update, the file is cleared until more sales orders are invoiced. Typically, this process is scheduled and runs automatically.
To run the Sales Order Processing ‘B’ Update:
1
Select Order Processing > Processes > Process Invoices – (‘B’ Update) to open the Sales Order Processing ‘B’ Update screen.
2
Enter an Order Number if you want to limit the update to just one order or leave this field blank to include all invoiced orders.
3
Enter a Warehouse if you want to limit the update to just one warehouse or leave this field blank to include all warehouses.
4
Once you choose Execute, the following information is updated in the system.
•
Rebates. The rebate amount of each sales order is updated. The commission amount for the salesman is less the rebate cost.
•
Blanket Contract. Act Qty Shipped, Total Price Ext, Total Cost Ext, Last Ship Date, Last Ship Qty, # of Orders in Blanket Contract Maintenance are updated for the blanket contract number on the invoiced sales order.
•
Commissions. The sales representatives commissions are updated if Commission is set to Y in File Update Parameters (refer to the Commission Subsystem for more information on commissions).
•
Credit Reason Statistics. The credit reason statistics are updated if Update Credit Rsn Stats is set to Y for the credit reason code in Credit Reason Maintenance.
•
Order Pad Inquiry. Updated with sales history information for a product if Order Pad is set to Y in File Update Parameters.
•
Sales Analysis. The sales analysis files are updated for all invoice sales orders.

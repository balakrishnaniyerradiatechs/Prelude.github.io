Creating Lost Sales Reason Codes
If you track lost product sales, you can set up lost sales reason codes to identify why a product was deleted from an order. For each deletion, you can also determine whether or not to update product usage for the lost sale.
File Update Parameters
To track lost product sales, you must set Lost Sales in File Update Parameters to Y so that the salesperson is required to enter a lost sales reason code.
To create lost sales reason codes:
1
Select Order Processing > Maintenance > Misc Code Files > Lost Sales Reason Codes to open the Lost Sales Reason Maintenance screen.
2
For Lost Sales Reason#, enter the 1 or 2 digit alphanumeric code that represents an order cancellation reason code.
3
Enter the Description of the code that explains why the order was lost.
4
For Update Product Usage, determine if product usage is updated on a sales order or quote when the product is deleted:
•
Y - Update product usage for the lost sale (e.g., out of stock).
•
N - Do not update product usage for the lost sale (e.g., price).
5
For Update Product Alerts, determine if a lost sale with this reason code should update the Product Alerts Queue in the Buyer Queue Inquiry.
•
Y - Update the Product Alerts buyer queue. This reason code will display as the Reason the product is in the queue.
•
N - Do not update the Product Alerts buyer queue.
6
Press F2-Save to save the lost sale reason code.
Refer to “Product Usage” in the Inventory documentation for more information on updating product usage.
Automatic Cancellation Codes for Backorders
To track lost sales for auto-cancelled backorders (refer to Automatically Cancelling a Backorder) you must create a lost sales reason code of CD (Cancellation date) to use on sales orders when a backorder is not received by a date. CAN displays in the Order Log Inquiry when an order is automatically cancelled.

Consignment Types screen
The Consignment Types screen appears when you enter an 07 Consignment order type in the Sales Order Header screen to allow you to define consignment information for the order.
Menu Path: Order Processing > Sales Order Entry > F5-Hdr Scr > Order Type = 07
Field and Function Key Descriptions
Field/Function Key
Description
Consign Type
The type of consignment order:
•
01 - Salesperson sample.
•
02 - Display.
•
03 - Consigned to customer.
•
04 - Reserved for customer.
•
05 - Invoice when complete.
Accumulate
Determines if all 07 orders are accumulated into one type 08 consignment order when the order is confirmed:
•
Y - Accumulate orders by customer. All consignment sales orders invoiced for a customer are accumulated on one order. Each time a product is added to the consignment order, it is added as a new line on the confirmed sales order.
•
C - Accumulate orders by product. All consignment sales orders invoiced for a product are accumulated on one line on a new consignment order if the selling warehouse and price for the item are the same.
•
A - Always accumulate by product number. Works the same as C except that all consignment sales orders invoiced for a product are accumulated on one line on a new consignment order even if the price for the item is different. The last price on the order to be consolidated is used.
•
N - Do not accumulate orders.
•
E - Expense (see your systems administrator to use this option).
Review Date
Indicates when the order should be reviewed for consignment items on loan to the customer. The date is used for the Consignment Review Date Report.
F2-Save
Saves the consignment information and returns to the Sales Order Header screen.

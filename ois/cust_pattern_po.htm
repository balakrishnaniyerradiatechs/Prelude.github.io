PO Pattern Match Screen
The PO Pattern Match screen allows you to establish purchase order patterns that must match the purchase order pattern on sales orders for this customer. You can set patterns that contain letters, numbers, both letters and numbers, dashes, and strings. You can enter multiple patterns, but only one pattern per line.
If the customer or customer ship-to has a default purchase order number, the pattern is validated if you access the Customer P/O# field.
Menu Path: Order Processing > Maintenance > Customer Maintenance > F7-Order > Require P/O Number = Y > F5-Po Patterns
Field and Function Key Descriptions
Field/Function Key
Description
Po Patterns
The alpha, numeric, alpha-numeric, dashes, or strings to include in required purchase orders for this customer. Refer to the following examples (change the number as necessary):
•
4A - The purchase order must be 4 alpha characters (e.g., PRNG)
•
4N - The purchase order must be 4 numeric characters (e.g., 4934).
•
8X - The purchase order must be 8 characters that could be both alpha or numeric (e.g., 4BC23999).
•
3A4N - The purchase order must have 3 letters followed by 4 numbers (e.g., ACT3398).
•
3A-4N - The purchase order must have 3 letters followed by a dash and then 4 numbers (e.g, ACT-3398).
•
3A.6N - The purchase order must have 3 letters followed by a decimal and then 6 numbers (e.g, ACT.339821).
•
“A”4N - The purchase order must be the letter “A” followed by 4 numbers (A3499).
•
0A - The purchase order can be any number of alpha characters.
•
0N - The purchase order can be any number of numeric characters.
•
0X - The purchase order can be any number of both alpha and numeric characters.
F2-Save
Saves the purchase order information for this customer.

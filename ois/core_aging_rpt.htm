Core Returns Due Aging Report
The Core Returns due aging report list pending cores not yet received. This information can help determine what products remain with the customer and when they are expected to arrive.
A core product is the re-usable part of a product that a customer returns either for credit or for a discount against a new item. If you set up a product as a Core Product (Product Maintenance > F5-Add Info > F5-Add Info), you should also set up a product to exchange for the core product when it is returned.
A pending core is created when you sell an exchange product to a customer but do not charge them for the core (reusable) part of the item. The core charge is in a pending status until it is billed to the customer or a core bank item is applied to it. A pending core is an open receivable.
For more information on cores, refer to “Tracking Pending Cores” in the Inventory documentation.
Menu Path: Order Processing > Reports > Miscellaneous Reports > Core Reports1 > Core Returns Due Aging Report
Selection Criteria
Use the following criteria to limit the information to print on the report:
•
Customer Range. A beginning and ending set of customer numbers, if you want to limit the report to just these customers.
•
As of Date <. The date to evaluate cores based on the core due date. The report lists the cores due in the appropriate columns based on this date.
•
Core Code. The core codes to include. The core type codes group products for core banking and are created in Core Type Code File Maintenance in the Inventory Module.
Column Descriptions
Column Heading
Description
Product Number
The product number.
Description
The product description.
Past Due
If the core product is past due, the quantity of the product displays in the column that corresponds to the number of days that it is past due.
Current
The quantity of the core product that is current.
Future
If the core product has a future due date, the quantity of the product displays in the column which indicates the number of days until it is due.

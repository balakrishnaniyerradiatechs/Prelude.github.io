Inspection Results Entry screen
Once a product is inspected, you can use the Inspection Results Entry screen to record the results and to identify the document where the product was first found to be defective (refer to Inspection Results Entry). You can also recommend a sample size to inspect based on the lot size (refer to Setting up Inspection Levels for Sample Sizes).
Menu Path: Order Processing > Additional Subsystems > Quality Inspection Subsystem > Inspection Results Entry
Field and Function Key Descriptions
Field/Function Key
Description
Inspection#
The inspection number that identifies the inspection results.
Entry Date
The date the inspection is entered.
User Id
The user ID of the person entering the inspection results.
Status
The current status of the inspection result:
•
O - Open.
•
P - Pass.
•
F - Fail.
Inspect Type
Identifies how the inspection request was generated:
•
C - Customer return.
•
P - PO receipts.
•
V - Vendor performed.
•
W - W/O manufacturing.
Once you enter the Inspect Type, the default Procedure# appears under Inspection Plan (refer to Assigning Default Procedures to Inspection Types).
Agent
Enter the Agent, that reported the defect or is assigned to the inspection (refer to Creating Quality Agent Codes).
Document#
The document number that corresponds with the inspection type (e.g., purchase order number, credit memo number, work order number).
Ln#
The line on the purchase order, credit memo number, or work order number.
Source
The vendor if you entered a purchase order number for Document#.
Product
The product number being inspected.
Lot Size
The total quantity received.
Inspection Plan
Procedure#
Defaults to the procedure established as the default in Quality Inspection Procedure Default Maintenance (refer to Assigning Default Procedures to Inspection Types).
Sample Size
Defaults to the recommended sample size to inspect, based on the Lot Size. The lot size information from Sample Size code Letter Maintenance (refer to Setting up Inspection Levels for Sample Sizes) determines the sample size (refer to Using Single Sampling Tables).
Std.Level
The recommended General Inspection Level based on the lot size. This information is from Sample Size code Letter Maintenance (refer to Setting up Inspection Levels for Sample Sizes). If you change the level, the Sample Size adjusts according to the value set up in sampling tables (refer to Using Single Sampling Tables).
Number/Cartons
The number of cartons in the shipment.
Inspection Results
Type
The code for the severity of the problem found during inspection:
•
C - Critical.
•
J - Major.
•
N - Minor.
Code
The defect code that represents the problem found.
Qty
The defective quantity found.
Description
An explanation of the problem.
F2-Save
Saves the inspection results entry.
F4-Del
Deletes the inspection results entry.
F5-Summary
Opens the Inspections Summary Statistics screen to display a summary of inspection statistics by severity type. (e.g., pass, fail).
F6-Instructions
Opens the Inspections Procedures screen to allow you to view any instructions set up for the product in Product Maintenance > F5-Add Info > Qual Insp Required? = Y > Instructions. Refer to Assigning Inspection Procedures for Products.

Credit Reason Maintenance screen
Credit reason codes are required when you enter a credit memo to issue credit to a customer for returned material. The credit reason codes define why a customer is returning material, update inventory, and post to the appropriate general ledger account. Refer to Creating Credit Reason Codes for more information.
Menu Path: Order Processing > Maintenance > Misc Code Files > Credit Reason Codes
Field and Function Key Descriptions
Field/Function Key
Description
Credit Reason Code
The 1 or 2 digit code to use on credit memos and RGAs to identify why a product is returned. The following are examples of commonly used credit reason codes:
•
01 - Shipped wrong product.
•
02 - Defective product. Refer to Creating Warranty Tags for Returned Products.
•
03 - Policy credit.
•
04 - Customer did not need.
•
05 - Pricing error.
•
CE - Core exchange credit.
•
EQ - Equipment trade-in credit.
Product Line
The product line number if you want to set up a general ledger account just for this credit reason/product line combination. Leave blank to use the general ledger account assigned to the product line or to use a general ledger table.
Description
The description of the credit reason.
Restock Charge Percent
The restocking charge percentage to use to calculate the restocking charge per line item to add to the order when this credit reason code is used. This percentage overrides any restocking charge percentage at the company level. Refer to Creating Credit Reason Codes for more information on restocking charge flags.
Default Inventory Type
The inventory type for returned items that use this credit reason code:
•
D - Damaged. The quantity will be moved into damaged inventory.
•
R - Return to stock. The quantity will be moved to regular stock.
•
S - Scrapped. The quantity will be scrapped or thrown away.
•
W - Warranty/scrapped. The quantity will be put in Warranty Inventory.
If this field is blank, then the inventory bucket defaults to the inventory type in Vendor Maintenance for the product’s primary vendor. If the primary vendor does not have an inventory type, then it defaults to the inventory type defined in Additional Option Parameters. If that is blank then it defaults to Return to Stock.
Update Credit Rsn Stats
Determines if credit reason statistics are updated when this reason code is used:
•
Y - Update credit reason statistics.
•
N - Do update credit reason statistics.
Update Customer Stats
Determines if customer statistics (YTD Returns) are updated when a credit memo with this credit reason code is invoiced:
•
Y - Update customer statistics.
•
N - Do not update customer statistics.
Update Product Usage
Determines if product usage is updated when a credit memo with this credit reason code is invoiced:
•
Y - Update product usage.
•
N - Do not update product usage.
Exclude From Quality Reporting?
Determines if the credit reason code is excluded on the Customer On-Time Shipments/Quality Report when the credit memo is invoiced:
•
Y - Exclude the credit reason code.
•
N - Include the credit reason code.
Equipment Trade In
Determines if the trade-in serial number information is required on credit memos that use this credit reason code:
•
Y - Require equipment trade-in information to update the make, model, price, and cost for the new serial number.
•
N - Do not require trade-in information.
Warehousing Specific Information
Update Picker Stats
If you use the Warehousing module, you can determine if pick error information is updated when a credit memo with this credit reason code is invoiced:
•
Y - Update pick error information.
•
N - Do not update pick error information.
F2-Save
Saves the credit reason code information.
F4-Del
Deletes the credit reason code information.
F5-G/L Accounts
Allows you to override general ledger account information for the credit reason code.
F6-Disposition Inst
Allows you to enter instructions for disposing of the material for this credit reason code.
F5-Restock
Opens the Restocking Fee Ranges screen to allow you set multiple restocking charges to apply depending on the number of days since the invoice date, for this reason code.

Closing a Credit Card Batch
You must close a credit card batch so that Protobase can complete the transfer of money from your customer’s account into your bank account.
If you use credit cards to pay open customer A/R balances (refer to “Receiving Credit Card Payments” in the Accounts Receivable documentation), make sure that any A/R Credit Card Payment Entry transactions are completed before you close the batch.
The timing in closing a batch is crucial to balancing with Protobase. Call Activant if you need help balancing or have any questions.
To close a credit card batch:
1
Select Order Processing > Processes > Credit Card Processes > Batch Close to open the Credit Card Batch Close Process screen.
2
Enter the Selling Warehouse number for the batch close.
3
For Credit Card Type, accept the default of VISA unless you use multiple merchant numbers for a credit card type.
4
For End Date, enter the post-authorization date. If the post-authorization was done yesterday, enter yesterday’s date.
5
For Close all Branches, choose from the following:
•
Y - Close all merchant IDs if you have more than one merchant ID per branch.
•
N - Only close the branch entered in Selling Warehouse.
6
Once you choose Execute, the following message appears:
Do you wish to Print Preview or Close the Batch?
7
Choose from the following:
•
Print - Prints the Protobase Administrator Terminal Report to allow you to verify the credit card transactions for the current batch. Choose this option the first time you attempt to close so you can match the batch total on this report with the batch total on the Credit Card Post Authorization Approval Report. Any discrepancies indicate a failure between Prelude and Protobase (e.g, if the post-authorization was sent to Protobase multiple times or if Protobase did not receive a post-authorization request). Make sure that the reports balance before you re-run and choose Close.
•
Close - Works the same as Print, except that after the report prints you can then choose to close the batch. Once you close, a final settlement report prints to list what the bank says you transmitted and it approves. If the Final Protobase Administrator Terminal Report does not match the Credit Card Post Authorization Approval Report, then you may need to contact Protobase to adjust the client’s card immediately so that it does not carry on to the next day.
Protobase Terminal Report - Column Descriptions
Column Heading
Description
Ref #
The credit card batch number.
Account
The customer’s credit card number.
Exp
The expiration date of the credit card.
Type
The Protobase terminology for transaction type (e.g., prior authorization).
Amount
The amount charged to the credit card, including tax and other charges.
Date
The date the credit card was used to pay for the transaction.
Bus.
The date the transaction was post-authorized.
Auth#
The authorization code from the Additional Credit Card Information screen when the sales order was pre-authorized. Refer to Pre-Authorizing a Sales Order.
Cashier
The order taker.
Order Num.
The sales order number.
AVS
The Address Verification Service approval number if you use AVS.
Zip Code
The zip code of the billing address for the credit card.
Billing Address
The billing address for the credit card.
Credit Totals for Terminal
Card Type
The type of credit card (refer to Assigning Credit Bureau Codes).
# Auth
The number of transactions authorized for this terminal.
Auth $
The total dollar amount authorized for this terminal.
# Sale
The number of sales made at this terminal.
Sales $
The total sales generated at this terminal, including authorized and non-authorized.
# Ret
The total amount of returns for this terminal.
Return $
The total value of returned material.
The remaining fields may apply to some Protobase customers but are not used by Prelude.

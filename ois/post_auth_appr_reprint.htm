Credit Card Post Authorization Approval Report
The Credit Card Post Authorization Approval Report generates when you post-authorize a credit card batch (refer to Post-Authorizing Credit Card Transactions). You can also use the Post Authorization Approval Report Reprint if you need to re-print the report. This report lists what Prelude post-authorized versus what the bank shows as correct.
Menu Path: Order Processing > Processes > Credit Card Processes > Additional Batch Functions > Post Authorization Approval Report Reprint
Selection Criteria
Use the following criteria to limit the information to print on the report:
•
Whse. The selling warehouse where credit card transactions took place.
•
Cc Type. The credit card types to include.
•
Post Auth Date Range. The beginning and ending dates that the transactions were post-authorized.
•
Tran Date. The date the sales orders were entered.
Column Descriptions
Column Heading
Description
Type
The type of credit card. Refer to Assigning Credit Bureau Codes.
Account Number
The credit card number.
Exp
The expiration date of the credit card.
Appr
The approval code assigned in Authorization Code on the Additional Credit Card Information screen when the sales order was pre-authorized. Refer to Pre-Authorizing a Sales Order.
AVS
The Address Verification Service approval number if you use AVS.
Order
The sales order number.
Pay Amt
The amount approved and paid on the invoice.
Time
The time the invoice was post-authorized.
Date
The date the invoice was post-authorized.

Order Information Screen
The Order Information screen allows you to customize information for a customer’s ordering, invoicing, and pricing requirements.
Menu Path: Order Processing > Maintenance > Customer Maintenance > F7-Order
Field and Function Key Descriptions
Field/Function Key
Description
Ship Via
The default ship via code that determines how orders generally ship to the customer. You can override the ship via for each ship-to address and for each order or each line item. These codes are created in the Ship Via Maintenance Screen.
Selling Warehouse
The default selling warehouse for the customer that receives credit for the customer’s purchases. You can override this warehouse for each ship-to address and for each order. Selling Allowed in Warehouse Maintenance > F5-Add Info must be set to Y to use a warehouse as a selling warehouse.
Shipping Warehouse
The default shipping warehouse for the customer that generally ships orders to this customer. You can override the shipping warehouse for each ship-to address and for each sales order or line item.
Contract Customer
The customer number to use for pricing orders for this customer. Refer to Contract Pricing.
CPN Customer Number
The customer number to reference when using customer part numbers. If several customers use the same customer part numbers for referencing products (e.g., divisions of a large company), you can create the part numbers for the main customer and then share the information with other customers. Refer to Setting Up Customer Product Numbers for more information.
Customer Rebate#
The rebate program to assign to this customer. Rebates are set up in the Customer Rebate Maintenance Screen to calculate rebates due to the customer for products purchased. Refer to “Generating Customer Invoice Rebates” in the Accounts Receivable documentation for more information.
Rebate Start Date
The date to begin calculating rebates for the customer.
Min Invoice Amount
The minimum invoice amount for this customer. If an order for this customer is less than this amount, then a Minimum Order Charge is added to the invoice. If you leave this field blank, then the minimum order amount for the order’s Ship-Via Code applies. This field overrides the minimum invoice amount at the company level.
#Invoice Sets
The number of copies of each invoice for this customer. If this field is set to zero, the customer will not receive printed/faxed/emailed invoices. Refer to Printing Invoices.
Fob
The default freight-on-board instructions for the customer’s orders.
Language Indicator
The code that indicates which language should appear on pick tickets, packing lists, quotes, and invoices for this customer. You can change this code for each ship-to or each for order.
Require Job Number
Indicates if this customer requires a job number on orders:
•
J - A valid job number is required on the customer’s orders. The job number must exist in the Customer Job Maintenance Screen. You cannot create a job number within a sales order.
•
N - A job number is optional on orders.
•
R - A job number is required on sales orders, but is not validated in the customer’s job account. You cannot create one in an order.
•
S - A job number is optional but is validated. You cannot create a job number within an order.
•
V - A job number is required but is not validated against the customer’s job account. You cannot create a job within an order.
•
Y - A job number is required but not validated on all sales orders for the customer. If a job account does not exist, you can create one while in the sales order. The job number is not validated against the Customer Job Maintenance Screen.
Refer to Customer Job Maintenance.
Customer Job Number
The default customer job number to default on all orders for this customer. This job must be a valid job number set up in the Customer Job Maintenance Screen.
Price Code
The price column for this customer:
•
0 - Suggested retail price.
•
1 - Price column 1.
•
2 - Price column 2.
•
3 - Price column 3.
•
4 - Price column 4.
•
5 - Price column 5.
Price Group
The price group classification for the customer, if applicable. Price groups are created in the Customer Price Group Maintenance Screen to group customers for discounts or special pricing. If you enter more than one price group for a customer, enter the primary price group first so that it is the default for retail price calculations. If the primary price group does not have the best price, the price group with the lowest price is used. You can also set up customer part numbers for a price group so that the products may be used for all customers assigned to the same price group (refer to Setting Up Customer Product Numbers). If the customer has Contract Pricing set to Y in Price Matrix Maintenance, the contract price overrides all price groups listed here. Refer to Contract Pricing.
Consign Consolidate Method
Determines how the customer consolidates consignment orders:
•
C - Consolidate at the customer level.
•
S - Consolidate at the customer ship-to level.
•
Blank - Use the system level setting in Order Processing Parameters > F5-Add Info > Consolidate Consign by Cust/ShipTo.
Require Acct Code
Determines if the customer requires an accounting code for each line item on a sales order:
•
Y - Opens the Acct Code Pattern Match Screen so you can set up accounting code patterns that must match the Customer Acct Code for a line item on sales orders (Sales Order Entry > Line Item Detail screen > F5-Add Info). If you do not enter any patterns, the accounting code is still required on sales order line items but it does not have to match any pattern. This accounting code is overridden by accounting codes in Customer Part Number Maintenance. If a customer part number exists, the accounting code in Customer Part Number Maintenance is assigned to the line items on the sales order (refer to Setting Up Customer Product Numbers).
•
N - Line items on orders for this customer do not require an accounting code.
Require P/O Number
Determines if a purchase order number is required on sales orders for this customer:
•
Y - The customer requires a P/O number. Opens the Customer Blanket P/O Numbers Screen to allow you to enter blanket purchase order information. You can also set P/O patterns that must match the pattern on the customers orders.
•
N - P/O numbers are not required for this customer.
•
O - Orders with no P/O number go on Administrative Order Hold.
•
I - Same as O, and you can enter the Administrative Invoice Hold limit for sales orders.
Refer to the Administrative Order Inquiry for information on using Administrative Order Hold.
Require Release Number
Determines if a release number is required on orders for this customer:
•
Y - Any order for the customer must have a Cust Release# on the sales order header.
•
N - The customer does not require release numbers.
Require P/O Line Number
Determines if a purchase order number is required for each line item on sales orders for this customer (Sales Order Entry > Line Item Detail screen > F5-Add Info > Cust P/O Line#):
•
Y - Any quote or sales order for the customer must have a Cust P/O Line# for each product on the order.
•
N - The customer does not require line item P/O numbers.
Allow Ship-To Creation
Determines if you can create a ship-to account for the customer when entering an order:
•
Y - You can create a ship-to address for the customer.
•
N - You cannot create a ship-to address for the customer.
Ration Type
The type of ration code to assign to this customer, if you use sales order rationing to limit the amount of a product or product line that a customer can purchase over a period of time (refer to Entering Rationed Products). You can set up ration types in Rationing Maintenance and then assign them to each customer in this field. Refer to “Customer Ration Types” in the Inventory documentation for information on customer rationing.
Package Insert Code
The insert code that indicates that a promotional item (e.g., brochure, catalog, free items) should be given to the customer based on product and sales order criteria. Refer to Creating Package Inserts for more information.
F2-Save
Saves the order information for the customer.
F5-Add Info
Opens the Additional Order Information Screen. You can set additional flags regarding orders for this customer.
F6-Sls Comm
Opens the General Notes Entry screen to allow you to enter or view comments for this customer regarding sales orders. These comments display in orders and quotes for the customer.
F7-Frt Info
Opens the Freight Information Screen. You can enter freight terms and calculations for the customer’s sales orders.
F8-Inc Ship Via
Opens the Incompatible Ship Via Maintenance Screen. You can enter any ship via codes that cannot be used when shipping orders to this customer.
Action Bar > Add’l Options
Alternate Shipping Warehouse
Opens the Alternate Shipping Warehouse Screen so you can establish a secondary shipping warehouse if the product is not available in the original warehouse. Applicable for EDI/DOE transmissions.
EDI Information
Opens the EDI Transmission Info Screen to allow you to enter EDI transmission information.
Government Information
Opens the Government Information Screen. You can enter information required by the government, if applicable.
Order Source Matrix
Opens the Order Source Matrix Screen to allow you to access additional discount information for orders for this customer, depending on how the order is generated.
Order Type
Opens the Order Types Screen. You can enter a default order type (e.g., 01, 02) for this customer.
Route
Opens the Customer Route Maintenance Screen. You can enter the route number normally used to deliver to the customer. If you enter more than one route, Sale Order Entry chooses the best route based on the order date.

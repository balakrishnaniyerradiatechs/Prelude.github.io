Customer Department File Maintenance screen
You can use the Customer Department File Maintenance screen to group products by department and accounting codes when you set up a crib warehouse. You can also enter the customer’s purchase order numbers to reference each accounting code, if an accounting code is required for each line item on a sales order. Refer to Maintaining Customer Department File Information.
Menu Path: Order Processing > Maintenance > Customer Control Files > Customer Crib Maintenance > Customer Department Maintenance
Field and Function Key Descriptions
Field/Function Key
Description
Customer
The crib customer number.
Customer Ship-To
The customer ship-to number.
Department
The department number to assign to the customer. This is used to separate expenses within a warehouse.
Description
The description of the department for just this customer.
Shrinkage
Indicates if inventory shrinkage is updated for the Customer Ship-to number. Choose from the following:
•
Y - Update shrinkage.
•
N - Do not update shrinkage.
Acct Code 1
The crib accounting code to display in the Crib Information screen when a product is entered on a crib sales order.
Code 2
The second crib accounting code to display the Crib Information screen when a product is entered on a crib sales order.
Customer P/O#
The customer’s purchase order number to use that must match the accounting code for each line item on a crib sales order.
PO Ln#
The line number from the original purchase order.
F2-Save
Saves the customer department file information.
F4-Del
Deletes the customer department file information.

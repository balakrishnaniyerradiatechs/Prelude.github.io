Prospect Conversion Screen
If you have a prospective customer set up in the Marketing Module that you want to convert to a regular customer, you can you use the Prospect Conversion screen to create the customer record without re-entering the existing customer information. You can only access this screen in customer maintenance before you enter a customer number.
Menu Path: Order Processing > Maintenance > Customer Maintenance > F6-Prospect Conv (before you enter a Customer)
Field and Function Key Descriptions
Field/Function Key
Description
Prospect Number
The existing prospect number created in Prospect Maintenance in the Marketing Module.
Description
The customer description.
Address Line One
The first line of address for the customer.
Address Line Two
The second line of address for the customer.
Address Line Three
The third line of address for the customer.
City, State, Zip
The city, state, and zip of the customer.
Customer Number
The customer number. You can assign a new number if necessary.
Price Group
The price group classification for the customer, if applicable. Price groups are set up in the Customer Price Group Maintenance Screen to group customers for special pricing. You can set up price matrices to establish discounts or pricing structures for each price group.
Salesrep One
The primary sales representative that appears as the default Salesrep One (Sales Order Header screen > F5-Add Info) on sales orders for the customer. You can override this ID on sales orders, but if sales representatives are assigned in the Authorized List of SalesRep #1 Screen, then you can only override Salesrep One with one of those names.
Salesrep Two
The second sales representative assigned to the customer (e.g., inside sales rep), if applicable. This sales representative number appears as the default Salesrep Two on sales orders (Sales Order Header screen > F5-Add Info) assigned to the customer. You can override this ID on sales orders, but if sales representatives are assigned in the Authorized List of SalesRep #2 Screen, then you can only override Salesrep Two with one of those names.
Freight Terms
The default freight terms code to use for orders for this customer. These codes are set up in the Freight Terms Maintenance Screen to indicate when inbound and/or outbound freight is normally charged to the customer. You can override the freight terms for a customer in Sales Order Entry.
Ship Via Number
The default ship via number to use to ship to this customer. Ship via codes are set up in the Ship Via Maintenance Screen, to specify carriers and charges for shipments. You can override this code in Sales Order Entry.
Payment Terms
The default accounts receivable payment terms code for the customer, which determines the cash discount percent, net due date, and method of payment. If you leave this field blank, you must enter an A/R terms code for each sales order for this customer.
Tx Jur
The tax jurisdiction code that determines the taxes to collect for this customer/location. These codes are set up in the Tax Jurisdiction Maintenance Screen for taxing structures and G/L allocations.
Description
The description of the tax jurisdiction.
Tx?
Indicates if tax is calculated for this tax jurisdiction:
•
Y - Calculate tax.
•
N - Do not calculate tax.
Exempt Number
The tax exempt number issued by the government if the customer is exempt from sales tax.
F2-Save
Returns to the Customer Maintenance Screen.

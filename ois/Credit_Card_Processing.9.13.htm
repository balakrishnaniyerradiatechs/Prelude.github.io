Pre-Authorizing a Sales Order
Once your system is set up for credit card processing (refer to Setting up Your System for Credit Cards), you can pre-authorize the sales order before releasing the order for pick ticket printing or invoicing.
Direct Ships
When you enter a Direct Ship order using a credit card, use the 01 regular order type instead of the 04 direct order type. Set the supply code for each line item to D for direct/drop ship so that the pre-authorization balances.
Invoice Hold
Use the Administrative Invoice Hold option in Order Processing Parameters to prevent invoicing confirmation (06) and direct (04) sales orders that are not pre-authorized.
To pre-authorize a credit card sales order:
1
Enter the sales order as usual (refer to Entering a Sales Order). When you complete the order, you must open the Sales Order Completion screen to access credit card information. Depending on the type of order, the sales order completion screen may automatically appear:
•
If the payment terms for the sales order are set up as credit card payment terms (refer to Setting up Credit Card Payment Terms) and Credit Card Entry Mode in the Payment Terms type is set to Y, when you finish entering the line items and press F2-Save, the Sales Order Completion screen automatically appears.
•
If the payment terms for the order are not set up as credit card payment terms (e.g., cash sales order or any other customer who wants to pay by credit card), when you finish entering the line items, press F2-Save and choose Complete. The Sales Order Completion screen appears.
2
Verify that the Terms Code is set to credit card or cash. Refer to Using the Sales Order Completion Screen for an explanation of the remaining fields on this screen.
3
Press F2 to save the Sales Order Completion screen, the Cash Application screen appears.
4
Enter a Pay Type of 03 for Bank Card.
5
Verify the Payment Amt and press ENTER to open the Additional Credit Card Information screen.
6
Depending on system settings and company procedures, choose one of the following to obtain credit card information:
•
If the order has a credit card Terms Code and default credit card information (refer to Setting up Your System for Credit Cards), the information defaults in the appropriate fields.
•
If no credit card information exists, enter the Credit Card Type (e.g., Visa, M/C), the Credit Card Number, and the Expiration Date.
•
If you have a credit card swipe machine and no credit card information exists, press F6-Swipe Card and swipe the card to fill in the information.
7
For CVV number, if required, enter the 3 or 4 digit code on the back of the card. This number is used by some companies to prevent theft. It is not stored in the system.
8
If you use the AVS (address verification service) enter the Address Line One and the Zip Code for the card user’s billing address. This information defaults from the customers Bill To Information Screen if the customer is already set up with credit card information (refer to Assigning Credit Card Information to a Customer). If you do not use address verification, leave these fields blank.
9
For Card ID#, enter the purchase card ID number, if applicable. This ID is for American Express or Visa cards to define the user of a corporate card. For example, a company with multiple stores may have one account but each location could have an additional ID number.
10
For Description, if applicable, enter a code that is passed on by credit card companies to Protobase. This information is used by American Express purchase cards and allows you to enter 4 lines of text stating what the purchase was for and to print on the statement. These fields are only used with Level II reporting.
11
Press F5-Authorize to send the authorization request. The Authorization Info screen appears.
12
If the authorization is successful, the approval code information appears. This code guarantees the credit card for 3 to 30 days (depending on the customer’s bank) and deducts the amount of the sales order from the cardholder’s available-to-buy. If the card is declined, a decline message appears (e.g., 0060DECLINE).
If your credit card verification system is down, you can manually enter an authorization code (refer to Manually Adding an Authorization Code).
If the message “Call” or “Referral” appears during pre-authorization, you must call for authorization by phone. Then you can enter the Authorization Code in the Additional Credit Card Information screen.
13
Press OK to return to the Additional Credit Card Information screen.
14
If the transaction was successful, the Authorization Code appears.
15
Press F2-Save to return to the Sales Order Completion screen and complete the order.
If Suppress CC Save Prompt in the customer Credit Card Information Screen (Customer Maintenance > F6- A/R > F7-Credit Card) is set to N, a message appears to ask if you want to update the customer’s credit card information.

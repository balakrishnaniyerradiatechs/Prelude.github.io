Price Override Reason Code Mnt screen
If you want to track price overrides on sales orders or quotes, you can create price override reason codes in Price Override Reason Code Maintenance. System settings allow you to determine who is allowed to override prices and if price override reason codes are required.
Option Parameters
Price Override Reason Prompt (Order Processing Parameters > F5-Add Info > F5-Add Info) determines if a price override reason code is required when a price on a sale order or quote is changed for a stock item:
•
Y - A price override reason code is required when a price is changed.
•
N - A price override reason code is not required when a price is changed.
User Identification Maintenance
Price Override Code (User Identification Maintenance > F6-Add Info) determines if the user can override a price without causing the order to go on administrative hold:
•
A - The user can change a price to any value, including below cost, without the order going on hold.
•
C - The user can change a price, but not below cost, without going on administrative hold.
•
G - The use can change a price, but if the gross profit percent falls below the minimum defined for the product line, the order will go on administrative hold.
•
N - The user cannot access any price fields.
•
R - The user can raise the price above the customer’s default price. If the price is lowered below the default price, the order will go on administrative hold.
Menu Path: Order Processing > Maintenance > Misc Code Files > Price Override Reason Codes
Field and Function Key Descriptions
Field/Function Key
Description
Price Override Reason Code
A 2 digit code to use on sales orders and quotes to identify a price override reason.

Acct Code Pattern Match Screen
The Acct Code Pattern Match screen allows you to assign accounting code patterns that must match each line item on the customer’s sales order (Sales Order Entry > Line Item Detail screen > F5-Add Info > Customer Acct Code). A pattern can contain letters, numbers, and dashes. You can enter multiple patterns, but only one pattern per line. If an Accounting Code exists in Customer Part Number Maintenance (refer to Setting Up Customer Product Numbers), it overrides this pattern.
Menu Path: Order Processing > Maintenance > Customer Maintenance > F7-Order > Require Acct Code = Y
Field and Function Key Descriptions
Field/Function Key
Description
Accounting Code Patterns
The alpha, numeric, dashes, or strings to include in required accounting codes for this customer. For example:
•
4A - The accounting code must be 4 alpha characters (e.g., PRNG).
•
4N - The accounting code must be 4 numeric characters (e.g., 4934).
•
8X - The accounting code must be 8 characters that could be either alpha or numeric (e.g., 4BC23999).
•
3A4N - The accounting code must have 3 letters followed by 4 numbers (e.g., ACT3398).
•
3A-4N - The accounting code must have 3 letters followed by a dash and then 4 numbers (e.g, ACT-3398).
•
3A.6N - The accounting code must have 3 letters followed by a decimal and then 6 numbers (e.g., ACT.339821).
•
“A”4N - The accounting code must be the letter “A” followed by 4 numbers (A3499).
•
0A - The accounting code can be any number of alpha characters.
•
0N - The accounting code can be any number of numeric characters.
•
0X - The accounting code can be any number of both alpha and numeric characters.
F2-Save
Saves the accounting code pattern(s) for this customer.
F4-Del
Deletes the accounting code pattern(s) for this customer.

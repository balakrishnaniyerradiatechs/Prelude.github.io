Customer Blanket P/O Numbers Screen
The Customer Blanket P/O Numbers screen allows you to set up and track blanket purchase order information for a customer (refer to Blanket Contracts). You can also maintain multiple purchase orders and determine if a purchase order pattern must be matched for orders for this customer. This screen is also used for Administrative Invoice Hold and Administrative Order Hold options. Refer to Administrative Hold Inquiries.
Menu Path: Order Processing > Maintenance > Customer Maintenance > F7-Order > Require P/O Number = Y, 0, or I.
Field and Function Key Descriptions
Field/Function Key
Description
P/O Customer
The customer number to use for blanket purchase orders, if applicable. You can set up a blanket purchase order for one customer to maintain blanket purchase order information for multiple customers. When the Customer P/O# is entered on an order, the blanket purchase order information applies. Leave this field blank if no other customers are associated with this customer’s blanket purchase order.
P/O Required Amt
If you enter a value, sales orders that exceed this amount require a purchase order.
Admin Invoice Amt
Only accessible if you entered I for Require P/O Number. Enter the Administrative Invoice Hold limit:
•
Sales orders are placed on Administrative Invoice Hold if the order is below this minimum and more than the P/O Required Amt. Refer to the Administrative Invoice Inquiry.
•
Sales orders are placed on Administrative Order Hold if they exceed this amount. Refer to Administrative Order Inquiry.
Beginning PO#
The beginning PO number in a range of purchase order numbers if you want the system to assign a purchase order number in the Sales Order Header or the Quote Order Header screens. The system will assign the next available number only if you enter NEW. You cannot use this feature if you are using PO patterns.
Ending PO#
The ending PO number in a range of purchase order numbers if you want the system to assign a purchase order number in the Sales Order Header or the Quote Order Header screens. The system will assign the next available number only if you enter NEW.
Current PO#
The current PO number which is updated when a sales order or quote is entered (if you established a Beginning PO# and an Ending PO#) . If you edit this number, it must exist within the PO# range established in the previous fields.
Customer P/O#
The customer’s blanket purchase order number. Refer to Creating a Blanket Contract for more information.
Start Dt
The date the customer’s blanket purchase order takes effect.
Exp date
The date the blanket purchase order expires. It can still be used after this date, but you are notified that it is expired.
P/O Amount
The amount allowed on this blanket purchase order. If sales exceed this amount, a warning message appears on the subsequent order.
P/O Balance
The balance remaining on the blanket purchase order, which is the P/O Amount less the value of any orders generated using this blanket purchase order number for this customer. Refer to Tracking Blanket P/O Information for a Customer.
P/O Desc
A description to reference this blanket purchase order number.
F2-Save
Saves the blanket purchase order information.
F5-Po Patterns
Opens thePO Pattern Match Screen. You can establish purchase order patterns that must match the pattern in the Customer P/O# on any sales order or quote for this customer. This feature cannot be used if you entered a beginning and ending PO# range.

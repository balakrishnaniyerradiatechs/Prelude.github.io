Formatting Price Labels
To design the format of a price label:
1
Select Order Processing > Maintenance > Pricing Structures > Price Label Format Maintenance to open the Price Label Format Maintenance screen.
2
For Format Code, enter a 1 - 6 digit alphanumeric code to represent the label design.
3
For Printer Type, choose from the following to print the labels:
•
Z - SPL language code (zebra printer).
•
E - EPL Language code.
•
O - Other.
4
For Stickers Across, enter the number of labels to print on a row (the number of labels across the form).
5
For Sticker Spaces, enter the number of spaces between each label in the row.
6
For Sticker Width, determine the maximum number of characters to print across on a single label.
7
For Sticker Depth, enter the number of lines to print on a single label. One line is added between each label.
8
For Type, indicate what data is to print on the label. Choose from the following:
•
C - Customer.
•
S - Customer ship-to.
•
CPN - Customer part number.
•
P - Product header.
•
W - Product warehouse.
•
PV - Product vendor.
•
V - Vendor.
•
O - Order.
•
OL - Order line.
•
D - Date.
•
L - Literal.
9
For Attr, enter the position in the raw data file (e.g., from /FLI) of the data to print.
10
For Mult Value, define the position where the data is extracted from if the Attr is multi-valued. If the Attr is not multi-valued, enter zero.
11
Enter the Line number on the label where the data will print. The fields to print must be in ascending order (e.g., 1, 2, 3, 4, 5).
12
For Print Pos, define the starting position for the data on the label (normally 0 or 1). To indent a field by 2 positions, enter 3.
13
For Length, define the number of characters to print on a line, which cannot exceed the Sticker Width.
14
For Start Pos, define the beginning digit to print on the label. Normally 1 is the first character. But, if you use product prefixes and you do not want to print a 3 digit prefix, enter 4 here to start with the 4th character of the part number.
15
If the data field is a price, use In Dec to define the number of decimal places to store in its raw data record.
For example, a price of 55.385 is stored internally as 55385. If you enter 3 in In Dec, then the price is stored as 55.385. If you enter 2 in In Dec, the price is stored as 553.85.
16
For Out Dec, indicate the number of decimal spaces to use to print the price. For example, if the internal price is 55385 (internal decimal of 3) and you set Out Dec to 3, then 55.385 will print. If you set Out Dec to 2, then 55.39 will print.
17
For Just, indicate how to justify the printed data on the label. Choose from the following:
•
R - Right justified.
•
L - Left justified.
•
D - Date (mm/dd/yy).
18
For Supp Lead Zero, indicate if leading zeros are suppressed:
•
Y - Suppress leading zeros (e.g., customer 000001 prints as 1).
•
N - Do not suppress leading zeros.
19
For Code Num Fld, indicate if you want to encrypt the price label data by assigning a character to each number from 0 - 9. Choose from the following:
•
Y - You want to encrypt the data to print on a label:
-
Select F6-Coded Numeric Field Definition to open the Coded Numeric Field screen.
-
Enter a character to print for each number. For example, if you enter D for 3, F for 5, and K for 9, then 55.39 prints as FFDK.
•
N - You do not want to encrypt the printed data.
•
B - Allows you to enter barcode type and dimension information.
20
Press F2-Save to save the price label format information.
Shelf Stickers
If you want to print shelf labels for your warehouse, select F5-Shelf Sticker. Enter the information in this screen in the same way as you would enter the information in the Price Label Format Maintenance screen.

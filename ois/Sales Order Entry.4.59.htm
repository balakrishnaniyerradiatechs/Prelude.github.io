Billing Consignment Orders
You can invoice the order using the normal invoice process (refer to Invoicing) to turn the order into an 08 order type, but the customer is not billed until the material is actually sold.
The first time an order is invoiced, the following occurs:
1
The quantities are moved from on-hand inventory to consigned inventory.
2
The Inventory G/L account is credited and the Inventory Consigned to Customer G/L account is debited for the total cost of the shipment.
3
A consignment invoice is produced for zero dollars, showing the items on consignment. A/R records are created for freight and handling but not for inventory.
4
A consigned order history record is created for the type 07 order and a type 08-Confirmed Consignment order is generated with a new order number. The new order number can be found in Inventory Inquiry, Order Log Inquiry, or on the consignment invoice. The 07 order can be found in Consignment Order History Inquiry.
5
The new 08 order is used to invoice the customer for the used consigned material or to return the unused consigned material back to stock when the 08 type order is invoiced.
6
If the material is used, the order is invoiced. If the material is being returned to stock, the material goes back into on-hand inventory.
Once you decide to bill the customer for the products, you can use one of the following methods for invoicing:
•
Automatically Billing Consignment Orders.
•
Manually Billing Consignment Orders.

Core (Issues) Sold on Exchange Report
The Core (Issues) Sold on Exchange report displays core products sold as an exchange product.
For a pending core to appear on this report, Auto Core Charge? (Order Processing Parameters > F5-Add Info > F5-Add Info) must be set to N. This option allows you to determine if you want to bill the core charge during invoicing.
A core product is the re-usable part of a product that a customer returns either for credit or for a discount against a new item. If you set up a product as a Core Product (Product Maintenance > F5-Add Info > F5-Add Info), you should also set up a product to exchange for the core product when it is returned.
A pending core is created when you sell an exchange product to a customer but do not charge them for the core (reusable) part of the item. The core charge is in a pending status until it is billed to the customer or a core bank item is applied to it. A pending core is an open receivable.
For more information on cores, refer to “Tracking Pending Cores” in the Inventory documentation.
Menu Path: Order Processing > Reports > Miscellaneous Reports > Core Reports1 > Cores (Issues) Sold on Exchange Report
Selection Criteria
Use the following criteria to limit the information to print on the report:
•
Warehouse. The warehouse that shipped the core exchange product.
•
Customer. The customer who purchased the exchange product.
•
Invoice Date Range. The beginning and ending invoice date range to include.

Column Heading
Description

The customer numbers.

The customer description.

The customer’s phone number.

The invoice the exchange product was billed on.

The sales order for the core exchange product.
Ship Date
The date the exchange product shipped to the customer.
Exchange#/
Description
Line 1: The exchanged part number.
Line 2: The description of the exchanged part number.
Core#/
Description
Line 1: The core part number.
Line 2: The description of the core part number.
Shp Qty
The quantity of the core part number that shipped.
Core Chg
The core charge applied to the shipped quantity.
Qty Appl
The number of core products applied to the pending core.
Last Applied
The date the core was last applied to the pending core.
Qty Due
The number of cores that still need to be returned or billed.
Extended Charges
The total shipped core charges (Shp Qty x Core Chg).

Transfer Entry
•
A new function key was added to the Product Tagging screen in Transfer Entry. F7- Resort enables the user to modify the sort option that was established as the default. The sort options available are P for product number or D for product description. For example, if the Product Tagging Default Sort flag in Default Parameters Maintenance is set to P in which the products will be displayed in ascending order by product number, and the user prefers the products to be displayed in ascending order by product description, this function key can be used to change the product sort in the tagging screen.
•
A new F8- Switch Entry Mode function key was added to the Product Tagging screen in Transfer Entry. This option allows the user to modify the position of the cursor from product number to quantity or vice versa when tagging products. A default of P for product number or Q for quantity is established in the Product Tagging Entry Mode field in Default Parameters Maintenance. For example, if the default is set to P, the cursor will be positioned at the product number, and the user prefers the cursor to be positioned at the quantity field, this function key can be used to change the cursor positioning in the tagging screen.
•
A new feature was added to Version 16 to allow inventory to be reserved for a customer. Therefore resulting in some modifications to Transfer Entry. The available quantity field in the line item detail screen will include reserved inventory. In addition, there is a new RSVD ( ) field in this screen to notify the user of the quantity reserved. However, if the quantity entered for this product produces an inventory level, below the reserved inventory level for a customer, the Insufficient Stock Requested Reserved Inventory! Placing Line Item on Hold! warning message will then be displayed, placing the sales order on inter-branch hold for review.
•
The Transfer Entry totals box was updated with a new Cube and Weight feature. Once a line item is entered, and processed through the line item detail screen, the totals box will display the cube and weight information for this item. If multiple items are entered on a transfer, the cube and weight information for each item will be accumulated as products are added and processed. The Cube and Weight information for products are established in Product Maintenance> F10> Addl Options> Package Information.

Product Cycle Count Request Entry
The Product Cycle Count Request Entry screen was modified for Version 16. Previously, this screen would prompt for a Warehouse Number and Product Number. The new version contains a Request Type field. This new field allows a type to be entered to cycle count products. The valid Request Types are L for Product Line, P for Product Number and V for Vendor Number. Once the request type and warehouse number were entered and the F2-Save function key was used, a screen will be displayed based on the request type selected. This screen prompts the user to enter either the Product Line(s), Product Number(s) or Vendor Number(s) that are to be cycle counted. Regardless of which request type was chosen, all the products that pertain to that type will be updated to the Cycle Count 2nd Level Entry screen.

The Current Value-Reserved report was added to Version 16 to provide a current valuation of reserved inventory products. Products MUST be established as Reserved Inventory products in Product Warehouse Maintenance to be included on this report.
Miscellaneous Features
Vendor Consigned Inventory
A modification was made to Work Order Issues Entry if Vendor Consigned Inventory is being used. The components entered on the work order in Work Order Issues Entry are now updated to the CONSIGNED (Attribute 6) and CONSIGNED.SOLD (Attribute 10) files.
Program Changes
The Stock Receipts Entry, Inventory Adjustments Entry and the Credit Memo Entry screens were affected by a program change that was put in place for both Version 15 and 16. This change was made to disallow a previously entered warranty tag number for a product to be duplicated.

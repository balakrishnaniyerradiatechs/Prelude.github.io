Accounts Payable
Vendor Invoice/Claim Reconciliation
You can split receivers/claims by partial line item quantities. Previously, the split receiver option would only accommodate splitting receivers/claims based on the total quantity of the line itemin Qty-Inc on the Vendor Invoice Split Routine screen. However, to use this process correctly, criteria must be met.
•
The quantity must be equal to or less than the quantity received of the line item.
•
Serialized items- enter serials numbers for the split quantities. Delete the serial numbers that do not apply.
•
Lot items cannot be split by quantities; only the total quantity will be accepted.
•
Freight Amount- split according to total freight % for all products on the receiver. If the Freight Matrix is established, freight will be recalculated based on the matrix.
•
Duty / Brokerage Amounts-split according to the total duty/brokerage % for merchandise amount of each receiver.
•
If the Multiple P/O Sourcing flag is set to Y in Purchasing Parameters and a line item has multiple linkages, the line is NOT allowed to be split by quantities- only the total quantity will be accepted.
Recurring Vendor Invoice Posting
A New Invoice # field was added to the Recurring Vendor Invoice Posting process. This option allows you to enter a new invoice number for this recurring invoice. Leave Invoice # blank to use the same invoice number.

Vendor Discount Code Maintenance
This new maintenance allows you to create On and Off vendor discount codes. These discount codes allow discount amounts and G/L accounts to offset the discounts. An On discount is a negotiated discount from the vendor that is reflected in the invoice amount. The discount is refelcted in Miscellaneous Amountwith Vendor Invoice Reconciliation Entry. Multiple G/L accounts can be offset for the discount. An Off discount is a negotiated discount from the vendor that is not reflected in the invoice amount. The vendor receives two invoices (a credit invoice for the discount in addition to the original invoice). Once valid vendor discount codes were established, they are then used in the Vendor Maintenance screen as well as the Purchase Order Entry Header to apply the pre-determined amount.

Reports> Monthly A/P Reports> Lost A/P Discounts Report
A new Lost A/P Discounts Report was added to the Monthly A/P Reports menu. This report was created to provide information pertaining to lost discounts due to the issuance of A/P checks after the discount date. This report displays Vendor #, Vendor Description, Invoice Number, Discount Date, Check Date, Check Number, and Discount Amounts regarding lost discounts.
Program Changes
Accounts Payable> Check Processing> Payment Selection/Print
A program change was made for V15-17 regarding the Payment Selection/Print process. In V14 and earlier the Payment Selection/Print option (APPS) did not include the GAIN/LOSS account in the G/L postings for the currency exchange rate if it had changed. Manual Check Entry was required to be performed to settle the currency exchange rate since this was not automatically performed during the APPS process. Since this modification was made, the Payment Selection/Print option will now include the GAIN/LOSS account in the G/L postings for the currency exchange rate if changed. Manual Check Entry is no longer required to be performed since the currency rate exchange settlement is performed as part of the APPS process.

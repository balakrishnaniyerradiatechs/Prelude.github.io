V18 New Features
Version 18 Warning
Version 18 has some significant enhancements, but we wanted to give you information about areas that are new or were significantly changed from prior versions. Please study this document and test and/or train around these items if they apply to your company.
Indexes (B-Tree)
Indexes are operating system-level cross-references that increase speed in selecting large files and improve report execution time. Previously, Prelude had indexes for over 20 files that improved speed for selected and strategic reports. For example, in the Daily/Monthly Invoice Registers, the architecture of those indexes came from the Pick D3 heritage and was not used as widely as is currently possible using UniData’s architecture capabilities. In addition, those indexes were not readily available for you to use in your reports.
Indexes require additional disk space to store cross-references. Therefore, we will delete all indexes during the upgrade process that are in the primary files, which we used in the prior versions. We designed a new tool to help generate the new indexes and an area that helps you easily create indexes. You must determine if the indexes that we will delete are important and add those indexes back to the user area of the New Index Routine. We can help you find the files that have indexes and rebuild those indexes as they currently exist. We can also help you rewrite the routines that use the older index syntax to use the new model to not waste disk space.
Work Queues Inquiries
We significantly changed the logic that updates some of the work queue inquiries. The update routines were significantly affected. Therefore, you must review your hold logic and process to verify that the queues works as you expect for your business. The primary changes are:
•
Instant notification to the managers. Each manager (user) determines how they wish to be notified that an order is on hold. The current work queues are still in place and were not changed. The manager can also be notified via email immediately and take advantage of the instant messaging capability in their email client software. We have another option that we will provide more information about in the near future.
•
Unlimited notes between the operator and manager for transactions that are on hold.
•
Consistent presentation and functionality between the work queue inquiries.
These changes affect the following work queues:
•
Credit Hold Inquiry.
•
Administrative Order/InvoiceQuote/ Hold Inquiry.
•
Purchase Hold Inquiry. The purchase hold inquiry was also changed from only a single buyer design to use manager codes like the other inquiries.
Email is available for all work queues except branch manager and emergency cycle count.
Import Costs
Many years ago, we partnered with a customer and designed/implemented the concept of importing costs with options like Inland Freight, Insurance, Brokerage, and other defined fees with set methods of calculating those import costs. These defined fees became restrictive. Therefore, we redesigned import costs to be more flexible and handle the dynamic requirements for importing. The new design provides for unlimited importing fees. You can define the fees you wish to calculate as well as the rates, allocation method, and currency for the rates.
Note: If you use the previous importing technique and want to use this new design, you must set it up, train your users, and convert data. Please contact us once V18 is loaded on your TEST account, but before you upgrade, so we can discuss a conversion plan.
New Financial Report Generator
To accommodate some new requirements and suggestions from customers, we created a new financial report generator that allows you to more easily generate reports, view your output on the screen, and drill down from the financial numbers to research where the amounts came from. The current financial report generator is still available in version 18 and was not modified.
If you want to use this new report tool, you must prepare your accounts and convert your existing reports to this new format. The primary feature we did not bring over to this new report functionality is the wildcard concept on the actual account number. However, we implemented a group concept to bundle those accounts for easy access on the reporting tool. Please take a look at it and you can decide if you want to use this new tool.
RF (Radio Frequency) Applications
Previously, we started all function key options on RF application screens using F5 and then possibly F6 for additional options. We have come across some challenges with configuring the F5 key on some guns – especially the new guns. Therefore, we are starting to use F6 and beyond for executing additional options. If an RF gun had an option on F5, it was moved to F6 or F7. Therefore, please notify your warehouse personnel that use RF guns that the function key options may have moved to another function key spot.
Documentation
We have begun a new generation of online documentation. We still have the historical field sensitive (F1) help, but we have added a new Help option to the Action Bar that launches our new screen-level online documentation in a browser. This documentation provides more expanded help for every field and function key on the screen and links to supporting or related documentation. Because this documentation is new, we could not completely document the entire application for this release. To allow us to easily add and update the documentation, we added the Help option to all screens in the application even though the documentation is not complete. Therefore, for Version 18 at this time, the Help option on the screens is only activated in Purchasing, Forecasting, and Shop/Repair. We are working on Accounts Payable, General Ledger and Inventory. As we complete modules, we can send them to you and activate them under your current version 18 release.

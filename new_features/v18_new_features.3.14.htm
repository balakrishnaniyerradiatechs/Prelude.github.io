New Counter Range Rollover w/Alphas Flag
This new Rollover w/Alphas flag determines if the counter range is rolled over to the beginning of the counter with an alpha prefix. This flag is only used if you are using prefixes to define your document number scheme. For example, a prefix of 10 for warehouse 10 can be used for a counter to update as 10xxxxxx. When the cycle counter rolls over to 999999, it will roll back over to the beginning number and assign document numbers that were used previously. If this flag is set to Y, the system will insert alphas and begin a new counter to retain unique numbers. For Ex., document 10999999 would roll over to 10A00001 versus 10000001 (which was used when originally setup).

New Auto Shp Whse Correction Flag
The Inventory Parameters screen was updated with a new Auto Shp Whse Correction flag. This flag determines if an over shipment received in Transfer Receipts Entry should prompt the system to generate an inventory adjustment. This process will update the shipping warehouse on the stock transfer to reflect the inventory count. If this flag is set to Y and an over shipment is received, the system will automatically create an inventory adjustment at the from warehouse behind the scenes. This adjustment will be performed for the shipping warehouse on the stock transfer, therefore reducing the inventory count. The bin associated with the inventory adjustment will be updated as TRNERR. If this flag is set to N, the system will not automatically generate an inventory adjustment, therefore requiring the user to manually create an inventory adjustment for any over shipments to correct the inventory count for the shipping warehouse. with this new process, a new Default Reason Code has also been added to this screen.
New Default Reason Code Field
with the new Auto Shp Whse Correction process, a new Default Reason Code field has also been added. If using the Auto Shp Whse Correction feature, setting the Default Reason Code field will be mandatory. The reason code entered, that is to be used for the automatically generated inventory adjustments, will be displayed on the Reason/Description line of the Inventory Adjustment. The valid Default Reason Codes are in the Inventory Adjustment Reason Code maintenance screen. (Inventory> Maintenance> Misc Code Files> Inventory Adjustment Reason Codes.)
New Auto W/O Deletion From SOE Flag
A new Auto W/O Deletion From SOE flag was added to the BMW/WO Parameters screen. This option was added to allow linked work orders to be automatically deleted if the sales order line linked to the work order is deleted (using the backslash command for line deletion or the F4-Del function key). If this flag is set to Y, the system will automatically delete the linked work order if the sales order line that is linked is deleted. The deletion will then be processed using the normal work order deletion process. However, for this deletion to be processed, the work order must have a status of ENTERED (unreleased). If the user attempts to delete the sales order line linked to a work order that was previously been released for production and currently has a status of RELEASED, the deletion will be denied. The Deletion Denied, Linked Work Order was Released For Production! warning message will be displayed. Therefore, the linked work order must be deleted in Work Order Entry. An invalid work order deletion will prevent the deletion of the sales order line.
Note: To ensure that this feature works correctly, set Auto Release to N, preventing the work order from automatically being released for production when created from in Sales Order Entry. If this flag is set to N, the automatic deletion of the linked work order when deleting the linked sales order line will not take place.
New Allow WO Over To Linked Xfer Flag
The BMW/Work Orders Parameters screen was updated with a new Allow WO Over To Linked Xfer flag. This is an Y/N flag that will determine if overages received in Work Order Completion will affect the linked stock transfers. If this flag is set to Y and the line item is linked, the system will automatically allocate the overage to the linked stock transfer. Therefore, the line item quantity of the stock transfer will be increased by the overage quantity. If this flag is set to N and the line item is linked, the system will not allocate the overage to the stock transfer as a result, not changing the quantity of the line item. The overage quantity is updated to regular stock.
Note: This flag only pertains to linked items.
New #/Months Classification Field
A new #/Months Classification field was added to the Purchasing Parameters screen. This new feature allows a number of months between 1 and 12 to be entered to determine how many months a product needs to be in the system before product classification will start. If this field is left blank, the system will default to 12 months. For example, if 6 is entered, the system will only require products to have six months of usage to be included in product classification. Previously the system required 12 months of usage. with this new feature, the ICS End of Month Calendar Close, and the Rebuild Product Classifications processes (Referenced in the Inventory New Features list) are also updated to check the number entered in the new #/Months Classification field.
New Administrative Order Hold Type

The E type will place orders on administrative order hold if the sales order has exceeded the blanket purchase order amount for the customer p/o on the order. to reference that this order was placed on hold for this reason, an E will be displayed at the header level in Administrative Order Release Inquiry. with this new AOH type, a P/O Customer # was added to Customer Maintenance and an update in functionality in Customer P/O # in Sales Order Entry.

Emailing
Updated Email Document Maintenance Functionality
A program change was made to the Email Document Maintenance options. If there is a format maintenance record with no text setups for a document type (OACK, OSHP), the program cannot determine what text record to use. The program will fail and no text email will be sent. At least one text record entry should be setup for each document type used. In format maintenance, the first text code setup in the list, for a document type, will be used as the default. If the program cannot find the default item defined in format maintenance, it will attempt to find a combination of a language code/text code record item (even though it may not be defined in Email Document Format Maintenance). If an exact match for the text code and language code combination is not found, the default will be used. For clients with a majority of English speaking customers, we recommend using the blank language code for the default. For shipping notifications, the program will first look for a format record for the shipping warehouse in the release since the selling and shipping warehouse may be different. For acknowledgement notifications, the selling warehouse will be used. If no warehouse setup exists, the system will try to find the default company format record. If neither the warehouse or company format record is found, the program will fail and no text email notification will be sent.

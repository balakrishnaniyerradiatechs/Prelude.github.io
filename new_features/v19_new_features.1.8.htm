Order Processing
Administrative Holds
New Admin Order and Quote Hold Types
•
P - Price Violation in Order Processing Parameters. Put an order on administrative order or quote hold if a price is lowered below cost, a gross profit percent, or a default price.You can now choose whether the order should go on admin order or admin invoice hold instead of always going on both types of hold.
WARNING! If you do not set parameters for the P option, none of the pricing violations (minimum price, high gross profit, or low gross profit) will put the order on hold.
•
V - Violated Mgr High Level GP% in Order Processing Parameters. Occurs when the GP% is exceeded for a product on an order or quote. Must also use the P option.
•
L - Always Quote Hold. Enter Q in Admin Hold Override in Customer Maintenance to place all quotes for the customer on admin quote hold. Administrative Quote Release Inquiry displays the L reason code for all quotes for this customer.
Admin and Credit Hold for Suspended Orders
The new Suspend Order From Activities flag in Order Processing Parameters determines if a sales order goes on administrative hold and credit hold if the order is suspended.
Un-Allocate Inventory for Orders on Admin Order Hold
The new Hold/Unallocate Inv flag in Order Processing Parameters allows you to un-allocate inventory committed to a sales order if the order is on admin order hold.
Admin Invoice Hold for Pre-authorized %
In the Exceed Preauth% field in Order Processing Parameters, enter a percent that the total invoice must not exceed or it will go on Administrative Invoice Hold. The Administrative Invoice Inquiry displays an X for this hold option.
Message Option for Orders on Hold
When an order goes on admin order hold or admin quote hold, you can enter a message that appears in the Administrative Order Release Inquiry or the Administrative Quote Release Inquiry.
Sort Option Added to Administrative Releases
A new sort option on the Action Bar allows you to sort orders by customer, ship-to, sales rep, ship via, order type, user ID, order header reason code, or by order line reason code in Administrative Order Release Inquiry and Administrative Quote Release Inquiry.
Credit Holds
Credit Hold for Zero Value Sales Orders
Sales orders with zero dollars now go on credit hold if the order has at least one line item. The system allocates any inventory to backordered line items so you don’t have to run the full inventory allocation process when the sales order is released from credit hold.
Credit Reason Codes for Orders Always on Hold
When you enter a Credit Code of H (always hold orders) in Customer Maintenance > F6, you can enter the credit reason code from Valid Response Maintenance.
Credit Hold for Floor Plan Customer
When you enter a sales order for a floor plan customer (the customer responsible for payment), the floor plan customer’s credit manager ID is checked to determine if the order should go on hold. Previously, the credit manager ID for the original customer was checked.
Pricing
Reprice Line Items by a Percentage
The new Increase/Decrease Percentage option on sales orders and quotes allows you to re-price a line item based on a percentage. To use this new feature, go to:
•
Sales Order Entry > Action Bar > Order Func > Reprice Order Options > Increase/Decrease Percentage.
•
Quotation Order Entry, go to Action Bar > Quote Func > Reprice Quote Options > Increase/Decrease Percentage.
Flexible Contract Dates in Contract Price Maintenance
You can now change the start and expiration dates for a contract price in Contract Price Maintenance instead of Price Matrix Maintenance.
Promotional Price for Linked Blanket P/Os
The new Blanket Contract field in Promotion Type Matrix Maintenance allows you to enter a blanket contract number from Blanket Contract Maintenance, which must match the customer’s P/O number to receive the promotional pricing. When you sell a product with a promotion matrix for a blanket contract, the Customer Promotion Matrix Inquiry lists the available promotions.
Assign a Vendor for Price Matrix Purchasing Cost
The new Vendor Number field (Price Matrix Maintenance > F5-Purchase Cost) allows you to assign a vendor to use for the purchasing cost for the price matrix. When you sell a product from this vendor using the B supply code in the line item detail screen, the purchase cost and assigned vendor automatically appear when you generate the purchase order.
Check for Best Price on Customer’s Contracts
The price matrix logic now checks for the best price of all contract matrices if Best Price is Y. Previously, the first contract price matrix was automatically used.
Copy Prices Matrices
The new Price Matrix Mass Update option on the Price Matrix Maintenance menu allows you to copy prices matrices from one product to another.
New Price Matrix Types
The following additional price matrix types are now available in Price Matrix Maintenance:
•
28 - Customer/vendor/product line
•
35 - Product line
•
36 - Major group
•
55 - Price group/rental product
New Sales Tax Matrix Types
The following additional sales matrix types are now available in Sales Tax Matrix Maintenance:
•
5 - Customer ship-to/product
•
6 - Customer ship-to/tax group
New Promotion Type Matrix
The new Matrix Type 2 (order) Promotion Type Matrix Maintenance allows you to enter a discount percent for each buy amount. The One Time Promo flag determines if the promotion matrix is a one time promotion for each customer.
New Rebate Method in Price Matrix Maintenance
The new rebate method P (rebate percent - price basis) calculates the rebate amount as the difference between actual cost and rebate cost. This method applies when the price fields used as the basis for the rebate are higher than the actual cost. If the system finds a rebated cost that is higher than the actual cost, a rebate is not calculated for the line item.
New Override Price Matrix Option
The new H option for Ov (override) in Price Matrix Maintenance allows you to increase a price, but not to decrease it lower, than the original gross price.
Create a Price Matrix From a Quote or Sales Order
The new Quote/Order Created Price Matrix process (Maintenance > Pricing Structures > Price Matrix Maintenance) allows you to create a price matrix based on a quote or sales order. Updates are recorded in the Order Log Inquiry (PMM code) and the Pricing Audit Reports.
Using Price Matrices with Child and Base Products
Prices matrices apply to base, not child products (Product Maintenance > Special Operations = CB or CC). When you order a child product the price matrix for the base product is used.
New Blanket Contract Method for Additional Discounts
When you enter a blanket contract for a customer in Blanket Contract Maintenance, the new A option under Mtd (method) allows you to enter an additional discount percent to apply to the line item on the blanket contract. This value overrides the Add Discount % field in Customer Maintenance > F7 > F5.
Adding Customers to a Master Customer Blanket Contract
If you enter a master customer number in Blanket Contract Maintenance, now only child customers for this master customer appear when you lookup a customer to add to the contract (F5-Hdr Scr - F5-Add Cust). If more than one child customer appears, you can use the space bar to tag multiple customers to add to the contract.
New Flag Determines if Backorders are Accepted
The new Accept Backorder flag in the Blanket Contract Maintenance header screen allows you to determine if backorders are accepted for items on the blanket contract for the customer.
Kits/Components
Pick Quantities Not Recalculated for Kit Components
Pick quantities for kit components on confirmation sales orders are no longer recalculated. As a result, you no longer have to manually update the kit components once the backordered quantities are allocated. Previously, ship, pick, and backorder quantities were recalculated based on the kit parent available when the kit parent was entered.
Rollup Component Prices for Kit Parents
The new Precost Kit From Rollup flag in Order Processing Parameters (F5- Add Info > F5- Add Info) determines if a kit parent item defaults to the cost from the rollup of the components.
Shipping/Selling Warehouse
Allocate Inventory from Available Warehouses
The new Shipping Warehouse Split Reassignment flag in Order Processing Parameters (> F5-Add Info > F5-Add Info) determines how inventory allocates to backorders:
•
Y - Prompts the salesperson to allocate inventory from an available warehouse
•
A - Allocates automatically
•
N - Backorders rather than allocate from another warehouse
Allow/Disallow Selling Warehouse
The new Selling Allowed flag in Warehouse Maintenance > F5 determines if you can use a warehouse as the selling warehouse for quotes and sales orders. Once the flag is changed to N, you cannot use that warehouse as a selling warehouse on new orders.
Geocode Default Shipping and Selling Warehouses
The new Selling Warehouse and Shipping Warehouse fields in Geographic Code Maintenance allow you to enter shipping and selling warehouses to default for this geocode in Customer Maintenance or Customer Ship-To Maintenance.
Update Shipping Warehouse in Duplicate Orders
The New Warehouse Number? field allows you to change the shipping warehouse when you duplicate a sales order or quote (Sales or Quotation Order Entry > Action Bar).
Changing the Shipping Warehouse for an Auto Reversed Order
The Shipping Whse Assign field was added to the Auto Reverse (Sales Order Entry > Action Bar) screen. You can keep the original warehouse on the order or change it to the correct shipping warehouse. Every item on the voided order updates the assigned shipping warehouse.
D Ship-To Address Included in F3 Search for Ship-To Number
The F3 search for Ship-To Number now includes the D address from Customer Maintenance.
Consolidating Shipping
The new Shipping Whse Consolidation flag in Order Processing Parameters determines if product shipment is consolidated. The following options are available:
•
S - Manually consolidate shipping in the Shipping Consolidation screen in Sales Order Entry.
•
A - Automatically consolidate shipping. If at least 5 items can be filled from a warehouse, as many items as possible are shipped from that warehouse.
•
N - No consolidation.
Viewing Delivery Days for Consolidated Shipments
The value in the new Delivery Days field in Ship Via Maintenance > F6 displays in the Shipping Consolidation screen in Sales Order Entry if Shipping Whse Consolidation in Order Processing Parameters is S.
Sales Tax Jurisdiction Adjusted for Ship-To Number P
If you enter P (pick-up) for the Ship-To Number in the Sales Order Header screen, the tax jurisdiction from the shipping warehouse applies. If you change the Ship-To Number from P to another ship-to account, the tax jurisdiction changes accordingly.
Consolidate Releases for PSS Shipments
The new PSS Consolidation flag in Customer Maintenance > F7 > F7 allows you to consolidate releases into one shipment using the PSS Manifest Interface. You can update and hold the releases in Release Confirmation Entry > F7 and then accumulate them by customer, warehouse, and ship-to address.
Group Route Codes for Surcharges
If you use Route Processing, you can use the new Route Code Group Maintenance to group route codes to add surcharges. The group’s surcharge amount is added to the outbound freight for the first release of the customer/ship via/route.
Voided Sales Allowed in the Point of Sale Subsystem
The new Disallow V/S From Alt Whse flag in Point of Sale Options Maintenance determines if a sale can be voided if the Startup Maintenance selling warehouse does not match the selling warehouse on the order.
Quotes
New Frequency Options for Standing Quotes
Quote Type 04 (Standing Quote) now allows you to enter a Frequency of B (semi-monthly), Q (Quarterly), or S (semi-annually). Previously, you could only enter a frequency by number of days or by a day of the month.
Record Lost Sales for Deleted Quotes
If you track deleted sales orders (Keep Deleted Orders? in File Update Parameters must be set to Y), you can enter a lost sales reason code when you delete a quote. You can track this information on the Monthly Lost Sales and Monthly Lost Sales by Vendor Reports.
Reprinting Quotes
When you re-print a quote, the warning message now warns you that regardless of how you transmitted the quote the first time, it will automatically be sent to the printer. If you want to email or fax the quote, you must go back into Quote Order Entry to select the method of transmission.
Credit Cards
Masking Credit Card by User ID
The new View Credit Card# flag in User Identification Maintenance allows you to determine if an employee can see the first 12 digits of a customer’s credit card number in all of the order entry and inquiry programs. Previously, the first 12 digits were masked for all users.
Defaulting AVS Information for Credit Cards
The new Default AVS For Credit Card flag in Option Parameters determines if Address Verification System (AVS) information defaults from Customer Maintenance.
Manually Moving Credit Card Transactions to History
The new Credit Card Rejection Maintenance program (Order Processing > Processes > Credit Card Processes > Additional Batch Functions) allows you to manually move a rejected credit card transaction into history so that it does not appear on the rejection report. Post authorize the transactions after you move the rejected transactions.
Credit Card Markup Authorization
The new Add Markup To Pre-Auth Amt flag in Order Processing Parameters > F5 determines if the pre-authorization amount for the credit card is increased based on the merchandise amount and a percent in Credit Card Markup Table Maintenance in Order Processing Controls. You can add the Credit Card Markup Table number to a code in Ship Via Maintenance.
Miscellaneous
Validating Geocodes for Updated Addresses
If you use vertex to calculate sales tax, the city, state, zip, and county fields are now validated against the Vertex address information. If you change the address, the geocode is validated to make sure the address matches the zip code.
New Comment Codes
The following codes are not linked to the product on the previous line on a quote or sales order and always appear at the bottom of the quote, pick ticket, or invoice:
•
CS - removed after first invoice.
•
CSB - remain for backorder items and all releases.
Update Serial Number Information for Trade-Ins
The new Equipment Trade In flag in Credit Reason Codes determines if you can enter the make, model, price, and cost for the new serial number when you enter a credit memo for an equipment trade-in.
Exchange Option added When Billing a Core Charge
The new Exchange option automatically updates the core charge and the core credit on the order.
New Commission Matrix
The new type15 matrix in Commission Matrix Maintenance allows you to set up a commission matrix by warehouse/route/product line.
New Purge Programs

Customer Expired Credit Cards

Rebates

Build Point of Sale

Purchasing
Buyer Queue Inquiry
The screen layout in the Buyer Queue Inquiry was rearranged to have the work related queues on the left and other alerts and inquiries on the right. Some of the queues were renamed and/or modified, and three new work queues were added.
Renamed queues:
•
Purchase Order Conversion changed to: Recommended Purchase Orders.
•
Work Order Conversion changed to: Recommended Work Orders.
•
Linked Purchase Order Inquiry changed to: Linked P/O Changes.
Modified queues:
•
Recommended Purchase Orders:
- If you choose to edit the purchase order before releasing it, if you go into F7-Targets, the new F5-Evaluate Line key allows you to view products not included on the recommended purchase order because they currently have not reached replenishment levels. You can enter a quantity for each line item to add to the recommended purchase order.
•
Recommended Work Orders:
- The Emg flag in the Recommended Work Order Inquiry now notifies buyers of existing sales orders and transfer backordered quantities. If these types of orders are found in the file, a Y displays in the Emg column indicating that these orders exist and should be moved up in the production queue. A new Backorder inquiry feature on the Action Bar displays the total number of sales order and transfer backordered quantities for each buyer. This inquiry screen is updated once Nightly Processing is completed. If the Display Cost? flag in User ID Maintenance is set to N, the Cost/Ext will not display any dollar amounts due to the fact that this user is not permitted to view costs. Important Note: to use this new feature, it is required that the M Replenishment Method is in Purchasing Parameters.
•
Purchase Order Expedite :
- The F6- now defaults the From Phone & From Fax fields from the warehouse number. If multiple purchase orders exist for the vendor from different warehouses, the From Phone & From Fax fields use the warehouse number from the first purchase order expedite item.
•
Purchase Hold Release :
- When the purchase order is sent to the Purchase Hold Release buyer inquiry due to the p/o minimum or direct p/o minimum for the vendor not being met, the buyer can release the purchase order. Previously, the buyer had to correct the P/O amount or delete the purchase order.
- When an order goes on hold in Purchase Order Entry, the buyer now has an option to send notes to the Purchasing Manager. If the e-mail notification is set for the user id of the purchasing manager, an e-mail is sent to the purchasing manager referencing the purchase order number and user id of the person who entered the purchase order. Notes are included in the body of the e-mail.
- The Purchase Hold Release inquiry now works with the new notification method in User Id Maintenance as well as the new purchasing manager id. Several of the fields and Action options have moved or revised as a result of this update. In addition, a reason column was added for the inquiry. The new reason codes are J-P/O limit exceeded; V-vendor minimum not met-regular or direct; H-vendor always placed on hold (purchase hold flag=Y).
New queues
•
Recommended Stock Transfers: Includes stock transfers created by Daily/Nightly Processing. You can convert the orders into open transfers.
•
Vendor Claims: Includes Vendor claims created in Vendor Claim Entry, the DOE convert process or from rules set up in Vendor Maintenance. You can review and edit the claims.
•
Product Alerts: Includes products whose usage falls above maximum levels set for the product, by the percentage defined in the Spike Exception Min/Max settings in Purchase Parameters Maintenance. Also, products that fall outside of the new low % and high % safety allowance parameters in stock receipts.
Purchase Orders
Product Number Previously Entered - New Message
When entering line items, the Warning!! This Product Number was Previously Entered message now has a new Return To Product option. Previously, the only options available were Return to Order which returns you back to the order to allow you to enter a different part number and Continue which returns you to the Line Item Detail screen to enter the duplicate product. The new Return To Product option takes you to the first occurrence of the product number and removes the duplicate product number entered.
Purchasing Warehouse Sequence Feature Added
A new F5-Purchasing Seq feature added to the Warehouse Sequences Maintenance screen in Inventory Controls allows you to create a Purchasing List to customize the order in which warehouses, as well as their surplus inventory are displayed and checked. The Purchase Order Entry screen now checks the Warehouse Sequence Maintenance for an existing Purchasing List when determining surplus inventory. The Purchase Order Conversion also has the same modification. The Nightly Processing routine and the Recommended Purchase Order Action Report are both updated to check the Warehouse Sequence Maintenance for any existing Purchasing List prior to generating any temporary purchase orders.
Note: If the line item is linked on the temporary purchase order, the system uses the list of warehouses in the PROD.STAT record to determine the line item's surplus inventory. The new Purchasing List feature is used when Nightly Processing generates a temporary purchase order for a backorder (non-linked item).
Pay To Vendor – Override Field Added
Typically, the vendor currently set up in the Pay To Vendor field in the F6-A/P screen of Vendor Maintenance is used when paying vendor invoices. The new Ovr Pay to Vendor field in the Purchase Order Header > F5-Add Info screen, allows you to enter a different vendor number to override the default. If you use this feature, a message displays in VIRE to reference the override.
Merge With Suspended P/O - Combines Duplicate Quantities
Previously, the Merge w/Suspended P/O process (Action Bar > Prod Func > Merge w/Suspended P/O) created an additional line for any duplicated product on the purchase order. It no longer adds another line, but combines the quantities of the duplicate products on the existing line. Also, the merge option now includes product notes from the temporary purchase order when merged into a suspended purchase order.
Vendor Part Number (VRFQ Entry) – Updated With VPN#
The Vendor Part Number field (Vendor RFQ Entry > VRFQ Detail Information screen > F6-Addit Info) is now automatically loaded with the vendor part number for each vendor entered on the VRFQ Header Information screen. If additional vendors are added in the header screen, the additional vendor part numbers are updated to the Additional Information screen for the existing product number. You can override the default Vendor Part Number. The vendor part number prints on the vendor request for quote user-form.
Purchase Requisition Entry – Header Screen Modified.
The Purchase Requisition Entry header screen, including F5-Add Info screen, are now identical to the Purchase Order Entry header screen.
Frozen Controls – New Field
The Recommended P/O Inquiry in the Purchase Order Conversion Entry screen now displays a Frozen field. If the Frozen Controls flag in the Purchasing Parameters Maintenance screen is set to Y, an * (asterisk) displays next to the Frozen field in the inquiry. If a Thaw Date exists, it appears to the right of the * (asterisk).
Purchase Order Log Inquiry - Codes Added/Modified
The following features are new to the Purchase Order Log Inquiry:
•
The Req Date field now only applies to direct purchase orders. Previously, this referred to 01 purchase orders.
•
The PLS code was added to reference a line item spit in Purchase Order Entry. This was previously combined with the SPL code.
•
The SPL code now just displays the receiver record.
•
The MRG code just displays the P/O history inquiry and not the receiver.
Blanket P/Os
Blanket Purchase Order Entry – New Header Information Fields
The header screen in Blanket Purchase Order Entry has the following new features:
•
The new Ship From field allows you to choose a Ship-From vendor in Vendor Maintenance.
•
The new F6-Ship-From key allows you to view Ship-From vendors from Vendor Maintenance.
•
The new E-Mail Address field allows you to email a blanket P/O and works with the PO Document flag in Vendor Maintenance > F5-Add Info.
•
The Contact field defaults to the vendor contact according to the hierarchy in Vendor maintenance.
Blanket P/O at Product Level - New Flag
The Purchase Order Entry screen is updated with a new Blanket P/O at Product Level flag added to Purchasing Option Parameters. The system now checks the Blanket P/O Product Level flag to determine which blanket purchase orders are displayed in the Blanket P/O Availability screen.
If the Blanket P/O at Product Level flag is set to Y, the system checks blanket purchase orders for all warehouses to find the lowest cost for sourcing. Example: If a sales order for warehouse 10 generates a temporary purchase order, the system sources from the blanket purchase order with the lowest cost even if the blanket purchase order is from a different warehouse. If the Blanket P/O at Product Level flag is set to N, the system checks blanket purchase orders for the warehouse to find the lowest cost for sourcing. For example, if a sales order for warehouse 10 generates a temporary purchase order, the system only sources from the blanket purchase order with the lowest cost for warehouse 10 only.
Maintenance
Purchasing Manager - Field Added
A new Purchasing Manager field was added to the F5-Add Info screen in Vendor Maintenance. This option allows you to establish an override purchasing manager to the warehouse level purchasing manager. Note: The buyer number to the left of the purchase order hold flag was removed and replaced with the new purchasing manager field.
Cage Code –Field Added
A new Cage Code was added to the Vendor Maintenance > Add Info screen to allow you to assign a code to print in the DD250 form (block 16) immediately following the product number. Note: The vendor number must be the product’s primary vendor for the code to print.
Vendor Claims Doc Option – Field Added
The Vendor Maintenance > F5-Add Info screen has a new Vendor Claims Doc Option field to determine how a vendor claim is transmitted. Previously the PO Document Option determined transmission options.
Vendor Type Checked For Recommended P/O
The Vendor Type (Vendor Maintenance > F5-Add Info) is now checked to determine if P/O type 01 is setup for this vendor type. If P/O types 01 and 10 are setup for the Vendor Type, Nightly Processing uses type 01 to generate the temporary P/O. If P/O type 01 is not setup and type 10 is setup, Nightly Processing uses type 10 to generate the temporary P/O.
ISO Qualified - New Flag
The ISO Qualified field is a new field the Vendor Maintenance > F5-Add Info screen. This setting determines if this vendor is included in the ISO Qualified preference.
•
Y - To include the vendor in the ISO Qualified preference as Qualified. A Vendor ISO Information box displays to prompt you to enter the date the vendor was ISO certified as well as the user id of the person who qualified the vendor for ISO certification. If notes are entered (F5-Notes), an asterisk (*) displays next to the ISO Qualified field.
•
P - To include the vendor in the ISO Qualified preference as Non-Preferred.
•
N - To include the vendor in the ISO Qualified preference as Disqualified.
The flag setting for this field is checked for defaulting in the accurate setting in the Inspection Req field in the Purchase Order Entry >Line Item Detail > F5-Add Info screen. The Qual Insp Required flags at the product warehouse level/product level override this flag. If a temporary purchase order is created from Nightly Processing, the system updates the Inspection Req flag accordingly when the temporary p/o is converted to a live purchase order. Note: This flag groups vendors for reporting purposes. The flags are:
•
If the ISO flag is set to Y = The Inspection Req flag for the purchase order is set to N
•
If the ISO flag is set to N = The Inspection Req flag for the purchase order is set to Y
•
If the ISO flag is set to P = The Inspection Req flag for the purchase order is set to N
The hierarchy followed when the Quality Inspection fields are in the system.
•
Product Warehouse Maintenance Qual Insp Required field (product warehouse level).
•
Maintenance > F5-Add Info-Qual Insp Required field (product level).
•
3rd Vendor Maintenance > F5-Add Info-ISO Qualified field (vendor level).
Vendor Keyword Maintenance
A new Vendor Keyword Maintenance menu option allows you to establish keywords for valid vendors. However, existing keywords cannot be modified or deleted in this screen. to make the necessary changes, you must go to Vendor Maintenance > F5-Add Info > F5-Keywords.
Freight Rate in VLM – Field Added
A new Freight Rate field was added to Vendor Line Maintenance. All products with the vendor line in Product Vendor Maintenance are subject to this rate. The hierarchy for the Freight Rate is:
•
1st - Product Warehouse Maintenance Freight Rate field (product warehouse level).
•
2nd - Vendor Line Maintenance Freight Rate field (vendor line level).
•
3rd - Freight Matrix Maintenance Freight Rate field (warehouse/product line level).
Import Cost Factor Maintenance – new menu option.
The new Import Cost Factor Maintenance feature allows you to enter an unlimited number of cost factors. The old Import Cost screen in Purchase Order Entry was replaced.
Reports
Product Line Turns Report – Page Break By Warehouse
The Product Line Turns now page breaks for each warehouse listed on the report
Vendor Turn Reports – Stock Status Options Added
A Stock Status option is now available in the vendor turns reports. You now can select S to only include stock items, N to only include nonstock items or B to include stock and nonstock items.
Package Quantity On Recommended Stock Transfers
The PKG column of the Recommended Stock Transfer Report now reflects the quantity entered in the Transfer Pkg Qty field in Product Maintenance. If no quantity is entered here, then the Package Quantity field in Product Vendor Maintenance is used.
Fill Rate Reports – New Reports
Two reports were added to the purchasing reports menu under the new Fill Rates at Order Entry Reports menu option. The Fill Rate Summary Report By Buyer/Planner Report displays all fill rate information updated at order entry for a buyer number and date range. The Fill Rate Detail Report displays fill rate information updated at order entry for a warehouse, date range, and product line. See the new on - line help feature for an explanation of the columns
New Entry Date and Entry Time Columns On Reports
The Venodr Audit reports are updated with a new Ent-Date and Ent-Time column. These columns display the date and time that the modifications were made to customers, pricing, products, vendors and contacts.
Report Writer Format Added to Reports
The Open Purchase Order Detail and Open On The Water Purchase Order Detail Reports use the Report Writer format. The email, fax, and print to screen options are now available.

Bill of Material/Work Order
Work Order Entry
•
The Work Order Entry screen was enhanced to display new pricing and cost fields. The estimated price and cost fields are updated at the time the Work Order is entered. Once the Work Order was issued in Work Order Issues Entry, the ISS (actual) Price and Cost fields are updated with the accurate totals. If the Rollup Pricing flag was set to N for the bill of material entered on the work order, the estimated and actual costs fields are the only fields that are updated given that the prices of the components are not used to make up the component's price.
•
Work Order Entry was updated to now include the ability to enter Line Item notes. Once Line item notes are entered for component lines on the work order quote, an * is displayed adjacent to the product description of the line in which the notes were entered. Both the Work Order Inquiry and Work Order History Inquiry are updated to display the * to identify that notes are attached to the line item.
•
The Header screen has recently been updated with a new 07 order type. The Work Order type 07 is predominately used for one step work orders that were physically completed but not yet entered into the system to update the Inventory and G/L.
•
The Detail Information screen in Work Order Entry was updated with the option to view cost of components in a Work Order. To allow users the ability to view component cost, the Display Cost? Flag in the User Id Maintenance screen must be set to Y or A for each user.
•
Line Notes in the F10-Action Notes option was modified to display comments that have previously been entered utilizing the F-5 Comments option in Miscellaneous Entry. If a W/O number and a burden item are entered in A/P Manual Invoice Entry (/APIE), the Line Notes are updated for that W/O number with the vendor number, vendor description, and invoice number of that A/P manual entry.
Work Order Processing
The Miscellaneous Entry screen was enhanced with a new F-5 Comments option. This feature allows the user to enter comments to be associated with the miscellaneous entry and the burden number entered. These comments will update the Line Notes in Work Order Entry once the burden number is added to the order.
Work Order Quote Subsystem
Work Order Quote Entry was updated to now include the capability of entering Header and Line Item notes. Once Header notes are entered, an * is updated in the notes column in Work Order Inquiry for the Work Order Quote. When Line notes are entered for component lines on the work order quote, an * is updated adjacent to the product description of the line in which the notes were entered.
Processes
ADS will no longer print work order pick tickets using the continuous printing method established in Warehouse Maintenance for work order pick tickets. If the sales order linked to a work order, the order will automatically be placed on credit or administrative order hold. Program Change
Maintenance
Bill Of Material Maintenance
A new warning message was added to this Maintenance option. This message will be displayed to notify the user of any changes that are made to the bill of material such as, component additions or deletions, as well as open work order documents that may exist for that bill of material once they are filed. The user has the ability to locate the open documents and update them with the items added or deleted in Bill Of Material Maintenance. For the warning message to be displayed for fields in the open documents, the fields need to be established in Log Inquiry Setup. For example, if changing the Assembly days in the Bill of Material Header screen, the warning message will only display once the bill of material is filed if open documents exist and the Assembly Days field in the BOM Change Log Setup screen.
Product Characteristics Code Maintenance
The Product Characteristics Code Maintenance option was enhanced with two new fields Style Template and Service Product # 100600A
•
The Style Template allows you to determine which template is to be used with the Product Characteristic code. Select 1 – No Default to load the product characteristics screen and allow product characteristics to be entered. Option 1 is the same screen as previous versions of the software. Select 2 – Use Last Imprint to load the product characteristic information automatically from the last historical information for the customer/product number or purchasing contact/product number entered. Select 3 – Inquiry Selection to display an inquiry of previous sales orders for the customer/product number or purchasing contact/product number to be used for the current sales order.
•
Service Product # was added to allow the service product number to be added to sales orders when a new product template is created for the sales order. This field is only accessible if Style Template 3 is selected.
Inquiries
Work Order Inquiry
Work Order Inquiry screen was updated to include Sort and Filter options, as well as a Work Order Inquiry Filters screen. These functions allow the user to limit the inquiries displayed on the screen to work orders, based on the criteria selected in the sort and filter fields. You also have the ability to set a Default sort for this screen in Default Parameters Maintenance (/DSS).
•
Sort field was added to the Work Order Inquiry screen to allow the user the ability to select how the work orders will be sorted and displayed. The user can choose to sort by B for Bill Number, D for Required Date, P for Product Line, S for Status or W for Work Order Number. If the user chooses to sort by S for Status, the work orders will be displayed in the following order: COMPLETED, ENTERED, RELEASED.
•
Filters flag was added to the Work Inquiry screen to limit the orders displayed. If you wish to display all the work orders, set the Filter flag to N. If you would like to use the filter option, select Y. If you have selected Y, the Work Order Inquiry screen will now be displayed to allow you to enter your criteria selections.
•
Work Order Inquiry Filters screen is a new screen that was added to enable you to limit the display of work orders, based on your filter selection. You now have the choice to filter work orders by one or more of these options. Customer Number, Keyword, Product Line, Status, Type, Beginning Required Date, or Ending Required date. Once you have completed your filter selections, F2 to save the filter information and display the corresponding work orders.

Bill Of Material Inquiries
Bill of Material Log Inquiry is a new inquiry that was added to BMW/Work Order Inquiries. This inquiry will display additions, deletions, or changes to a bill of material in Bill of Material Maintenance. Bill of Material Log Inquiry Codes
•
ADD Bill of Material line item addition
•
ASG Bill of Material number assignment / creation
•
CHG Bill of Material header / line item change
•
DEL Bill of Material deletion
•
DUP Bill of Material duplication
•
ENT Re-entered Bill of Material
•
LDE Bill of Material line item deletion
The Bill of Material Log Inquiry will only be updated for new bill of materials created or processed after receiving the new version of the software (V15) that contains this feature. Any bill of materials created in Bill of Material Maintenance prior to Version 15, will not be displayed in the Log Inquiry until the bill of materials are entered in Bill of Material Maintenance again and changes are made or just reprocessed.
Reports
The Work Order Budget Comparison Report is a new report added to the BMW/Work Order Module. This report will compare and display the estimated budget vs. the actual budget used for completed work orders.
Misc
Allow Completions in WOIE
A new System flag was added to BMW/Work Order Parameters to allow the user the option to complete and close work orders at the time of Work Order Issues Entry or to continue using Work Order Completions. If the Allow Comp in W/O Issues flag is set to Y, the normal prompt that is usually displayed in Work Order Completions will not be displayed in Work Order Issues Entry. The Work Order is then automatically closed, once the work order is completed. The setup for the Allow Comp in W/O Issues is located in System Parameter Maintenance Option Parameters BMW/Work Order Parameters.
Allow Overages in W/O
A new system flag was added to the BMW/Work Order Parameters screen to allow overages received on a work order to be allocated to the sales order quantity on the linked sales order. Select Y to allocate the overage to the line item linked on the sales order. The sales order line item quantity will be increased by the overage quantity. Select N to not allocate the overage to the line item linked on the sales order. The sales order line item quantity will not be increased by the overage quantity. The overage quantity will be added to stock for the bill of material. 091800F
Credit Release Inquiry (/CRI)
A sales order linked to a work order can go on credit hold for numerous reasons. Credit hold will prevent the linked work order from printing a continuous pick ticket. For example, S/O 101766 was put on credit hold for customer 000450 in Credit Release Inquiry due to the sales order going over the customers credit limit of $50.00. W/O 100118 linked to S/O 101766 has not been updated with the CONT code representing the work order as being printed using the continuous printer established. Once the sales order was released from credit hold, the CONT code will be updated for the work order in /WLI when the Release Pick Tickets process (BMW/Work Order > Processes) is completed for work orders. Work Order Printing (BMW/Work Order > Work Order Processing) does not allow printing a manual work order pick ticket until the linked sales order was released from credit hold.
Administrative Order Inquiry (/AOI)
A sales order linked to a work order can go on administrative hold for many reasons. Administrative order hold will prevent the linked work order from printing a pick ticket. For example, S/O 102153 was put on administrative order hold for customer 000400 in Administrative Order Inquiry due to the freight terms overwritten. W/O 100151 linked to S/O 102153 has not been updated with the CONT code representing the work order as being printed using the continuous printer established. Once the sales order was released from administrative order hold, the CONT code will be updated for the work order in /WLI when the Release Pick Tickets process (BMW/Work Order > Processes) is completed for work orders. Work Order Printing (BMW/Work Order > Work Order Processing) does not allow printing a manual work order pick ticket until the linked sales order was released from administrative order hold.

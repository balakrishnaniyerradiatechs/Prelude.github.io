The functionality of the Purchase Order Inquiry has recently been modified to accommodate a new S code displayed in the S column in the inquiry. Buyer Queue Inquiry
•
A new Purchase Order Expedite Items work queue was added to V17. This queue will display the number of purchase order expedite items that currently exist for the buyer. for system to check for expedite levels, the purchase order must be printed prior to performing the Nightly Processing routine. The buyer has the option of e-mailing or faxing the purchase order expedite items for a vendor from in the F5-Review screen. This inquiry will only be updated if the Fax Vendor Expedite Reports are not generated during the Nightly Processing. The system will check the vendor's Fax Expedite Flag (Purchasing>Maintenance>Vendor Maintenance> F5-Add Info screen) to determine if the purchase order expedite items are faxed during Nightly Processing or updated to the buyer's queue for review. This is to make certain that the vendor will not receive notification of expedite items twice. To ensure that the system always updates the buyer queue once Nightly Processing was performed, the Fax Vendor Expedite Reports flag will need to be set to N in the Nightly Processing Setup screen.

Select Yes to recalculate the purchasing parameters. Suspended Purchase Orders inquiry. This new inquiry will display all the Purchase Orders that are currently suspended by a User Id. Information such as the P/O #, Vendor, Weight, Dollars and # of Lines will be displayed. In addition, function keys are also available to allow you to review or edit the P/O listed as suspended. F5-Add Info

P/O Minimum - This new flag was added to allow amounts be established to ensure that the minimum dollar amount required for purchase orders for a vendor are met prior to releasing the P/O for printing. If a P/O is entered and does not meet the required amount, the following message appears: This Purchase Order Does Not Meet the Vendor Minimum and Will Be Placed On Purchase Hold! Please Correct The P/O Or Delete!. Once Continue was selected, this purchase order will automatically be placed on P/O Hold. P/Os created from in /SOE (auto mode) will also be placed on hold however, the warning message will not be displayed. As a result, any purchase orders not reaching the required minimum cannot be printed until it was corrected. to correct or delete this P/O, you must re-enter the purchase order. The Continue option does not return you back into the P/O. Therefore, only P/Os that reach the minimum amount allocated will be available for printing and stock receipts. This message is also displayed in /POE and /POC. Considering the purchase orders cannot be released from in the Purchase Hold Release Inquiry, only P/Os that do not meet the required minimum will not be displayed.

A new B flag was created for the Round Pack Qty flag (the Rounding refers to the Recommended Qty in the Purchasing Parameters for a item). The B flag setting was established to prevent any rounding to take place in Nightly Processing as well as ICS End of Month Calendar Closing when the vendor is identified as a primary vendor in Product Vendor Maintenance. Off Discount Code - with the new feature of calculating On and Off discounts for vendors, a new Off Discount Code was added to Vendor Maintenance. An Off discount is a negotiated discount given to the vendor that is NOT reflected in the invoice amount. The vendor receives two invoices (a credit invoice for the discount, in addition to the original invoice). Once a valid vendor discount code that includes the discount amount, as well as the G/L account that will be offset for the discount is established in the new Vendor Discount Code Maintenance screen (located in both the Purchasing and Accounts Payable Maintenance screens), a valid code can then be entered in this field. If no discounts are required for a vendor, leave this field blank.
•
On Discount Code - with the new feature of calculating On and Off discounts for vendors, a new On Discount Code was added to Vendor Maintenance. An On discount is a negotiated discount given to the vendor that is reflected in the invoice amount. The vendor's invoice is updated with the discount amount in the Miscellaneous Amount field in Vendor Invoice Reconciliation Entry. Multiple G/L accounts can be offset for the discount. Once a valid vendor discount code that includes the discount amount, and the G/L account/s is established in the new Vendor Discount Code Maintenance screen (located in both the Purchasing and Accounts Payable Maintenance screens), valid codes can then be entered. Multiple codes can also be entered. The codes entered here will be defaulted into the On Discount Code on the F5-Add Info screen in the Purchase Order Header. However these can be overridden when creating a Purchase Order.

Vendor Discount Code Maintenance
The maintenance screen in the Purchasing menu has recently been updated with a new Vendor Discount Code Maintenance feature (also available in Accounts Payable> Maintenance). This new maintenance routine was created to provide the ability of establishing On and Off vendor discount codes. These discount codes allow discount amounts as well G/L accounts to offset the discounts be setup for codes. An On discount is a negotiated discount given to the vendor that is reflected in the invoice amount. The vendor's invoice is updated with the discount amount in the Miscellaneous Amount field in Vendor Invoice Reconciliation Entry. Multiple G/L accounts can be offset for the discount. An Off discount is a negotiated discount given to the vendor that is NOT reflected in the invoice amount. The vendor receives two invoices (a credit invoice for the discount in addition to the original invoice). Once valid vendor discount codes were established, they are then used in the Vendor Maintenance screen as well as the Purchase Order Entry Header to apply the pre-determined discount amount.

Surplus Reports> Surplus Report with On Order Merchandise
A new Surplus Report with On Order Merchandise report was added to the Purchasing module for V17. This report was created to provide a list of surplus inventory that contain on order merchandise. The results from this report will are different in comparison to the Surplus Inventory Report in the same Purchasing> Reports menu, in that this report will include ONLY items that contain BOTH surplus inventory as well as on order merchandise such as anything included on any open purchase orders, work orders or stock transfers. If items contain surplus inventory but do not have any on order merchandise pending, these items are not included on the report.

When the Multiple Bin Location flag is set to Y, the default bin locations for products that exist on type 03 purchase orders that are linked to sales orders will be updated. Previously it was required that the detailed information screen in Purchase Order Entry be processed for the bins to be updated.

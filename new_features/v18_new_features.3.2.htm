Accounts Payable
New Form 1099 Printing Options
The Form 1099 Printing process was updated with two new types of printing options. The 1099-Int for Interest, 1099-Div for Dividends options were added. Previously, only the 1099-Misc for Miscellaneous option was available. with these new types, a new Enter 1099 Form Type For This Run field has also been added. This new field allows you to enter the 1099 type to be printed. Select I for 1099-Int (Interest). Select D for 1099-Div (Dividends). Select M for 1099—Misc (Miscellaneous).
New A/P Transaction Audit Features
A/P Transaction Audit Inquiry
This inquiry displays information pertaining to A/P transactions updated for the vendor.
A/P Transaction Audit Purge
This purge routine removes A/P Transaction Audit Inquiry records before the date entered. Once the records are purged, a FWD code is displayed for the vendor in A/P Transaction Audit Inquiry. The FWS code references the accumulated balance of all the A/P transactions purged up to the date entered.
A/P Transaction Audit Rebuild
This process rebuilds all A/P transaction audit inquiry records. Run after purging.
A/P Transaction Audit Conversion
This new process converts all A/P records for the beginning date entered up to the current date to be displayed in the A/P Transaction Audit Inquiry. A CNV code is updated for each invoice converted for the vendor. Note: Rebuild Vendor Invoice Balances (Systems > Rebuild) before the data is converted.
Manual Invoice Entry
New Sort By Product Function Key
The Manual Invoice Entry process was updated with a new F5-Sprt By Product function Key. This new option sorts the consigned inventory records by product number.
Updated P/O Number Field
The P/O Number field in Manual Invoice Entry was modified. Previously, if a contact name was in Vendor Maintenance and a vendor number was entered in Vendor Number field the contact name would be updated in the P/O Number field. This has since been modified. Once the invoice number and invoice date fields are entered, the system clears out the contact name from the field.
Updated F4-Delete Functionality
The F4-Delete & CC Payment Vendor Invoice was updated. When the user manually deletes the C/C.PAID invoice for the CC Payment Vendor, the system performs the void check procedure behind the scenes. The CC Payment Vendor invoice (C/C.PAID) is voided. The original invoices paid off are voided and setup for payment selection again.
New F5-Sort By Product Function Key
A new F5-Sort By Product function key was added to the Consigned screen. This function key allows you to view the consigned inventory records in order by product number.
New FA Chrg Type Field
The new FA Chrg Type field allows you to enter the fixed asset charge type that is used when updating the equipment id. This field defaults from the Asset Tracking Charge Type field in Chart Of Accounts Maintenance for G/L account entered. This flag is accessible if the Update Inv? Flag is set to Y or N.
New Update Inv Field
A new Update Inv field was added to the Equipment ID Number screen in Manual Invoice Entry. This is a Y/N flag that determines if a cost adjustment is created for the serial number associated with the equipment id. Setting the flag to Y created a cost adjustment for the invoice amount. Setting the flag to N will not create a cost adjustment. The Equipment ID Number screen is prompted when a G/L account number is entered that has the Asset Tracking set to Y in Chart of Accounts Maintenance.
New Reas Code Field
The Equipment ID Number screen has a new Reas Code field where you can enter the reason code for the cost adjustment if Update Inv is Y.
Updated Serial Number History Inquiry
The Serial Number History Inquiry now includes a CST code to reference the cost adjustment created.
Update to EDI Freight Inquiry Functionality
This process now checks if the vendor’s freight invoice already exists in the OAP.HIST file. If the invoice does exist, a warning message is displayed notifying the user that the invoice already exists. The manual invoice then is not created in the OAP file. The freight invoice is also removed from the inquiry screen. Note: This process only checks the key (company number!vendor number!invoice number!) in OAP.HIST to find a match.
Invoice Reconciliation Entry
Updated Notes Functionality
The notes screen was updated with the standard notes screen for updating the date and time stamp along with the user id when notes are entered using the F5-Edit Notes function and Update Notes in History flag is set to Y or R in General Parameters. The F6-Edit Notes function key is available for editing notes if the user id is setup accordingly. Previously, the date/time stamp was not updated.
Updated Override Message Displayed
The Override message that is currently displayed in VIRE after entering the Receiver number was slightly modified to accommodate a new Ovr Pay To Vendor flag. The Ovr Pay To Vendor is a new flag added to the POE and POE Conversion screens that will allow a user to manually enter a different pay to vendor for a purchase order (The Ovr Pay To Vendor feature is referenced in the Purchasing section of the New Features list). Typically, the override message would use the pay to vendor that is currently in Vendor Maintenance. If this new feature is used, the override message displayed will now reference the Ovr Pay To Vendor rather than the Pay To Vendor. This modification also applies to the F5-Split and F6-Merge process as well. If the Ovr Pay To Vendor field is left null the Pay To Vendor will be displayed.
Updated F5-Split Logic
The F5-Split logic was modified to accommodate a new disable feature pertaining to the splitting of core items. Users will no longer be permitted to split a core item that was a result of purchasing an exchange product. This process was allowed in earlier versions of the software, however is no longer available. Additional logic has also been added to automatically split the core product if the exchange product is split from the receiver.
New Invoice Status Flag
A new C Credit Card invoice status flag was added for Invoice Reconciliation Entry. This field defaults to C if the Payable Method flag is set to C for the vendor.
New Tax Amount Field
A new Tax Amount field was added to the Invoice Reconciliation screen. This field allows you to enter the tax amount for the vendor invoice. Note: This field is not automatically updated. You must enter the tax amount manually. If the Include flag is set to Y, the tax amount is included as part of the accounts payable amount for the G/L posting. If the Include flag is set to N, the tax amount is not included as part of the accounts payable amount for the G/L posting. The A/P Tax Amount G/L account in the General Ledger Table is used for the G/L posting for the tax amount entered. The tax amount is not included in the landed cost of the products of the vendor’s invoice. Therefore, the Include flag will default to Y so the tax amount will be included in the vendor’s invoice. The A/P tax amount will be a debit posting.
Payment Selection/Print
Updated Payment Selection/Print Functionality
The functionality of the Payment Selection/Print process was updated to now prompt stationary and Unix id messages when certain conditions exist for check printing. This modification was made to provide more informative messages during check printing if the incorrect stationary is chosen. In addition, another change was made to not allow a user id to print checks if a user id is not a member of the Unix printq group. The administrator would have to make the change in Unix for the user to be setup in the Unix printq group to print checks. Previously, the stationery message that appeared before printing was basically ignored and the enable/disable messages would appear on the screen at the end of the print job for checks.
New Batch Type Flag
A new Batch Type C was added for credit card payments. If Batch Type is set to C, the system prompts for the A/P account for use with the g/l posting of the credit card payment. Also, the system ONLY checks for invoices with status set to C (credit card) when using the F5-Auto Select option to select available invoices for payment. When F8 is selected to print checks, the system does not print any actual checks to the printer. The system updates the invoices as paid and creates a new invoice for the total amount of all invoices paid for the CC Payment Vendor. The check number begins with a C prefix followed by the next available check number using an internal sequential counter. The format for the invoice number is as follows: C/C.PAID|check number (C/C.PAID|27 as an example). The system updates notes for the original invoices paid off as well as the new invoice created for the CC Payment Vendor for reference. The system does not allow the invoices paid off with the credit card to be voided. The user will have to manually delete the new invoice created for the CC Payment Vendor. It is recommended to setup a bank code (VISA for example) for use with the A/P Credit Card Clearing account to keep the g/l postings of credit card payments separate from the regular a/p payments. When using the F5-Auto Select option to select available invoices for payment, the system recognizes the original invoices paid off that were voided and setup for payment again.

Checks
Updated Manual Check Register Report Functionality
The Manual Check Register was updated to now print remittance comments after each invoice number. Note: The remittance comments only print if Totals Only is set to N.
Updated System Date Default
The Void Date field in Void Check Entry now defaults to the system date (today’s date). Previously the system defaulted to the original check date.
Updated Select Function Key
With the new Single Chk Per Inv flag in Vendor Maintenance, the F5-Auto Select and F6-Manual Select options were updated. If the manually selected vendor has Single Chk Per Inv set to Y you must manually enter each invoice. You must enter the vendor number for each corresponding invoice number. After the vendor number is entered once, a warning message indicates vendor number was entered.
Reports
New A/P Tax Report
A new A/P Tax Report was added to the Monthly A/P Reports Menu. This report displays information on vendor invoices updated with a Tax Amount value in Vendor Invoice Reconciliation Entry or A/P Manual Invoice Entry for a fiscal period and vendor number entered. The tax value information is pulled from the VAT# (value added tax) field in Vendor Maintenance.
Updated Open Receiver Invoice Report
The Open Receiver Report was modified to break by currency type to provide a subtotal for each currency.
New Sort By? Field - Month End A/P Aging Report
The Month End A/P Aging Report was updated with a new Sort By? field. This new feature was added to allow users the ability of selecting how the information displayed is to be sorted. Enter D to sort by vendor description. Enter N to sort by the vendor number. Previously the report would only be sorted by the vendor description.
Updated Vendor Invoices on Hold Report Functionality
The Fiscal Period field was removed from the GRIM screen. In addition, the report now includes invoices on hold (Hold status option selected in VIRE). The status column displays I for invoice hold or V for vendor hold. The hold status is only displayed on the detail version of the report.
Updated Mont End A/P Aging Report Functionality
Fiscal Period was removed as selection criteria. In addition, the report now includes invoices on hold (Hold status option selected in VIRE). The status column displays I for invoice hold or V for vendor hold. The hold status is only displayed on the detail version of the report.
Updated Mont End Multi-Currency A/P Aging Report Functionality
The Month End Multi-Currency A/P Aging Report has a new status column with I for invoice hold or V for vendor hold. The hold status appears on the detailed report.
New G/L Warehouse Field - A/P Distribution Register
The A/P Distribution Register was updated with a new G/L Warehouse field. This new field allows G/L Warehouse(s) to be entered to determine which G/L Warehouses appear on the report.

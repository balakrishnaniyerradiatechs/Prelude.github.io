Purchase Order Entry
Header Level
F5-Add Info Language Indicator is a new field that defines the dialect used when printing the purchase order. The product description are printed on the form based on the language indicator on the purchase order. This field defaults to the Language Indicator in the vendor record. See Vendor Maintenance and Inventory Product Maintenance for more information. Note: this feature requires User-Form modifications that are billable on a Time & Material basis. Standard forms utilize both English and French.

Maintenance
Vendor Maintenance
•
F5-Add Info - Language Indicator is a new field that defines in which dialect the product description will print on purchase orders for the vendor. For example: if a vendor is set to 2 for French, then the French description entered in Product Maintenance will print on purchase orders for the vendor. This field is the default but may be overridden in purchase order entry. See Purchase Order Entry and Inventory Product Maintenance for more information.
Note: this feature requires User-Form modifications that are billable on a Time & Material basis. Standard forms utilize both English and French.
•
Vendor Type is a new field that defines which P/O types are valid in purchase order entry, stock receipts entry, and automatic purchase order generation from sales order entry. Leave blank to allow all P/O types. For example, a vendor may only ship to your store or they will allow you to come pick up the item. A vendor type code could be set up defining 01 and 03 as valid P/O types. That code could then be set as the Vendor Type for the vendor. Then only 01 and 03 type purchase orders would be allowed for the vendor. See Vendor Type Code Maintenance for more information.
Vendor Ship-From Maintenance
F10-Action - Auth Whses is a new option that allows you to define warehouses that may use a certain vendor’s ship-from address. Multiple warehouses may be defined. If the Auth Whses field is empty, the ship-from may be used in all locations. Any time a purchase order is created, the system will determine if it can set a default ship-from by cycling through the ship-froms and finding a match on the warehouse. The following rules apply:
•
If the system does not find a ship-from for the vendor then the Ship-From Number field in the purchase order header will be set to D.
•
If the system finds a single ship-from for the vendor with authorized warehouses and the current purchasing warehouse is not in the list then the Ship-From Number in the purchase order header will be set to D.
•
If the system finds a single ship-from for the vendor and it contains no authorized warehouses then the Ship-From Number in the purchase order header will be set to that ship-from number.
•
If the current purchasing warehouse is authorized for multiple ship-froms for the vendor, the Ship-From Number will be set to the first ship-from the system finds.
•
If the system finds multiple ship-froms for the vendor, all without authorization warehouses, then the Ship-From Number in the purchase order header will be blank. User will have to select the ship-from.
Discount Break Table Maintenance
•
Sum and Recost is a new field that determines if a vendor line using the current discount break table allows sum and recost for all line items on a purchase order in that vendor line. In other words, should purchase order line items whose vendor line is using this discount break table be included in recosting? This flag works for both Nightly Processing and the Recost Purchase Order feature in Purchase Order Entry. If this flag is set to Y, purchase order line items are accumulated by vendor line and recosted if a break level is reached. If this flag is set to N, purchase order line items are recost based on their quantity or dollar value.
•
Vendor Type Code Maintenance is a new maintenance routine used to create and maintain vendor type codes used in vendor maintenance to define valid purchase order types for the vendor. These codes limit vendors to P/O types. User is prompted for a vendor type code, description, and valid P/O types. Multiple P/O types may be entered. See Vendor Maintenance for more information.

Emergency/Direct Line Items
•
The Buy field on the F5-Review screen was modified to display an H if the order is on credit hold, an O if the order is on administrative order hold, or an I if the order is on administrative invoice hold. If a work order is linked to a sales order, the sales order will be checked to determine if it is on credit or administrative hold. In other words, the value displayed will be for the sales order and not for the work order. If the sales order is on hold, this field cannot be changed till the order is released. A purchase order will not be generated for these type items.

Added a Change Vendor option to the F5-Review > F10-Action bar that will allow you to change the purchasing vendor. For example, you may be able to get the item quicker from a vendor other than the primary vendor. When you enter and save the vendor it is immediately changed. Note: Nightly Processing rebuilds this information so any vendor changes will revert back to the original setting if the item is not converted.
Linked Purchase Order Inquiry
Displays purchase orders that were affected by changes to a linked sales order. Drill down on this inquiry displays the purchase order affected, the vendor, the reason code, and the first line of notes from the originator of the sales order. User can review the purchase order, view and send notes to the order originator, and release the purchase order. Note: the purchase order is not placed on hold but may be processed as usual. When a purchase order is released, the user is prompted for notes to the order originator. If notes are entered, the order will be sent to the order originator’s queue for review. When this inquiry is exited, all purchase orders that were released will be removed from the queue. All purchase orders that have notes to the order originator will be forwarded to that person’s queue. See System Purchasing Option Parameter Maintenance, System Miscellaneous Controls, and Sales Rep Inquiry for more information.
Reports
The Overdue Purchasing Report was enhanced to give the user the option to print the report to the printer or to the screen.

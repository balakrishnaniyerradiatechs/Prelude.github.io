Accounts Receivable Parameters
The Accounts Receivable Parameters screen was updated with a new Avg Days to Pay Months field. This flag enables the user to enter a number up to 24 to months to be calculated by the system to determine the average days to pay, rather than using the average number of days from the time a customer was established in the system. If this field is left blank, the total cash applied since the beginning of time for that customer is used as the basis for the calculation of the average days to pay. If a number is entered, the system will use the total cash applied beginning with the current month to date and going back for the number of months entered as the calculation of average days to pay. For example, if 24 is entered, the system will use the current month and the previous 23 months of cash applied.

Order Processing> Consignment Order History
A new purge routine was added to Version 16’s Purge Utility Menu. The Consignment Order History purge removes all consignment history records for the invoice date entered and prior. The Purge Log Report has also been updated with a Purge Type 37 to only include purged records for consignment order history records.
Marketing> Marketing Activity Queue
The Marketing Activity Queue is a new purge routine that was added to the Purge menu. This new purge provides the user with the capability of transferring marketing activity queue records from one user to another. If the user does not wish to move activity information from one user id to another, both the activity information and the queue records will be deleted. Please note that the Purge Log Report will not be updated to reflect the purging or transferring of the marketing activity queue records.
Marketing> Marketing History
with the new features added to the Contact Diary System, a new purge routine was added to the Purge Utility Menu. The Marketing History purge removes all marketing history records created prior to the Ending date entered for this purge. Once the purge routine is completed, the Purge Log Report will be updated with a Type 40 (Marketing History) code.
Accounts Payable> Open Receive Purge
The Open Receive Purge option is a new process that was added to the Accounts Payable purge menu. This purge allows you to remove all open receivers that are at least 90 days old or older for a Warehouse Number, Vendor Number, Receive Date, and P/O Type. This process is the same as manually deleting open receivers in Vendor Invoice Reconciliation Entry using the F4-Delete but allowing you the ability to perform a mass deletion of receivers rather than deleting them. The Purge Log Report has also been updated as well, with a Purge Type 34 to indicate which open receivers were purged.
Accounts Receivable> Transaction Audit
This Transaction Audit process is a new purge routine that was added to the system. This purge enables a user to enter a transaction date, then removes all A/R Transaction Audit Inquiry records created prior to the transaction date entered. Once these records were purged, a new FWD code is updated for each customer's records that were purged. Miscellaneous Features
A new Y/N flag was added to the Additional Parameters screen. This new Use Serialized Cost with FIFO flag determines if serialized items will use the serial cost of the item once FIFO costing was established. If FIFO costing is used, FIFO layers are relieved based on the first in first out method and the cost layer being relieved. If you currently have serialized items, FIFO typically ignores the serial cost and updates by layer. Select Y to allow the serial cost of the item to be used instead of the FIFO layers cost. The layer’s on-hand cost will be relieved appropriately. Select N to allow the FIFO layer cost of the item to be used.

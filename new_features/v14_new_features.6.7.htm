Accounts Receivable/Accounts Payable
Cash Receipts Entry
•
This routine was modified to allow an incomplete batch to be placed on hold till the user can complete it at a later time. An incomplete batch is one where the Deposit Total (batch total) amount does not match the Batch Total Applied amount. Previously, the user was not allowed to exit this process till the total cash applied matched the batch total. Now the user can exit a batch that is out of balance and call it up at a later time for additional cash applications. If a batch is incomplete, a general ledger posting will be made for the total cash applied amount, keeping the general ledger in balance. Batches on hold will not show up on any cash receipts reports. They will only display on the Incomplete Cash Receipts Report. See Reports for more information.
And Hold Flag Equal To is a new prompt on Batch Number F3-lookup that allows only those batches on hold to be displayed. If this field is set to H, only those batches on hold will be displayed. If this field is left blank, all batches will be displayed. The lookup screen will display the hold flag for all batches displayed so the user can identify batches that are on hold.

Chargeback Invoices option parameter and the Charge Back Invoice by Customer P/O option parameter into account to determine how the charge back invoices will be created. This process was modified to update, for each invoice paid, the amount paid, whether or not the payment was received in the Eligible Days to Pay days from the Invoice Date (hereafter know as the eligibility period), and the percent of the order paid in the eligibility period. These fields are used in the rebate reports to calculate the rebate amount. If part or all of an invoice is paid before the eligibility period expires, that invoice is eligible for a rebate. The system updates the percentage of the invoice paid before the eligibility period expires. The percentage is updated by each cash receipt so that it will reflect the cumulative payments made prior to the expiration of the eligibility period. For example, if only half the invoice is paid during the eligibility period, then the customer will only receive a rebate on half the eligibility amount and not on the whole amount. See Order Processing Customer Rebates Maintenance, Order Processing Invoice Processing, Accounts Receivable Customer Rebate Maintenance, and Order Processing Reports for more information.
Service Charge Processing
Service charges generated through Auto Service Charge Entry or entered in Manual Service Charge Entry will be created in the customer’s currency defined in Customer Maintenance. If the customer does not have a currency rate defined, the system will determine if the daily exchange rate will be used or the currency table.
Customer Rebate Maintenance
Customer Rebate Maintenance is a new maintenance routine used to view rebate information for a customer’s invoices and to maintain rebates calculated during the cash receipts procedure. It can also be used to turn invoices off or on for rebates. This screen displays the invoice number, the Eligible Days to Pay (number of days from the invoice date that the rebate is good for), the invoice amount, the amount paid to date, the amount eligible for a rebate (for the entire invoice not for the paid amount), the percentage eligible for a rebate (from payments), the Credit Report Freq, and whether a rebate will be given or not (Y/N). The system also gives you the ability to view invoice information. See Accounts Receivable Cash Receipts Entry, Order Processing Invoice Processing, Order Processing Customer Rebates Maintenance, and Order Processing Reports for more information.
Inquiries
•
Customer Invoice Inquiry - The Stat field was modified to display a P if the invoice number is specified on an RGA. A P will only display if the invoice is open and has no prior status (disputed, etc). The P is removed once the RGA is converted to an actual credit memo and there are no other open RGAs against the invoice or if the invoice status is manually changed. Pending RGAs may be viewed by invoking the F5-Review Invoice > F7-Pending RGAs option for the current invoice. See Invoice Detail Inquiry for more information.
•
Invoice Detail Inquiry - F7-Pending RGAs is a new function key that displays all pending RGAs for the current invoice. User has the ability to drill down into the RGA. See Customer Invoice Inquiry for more information.
•
Cash Receipts Reports - Incomplete Cash Receipts Report is a new report that displays batches that were placed on hold in Cash Receipts Entry because the total cash applied did not match the batch total. This report can be run for a bank code, deposit date range, and/or batch number range. See Cash Receipts Entry for more information.
AP/PO Subsystem
Vendor Invoice Reconciliation
Added logic to capture the actual freight amount per serialized item. The system proportions the freight amongst the line items and adds it to the landed cost. If the item is a serialized product, the freight amount per serial number is captured and stored in the Prod.SN file. The actual freight will replace the estimated freight captured at stock receipts time. See Inventory Stock Receipts for more information.
Vendor Claim Reconciliation
For the Receiver Number field, added Vendor RMA# to the lookup options which gives you the ability to search for a receiver by entering an RMA number. Only an exact match will be returned. For example, if the RMA number is RTN-3095, entering RTN or 3095 will not find the receiver. You must enter this number exactly to find the corresponding receiver. If the system finds more than one receiver with the same RMA number then those receivers will be displayed in a selection box for you to determine the correct receiver to process.

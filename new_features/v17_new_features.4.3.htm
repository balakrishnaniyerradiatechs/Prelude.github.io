General Ledger
Reoccuring Journal Entries
•
Reversal Date was added to allow a journal entry to be reversed in the next fiscal period. Reversing Journal Entries are typically used to accrue an expense for one month, and the Reversing Entry backs out the transaction in the next month. Any Journal Entry may contain a reversing date. This date is validated on the Fiscal Period Table. The reversal date should be greater than the Transaction Date. At the time of posting the JE, the system will create two entries. The first will be for the Transaction Date for the amounts as shown on the data entry screen, the second will be for the Reversal Date. All Debits will be posted as Credits, and Credits as Debits. This effectively removes the Journal Entry effect from the Ledger.
•
If a Recurring Journal Post has a reversal date, it is cleared for the recurring journal entry. If the recurring journal entry is setup for multiple fiscal periods and requires a reversal entry for each month, you must open the journal entry in Recurring Journal Entry after each month's posting to update the reversal date before next month's postings.
GL Balances Postings
The GL Balance Posting routine was modified for V17+. To simplify this process, a GRIM (General Report Interface) screen was created to allow you to enter information pertaining to the posting process. The GRIM screen contains four fields that allow you to customize the posting process to your needs.
•
Requested Module To Post = code to be posted (AP, for example) to reduce the posting routine to a module. Leave this field blank to post ALL modules.
•
Fiscal Period = Enter the fiscal period for the un-posted transactions that are currently available for posting. Enter C if you would like the system to assign the current fiscal period and run the posting routine on a daily basis. Note: If the scheduler uses the C Current Fiscal period, the posting is based on the time that the schedulerruns. For example, if the scheduler posts at 1:00 am, based on your Fiscal Period setup, it could be posted to the next period.
•
Print G/L Summary Report = determines if the G/L Summary Posting Report is printed for all G/L accounts updated during the posting routine. If the Scheduler is used to schedule the posting process at a date and time, this flag is ignored.
•
Email Address Notification = Enter the email address that the system is to use for the recipient of e-mails pertaining to any G/L errors occur during the posting routine. Due to the fact that the posting routine is running in the background, you are not aware of problems that may have occurred during the posting process. The email address is only used if the posting process was setup in the Scheduler. This flag is ignored if the posting process is performed at that time (Execute option) since you see the G/L Posting Error message.

Contact Diary System
Contact Diary Entry
The process for the Contact Diary Entry screen was modified. Previously of the software, the contact diary entries were updated to the CONT.DETAIL file. The program that updated the contact entries that occurred in this screen was rewritten, and will now update the information to the MKT.HISTORY.UPD file.
Contact Diary Report
The Contact Diary Report was was changed in a couple of ways. First, in previous versions, the Contact Diary Report and Contact Diary Inquiry were combined into one option. For version 16, this was modified and now the Contact Diary Report and Contact Diary Inquiry exist as two separate options. In addition, the Report is now formatted as a GRIM (General Report Interface Maintenance) allowing the user to modify this report to include additional information if desired. Secondly, the file used in updating this report is now the MKT.HISTORY.UPD file. Which contain transactions entered in Marketing Activity Entry, Marketing Activity Contact Entry and Contact Diary Entry as well as sales orders, quotes and work orders that may be entered for a contact number. Prior to Version 16, contact entries were updated to the CONT.DETAIL file.
Contact Diary Inquiry
The Contact Diary Inquiry was was changed in a several ways. First, in previous versions, the Contact Diary Inquiry and Contact Diary Report were combined into one option. For version 16, this was modified and now the Contact Diary Inquiry and Contact Diary Report exist as two separate options. In addition, the Inquiry has added two new fields, Sort and Filter. The Sort option allows the user to choose the sequence in which the information retrieved for the inquiry will be displayed. The Sort field will automatically contain the default sort code that has previously been established in Default Parameters Maintenance. However, the user does have the ability to enter a different selection to choose an arrangement that best suits this user's needs. The available sort options are C-Contact Number, E-Entry Date, M-Contact Method, P-Product, S-Action Summary, T-Document Type, and U- User ID. The Filter option allows even more customization abilities for displaying information in this inquiry. This feature filters information such as Product Numbers, Major Groups, Customer P/Os, Invoice # etc. In addition, given that this feature is in a GRIM (General Report Interface Maintenance) format, the user also has the capability of modifying the inquiry even further to include additional search criteria. The information displayed in this inquiry is pulled from the MKT.HISTORY.UPD file which contain transactions entered in Marketing Activity Entry, Marketing Activity Contact Entry and Contact Diary Entry as well as sales orders, quotes and work orders that may be entered for a contact number
Processes
Copy Contact To New Customer
A new Copy Contact To New Customer option was added for Version 16. This feature allows an established contact for one customer to be copied to another customer. Once the From Customer, Contact and To Customer fields were completed, the Customer Contacts screen will then be displayed. At this point, the Contact will be assigned the next available contact number as well as having the ability to modify any information pertaining to this contact. If open sales orders or quotes currently exist for this customer/contact number, a message will be displayed informing the user that open documents exist and the copy process will not be performed.
Note: Historical data will NOT be transferred to the new contact number with the exception of notes that were previously entered for the original contact number. In addition, the system will automatically update the Sales and Obsolete flag settings to Y for the original contact number used in the copy procedure.
Merge Contacts in Customer
Version 16 was updated with a new Merge Contacts in Customer option in the Marketing module. The process allows customer contact information as well as historical data for a customer to be merged into another contact that is currently established for the same customer.
Form Letter Batch Print
A new process was added to the Marketing module. The Form Letter Batch Print option will print all Form Letters that currently are established to be printed in batch mode, including the Form Letters that were not printed at the time of entry. This GRIM (General Report Interface Maintenance) screen allows the user to enter a Form Letter Number as well as a Customer Number to limit the batch printing to just ones that are associated with this customer or form. In addition, the user also has the option of leaving these fields blank to print all Form Letters for all Customers that are to be printed in batch mode.

A new F5-Update option was added to the Marketing Activity Inquiry screen for Version 16. This new feature enables the user to update an activity displayed on this inquiry without having to manually go to the necessary screen through the usual process. This new option simplifies the steps for the user to better use their time focusing on closing potential accounts rather than having to go through multiple steps to update the profile itself. Based on the Document type associated with a activity, the F5-Update function key will automatically take the user to the necessary screen to update this activity. If the Document type is Q, when the F5-Update key is used, the user will be taken directly to Quotation Order Entry to allow for a Quote to be entered. If M, P, or T are the associated document types, the F5-Update will option will take the user directly to the Marketing Activity Entry screen for further updating.

This new option was added to allow the user the option of updating the time and date of a activity displayed in the inquiry. This new feature is beneficial to the user in that the user now has the ability to update the activity itself, or the follow up date and time to make the necessary follow up phone calls, complete to do items, or enter quotes for a potential customer, without having to go through the multiple steps of getting to the update screen. Once the time and date were changed, the inquiry screen will then be resorted and redisplayed to list items in the revised order.

Activity Defaults
The Marketing module was updated with a new maintenance option. The Activity Defaults menu is a very beneficial feature for customizing follow up processes to better track results pertaining to prospect and contact activity. This new feature allows a start menu to be created and associated with customized default information such as user IDs, dates, times and actions that will be used in the Marketing Activity Entry and Marketing Activity Contact Entry screens. To use this feature, information will need to be established.
•
Step 1: Establish the name of the customized start menu in the Activity Queue Menu field in Marketing Parameters screen. To use this feature, it is required that the Start Menu name begins with CUSTOM.
•
Step 2: Determine the sub menus and menu options that are to be available to the user for entering information such as phone calls, follow up visits, successful contacts etc.
•
Step 3: Create a id that will pertain to information that is to be viewed. Once you have created and entered the Default ID, enter the default information that you would like to be displayed on the Marketing Activity Entry and Marketing Contract Entry screens. The default fields will consist of information such as To Do, Phone and Open defaults that can be established for a menu option.
•
Step 4: Create a menu in the Menu Definitions screen to include the menu options that are to be displayed in the Marketing Activity Entry and Marketing Contact Entry screens. To use this option, it is required that the menu name begins with CUSTOM. Ex: CUSTOM. PROS.ACTIVITY, once this was entered, the remainder of the necessary information has can be entered for this menu option.
Once this criteria was established for using this new feature, the customized start menu will be available to the user in the Marketing Activity Entry and Marketing Activity Contact Entry screens. This will allow the user to view and select menu options from the start menu as well as sub menus to choose which default information is to be entered into these activity screens. This features helps to reduce the number of keystrokes by the user, enabling users to complete updating activity information in a more timely manner.
Reports
Send Literature Report
A new feature was added to the Send Literature Report. This will prove to be a very valuable time saving feature to the Marketing module. The Send Literature report now offers the ability to print mailing labels for each address listed on the report. Previously, this option was not available therefore, any mailing labels that were needed, would have to be done manually. Once the report has printed to the screen or the printer, the user will be prompted with a Do You Wish To Print Mailing Labels For Literature Requests Just Printed! message. This allows the user to choose if labels are to be printed, not printed, or if the user is to be re-prompted with the same message. Once a selection was made, system will display an additional message asking the user if the labels have printed correctly. If the user chooses Yes the report will then be purged and this report will be not be available to be reprinted. Contact Management flag will determine which address will be used on the labels. If the Contact Management flag is set to Y the contact's address established in Customer Contacts will be used. If the flag is set to N, the address established in Prospect Maintenance will be used.

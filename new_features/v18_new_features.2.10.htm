Order Processing
Sales Orders
Updated Customer P/O# Functionality
The functionality of the Customer P/O# field was updated to accommodate the new E Administrative Order Hold type. Versions 18 & higher were enhanced with a new Administrative Order Hold process that will place orders on hold that exceed the pre-determined blanket purchase order amount. This update works with the new E AOH type (Referenced in the Systems New Features list) as well as the new P/O Customer # field in the Require P/O Number screen in Customer Maintenance. Once a customer number is entered in the Customer P/O# field, the system will review this customer's blanket p/o information. The F3-Look up option is also available to display all the blanket P/Os for this customer to allow you to select one if multiple blanket P/Os exist. If the total dollar amount of the items entered on the sales order exceed the amount available on the blanket P/O, the message Order was Placed on Administrative Hold! is displayed after you select Pick Ticket. In addition, the message Warning, Blanket Purchase Order Limit was Exceeded! will also be displayed, notifying the customer of the reason this order was placed on hold.
New Auto Cancellation Date Field
The Sales Order Entry screen was updated with a new Auto Cancellation Date field. This new feature provides users with the capability of entering a date that will be evaluated during Nightly Processing when the date is reached, therefore, determining if this order should be cancelled. Each line on the sales order will be assessed separately and cancelled if no linkages or pick quantities currently exist. This option is primarily beneficial for canceling sales orders that contain items on backorder. In addition, an Auto Cancellation Date field has also been added to the F5-Add Info screen at the Line Item level. This allows users to establish cancellation dates for each line item. The date at the Line Item level will override the date entered at the header level. However, if the cancellation date is null at the line item level, the date set up at the header level will be used. With this new option, several updates throughout the system have also been made to incorporate this feature. The logic in the Nightly Processing routine was modified to accommodate this new auto cancel option. The Monthly Lost Sales and Monthly Lost Sales By Vendor reports were updated to include a CD lost sales reason code to identify which sales were lost due to the Auto Cancellation Date process. The Sales Order Log Inquiry has also been updated with a new CAN code. This code will be displayed indicating that a line item was deleted during the Nightly Processing routine due to the auto cancellation date. The Reference column in the Order Log Inquiry will display the product number that was deleted. In addition, the Order Log Inquiry will also display DEL indicating that the entire sales order was deleted due to the auto cancellation date. The Reference column will be updated with the Auto B/O Cancellation reference.
Note: To update product usage due to the auto cancellation date, set CD as a Lost Sales Reason Code.
New Return To Product Option
The Warning!! This Product Number was Previously Entered. message was updated with a new Return To Product option. Previously, the only options available were Return to Order which returns the user back to the order to allow a different product to be entered. Continue which returns the user to the Line Item Detail screen and allow the duplicate product to be entered. The new Return To Product option takes the user to the first occurrence of the product number and removes the duplicate product number.
Updated A/R Terms Functionality In Sales Order Entry
The completion screen in Sales Order Entry was modified with a new feature. This applies to customers associated with an A/R Terms code that have the Deposit Required flag set to Y. The Deposit Required flag is a new flag that was added to the A/R Terms Code screen in the Customer Control Files option. If a sales order is entered for a customer that currently is attached to an A/R terms with the Deposit Required flag set to Y and the Acknowledge Flag is not set to Print, Fax or E-mail. A message will be prompted. The message Do You Wish To Print An Acknowledgement For Receipt Of Deposit? will be displayed, providing the user with the option of printing an acknowledgement for this order. Acknowledgements will only be printed for sales orders generated from in SOE or converted from quotes. This feature does not apply to orders generated from DOE. In addition, several other areas were updated or added to incorporate this new feature. A new DPA Order Log Inquiry code as well as a new Summary G/L Recap option was added in the Open Sales Order Inquiry.
Note: A USER-FORMS modification will be required for customers upgrading from Versions 17 and earlier.
New Promotional Matrix Functionality
The Sales Order Entry screen was updated with a new functionality when entering a product that is currently associated with a promotional matrix. Once a product is entered that contains a promotional matrix, the Customer Promotion Matrix Inquiry screen will be displayed, providing a list of promotions currently available pertaining to this product. Once reviewed and F2 is selected, the detail screen will then be displayed allowing the user to select the quantity for this promotional item. If multiple promotional matrices exist, the promotional item given to the customer is based on the promotion that matches the quantity entered. When the order is complete, the Promo List Display box lists all the promotional items added to the order.
Note: The Promo List Display feature is not available with confirmation orders (06).
New Currency Conversion Screen
The completion screen in Sales Order Entry was updated with a new Currency Conversion screen to allow you to automatically convert the US dollar amounts entered for additional charges to the correct currency type for this customer. This new option simplifies the process of applying additional charges such as Inbound and Outbound freight, Handling, Delivery, Pickup, Restock, or Minimum charges to an order that is not currently associated with a US currency customer. When entering additional charges on an order for a non-US currency customer, the user can select the Currency Conversion option this screen will display the currency rate for this customer. The user then has the ability of entering the dollar amounts in US currency for each of the additional charges that apply. Once completed, select F2 to convert the amounts entered to the correct currency for this customer. If changes need to be made to the amounts once they were converted, the user can simply select the conversion process again and re-enter the amounts. After the F2-Save option is selected, the completion screen will be updated with the modified dollar amounts.
Updated Salesrep Number 2 Functionality
The functionality of the Salesrep Number 2 related to the Sample Request Entry process was modified for Versions 18 & higher. If the Salesrep Number 2 in the specifier ship-to is left null, the Salesrep 2 for the customer ship-to number will default into the Salesrep Number 2 field. Once this process was completed, the system will be updated with the Salesrep 2 from the customer ship-to as the Salesrep Number 2 to be used for the ship-to.
Updated Equipment Maintenance Functionality
The functionality of the Equipment Number has recently been enhanced. Now, when a non-rental serialized item is entered on a sales order and the order was invoiced, the system will update both the equipment and fixed asset record for that serial number. The disposal information will be updated in Equipment Maintenance, whereas in earlier versions of the software, this information was not updated at all.

Picking & Shipping
Update to Warehouse Release Confirmation Entry functionality
The functionality of Warehouse Release Confirmation Entry now allows you to choose to print a packing list. Select Y to send the packing list to the continuous printer. Select N to not print the packing list.
Note: The continuous printer must be setup to print sales order packing lists to use this option.
Update To The Fill Rate Detail Report Functionality
Due to the new Fill Rate Purge process, the Fill Rate Detail Report will no longer prompt the user to purge the data at the end of running the report.
New Stand-Alone Email Shipping Notice
A new Stand-Alone Email Shipping Notice option was added to the Optional Entry & Printing menu. This new feature enables users to send text e-mail shipping notices for release numbers. Once the Release Number was entered, (which is pulled from the ORDER file or ORDER.HISTORY file based on if the order was invoiced yet or not) the Customer Number and email associated with the release are defaulted. You can change the e-mail.
Note: To use this feature, the Email Ship Notice flag for customers must be set to N (Customer Maintenance> F7-Order> F5-Add Info). In addition to this new option, a new %CPO% tag has also been added as a valid tag in the Email Document Text Maintenance screen. This new tag denotes that the customer p/o number will be printed in the body of the text e-mail for order acknowledgements and shipping confirmations.
Inquiries
New Summary G/L Recap Option
A new Summary G/L Recap option was added to the Order Info menu in SOI. This new feature was created to display the G/L Entry for the deposit amount that was received for this sales order. This works with the new Deposit Required feature added in the Customer Control Files.
Note: These new types can only be used if Product # Prefix is Y in Inventory Parameters. To view the G/L Entry information, the Display Cost? field in your User ID Maintenance must be set to A.
New Sales Order Log Inquiry Codes
•
CAN - To accommodate the new Auto Cancellation Date field in the Add Info screens at both the Line Item level and Header level, the Sales Order Log Inquiry was updated. A new CAN code will be displayed indicating that a line item or multiple line items were cancelled due to the Auto Cancellation Date process. The CAN code will be displayed for each line item cancelled. In addition, the reference column in the inquiry will reference the actual item number that was deleted. Note: The CAN code is primarily for items; DEL is for entire sales order.
•
DEL - To accommodate the new Auto Cancellation Date field in the Add Info screens at both the Line Item level and Header level, the Sales Order Log Inquiry was updated. The DEL code will now also represent sales orders that were deleted due to the Auto Cancellation Date process. In addition, the reference column in the inquiry will reference Auto B/O Cancellation.
•
DPA - With the new Deposit Required flag added to the A/R Terms Code Maintenance screen, a new DPA code was created to distinguish that a deposit was applied for an order. The reference column displays Deposit Applied and the amount.
•
H-C - With a new Credit Limit Amount field added to User ID Maintenance, the Sales Order Log Inquiry was updated. The inquiry will now reference any Sales Orders, Credit Memos and RGAs that were placed on Administrative Invoice Hold due to exceeding the credit limit amount in the Credit Limit Amount field. The reference for the AIH code will be displayed as the Manager Code followed by H-C.
•
J - With a new Quote Limit field (Referenced in the Systems New Features list) added to User ID Maintenance, the Administrative Quote Release Inquiry was updated to incorporate this new feature. Once a quote is updated from Quote Hold due to exceeding the Quote Limit in User ID Maintenance, the J code will be updated to reference this. When the quote is released from hold, the system will now prompt you to Print or Suspend. Select Print to set the quote up for printing or. The system will then determine the print output, whether it be to print, email or fax the quote based on the print box selected at the time of processing the quote in Quotation Order Entry. If Suspend is selected, the quote is suspended and not available for printing.
Updated Continuous Printer Functionality
The functionality of the Continuous Printer for Administrative Order Release Inquiry was modified for V18+. Now when a sales order that is linked to a work order is released from credit hold in Credit Release Inquiry, the system now checks the linked work order to see if released (Auto Release flag set to Y in BMW/Work Order Parameters) and continuous printing is used. If the criteria is met, the system sends the work order pick ticket to the continuous printer. Previously, you had to run the Release Pick Tickets process to find the available work orders released from administrative order hold available for printing.
Reports
New Page Break Feature
A new Page Break on Salesrep/WB field was added to the GRIM screen for the Open Sales Order Detail Report. This is a Y/N field that allows the user to select if a page break is to take place when generating this report. The page break will occur after the salesrep or the written by, based on the flag setting in the Sort By Cust, Salesrep, WB flag. If the sort by flag is set to S, the report will break after each salesrep's information. If the sort by flag is set to W, the report will break after each written by information. If this flag is set to N, this report will not be generated with any page breaks.
New Cash Sales Breakdown Register Report
A new report was created for the Point of Sale Subsystem. Generating this new Cash Sales Breakdown Register report will provide cash register sales breakdown information by Cash, Check, Credit Card and Gift Certificates for a date. However, gift certificate information will not be updated until the 05 order is invoiced.
New Daily Deposit Received Report
A new Daily Deposit Received Report was added with the new Deposit Required flag. This report will display information pertaining to deposits received from sales orders that are associated with A/R terms codes that have the Deposit Required flag set to Y. The information on this report will include Order #, Customer #, Payment, Terms and Deposit amounts. In addition, this report breaks and totals for each cash pay type.
Updated Monthly Lost Sales Report
To accommodate the new Auto Cancellation Date field in the Add Info screens at both the Line Item level and Header level, the Monthly Lost Sales report was modified. This report will now display a CD lost sales reason code to indicate that a line item or the entire sales order was deleted due to the Auto Cancellation Date process.
New ‘Ent-Date and Ent-Time columns
The Customer, Pricing, Product, Vendor, and Contract Audit reports are updated with a new Ent-Date and Ent-Time column. These columns display the date and time that the modifications were made to customers, pricing, products, vendors and contacts.

New Retail Credit Sales Entry Option
The POS Subsystem was enhanced with a new Retail Credit Sale Entry option. This new feature enables credits to be entered for cash refunds on returned items. This procedure is similar to the Credit Memo Entry process in which the line item detail screen defaults to a negative quantity and refund amount. However, you can still enter a positive quantity as well as a positive amount for any re-bill situations. It is recommended to use this entry process for entering returns if the Suppress Line Item Detail flag in Point of Sale Options Maintenance is set to Y. Due to the fact that the detail information screen is not prompted in Retail Cash Sale Entry.

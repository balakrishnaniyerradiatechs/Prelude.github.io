Bill of Materials/Work Order
Work Order Entry
Line Item Level
•
Work Order Entry was modified to better deal with product notes for a bill of material. Previously, the product notes were not displayed if a bill of material was entered as a component and exploded. Now, the product notes will appear after the comment line stating Beginning Of BOM.
•
Work Order Entry was enhanced to allow automatic transfer generation for line items with a T or R supply code. The system will check the user id record to determine if the user has authority to generate transfers from in Work Order Entry. The Automatic Transfer Generation procedure will work the same way it does in sales order entry. See System User Id Maintenance for more information.
Line Item Detail Level
The Type field has a new option of V (vendor claim expense) that is used to define products for which a vendor credit is expected. Setting the Type to V for a line item tells the system to track the item as a vendor claim expense allowing the user to create a vendor claim once the work order was completed and to post the item accordingly. You are prompted for an Expense Reason Code and the Vendor Number.
•
The Expense Reason Code field defines why the item is expensed as well as the account used when posting the expense line item in Work Order Completions. The account defined for the reason code in Work Order Expense Reason Code Maintenance is also used for the vendor claim miscellaneous charge line item when the vendor claim is automatically generated. If the reason code does not contain an account number in Work Order Expense Reason Code Maintenance, the system will use the Expense Work Order account defined in General Ledger Table Maintenance.
•
The Vendor Number field defines the vendor to claim the credit from. This field defaults to the product’s primary vendor but may be overridden.
This feature offers the ability to expense products on a work order for which a credit is expected from the vendor. These are items that were repaired in-house but for which the vendor will issue a credit. For example, a unit is returned from the customer because it failed to perform. You decide to repair the unit in-house and issue a claim to the vendor instead of sending it back to the vendor for repairs. This is accomplished by defining the lines on the work order that are to be expensed (claimed for credit) and then creating a vendor claim with a miscellaneous charge line item for the expected credit amount. The vendor claim may be automatically generated after the work order is completed through Work Order Completions.
To identify those products for which a claim is expected, set the Type field to V (vendor claim expense) in work order entry. Print and Issue the work order as usual. See Work Order Completions for the rest of this procedure. Refer to the ADS BMW/Work Order documentation for examples, posting information, and additional information.
Work Order Completions
This routine was modified to work with vendor claim expense line items. Work Order Completions will accumulate, by vendor by reason code, the cost of each line item on the work order with the Type field set to V. This allows the system to post certain lines on the work order to specified expense accounts and the remaining line items to Inventory thus updating average cost appropriately. Expense line items are excluded from reaveraging of cost. If the work order contains more than one expense reason code and different accounts are referenced, the system will accumulate cost by reason code and post accordingly. Once the work order is completed and if the work order contains a line item with Type set to V, the system will ask if you wish to create a vendor claim for vendor claim expense line items.
•
If you respond No, you will be taken back to the work order completion screen. You must manually create the vendor claim with a miscellaneous charge line item and reference the correct account. Be aware that at this time there are no reports to inform you of expensed items that are not on a vendor claim.
•
If you respond Yes, the Generate Vendor Claim for Expense Items screen is displayed allowing you to generate the vendor claim(s) for the expensed items. This screen displays each vendor a claim needs to be created for, the warehouse it will be created in, and the number of lines that will be placed on the claim. The vendor claim number is displayed after the claim is created.
A regular (type 01) vendor claim is created for the vendor(s). A miscellaneous charge line item is created for each reason code. The amount is the total of all lines with that reason code/vendor. An itemized detail listing of all the work order expense line items associated with the vendor claim and the cost associated of each is in the vendor claim header notes.
See Work Order Entry Line Item Detail for more information on this feature. A complete write-up is available on the system in the ADS BMW/Work Order help menu. This write-up contains examples, posting information, and additional conditions you should be aware of before using this feature.
Release Pick Tickets
•
Sales Orders, Work Orders, and Transfers are three new prompts that allows you to specify what documents to release. Based on how these flags are set (Y/N), the process will release the appropriate documents. The default for these fields is Y.
•
Printer Form Queue is a new prompt that allows you to redirect the print job for sales orders, transfers, and/or work orders to another form queue. These prompts are only accessible if this process is ran for a warehouse and if that warehouse is set up for continuous printing. For example, if in warehouse 10 sales order pick tickets are set up to print continuous but work order and transfer tickets are not, then you will only be prompted for a form queue for sales orders when this process is ran for warehouse 10. If the form queue is left blank, the standard default will apply. Note: if a pick ticket process that meets all the other requirements to prompt for a form queue is not selected, the system will skip the prompt. Using the previous example, if Sales Orders is set to N then you will not be prompted for a form queue.
This field was modified so that if a form queue is entered, the system will check to see if it is set up in Form Queue Maintenance. If it is, it will then verify that the form queue is valid for the specified warehouse. If the form queue is not established in Form Queue Maintenance, the system will print to that form queue without validating the warehouse. See System Maintenance Printer Set-up for more information.

V14 New Features
General Information
General Report Interface
Expanded the fiscal period prompts to provide the ability to enter a valid fiscal period, C for Current, T for Today Current, and -n or +n to evaluate current date and go back or forward n number of periods. For example, for a report ran on 11/10/99, C and T would be interpreted as 99-11, -1 would be interpreted as 99-10, and +1 would be interpreted as 99-12. This change was made for use with the scheduler. However, this modification has also been made to fiscal period prompts outside the grim interface.
Menu Message Inquiries
Sales Rep Queue
•
Sales Rep Message Inquiry was moved under this option from the main menu.
•
Linked Sales Order Inquiry is a new option that displays sales orders that were affected by changes to a linked purchase order. Drill down on this inquiry displays the sales orders affected, the customer, the reason code, and the first line of notes from the buyer. User can review the order, view and send notes to the buyer, and release the order. Note: the order is not placed on hold because it is in this queue. When an order is released, the user is prompted for notes to the buyer. If notes are entered, the order will be sent to the purchase orders buyers queue for the buyer to review. When this inquiry is exited, all orders that were released will be removed from the queue. All orders that have notes to the buyer will be forwarded to that buyers queue. See System Purchasing Option Parameter Maintenance, System Miscellaneous Controls, and Buyer Queue Inquiry for more information.
Buyer Queue Inquiry
Linked Purchase Order Inquiry is a new option that displays purchase orders that were affected by changes to a linked sales order. Drill down on this inquiry displays the purchase order affected, the vendor, the reason code, and the first line of notes from the originator of the sales order. User can review the purchase order, view and send notes to the order originator, and release the purchase order. Note: the purchase order is not placed on hold but may be processed as usual. When a purchase order is released, the user is prompted for notes to the order originator. If notes are entered, the order will be sent to the order originator’s queue for review. When this inquiry is exited, all purchase orders that were released will be removed from the queue. All purchase orders that have notes to the order originator will be forwarded to that person’s queue. See System Purchasing Option Parameter Maintenance, System Miscellaneous Controls, and Sales Rep Inquiry for more information.

Sales Order Entry
•
When Change Order Control is required for the job number entered on the sales order, you are prompted if the order is an Original Order or a Change Order when the header screen is exited. The Original Order option is used to enter the initial job into Sales Order Entry. The Change Order option requires a change order number. You are limited to adding new detail lines and deleting existing lines. Prices and quantities cannot be changed. Sales Order Log Inquiry and Customer Job Inquiry will reference the change order number. See Customer Job Number Maintenance for more information.
If the sales order is re-entered, a prompt is displayed allowing you to choose Original Order, Change Order, Adjustment, Correction, or to Exit the order. The Original Order and Change Order options are described above. The Adjustment option only allows you to add line items with a zero price. This does not change the price of the job but is used for tracking extra costs. The Correction option lets you have full access to all line items. There are no restrictions with this option.
If products are to be released to the job site on a controlled basis rather than when available, the line item pick ticket date or the Tag and Hold order type can be used in Sales Order Entry. The Tag and Hold Release Entry can be used to release a partial or complete quantity of a item.
Certain order types are not allowed when using job change order control. These types are 07, 08, 10, 11, 12, 16, 17. Valid order types for job change order control are 01, 02, 03, 04, 05, 06, 13, and 14.
•
Job Pick Ticket Printing is a new process that will print a picking ticket for shipping the product to the customer. This format can be a standard picking ticket, same as the distribution pick ticket if desired. For the initial installation, a separate printing process was added. This process will do all of the same updates as the distribution pick ticket printing, including creating a release record.
•
Release Confirmation Entry will confirm the release generated by Job Pick Ticket Printing and set the sales order up for internal billing.
•
Internal Invoice Printing and Processing is the same routine used in Order Processing. Note: both order processing invoice printing and job accounting internal invoice printing will select all orders that were released for invoicing. The process being executed is the same; the menu option was duplicated to assist in logical processing flow.
Job controlled sales orders will be selected during the invoice printing and processing cycle. For sales order with job change order control, the processing is very different. Invoicing will relieve inventory for the products delivered to the customer, will remove the completed lines from the open sales order, and write a snapshot of the order to a new file, ORD.CUSTJOB and ORD.CUSTJOB.LINE. These files are for history purposes only. This process will not create an open A/R record for change order control jobs. Internal billing will credit the Inventory G/L account and debit the Jobs Work In Process account. This is the only general ledger postings that will take place during this first billing. The other postings, A/R for example, will be updated during the job billing process.
•
Job Billing Entry is used to bill a job. The amount being released for each line is entered. This is NOT a quantity, rather a lot billing based on completion. For example, if a product is one-third installed, then the billing amount entered will be one-third of the extended price. If the job is AIA controlled, then you can enter the current billing amount and a stored materials amount. Both amounts will be billed to the customer, but will reflect differently on the AIA Invoice Report. Note: you can also adjust the previously billed column if a partial amount was rejected on the previous month’s AIA Billing. An adjusted AIA report can be printed. This must be corrected manually before billing, or the current billing amount will be processed a second time, up to the amount of the line item. See Customer Job Number Maintenance and Job Accounting Reports for more information.
Job Invoicing
•
Job Invoice Preview Report will display all open job invoices to be processed during the Job Invoice Printing and Processing routine.
•
Job Invoice Printing and Processing will select jobs for billing. Any line items that contain a billing amount will be processed. A job invoice will be printed for the billing dollar amount. Quantities are not considered. This billing will credit the Job Work In Process account, debit the Cost of Goods Sold account, credit the Job Sales account, and debit the Account Receivable account. The Cost of Goods Sold account will be debited for the proportion of the cost of the item equal to the proportion of the extension being billed.
An A/R record will be created for this billing. If the job does not have a retainage percent set up in the customer job file, the A/R amount will equal the billing amount. If the job has a retainage percent established, the initial A/R record will be generated for the amount being billed, less the retainage percentage. This first A/R record will have a key of the invoice number followed by a -01 suffix. An additional A/R record will be created for the retainage percentage. The retainage A/R record will have a key of the invoice number followed by a -02 suffix. The retainage A/R record will be generated with a due date of the Retainage Due Date setup in Customer Job Maintenance. Together, both A/R records will equal the amount being billed to the customer. See Customer Job Numbers Maintenance for more information.
•
Job Accounting B Update is a separate Bupdate process for job accounting. This routine will select jobs that were processed through job billing. Each job is evaluated and will update the same Sales Analysis files as Sales Order Processing. Each job represents one unit, so partial job billings will post as a partial unit of one. For example, billing $500 of a $1,000 job will update as .5 units.
Maintenance
Customer Job Numbers > F5-Add Info
•
Change Order Control is a new field that controls access availability in Sales Order Entry. If this flag is set to Y, the user will be prompted for Change Order numbers. A Change Order can only add new line items or delete existing line items. Prices and quantities cannot be changed through a change order. A customer cannot use Mail Invoices, Summary Invoices, or Consolidated Invoices when using a job with change order control. These invoice flags must be turned off or the system will not allow you to set a job up for change order processing. See Job Accounting Sales Order Entry for more information.
•
Retainage Percentage is a new field that defines the percentage of each invoice in a job that the customer will retain until the job is complete. See Job Accounting Job Invoice Printing and Processing for more information.
•
Retainage Due Date is a new field that defines the estimated cutoff date the customer will wait before paying the retainage amount. See Job Accounting Job Invoice Printing and Processing for more information.
•
AIA Invoice is a new field that defines if the job requires an AIA Invoice Report. AIA stands for American Institute of Architects. Enter Y to include this job in AIA printing. Otherwise, enter N. See Job Accounting Job Billing Entry and Job Accounting Reports for more information.
•
Job Line Item Tax Flag Default is a new field that defines the default tax setting for the customer job line items as they are added to the job. Enter Y for Taxable, N for Non-Taxable, or U for Use Tax.
•
F5-Tax Jurisdictions is a new option that allows you to define tax jurisdictions for the job. Tax jurisdictions may only be set up if the Change Order Control flag is set to Y.

Stock Receipts> On The Water Maintenance
On The Water Maintenance is a new receipts method to update original purchase orders with merchandise received. This new feature allows products on open P/Os to be received once the product has arrived and compares the number of received items to the amount of open items on the open P/O. If the received quantity is larger than the open quantity, the following message appears: Quantity Entered Exceeds Total Quantity On Open Purchase Orders By (the overage quantity). Choose Accept Overage or Return to the screen to adjust the quantity. In addition, this also creates the Type 12 and 13 Purchase Orders. with this On The Water Maintenance, additional criteria are also required to be established for this feature to work correctly. Purchase Order Types 12-OTW-Stock and 13-OTW-Direct are new PO Types that were added to V17. A new P flag was added to the On The Water Conversion field in the Additional Option Parameters. In addition, the functionality of the Stock Receipts Entry program has also been updated.
Transfer Shipments Maintenance
The Stock Movements menu has recently been enhanced with a new Transfer Shipments Maintenance option. This process performs the same function as the Suspend option in Transfer Shipments Entry with the exception that packing slips are printed if continuous printing function is currently setup. You can use this as a maintenance entry screen for Transfer Shipments Entry. The quantity shipped, bin location, serial number, lot number, etc. will be updated for each line item currently on the release. Once the data is saved, the transfer release is suspended using a new SAV (Transfer Shipments Maintenance code (which references the Transfer Release #) rather than the SUS code. Therefore, a packing slip printes if you print them continuously.

Several new fields were added to the Unusual Usage & Exception Report GRIM screen. Move Product Usage
A new Move Product Usage feature has recently been added for V17. This menu option allows users the ability of copying usage information from one product to another product. in this feature, you can choose to copy the usage from the original product to the To product or if the usage is to be copied as well as cleared from the original product. Once this process was generated, the usage information that was copied will be reflected in the Override column in the Usage screens in Product Warehouse Maintenance of the To product. Please note that the quantity displayed in the Override column is not the amount of items that were moved as the result of the Move Product Usage process but the combined total usage of both the original product that was moved as well as the usage of the to product. In addition, if the Capture Weekly Usage flag in Inventory Parameters is set to Y, the weekly usage information will also be available to be copied along with the monthly usage. with this new option, a new MOV code has also been created for the Usage Audit Inquiry to indicate that the usage transaction in UAI is a result of a Move Product Usage process.

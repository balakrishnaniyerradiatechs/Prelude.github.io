Line Item Level
F10-ActionNotes
Conversion Notes is a new option that displays DOE conversion notes for a sales order. These are error messages that occurred during the order conversion process.
F10-ActionOrder FuncPriority Code
This is a new Priority Code function that allows the user to change the Priority Code that was defined at the order header level throughout Sales Order Entry. Any line items that are entered once the default was changed will then be updated with that priority code. This new feature is very beneficial for entering multiple flyers or promotional campaigns without having to enter the priority code at each line item level.
The Line Item Detail screen in Sales Order Entry was modified to display a new information feature. KIT (Available) was added, to display how many components are available to build the kit parent.
Release Confirmation Entry
Release Confirmation Entry was enhanced to allow Product Substitutions during the RCE process. At Release Confirmation time, the user has now has the ability to type a substitute product over the original product line entered, without modifying the price of the original product. If the replacement product is not currently established as a valid substitute, the user will then can Create the substitute product or Return to the product line to re-enter another product substitution. Once the product was substituted, the inventory on both the original and substituted products will be updated. in the Order Log Inquiry, the CHG code line is updated for each product substitution that was processed during RCE.
Note* If the Warehousing Module is currently activated, the substitution option will not be available for that warehouse.

The Price Group field now has the capability of recognizing that a customer can be assigned to more than one customer price group to receive the Best Price. The first price group entered will be considered the primary price group. If this group does not reflect the best price, the system will then search for the best price from one of the secondary price groups for this customer. Please note that if this customer is utilizing contract pricing, the current contract price will override all price groups listed.

The user cannot create the job number when using flags S or J so the job number must be created in Customer Job Maintenance for the job number to be entered in Sales Order Entry.

View Maintenance
A View Maintenance menu option was recently been added to ADS. This option allows you to establish the default view for each customer and contact. The system has the ability to setup Major Groups and Product lines to be included or excluded in each category. Major Groups and Product Lines can be setup to be included/excluded in each category. When using major groups and product lines exist in those major groups, the product lines do not have to be entered under the Product Lines section of the screen. Only when major groups are not entered or if product lines are to be excluded from the category are product lines to be entered. Enter the Major Group Codes under the Major Group section of the screen. For the Product Line section of the screen, enter EQ for equal to or NE for not equal to along with the product line number. The EQ represents the product line is equal to the number entered. All products used from the product line number will be displayed in sales analysis in the Contact Management Statistics Inquiry when EQ is used. The NE represents the product line is excluded from this category. All products used from the product line number will not be displayed in sales analysis in the Contact Management Statistics Inquiry for this category when NE is used. The NE should only be used when a major group is entered to exclude a product line from the major group totals

F10-Action
The Copy option was updated with the D Delete Records selection. When using the D option, the system will delete a single customer, single customer price group, multiple customers, multiple customer price groups or a list of customers / customer price groups. Once this screen was exited, the Price Matrix records that meet the criteria entered, will be deleted. This option was added to Price Matrix Maintenance and Retail Price Matrix Maintenance as well. Copy Option: the D delete records will delete price matrices for the customer or customers defined. The entire price matrix record for the customers will be deleted. For the deletion to take place, the same combination of matrices must be setup for the customer or customer price groups. For example, if deleting Price Matrix Type 1 records, customers 981234, 001000, and 000100 must have the same product entered on their matrices as the matrix record where the Copy procedure is performed. Only records with the same product will be deleted. The price matrices where range numbers are entered as part of the matrix must have the same range number for the customers or customer price groups entered in the Copy procedure to be deleted.

The Sort field determines how the sales order records are to be sorted. You can sort by Order Number, Order Date, Required Date, Customer P/O Number, or by Order Type.

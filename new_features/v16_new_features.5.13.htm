Service Order Entry
Display Open Orders Inquiry in SVOE
This module was updated with the option to view open service orders in an Open Order Inquiry in the Service Order Entry screen. The Display Open Orders Inquiry in SVOE Y/N flag is required to be set in the Order Processing ParametersÆ F5-Add InfoÆF5-Add Info screen, to have the inquiry displayed. Any open service orders that currently exist will be displayed once the System ID # was entered. If no open service orders are on file, the inquiry will not display.
Display Quote/Meter Info in SVOE
A new flag was added in the Order Processing ParametersÆ F5-Add InfoÆF5-Add Info screen. This is a Display Quote/Meter Info in SVOE Y/N flag that will determine if the Quote/Meter screen will be displayed for entries or inquiries in Service Order Entry.
Secondary Engineer
The Secondary Engineer flag was created to allow an additional engineer to be entered that could be designated to this service order if the primary engineer is unavailable.
Service Quote Entry
The Secondary Engineer flag was created to allow an additional engineer to be entered that could be designated to this service order if the primary engineer is unavailable at that time.
Service Document Printing
Service Document Printing is a new printing process that was added to ADS. This process enables you to print service documents in batch mode. You also have the option of printing service documents for a warehouse and/or sales order number.
Timecard Process
Service Timecard Entry
A new feature was added to the Service module. The new Service Timecard Entry process was added to allow a mass number of labor hours to be entered for Service and Work Orders as well as for tracking purposes. This process will enable multiple service and work orders to be updated with the number of labor hours worked by a employee without having to manually enter hours for each service or work order.
Timecard Audit Report
The new Timecard Audit Report that was added to the Service module displays all timecard entries that have not yet been processed using the Timecard Batch Update.
Timecard Batch Update
Timecard Batch Update will update all of the unposted timecard entries. If any billable service order hours are entered, the order will be updated with the product number that was established in Burden Maintenance for the burden type code. This is the code that was entered on the timecard entry screen. This process works the same for work orders as well.
Maintenance
Tertiary Engineer
This new Tertiary Engineer field was added to the Systems Maintenance screen as well as in a few other maintenance screens. This field allows a third engineer to be assigned to this customer if the Primary and Secondary Engineers are not available at that time. Therefore enabling service engineer information to be maintained separately from the Salesrep #1 and Salesrep #2.

Salesrep #1 and Salesrep #2 fields were added to the Service Information option of the Customer Ship-To Maintenance screen. The primary sales representative for this customer's service orders can be entered in the Salesrep #1 field. The secondary sales representative can be entered in the Salesrep #2 field. This will provide the ability to maintain which salesmen are responsible for service orders for this customer.
Burden Maintenance> Product Number
A new flag was added to Burden Maintenance. The new Product Number flag allows the user to enter a product number that will be used to update a service order once a burden code type is used in the Service Timecard Entry process. The product number used in this field is typically established in Product Maintenance as a service item.
Reports
A new Preventative Maintenance Due report was added to the Service module. This report will display information pertaining to the preventative maintenance due associated with the systems that were previously established in Systems Maintenance. The user has the ability of generating this report for a Warehouse Number, Customer Number as well as selecting a Due Date.

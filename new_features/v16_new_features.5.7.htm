Accounts Payable
Vendor Invoice Reconciliation
The Pay On Receipt Vendor APPO Mass Update is a new update that was added to the APPO Subsystem. This new feature will perform the Vendor Invoice Reconciliation Entry for stock receipts entered with pay on receipt information. This was created to process only the stock receipts associated with vendors that contains an A Pay On Receipt flag setting.
Claim Reconciliation Entry
The F8-Buyer Hold function key was enhanced with the functionality of placing a vendor claim on buyer hold. Previously, the F8-Buyer Hold function key was displayed but not accessible. This was modified for Version 16. Press F8-Buyer Hold to place a vendor claim on buyer hold in Claim Reconciliation Entry. To release this claim once it was placed on hold, the user will be required to view the claim in the Accounts Payable Discrepancy Inquiry > F5-AP/PO Entry option. After the claim was reviewed and F2-Save function key was selected, the message Do You Wish To Release This Receiver For Accounts Payable Invoicing? will be displayed, if Yes is selected, the claim will then be released.
Void Check Batch Entry
A new Void Check Batch Entry option was added for Version 16. This new feature allows the user the ability to void multiple checks based on a range of check numbers for a bank code. Therefore preventing the user from having to void each check.
Accounts Payable Inquiry
The Accounts Payable Inquiry F9-Notes screen was updated with a new option. The user now has the ability to enter or modify notes in the notes screen as well as view them. Previously, the user could only view notes that had previously been added.
Program Changes
Previously of ADS, if an insufficient quantity existed in the FIFO layer and an AP/PO Cost Adjustment was made to a stock receipt, no adjusting entry was made. A Program change was made for Versions 16 and higher for the Cost for Sales entry to be included in the G/L postings for the re-costing amount.

Accounts Receivable
Inquiries
Updated Administrative Invoice Release Functionality
This inquiry was updated to work with the new notification method in User Id Maintenance and AIH mgr id. The notification method is used when the order is moved to another manager id queue (*Action > Alternate Mgr) & that manager id has notification method setup. The notification method is also checked when F8-Release function is used to update the user id queue when released from hold.
Updated C Code
with a new Credit Limit Amount field added to User ID Maintenance, the Administrative Invoice Release Inquiry was updated. The inquiry will now reference any Sales Orders, Credit Memos and RGAs that were placed on Administrative Invoice Hold due to exceeding the credit limit amount inCredit Limit Amount. The C code will be updated to reference the reason this was placed on hold.
New Salesrep One Field
The A/R Credit Comments Inquiry was updated to now display the Salesrep One field.
Reports
New Totals Only GRIM Screen
The Totals Only option has now been placed on the GRIM screen. Previously, you had to choose the Totals Only option immediately before generating the report.
New Suppress Customer Detail Field
The Aged Accounts Receivable-Month End report was enhanced with a new Suppress Customer Detail field. This Y/N field allows lets you determine if customer information such as Contact Name, Phone Number and A/R Terms. If this flag is set to Y to Suppress Customer Detail, the information will not be displayed. If this flag is set to N, the information will not be suppressed therefore will be included on the report.
Old Month-End Report
The original version of the Month-End report was modified for V17+. The original Month-End report before any modifications is now available again for V18+. The A/R Aging Reports menu has both Month-End reports.
New Cost of Sales G/L Accounts
Cost of Sales Adj
The Product Line Maintenance screen was updated with a new Cost of Sales Adj G/L account. This account override was created for use with the Cost of Sales Adjustments at the product line level. This account will be posted to during the re-costing of the receiver using the F7-Recost function in Vendor Invoice Reconciliation Entry. This account is used to reflect the change in cost of sales for the receiver. In V17, the Cost of Sales Adj account in the General Ledger Table was used for the re-costing process.
Cost of Sales Adj-Direct
The Product Line Maintenance screen was updated with a new Cost of Sales Adj-Direct G/L account. This account override was created for use with the Cost of Sales Adjustments at the product line level. This account will be posted to during the re-costing of the receiver linked to a direct ship sales order by using the F7-Recost function in Vendor Invoice Reconciliation Entry, since the direct ship sales order linked to the purchase order has already been invoiced. This account is used to reflect the change in cost of sales for the linked sales order. In V17, the Cost of Sales Adj-Direct account in the General Ledger Table was used to re-cost.
Cost of Sales Adj-Indirect
The Product Line Maintenance screen was updated with a new Cost of Sales Adj-Indirect G/L account. This account override was created for use with the Cost of Sales Adjustments at the product line level. This account will be posted to during the re-costing of the receiver linked to a sales order by using the F7-Recost function in Vendor Invoice Reconciliation Entry, since the sales order linked to the purchase order has already been invoiced. This account is used to reflect the change in cost of sales for the linked sales order. In V17, the Cost of Sales Adj-Indirect account in the General Ledger Table was used for the re-costing process.
Printing/Faxing/Emailing
New GRIM Screen For Credit Collections Nightly Process
The Credit Collections Nightly Process was updated with the standard GRIM interface.
Updated Continuous Printer Functionality
Now when a sales order that is linked to a work order is released from credit hold in Credit Release Inquiry, the system now checks the linked work order to see if released (Auto Release flag set to Y in BMW/Work Order Parameters) and continuous printing is used. If the criteria are met, the system sends the work order pick ticket to the continuous printer. Previously, you had to run Release Pick Tickets to find the released work orders.
Updated Customer Statement Printing Functionality
The Customer Statement Printing process checks Statement Fax when faxing customer statements. The Statement Fax field is in the F6-A/R screen in Customer Maintenance. If this flag is blank, the fax number on the main Customer Maintenance screen is used. Note: to use this feature you must request a USER-FORMS change.

Company Maintenance
F5-Add Info> G/L Rep Cst Default> C
The functionality of the C flag setting for this option was modified. Previously, the system would update the line item cost for the credit memo with the replacement cost if the average cost was $0.00. Therefore, if the replacement cost was also $0.00, the line item entered on the credit memo would then be updated with a $0.00 cost as well. This function has since been altered to provide the ability of allocating a product cost to line items that contain a $0.00 average and replacement cost. The system will now update the line item with the product cost entered at the time of entry when creating a credit memo regardless if the average and replacement costs are currently at $0.00.
F6-Modules> Marketing> Update Prospect Email
The Marketing options screen was enhanced with a new Update Prospect Email flag. This is a Y/N flag that determines if the prospect e-mail address will be updated with the new e-mail address if it is changed in the Customer file. If this flag is set to Y, and the e-mail address is modified, the prospect file will then be updated with the e-mail change to match what is currently listed in the customer file. Therefore, the address in both the customer and prospect file would be the same. Select N to not update the prospect file with any e-mail address modifications made in the customer file.
F6-Modules> Vertex Sales Tax> Use Taxable Status From Ship-To?
A program change has recently occurred for the Use Taxable Status From Ship-To? flag in the Vertex Sales Tax option for V15-17. From this point forward, if the Use Taxable Status From Ship-To? flag is set to Y, the system will then check the ship-to record for the Taxable flag setting as well as the Exempt Number for that sales order. If this flag is set to N, the system will automatically use the Taxable flag setting and Exempt Number information from the customer record.

F5-Add Info> F5-Add Info> Pick Ticket Minimum Dollar
The Order Processing Parameters menu was updated with a new Pick Ticket Minimum Dollar flag. This new flag was added to provide the ability to establish a minimum dollar amount to be required for each warehouse prior to releasing a pick ticket for processing. The Pick Ticket Minimum Dollar flag contains logic that will check the dollar amount for each pickable line item assigned to the same warehouse, calculate the pickable total and compare it to the minimum dollar amount previously entered for this flag. If the total amount of the pickable items is less than the amount established in the Order Processing parameters, a pick ticket will not be released for picking for this warehouse. If the pickable total meets or exceeds the pre-determined amount, the pick ticket release will continue to be processed. The logic for this option will in fact calculate the totals by each warehouse, however a Minimum dollar amount cannot be established per warehouse. The minimum dollar amount instituted will be used as the amount required to be met for each warehouse that pertains to the order. Therefore if multiple warehouses are used, each warehouse must meet the minimum dollar amount to release the pick ticket. If a sales order contains multiple warehouse items and only one of the warehouse pickable totals meet the minimum dollar amount a pick ticket for only this warehouse will be released. Note: If using both features the Pick Ticket Minimum Dollar as well as the Pick Ticket Minimum Percent, only one of these conditions is required to be met prior to releasing the pick ticket for this order.
F5-Add Info> F5-Add Info> Pick Ticket Minimum Percent
with the Pick Ticket Minimum Dollar flag, a Pick Ticket Minimum Percent flag has also been added to allow a percentage to be required for sales orders prior to releasing pick tickets for processing. The Pick Ticket Minimum Percent flag contains logic that will check the dollar amount for each pickable line item assigned to the same warehouse, calculate the pickable total and divide it by the entire amount total of the line items allocated to the same warehouse. This calculation will determine if the percentage pickable amount meets the percentage amount previously established in the Order Processing Parameters menu. If the total amount of the pickable items is less than the percentage amount established, a pick ticket will not be released for picking for this warehouse. If the pickable total meets or exceeds the pre-determined percentage amount, the pick ticket release will continue to be processed. The logic for this option will in fact calculate the totals by each warehouse to determine if pick tickets will be printed, however a Minimum percent amount cannot be established per warehouse. The minimum percent amount instituted will be used as the amount required to be met for each warehouse that pertains to the order. Therefore if multiple warehouses are used, each warehouse must meet the minimum percentage amount to release the pick ticket. If a sales order contains multiple warehouse items and only one of the warehouse pickable totals meet the minimum dollar amount a pick ticket for only this warehouse will be released. Note: If using both features the Pick Ticket Minimum Percent as well as the Pick Ticket Minimum Dollar, only one of these conditions must be met to release a pick ticket.

Default Consign Accum Flag
A new A option was created for the Default Consing Accum flag. This A setting was added to allow products to always be accumulated on a consignment order for a customer, regardless of any price differences. When creating a consignment order for a customer, the product numbers will be accumulated even though a price variance might exist for this product. The price will be updated on the type 08 consignment order each time the product is entered on an 07 consignment order and invoiced. The price of the accumulated products will be updated based on the allocated price of the last product entered and invoiced.

Purges
Order Processing> Quotations
A new Quotations option was added to the Purge Utility Menu. This new process will purge open quotes that were converted and/or deleted with a disposition code without printing the Quotation Disposition Reports. This process can be used for converted quotes, lost quotes (quotes that were deleted with a disposition code) or for both converted and lost quotes. Once this process is performed, each quote will be removed from the quote file and updated into the quote history file. The Quotation Log Inquiry will then be updated with the PUR code referencing that a purge transaction has taken place however, the Purge Log Report will not be updated for this purge. In addition, the Quote Order History Inquiry Quote History Product Inquiry. This purge routine is identical to the process performed following the printing of the Quotation Disposition Summary Report. The GRIM (General Interface Report Maintenance) screen that will be displayed is a duplicate of the report screen however, only the purge process will be performed.

This purge was added to allow users to purge vendor master records from a saved-list or criteria that currently do not contain an A/P Balance or Open Purchase Orders.

Stock Receipts…
Stock Receipts Entry
A new barcode printing option was added to Stock Receipts Entry as well as to Radio Frequency Stock Receipts Entry. ADS now has the ability to print barcode labels at the point of product receipt for vendor items that are not bar-coded. It is not necessary for the Warehousing Module to be activated to use this option, however, there are flags in Warehouse, Vendor, Product Vendor and Product Maintenances, that are required to be setup.
Direct Ship Receipt Entry
A new Direct Ship Receipt Entry option was added to the Stock Receipts menu. This menu option allows for the receipt of a drop ship purchase order, therefore the invoice for the customer then can be generated. Once a direct ship receipt is processed, the sales order linked to the direct ship receipt is updated with a confirmed status and setup for invoicing. Previously, the purchase order was not received and updated for invoicing until Vendor Invoice Reconciliation Entry was processed. In utilizing this option, the Direct Ship Entry process will create a SR module posting updating inventory and accounts payable not invoiced accounts. Vendor Invoice Reconciliation Entry will create an AP module posting reducing the not invoiced accounts and update inventory direct shipped and accounts payable accounts accordingly.

Nonnettable File Update Entry
Nonnettable File Update Entry is a new option primarily used with a carousel interface to update items in the non-nettable bucket. In other words, it takes the inventory out of the non-nettable bucket and puts it into the available bucket. You are prompted for a product number and a warehouse number. All purchase order receivers and transfer receivers for the combination are displayed along with the quantity received, the receipt date, and the receipt time. Selecting F4-Del for a product/warehouse combination will remove all receipts into the non-nettable bucket, update the Available quantity, and run allocations for the item. Note: at this time this procedure updates for the full quantity in the non-nettable bucket. You do not have the ability to update for only one receiver. See also System Warehouse Maintenance and Inventory Miscellaneous Reports.
Maintenance…
Product Maintenance…
•
The Product Maintenance screen was updated to include a new Availability Type flag. This flag determines the availability of the product for order entry. Select Y if this product is always available for sale regardless of the available quantity and will always pick. Select N if this product will pick what is available and will backorder the remaining quantity. Select B if this product will always backorder and will never be allocated to. The user must manually allocate inventory to this product. This type product usually does not involve picking and shipping. Select D if this product will always be backordered and default to a D supply code if no available inventory exists for the product Sales orders and Quotes are defaulted to the product’s supply code in the Line Item Detail screen when the product number is entered.
•
The Product Maintenance screen was updated with a new F-9 Selection option. This new feature enables the user to review the last list in memory of a product search. Once a user completes the search for a product using the F3 key, the F-9 option will allow you to retrieve the same list without having to perform the entire search.
•
The Substitutes option has recently been updated with a new Auto Sub Prompt in SOE/WOE? flag. If using Product Substitution type A or P, this flag determines if the user should be prompted, prior to substituting products in Sales Order and Work Order Entry. If the flag is set to Y, the system will not prompt the user to answer the Substitute Product? message when the requirements are met for the substituted product. If this flag is set to N, the user will then be prompted prior to substituting the products that meet the product substitution requirements. If Product Substitution types B or S are used, this flag will not apply.
Quick Product Maintenance
Quick Product Maintenance is a new maintenance option that was added to allow the quick setup of products for multiple warehouses when only basic information is required. When setting up a warehouse record for a product, the user now cancan enter multiple warehouse numbers or enter ALL to create warehouse records for all warehouse address records defined on the system. The criteria that is entered for the warehouses is updated to those warehouse records The WAREHOUSE and WHSE.STAT files are updated. The PROD.STAT file has the warehouse list updated for the warehouse numbers entered.
Product Characteristics Codes
The Product Characteristics Code Maintenance routine has added the new fields, Description, Mandatory, History, and Add Info to better use the Product Characteristics codes for products. In addition, the Sales and Quotation Order and Order History Inquiries are updated as well to reflect the descriptions entered on the Product Characteristics screen.

Product / Serial Number
•
The Serial Number Inquiry was updated to reflect the new serial number of an item that was changed in the Serial Number Change Maintenance routine. This inquiry displays the new serial number using the original serial number's equipment id number and rental unit number.
•
This inquiry was updated with a CHG code to be displayed to reference the serial number change made in Serial Number Change Maintenance. Both the original and new serial numbers are updated. The Ref# column for the new serial number is updated with the original serial number, if viewing the original serial number, the Ref# column will display the new serial number.
Vendor Claim Log Inquiry
The Vendor Claim Log Inquiry has added a new drill down feature utilizing the F-5 Review option in the inquiry screen. This allows the user to view each transaction that has taken place during the vendor claim process. In addition, a Vendor Claim Log Notes screen can also be displayed to show additional information regarding any transactions displaying the CHG code. For the detailed information on the CHG transaction updates to be viewed, the fields that are to be tracked, must be established in the APPO Change Log Setup and the APPO Line Change Log Setup in Log Inquiry Setup.
Bin Movement Inquiry
The Bin Movement Inquiry was updated with a new F9-Goto Line option. This new function key allows the user the capability of going directly to a line item. This function can be used by entering the line number to be accessed by entering the line item number in the F9-Goto Line field.

Misc
TAI Upd Of Service Items
This is a new flag that was added to Inventory Parameters to determine if service items are available to be reviewed in Transaction Audit Inquiry. Select Y to allow the transaction audit inquiry to be updated with service items. Select N to not allow the transaction audit inquiry to be updated with service items.
Program Changes
The Stock Receipts Entry, Inventory Adjustments Entry and the Credit Memo Entry screens were affected by a program change that was put in place for both Version 15 and 16. This change was made to disallow a previously entered warranty tag number for a product to be duplicated for the same product.

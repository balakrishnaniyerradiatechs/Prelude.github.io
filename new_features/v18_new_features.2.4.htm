Inventory
New Cycle Count Options
These new cycle count options only pertain to the terminal version of Product Cycle Count Request Entry.
•
C - enter a zone for cycle counts. The system locates products with a primary bin in that zone. Those products are added to the second level queue. This is a similar to the by product line or by primary vendor process.
•
B - enter bin locations for cycle counts. When selecting the B type, a Cycle Count Request Entry By Bin screen allows you to enter the bin locations to cycle count. The bins are validated against BIN.LOC.
•
M - manually enter criteria for cycle counts. When selecting the M type, a Manual Product Cycle Count Request Options GRIM screen allows you to request the cycle count. The product number is not validated. If you enter an invalid product or wildcard, the data is not updated. The data is from the WHSE.STAT file.
Updated Rebuild Product Classifications Functionality
The Rebuild Product Classifications process was updated with a new #/Months Classification field added to the Purchasing Parameters. (Referenced in the Systems New Features list) This product classification process will now check the #/Months Classification field for product classification. The system will update the inventory class for the products based on the number entered in the #/Months Classification field. If the new field is left blank, the default will be 12 and the process will remain the same as previous versions.
Tracking
New Sort Sequence
The Sort Sequence field in the Transaction Audit inquiries was updated with a new F option. Selecting F will display a Transaction Audit Inquiry Filter Options screen. This allows a range of dates as well as transaction codes to be entered to retrieve results. Based on the selection criteria entered, the report will display a range of entries. The system will automatically default the Sort Sequence flag setting to F. This default is fromTransaction Audit Inquiry in Default Parameters Maintenance.
New Ent-Date and Ent-Time Columns
The Customer, Pricing, Product, Vendor, and Contract Audit reports have new Ent-Date and Ent-Time columns that show the date and time the data was changed.
Transfer Entry and Shipment
New Substitue Functions
A new F6 function key was added to the Line Item Detail screen to allow you to view and select substitute products. A new SUB code is displayed in the Line Item Detail screen in Transfer Entry to reference the item has a substitute product.
Update to Transfer Shipments Maintenance Functionality
The functionality of the Transfer Shipments Maintenance was modified to now prompt for the option of printing a packing list. Select Y to send the packing list to the continuous printer. Select N to not print the packing list.
Note: The continuous printer must be setup to print transfer packing lists to use this option.
New Dup Tran Hist Option for Transfer Entry
A new Dup Tran Hist option was added to allow users the capability of duplicating historical stock transfers. This process was created for transfers that have already been processed in Transfer Receipts Entry. However, if it is necessary to duplicate transfers that have not yet been processed in Transfer Receipts Entry, the Dup Transfer located on the same menu is available for this function. In addition, Ship Via Codes, Instructions, Unusual Usage flag as well as the F7-Ship-To address fields in the header screen will be copied from the original transfer. The User ID will also be updated with the User ID that has performed the duplication.
Note: Warranty transfers cannot be duplicated.
Automatic Substitutes in Stock Transfers
Previously, the Auto Sub Prompt in SOE/WOE/TE? flag in Product Maintenance was referenced as Auto Sub Prompt in SOE/WOE? This flag is now referred to as Auto Sub Prompt SOE/WOE/TE?. In addition, a new A setting is now available. This setting allows for automatic substitution. If only using Sales Order Entry and Work Order Entry for automatic substitution, set flag to Y. If no automatic substitution is required, set the flag to N.
Stock, Transfer, and Direct Receipts
New Recv Amount Field for Receipt Entry screens
A new Recv Amount field was added to several of the Receipt Entry screens. These Receipt Entry processes were updated with this new feature to display a running total of the received amount. Once each line item is processed, the Recv Amount field will be updated displaying the current dollar amount received. The running total will be updated if using the Line Item or Exception option of processing the stock receipt.
New GRIM Screen For Batch Updates
The Batch Update Procedure was updated with the standard GRIM interface. Due to upgrade to this process, the message Do You Wish To Continue… previously displayed on the non-GRIM screen will not appear.
Printing Barcode Labels
User-Forms Change: To use this new feature, the USER-FORMS programs need to be modified or a CUSTOM modification is required to include the necessary logic for this process. The Receiving Documents Report was enhanced with the option of printing barcode labels for each product and purchase order if the barcode label print flags are set accordingly. If emailing or faxing the report, the barcode labels for each product and purchase order number will not print.
Note: for the barcode labels to print on the report, Stock Receipts Printer Type must be set to the appropriate printer type in the Warehouse Maintenance> F7-RTS Maint. screen.
New Doc B/O Field
The Receiving Documents Report was updated with a new Doc B/O field on the report. This field displays the total backordered quantity for a item for sales orders, work orders and stock transfers. The Doc B/O prints below the Qty-Opn column on the report.
Note: Backordered quantities of an item with backorders or links are excluded from the report.
New Inspection Messaging option
A new messaging option is now available. If Prelude Notify Messaging option is the notification method being used and a stock receipt is placed on quality inspection hold, the system displays a popup message. If email notification is set for the user id of the quality inspection manager, an e-mail I sent to the manager with reference to the stock receipt number placed on hold.
New Rental Return ‘H Hold Status
A new H option for the Rental Return Status (Action Bar > Addtl Opts> Rental Information) field in Product Maintenance was created. If a rental serialized item is setup with Rental Return Status of H and the rental item is returned (Rental Complete must be set to Y), the system sets the status of the serial number to H (hold) in the PROD.SN file.
Month End
The Month End Closing Date field was modified for the GRIM screen. The Date field will now default to the last day of the previous or current fiscal period, based on the current system date. If the system date is between 1 and 25, the Date field automatically defaults to the last day of the previous fiscal period. However, if the date is between 26 and the end of the month, the Date field will then default to the last day of the current fiscal period.
The ICS End of Month Calendar Close process was updated with a new #/Months Classification field added to the Purchasing Parameters. (Referenced in the Systems New Features list) This end of month process will now check the #/Months Classification field for product classification. The system will update the inventory class for the products based on the number entered in the #/Months Classification field. If the new field is left blank, the default will be 12 and the process will remain the same as previous versions.
Vendor Claims
Update To F5-Vendor Claim Reprint Functionality
New F6-Updated Vendor Claim Reprint Function Key
You can reprint a vendor claim before updating it. Previously, you could only print vendor claims once they were updated. To print vendor claims after they are updated, press F6-Updated Vendor Claim Reprint. You must indicate with vendor claim release (-#) you want to preprint.
Note: Document Options defaults from Vendor Claims Doc Option in Vendor Maintenance.
New Vendor Field For The Nonreturnable Flag
The Nonreturnable Flag was updated with a new Vendor field. This is a Y/N flag that determines if a product will be allowed to be returned directly from stock. Set this flag to N, if products are not allowed to be returned directly from stock. to further enhance this new feature, a WARNING message has also been created for when the flag is set to N. If a user attempts to return a product back to the vendor from stock in Vendor Claim Entry, the message This Item Is Not Returnable from Stock! will be displayed, disallowing this product number to be entered. If this flag is set to Y or left null, the user will be permitted to enter the product number on the vendor claim returning the product back to the vendor from stock.
Note: This flag does not pertain to warranty inventory or warranty tag returns.
Updated Equipment Number Functionality
When a non-rental serialized item is entered on a vendor claim and the vendor claim was updated, both the equipment and fixed asset record for that serial number are updated. The disposal information will be updated in Equipment Maintenance.
Price Tape Update
New Skip P/N if Vendor Doesn’t Match Field
The Price Tape Update process was enhanced with a new Skip P/N if Vendor Doesn't Match field. This is a Y/N option. This new feature was added to allow users to determine if the product number should be skipped if the product warehouse vendor (primary vendor in Product Vendor Maintenance) does not match the vendor in the Price Tape setup screen. If this flag is set to Y, the Product Number will be skipped, if the product warehouse vendor does not match the vendor that is currently in the Price Tape. If this flag is set to N, the product number will not be skipped.
Updated Price Tape Update Functionality
The functionality of the Price Tape Update process was updated. The update will now recomputed price and cost changes automatically when the Price Tape Update process is performed for the price and cost fields that have a Basis.
Note: Once upgraded to version 18, the Recompute Prices procedure is not required because the prices and costs recalculate automatically with Price Tape Update.
New Clear Product Keyword Xref utility
The new utility clears the additional keywords in the PRODUCT & NS. PRODUCT files in preparation for performing the Price Tape update. When this option is selected, the following message is displayed: This Rebuild will remove all additional Keywords form the Product and NS Product cross reference file in preparation of new additional keywords being added by the Price Update Tape process. Once Continue is selected, the GRIM selection screen for the Clear Product Keyword Xref is displayed. In the Product Line prompt, enter the product line(s). In the Product Prc Group prompt, enter the product price group(s).
Prices and Costs
New Future Option in Product/Vendor/ Region Maintenance
The Product/Vendor/Region Maintenance option has updated with a new Future option in the Action menu to establish future prices for products. The information entered in the Cost Region field will determine which records will be updated with the future prices. If the Cost Region field is left null, future prices will be updated to the product header. If a cost region is exists for the product number, each product warehouse record associated with that cost region will be updated with the future prices.
Notes Option For Cost Adjustment Entry
The Cost Adjustment Entry screen was updated with a new Notes option added to the *Action menu. Users will now can enter, modify or view any notes that were created pertaining to the cost adjustment entry. In addition, the Notes function also includes date and time stamp information. The functionality of the Adjustment Number in the Average Cost Adjustment Inquiry has also been modified to incorporate this new feature. If notes currently exist for an adjustment number, an * (asterisk) will then be displayed before the number to reference that this adjustment contains notes. Note: These notes are for internal purposes only, therefore will no be printed on any cost adjustment reports.
Note: If allowing users to edit existing notes using the F6-Edit Notes option of this feature, the Notes Edit Authorizations in the User Options menu of User ID Maintenance must be set to ALL for the user.
Update To Product Inquiries Functionality
Product Inquiry, Product Warehouse Inquiry, and Product Vendor Inquiry now check the Display Costs flag in User Id Maintenance to determine if the cost fields should appear. If Display Costs is N, the following will apply:
•
Product Inquiry – The Replacement Cost, Standard Cost, Cost for Sales, and Direct Ship Cost fields are blank on the F7-Pricing screen, the F7-Pricing > Action > Future screen and the F7-Pricing >Action > Previous.
•
Product Warehouse Inquiry – The Replacement Cost, Standard Cost, Cost for Sales, and Direct Ship Cost fields are blank on the F7-Price/Cost screen, the F7-Price/Cost > Action > Future screen and the F7-Price/Cost > Action > Previous.
•
Product Vendor Inquiry – Unit Cost, New Unit Cost, Direct Ship Cost, and Last Purchase Cost fields are blank.

This new Ship Via Code will allow you to enter the ship via code on the OTW purchase order created.
Update To F2-Save & Update Screen
Once the On The Water Maintenance entry is processed using F2-Save & Update, the system updated the print status for the OTW purchase order as Y. If changes are made to the purchase order in POE and selects Print, the message Purchase Order Has Already Been Printed is displayed.
New Bin Product OTW Onhand Report
The Bin Product OTW Onhand Report is a new report added to provide information pertaining to product bin onhand for on the water bins for a warehouse, bin number, as well as bin and storage types. This report is identical to the Bin Product Onhand Report that currently exists, with the exception of two additional Cost and Ext Cost columns included on the new report. The Bin Product OTW Onhand Report will display Bin Location, Product Number, Description, Bin Onhand, Bin Committed, Bin Available, Unit of Measure, Lot Numbers, Product Classifications, Costs and Extended Costs. This report also has a P Product List Sort option which allows you to sort by product number for each bin location. This field can be left null to sort by product number.

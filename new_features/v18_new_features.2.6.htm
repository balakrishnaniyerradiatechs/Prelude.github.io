General Ledger
Financial Reports
Updated Financial Report Printing Functionality
The Financial Report Printing process now uses historical currency exchange rates if they exist.
New Financial Report Manager Option
The General Ledger menu was updated with a new Financial Report Manager option. This menu provides maintenance routines for designing financial reports such as balance sheets and income statements using the data maintained in the general ledger files.
•
Financial Report Master Maintenance – Allows financial reports such as balance sheets to be created.
•
Financial Report Group Maintenance – Allows you to create financial report groups such as assets, liabilities, income, and expenses.
•
Financial Report Column Maintenance – Allows you to edit financial report columns and headings.
•
Financial Report Printing – Allows the financial reports entered to be printed. The Report parameters are not needed on this screen if the print parameters were defined for the report in Financial Report Master Maintenance > F8-Print Param screen.
G/L Setup
New G/L Account
A new A/P Tax Amount account was added to the General Ledger Table. This new account is used with G/L postings for amounts entered in the Tax Amount field in Vendor Invoice Reconciliation Entry or A/P Manual Invoice Entry.
New Excel Output Option
The Detail General Ledger Inquiry was updated with a new F7-Detail Output – Excel option. The Excel option L was added to the list of available output options for sending the Detail Report to a file in Excel.
Updated Fiscal Period Maintenance Functionality
The Fiscal Period Maintenance functionality was modified for versions 18 and higher. When the Open? flag setting is changed from Y flag setting for a fiscal period, the system now prompts with a message to update historical currency exchange rates. If Y is selected, the Month End Currency Rate Update screen is displayed. The user can accept the default currency exchange rates or override the values. Once the rates are entered, the system updates these rates for the fiscal period in Currency Type Maintenance > F5-History. If No is selected, the history currency exchange rates for the fiscal period are not updated.
New Eval by Fisc Period Flag
A new Eval by Fisc Period flag was added to the Depreciation Maintenance screen. This flag determines if the depreciation for the fixed asset is calculated using the fiscal period table to account for the date the asset was purchased. Set this flag to N to not account for the fiscal period the fixed asset was purchased. The depreciation is calculated using the standard 12 months for the percentage in the table for the depreciation method. Set this flag to Y to account for the fiscal period the fixed asset was purchased. The depreciation is calculated using the number of months remaining in the year from the time the asset was purchased for the percentage in the table for the depreciation method. If using multiple fiscal periods, the system will check for the fiscal period in Multi-Fisc Period Table Maintenance to determine the depreciation year and percentage from the table in Depreciation Maintenance.
New Function Keys
Some of the General Ledger Inquiries were updated with new function keys to allow you to view blocks of records. The number of records is in the Records To Block field in the GUI Options of Company Maintenance.
•
sF5-Prev Group (Shift-F5) shows the previous block of records. The number entered in the Records To Block field in Company Maintenance determines the number of records that display on the inquiry at a time.
•
sF6-Next Group (Shift-F6) shows the next block of records. The number entered in the Records To Block field in Company Maintenance determines the number of records that display on the inquiry screen at a time.
New Sort Flag
A new Expanded G/L Inquiry Sort flag was added to the Default Parameters screen. This new option allows a default sort option to be entered for the GRIM interface screen for Expanded General Ledger Inquiry and Expanded Trial Balance Inquriy. The Sort options available are:
•
AC – Ascending Credit Amount
•
AD – Ascending Debit Amount
•
AP – Ascending Posting Date
•
DC – Descending Credit Amount
•
DD – Descending Debit Amount
•
DP – Descending Posting Date
•
M – Module
New Expanded G/L Inquiry
This new inquiry includes G/L posting information for the G/L prefixes/account entered. The Detail General Ledger Inquiry located on the same menu only displays the G/L posting information for a G/L prefix. This inquiry allows each G/L prefix to be displayed on the screen that is included in the GRIM criteria with a drilldown available for G/L posting information. For the GUI version, the F5-Review > F6-Report function key will not be highlighted for all module codes. The report option is only available for AC, AM, AP, AV, and OE module codes.
Inventory Balancing
New Menu Option
This is a new menu option that contains several reports to be used with inventory balancing. These reports will facilitate balancing most of the inventory accounts, e.g. stock, linked, warranty. In addition, there are reports to help balance the accrual accounts as well, e.g. inventory received but not invoiced, liked inventory received not invoiced.
Updated Balance Save Functionality
The Month-End Inventory Balance Save process was updated. This procedure now saves warranty inventory (returned and scrapped) and consigned inventory values. The following attributes are updated in the WHSE.STAT file for warranty and consigned inventory.
•
Attribute 140 – Consigned Inventory
•
Attribute 141 – Warranty Inventory (Returned)
•
Attribute 142 – Warranty Inventory (Scrapped)
New Warehouse Number Field
The GRIM screen in the Month-End Inventory Balance Save was enhanced with a new Warehouse Number field. This new field allows you to select one or more warehouse(s) that are to be included when generating this process. This field can also be left blank for all warehouses to be included. Previously, this field was not available therefore, automatically including all the warehouses when running this process.
Chart of Accounts
Updated Chart of Accounts Menu Name and Functionality
The Chart of Accounts Maintenance option was renamed Chart of Accounts-Master. In addition to the name change, the functionality has also been updated. This maintenance now allows a chart of account to be updated for all prefixes. use this maintenance option to make changes to the chart of account. The chart of account can be deleted if no balances or budgets exist for the chart of account.
Updated Financial Report Group Requirement
For V18+, Fin Rpt Grp is in Chart of Accounts Master so the account is included in the financial reports.
Chart of Accounts Balance Report
•
The Chart of Accounts Balance Report was updated with a new Y/N Suppress Zero Balance Accts? field. Select Y to suppress the chart of accounts prefix that contains zero balance. Select N to not suppress the chart of accounts prefix that contains a zero balance.
•
A new Location field was added to the GRIM screen of the Chart of Accounts Balance Report. This field allows you to enter the location(s) of which information is to be retrieved for the report.
Recurring Journal Entries
New Project ID Option
The Recurring Journal Entry process was updated with a new Project ID option. Enter the project id to be tracked for the G/L account number entered. Once the project id is entered and the screen is processed, the system will update the Project General Ledger Inquiry with the project id and debit/credit amount for the G/L account number when the Recurring Journal Post procedure is performed for the recurring journal entry. If a Reverse Date is entered for the journal entry, the system will NOT prompt for a project id for the account number in the Recurring Journal Entry screen since the G/L account be will reversed out in the following month. The user must manually process all fields on the Recurring Journal Entry screen if the recurring journal entry has not been posted yet that pertain to the G/L account setup for project id tracking for the system to display the project id prompt after the debit/credit amount fields. If the Copy procedure (*Action menu) is performed for the recurring journal entry, the Project ID is copied to the new recurring journal entry created from the copy process. Once the user performs the Recurring Journal Post procedure, the Project ID is updated for the original recurring journal entry as well as the new recurring journal entry created from the copy process.
New Copy Option
The Recurring Journal Entry screen was updated with a new Copy option. This feature will create a copy of the original journal entry using the next available recurring JE number. If a Reverse Date is in the original recurring journal entry to be copied, the Reverse Date will be cleared for the new recurring journal entry created from the copy process. If the recurring journal entry affects multiple fiscal periods and requires a reversal entry for each month, the user will be required to bring up the journal entry in Recurring Journal Entry after each month's posting and update the reversal date prior to the next month's posting. If a Batch Number exists for the original recurring entry, the Batch Number field will be cleared for the new entry created from the copy process. The user would be required to re-enter the batch number if using the same batch number as the original journal entry. If a Project ID exists for the original recurring journal entry, the Project ID is updated for the new recurring journal entry during the copy process. Once post the Recurring Journal Entry, the Project ID is updated for the original recurring journal entry as well as the new one created.
Updated Recurring Journal Post Functionality
The Recurring Journal Post process was updated for versions 18 and higher. When this process is performed and the recurring journal entry contains a project id, the Project General Ledger Inquiry for the account number is updated to reflect the project id.
Updated Project General Ledger Inquiry Functionality
The Project General Ledger Inquiry was updated for versions 18 and higher. This inquiry now includes debit and credit information for G/L accounts that have Project ID Required flag set to Y. The JE code is updated for Journal Entry and Recurring Journal Entry.

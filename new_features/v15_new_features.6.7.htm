Company Maintenance
•
F-5 Add. Info was updated with a new I option flag in the Freight Allocation-P/O field. The I flag was added to disallow the performance of freight allocation for direct ship purchase orders. All other types of purchase orders will perform freight allocation. If freight is manually entered in AP/PO, the Freight Paid account will be used for posting that freight charge. The Y and N flags will continue to use the same logic as before.

Contact Management is a new Y/N flag that was added to allow you to determine if contact management options should be accessible in ADS.
Warehouse Maintenance
The Warehouse Maintenance screen has recently been updated with two new flags. The
•
Bin Minimum Percent: This flag allows you to enter the bin minimum percent to be used in the recalculation of bin minimum and maximums. The formula: avg monthly usage * bin minimum percent.
•
Bin Maximum Percent:: This flag allows you to enter the bin maximum percent to be used in the bin recalculation of bin minimum and maximums. The formula: avg monthly usage * bin maximum percent
•
Carousel Interface is a new field that determines if the carousel interface is activated for a warehouse. This flag must be turned on for the system to correctly handle carousel inventory. It also defines whether or not inventory is received as non-nettable. Set this flag to Y to activate the carousel interface for the current warehouse and to update the non-nettable quantity for received products with CAR as the primary bin. This option only sends products whose primary bin is CAR to the non-nettable bucket. Set this flag to A to activate the carousel interface for the current warehouse and to update the non-nettable quantity for all received products. This option sends all products to the non-nettable bucket no matter what their primary bin is. Select N to not use the carousel interface for the current warehouse. For the carousel interface to work, the Multiple Bin Locations and Warehousing flags must be set to N. See also Inventory Processes and Inventory Miscellaneous Reports.

Order Processing Parameters
•
Soe Restock Charge Method, is a new flag that was added to allow restock charges on all types of returned goods including warranty and scrapped. Select S to allow restocking charges on returned goods type only. Select A to allow restocking charges on all types of returned goods. Previously, the restocking charge was only added to return goods with return to stock quantities. This flag, only needs to be established in Order Processing Parameters if the system is to calculate restocking charges on all types of return goods. If this flag is Null, the standard software will only apply restocking charges to returned goods type (same as S).
•
When a credit memo is entered and applied against the original invoice for warranty inventory, for example, this flag would need to be set to A to calculate a restocking charge for the return goods updated into warranty. The restocking charge will update the Completion Screen of the credit memo if a restocking charge applies. The amount of the restocking charge depends on the credit reason code entered and the restocking charge percentage established for the credit reason code in Credit Reason Code Maintenance.
Inventory Parameters
A new Check Digit for UPC flag was added to the Inventory Parameters screen. This flag will determine if the system is to use the 12 digit check sum logic for creating the UPC code. This flag is primarily used when the Generate UPC Codes is set to N because the user does not want to automatically generate a UPC code when a new product is created. When entering N for NEW in the UPC code field in Stock Receipts Entry or an alternative product maintenance field, the system will then generate a UPC code using the 12 digit check sum logic. If the flag, Generate UPC Codes, is set to Y, this flag will be used for the check sum logic. If the flag, Generate UPC Codes is set to Y, this flag will be used for the check sum logic when creating additional UPC codes using N for NEW . If the flag, Generate UPC Codes, is set to C, this flag is not used because C is using the check sum logic automatically when creating the code.
BMW / Work Order Parameters
A new Allow W/O Release for Partial flag was added to the parameters screen. This flag determines if work orders should be released with a partial quantity. If partial or full quantities are to be released for a work order, select Y. If you do not wish to have partial work orders released, select N.
Marketing Parameters
Campaign Priority Code
The Campaign Priority Codes were updated with a new O option. Previously you had the option of selecting Y to require a campaign code in Sales Order Entry, or N to not require one. Now you also have the option of selecting O to establish that the campaign code is optional, and not required to complete the process of the sales order.

ADS Utilities…
Create a New Warehouse From Another Warehouse
This is a new process that allows you to copy warehouse records to create a new warehouse. These new conversion programs were added to be used when transactions exist on ADS, and the customer wants the Warehousing/Bin Locator flag to be set to Y to use the warehousing module. These conversion utilities must be run at a time when there are no transactions being entered on the system.
Bin Location Document Update
This process will cycle through all the open documents on the system and load the bin locator tracking flag. The primary bin will be updated for all confirmation document types and open releases. If no primary bin exists for the product warehouse record, a bin CONV is loaded into the field.
Bin Location and QOH Update
This process will select all product warehouse records, load the primary bin location into the warehouse bin location tracking data, and set the primary bin onhand equal to the warehouse onhand quantity
BarOne Utilities…
ADS has added BarOne utilities for printing labels. BarOne, a 3rd party label software program, looks for a UNIX file created from ADS for the BarOne software to use for label printing. The BarOne software must be running on a terminal. Certain maintenance screens must be established for basic setup of the label. Additional custom programming must be completed to actually print the label.
On Demand General Label Maintenance
This maintenance routine is used to setup a BarOne label to print on demand from ADS.
Subroutine General Label Maintenance
This maintenance routine is used to setup the BarOne label to be printed as part of another process such as Pick Ticket printing. The label will print behind the scenes as part of the printing process.
Warehouse Multiple Printer Maintenance
The maintenance routine sets up label printers per warehouse. A prompt will display to ask you which printer is used. The multiple warehouse printer prompt is not used with the Subroutine General Label Maintenance routine because that process prints labels behind the scenes as part of another process.
Print On-Demand Label
This process prints the stand-alone label from On-Demand General Label Maintenance .

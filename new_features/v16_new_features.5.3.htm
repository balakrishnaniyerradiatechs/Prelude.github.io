New Purchasing Features
Purchase Order Entry
•
A new function key was added to the Product Tagging screen in Purchase Order Entry. F7- Resort enables the user to modify the sort option that was established as the default. The sort options available are P for product number or D for product description. For example, if the Product Tagging Default Sort flag in Default Parameters Maintenance is set to P in which the products will be displayed in ascending order by product number, and the user prefers the products to be displayed in ascending order by product description, this function key can be used to change the product sort in the tagging screen.
•
A new F8- Switch Entry Mode function key was added to the Product Tagging screen in Purchase Order Entry. This option allows the user to modify the position of the cursor from product number to quantity or vice versa when tagging products. A default of P for product number or Q for quantity is established in the Product Tagging Entry Mode field in Default Parameters Maintenance. For example, if the default is set to P, the cursor will be positioned at the product number, and the user prefers the cursor to be positioned at the quantity field, this function key can be used to change the cursor positioning in the tagging screen.
•
The Purchase Order Entry screen was updated with a new Delete P/O Reason Code prompt for deleted P/Os. This prompt enables users to enter deletion reason codes that were in P/O Deletion Reason Code Maintenance for P/Os that are deleted in Purchase Order Entry. This new feature works with the new P/O Deletion Reason Code Maintenance, P/O Deletion Reason Code Listing, and Deleted Purchase Orders Report.
•
A new Orig Line Num field was added to the F5-Add Info screen in the line item detail box. This new field allocates a number to each line item entered on a purchase order. This number identifies that line item with the allocated number. This feature is used in Stock Receipts Entry, Batch Stock Receipts Entry, or Direct Ships Entry by selecting the F9-Go To option and entering the desired line number. This will place the cursor at the line item allocated to this original line number. The Orig Line Num is also displayed in the F10-Examine option for Purchase Order Inquiries.
•
A new feature was added in Purchase Order Entry for Version 16. The Linkage Deletion option was created to allow users the ability to delete items linked to a sales order. However, there are items that cannot be deleted using this process. 051001A Disallowed Items:
-
* Astrik Items- The message Linkage Deletion Disallowed For An * Item! Must Delete Line to Remove will be displayed if trying to perform a deletion on a * item.
-
N Status Items-The message Linkage Deletion Disallowed For A Nonstock Status Item! Must Delete Line to Remove will be displayed if trying to perform a deletion on a non stock status item.
-
04 TypeP/Os-The message Linkage Deletion Disallowed For An 04 Type Purchase Order! If trying to perform a deletion of an 04 type P/O.

This was incorporated into the process to calculate the usage of products to determine the available inventory for each product. Maintenance
Vendor Maintenance
•
The F5-Add Info screen in Vendor Maintenance was enhanced with a new Pay On Receipt feature. This new option was added to Version 16 to cut down on the time being spent on the Vendor Invoice Reconciliation processes. The Pay On Receipt flag can be set to Y, N or A to determine if the system will provide the user the ability to enter necessary information that will allow the Vendor Invoice Reconciliation Entry process to be performed behind the scenes for stock receipts. If this flag is set to Y or A the system will prompt the user in Stock Receipts Entry to enter the Pay On Receipt information such as invoice number, invoice date, period date, and freight amount. Once this was entered, the VIRE process will be performed. If this flag is set to A, the receiver will then be updated with the invoice number, period date, etc. However, the Vendor Invoice Reconciliation will NOT be automatically performed. It will be necessary for the user to either run the Pay On Receipt Vendor Mass Update process or enter Vendor Invoice Reconciliation for each receiver to process the vendor invoice. If this flag is set to N, this feature will not be available, therefore not prompting the user to enter Pay On Receipt information. As a result the stock receipts will be required to be processed separately. Note: This flag does NOT pertain to Direct Ship Receipts.
•
A new Y/N Purchase Hold? flag was added to the Additional Info screen in Vendor Maintenance. This new option allows purchase orders for a vendor to be automatically placed on hold if this flag is set to Y. Once this flag is set to Y it is mandatory to enter the buyer id associated with the buyer responsible for approving purchase orders for this vendor.
•
The new Vend Claim RGA# Required? flag was created to allow the company to determine if it should be mandatory to enter a RGA# when placing a vendor claim for this vendor. If this flag is set to Y, the user will be prompted to enter a RGA# at the time of Vendor Claim Entry. If this flag is set to N, the user will not be prompted.
•
The Label Information option was added to Vendor Maintenance for Version 16. This new feature allows information to be established for vendors that require contents tracking when labels are printed. The information fields added to accurately track contents are Contents Tracking, UCC Mfg Number, SSCC Label Program, SSCC Label Execution, PC Label Program and PC Label Execution. These fields work with the new Vendor Claim Serial Shipping Entry option.
•
A new feature was added to the Addl Options screen in Vendor Maintenance. This option allows a Vendor Product Type to be attached to a vendor to be used as a keyword for vendor lookup screens. Once a vendor type was established as a valid product type code in the new Vendor Product Types Maintenance screen, it can then be attached to a vendor. This option also allows multiple product types to be attached to a single vendor.

This maintenance screen was added to the Purchasing menu to allow reason codes and their descriptions for deleted P/Os to be established. These codes are used in the Deleted P/O Reason Code prompt that is displayed in Purchase Order Entry once a purchase order was chosen to be deleted. The P/O Deletion Reason Code Maintenance option is one of several new options for the deleting P/Os feature. This new addition works with the new flag located in File Update Parameters, the P/O Deletion Reason Code Listing, as well as the Deleted Purchase Orders Report.
Vendor Product Types Maintenance
The maintenance menu in the Purchasing module was enhanced with a new maintenance screen. The Vendor Product Types Maintenance option was added to allow Product Type Codes to be established and used as keywords for Vendor lookup screens. Product type codes and descriptions are created and maintained in this screen. Once these codes were established, they can then be assigned to a vendor as a Vendor Product Type in the Vendor Maintenance> F10-Action> Addl Options Vendor Product Types screen.

Emergency / Direct Buyer Inquiry
The Emergency Direct/Buyer Inquiry was modified for Version 16. Previously the buyer was allowed to generate a purchase order for a item on a sales order regardless if the order was still in process, this was based on the Y option in the Buy? Field of the inquiry. The change made for Version 16 will now display a L (Locked by another process) code to inform the buyer that in fact this order is still in process. Therefore not allowing a purchase order to be generated for this item until the order is exited.

Lead Time Trend Report
The Warehousing module has added a new Lead Time Trend Report for Version 16. This report displays information such as Warehouse, P/O Number, P/O Date Product Number, Description, Received Date, Lead Time, Quantity Ordered, Quantity Received and Cost. This information pertains to lead time trends for receipt dates, vendor numbers and product numbers. However, there are a few Purchase Orders that will be excluded from the report. The following P/Os are examples of what will NOT be included. 100801D
•
If a Purchase Order contains partial receipts, only the 1st Stock Receipt processed for that purchase order will be included on this report.
•
If the Unusual Delivery flag is set to Y for a Purchase Order, this stock receipt will excluded from this report.
•
If the Emergency flag is set to Y for a Purchase Order, this stock receipt will not be included on the report.
•
If the Purchase Order Type is 03 (Will Call) and the lead time is less than 0, this stock receipt will be excluded from the report.
•
If the Lead Time is greater than the Lead Time Cutoff Days in Purchasing Parameters (calculated as 180 days if left blank) the stock receipt will not be included on this report.

Warehousing
Bin Replenishment Maintenance
A new option was added to the Frozen Control flag for Version 16. This new M option will freeze only the Bin Max Quantity and allow the Bin Replenishment Min/Max update to receive the Min Qty. If the bin minimum calculation is greater than the frozen bin maximum, the system will set the bin minimum to the frozen maximum quantity minus 1.
Bin Replenishment Min/Max Update
The Bin Replenishment Min/Max Update was enhanced to now include a new Preview or Update field. If P is selected, a preview report will then be generated. If this flag is set to U, the update process will be performed without printing a preview report first.
Summary Bin Queue
The Summary Bin Queue Inquiry is a new inquiry that was added to display the number of pending transactions that are to be picked for a warehouse, using the RF unit. These transactions include Sales Orders, Stock Transfers, Work Orders, Bin Replenishments and 1st & 2nd level cycle counts. The inquiry screen will be updated by reducing the number that is displayed in the #Documents and #Lines columns, once the transaction was picked and processed using the RF Unit.
Product Label Print By Receiver
To provide users the ability to print barcodes for a Purchase Order or Transfer Receiver #, as well as Product Numbers, a new feature was added to Version 16. The Product Label Print By Receiver is a new printing process that allows you to enter a Warehouse #, Printer Form Queue, Document Type, Receiver Number, or Product Number information for barcode printing. If Document Types P (Purchase Order) or T (Transfer) are selected, and the valid Receiver number is entered, the user will then be prompted to choose one of three barcode printing options. The Single option will display all the products listed on this receiver. The # Labels column will be defaulted to display O for labels to print, the user then has the option of entering the number of labels required for this product. If the Exception option is chosen, only the products that have a received quantity will be displayed, the # Labels field in this screen will automatically be defaulted with the quantity received for this product. However, the user has the capability of changing this number if needed. If Complete is selected, the system will automatically calculate the # of labels needed based on the quantity received and print that number of barcode labels. If Document Type M is entered for this process, an additional screen will be displayed, prompting the user to enter the product and the number of labels to be printed for this item.

For Version 16, three attributes were added to the PICK.STATS file to better track pick errors associated with Credit Reason Codes. As far as the additions made to the Picker Statistics Report, three new columns were added to represent the new attributes added. The Picker Statistics Report was updated with a new Unt/Zn column. This new column will represent the total number of units picked per zone. The information that will be displayed in this field is generated from Attribute 5.M (NUM.UNITS.ZONE) in the PICK.STATS file once the Sales Order Bin Allocation was processed and the release was updated. Please note, that the 5.M Attribute is multi valued with the zone being contained in Attribute 1.M. In addition to the updating of the sales orders, work orders, stock transfers, bin replenishments, and any other processes that may update this file will display total number of units picked as well.
Receiving Statistics Report
A new Receiving Statistics Report has currently been added to the Warehousing module. This report displays information regarding any warehousing activities that were received through Stock Receipts (terminal or RF process) or Stock Transfers. In addition, this report was formatted to list the total number of received lines for each user and transaction date under the #Received column. To better identify the transactions that pertain to this report, two new Transaction Types were created as well. The SRR-Stock Receipts type identifies entries received in Stock Receipts Entry (/SRE), RF Stock Receipts (/RFSRE), RF Multi P/O SRE (/RFMSR), Direct Ship Receipt Entry (/DSRE) and Batch Update Procedure (/BSRE). The TRR-Stock Transfer Receipts type references transactions received through Transfer Receipts Entry (/TRE) and Batch Update Procedure (/BTR.UPDATE.PROCESS).
Rf Release Picked Report
The Warehousing module has added a new RF Release Picked Report in the Reports screen. This new report displays all releases that were completely picked on RF Units and are waiting to be updated through the Release Confirmation Entry process. Once the releases were confirmed through RCE, they will no longer appear on the report.
Warehouse Activity Report
Version 16 has added a new report to the Warehousing module. The Warehouse Activity Report was created to display warehouse activity by a warehouse as well as beginning and ending activity dates. This report displays information such as Sales Order Pick Lines (updated at time of Pick Confirmation By Range or Pick Confirmation By Release). Sales Order Pack Lines (updated at time of SSCC Entry). Transfer Pick Lines (updated at time of Transfer Picking Entry). Transfer Pack Lines (updated at time of Transfer SSCC Entry). Work Order Pick Lines (updated at time of Work Order Picking Confirmation). Replenishment Lines (updated at time of Replenishment Picking Entry). Cycle Count Lines (updated at time of Cycle Count Entry-1st & 2nd level). This information is advantageous to those users that are required to view warehouse activities for a warehouse or activity date. In addition, a new Warehouse Activity Purge routine has also been added for this report. This purge routine will purge all warehouse activity up to the activity date selected. A Purge Type 38 has also been created for the Purge Log Report. This new type can be entered to view only purged information that relates to Warehouse Activity.
Bin Velocity Summary Report
The Bin Velocity Summary Report was added to the Warehousing module reports to provide users with the capability of generating a report that summarizes how many times a bin was picked from. This report can be processed for a Warehouse Number, Bin Number or Product Number. The BIN.VELOCITY file is used to retrieve the information necessary for this report.
Bin Velocity Detail Report
The Reports option in the Warehousing module was updated with a new Bin Velocity Detail Report. This allows the user to generate a report to provide detailed information pertaining to how many times a bin was picked from as well as the transaction type, such as Sales Orders, Stock Transfers, Bin Replenishments and Work Orders. The BIN.VELOCITY file is used to update the records for this report.

Putaway Ent.
•
The Putaway Ent option was restructured to allow a user to have the ability to update a primary bin during the RF Putaway Entry process. This option can only be used if the new RF Putaway Bin Update flag in User Id Maintenance is set to Y.
•
A modification was made to the Putaway Ent process. If the FROM bin location has a bin type D for dock and the quantity being putaway is less than the quantity displayed, the user will be prompted to choose if the putaway of this item should be placed in different bin location. The user will be prompted with the following message: Complete Putaway of this Item to Another Bin? If the user selects No, the process continues on to the next product for putaway. The remaining quantity will remain in the dock bin until the next putaway entry for that product is performed. If the user selects Yes, a new line will be inserted in the bin move record duplicating the product number and the FROM bin location with the remaining quantity. The TO bin location is left blank for the user to decide which bin location should be used for the remaining quantity.
Stock Receipts
•
The Stock Receipts function in the RF process was enhanced with a new feature for Version 16. The user now has the ability to create UPC codes while in the RF Stock Receipts Entry screen. Two new flags were added to the Inventory Parameters screen to accommodate this new feature. If the Generate UPC Code flag in Inventory Parameters is set to R, UPC codes can be generated at time of RF Stock Receipts. If the flag is set to D UPC codes can be created using the check digit sum logic when in RF Stock Receipts Entry. If the flag is set to either R or D and the product entered does not contain an alternate product number that matches the company prefix, the user will then be prompted with a UPC Not On File! Do You Wish to Exit, Scan, or Generate message. At that point if the user chooses Exit, the user will be returned to the Product Number prompt. Or the user can choose Scan, which allows the product to be scanned or the UPC code to be entered. Once this function was processed, the UPC code will be updated as an alternate product number in Product Maintenance. If the user selects Generate, a UPC code will then be generated. Once this was processed, the UPC code will then be updated as an alternate product number in Product Maintenance. If the Generate UPC Code flag is set to R or D and the product entered in the Stock Receipts, does in fact already have an alternate product number attached that matches the company prefix, the system will continue processing the RF Stock Receipt and the user will not be prompted with the generate UPC Codes message.

The functionality of the Bin Location prompt in the Stock Receipts screen was updated for Version 16. Previously, it was required that the user manually enters the bin location for each product. The Bin Location prompt now will use the 1st bin location entered for the product received as the default location until the user changes it.

Is Quantity Received Correct? will be displayed if the quantity received in Stock Receipts is greater than the open quantity on the Purchase Order, by more than the percentage established in the Over Rcvd Unit Var Pct field in the Warehouse Maintenance> F5-Add Info screen. Therefore allowing the user the ability to modify the amount being received or accept it.

The Weight Required field in the Inventory parameters was updated with a new functionality, therefore creating a modification in the RF Stock Receipts screen. If a product is entered that does not contain a weight in the Product Maintenance> F10> Addtl Opts> Package Information screen, it will be required for the user to enter one before the receipt can be updated. However, once the weight is entered and the transaction is updated, the weight information will then be updated to the Product Maintenance file. Therefore not requiring the user to continue entering the weight for the same product over and over again.

Alternate Pn Mnt
To implement some better time management practices, Version 16 has updated the Alternate PN Maintenance with new fields to provide users the ability of adding UCC Codes on the fly in the RF process and picking screens. If an alternative product does not exist for a 13 digit barcode that is scanned, the user will be prompted with the message Product Does Not Exist! Set Up as Alternate Product?. If the user selects Y, the RF ADD screen will then be displayed, allowing the user to enter the ADS stocking product number that is to be updated as an alternate product with the barcode number scanned in Product Maintenance. Therefore, ensuring that this product will be recognized as an alternate product when scanned in the future. Once the product number is entered, and if alternate products currently exist, the Alternates field will then display the first alternate product listed for this product. If multiple alternates exist, the down arrow key can be used to move through the list of alternates. Once the Alternate number was selected, the remaining information pertaining to this alternate can then be entered. The UPC field is a Y/N flag that determined if this product should be identified as a UPC code. The UCC flag determines if this alternate should be established as a UCC Code. This code can be identified as a M-Master Pack, P- Pallet Pack, S-Shelf Pack, or select N to not update this product as an UCC code. If something other than N is selected for the UCC flag, the user will then be prompted to enter additional UCC information. Enter the quantity of items for this UCC Code in the UCC Qty field. This quantity will automatically be defaulted to the Serial Ship Container Entry for label printing purposes. Next, the user will then be required to enter the Unit of Measure in the U/M field. The user is then brought in to the Cat ID field to enter a valid Catalog id code for this product. The Alt Ppg field requires that the associated alternate product price group to be entered for this UCC code in this field. In addition to this screen, the option of adding UCC Codes on the fly was added to several other screens as well. Such as, Individual Bin Movement, Emergency Cycle Count Entry, Cycle Count Entry-2nd Level, Putaway Entry, Stock Receipts, Multi P/O SRE, SR Putaway Entry, RGA Entry, RF SSCE Entry, RF Sales Order Entry, Bin Repl Picking and Transfer Picking.

A new Skip From Bin field was added to the Warehouse Maintenance screen. This is a new Y/N flag that determines if the RF picker is prompted for the FROM bin field during RF Order Picking. If this flag is set. To Y, the FROM bin WILL be skipped, and the RF picker will not have the option of picking products from a different location. The cursor will then go directly to the Qty field. A new flag was added to the Warehouse Maintenance screen. The Allow New Products in RF SRE allows ADS established products that do not exist on an original P/O to be received in RF Stock Receipts. to use this option, the Warehousing Flag must be set to Y. Once the Allow New Products in RF SRE was established with the flag set to Y, new products can then be received on the RF unit without having to first add them to the open purchase order. When a new line item is added, the system will NOT prompt for an open order quantity or unit of measure. Any line items added at the time of RF Stock Receipts will be included on the Stock Receipts Exception Report once the P/O is received.

If this flag is set to Y, the destination bin prompt will be displayed allowing the user the ability to enter the preferred staging bin for a order.

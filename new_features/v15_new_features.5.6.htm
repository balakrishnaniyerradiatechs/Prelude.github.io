Rental
Rental Order Entry
Header Level
The Rental Order Entry screen was updated with a new Return Warehouse field to specify which location the rental item is being returned. If the return warehouse is not specified in this field, the return warehouse will default to the same location that appears on the original rental order that the item was placed on.
F-10 Action
•
in the Rent Info option, the Rental Start Date field allows the user to enter the rental start date to default in the Rental Information screen in Release Confirmation Entry
•
A Rental Start Time field was added to the Rent Info option to enter the rental start time to default in the Rental Information screen in Release Confirmation Entry.
The date and time entered in the Rental Header screen will default in the rental information screen in Release Confirmation Entry. These fields can be overridden. If these fields are left blank, the system will default to the system date and time in Release Confirmation Entry. If a date and time is entered in the Rental Header and the flag, Start Date/Time in RCE?, is set to N, the system will not use the date and time entered in the header screen. The flag will override the date and time by using the pick ticket printing date and time.
Line Item Level > F-5 Add. Info
The F5- Add Info option in the line item detail of Rental Items was modified by adding a Rental Rate Method flag. This flag is used to enter the method for the rental item. If a method is established here, this method will override the rental rate method defined at the header level. If multiple rental products exist on the order, each rental item can be established with its own rental rate method. For example, rental item 9801-2 can be set to B and rental item 9801-OC can be set to A. Billing will be determined by the line item’s method. If this field is left Null, the rental rate method will be used from the header.
Rental Exchange Printing
•
Rental Exchange Printing was modified to incorporate the continuous printing option. To integrate this feature, the new Cont. Rental Exchange flag will need to be established in Warehouse Maintenance. This flag determines if continuous printing will be used for Rental Exchanges based on the flag setting. This will allow the continuous printing for the Rental Exchange documents to be performed during the Rental Exchange Printing batch routine.
•
A new F-5 Rental Xchg Reprint option was added to the Rental Exchange Printing process to allow the user to reprint a rental exchange document to the form queue entered.
•
In the Previous versions of the software, Rental documents such as the Rental Agreement, Rental Checkin, Rental Checkin History and Rental Checkin History Reprint displayed all comment lines on the printed documents. This was modified to only display the C Comments on documents printed.
Serial Number Change Maintenance
This Serial Number Change Maintenance routine is a new option that was added to ADS to allow the correction of incorrect serial numbers attached to products. This process enables the user to change an incorrect serial number to the correct serial number of an item that was received with the incorrect one. This process cannot be used for any incorrect serial numbered items that exist on any open documents such as transfers, sales, work, or rental orders, etc. The Serial Number History Inquiry was updated with the CHG code to reference the serial number change made in the Serial Number Change Maintenance option. Both the original and new serial numbers are updated with this CHG code. The Serial Number Inquiry was updated as well, to reflect the new serial number using the original serial number's EIN and rental unit number.
Inquiries
Rental Order Inquiry
The Rental Order Inquiry screens was updated to include sort and filter options for allowing additional criteria to limit the rental orders displayed in the respective inquiries. When the flag, Filters, is set to Y, the Rental Order Inquiry Filters screen is displayed.
Rental Order History Inquiry
Sort and filter options were added to the Rental Order History Inquiry to limit the inquiry to rental orders depending on the criteria entered. When the flag, Filters, is set to Y, the Rental Order History Inquiry Filters screen is displayed.
•
Sort: this field determines how the rental order records will be sorted on the inquiry screen. The default sort can be established in the field, Default Order History Inq., in Default Parameters Maintenance. Press F3 to display the available sort options. Select O for Order Number, D for Order Date, P for Customer P/O Number, I for Invoice Date, or N for Invoice Number.
•
Filters: this flag determines if filters are used to limit the rental orders displayed on the screen. Select N to not allow filters and display all the rental orders for the customer number entered using the sort established in the Sort field. Select Y to allow filters to be used for limiting the rental orders displayed on the inquiry screen.
•
F10-Examine - The F-10 Examine option in Rental Order Inquiry and Rental Order History Inquiry was updated to display the rental rate method, if one was established at the line item.
Transfer Rental Return Inquiry
This is a new inquiry that was added to display the auto transfer transactions generated at invoicing when the rental item is returned to a different location.
Rental Summary Inquiry
A new inquiry Rental Summary Inquiry was added to the Rental Module. This inquiry displays rental orders that have not yet been invoiced, for a customer and job number as well as the total units of the product number entered for a job.
•
Sort: this field determines how the rental order records will be sorted on the inquiry screen. The default sort can be established in the field, Default Sales Order Inq., in Default Parameters Maintenance. Press F3 to display the available sort options. Select O for Order Number, D for Order Date, R for Required Date, P for Customer P/O Number, or T for Order Type.
•
Filters: this flag determines if filters are used to limit the rental orders displayed on the screen. Select N to not allow filters and display all the rental orders for the customer number entered using the sort established in the Sort field. Select Y to allow filters to be used for limiting the rental orders displayed on the inquiry screen.
Reports
Periodic Billing Report
•
The Periodic Billing Report was modified to display any comments that were entered on the original Rental Order.
•
The Periodic Billing report has recently been enhanced to include the Grand Totals of the invoice amounts of all the periodic billing entries listed on the report.
Rental Items Sold Report
The Rental Items Sold Report is a new report option that was added to the Rental Module to display rental items sold on sales orders for a division, warehouse, product, customer, or fiscal period depending on the criteria entered.
Daily Transfer Rental Return Report
This new report was created to allow the user the ability to generate reports on all rental orders that were returned to a warehouse other than the warehouse it was originally invoiced out of. You can set the parameters of the report to display information for the Transaction Date, Division Number, the From Whse and To Whse.
Monthly Transfer Rental Return Report
This new report was created to allow the user the ability to generate reports on all rental orders that were returned to a warehouse other than the warehouse it was originally invoiced out of. You can set the parameters of the report to display information for the Period, Division Number, the From Whse and To Whse.
Equipment Maintenance
A new F6-Parts Copy function key was added to the Equipment maintenance screen to allow a parts list to be copied from one equipment id to another equipment id. This copy function can only be used if the equipment id to be copied to, contains a blank parts list.
Equipment Rates / Availability Listing
This new listing has the ability to sort each product number in serial number order. The Equipment #, Product #, Product Description, Serial #, Rental Unit #, Make, Model, Product Line, Minimum Rental period, Unit of Measure, and Rate Information is displayed on this listing
Rental Tax Rate %
To calculate a sales tax rate on rental items, a new field was added in Tax Jurisdiction Maintenance. This field enables you to enter the Rental Tax Rate % that is to be applied to Rental orders. The tax % entered in the Rental Tax Rate % field will override that tax% entered in the Tax Rate% field. If the Rental Tax Rate% field is left blank, the tax % entered in the Tax Rate% will be applied to the items on the Rental orders. If a rental item is sold in Sales Order Entry utilizing the Rental Unit Sold function (F10-ActionProd Func) the sales tax percentage in the Tax Rate% field will be used to calculate the sales tax since the rental item is being sold as regular inventory on a sales order. The maintenance routine to setup the Rental Tax Rate% field is located in Order Processing Maintenance Sales Tax Tax Jurisdictions.
Rental Sales Representatives
Two new salesrep fields were added to the system to to have the ability to assign Rental Sales Representatives ids to Rental Orders. The new fields are located in both Customer and Customer Ship-To Maintenance screens. in Customer Maintenance, the Salesrep #1 field allows you to enter the first sales representative to be assigned to the rental orders, the Salesrep #2 field allows you to enter the second sales representative to be assigned to the rental orders. When a customer ship-to number is used on a rental order, the system will default to the sales representatives established in Customer Ship-To Maintenance. The maintenance routine to setup the Rental salesreps is located in Order Processing Maintenance Customer/Customer Ship-To Maintenance F7-Order F5-Add Info
F6-Rental
Note: If a rental item is sold using the Rental Unit Sold function in Sales Order Entry, the salesrep on this order will not be automatically changed from the Rental Sales Rep to the Sales Rep. If the rental items sold are to be tracked by the Sales Rep and not by the Rental Sales Rep, the user will need to manually change the salesrep id for that sales order.
Rental Contract Customers
•
Rental Contract Customer # - A new Price Matrix Maintenance type 50 (Customer Number/Rental Product Number) option, was added to the system to allow a matrix type to be setup for Rental Contract Customers. Once a rental contract number is established in Customer Maintenance as the master customer number, the master customer number can then be entered in the Rental Contract Customer # field in the type 50 rental matrix. Multiple rental customer numbers can use the same rental contract customer # to benefit from rental discounts in Rental Order Entry.
•
Rental Lost / Damaged Quantity - A new feature was added to the Rental Order Entry screen to allow the automatic billing of lost or damaged rental items. The system enables the user to enter the quantities of the lost or damaged rental items in the Lst/Dmg field on the line item level. Once a quantity was entered in this field, a Lost/Damaged Information screen will be displayed. This screen allows the user to enter the Selling price and Comments for this rental item. In addition Rental Lost/Damage Comments can be setup in Default Parameters Maintenance to allow comments to be entered on the Rental Return.

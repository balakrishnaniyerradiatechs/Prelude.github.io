Shop Repair
Shop Repair Order & Quote Entry
Shop Repair Quote Subsystem – New Subsystem
The Shop Repair module has a new subsystem that allows you to quote a customer on a repair. Shop Repair Quote Entry is like Shop Repair Order Entry procedure with job status of Q for Quotation. The system includes:
•
Shop Repair Quote Entry
•
Open Repair Quote Inquiry and History Inquiry
•
Quote Repair Pickup and Delivery Ticket Printing
Continuous Printers
The USER FORMS programs must be modified. The Shop Pickup and Shop Delivery flags in Warehouse Maintenance (F6-Printing) are now checked for quote repair pickup ticket documents using continuous printing.
Rental Returns – New Option
The new Internal Shop Customer field in Rental Parameters allows you to enter a customer number to generate a shop repair order at the time of rental return in Rental Order Entry. The Rental Status field for each product (Product Maintenance > Action Bar > Addtl Opts > Rental Information) has a new R option to signify that it is a Shop/Repair Order. If a rental serialized item is set up with the Rental Return Status of R, and the rental item is returned, the system prompts you to setup a shop/repair order for this item. The Shop Repair Job Entry screen is prompted for each line item on the rental order. A default description of the job referencing the original rental order is updated for the job description. If multiple line items are on the rental order, the system creates a separate shop/repair order for each line item. If the rental is returned to another warehouse, the system creates the shop/repair order using the transfer return warehouse where the serial number is updated.
Create Job Codes When Entering a Job
You can now create a shop repair job code on the fly while in the Shop Repair Job Entry screen. When you are in the description field, if the description is not a valid job code, you are prompted to create a job code.
Recap Report – New Option
The Shop Repair Recap Information screen has a Recap Report option to view profit for labor and parts.
Model Number Field – F3 Lookup Modified
The F3 lookup on the Model Number field on the Shop Repair Information screen now displays the model numbers bought by or repaired for the customer that you are entering the shop repair order.
Rent Info – New Option
The Shop Repair Order Entry Action Bar has a new Rent Info option on the Action Bar. You can change the Repair Status to D (down for repair) so that the rental serialized item is not available for rental.
Invoice Preview – New Flag
The new Invoice Preview flag appears on the Shop Document Selections screen when you choose to print a shop repair order. This Y/N flag determines if an invoice preview document is generated to include all charges on the sales order completion screen for the shop repair order as well as sales tax and deposit information. The invoice preview allows you to print a copy of the invoice before the order is invoiced through Invoice Printing.
You can set the Invoice Preview flag for continuous printing in Warehouse Maintenance > F6-Printers.
Service Bulletin - Alert Type 4 Added
The Alert Level field in Service Bulletin Maintenance now has a new type 4 - Information Bulletin option. The alert type displays in Shop Repair Order Entry.
Jobs
Parts Entry – New Shop Repair Job Entry Screen Function
The new Parts Entry option on the Shop Repair Job Entry screen Action Bar allows you to add parts for a job code. If the job code entered exists in Job Code Maintenance for the product entered, the system prompts with the parts entry screen. You can add new products to the parts list and update the job code file.
Job Code Maintenance – New Fields
The new Model Number field in Job Code Maintenance allows you to enter a model number for the job code entered. If you do assign a model number to a job code, then you can only use that job code for the assigned model number. If a model number is not entered in Job Code Maintenance, then the job code can be used for all model numbers. The new Cd field in Job Code Maintenance > F7 Products, determines if the product number is required for the job code. Set the flag to R to require the product to be entered on the shop repair order when the job code is entered. Set the flag to O to allow the product to be optional when the job code is entered. A new F8-Comments function key in Job Code Maintenance allows you to enter comments to appear on the order, for a shop repair code.
Examine Option on Sales Order Inquires - Updated
These inquiries (F5-Review > Action Bar > Examine) now display the entry date, technician, and hours worked, if the hours from Shop Time Clock Entry or Technician Job Billing Entry.
Job Billing, Shop Charges, & Pay Types
Technician Job Billing – New Feature
The new Technician Job Billing Entry feature allows you to allocate a base number of hours across multiple shop repair orders. Since all of the work done during the day is entered at one time, orders cannot be invoiced until the Technician Job Billing Entry is completed for all technicians. When entering labor for each technician, the Document # field allows you to enter work completed by Order number or by Shop Time Clock code.
The rate and price for each shop repair order is determined by the rate entered. Once the hours are entered for each job, the system updates the service labor items for each job on the shop/repair order. The cost posted is based on the type entered. If a service item already exists with a different cost, the service item is added as a second line with the new cost. You can also enter labor hours for shop quotes and regular sales orders using the shop time codes. Note: Time posted to quotes is converted automatically; time added manually is not.
The Technician Job Billing Inquiry displays technician job billing entries. The Technician Shop Rate Maintenance program allows you to set up each technician with labor rates for work performed. Rates are entered for work during regular hours, overtime hours, or work done at a premium.
Move Service Labor to Another Job – New Option
The Move to Another Job option (Shop Repair Order Entry > Action Bar > Job Func ) allows you to move the items (parts or labor) on a shop repair order to another shop repair order.
Shop Labor Type Maintenance – New Feature
The new Shop Repair Labor Type Maintenance program allows you to set up a default labor item and type (regular, overtime, premium rate) for defaulting in Technician Job Billing Entry. Once the service labor item is assigned, the system prompts with this labor item for every shop repair order in Technician Job Billing Entry.
Paid Option on Shop Time Clock Codes – New Field
The new Paid field in Shop Time Clock Code Maintenance determines if the shop time clock code is considered a paid code. The field setting here determines where the shop time clock code is printed on the Shop Timekeeping Payroll Report.
Basis flag – new B Option
Basis in Shop Charges Maintenance has a new B option to use both labor and parts to calculate shop charges.
Capitalize Charges For Equipment ID
The new Captilize flag in Pay Type Maintenance determines if the dollars for the shop/repair pay type on the order are considered no charge and posted to the Add –On Capital column for the equipment id number in Equipment Maintenance. When the Capitalize flag is set to Y and Charge Customer flag is set to N, the system creates the G/L entry for the shop/repair order invoiced using the Repair A/R Offset account in the General Ledger Table instead of the A/R account. The system also creates a sub asset automatically for the main asset used on the shop/repair order for the no charge amount on the shop/repair order.
Shop/Repair Option Parameters
Require Tech on Job – New Flag
The Require Tech on Job flag in System Parameters determines if the technician is required for the job code before any labor hours can be updated to the job.
Require Problem Notes – New Flag
The Require Problem Notes flag in System Parameters determines if the Problem Notes are required in the job entry screen in Shop Repair Order Entry.
Department and Cost Center in Shop/Repair Parameters
The Dep# (Department Number) and Ccr# (Cost Center number) fields were added to Shop/Repair Parameters, to apply each time a Setup shop repair order is entered.
Print Pick Ticket – New Flag
The USER FORMS programs must be modified. The new Pick Ticket flag determines if the pick ticket is printed for shop repair order – if continuous printing is set up for pick tickets. The standard pick ticket for shop repair orders does not include service labor items and is treated like a staging document.
Entering Hours Only for Technician Job Billing
The Enter Hours Only Flag in Shop Repair Parameters determines if the Technician Job Billing Entry screen lets you enter hours for the system to calculate the beginning and ending start times for each job.
The new Hours field in Technician Job Billing Entry is only accessible if the above flag is set to Y. If a non-shop repair order is entered in the Document # field, the job field is skipped and you can enter labor hours.
Technician Job Billing Inquiry has a new Hours field to display the hours worked.
Inquiries
Shop Repair Vendor Claim (SHVC) – New Code
When a shop repair order is invoiced, if the pay type on the order has the Warranty flag set to Y in Pay Type Maintenance, the Sales Order Log Inquiry is updated with the SHVC code to reference that the vendor claim was created for the failed product number.
Serial Number History Inquiry Screen
You can now view the Serial Number History Inquiry screen from in the History Shop Repair Inquiry (F5-Review > F5-Hdr Scr > F7-Repair History > F7-SN History) and the Model/Serial Repair Inquiry (F7-SN History).

Updated EDI Reports Functionality
The Daily Audit Log, Acknowledgement Report, Unconverted/Formatted Data Report, and Unconverted/Formatted Historical Data Report were updated with a delete process. The process names for these reports are now deleted from the SAVEDLISTS file. This modification was made to clean up the SAVEDLISTS file. The process name is deleted from the file whether or not the purge process is performed. This change does not affect the results of the reports.
Update to Convert Ship-To Functionality
The Phone Extension field is correctly updated in Customer Ship-To Maintenance when the DOE is converted.
New Convert Vendor Claims Process
A new Convert Vendor Claims process was added to the Electronic Entry module of Prelude. This new feature allows vendor claim data that was updated in the DOE convert files to be created into new vendor claims in the system. This process works with the vendor flag added for the Nonreturnable field in Product Maintenance (Product Maintenance> F5-Add Info> Nonreturnable Flag> Vendor). When using a product with the Vendor field set to N for the Nonreturnable flag for the DOE Convert Vendor Claims process, the message that typically prompts in Vendor Claim Entry will be suppressed. However, will be updated in the Header Notes for the vendor claim created. Since the product is not allowed in Vendor Claim Entry, the DOE Convert Vendor Claims process updates the product on the vendor claim as an invalid product number.

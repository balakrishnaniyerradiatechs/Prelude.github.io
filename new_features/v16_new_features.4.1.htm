V16 New Features
Order Processing
Sales Order Entry
Header Level
•
The Purchasing contact in the Sales Order Entry header was updated to display the purchasing contact created in Customer Contact Management Maintenance. If no purchasing contact is currently established, this field will remain blank.
•
A new Blind Shipment flag was added to the Sales Order Header screen. This flag allows you to send out a shipment to a customer anonymously by not including the company logo, name or address on the packing slip. If you are currently using the 3rd party software package Optio, a user-forms change can be made to your existing user-forms program to NOT include the company information. If this flag is set to Y both the pick ticket and packing list will be printed without the company information and the Price Pick Ticket flag will automatically be set to Y. If a manual ship-to address is entered, the system will prompt the user to select Yes, No or Blind Shipment on the Price Pick Ticket option. If the Blind Shipment option is selected, both the Price Pick Ticket and Blind Shipment flags will be set to Y automatically. This new feature was added to Quotation Order Entry as well.
•
The sales order header was updated with a new Consolidated Inv? Flag. This flag setting will determine if this order will be considered a consolidated invoice. If this flag is set to Y, this order will not be printed but will be processed as if the invoice had actually been printed. The sales order will then be closed, an A/R record created and inventory relieved. Once the invoice number is entered in the Consolidated Invoice Printing screen, the original invoice will be marked as paid, and a new consolidated invoice will be created. Note: A User-Forms modification is required to be made to the Consolidated Invoice Printing User-Forms program to use this feature.
•
A new function was added to the Reprice Order Options in Sales Order Entry. This option now requires a password prior to allowing the user to enter the Repricing menu. The password is established in the Systems> Maintenance> Program & Menu Passwords using the process defined as REPRICE.OPTION.CHK.
•
A new function key was added to the Product Tagging screen in Sales Order Entry. F7- Sort enables the user to modify the sort option that was established as the default. The sort options available are P for product number or D for product description. For example, if the Product Tagging Default Sort flag in Default Parameters Maintenance is set to P in which the products will be displayed in ascending order by product number, and the user prefers the products to be displayed in ascending order by product description, this function key can be used to change the product sort in the tagging screen.
•
A new Order Function was added to the Sales Order Entry screen. This new feature was created to allow users to reallocate inventory for backordered items from future require date sales order to current sales orders. Once this option was selected, and if inventory is available, for this item, backordered, on-hand, committed and available quantities will be displayed. Reallocation will only be permitted if the inventory is to be received prior to the future required date. Although kit parent items and bill of material products are excluded from this option, open stock transfer and work order lines are checked for available reallocation.
•
A new F8- Switch Entry Mode function key was added to the Product Tagging screen in Sales Order Entry. This option allows the user to modify the position of the cursor from product number to quantity or vice versa when tagging products. A default of P for product number or Q for quantity is established in the Product Tagging Entry Mode field in Default Parameters Maintenance. For example, if the default is set to P, the cursor will be positioned at the product number, and the user prefers the cursor to be positioned at the quantity field, this function key can be used to change the cursor positioning in the tagging screen.
Order Level
•
Sales Order Entry was updated with a new suspended order feature. You can require the user to enter a password for any sales orders that are to be suspended. to activate this feature, the ORD.SUSPEND process must be established in the Program & Menu PassWords maintenance screen.
•
Sales Order Entry was updated with a new prompt for password feature. You can require the user to enter a password prior to selecting No to the messages Are You Ready to Release This Order For Invoicing? (06 confirmation orders) and Print Invoice? (05 counter invoices) in Sales Order Entry. to activate this feature, the NO.INVOICE process must be established in the Program & Menu PassWords maintenance screen.
•
A new option was added to Sales Order Entry. Previously, once /OLI was selected, the Sales Order Number field in the Order Log Inquiry screen would appear blank, the user now has the option of typing /OLI in Sales Order Entry and the Order Log Inquiry screen will now default to the order number currently on the screen to display transactions that have taken place regarding this order. This same feature has also been added to Sales Order Inquiry and History Sales Order Inquiry.
•
A new accessory option was added to Version 16. A new Group option was created to allow multiple accessory items to exist in a single group. Therefore, in Sales Order Entry, the user has the option of reviewing multiple accessory items for a product. If a product is entered that contains accessory products, the user will be prompted with the message Do You Wish To Continue With Accessory Items Entry. If the user selects Yes, the Product Accessory Group Information screen will be displayed. To view product detail information, select the F5-Product Detail action key, this function will display the available accessory items that can be added to the sales order. Multiple products can be selected from accessory groups. Once the accessory groups were reviewed and the products were selected, the system will then add all accessory items to the sales order at that time.You must set these system flags to use this feature accurately.
•
A new feature was added to Version 16 to allow inventory to be reserved for a customer. Therefore resulting in some modifications to Sales Order Entry. The available quantity field in the line item detail screen will include reserved inventory. In addition, there is a new RSVD ( ) field in this screen to notify the user of the quantity reserved. However, if the quantity entered for this product produces an inventory level, below the reserved inventory level for a customer, the Insufficient Stock Requested Reserved Inventory! Placing Line Item on Hold! warning message will be displayed, placing the sales order on inter-branch hold.
•
The Sales Order Entry screen was updated to incorporate a revised Obsolete Product Warning Message. Previously, the Obsolete warning message was displayed at the bottom of the screen. This was modified for Version 16. To ensure that the user observes the warning message and is required to confirm that it has in fact been viewed. Once an obsolete product number is entered, a dialog box will be displayed with the message This Product was Classified as Obsolete. Sell Available Units Only!. At that point, the user will then be prompted to select Reprompt to notify the user again, that this product is in fact obsolete. Continue is the other available selection. This will allow the user to continue the entry of this product.
Completion Screen
Charges that were entered in any of the Handling, Delivery, Pick Up, Restocking Charge, Minimum Charge, or Disc/Allowance fields in the sales order completion screen, will be reflected on a new Other column that was added to the Invoice Detail Inquiry in the Accounts Receivable Inquiries.
Release Confirmation Entry
•
The Release Confirmation Header screen was updated to include a new Handling field. This field allows you to enter any additional handling charges that are to be charged to the customer at the time of invoicing. The handling charge amount entered, will be updated in the handling field in the sales order completion screen as well.
•
A new Warning message was added to the Release Confirmation Entry screen. The message Quantity Exceeds Available For Bin! Continue? will be displayed to warn the user that the bin quantity entered is greater than the bin available. This new message enables the user to determine if they in fact would like to continue using this bin, in which will cause the bin quantity to go negative, or chose a bin with the available quantity. This feature is also used for Type 06 and 08 orders in Sales Order Entry when the Bin Location information is entered in the Qty-Shp field and the quantity entered is greater than the available bin quantity. Note: this warning message will only be displayed for sales orders that are entered for branches using warehousing.
•
Component Entry is a new function that was added to Version 16. This feature allows the user to enter component items in place of the original products on the order during Release Confirmation Entry. The component items are not validated for on-hand inventory, the system assumes if this product is entered, the products listed are equivalent in stock. Therefore the original product on the sales order is not reflected in the on-hand quantity since the component items are relieved from inventory. Once the Release Confirmation Entry was processed, the original product on the sales order is updated as a kit item.

The Quote Order Entry screen has added a new option due to the change in functionality of the Campaign Priority Code in the Marketing Parameters. If the Campaign Priority Code flag is set to Y, it is mandatory to enter a campaign code prior to entering a order in Sales Order Entry, but is NOT mandatory in Quote Order Entry.

F7-Order
•
A new flag was added to the Customer Maintenance option. The Consolidate Type flag determines which type of consolidated invoices will be included in the consolidated invoice printing process. The Consolidate Invoice flag must be established for this new feature to be used. The invoice printing process will check the Consolidate Type flag setting to determine if this customer’s sales orders are to be recognized as part of consolidated invoicing. If this flag is set to N the invoice printing process will proceed as normal. If this flag is set to B, S, V or left blank, the sales order will not be printed, however be processed as if it had. The sales order will then be closed, and A/R record created and inventory will be relieved. Once the Consolidated Invoice Printing Process was completed, the invoice will be included in the consolidated invoice that is printed. The available flag settings are:
-
B – Sales & Service Invoices are included in Consolidated Invoice Printing
-
S – Sales Invoices only are included in the Consolidated Invoice Printing
-
N – Sales & Service Invoices are not included in Consolidated Invoice Printing
If this field is left blank, the system will consider this setting the same as the B flag and include both Sales & Service Invoices in the Consolidated Invoice Printing process. Note: A User-Forms modification is required to be made to the Invoice Printing and Consolidated Invoice Printing User-Forms program to use this feature.

Open Release Inquiry
The Open Release Inquiry was enhanced with a new F5-Access Tracking feature for Version 16. This new option allows the user to view the tracking information for a order by automatically connecting to the carrier provider's (UPS, Fed Ex, Airborne) website, via the internet. This feature enables the user to enter a order number and select the actual box to be tracked. Once selected, the user will then automatically be connected to the carrier's tracking information website page that pertains to that box. This feature is accessible through the F5-Review Release> F5-Hdr Scr> F5-Freight Detail> F5-Access Tracking. This is an essential feature for gathering tracking information for orders on a need to know now basis. To use this feature, it is required that SB Client as well as Internet Explorer are installed on your machine. Otherwise tracking information cannot be viewed. Additional Ship Via information is also required to accurately use this feature. The Ship Via Maintenance > F10-Action>Addl Optns> Via Overrides screen will need to be completed with the necessary Tracking URL Prefix and Suffix information. Currently the Fed Ex and UPS carriers are established to connect the user directly to the tracking information page for the box selected. However Airborne and any other User-defined carriers will connect the user to the general tracking information website page and the user will then be required to manually enter the tracking number for the box or boxes being researched.

Monthly Lost Sales By Vendor
To provide companies the ability to track lost sales by a Primary Vendor, Reason Code and Beginning and Ending Lost sales dates, a new Monthly Lost Sales By Vendor Report was added to the Order Processing module. This report will display information such as Customer #/Vendor #/Description, Order Number, Update Date, Product Number, Reason Code, Units, U/M, Ext. Price and Gross Profit that pertain to any lost sales that may have occurred for a vendor and time period. The lost sales information will be displayed on the report sorted first by Vendor, then by Reason Code.
Late Sales Order Reports
•
A new Late Sales Order Report was added to the Reports option of Version 16. This new report provides an overview of late orders including days late by warehouse for the Current MTD as well as the previous two months. To use this report, the Keep Late Orders? flag in File Update Parameters must be set to Y for the system to recognize that this information needs to be tracked.
•
A new Late Sales Order Detail Report was added to the Reports option of Version 16. This new report provides an overview of late orders including days late by warehouse for the Current MTD as well as the previous two months. This report allows the user to enter criteria such as Warehouse Number, Days Late and by Warehouse or Salesrep to generate a report for information. To use this report, the Keep Late Orders? flag in File Update Parameters must be set to Y for the system to recognize that this information needs to be tracked.
Pricing Audit Reports
•
A new pricing audit report Price Maintenance Audit Report By User Id was added to display any changes made to the Price Matrix Maintenance screen by a user Id and date range. Once this report has completed, the user will be prompted with the option to purge the information on this report. For this report to display the changes that have occurred in Price Matrix Maintenance, the Keep Price Matrix Maint Audit flag in the File Update Parameters must be set to Y, to create an audit file to track changes.
•
A new pricing audit report Price Maintenance Audit Report By Type was added to display any changes made to the Price Matrix Maintenance screen by a matrix type and date range. Once this report has completed, the user will be prompted with the option to purge the information on this report. For this report to display the changes that have occurred in Price Matrix Maintenance, the Keep Price Matrix Maint Audit flag in the File Update Parameters must be set to Y, to create an audit file to track change.
Miscellaneous Features
Sales Analysis
The Sales Source field default in the General Report Interface screens for all the Sales Analysis reports was changed. Previously, the Sales Source field would default to the WDI sales source code. This field no longer contains a default. This field will remain blank to allow the user to enter the desired sales source code for the report. In addition, service orders are now updated with the S sales source in the sales analysis files by the B Update process.
Qte Line Conversion in SOE
The Qte Line Conversion in SOE is a new flag that was added to allow the conversion of existing quotations to be processed in Sales Order Entry. If this flag is set to Y, the user will be prompted with a message that displays Customer Has Open Quotations For This Product! Do You Wish To Convert A Quote Line? If Yes is selected, the Quote Line Item Conversion screen will be displayed. For the Conv flag of the quote line item to be converted to a sales order, Y needs to be selected in the conversion screen. If this line was converted on a previous sales order, this message will be displayed, This Line Has Already Been Converted, Continue With Conversion?. If trying to convert a quote that is currently on Administrative Quote Hold, the quote item will be displayed on the screen, but will not be available to be converted. 022301A
Note: For this feature to be used, the Display Open/Quote/RGA Message must be set to Y and the Open Quote Message flag must be set to L or B.
Optio Pick Ticket & Packing List User-Forms
user-forms were modified to not print company information on Pick Tickets and Packing Lists based on the setting of the Blind Shipment Flag. If this flag is set to Y the user-forms will not print any company information such as logos name or addresses. This option allows you to send out shipments anonymously. If the flag is set to N, the information will be printed on both the pick tickets and packing lists.
Program Changes
A program change was made to the Sales Order completion screen. Previously, the user had the ability to change the tax jurisdiction code and the taxable flag in the completion screen. However if the user opted to change either one of these flags, the tax amount would not be recalculated. Therefore the modification made for this version now disallows any changes to be made to the tax jurisdiction or the taxable flag (with the exception of deleting them entirely) if a line item is associated with a sales tax matrix in Sales Tax Matrix Maintenance.

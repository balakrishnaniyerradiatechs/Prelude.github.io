Stock Receipts
Added logic to stock receipts to capture the estimated freight amount per serialized item. The system proportions the freight amongst the line items and adds it to the landed cost. If the item is a serialized product, the freight amount per serial number is captured and stored in the PROD.SN file. For example, stock receipts are done on a purchase order containing one line item for 2 pieces of a serialized product. The product’s cost is 10.00 and the estimated freight is 6.00. The system proportions the freight between the two products giving each a landed cost of 13.00 and a freight amount of 3.00 per serial number. Note: remember that a proportional freight amount is calculated per line item. See Accounts Payable Vendor Invoice Reconciliation for more information.

Core Bank/Issues Entry is a new option used to identify available core bank items and apply them to pending cores for a customer. You are prompted for the customer number and the return type. Return Type allows you to define if all pending cores for the customer are displayed or if only those pending cores with a remaining quantity are displayed. This routine displays pending cores records for the selected customer along with the quantity shipped, quantity applied, and quantity remaining for each record.
-
Pressing F5-Review on a record will drill down into the Core Order Info screen. This screen displays the pending cores for the selected order. For each core product number, the core type code (from product maintenance), the quantity shipped, the quantity applied, the quantity remaining, and the bank available quantity is displayed. Bank Avail is the number of cores available in the core bank for the current product. This screen gives you several options. You can press F6-Apply and apply a banked core to the pending core or reverse a previously applied core bank. You can press F7-Write Off and manually apply/write off the pending core. A match to a core bank record is not required with the write-off option.
-
The F6-Apply screen gives you the option to display all core bank records for the customer with a quantity available, all core bank records for the current customer/product that have a quantity available, or all core bank records that were fully applied. This screen will display the core bank product number, the adjustment number, core type, the quantity placed into the core bank, the quantity applied to a pending core record, and the quantity available. To apply a core, choose the banked core to apply and enter a quantity in the Quantity field. This process allows the quantity applied to be different than the quantity in the pending core record. The cost difference would be posted to the Core Cost Variance account. When this screen is exited and if a quantity is entered for one or more core bank records, the pending core record is updated with the quantity applied, the current date, and the associated transfer line item key for the associated core bank item. The core bank record is update with the associated pending core line item key. This provides a proper link between the pending core and the core bank records. You are then taken back to the initial Core Bank Issue Entry screen and the record shows applied or is no longer visible depending on the Ret Type flag. This screen also gives you the ability to reverse the application of a core bank to a pending core such as to correct an error. The system allows you to enter a negative quantity to reverse a previous application. This would reverse the general ledger postings and also establish the quantity back on the pending cores and core bank records.
-
F7-Write Off gives you the ability to write off a pending core record. A match to a core bank record is not required. The quantity entered is the quantity that will be written off. The update process will reduce the remaining quantity on the pending core record by the quantity entered and makes the appropriate postings to the general ledger. Note: the quantity entered cannot be greater than the quantity remaining for the pending core record.
See Order Processing Sales Order Entry Line Item Level, Inventory Processes Auto Core Application Process, Order Processing Sales Order History Inquiry, and General Ledger G/L Table Maintenance for more information.
•
Automatic Core Application Process is a new option that will automatically apply available core bank items to pending core records. The system selects all customers with available core bank items and then all pending core records for those customers. Only E type core bank items are used in this process. The system will match each core bank item with a pending core of the same core product and it will use the oldest core bank item first. The pending core record is updated with the quantity applied, the current date, and the adjustment line item key for the applied core bank item. The quantity available for the core bank record is reduced by the quantity applied and the core bank record is update with the pending core line item key it was applied to. This provides a proper link between the pending core and the core bank records. See Order Processing Sales Order Entry Line Item Level, Order Processing Sales Order History Inquiry, and General Ledger G/L Table Maintenance for more information.

The Special Operation field’s Length (L) option was changed to prompt for Length Pattern and Length Rounding. These should be set up if an XXXX generic length product number is NOT being used. Previously you had to have an XXXX generic length product number to set this flag to L but this locked you into the XXXX length pattern. Now, a regular base product may be used and the length pattern defined when this flag is set to L. This allows you to retain the same functionality as the generic length product number but gives you flexibility in the length pattern.

Length Pattern defines a length pattern for a product if the generic length pattern of XXXX is not desired. For example: product XYZ has a length pattern of 999.999. In Sales Order Entry, a length of 159 inches is entered. The 159 will be zero filled to the length pattern (159.000) and the result will be appended to the product XYZ (XYZ159.000). If this product already exists, processing will proceed as normal. If it does not already exist, then a new product, warehouse, bill of material, etc. records will be created for the new product XYZ159.000.
•
Length Rounding defines the increment to use for rounding the length. For example, if .50 is entered, the length will be rounded up to the next half foot. For example, a product has a length pattern of 999.99 and a rounding of .50. In sales order entry, 99 is entered as the length. The product would be XYZ099.000

F6-Xref/Notes - Substitutes
•
Product Substitution Type is a new field that defines how the substitute product is added to the order. The options for this field are for customers that allow substitutes. For those customers that do not allow substitutes, the system will treat the current product as if this flag were set to S or Null with the excepting of pricing which works as defined for the setting. See Order Processing Sales Order Line Item Level for more information.
-
A - when the product is added to an order and there is a backordered quantity, the system will automatically replace the product with the first substitute product with available inventory. Pricing is based on the substitute product.
-
P - when the product is added to an order and there is a backordered quantity, the system will automatically replace the product with the first substitute product with available inventory but the pricing is based on the original product.
-
B - when the product is added to an order, the system will automatically replace the product with the substitute product regardless of the availability of the original product. This type substitution is used for special promotions such as when the customer receives the requested product and some extra for free. For example, the vendor provides a pack of four instead of the usual pack of three. Once the bonus product runs out, the system will default back to the original product. Pricing is based on the substitute product.
-
S or Null – when the product is added to an order, no automatic substitution will take place. Instead, the user must manually choose to substitute a product by selecting F6-Subs at the line item detail level and choosing the substitute product from the available list. Pricing is based on the substitute product.

F5-Add Info - Multiple Fiscal Periods is a new field that determines if multiple fiscal periods are used when creating locations, chart of accounts, chart of account descriptions, and general ledger postings. Select Y to allow for multiple fiscal periods. Select N to not allow for multiple fiscal periods.
This flag provides a way to have multiple companies share inventory but post to different fiscal periods. This is done by setting up a live account for the corporation and then fiscal accounts for each fiscal year-end month or company. For example, ABC Supply has stores in Dallas, Houston, Tulsa, and San Antonio. These stores need the ability to share inventory but the fiscal year end for Dallas is March, Houston’s fiscal year end is June, and Tulsa and San Antonio’s fiscal year end is October. So a live account is set up for ABC, which allows all the stores to share inventory, and fiscal accounts are set up for MAR, JUN, and OCT for fiscal period postings. Tulsa and San Antonio will share the same fiscal account. When a posting routine is ran, the system will post on the live account but it will also, behind the scenes, post to the correct fiscal period on the fiscal account. The correct fiscal period for the fiscal account is found by using a cross-reference fiscal period table.
Note: all postings and account creation/maintenance should be performed in the live account.
Once the fiscal accounts and location records are set up, run /MULT.FISC.COPY from the live account to set up related information in each fiscal account. You are prompted for a Fiscal Account Name. This should be the name of the fiscal account you are setting up. This procedure sets up everything by mass copying existing related records over to each fiscal account. It copies the appropriate CA, CADF, and CA.STAT records from the live account to the specified fiscal account. It also copies all appropriate division/region/location type records.
See Location Maintenance, General Ledger Chart Of Accounts, General Ledger Chart Of Accounts Description, General Ledger Chart Of Accounts Duplication, and General Ledger G/L Balances Postings for more information.

Replenish Rounding Method is a new field that defines if a recommended replenish quantity is rounded down to a full case or a full pallet. If this flag is set to C or null, the quantity will be rounded down to a full case. If this flag is set to P, the quantity will be rounded down to a full pallet. For either setting, a quantity less than a single case/pallet will default to a quantity of 1 case/pallet. The case and pallet quantities are retrieved from the product master file. If there is not a case quantity or a pallet quantity, the system will default to a case/pallet quantity of one. The rounding method is only used for replenishing the primary bin location. This calculation is not used in the primary full case bin evaluation.

Replenish Bin Sorting Method is a new field that defines how the list for eligible bins will be sorted for the replenishment process. If this flag is set to O, the sort will be by ascending bin on-hand as it was previously. This method helps clean up the supplying bins with smaller bin on-hand quantities thus releasing bin space for other products. If this flag is set to D, the sort will be by ascending bin transaction date. This method will attempt a FIFO rotation of inventory. This is not a 100% pure FIFO rotation of stock.
Lot control should be used if you have a concern with product age or expiration dates. Lot controlled products are excluded from this option and are allocated by oldest lot creation date.

Check For Existing Orders is a new option parameter that determines if the user is prompted in Sales Order Entry to add items to an existing order instead of creating a new order. This allows the user to add to an existing order for a customer until a pick ticket is printed.
If this flag is set to Y, upon exiting the sales order header screen for a new order, the system checks to see if previous orders exists for the customer/customer ship-to combination. If any orders are found matching the customer/customer ship-to combination, the system will then check that a pick ticket has not yet printed for the sales order, that the order is a type 01, and that the customer does not require a purchase order number. If an order is found meeting all these conditions, the user is prompted Do You Want To Add To Existing Sales Order? If the user responds Yes, the first sales order that is not in the pick ticket process will be displayed allowing the user to enter the items on the existing order. If the user responds No, the user will continue to enter the items on the newly created sales order.
If this flag is null or set to N, the user will not be prompted to add to an existing sales order.
This option parameter has a new option of P that works similar to the Y option except that the system will only evaluate orders with a pickable quantity. If no order is found with a pickable quantity, the user will be returned to their original sales order.

Printer Set-up
Form Queue Maintenance is a new routine used to set up form queues and the warehouses that may print to those form queues. At this time, this process is only used in Release Pick Tickets to validate that the warehouse may print to a certain form queue. It is not used to validate the form queue itself.
A form queue may be assigned to a warehouse, multiple warehouses, or to all warehouses. Leave the Warehouse# List field null to print to all warehouses. If a form queue is entered In Release Pick Tickets, the system will first verify that the form queue entered is established here. If so, it will then verify that the form queue is valid for the warehouse the document is being printed in. If the form queue is not established here, the system will print to that form queue without validating the warehouse. For example using the screen above, you may only print documents for warehouse 20 in form queues 24 and 56 or in a form queue not defined in this maintenance routine. See Inventory Processes for more information.

Create Index Procedure
This option was modified to allow you to rebuild indexes for selected files. Previously, your only choice was to rebuild all files. An upfront screen of standard ADS indexes was added so you can define the indexes to rebuild. When this option is selected, the default for each file is N. You will have to select each file that needs to be rebuilt. If you want to rebuild each file then you need to set each one to Y.
